# 부록 A. AI 질문 역량(협업적 추상화) 평가 루브릭

## A-1. 루브릭 개요

| 항목 | 내용 |
|------|------|
| 평가 대상 | AI와의 대화 로그 및 최종 산출물 |
| 평가 영역 | 5개 영역 |
| 점수 범위 | 각 영역 1-4점, 총점 5-20점 |
| 평가 방식 | 2인 독립 채점 후 협의 |

---

## A-2. 평가 영역별 상세 루브릭

### 영역 1: 초기 의도 표현 (Initial Intent Expression)

| 수준 | 점수 | 기준 | 예시 |
|------|------|------|------|
| 미흡 | 1 | 의도 파악이 어려운 모호한 프롬프트 | "프로그램 만들어줘" |
| 보통 | 2 | 목표만 명시하고 맥락이나 제약조건 부족 | "구구단 프로그램 만들어줘" |
| 양호 | 3 | 목표와 맥락을 명시하고 일부 제약조건 포함 | "초등학생용 구구단 연습 프로그램 만들어줘" |
| 우수 | 4 | 목표, 맥락, 제약조건, 예상 결과 형식을 모두 명시 | "초등학교 3학년 학생이 2-9단을 연습할 수 있는 프로그램을 만들어줘. 랜덤으로 문제가 나오고, 정답을 입력하면 맞았는지 확인해주고, 10문제 후 점수를 알려줘" |

**채점 지침:**
- 목표(Goal): 무엇을 만들려고 하는가?
- 맥락(Context): 누가, 어디서, 왜 사용하는가?
- 제약조건(Constraints): 범위, 형식, 기능적 요구사항
- 예상 결과(Expected Output): 결과물의 형태나 특성

---

### 영역 2: 대화적 구체화 (Dialogic Specification)

| 수준 | 점수 | 기준 | 예시 |
|------|------|------|------|
| 미흡 | 1 | AI 응답 후 후속 질문이나 수정 요청 없음 | (첫 응답을 그대로 수용) |
| 보통 | 2 | 단순 반복이나 막연한 수정 요청 | "다시 해줘", "더 좋게 만들어줘" |
| 양호 | 3 | 구체적인 부분을 지적하며 적절한 수정 요청 | "오답일 때 정답을 보여주는 기능을 추가해줘" |
| 우수 | 4 | 전략적 수정 요청, 대안 탐색, 설명 요청 포함 | "이 방식 말고 게임처럼 점수가 쌓이는 방식은 어떨까? 그리고 왜 while 대신 for를 사용했어?" |

**채점 지침:**
- 후속 질문의 구체성
- 수정 요청의 명확성
- AI 응답에 대한 이해도
- 대안 탐색 시도 여부

---

### 영역 3: 반복적 정련 (Iterative Refinement)

| 수준 | 점수 | 기준 | 예시 |
|------|------|------|------|
| 미흡 | 1 | 1회 시도 후 종료 또는 단순 반복 | 첫 응답 수용 또는 같은 질문 반복 |
| 보통 | 2 | 2-3회 시도, 양적 변화만 있음 | 기능 추가 요청만 반복 |
| 양호 | 3 | 4-5회 시도, 질적 변화 동반 | 구조 개선, 오류 수정, 기능 통합 |
| 우수 | 4 | 5회 이상 시도, 체계적 개선 과정 | 단계별 개선, 테스트 후 수정, 최적화 |

**채점 지침:**
- 대화 턴(turn) 수
- 각 턴에서의 변화 정도
- 개선의 방향성
- 최종 결과물의 완성도 향상 정도

---

### 영역 4: 메타인지적 모니터링 (Metacognitive Monitoring)

| 수준 | 점수 | 기준 | 예시 |
|------|------|------|------|
| 미흡 | 1 | 자신의 의도와 AI 결과 간 불일치 인식 없음 | 오류나 불일치를 인식하지 못함 |
| 보통 | 2 | 불일치 인식하나 원인 분석 부족 | "이게 내가 원한 게 아닌데..." |
| 양호 | 3 | 불일치의 원인을 분석하고 표현 | "내가 처음에 범위를 안 알려줘서 그런 것 같아" |
| 우수 | 4 | 원인 분석 후 질문 전략을 명시적으로 수정 | "아, 내가 너무 추상적으로 물어봤구나. 구체적으로 다시 물어볼게. 2단부터 9단까지..." |

**채점 지침:**
- 자기 질문(self-questioning) 증거
- 불일치 인식 여부
- 원인 귀인의 적절성
- 전략 수정의 명시성

---

### 영역 5: 최종 통합 (Final Integration)

| 수준 | 점수 | 기준 | 예시 |
|------|------|------|------|
| 미흡 | 1 | 미완성 또는 원래 의도와 무관한 결과물 | 실행 불가, 의도와 다른 결과 |
| 보통 | 2 | 부분적으로 완성, 핵심 기능 일부 누락 | 기본 기능은 있으나 중요 요소 빠짐 |
| 양호 | 3 | 대부분 완성, 사소한 개선 여지 | 핵심 기능 모두 포함, 세부 조정 필요 |
| 우수 | 4 | 원래 의도와 완전히 일치, 기대 이상 | 모든 요구사항 충족, 추가 개선까지 |

**채점 지침:**
- 초기 의도와의 일치도
- 기능적 완성도
- 실행 가능성
- 품질 수준

# 부록 B. AI 협업 과제 지시문

## B-1. 과제 안내문

---

### AI 협업 프로그래밍 과제

**과제명:** 초등학교 곱셈 학습 프로그램 제작

**소요 시간:** 30-40분

**사용 도구:** Claude (claude.ai)

---

#### 과제 설명

여러분은 초등학교 3학년 담임교사입니다. 학생들이 곱셈을 재미있게 연습할 수 있는 교육용 프로그램을 AI의 도움을 받아 만들어 보세요.

#### 기본 요구사항

프로그램에는 다음 기능이 포함되어야 합니다:

1. 곱셈 문제를 제시하는 기능
2. 학생이 답을 입력하는 기능
3. 정답 여부를 확인하는 기능
4. 점수나 결과를 보여주는 기능

#### 추가 고려사항

다음 사항들을 자유롭게 고려하여 프로그램을 발전시켜 보세요:

- 대상 학생의 수준 (몇 단까지?)
- 문제 수와 난이도
- 오답 시 피드백 방식
- 학습 동기 유발 요소
- 사용자 친화적 인터페이스

#### 수행 방법

1. Claude(claude.ai)에 접속합니다.
2. AI와 대화하며 프로그램을 만들어 나갑니다.
3. 원하는 결과가 나올 때까지 자유롭게 대화를 이어갑니다.
4. 최종 결과물(코드)을 저장합니다.
5. 대화 로그는 자동으로 저장됩니다.

#### 유의사항

- 정해진 "정답"이 없습니다. 자유롭게 창의적으로 접근하세요.
- AI와 여러 번 대화해도 됩니다.
- 코드를 직접 수정해도 됩니다.
- 질문이 있으면 언제든 연구자에게 문의하세요.

#### 평가 안내

이 과제는 **프로그램의 완성도**가 아니라 **AI와 협업하는 과정**을 평가합니다:
- 어떻게 질문했는가?
- AI 응답을 어떻게 활용했는가?
- 원하는 결과를 위해 어떤 노력을 했는가?

# 부록 E. 데이터 분석 코드 (Python)

## E-1. 필요 라이브러리

```python
import numpy as np
import pandas as pd
from scipy import stats
from scipy.stats import pearsonr, shapiro
import statsmodels.api as sm
from statsmodels.stats.outliers_influence import variance_inflation_factor
```

## E-2. 데이터 로드 및 확인

```python
# 데이터 로드
df = pd.read_csv('research_data.csv', encoding='utf-8-sig')

# 기본 정보 확인
print(df.info())
print(df.describe())
```

## E-3. 기술통계 분석

```python
# 분석 변인 선택
analysis_vars = ['메타인지', '인지욕구', '모호함인내', 'AI경험', 'AI질문역량']

# 기술통계
desc_stats = df[analysis_vars].describe().T
desc_stats['왜도'] = df[analysis_vars].skew()
desc_stats['첨도'] = df[analysis_vars].kurtosis()

print(desc_stats)

# 정규성 검정
for var in analysis_vars:
    stat, p = shapiro(df[var])
    print(f"{var}: W = {stat:.3f}, p = {p:.3f}")
```

## E-4. 상관분석

```python
# 상관행렬
corr_matrix = df[analysis_vars].corr()

# 유의확률 계산
p_matrix = pd.DataFrame(np.zeros((len(analysis_vars), len(analysis_vars))), 
                         columns=analysis_vars, index=analysis_vars)

for i, var1 in enumerate(analysis_vars):
    for j, var2 in enumerate(analysis_vars):
        if i != j:
            r, p = pearsonr(df[var1], df[var2])
            p_matrix.loc[var1, var2] = p

print("상관행렬:")
print(corr_matrix.round(3))
print("\n유의확률:")
print(p_matrix.round(3))
```

## E-5. 위계적 회귀분석

```python
# 종속변인
y = df['AI질문역량']

# 1단계: 통제변인 (AI 사용 경험)
X1 = sm.add_constant(df[['AI경험']])
model1 = sm.OLS(y, X1).fit()
print("1단계 결과:")
print(model1.summary())

# 2단계: 인지적 성향 변인 추가
X2 = sm.add_constant(df[['AI경험', '메타인지', '인지욕구', '모호함인내']])
model2 = sm.OLS(y, X2).fit()
print("\n2단계 결과:")
print(model2.summary())

# R² 변화량 검정
r2_change = model2.rsquared - model1.rsquared
df1 = model2.df_model - model1.df_model
df2 = model2.df_resid
f_change = (r2_change / df1) / ((1 - model2.rsquared) / df2)
p_change = 1 - stats.f.cdf(f_change, df1, df2)

print(f"\nΔR² = {r2_change:.3f}")
print(f"ΔF({df1}, {df2}) = {f_change:.3f}, p = {p_change:.4f}")
```

## E-6. 다중공선성 진단

```python
# VIF 계산
X_vif = df[['AI경험', '메타인지', '인지욕구', '모호함인내']]
vif_data = pd.DataFrame()
vif_data['변인'] = X_vif.columns
vif_data['VIF'] = [variance_inflation_factor(X_vif.values, i) 
                   for i in range(X_vif.shape[1])]
vif_data['Tolerance'] = 1 / vif_data['VIF']

print("다중공선성 진단:")
```

## E-7. 표준화 회귀계수 계산

```python
# 표준화 회귀계수 계산
def standardized_beta(model, X, y):
    betas = {}
    for var in X.columns:
        if var != 'const':
            beta = model.params[var] * df[var].std() / y.std()
            betas[var] = beta
    return betas

betas = standardized_beta(model2, X2, y)
print("\n표준화 회귀계수:")
for var, beta in sorted(betas.items(), key=lambda x: abs(x[1]), reverse=True):
    print(f"  {var}: β = {beta:.3f}")
```

| 용어 | 영문 | 설명 |
