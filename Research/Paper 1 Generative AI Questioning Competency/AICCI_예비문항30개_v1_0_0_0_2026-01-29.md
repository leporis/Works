## 2. AICCI 예비 문항 (전문가 검토용)

### 지시문

> 다음은 AI와 협력하여 코딩하는 것에 대한 여러분의 생각과 느낌을 묻는 문항들입니다. 각 문항을 읽고, 현재 자신의 생각과 가장 일치하는 번호에 표시해 주세요.
>
> 정답은 없습니다. 솔직하게 응답해 주세요.
>
> 1 = 전혀 그렇지 않다 / 2 = 그렇지 않다 / 3 = 보통이다 / 4 = 그렇다 / 5 = 매우 그렇다

---

### 요인 1: AI 출력물 비판적 검증 인식 (CVI)

**구인 정의:** AI가 생성한 결과물을 무조건 수용하지 않고, 오류 가능성을 인식하여 비판적으로 검토해야 한다는 인식

| 번호 | 문항 | 부정 |
|:----:|------|:----:|
| CVI-1 | AI가 생성한 코드는 반드시 직접 실행해보고 결과를 확인해야 한다. | |
| CVI-2 | AI는 자신 있게 답변하더라도 틀린 정보를 제공할 수 있다. | |
| CVI-3 | AI가 만든 코드라도 오류가 없는지 내가 직접 검토해야 한다. | |
| CVI-4 | AI의 답변이 논리적으로 맞는지 스스로 판단하는 것이 중요하다. | |
| CVI-5 | AI가 생성한 결과물을 그대로 사용하면 예상치 못한 문제가 생길 수 있다. | |
| CVI-6 | AI가 제시한 코드가 틀릴 수도 있다는 것을 항상 염두에 두어야 한다. | |
| CVI-7 | AI가 생성한 코드는 충분히 검증되었으므로 그대로 사용해도 된다. | *(R)* |
| CVI-8 | AI 결과물을 검증하는 것은 시간 낭비이다. | *(R)* |

---

### 요인 2: AI 협력 효능감 (ACE)

**구인 정의:** AI와 협력하여 원하는 결과물을 만들어낼 수 있다는 자기 효능감

| 번호 | 문항 | 부정 |
|:----:|------|:----:|
| ACE-1 | 나는 AI와 협력하여 원하는 기능을 가진 프로그램을 만들 수 있다. | |
| ACE-2 | AI와 여러 번 대화하며 결과를 개선해 나갈 수 있다. | |
| ACE-3 | AI가 처음에 원하는 결과를 주지 않더라도, 대화를 통해 원하는 결과를 얻을 수 있다. | |
| ACE-4 | 나는 AI와 협력하면 혼자서는 만들기 어려운 프로그램도 만들 수 있다고 생각한다. | |
| ACE-5 | 나는 AI와 협력하여 코딩 문제를 해결하는 것이 자신 있다. | |
| ACE-6 | AI를 활용하면 내가 구상한 프로그램을 실제로 구현하는 것이 가능하다고 생각한다. | |
| ACE-7 | AI와 협력하여 코드를 작성하는 것은 나에게 너무 어려운 일이다. | *(R)* |

---

### 요인 3: 프롬프트 설계 자신감 (PDC)

**구인 정의:** AI에게 의도를 명확히 전달하고 효과적인 프롬프트를 작성할 수 있다는 자신감

| 번호 | 문항 | 부정 |
|:----:|------|:----:|
| PDC-1 | 나는 AI에게 원하는 결과를 얻기 위해 구체적이고 명확하게 요청할 수 있다. | |
| PDC-2 | 나는 AI가 내 의도를 잘 이해할 수 있도록 맥락 정보를 포함하여 요청할 수 있다. | |
| PDC-3 | 나는 AI에게 단계별로 구체적인 조건을 설명하며 원하는 코드를 요청할 수 있다. | |
| PDC-4 | AI에게 요청할 때 어떻게 표현해야 좋은 결과를 얻을 수 있는지 알고 있다. | |
| PDC-5 | 나는 AI에게 원하는 결과를 얻지 못했을 때 요청 방식을 수정하여 다시 시도할 수 있다. | |
| PDC-6 | 나는 AI에게 프로그래밍 요구사항을 체계적으로 전달할 자신이 있다. | |
| PDC-7 | 나는 AI에게 무엇을 어떻게 요청해야 할지 잘 모르겠다. | *(R)* |

---

### 요인 4: AI 역할 인식 (ARP)

**구인 정의:** AI를 단순한 '정답 제공자'가 아닌 인간과 역할을 분담하는 '협력적 파트너'로 인식하는 정도

| 번호 | 문항 | 부정 |
|:----:|------|:----:|
| ARP-1 | AI는 내가 지시하고 검토하는 협력적 파트너이다. | |
| ARP-2 | AI와의 협력에서 최종 판단과 책임은 나에게 있다. | |
| ARP-3 | AI는 코딩의 모든 것을 대신해주는 것이 아니라, 내가 구상한 것을 구현하는 데 도움을 주는 도구이다. | |
| ARP-4 | AI와 협력하여 코딩할 때, 내 역할은 AI에게 방향을 제시하고 결과를 검토하는 것이다. | |
| ARP-5 | AI는 정확하고 완벽한 답을 주는 도구가 아니라, 함께 문제를 해결해가는 협력자이다. | |
| ARP-6 | AI를 활용한 코딩에서도 인간의 창의적 사고와 비판적 판단이 여전히 중요하다. | |
| ARP-7 | AI가 코드를 생성해 주면 나는 그것을 그대로 활용하면 된다. | *(R)* |
| ARP-8 | AI가 있으면 프로그래밍에서 내가 생각하거나 판단할 필요가 없다. | *(R)* |

## 4. 전문가 내용 타당도 검토 양식 (CVR)

### 4.1 CVR 검토 양식

> **검토 의뢰:** 다음 각 문항이 해당 구인을 측정하는 데 적합한지 평가해 주십시오.

**평가 기준:**
- **필수적:** 이 문항은 해당 구인을 측정하는 데 반드시 필요하다
- **유용하나 필수적이지 않음:** 이 문항은 도움이 되지만 필수적이지는 않다
- **불필요:** 이 문항은 해당 구인 측정에 적합하지 않다

**CVR 계산:** CVR = (ne - N/2) / (N/2), ne = "필수적" 선택 전문가 수, N = 전체 전문가 수

**기준:** CVR ≥ .99 (5인 패널 기준: 5명 모두 "필수적" 선택 시 CVR = 1.00)

## 7. 이론적 근거 (문항별)

### CVI 문항 이론적 근거

| 문항 | 이론적 근거 |
|------|------------|
| CVI-1~6 | AI 환각(Hallucination) 현상에 대한 인식 (Bender et al., 2021); '증강 패러독스(Augmentation Paradox)' — AI 활용이 오히려 인간의 비판적 검증 필요성을 증가시킴 (Parasuraman & Riley, 1997) |
| CVI-7, 8 | 자동화 편향(Automation Bias) — AI 출력물을 맹목적으로 신뢰하는 경향 (Skitka et al., 1999); 역채점으로 자동화 편향 측정 |

### ACE 문항 이론적 근거

| 문항 | 이론적 근거 |
|------|------------|
| ACE-1~6 | Bandura(1977)의 자기효능감 이론 — 특정 과제를 수행할 수 있다는 능력 신념; AI 협력 맥락으로 확장 |
| ACE-7 | 역채점 문항 — 자기효능감 낮음을 직접 측정 |

### PDC 문항 이론적 근거

| 문항 | 이론적 근거 |
|------|------------|
| PDC-1~6 | 프롬프트 리터러시(Prompt Literacy) — AI와의 효과적 의사소통 능력 (Brachman & Levesque, 2004; Liu et al., 2022); CLEAR 프레임워크 (Concise, Logical, Explicit, Adaptive, Reflective) |
| PDC-7 | 역채점 문항 — 프롬프트 무력감 직접 측정 |

### ARP 문항 이론적 근거

| 문항 | 이론적 근거 |
|------|------------|
| ARP-1~6 | 인간-AI 협력 모델(Human-AI Collaboration) — 인간의 주도성과 AI의 보조적 역할 (Shneiderman, 2022); 분산 인지 이론 (Hutchins, 1995) |
| ARP-7, 8 | 역채점 문항 — AI를 정답 제공자로만 인식하는 관점 측정; 자동화 의존성(Over-reliance) 개념 |
