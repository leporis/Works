## 2. 1주차 평가: 이론 퀴즈 (10점)

### 2.1 기본 정보

| 항목 | 내용 |
|:---|:---|
| 실시 시기 | 2주차 1차시 시작 |
| 실시 방법 | **LMS 퀴즈 코너** 온라인 응시 |
| 제한 시간 | 10분 |
| 출제 범위 | 1주차 전체 내용 (AI의 이해 + 기계학습의 원리) |
| 문항 구성 | 5지선다형 10문항 + 단답형 5문항 (총 15문항) |
| 배점 방식 | 5지선다형 각 0.5점 (5점) + 단답형 각 1점 (5점) = 10점 |

| 영역 | 예상 문항 수 | 주요 출제 내용 |
|:---|:---:|:---|
| AI 정의와 유형 | 3문항 | AI 정의, 약한/강한 AI 구분, 기능별 분류(인식/예측/분류/생성) |
| AI 발전 역사 | 2문항 | AI의 봄과 겨울, 주요 사건(튜링 테스트, AlphaGo 등) |
| 기계학습 개념 | 2문항 | ML 정의, AI/ML/DL 포함 관계 |
| 학습 유형 | 3문항 | 지도학습, 비지도학습, 강화학습의 정의와 사례 구분 |

**단답형 (5문항 × 1점 = 5점)**

| 영역 | 예상 문항 수 | 주요 출제 내용 |
|:---|:---:|:---|
| AI 유형·분류 | 2문항 | 약한/강한 AI 구분 이유 서술, 기능별 분류 사례 기술 |
| 기계학습 원리 | 2문항 | 학습 과정(수집→학습→평가→예측) 단계 설명, 학습 유형 비교 |
| 통합 이해 | 1문항 | AI/ML/DL 포함 관계 설명 또는 사례 적용 |

## 4. 3주차 평가: 엔트리 AI 블록 활용 작품 (10점)

### 4.1 기본 정보

| 항목 | 내용 |
|:---|:---|
| 제작 시간 | 3주차 2차시 종료 전 20분 |
| 과제 내용 | 엔트리 인공지능 블록 1종 이상을 활용한 AI 작품 |
| 제출물 | 엔트리 작품 URL |
| 필수 조건 | AI 블록(음성인식/번역/읽어주기/사물인식/사람인식 중 1종+) + 블록 코딩 연동 |

### 4.2 평가 영역 및 배점

| 평가 영역 | 배점 | 세부 항목 |
|:---|:---:|:---|
| A. AI 블록 활용 | 4점 | 블록 선택·적용(2) + 기능 작동(2) |
| B. 블록 코딩 구현 | 4점 | AI 결과 연동(2) + 동작 구현(2) |
| C. 제출 요건 | 2점 | 기한 준수(1) + 작품 실행 가능(1) |

### 4.3 항목별 평가 기준

#### A. AI 블록 활용 (4점)

**A-1. AI 블록 선택·적용 (2점)**

| 점수 | 기준 |
|:---:|:---|
| **2점** | 인공지능 블록을 2종 이상 조합하여 활용하거나, 1종을 창의적으로 심화 활용 |
| **1점** | 인공지능 블록을 1종 활용했으나 시연과 유사한 수준 |
| **0점** | 인공지능 블록을 활용하지 못함 |

**A-2. 기능 작동 (2점)**

| 점수 | 기준 |
|:---:|:---|
| **2점** | AI 블록 기능이 안정적으로 작동하고, 결과가 작품 흐름에 자연스럽게 통합됨 |
| **1점** | AI 블록 기능이 작동하나 불안정하거나, 작품과의 연계가 부자연스러움 |
| **0점** | AI 블록 기능이 작동하지 않음 |

#### B. 블록 코딩 구현 (4점)

**B-1. AI 결과 연동 (2점)**

| 점수 | 기준 |
|:---:|:---|
| **2점** | AI 블록의 출력(인식 결과, 번역 결과 등)을 블록 코딩에서 조건 분기나 변수로 활용 |
| **1점** | AI 블록 출력을 단순 표시만 하거나, 연동이 형식적 |
| **0점** | AI 블록 출력을 블록 코딩에 연동하지 못함 |

**B-2. 동작 구현 (2점)**

| 점수 | 기준 |
|:---:|:---|
| **2점** | AI 인식 결과에 따라 2가지 이상의 서로 다른 동작이 정상 작동 |
| **1점** | AI 인식 결과에 따른 동작이 1가지이거나, 일부 동작이 오류 |
| **0점** | AI 결과와 무관한 동작이거나 작동하지 않음 |

#### C. 제출 요건 (2점)

**C-1. 기한 준수 (1점)**

| 점수 | 기준 |
|:---:|:---|
| **1점** | 수업 종료 시간까지 제출 완료 |
| **0점** | 수업 종료 시간 초과 |

**C-2. 작품 실행 가능 (1점)**

| 점수 | 기준 |
|:---:|:---|
| **1점** | 제출된 URL로 접근 가능하고, 작품이 정상 실행됨 |
| **0점** | URL 접근 불가 또는 작품 실행 오류 |

| 제출물 | 엔트리 작품 URL + 생성형 AI 활용 프롬프트 전문 |
| 필수 조건 | AI 모델 학습 활용(이미지 분류 또는 테이블 학습) + 생성형 AI 활용 과정 기록 |

### 5.2 평가 영역 및 배점

| 평가 영역 | 배점 | 세부 항목 |

### 5.3 항목별 평가 기준

#### A. 작품 완성도 (4점)

**A-1. AI 모델 활용 (2점)**

| 점수 | 기준 |
|:---:|:---|
| **2점** | 이미지 분류 모델 또는 테이블 학습(선형회귀/KNN/K평균)을 활용하여 작품에 통합, 학습 데이터가 충분하고 결과가 안정적 |
| **1점** | AI 모델 학습을 시도했으나 학습 데이터가 부족하거나 모델 정확도가 낮음 |
| **0점** | AI 모델 학습이 완료되지 않았거나 작품에 활용되지 않음 |

**A-2. 동작 구현 (2점)**

| 점수 | 기준 |
|:---:|:---|
| **2점** | 모델 학습 결과(분류, 예측 등)에 따른 조건 분기 동작이 2개 이상 정상 작동 |
| **1점** | 기본적 동작은 하나 일부 오류 또는 단순 동작만 구현 |

| **1점** | 요청은 있으나 조건이 모호하거나 구조화 부족 (1~2개 조건) |
| **0점** | 프롬프트가 없거나 "코드 짜줘" 수준의 단순 요청 |

**B-2. 활용 적절성 (2점)**

| **2점** | 생성형 AI를 아이디어 구상, 코드 설계, 디버깅 등 2가지 이상 용도로 활용 |
| **1점** | 생성형 AI를 1가지 용도로만 활용 |
| **0점** | 생성형 AI를 실질적으로 활용하지 않음 (프롬프트 형식적 기록만) |

### 6.1 기본 정보

| 항목 | 내용 |
|:---|:---|
| 제작 시간 | 5주차 2차시 종료 전 30분 |
| 과제 내용 | 생성형 AI의 도움을 받아 AI 활용 초등학교 수업 지도안 작성 |
| 분량 | A4 1~2쪽 |
| 제출물 | 수업 지도안 + 생성형 AI 활용 프롬프트 전문 |
| 필수 포함 요소 | 학년/주제/성취기준/학습목표/도구활용계획/학생활동/평가계획 |

### 6.2 평가 영역 및 배점

| 평가 영역 | 배점 | 세부 항목 |
|:---|:---:|:---|

### 6.3 항목별 평가 기준

#### A. 지도안 내용 (4점)

**A-1. 성취기준 연계 (2점)**

| 점수 | 기준 |
|:---:|:---|
| **2점** | 2022 개정 교육과정 성취기준을 명시하고, 학습목표가 성취기준과 명확히 연결됨 |
| **1점** | 성취기준을 명시했으나 학습목표와의 연결이 불명확하거나 형식적 |
| **0점** | 성취기준 미명시 또는 학습목표 없음 |

**A-2. 활동 설계 (2점)**

| 점수 | 기준 |
|:---:|:---|
| **2점** | 초등학생 수준에 적합한 구체적 학생 활동 포함 (도입-전개-정리 구조, 시간 배분 명시) |
| **1점** | 활동은 있으나 구체성 부족 (시간 배분 없음, 활동 내용 모호) |
| **0점** | 학생 활동이 없거나 초등학생 수준에 부적합 |

#### B. 도구 활용 (4점)

**B-1. 도구 연계 계획 (2점)**

| 점수 | 기준 |
|:---:|:---|
| **2점** | 엔트리 AI 또는 생성형 AI를 수업에 활용하는 구체적 계획 포함 (어떤 도구를, 어떤 활동에서, 어떻게 사용할지 명시) |
| **1점** | 도구 활용 계획은 있으나 구체성 부족 ("AI를 활용한다" 수준) |
| **0점** | 도구 활용 계획 없음 |

**B-2. 생성형 AI 활용 (2점)**

| 점수 | 기준 |

## 8. 부정행위 처리 기준

### 8.1 적용 범위

본 기준은 이론 퀴즈(1~2주차), 엔트리 AI 작품(3~4주차), 수업 지도안(5주차) 등 모든 평가에 공통 적용된다.

### 8.2 부정행위 의심 시 처리 절차

| 단계 | 주체 | 내용 |
|:---:|:---:|:---|
| 1단계 | 교수 | 부정행위가 의심되는 경우, 교수는 해당 학생에게 확인을 요청할 수 있다. |
| 2단계 | 학생 | 학생은 교수의 확인 요청에 대하여 의무적으로 해명하여야 한다. |
| 3단계 | 교수 | 교수가 납득할 만한 해명이 이루어지지 않거나 학생이 해명을 거부하는 경우, 해당 퀴즈/과제의 평가 점수는 **0점** 처리한다. |

### 8.3 부정행위 유형 예시

- 이론 퀴즈: 타인의 답안 참조, 허용되지 않은 자료 사용
- 엔트리 AI 작품: 타인의 작품을 그대로 제출, 본인이 제작하지 않은 작품 제출
- 수업 지도안: 타인의 지도안을 복사하여 제출, 생성형 AI 활용 과정을 허위로 기록

> **유의사항**: 생성형 AI를 활용하는 과제(4~5주차)의 경우, 생성형 AI 사용 자체는 부정행위가 아니다. 단, 프롬프트 전문을 성실하게 기록·제출하지 않거나, 타인의 프롬프트와 결과물을 그대로 복사하여 제출하는 행위는 부정행위에 해당한다.

### 8.4 교수의 의무

교수는 평가 과정과 결과에 대한 학생의 문의에 성실하게 답변하여야 하며, 학생이 납득할 때까지 충분히 설명할 의무가 있다.
