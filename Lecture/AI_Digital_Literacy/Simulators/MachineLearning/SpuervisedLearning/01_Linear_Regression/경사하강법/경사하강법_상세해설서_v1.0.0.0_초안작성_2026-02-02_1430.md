## 1. 경사하강법이란?

### 1.1 정의

**경사하강법(Gradient Descent)**은 함수의 최솟값을 찾기 위한 **반복적 최적화 알고리즘**입니다. 현재 위치에서 함수의 기울기(경사)를 계산하고, 기울기의 **반대 방향**으로 조금씩 이동하여 최솟값에 도달합니다.

### 1.2 왜 경사하강법이 필요한가?

머신러닝에서 모델을 학습한다는 것은 **손실 함수를 최소화하는 파라미터를 찾는 것**입니다.

| 문제 | 설명 |
|------|------|
| **목표** | 예측값과 실제값의 차이(오차)를 최소화 |
| **방법** | 오차를 측정하는 손실 함수 정의 |
| **해결** | 손실 함수를 최소화하는 가중치 찾기 |

단순한 경우에는 수학적으로 **정확한 해(닫힌 형태 해, Closed-form Solution)**를 구할 수 있지만, 다음과 같은 경우에는 경사하강법이 필요합니다:

- 데이터가 매우 클 때 (수백만 개 이상)
- 비선형 모델 (신경망 등)
- 닫힌 형태 해가 존재하지 않을 때

### 1.3 핵심 아이디어

```
현재 위치에서 가장 가파른 내리막 방향으로 한 걸음씩 이동하면
결국 가장 낮은 곳(최솟값)에 도달한다!
```

## 2. 직관적 이해: 산에서 내려오기

### 2.1 비유: 안개 낀 산에서 내려오기

눈을 감고 안개 낀 산꼭대기에 서 있다고 상상해보세요. 가장 낮은 곳(골짜기)으로 내려가고 싶지만, 전체 지형을 볼 수 없습니다.

**해결 전략:**
1. 현재 발밑의 **경사(기울기)**를 느낀다
2. **가장 가파르게 내려가는 방향**을 찾는다
3. 그 방향으로 **한 걸음** 이동한다
4. 평평한 곳(골짜기)에 도달할 때까지 **반복**한다

### 2.2 경사하강법 vs 산 내려오기

| 산 내려오기 | 경사하강법 |
|------------|-----------|
| 현재 위치 | 현재 파라미터 값 (w, b) |
| 고도 | 손실 함수 값 (MSE) |
| 경사의 가파른 정도 | 기울기 (Gradient) |
| 한 걸음 크기 | 학습률 (Learning Rate, η) |
| 평평한 골짜기 | 손실 최솟값 |

### 2.3 2D 시각화

손실 함수가 하나의 파라미터 w에 대해 포물선 형태라고 가정하면:

```
손실
  │
  │    ●  시작점
  │     ╲
  │      ╲  ← 기울기 (음수 방향으로 이동)
  │       ╲
  │        ●
  │         ╲
  │          ╲
  │           ●  최솟값 도달!
  │────────────────────────── w
```

**핵심 포인트:**
- 기울기가 **양수**면 w를 **감소**시킴
- 기울기가 **음수**면 w를 **증가**시킴
- 기울기가 **0**이면 최솟값 도달!

## 3. 수학적 기초

### 3.1 손실 함수 (Loss Function)

선형 회귀에서 가장 널리 사용되는 손실 함수는 **평균 제곱 오차(MSE, Mean Squared Error)**입니다.

**MSE 공식:**

```
        1   n
MSE = ─── × Σ (yᵢ - ŷᵢ)²
        n  i=1
```

여기서:
- n: 데이터 개수
- yᵢ: i번째 실제값
- ŷᵢ: i번째 예측값 (= w × xᵢ + b)

**왜 제곱을 사용하는가?**

| 이유 | 설명 |
|------|------|
| 양수 보장 | 음수 오차와 양수 오차가 상쇄되지 않음 |
| 미분 가능 | 최적화에 필요한 기울기 계산 가능 |
| 큰 오차 강조 | 제곱으로 인해 큰 오차에 더 큰 패널티 |

### 3.2 기울기 (Gradient)

기울기는 **손실 함수가 각 파라미터에 대해 얼마나 빠르게 변하는지**를 나타냅니다.

**편미분 정의:**

```
       ∂MSE
∇w = ──────  (w에 대한 손실의 변화율)
        ∂w

       ∂MSE
∇b = ──────  (b에 대한 손실의 변화율)
        ∂b
```

### 3.3 기울기 계산 유도

**단계별 유도:**

1) 손실 함수 전개:
```
        1   n
MSE = ─── × Σ (yᵢ - (w×xᵢ + b))²
        n  i=1
```

2) w에 대한 편미분 (연쇄 법칙 적용):
```
∂MSE     1   n
──── = ─── × Σ 2(yᵢ - (w×xᵢ + b)) × (-xᵢ)
 ∂w      n  i=1

        2   n
     = ─── × Σ (ŷᵢ - yᵢ) × xᵢ
        n  i=1
```

3) b에 대한 편미분:
```
∂MSE     1   n
──── = ─── × Σ 2(yᵢ - (w×xᵢ + b)) × (-1)
 ∂b      n  i=1

        2   n
     = ─── × Σ (ŷᵢ - yᵢ)
        n  i=1
```

### 3.4 파라미터 업데이트 규칙

**핵심 공식:**

```
w_new = w_old - η × ∇w

b_new = b_old - η × ∇b
```

여기서 η(에타)는 **학습률(Learning Rate)**입니다.

**왜 빼는가?**
- 기울기가 양수 → 오른쪽으로 가면 손실 증가 → 왼쪽(반대)으로 이동
- 기울기가 음수 → 왼쪽으로 가면 손실 증가 → 오른쪽(반대)으로 이동

## 4. 선형 회귀에서의 경사하강법

### 4.1 전체 학습 과정

선형 회귀 모델: **ŷ = w × x + b**

```
┌─────────────────────────────────────────────────────────────┐
│                    경사하강법 학습 흐름                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐  │
│  │ 1. 초기화│ → │ 2. 예측 │ → │ 3. 손실 │ → │ 4. 기울기│  │
│  │  w, b   │    │ ŷ=wx+b │    │  계산   │    │  계산   │  │
│  └─────────┘    └─────────┘    └─────────┘    └─────────┘  │
│       ↑                                            │        │
│       │         ┌─────────────────────────────┐    │        │
│       └─────────│ 5. 파라미터 업데이트        │←───┘        │
│        반복!    │ w = w - η×∇w, b = b - η×∇b │             │
│                 └─────────────────────────────┘             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 단계별 상세 설명

**[단계 1] 파라미터 초기화**

```python
w = 0  # 또는 작은 랜덤 값
b = 0
```

**[단계 2] 예측값 계산**

모든 데이터에 대해 현재 w, b로 예측:

```python
for i in range(n):
    ŷᵢ = w × xᵢ + b
```

**[단계 3] 손실(MSE) 계산**

```python
MSE = (1/n) × Σ(yᵢ - ŷᵢ)²
```

**[단계 4] 기울기 계산**

```python
∇w = (2/n) × Σ(ŷᵢ - yᵢ) × xᵢ
∇b = (2/n) × Σ(ŷᵢ - yᵢ)
```

**[단계 5] 파라미터 업데이트**

```python
w = w - η × ∇w
b = b - η × ∇b
```

**[반복]** 손실이 충분히 작아지거나 정해진 에포크만큼 반복

### 4.3 의사코드 (Pseudocode)

```
함수 경사하강법(X, Y, 학습률, 에포크수):
    
    # 초기화
    w ← 0
    b ← 0
    n ← 데이터 개수
    
    # 학습 반복
    반복 epoch = 1 부터 에포크수:
        
        # 기울기 누적 변수 초기화
        grad_w ← 0
        grad_b ← 0
        total_loss ← 0
        
        # 모든 데이터에 대해
        반복 i = 1 부터 n:
            
            # 예측
            ŷᵢ ← w × xᵢ + b
            
            # 오차
            error ← ŷᵢ - yᵢ
            
            # 기울기 누적
            grad_w ← grad_w + error × xᵢ
            grad_b ← grad_b + error
            
            # 손실 누적
            total_loss ← total_loss + error²
        
        # 평균 기울기
        grad_w ← (2/n) × grad_w
        grad_b ← (2/n) × grad_b
        
        # 파라미터 업데이트
        w ← w - 학습률 × grad_w
        b ← b - 학습률 × grad_b
        
        # 평균 손실 출력
        출력 "에포크", epoch, "손실:", total_loss/n
    
    반환 w, b
```

## 5. 학습률의 역할

### 5.1 학습률이란?

**학습률(η, Learning Rate)**은 한 번 업데이트할 때 **얼마나 크게 이동할지**를 결정하는 하이퍼파라미터입니다.

```
w_new = w_old - η × ∇w
              ↑
           학습률
```

### 5.2 학습률의 영향

| 학습률 크기 | 특징 | 문제점 |
|------------|------|--------|
| **너무 작음** (0.0001) | 안정적이지만 매우 느림 | 수렴에 오랜 시간 소요 |
| **적절함** (0.01) | 빠르고 안정적 | 이상적 |
| **너무 큼** (1.0) | 빠르지만 불안정 | 발산 가능! |

### 5.3 학습률에 따른 수렴 패턴

**적절한 학습률:**
```
손실 │
     │╲
     │ ╲
     │  ╲
     │   ╲__________  ← 안정적으로 수렴
     │
     └────────────────── 에포크
```

**너무 작은 학습률:**
```
손실 │
     │╲
     │ ╲
     │  ╲
     │   ╲
     │    ╲
     │     ╲
     │      ╲________  ← 너무 느림!
     └────────────────── 에포크
```

**너무 큰 학습률:**
```
손실 │
     │    ╱╲    ╱╲
     │   ╱  ╲  ╱  ╲
     │  ╱    ╲╱    ╲  ← 발산!
     │ ╱
     │╱
     └────────────────── 에포크
```

### 5.4 학습률 선택 가이드

| 권장 학습률 | 적용 상황 |
|------------|----------|
| 0.1 | 빠른 실험용 |
| 0.01 | 일반적 시작점 |
| 0.001 | 정밀 조정 |
| 0.0001 | 미세 조정 |

**팁:** 처음에는 0.01로 시작하고, 손실이 진동하면 줄이고, 너무 느리면 늘리세요.

## 6. 경사하강법의 종류

### 6.1 배치 경사하강법 (Batch Gradient Descent)

**특징:** 전체 데이터셋을 사용하여 한 번 업데이트

```
1 에포크 = 전체 데이터로 1번 업데이트
```

| 장점 | 단점 |
|------|------|
| 안정적인 수렴 | 대용량 데이터에서 느림 |
| 정확한 기울기 계산 | 메모리 많이 필요 |

**적합한 경우:** 작은 데이터셋 (수천 개 이하)

### 6.2 확률적 경사하강법 (SGD, Stochastic Gradient Descent)

**특징:** 데이터 1개씩 사용하여 업데이트

```
1 에포크 = n개 데이터 × n번 업데이트
```

| 장점 | 단점 |
|------|------|
| 매우 빠름 | 불안정한 수렴 |
| 온라인 학습 가능 | 최솟값 주변 진동 |

**적합한 경우:** 실시간 스트리밍 데이터

### 6.3 미니배치 경사하강법 (Mini-batch Gradient Descent)

**특징:** 작은 그룹(배치)으로 나눠서 업데이트

```
1 에포크 = (n / 배치크기) 번 업데이트
```

| 장점 | 단점 |
|------|------|
| 속도와 안정성 균형 | 배치 크기 선택 필요 |
| GPU 병렬 처리 효율적 | |

**일반적 배치 크기:** 32, 64, 128, 256

### 6.4 세 가지 방식 비교

```
┌──────────────────────────────────────────────────────────────┐
│                   경사하강법 종류 비교                        │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  [배치 GD]          [미니배치 GD]        [SGD]               │
│                                                              │
│  전체 데이터         32~256개 배치        1개씩               │
│      ↓                  ↓                  ↓                 │
│  1번 업데이트       여러 번 업데이트    n번 업데이트          │
│                                                              │
│  ════════════       ────────────        ∿∿∿∿∿∿∿∿∿            │
│  (안정적)           (균형)              (불안정)             │
│                                                              │
│  느림               중간                빠름                  │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 7. 실제 계산 예제

### 7.1 문제 설정

사과 가격 예측 문제:
- 입력(X): 품질 점수 (0~100)
- 출력(Y): 가격 (원)

**샘플 데이터:**

| 번호 | 품질(x) | 가격(y) |
|:----:|:-------:|:-------:|
| 1 | 20 | 1,600 |
| 2 | 40 | 2,100 |
| 3 | 60 | 2,800 |
| 4 | 80 | 3,400 |

**목표:** y = w × x + b 에서 최적의 w, b 찾기

### 7.2 학습 과정 (수동 계산)

**초기값:**
- w = 0
- b = 0
- η (학습률) = 0.0001

---

**[에포크 1]**

**Step 1: 예측**

| i | xᵢ | 예측 ŷᵢ = 0×x + 0 | 실제 yᵢ |
|:-:|:--:|:-----------------:|:-------:|
| 1 | 20 | 0 | 1,600 |
| 2 | 40 | 0 | 2,100 |
| 3 | 60 | 0 | 2,800 |
| 4 | 80 | 0 | 3,400 |

**Step 2: 오차 계산**

| i | ŷᵢ - yᵢ | (ŷᵢ - yᵢ)² |
|:-:|:-------:|:----------:|
| 1 | -1,600 | 2,560,000 |
| 2 | -2,100 | 4,410,000 |
| 3 | -2,800 | 7,840,000 |
| 4 | -3,400 | 11,560,000 |

**MSE = (2,560,000 + 4,410,000 + 7,840,000 + 11,560,000) / 4 = 6,592,500**

**Step 3: 기울기 계산**

```
∇w = (2/4) × [(-1600)×20 + (-2100)×40 + (-2800)×60 + (-3400)×80]
   = 0.5 × [-32,000 + (-84,000) + (-168,000) + (-272,000)]
   = 0.5 × (-556,000)
   = -278,000

∇b = (2/4) × [(-1600) + (-2100) + (-2800) + (-3400)]
   = 0.5 × (-9,900)
   = -4,950
```

**Step 4: 파라미터 업데이트**

```
w_new = 0 - 0.0001 × (-278,000) = 27.8
b_new = 0 - 0.0001 × (-4,950) = 0.495
```

---

**[에포크 2]**

**새로운 파라미터:** w = 27.8, b = 0.495

**Step 1: 예측**

| i | xᵢ | 예측 ŷᵢ = 27.8×x + 0.495 | 실제 yᵢ |
|:-:|:--:|:------------------------:|:-------:|
| 1 | 20 | 556.5 | 1,600 |
| 2 | 40 | 1,112.5 | 2,100 |
| 3 | 60 | 1,668.5 | 2,800 |
| 4 | 80 | 2,224.5 | 3,400 |

**MSE가 이전보다 줄었는지 확인:**
- 에포크 1 MSE: 6,592,500
- 에포크 2 MSE: 약 1,034,500 (크게 감소!)

### 7.3 수렴 후 결과

여러 에포크를 반복하면:

```
최종 결과 (약 1000 에포크 후):
w ≈ 30 (품질 1점당 30원 상승)
b ≈ 1,000 (기본 가격 1,000원)

모델: 가격 = 30 × 품질 + 1,000
```

**검증:**
- 품질 50점 → 가격 = 30 × 50 + 1,000 = 2,500원 ✓

## 8. 수렴 조건과 종료 기준

### 8.1 언제 학습을 멈출까?

| 종료 조건 | 설명 | 예시 |
|----------|------|------|
| **에포크 횟수** | 정해진 반복 횟수 도달 | 500회 |
| **손실 임계값** | 손실이 충분히 작아짐 | MSE < 0.001 |
| **손실 변화량** | 손실 감소가 미미함 | ΔLoss < 0.0001 |
| **기울기 크기** | 기울기가 거의 0 | ‖∇‖ < 0.001 |

### 8.2 조기 종료 (Early Stopping)

검증 데이터의 손실이 더 이상 개선되지 않으면 학습 중단:

```
손실 │
     │   훈련 손실
     │  ╲__________
     │
     │     검증 손실
     │    ╲___╱╲___  ← 이 지점에서 멈춤!
     │              (과적합 방지)
     └────────────────── 에포크
```

## 9. 일반적인 문제와 해결책

### 9.1 문제: 발산 (Divergence)

**증상:** 손실이 증가하거나 NaN/Inf 발생

**원인:** 학습률이 너무 큼

**해결책:**
- 학습률을 1/10로 줄이기
- 데이터 정규화 적용

### 9.2 문제: 느린 수렴

**증상:** 손실이 매우 천천히 감소

**원인:** 학습률이 너무 작거나 특성 스케일 불균형

**해결책:**
- 학습률 증가
- 특성 정규화 (Z-score, Min-Max)

### 9.3 문제: 지역 최솟값 (Local Minimum)

**증상:** 손실이 더 이상 줄지 않지만 최적이 아님

```
손실 │
     │╲
     │ ╲    ╱╲
     │  ╲__╱  ╲    ← 지역 최솟값에 갇힘!
     │         ╲__  ← 전역 최솟값
     └────────────── w
```

**해결책:**
- 다른 초기값으로 재시도
- 모멘텀 사용
- 학습률 스케줄링

### 9.4 정규화의 중요성

**정규화 전:**
```
품질: 0~100
가격: 1000~4000

→ 스케일 차이로 기울기 불균형
→ 학습 불안정
```

**정규화 후 (Z-score):**
```
품질: -2 ~ +2 (평균 0, 표준편차 1)
가격: -2 ~ +2 (평균 0, 표준편차 1)

→ 균형 잡힌 기울기
→ 안정적 학습
```

**Z-score 정규화 공식:**
```
x_정규화 = (x - 평균) / 표준편차
```

## 10. 핵심 용어 정리

| 용어 | 영문 | 의미 |
|------|------|------|
| **경사하강법** | Gradient Descent | 기울기를 따라 최솟값을 찾는 최적화 알고리즘 |
| **손실 함수** | Loss Function | 예측 오차를 측정하는 함수 |
| **MSE** | Mean Squared Error | 평균 제곱 오차 |
| **기울기** | Gradient | 손실 함수의 편미분 값 |
| **학습률** | Learning Rate (η) | 파라미터 업데이트 크기 |
| **에포크** | Epoch | 전체 데이터로 한 번 학습하는 주기 |
| **배치** | Batch | 한 번에 처리하는 데이터 그룹 |
| **수렴** | Convergence | 손실이 최솟값에 도달함 |
| **발산** | Divergence | 손실이 증가하여 학습 실패 |
| **정규화** | Normalization | 데이터 스케일 조정 |
| **파라미터** | Parameter | 학습을 통해 결정되는 값 (w, b) |
| **하이퍼파라미터** | Hyperparameter | 사람이 설정하는 값 (학습률, 에포크) |

## 요약: 경사하강법 핵심 공식

### 선형 회귀 모델
```
ŷ = w × x + b
```

### 손실 함수 (MSE)
```
MSE = (1/n) × Σ(yᵢ - ŷᵢ)²
```

### 기울기
```
∇w = (2/n) × Σ(ŷᵢ - yᵢ) × xᵢ
∇b = (2/n) × Σ(ŷᵢ - yᵢ)
```

### 파라미터 업데이트
```
w ← w - η × ∇w
b ← b - η × ∇b
```

### 학습 과정
```
1. 초기화 (w=0, b=0)
2. 예측 (ŷ = wx + b)
3. 손실 계산 (MSE)
4. 기울기 계산 (∇w, ∇b)
5. 파라미터 업데이트
6. 반복 (2~5)
```
