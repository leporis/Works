## ⚡ 1분 초속성 실습

| 단계 | 행동 | 확인 |
|:---:|------|:----:|
| 1 | 좌측 **데이터셋** 드롭다운에서 `🏠 보스턴 주택가격` 선택 | 산점도에 파란 점이 나타났나요? |
| 2 | 우측 **⚡ 학습 실행** 버튼 클릭 | 손실 그래프가 L자로 꺾이나요? |
| 3 | 결과 패널에서 **R²** 확인 | 0.5 이상이면 🎉 성공! |

> 30초 만에 "집 크기로 가격을 예측하는 모델"을 학습시켰습니다!

## 🚀 빠른 시작 (3단계)

### 1️⃣ 데이터 준비

좌측 사이드바 상단에서 데이터를 선택합니다.

| 탭 | 용도 | 추천 시점 |
|:--:|------|----------|
| **데이터셋** | 내장 27종 중 선택 | 처음 시작할 때 |
| **CSV** | 내 파일 업로드 | 자기만의 데이터로 실습할 때 |
| **직접** | 숫자 입력 또는 차트 클릭 | 원하는 패턴을 직접 만들 때 |

**처음이라면**: `📚 공부시간과 성적` → 직관적이고 데이터가 적어서(25건) 학습이 빠릅니다.

데이터셋을 선택하면 아래에 **X변수**(독립)와 **Y변수**(종속)를 고를 수 있습니다.
- 다중 X변수를 선택하면 **가중치 슬라이더**가 나타나 변수 간 비중을 조절할 수 있습니다.

**🏷️ 범주형 필터** (일부 데이터셋):
Iris, 펭귄, MPG 등 9종 데이터셋에서는 칩 형태의 필터가 나타납니다.
- 파란 칩 = 포함, 회색 칩 = 제외
- 예: Iris에서 `setosa`만 선택 → setosa 품종만으로 회귀 실행

### 2️⃣ 학습 실행

우측 사이드바에서 설정 후 학습합니다.

| 설정 | 기본값 | 추천 범위 | 역할 |
|------|:------:|:---------:|------|
| **학습률** | 0.01 | 0.001 ~ 0.05 | 한 번에 얼마나 배울지 (너무 크면 발산!) |
| **에포크** | 500 | 300 ~ 1,000 | 데이터를 몇 번 반복 학습할지 |

설정을 마쳤으면 **⚡ 학습 실행** 버튼을 클릭하세요.

### 3️⃣ 결과 확인

학습이 끝나면 결과 패널이 나타납니다.

| 지표 | 의미 | 좋은 값 |
|------|------|:-------:|
| **훈련 MSE** | 훈련 데이터에서의 평균 오차 크기 | 작을수록 좋음 |
| **테스트 MSE** | 새 데이터에서의 평균 오차 크기 | 훈련 MSE와 비슷하면 좋음 |
| **R²** | 모델이 데이터를 얼마나 설명하는지 | 0.8 이상 ✅, 0.5~0.8 ⚠️, 0.5 미만 ❌ |

결과를 확인한 뒤 상단 **🔮 추론 모드** 탭을 클릭하면, X 값을 넣어 Y를 직접 예측해볼 수 있습니다.

| 🏷️ 필터 배너 | 주황색 "필터 적용 중" 표시 | 전체가 아닌 부분 데이터만 보고 있음을 경고 |
| 🔮 추론 차트 | 노란 점이 회귀선 위에 위치 | X 값에 대한 Y 예측이 수행됨 |
| 📊 결과 패널 | 훈련 MSE ≪ 테스트 MSE | ⚠️ 과적합 — 훈련 데이터에만 맞춰진 상태 |

## ⚙️ 주요 설정 안내

### 데이터 입력 방식

| 방식 | 데이터 수 | 특징 |
|------|:---------:|------|
| **내장 데이터셋** | 11~506건 | 27종, 실제 공개 데이터 기반 |
| **랜덤 생성** | 5~100건 (조절 가능) | 4가지 시나리오 + 노이즈 3단계 (5%/10%/20%) |
| **CSV 업로드** | 파일에 따라 다름 | 수치·범주형 자동 감지, 유니크값 ≤20이면 범주형 |
| **직접 입력** | 수동 추가 | 숫자 입력 또는 산점도 클릭 |

### 범주형 필터 대상 데이터셋

| 데이터셋 | 필터 가능 항목 | 예시 실습 |
|---------|--------------|----------|
| 🌸 Iris | 품종 | setosa vs versicolor 비교 |
| 🐧 펭귄 | 종, 섬, 성별 | Adelie만 필터 → 성별로 다시 필터 |
| 🍽️ 팁 | 성별, 흡연, 요일, 시간대 | 흡연자만 필터 → R² 변화 관찰 |
| 💎 다이아몬드 | 컷·색상·투명도 등급 | Ideal 등급만 필터 → 가격 예측 |

> ⚠️ **필터 사용 시 주의**: 필터는 표본 구성을 변경합니다. 필터 전후의 R²가 다르다면, "데이터 전체의 경향"과 "부분의 경향"이 다르다는 뜻입니다. 결과 해석 시 필터 조건을 반드시 함께 기록하세요.

### 학습 과정 재생

학습 완료 후, 우측 사이드바 하단의 **▶️ 재생** 버튼을 누르면:
- 에포크 1부터 마지막까지 회귀선이 움직이는 애니메이션을 볼 수 있습니다.
- **속도 슬라이더**로 재생 속도를 조절하세요 (1 = 느림, 10 = 빠름).
- ⏸️ 일시정지, ⏹️ 정지로 원하는 시점에 멈출 수 있습니다.

## 🆘 도와주세요!

| 증상 | 원인 | 해결책 |
|------|------|--------|
| 손실 그래프가 위아래로 춤춘다 | 학습률이 너무 큼 | 학습률을 **0.001**로 낮추세요 |
| 손실이 안 줄어든다 | 에포크가 부족하거나 학습률이 너무 작음 | 에포크를 **1,000**으로 늘리거나 학습률을 **0.05**로 높여보세요 |
| R²가 0에 가깝다 | 선형 관계가 없는 데이터 | 앤스컴 2(곡선)처럼 비선형 데이터일 수 있습니다. 다른 X/Y 조합을 시도해보세요 |
| 학습 버튼이 비활성화됐다 | 필터로 데이터가 3건 이하 | 필터를 넓히거나 **↺ 필터 초기화** 클릭 |
| "최소 2개 이상의 데이터가 필요합니다" | 데이터 부족 | 데이터셋을 선택하거나, 직접 입력에서 점을 더 추가하세요 |
| 추론 모드에 "--"만 보인다 | 아직 학습을 안 함 | 학습 모드에서 먼저 **⚡ 학습 실행** 후 추론 모드로 전환하세요 |
| 산점도가 비어 있다 | 변수 선택이 잘못됨 | X변수와 Y변수가 같은 변수이면 안 됩니다 |
| "재분할 불가" 메시지 | 데이터 5개 미만 | 데이터를 5개 이상 확보하세요 |
| 주황색 배너가 사라지지 않는다 | 필터가 활성 상태 | 좌측 필터 영역에서 **↺ 필터 초기화** 클릭 |

## 🔑 핵심 용어

| 용어 | 학술 용어 | 의미 | 쉬운 설명 |
|------|----------|------|----------|
| 회귀선 | Regression Line | Y = wX + b 직선 | 데이터의 "평균적 경향"을 나타내는 직선 |
| 기울기 | Slope (w) | X가 1 증가할 때 Y의 변화량 | "면적 1㎡ 늘면 가격이 얼마나 오르나?" |
| 절편 | Intercept (b) | X = 0일 때의 Y 값 | "면적이 0이면 기본 가격은?" |
| 학습률 | Learning Rate | 한 에포크에서 파라미터 업데이트 크기 | "한 걸음의 크기" — 크면 빠르지만 불안정 |
| 에포크 | Epoch | 전체 데이터를 한 번 학습하는 단위 | "교과서를 처음부터 끝까지 1번 읽기 = 1에포크" |
| MSE | Mean Squared Error | 예측 오차의 제곱 평균 | "모델이 얼마나 빗나갔는지"의 점수 |
| R² | Coefficient of Determination | 모델이 데이터 변동을 설명하는 비율 | "100점 만점에 몇 점?" (0.85 = 85점) |
| 훈련/테스트 분할 | Train/Test Split | 데이터를 80%/20%로 나눠 학습·평가 분리 | "모의고사용 문제와 실전 시험 문제를 분리" |
| 과적합 | Overfitting | 훈련 데이터에만 맞고 새 데이터에 약함 | "족보만 외워서 시험엔 약한 상태" |
| 경사 하강법 | Gradient Descent | 기울기 방향으로 파라미터를 조금씩 조절 | "눈 감고 산에서 내려갈 때, 발끝이 가리키는 가장 가파른 쪽으로 한 발짝" |

## 💭 생각해보기

> **1.** 학습률 0.001과 0.1로 같은 데이터를 학습시켜보세요. 손실 그래프의 모양이 어떻게 다른가요? 왜 그럴까요?

> **2.** Iris 데이터에서 세 품종을 모두 포함한 R²와, setosa만 필터링한 R²가 다릅니다. 어느 쪽의 모델을 "더 좋다"고 할 수 있을까요?

> **3.** 앤스컴 데이터 4종의 R²는 모두 거의 같습니다. 그런데 산점도는 전혀 다릅니다. 이것이 "R²만으로 모델을 평가하면 안 되는 이유"와 어떤 관련이 있을까요?

> **4.** 훈련 MSE는 매우 낮은데 테스트 MSE가 높다면, 이 모델을 실제로 사용해도 될까요?
