## 1. 퀵가이드의 정체성

### 1.1 퀵가이드란?

퀵가이드는 **소프트웨어 옆에 띄워두는 '내비게이션'**입니다.

```
┌─────────────────────────────────────────────────────────────┐
│                     퀵가이드의 역할                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ❌ 퀵가이드는 이것이 아닙니다:                             │
│      • 모든 기능을 설명하는 매뉴얼                           │
│      • 이론을 깊이 다루는 교재                               │
│      • 읽고 나서 소프트웨어를 사용하는 문서                   │
│                                                             │
│   ✅ 퀵가이드는 이것입니다:                                  │
│      • 소프트웨어와 함께 보는 실시간 안내서                   │
│      • "지금 뭘 눌러야 하지?" 순간의 해결사                  │
│      • 문제 발생 시 즉시 참조하는 트러블슈팅 카드             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 다른 문서와의 차이

| 문서 유형 | 목적 | 사용 시점 | 깊이 |
|----------|------|----------|:----:|
| **퀵가이드** | 즉시 실행, 빠른 참조 | 소프트웨어 사용 **중** | ⭐ |
| 사용설명서 | 전체 기능 상세 설명 | 소프트웨어 사용 **전/후** | ⭐⭐⭐ |
| 이론해설서 | 개념 및 원리 학습 | 별도 학습 시간 | ⭐⭐⭐⭐ |
| 활용가이드 | 수업 적용 방법 | 수업 준비 시 | ⭐⭐⭐ |

### 1.3 핵심 질문

퀵가이드 작성 시 항상 스스로에게 물어보세요:

> **"사용자가 소프트웨어 화면과 이 문서를 동시에 보고 있다면, 이 내용이 필요한가?"**

- "예" → 포함
- "나중에 읽어도 됨" → 다른 문서로 이동

## 2. 핵심 설계 원칙

### 2.1 Action First (행동 우선)

```
❌ 잘못된 예:
"선형 회귀는 독립 변수와 종속 변수 간의 선형 관계를 
모델링하는 통계적 방법입니다. 이 시뮬레이터에서는..."

✅ 올바른 예:
"[✨ 데이터 생성] 버튼을 클릭하세요. 점들이 나타났나요?"
```

**원칙**: 설명보다 행동 지시가 먼저 나와야 합니다.

### 2.2 Success First (성공 경험 우선)

```
┌─────────────────────────────────────────┐
│         전통적 순서 (❌)                │
│  이론 → 설명 → 실습 → 결과             │
├─────────────────────────────────────────┤
│         퀵가이드 순서 (✅)              │
│  실습 → 성공! → "왜?" → 이론           │
└─────────────────────────────────────────┘
```

**원칙**: 사용자가 **30초 내에 성공 경험**을 맛보게 하세요.

### 2.3 Observe, Don't Just Do (관찰 유도)

```
❌ 단순 지시:
"학습 실행 버튼을 누르세요."

✅ 관찰 유도:
"학습 실행 버튼을 누르세요. 
 → 손실 그래프가 'L'자로 꺾이며 내려가나요?"
```

**원칙**: 버튼을 누르는 것은 '조작', 변화를 인식하는 것이 '학습'입니다.

### 2.4 Escape Route (탈출구 제공)

```
❌ 문제 상황 무시:
(트러블슈팅 없음)

✅ 탈출구 제공:
"🆘 선이 춤을 춰요 → 학습률을 0.01 이하로 낮추세요"
```

**원칙**: 예상되는 문제 상황마다 **1줄 해결책**을 제시하세요.

## 3. 필수 구성요소

### 3.1 필수 섹션 (반드시 포함)

| 순서 | 섹션명 | 목적 | 권장 분량 |
|:---:|--------|------|:--------:|
| 1 | **📖 이 문서 사용법** | 퀵가이드 활용 방법 안내 | 0.5쪽 |
| 2 | **⚡ 초속성 실습** | 30초~1분 내 성공 경험 | 0.5쪽 |
| 3 | **🚀 빠른 시작** | 핵심 기능 3단계 | 0.5쪽 |
| 4 | **🖥️ 화면 구성** | UI 레이아웃 안내 | 0.5쪽 |
| 5 | **🔍 관찰 포인트** | 무엇을 봐야 하는지 | 0.5쪽 |
| 6 | **🆘 트러블슈팅** | 문제 상황별 해결책 | 0.5쪽 |
| 7 | **🔑 핵심 용어** | 용어 정의 (학술+쉬운) | 0.5쪽 |

### 3.2 선택 섹션 (필요시 포함)

| 섹션명 | 포함 조건 |
|--------|----------|
| 📚 문서 읽기 순서 | 관련 문서가 3개 이상일 때 |
| 📐 수식/계산 설명 | 수학적 개념이 포함될 때 |
| ⚙️ 파라미터 설명 | 조절 가능한 설정이 많을 때 |
| ❓ FAQ | 예상 질문이 5개 이상일 때 |
| 🎯 실습 활동 | 교육용 소프트웨어일 때 |
| 📝 기록표 | 반복 실습이 필요할 때 |
| 💭 생각해보기 | 대학/성인 교육일 때 |

### 3.3 섹션 배치 원칙

```
문서 상단 (즉시 행동)
├── 📖 이 문서 사용법
├── ⚡ 초속성 실습
└── 🚀 빠른 시작

문서 중단 (참조용)
├── 🖥️ 화면 구성
├── 🔍 관찰 포인트
├── 📐 수식/계산
└── ⚙️ 파라미터

문서 하단 (문제 해결/심화)
├── 🆘 트러블슈팅
├── ❓ FAQ
├── 🔑 핵심 용어
└── 💭 생각해보기
```

## 4. 작성 순서

### 4.1 권장 작성 순서

실제로 문서를 작성할 때는 **읽는 순서와 다르게** 작성합니다:

| 순서 | 작성 항목 | 이유 |
|:---:|----------|------|
| 1 | 🔑 핵심 용어 | 문서 전체에서 사용할 용어 확정 |
| 2 | 🚀 빠른 시작 | 핵심 워크플로우 정의 |
| 3 | 🖥️ 화면 구성 | UI 요소 명칭 통일 |
| 4 | ⚡ 초속성 실습 | 빠른 시작에서 핵심만 추출 |
| 5 | 🔍 관찰 포인트 | 각 단계에서 관찰할 것 정리 |
| 6 | 🆘 트러블슈팅 | 예상 문제 상황 수집 |
| 7 | 📖 이 문서 사용법 | 전체 구조 파악 후 작성 |

### 4.2 반복 검토 사항

작성 후 다음을 확인하세요:

```
□ 모든 버튼명이 실제 UI와 일치하는가?
□ 모든 용어가 일관되게 사용되었는가?
□ 각 섹션이 독립적으로 참조 가능한가?
□ 초속성 실습이 실제로 1분 내에 가능한가?
```

## 5. 용어 처리 전략

### 5.1 3층 용어 구조

모든 전문 용어는 **3가지 표현**을 준비합니다:

| 층위 | 용도 | 예시 |
|------|------|------|
| **학술 용어** | 정확한 개념 전달 | Slope, Learning Rate |
| **번역 용어** | 한국어 대응 | 기울기, 학습률 |
| **쉬운 설명** | 직관적 이해 | "품질 1점 = 가격 몇 원?" |

### 5.2 용어 테이블 형식

```markdown
| 용어 | 학술 용어 | 의미 | 쉬운 설명 |
|------|----------|------|----------|
| **가중치** | Weight, **기울기(Slope)** | 입력이 출력에 미치는 영향력 | 품질 1점 = 가격 몇 원? |
```

### 5.3 본문에서의 용어 사용

```
❌ 학술 용어만:
"Learning Rate를 조절하여 Gradient Descent의 step size를 변경합니다."

❌ 쉬운 말만:
"배우는 속도를 조절해서 한 걸음 크기를 바꿔요."

✅ 병기:
"학습률(Learning Rate)을 조절하세요. 
 → 한 번에 얼마나 배울지 결정합니다."
```

### 5.4 수학 연결 팁

대학생/성인 대상이라면 기존 지식과 연결하세요:

```markdown
💡 **수학 시간에 배운 y = ax + b 기억나세요?** 
   여기서 a가 기울기(가중치), b가 절편(편향)입니다!
```

## 6. 시각적 구성 원칙

### 6.1 ASCII 다이어그램 활용

복잡한 레이아웃은 ASCII 아트로 표현합니다:

```
✅ 좋은 예:
┌──────────┬─────────────────────────┬──────────┐
│  데이터   │  📉 손실    📈 산점도   │  학습    │
│  생성    │  그래프     (회귀선)    │  설정    │
└──────────┴─────────────────────────┴──────────┘
```

### 6.2 테이블 우선 원칙

장문의 설명보다 테이블을 사용합니다:

```
❌ 장문:
"학습률이 너무 높으면 선이 위아래로 요동치는 현상이 
발생합니다. 이 경우 학습률을 0.01 이하로 낮추면 
문제가 해결됩니다. 반대로 학습률이 너무 낮으면..."

✅ 테이블:
| 증상 | 원인 | 해결책 |
|------|------|--------|
| 선이 요동침 | 학습률 높음 | 0.01 이하로 |
| 선이 안 움직임 | 학습률 낮음 | 0.05로 높이기 |
```

### 6.3 이모지 사용 규칙

| 용도 | 이모지 | 예시 |
|------|:------:|------|
| 섹션 제목 | 📖 🚀 🔍 | ## 🚀 빠른 시작 |
| 성공/긍정 | ✅ 🎉 ✨ | ✅ 성공! |
| 경고/주의 | ⚠️ 🆘 ❌ | 🆘 문제 발생 |
| 팁/힌트 | 💡 🎯 | 💡 팁: |
| 단계 번호 | 1️⃣ 2️⃣ 3️⃣ | 1️⃣ 첫 번째 |

### 6.4 강조 사용 규칙

| 강조 유형 | 용도 | 예시 |
|----------|------|------|
| **굵게** | 핵심 키워드, 버튼명 | **[학습 실행]** 클릭 |
| `코드` | 수치, 파일명, 코드 | 학습률 `0.01` |
| > 인용 | 핵심 메시지, 질문 | > 이 문서는... |

## 7. 교육적 요소 삽입

### 7.1 관찰 포인트 작성법

단순 지시가 아닌 **관찰 질문** 형태로 작성합니다:

```
❌ 지시형:
"손실 그래프를 확인하세요."

✅ 관찰 질문형:
"손실 그래프가 'L'자로 꺾이며 내려가나요?"
"오차선 색상이 🔴빨강 → 🟢녹색으로 변했나요?"
```

### 7.2 분석가의 눈 유도

```markdown
💡 **분석가의 눈**: "왜 저 점은 오차가 클까?" 생각해보세요!
```

### 7.3 성찰 질문 (대학/성인용)

```markdown
## 💭 생각해보기

> **이 AI가 예측한 가격이 실제 시장 가격과 다르다면, 
> 누구의 책임일까요?**
> 
> 데이터의 문제일까요? 모델의 문제일까요?
```

### 7.4 기록표 (참여 유도)

```markdown
## 📝 나의 베스트 기록

| 시도 | 설정값 | **결과** |
|:---:|:------:|:-------:|
| 1 | | |
| 2 | | |
| **최고** | | **🏆** |

🎯 **도전 목표**: R² > 0.95 달성해보세요!
```

## 8. 품질 검증 체크리스트

### 8.1 구조 검증

```
□ 📖 이 문서 사용법이 최상단에 있는가?
□ ⚡ 초속성 실습이 문서 읽기 순서보다 앞에 있는가?
□ 🆘 트러블슈팅이 포함되어 있는가?
□ 🔑 핵심 용어에 학술 용어와 쉬운 설명이 병기되어 있는가?
```

### 8.2 내용 검증

```
□ 초속성 실습이 실제로 1분 내에 완료 가능한가?
□ 모든 버튼명/메뉴명이 실제 UI와 일치하는가?
□ 트러블슈팅이 실제 발생 가능한 문제를 다루는가?
□ 관찰 포인트가 구체적인가? (예/아니오로 답할 수 있는가?)
```

### 8.3 사용성 검증

```
□ 각 섹션이 독립적으로 참조 가능한가?
□ 소프트웨어 화면과 문서를 동시에 볼 때 유용한가?
□ 인쇄했을 때 A4 4~6페이지 이내인가?
□ 스마트폰/태블릿에서 읽기 편한가?
```

### 8.4 교육적 검증

```
□ 단순 조작 지시만 있지 않고 관찰 유도가 있는가?
□ 학술 용어와 일상 용어가 연결되어 있는가?
□ 기존 지식(수학 등)과의 연결점이 있는가?
□ 성찰 질문 또는 토론 주제가 있는가? (대학용)
```

## 9. 안티패턴: 피해야 할 것들

### 9.1 정보 과부하

```
❌ 안티패턴:
"선형 회귀(Linear Regression)는 통계학에서 종속 변수 y와 
한 개 이상의 독립 변수 X와의 선형 상관관계를 모델링하는 
회귀 분석 기법입니다. 이 방법은 1805년 르장드르가 
최소제곱법을 발표하면서 시작되었으며..."

✅ 개선:
"📈 품질 점수가 높을수록 가격이 올라가는 관계를 직선으로 표현합니다."
(자세한 내용은 이론해설서 참조)
```

### 9.2 추상적 지시

```
❌ 안티패턴:
"적절한 학습률을 설정하세요."

✅ 개선:
"학습률을 0.01로 설정하세요.
 (선이 요동치면 0.001로, 너무 느리면 0.05로)"
```

### 9.3 관찰 없는 행동

```
❌ 안티패턴:
"1. 데이터 생성 클릭
 2. 학습 실행 클릭
 3. 추론 모드 클릭"

✅ 개선:
"1. 데이터 생성 클릭 → 점들이 나타났나요?
 2. 학습 실행 클릭 → 선이 점들을 가로지르나요?
 3. R² 확인 → 0.9 이상이면 성공!"
```

### 9.4 탈출구 없음

```
❌ 안티패턴:
(문제 상황에 대한 안내 없음)

✅ 개선:
"🆘 문제가 생겼나요?
 • 선이 춤춰요 → 학습률 낮추기
 • 선이 안 움직여요 → 학습률 높이기"
```

### 9.5 과도한 친절

```
❌ 안티패턴:
"틀린 정도의 평균이에요. 쉽게 말해서 AI가 얼마나

✅ 개선:
| MSE | Mean Squared Error | 틀린 정도의 평균 (작을수록 좋음) |
```
