## 2. 데이터 구조 통합 패턴

### 2.1 문제 인식

시뮬레이터가 성장하면서 데이터 소스가 여러 객체로 분산되는 현상이 발생한다.

```
[전형적 성장 경로]

v1.0  프리셋 4개 (합성 생성기)
       └─ presets = { housing: {...}, exam: {...} }

v1.1  + CSV 업로드 기능
       └─ handleCSVUpload()

v1.2  + 내장 실데이터 4개
       └─ builtinDatasets = { iris: {...}, penguin: {...} }

v1.4  + 공개 데이터셋 23개 ← 여기서 구조 문제 폭발
       └─ presets(4) + builtinDatasets(4) + 새 데이터(19) = 3중 구조?
```

**문제 증상**:
- 동일 목적(데이터 제공)의 객체가 2개 이상으로 분리
- 각 객체마다 별도의 로드 함수가 존재
- UI에서 데이터 소스별로 다른 탭/영역에 흩어져 있음
- 새 데이터를 추가할 때 어디에 넣어야 할지 불명확

### 2.2 해법: 동질 통합 + 이질 분리

| 판단 기준 | 조치 |
|----------|:----:|
| 두 객체가 동일한 형태(headers + data)를 갖는다 | **통합** |
| 데이터 제공 방식이 근본적으로 다르다 (생성 vs 내장) | **분리** |
| 하나의 객체에 생성 파라미터와 실데이터가 혼재한다 | **분리 후 통합** |

### 2.3 적용 절차

```
[1단계] 기존 데이터 객체 전수 조사
        → 어떤 객체가 있는가? 각각 몇 개 항목? 어떤 필드 구조?

[2단계] 동질/이질 판단
        → headers+data 형태의 실데이터끼리 묶기
        → 합성 생성기(파라미터 기반)는 별도 분리

[3단계] 통합 객체 설계
        → 하나의 DATASETS 객체로 실데이터 통합
        → RANDOM_PRESETS 등으로 생성기 분리

[4단계] 로드 함수 단일화
        → loadSelectedDataset() 하나로 통합
        → 생성기 전용 함수는 별도 (generateRandomData())

[5단계] 기존 함수 래퍼 유지 (§9 참조)
```

### 2.4 적용 전 → 후

```javascript
// ❌ 적용 전: 2개 객체, 2개 로드 경로
const presets = {
    housing: { xRange: [20,150], slope: 15, noise: 100 },
    exam:    { xRange: [0,100],  slope: 0.7, noise: 5 },
};
const builtinDatasets = {
    iris:    { headers: [...], data: [...] },
    penguin: { headers: [...], data: [...] },
};
function loadPreset(key) { /* presets에서 생성 */ }
function loadBuiltinDataset(key) { /* builtinDatasets에서 로드 */ }

// ✅ 적용 후: 역할별 분리, 실데이터 통합
const DATASETS = {
    iris:    { name: '🌸 붓꽃', headers: [...], data: [...], ... },
    penguin: { name: '🐧 펭귄', headers: [...], data: [...], ... },
    // ... 23개 (확장 용이)
};
const RANDOM_PRESETS = {
    housing: { xRange: [20,150], slope: 15, noise: 100 },
    exam:    { xRange: [0,100],  slope: 0.7, noise: 5 },
};
function loadSelectedDataset() { /* DATASETS에서 로드 (단일 경로) */ }
function generateRandomData() { /* RANDOM_PRESETS에서 생성 (별도 경로) */ }
```

### 2.5 네이밍 컨벤션

| 구분 | 네이밍 | 이유 |
|------|--------|------|
| 통합 실데이터 | `DATASETS` (대문자, 복수) | 상수 + 다수 항목 |
| 분리된 생성기 | `RANDOM_PRESETS` (대문자, 접두사) | 상수 + 용도 명시 |
| 통합 로드 함수 | `loadSelectedDataset()` | 동작 명확 |
| 생성 전용 함수 | `generateRandomData()` | generate = 새로 만든다 |

## 3. 데이터셋 객체 표준

### 3.1 필수 필드 정의

**교육용코드개선지침서** §6.3의 `BUILTIN_DATASETS` 템플릿을 확장하여, 대규모 다변량 데이터셋에 대응하는 표준 구조를 정의한다.

```javascript
const DATASETS = {
    'dataset_key': {
        // ── 메타 정보 (UI 표시용) ──
        name: '🌸 붓꽃(Iris)',
        source: 'Fisher(1936), UCI ML Repository | CC BY 4.0',
        description: '붓꽃 150송이의 꽃받침·꽃잎 크기 측정값',
        defaultTarget: '꽃잎길이(cm)',
        recommended: '꽃잎길이 → 꽃잎너비 (r=0.96)',

        // ── 데이터 구조 ──
        headers: ['c0', 'c1', 'c2', 'c3'],
        headerLabels: ['꽃받침길이(cm)', '꽃받침너비(cm)',
                       '꽃잎길이(cm)', '꽃잎너비(cm)'],
        data: [[5.1, 3.5, 1.4, 0.2], ...]
    }
};
```

### 3.2 기존 템플릿과의 차이

| 항목 | 교육용코드개선지침서 §6.3 | 이 지침서 (확장) |
|------|-------------------------|-----------------|
| 변수 수 | 고정 (x1, x2, y) | 가변 (headers 배열) |
| 변수명 | 직접 필드 (`x1Label`) | 내부 키 + 표시명 분리 |
| 분류 정보 | classAName, classBName | 없음 (회귀용) |
| 메타 정보 | name, description | + source, defaultTarget, recommended |
| 데이터 형태 | 객체 배열 `[{x1:, x2:, label:}]` | 행 기반 2D 배열 `[[v1, v2, ...]]` |

### 3.3 필수 규칙

| # | 규칙 | 이유 | 위반 시 증상 |
|:-:|------|------|-------------|
| 1 | `headers`와 `headerLabels` 길이 동일 | 인덱스 매핑 의존 | 축 레이블 불일치 |
| 2 | `headers`는 짧은 내부 키 (`c0`, `c1`...) | 파일 크기 절감 | 불필요한 비대화 |
| 3 | `headerLabels`에 단위 포함 | 축 레이블에 그대로 사용 | 단위 누락 |
| 4 | `defaultTarget`은 `headerLabels` 값과 정확히 일치 | `indexOf()`로 매칭 | 기본 Y 미선택 |
| 5 | `data`는 숫자만 | 수치형 검증 생략 | NaN 에러 |
| 6 | `source`에 라이선스 명시 | 교육용 배포 시 저작권 준수 | 법적 문제 |

### 3.4 분류 알고리즘용 확장 필드

회귀가 아닌 분류 시뮬레이터에서는 다음 필드를 추가한다:

```javascript
// 분류용 추가 필드

}
```

## 4. 대규모 데이터 임베딩

### 4.1 문제

단일 HTML 파일에 수백~수천 행의 데이터를 포함하면 파일이 비대해진다. 교육용 시뮬레이터는 외부 파일 로드 없이 단독 실행되어야 하므로 임베딩은 불가피하다.

### 4.2 파일 크기 관리 전략

| 전략 | 적용 기준 | 효과 | 예시 |
|------|----------|------|------|
| 행 기반 압축 포맷 | 항상 | 줄 수 30~50% 감소 | N≤30: 3행/줄, N>30: 5행/줄 |
| 대규모 데이터 샘플링 | 원본 >1000행 | 파일 크기 상한 제어 | 1000행 → 500행 |
| 내부 키 단축 | 항상 | 반복 문자열 절감 | `sepal_length` → `c0` |
| 소수점 자릿수 통일 | 항상 | 일관성 + 크기 절감 | 소수 2자리 |

### 4.3 Python 자동 변환 스크립트

대규모 데이터셋 임베딩은 수동으로 하지 않는다. JSON → JS 자동 변환 스크립트를 사용한다.

```python
# 핵심 로직 (개요)
import json

with open('presets_data.json') as f:
    raw = json.load(f)

for key, ds in raw.items():
    columns = list(ds['columns'].keys())
    n_rows = len(ds['columns'][columns[0]])

    # 1. 컬럼 기반 → 행 기반 전치
    rows = [[ds['columns'][col][i] for col in columns] for i in range(n_rows)]

    # 2. 내부 키 생성
    headers = [f'c{i}' for i in range(len(columns))]

    # 3. 압축 포맷 (N 기준 행/줄 결정)
    per_line = 3 if n_rows <= 30 else 5

    # 4. JS 코드 문자열 생성
    # → data: [[v1,v2,v3],[v4,v5,v6],[v7,v8,v9],
    #          [v10,v11,v12],[v13,v14,v15]]
```

**원칙**: 데이터셋 10개 이상 추가할 때는 반드시 Python 스크립트를 사용한다. 수동 복사-붙여넣기는 오류 원인이 된다.

### 4.4 카테고리는 JS가 아닌 HTML에서 관리

```javascript
// ❌ JS 데이터에 category 필드 → 드롭다운 동적 생성 시 복잡
const DATASETS = { iris: { category: '자연과학', ... } };

// ✅ 카테고리는 HTML optgroup으로 직접 표현
<optgroup label="🌿 자연과학">
    <option value="iris">🌸 붓꽃(Iris) - 150개</option>
</optgroup>
```

**이유**: 카테고리는 UI 표시 관심사이므로 HTML에서 관리한다. JS 데이터 객체는 순수 데이터만 보유하여 관심사를 분리한다.

## 5. 셀렉터 UI 확장 패턴

### 5.1 전환 기준

선택지 개수가 증가하면 UI 형태를 전환해야 한다.

| 항목 수 | 적합한 UI | HTML 구조 | 전환 시점 |
|:-------:|----------|----------|----------|
| 2~6개 | 버튼 그리드 | `<button class="preset-btn">` | 초기 개발 |
| 7~15개 | 드롭다운 | `<select>` + `<option>` | 중간 확장 |
| 16개 이상 | 카테고리 드롭다운 | `<select>` + `<optgroup>` | 대규모 확장 |

### 5.2 전환 시 변경 범위

버튼 그리드 → 카테고리 드롭다운 전환 시 영향받는 영역:

```
┌──────────────────────────────────────────────┐
│  변경 영역            작업 유형              │
├──────────────────────────────────────────────┤
│  CSS                                         │
│  ├─ .preset-grid, .preset-btn 제거    삭제   │
│  ├─ .preset-btn:hover, .active 제거   삭제   │
│  ├─ .dataset-selector 추가            신규   │
│  └─ .dataset-info-box 추가            신규   │
│                                              │
│  HTML                                        │
│  ├─ <button> 그리드 제거              삭제   │
│  ├─ <select> + <optgroup> 추가        신규   │
│  └─ 정보 박스 div 추가                신규   │
│                                              │
│  JS                                          │
│  ├─ loadPreset(key) 제거/래퍼화       변경   │
│  ├─ onDatasetChange() 신규            신규   │
│  ├─ loadSelectedDataset() 신규        신규   │
│  └─ active 클래스 토글 코드 제거       삭제   │
└──────────────────────────────────────────────┘
```

```
                   ─────── ─────────────   ────
```

## 6. 공유 컴포넌트 추출 패턴

### 6.1 문제 인식

변수 선택 UI처럼 여러 데이터 소스(데이터셋 탭, CSV 탭)가 공통으로 사용하는 UI가 특정 탭 내부에 종속되면, 다른 탭에서 접근할 수 없다.

**판단 기준**: 해당 UI를 **2개 이상 탭/기능**에서 접근해야 하면 추출 대상.

### 6.2 추출 전 → 후

```
[❌ 추출 전: CSV탭 내부 종속]
┌─ 탭 컨테이너 ──────────────────────────────┐
│  [프리셋탭]   [CSV탭]          [직접탭]     │
│               ├─ 파일 업로드               │
│               ├─ 변수 선택 UI  ← 종속!     │
│               └─ 내장 데이터 버튼           │
└─────────────────────────────────────────────┘

[✅ 추출 후: 탭 밖 공유 영역]
┌─ 탭 컨테이너 ──────────────────────────────┐
│  [데이터셋탭]  [CSV탭]         [직접탭]     │
│  (각 탭 고유 콘텐츠만)                      │
└─────────────────────────────────────────────┘
┌─ 공유 영역 (hidden, 데이터 로드 시 표시) ──┐
│  변수 선택 UI                               │
│  ├─ 독립변수(X) 체크박스                    │
│  ├─ 종속변수(Y) 드롭다운                    │
│  └─ 가중치 슬라이더                         │
└─────────────────────────────────────────────┘
```

### 6.3 추출 절차 (4단계)

| 단계 | 작업 | 영향 범위 | 확인 사항 |
|:----:|------|----------|----------|
| 1 | HTML에서 해당 UI 블록을 탭 밖으로 이동 | HTML | 탭 닫는 `</div>` 뒤에 배치 |
| 2 | CSS 클래스명을 공유 의미로 변경 | CSS | `.var-select-group` → `.shared-var-select` |
| 3 | JS에서 해당 UI를 표시/숨김하는 모든 코드 점검 | JS | `getElementById` 호출처 전수 확인 |
| 4 | `resetAll()`에서 공유 UI 숨김 로직 추가 | JS | §11 부수 작업 참조 |

### 6.4 추출 후 체크리스트

| # | 확인 항목 |
|:-:|----------|
| 1 | 추출한 UI의 ID가 HTML에서 유일한가? (중복 ID = 치명적 버그) |
| 2 | 원래 위치의 코드가 **완전히** 제거되었는가? |
| 3 | 모든 데이터 소스(탭)에서 공유 UI에 접근 가능한가? |
| 4 | `resetAll()`에서 공유 UI를 숨기는 로직이 추가되었는가? |
| 5 | 탭 전환 시 공유 UI 상태가 올바르게 유지되는가? |
| 6 | 공유 UI를 표시하는 모든 경로(데이터셋 로드, CSV 로드)에서 정상 작동하는가? |

## 7. 조건부 UI 토글 패턴

### 7.1 문제

하나의 드롭다운에서 일반 항목(데이터셋)과 특수 항목(랜덤 생성)을 함께 제공할 때, 선택에 따라 다른 UI를 표시해야 한다.

### 7.2 패턴: classList.toggle

```javascript
function onDatasetChange() {
    const key = document.getElementById('datasetSelect').value;
    const isRandom = (key === '_random');

    // toggle(className, condition) — 한 줄로 표시/숨김 전환
    document.getElementById('datasetInfoBox').classList.toggle('hidden', isRandom);
    document.getElementById('randomControls').classList.toggle('hidden', !isRandom);

    if (!isRandom && DATASETS[key]) {
        document.getElementById('diSource').textContent = '출처: ' + DATASETS[key].source;
        document.getElementById('diDesc').textContent = DATASETS[key].description;
        document.getElementById('diRec').textContent = '💡 ' + DATASETS[key].recommended;
    }
}
```

### 7.3 classList.toggle vs if-else

```javascript
// ❌ 장황한 if-else
if (isRandom) {
    document.getElementById('datasetInfoBox').classList.add('hidden');
    document.getElementById('randomControls').classList.remove('hidden');
} else {
    document.getElementById('datasetInfoBox').classList.remove('hidden');
    document.getElementById('randomControls').classList.add('hidden');
}

// ✅ toggle(class, condition)으로 한 줄씩
document.getElementById('datasetInfoBox').classList.toggle('hidden', isRandom);
document.getElementById('randomControls').classList.toggle('hidden', !isRandom);
```

### 7.4 적용 한계

| 토글 대상 수 | 적합한 방식 |
|:-----------:|-----------|
| 2개 | `classList.toggle` |
| 3개 이상 | 상태 객체 + 루프, 또는 CSS 클래스 기반 상태 관리 |

## 8. 선택-실행 분리 패턴

### 8.1 문제

데이터셋이 소수일 때는 "선택 = 즉시 로드"가 자연스럽다. 그러나 데이터셋이 많아지면 사용자가 내용을 확인한 후 로드 여부를 결정할 수 있어야 한다.

### 8.2 1단계 → 2단계 전환

```
[1단계 (소규모)]
버튼 클릭 → 즉시 데이터 로드 + 차트 갱신

[2단계 (대규모)]
드롭다운 선택 → 정보 표시 (미리보기) → "로드" 버튼 → 데이터 로드 + 차트 갱신
```

### 8.3 구현

```javascript
// 1단계: 선택 시 정보만 표시 (onchange)
function onDatasetChange() {
    const key = document.getElementById('datasetSelect').value;
    const ds = DATASETS[key];
    if (!ds) return;
    document.getElementById('diDesc').textContent = ds.description;
    document.getElementById('diRec').textContent = ds.recommended;
    // 데이터 로드는 하지 않음!
}

// 2단계: 로드 버튼 클릭 시 실행 (onclick)
function loadSelectedDataset() {
    const key = document.getElementById('datasetSelect').value;
    if (key === '_random') { generateRandomData(); return; }

    const ds = DATASETS[key];
    if (!ds) return;

    csvHeaders = ds.headers;
    rawCsvData = ds.data.map(row => {
        const obj = {};
        ds.headers.forEach((h, idx) => { obj[h] = row[idx]; });
        return obj;
    });
    columnLabels = {};
    ds.headers.forEach((h, idx) => { columnLabels[h] = ds.headerLabels[idx]; });

    populateVarSelects(ds.headers);

    // defaultTarget 자동 선택
    const targetIdx = ds.headerLabels.indexOf(ds.defaultTarget);
    if (targetIdx >= 0) {
        document.getElementById('selectY').value = ds.headers[targetIdx];
    }

    document.getElementById('varSelectGroup').classList.remove('hidden');
    applyVariableSelection();
}
```

### 8.4 전환 판단 기준

| 조건 | 선택 방식 |
|------|:---------:|
| 데이터셋 ≤ 6개, 모두 친숙한 항목 | 1단계 (즉시 로드) |
| 데이터셋 > 6개, 또는 메타 정보가 중요 | 2단계 (미리보기 → 로드) |

## 9. 하위 호환 래퍼 패턴

### 9.1 문제

기존 함수명을 제거하면 HTML `onclick` 속성이나 다른 JS 함수의 호출에서 `ReferenceError`가 발생한다.

### 9.2 원칙

```
구 함수명은 래퍼로 유지하되, 내부는 신규 함수로 위임한다.
```

### 9.3 래퍼 작성 템플릿

```javascript
// ── 신규 함수 (실제 로직) ──
function loadSelectedDataset() { /* ... 전체 구현 ... */ }

// ── 호환 래퍼 1: 구 함수 → 신규 함수 위임 ──
function generateData() {
    loadSelectedDataset();
}

// ── 호환 래퍼 2: 인자 있는 구 함수 → 드롭다운 연동 ──
function loadBuiltinDataset(key) {
    if (DATASETS[key]) {
        document.getElementById('datasetSelect').value = key;
        onDatasetChange();
        loadSelectedDataset();
    }
}
```

### 9.4 래퍼 필요 여부 판단

```bash
# 기존 함수의 모든 호출처 확인 (함수 정의 제외)
grep -n "generateData\b" 파일.html | grep -v "function generateData"
grep -n "loadBuiltinDataset\b" 파일.html | grep -v "function loadBuiltinDataset"
```

| 결과 | 조치 |
|------|------|
| HTML `onclick`에서 호출됨 | 래퍼 필수 |
| 다른 JS 함수에서 호출됨 | 래퍼 또는 호출처 수정 |
| 호출처 0건 | 래퍼 불필요, 삭제 |

## 10. 안전한 코드 제거 절차

### 10.1 3종류의 제거 대상

| 유형 | 예시 | 위험도 |
|------|------|:------:|
| 전역 변수 | `let currentPreset = 'housing';` | 중 |
| 함수 | `function loadPreset(key) { ... }` | 높 |
| CSS 클래스 | `.preset-btn { ... }` | 낮 |

### 10.2 안전 제거 절차

```
[1단계] 제거 대상의 모든 참조 검색
        grep -n "currentPreset" 파일.html

[2단계] 참조가 0건 → 안전하게 제거 가능

[3단계] 참조가 N건 → 각 참조의 맥락 확인
        └─ 해당 코드도 함께 리팩토링 필요

[4단계] 제거 후 재검색으로 완전 제거 확인
        grep -c "currentPreset" 파일.html  # 기대: 0
```

### 10.3 DOM 요소 조건부 접근

UI 구조 변경으로 특정 DOM 요소가 항상 존재하지 않게 되면, `getElementById` + `addEventListener`에서 null 에러가 발생한다.

```javascript
// ❌ 항상 존재한다고 가정
function initSliders() {
    document.getElementById('dataCount').addEventListener('input', (e) => { ... });
}

// ✅ null 체크 선행
function initSliders() {
    const dataCount = document.getElementById('dataCount');
    if (dataCount) {
        dataCount.addEventListener('input', (e) => { ... });
    }
}
```

**적용 대상**:
- 탭/모드에 따라 조건부 렌더링되는 요소
- `hidden` 클래스로 숨겨졌다가 나타나는 요소의 자식
- 특정 선택지에서만 생성되는 동적 요소

## 11. 부수 작업 체크리스트

대규모 리팩토링 후 **메인 작업은 완료했지만 놓치기 쉬운** 부수 작업 목록이다.

### 11.1 필수 부수 작업 6가지

| # | 부수 작업 | 적용 조건 | 미수행 시 증상 |
|:-:|----------|----------|---------------|
| 1 | `resetAll()`에 신규 UI 숨김/초기화 추가 | UI 요소를 추가하거나 이동한 경우 | 초기화 후 UI 잔상 |
| 2 | `initSliders()`에 null 체크 추가 | DOM 요소 존재 조건이 변경된 경우 | 페이지 로드 시 에러 |
| 3 | `DOMContentLoaded`에 초기화 호출 추가 | 새 컨트롤이 페이지 로드 시 상태를 필요로 하는 경우 | 초기 상태 미설정 |
| 4 | 구 변수/함수의 참조 검색 후 제거 | 함수/객체를 리팩토링한 경우 | 데드 코드 잔존 |
| 5 | 제거된 CSS 클래스의 HTML 참조 검색 | CSS 클래스를 삭제/이름 변경한 경우 | 스타일 미적용 |
| 6 | 버전 배지 업데이트 | 모든 변경 | 버전 불일치 |

### 11.2 수행 순서

```
[메인 리팩토링 완료]
        ↓
[1] resetAll() 점검 — 신규/이동 UI가 초기화되는가?
        ↓
[2] initSliders() 점검 — 조건부 DOM에 null 체크가 있는가?
        ↓
[3] DOMContentLoaded 점검 — 새 컨트롤의 초기 상태가 설정되는가?
        ↓
[4] 데드 코드 검색 — 구 함수/변수가 남아있지 않은가?
        ↓
[5] CSS-HTML 정합성 — 제거된 클래스가 HTML에 남아있지 않은가?
        ↓
[6] 버전 배지 — 모든 버전 표기가 갱신되었는가?
```
