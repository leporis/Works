| 항목 | v1.2.0.0 (변경 전) | 문제점 |

|------|-------------------|----------|
| 데이터 분할 | ✅ 80:20 자동 분할 | 분할 개념 학습 |
| 학습 | 훈련 데이터만 사용 | 올바른 학습 과정 |
| 검증 | 테스트 데이터로 MSE | 일반화 성능 평가 |

```
전체 데이터
    │
    ▼ [Fisher-Yates 셔플 → 임의 분할]

    │                     │
    ▼ 경사하강법(trainData) │
    │                     │
회귀선 (w, b)             │
    │                     ▼
    └──────────────► 테스트 20%로 검증
                          │
                          ▼
                    테스트 MSE 평가
                          │
              ┌───────────┴───────────┐
              ▼                       ▼
           만족 → 완료          불만족 → [🔀 재분할 & 재학습]
```

### 3.4 데이터 분할 로직

```javascript
function splitData() {
    if (data.length < 5) {
        trainData = [...data];
        testData = [];
        return;
    }

    // Fisher-Yates 셔플
    const indices = data.map((_, i) => i);
    for (let i = indices.length - 1; i > 0; i--) {
        const j = Math.floor(Math.random() * (i + 1));
        [indices[i], indices[j]] = [indices[j], indices[i]];
    }

    const splitPoint = Math.floor(data.length * 0.8);
    trainData = indices.slice(0, splitPoint).map(i => data[i]);
    testData = indices.slice(splitPoint).map(i => data[i]);
}
```

### 3.5 핵심 원칙: 정규화는 훈련 데이터만으로

```javascript
function calculateNormParams() {
    if (trainData.length === 0) return;
    // 정규화 파라미터는 훈련 데이터만으로 계산
    const xValues = trainData.map(d => d.x);
    const yValues = trainData.map(d => d.y);
    normParams.x = { mean: mean(xValues), std: std(xValues) };
    normParams.y = { mean: mean(yValues), std: std(yValues) };
}
```

## 6. 교육적 의의

### 6.1 훈련/테스트 분리의 필요성
- 학생들이 "데이터를 나누는 이유"를 시각적으로 이해
- 훈련 MSE와 테스트 MSE의 차이를 그래프로 관찰

### 6.2 과적합 탐지
- 손실 그래프에서 훈련 MSE는 감소하지만 테스트 MSE가 증가하는 패턴 관찰 가능
- 에포크 수, 학습률에 따른 과적합 경험

### 6.3 재분할의 효과
- 같은 데이터를 다르게 분할하면 결과가 달라짐을 경험
- 데이터 분할의 무작위성과 모델 성능의 관계 이해
