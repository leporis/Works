## 5. 데이터 구조 리팩토링 패턴

### 5.1 이원화 구조를 단일 구조로 통합

**문제**: 동일 목적(데이터 제공)의 객체가 2개로 분리되면 코드 중복과 UI 분산 발생.

```
[문제 구조]
presets (합성 생성기)         builtinDatasets (실제 데이터)
  ├─ loadPreset()               ├─ loadBuiltinDataset()
  ├─ generateData()             ├─ CSV 경로 재사용
  └─ 프리셋탭 UI                └─ CSV탭 내부 UI
  = 2개 로드 경로, 2개 UI, 로직 중복
```

**원칙**:

| # | 원칙 | 설명 |
|:-:|------|------|
| 1 | **동질 데이터 통합** | 동일 형태(headers+data)의 데이터는 하나의 객체로 |
| 2 | **이질 로직 분리** | 합성 생성기는 별도 객체로 명확히 분리 |
| 3 | **로드 경로 단일화** | 통합된 데이터는 하나의 로드 함수가 처리 |

**적용 전 → 후**:

```javascript
// ❌ 적용 전: 2개 객체, 2개 로드 함수
const presets = {
    housing: { xRange: [20,150], slope: 15, noise: 100 },  // 생성 파라미터
};
const builtinDatasets = {
    iris: { headers: [...], data: [...] },                   // 실제 데이터
};
function loadPreset(key) { /* presets 사용 */ }
function loadBuiltinDataset(key) { /* builtinDatasets 사용 */ }

// ✅ 적용 후: 역할별 분리, 실데이터 통합
const DATASETS = {           // 실데이터 N개 통합
    iris: { headers: [...], headerLabels: [...], data: [...] },
    penguins: { ... },       // ... 23개
};
const RANDOM_PRESETS = {     // 생성기 전용 분리
    housing: { xRange: [20,150], slope: 15, noise: 100 },
};
function loadSelectedDataset() { /* DATASETS 사용 */ }
function generateRandomData() { /* RANDOM_PRESETS 사용 */ }
```

**적용 판단 기준**:

| 조건 | 조치 |
|------|:----:|
| 2개 이상 데이터 객체가 동일한 형태(headers+data)를 갖는다 | 통합 |
| 데이터 제공 방식이 근본적으로 다르다 (생성 vs 내장) | 분리 |
| 하나의 객체에 생성 파라미터와 실데이터가 혼재한다 | 분리 후 통합 |

### 5.2 통합 데이터셋 객체 표준 구조

```javascript
const DATASETS = {
    'dataset_key': {
        // --- 메타 정보 (UI 표시용) ---
        name: '🌸 붓꽃(Iris)',                                // 이모지 + 한글명(영문명)
        source: 'Fisher(1936), UCI ML Repository | CC BY 4.0',  // 출처 + 라이선스
        description: '붓꽃 150송이의 꽃받침·꽃잎 크기 측정값',    // 1줄 설명
        defaultTarget: '꽃잎길이(cm)',                          // headerLabels 중 기본 Y
        recommended: '꽃잎길이 → 꽃잎너비 (r=0.96)',            // 추천 변수 조합

        // --- 데이터 구조 ---
        headers: ['c0', 'c1', 'c2', 'c3'],                    // 내부 키 (짧게)
        headerLabels: ['꽃받침길이(cm)', '꽃받침너비(cm)',        // 한글 표시명 (단위 포함)
                       '꽃잎길이(cm)', '꽃잎너비(cm)'],
        data: [[5.1, 3.5, 1.4, 0.2], ...]                     // 행 기반 2D 배열
    }
};
```

**필수 규칙**:

| # | 규칙 | 이유 |
|:-:|------|------|
| 1 | `headers`와 `headerLabels` 길이 동일 | 인덱스 매핑 의존 |
| 2 | `headers`는 짧은 내부 키 (`c0`, `c1`...) | 파일 크기 절감, JS 키로 사용 |
| 3 | `headerLabels`에 단위 포함 | 축 레이블에 그대로 사용 |
| 4 | `defaultTarget`은 `headerLabels` 값과 정확히 일치 | `indexOf()`로 매칭 |
| 5 | `data`는 숫자만 (문자열 컬럼 사전 제거) | 수치형 검증 생략 가능 |
| 6 | `source`에 라이선스 명시 | 교육용 배포 시 저작권 준수 |

## 6. UI 확장 패턴

### 6.1 셀렉터 UI 전환 패턴

**문제**: 선택지가 증가하면 기존 버튼 그리드 UI가 화면 공간 초과.

**전환 기준**:

| 항목 수 | 적합한 UI | HTML 구조 |
|:-------:|----------|----------|
| 2~6개 | 버튼 그리드 | `<button class="preset-btn">` |
| 7~15개 | 드롭다운 | `<select>` + `<option>` |
| 16개 이상 | 카테고리 드롭다운 | `<select>` + `<optgroup>` |

**적용 전 → 후**:

```html

<div class="preset-grid">

        <option value="iris">🌸 붓꽃(Iris) - 150개</option>
        <option value="penguins">🐧 펭귄(Penguins) - 333개</option>

    <optgroup label="📈 경제/비즈니스">

    </optgroup>
```

**CSS 전환 대응표**:

| 제거 CSS | 추가 CSS |
|----------|----------|
| `.preset-grid` (grid 레이아웃) | `.dataset-selector` (select 스타일) |
| `.preset-btn` + `:hover` + `.active` | `.dataset-info-box` (정보 표시) |
| `.current-vars` (변수 표시) | `.random-controls` (조건부 컨트롤) |

### 6.2 정보 박스 패턴

**문제**: 항목이 많아지면 선택 전 내용 미리 확인 수단이 필요.

**패턴**: 드롭다운 선택 시 정보 박스를 동적 업데이트 → "선택"과 "로드" 분리.

```html
<div class="dataset-info-box" id="datasetInfoBox">
    <div><strong>출처:</strong> <span id="diSource">선택하세요</span></div>
    <div><strong>설명:</strong> <span id="diDesc"></span></div>
    <div><strong>추천:</strong> <span id="diRec"></span></div>
</div>
```

**원칙**: 사용자가 정보를 확인한 후 명시적으로 "로드" 버튼을 누르는 **2단계 흐름** 구성. 선택 = 미리보기, 버튼 = 실행.

### 6.3 조건부 UI 토글 패턴

**문제**: 하나의 드롭다운에서 일반 항목과 특수 항목(랜덤 생성 등)을 함께 제공할 때, 선택에 따라 다른 UI 표시.

**패턴**: `classList.toggle(className, condition)`

```javascript
function onDatasetChange() {
    const key = document.getElementById('datasetSelect').value;
    const isRandom = (key === '_random');

    // toggle(class, forceAdd): true면 add, false면 remove
    document.getElementById('datasetInfoBox').classList.toggle('hidden', isRandom);
    document.getElementById('randomControls').classList.toggle('hidden', !isRandom);

    if (!isRandom && DATASETS[key]) {
        document.getElementById('diSource').textContent = DATASETS[key].source;
        document.getElementById('diDesc').textContent = DATASETS[key].description;
        document.getElementById('diRec').textContent = DATASETS[key].recommended;
    }
}
```

**원칙**: `classList.toggle(class, condition)`으로 한 줄 처리. `if/else`로 `add`/`remove`를 분기하는 것보다 간결하고 실수가 적다.

### 7.1 탭 종속 UI를 공유 영역으로 추출

**문제**: 여러 데이터 소스가 공통으로 사용하는 UI가 특정 탭 내부에 종속되어 다른 탭에서 접근 불가.

**적용 판단**: 해당 UI를 2개 이상 탭/기능에서 접근해야 하면 추출 대상.

```
[❌ 적용 전: CSV탭 내부 종속]
┌─ 탭 컨테이너 ──────────────────────────┐
```

## 8. 함수 리팩토링 패턴

### 8.1 선택-로드 2단계 분리

**문제**: "선택 = 즉시 로드"를 "선택(미리보기) → 명시적 로드"로 분리해야 한다.

```javascript
// ❌ 적용 전: 선택 = 즉시 실행 (1단계)
function loadPreset(key) {
    currentPreset = key;
    varLabels = { x: presets[key].xLabel, y: presets[key].yLabel };
}

// ✅ 적용 후: 선택(정보 표시) → 명시적 로드 (2단계)
function onDatasetChange() {               // 1단계: 선택 시 정보만
    const ds = DATASETS[document.getElementById('datasetSelect').value];
    document.getElementById('diDesc').textContent = ds.description;
}
function loadSelectedDataset() {           // 2단계: 로드 버튼 클릭 시
    const ds = DATASETS[document.getElementById('datasetSelect').value];
    csvHeaders = ds.headers;
    rawCsvData = ds.data.map(/* ... */);
    populateVarSelects(ds.headers);
    applyVariableSelection();
}
```

**원칙**: 데이터셋이 많을수록 "선택"과 "실행" 분리 필요. 실수 방지 + 탐색 촉진.

### 8.2 하위 호환 래퍼 패턴

**문제**: 기존 함수명을 제거하면 HTML `onclick` 등에서 에러 발생.

**원칙**: 구 함수명은 래퍼로 유지, 내부는 신규 함수로 위임.

```javascript
// 신규 함수 (실제 로직)
function loadSelectedDataset() { /* ... */ }

// 호환 래퍼 1: generateData → loadSelectedDataset
function generateData() { loadSelectedDataset(); }

// 호환 래퍼 2: loadBuiltinDataset → DATASETS 검색 후 로드
function loadBuiltinDataset(key) {
    if (DATASETS[key]) {
        document.getElementById('datasetSelect').value = key;
        onDatasetChange();
        loadSelectedDataset();
    }
}
```

**적용 판단**:

| 조건 | 조치 |
|------|:----:|
| HTML `onclick`에서 기존 함수를 호출 | 래퍼 필수 |
| 다른 JS 함수에서 기존 함수를 호출 | 래퍼 또는 호출처 수정 |
| 호출처 없음 (`grep` 확인) | 래퍼 불필요, 삭제 |

**검증**: `grep -n "함수명" 파일.html | grep -v "function "` → 호출처 존재 여부 확인

### 8.3 불필요 상태 변수 제거

**문제**: 리팩토링 후 참조되지 않는 전역 변수 잔존 → 혼란 유발.

**절차**:

| 단계 | 작업 | 명령 |
|:----:|------|------|
| 1 | 모든 참조 검색 | `grep -n "currentPreset" 파일.html` |
| 2 | 참조 0건이면 선언부 제거 | `let currentPreset = 'housing';` 삭제 |
| 3 | 참조 남아있으면 해당 코드도 수정 | 호출처 리팩토링 |

**원칙**: 함수 리팩토링 후 **반드시** 전역 변수 참조 검사 수행. "선언만 있고 사용처 없음 = 데드 코드".

### 8.4 조건부 DOM 요소 안전 접근

**문제**: UI 구조 변경으로 특정 DOM 요소가 항상 존재하지 않게 되면 `addEventListener`에서 null 에러.

```javascript
// ❌ 적용 전: 항상 존재한다고 가정
function initSliders() {
    document.getElementById('dataCount').addEventListener('input', (e) => { ... });
}

// ✅ 적용 후: null 체크 선행
function initSliders() {
    const dataCount = document.getElementById('dataCount');
    if (dataCount) {
        dataCount.addEventListener('input', (e) => { ... });
    }
}
```

**적용 대상**: 탭/모드에 따라 조건부 렌더링되는 요소, `hidden`으로 숨겨졌다가 나타나는 요소, 특정 선택지에서만 생성되는 요소.

```python
# 입력: presets_data.json (컬럼 기반)
# 출력: datasets_block.js (행 기반 JS 객체)

columns = list(ds['columns'].keys())
n_rows = len(ds['columns'][columns[0]])
rows = [[ds['columns'][col][i] for col in columns] for i in range(n_rows)]

# 2. 내부 키 생성

# 3. 압축 포맷 (N 기준)
per_line = 3 if n_rows <= 30 else 5
```
