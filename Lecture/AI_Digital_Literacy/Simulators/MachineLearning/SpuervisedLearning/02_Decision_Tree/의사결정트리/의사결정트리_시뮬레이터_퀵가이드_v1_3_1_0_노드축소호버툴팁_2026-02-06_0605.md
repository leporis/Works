## 🚀 빠른 시작 (3단계)

### 1️⃣ 데이터셋 선택
```
헤더 드롭다운에서 데이터셋 선택 (🐧 펭귄 또는 🌺 붓꽃)
→ 또는 [📄 CSV 업로드] 버튼으로 직접 데이터 업로드
```

### 2️⃣ 트리 생성
```
[🧠 학습 모드] → 특성 체크 → 깊이/최소샘플 조절 → [🌳 트리 생성] 클릭
```
- 트리가 Canvas에 축소 노드로 표시됨 (마우스를 올리면 상세 정보 확인)
- 산점도에 결정 경계가 자동 표시됨
- 혼동행렬과 특성 중요도도 함께 계산됨

### 3️⃣ 추론 체험
```
[🔮 추론 모드] 클릭 → 테스트 데이터 선택 또는 슬라이더로 직접 입력
→ 트리를 따라가는 추론 경로가 노란색으로 강조됨
```

## 📄 CSV 업로드 방법 (v1.3.0.0)

### 업로드 절차
1. **[📄 CSV 업로드]** 버튼 클릭
2. CSV 파일 선택 → **미리보기 모달** 표시 (5행)
3. **분류 열** 드롭다운에서 클래스 열 선택
4. **[✅ 적용]** 클릭

### CSV 파일 요구사항

| 항목 | 조건 |
|------|------|
| 형식 | 쉼표(,) 구분 CSV |
| 헤더 | 첫 번째 행 = 열 이름 |
| 최소 행 | 5행 이상 |
| 숫자 특성 | 1개 이상의 숫자 열 필요 |
| 클래스 열 | 분류 대상이 되는 열 지정 |

### 업로드 후
- 숫자 열 → 자동으로 특성(feature)으로 인식
- 클래스별 이모지/색상 자동 할당
- 드롭다운에 "📄 CSV: 파일명" 표시
- ❌ 버튼으로 CSV 데이터셋 삭제 가능

## 🔍 트리 노드 호버 (v1.3.1.0)

트리 노드는 축소 표시되며, **마우스를 올리면 상세 정보**가 툴팁으로 나타납니다.

### 분기 노드 (조건 노드)
| 정보 | 설명 |
|------|------|
| 조건 | 분기에 사용된 특성과 임계값 |
| 샘플 수 | 해당 노드에 도달한 데이터 수 |
| 지니 불순도 | 데이터 혼합 정도 (0=순수) |
| 클래스 분포 | 각 클래스별 데이터 수 |

### 리프 노드 (결과 노드)
| 정보 | 설명 |
|------|------|
| 예측 클래스 | 최종 분류 결과 |
| 샘플 수 | 해당 리프에 도달한 데이터 수 |
| 신뢰도 | 예측 클래스 비율 (%) |
| 클래스 분포 | 각 클래스별 데이터 수 |

## 📊 결과 해석

### 혼동행렬 읽는 법
- **대각선** (초록): 맞게 분류한 수
- **비대각선** (빨강): 잘못 분류한 수
- 대각선 수가 많을수록 좋은 모델!

### 정확도 기준
| 상태 | 정확도 | 의미 |
|------|--------|------|
| ✅ 우수 | 90% 이상 | 분류를 아주 잘 해요! |
| ⚠️ 양호 | 70~90% | 꽤 괜찮지만 개선 여지 있음 |
| ❌ 부족 | 70% 미만 | 특성이나 설정을 바꿔보세요 |

### 특성 중요도
- 막대가 길수록 분류에 큰 영향을 미치는 특성
- 데이터셋마다 중요한 특성이 다름!

### 산점도 + 결정 경계
- **색상 영역**: 트리가 각 영역을 어떤 클래스로 분류하는지 표시
- **● 원**: 학습 데이터
- **◇ 다이아몬드**: 테스트 데이터
- X/Y축 드롭다운으로 특성 조합 변경 가능

## 🔄 데이터셋별 비교 팁 (v1.3.0.0)

### 🐧 펭귄 데이터
- 134개 샘플, 3종 (Adelie, Chinstrap, Gentoo)
- 부리 길이/깊이가 종 분류에 중요한 특성
- 체중만으로는 분류가 어려움 → 여러 특성 조합의 중요성 학습

### 🌺 붓꽃 데이터
- 150개 샘플, 3종 (Setosa, Versicolor, Virginica)
- 꽃잎 길이/너비가 가장 중요한 특성
- Setosa는 쉽게 분류되지만, Versicolor/Virginica 구분은 어려움

### 비교 실습
1. 같은 설정(깊이 4, 전체 특성)으로 양쪽 트리 생성
2. 트리 구조 비교: 어떤 특성이 루트 노드인가?
3. 특성 중요도 비교: 데이터마다 중요한 특성이 다른 이유는?
4. 결정 경계 비교: 모양이 어떻게 다른가?

## 🎯 실습 활동 예시

### 활동 1: 깊이 실험
1. 최대 깊이 1로 트리 생성 → 정확도 확인
2. 깊이 2, 4, 8로 차례로 변경 → 정확도 변화 관찰
3. 깊이가 깊으면 항상 좋을까? 토론해보세요!

### 활동 2: 특성 선택 실험
1. 특성 1개만 선택 → 가장 높은 정확도의 특성 찾기
2. 특성 2개 조합 → 어떤 조합이 최고인지 탐색
3. 특성 중요도 그래프와 비교해보세요!

### 활동 3: 데이터셋 비교
1. 펭귄 데이터로 트리 생성 → 구조와 정확도 기록
2. 붓꽃 데이터로 전환 → 같은 설정으로 트리 생성
3. 두 트리의 차이점은? 어떤 데이터가 분류하기 쉬운가?

### 활동 4: CSV 데이터 도전 (심화)
1. 학급에서 수집한 데이터를 CSV로 준비
2. 시뮬레이터에 업로드 → 어떤 특성이 분류에 중요한지 탐색
3. AI가 실제 데이터를 어떻게 학습하는지 체험!

## ❓ 자주 묻는 질문

**Q: 데이터셋을 전환하면 이전 트리가 사라져요**
→ 정상입니다. 데이터가 바뀌면 트리도 새로 만들어야 해요. [🌳 트리 생성]을 다시 클릭하세요.

**Q: CSV 업로드가 안 돼요**
→ CSV 파일인지 확인하세요. 쉼표로 구분된 파일이어야 하고, 첫 줄은 열 이름, 최소 5행이 필요합니다.

**Q: 정확도가 갑자기 낮아졌어요**
→ 특성을 너무 적게 선택하거나, 최대 깊이를 1로 설정하면 정확도가 떨어질 수 있어요. 설정을 조절해보세요.

**Q: 산점도가 안 보여요**
→ 특성을 2개 이상 선택해야 산점도가 표시됩니다. 체크박스를 확인하세요.

**Q: 추론 경로가 안 보여요**
→ [🔮 추론 모드]로 전환한 후, 테스트 데이터를 선택하면 노란색 경로가 표시됩니다.

**Q: 애니메이션이 너무 빨라요/느려요**
→ 속도 슬라이더를 조절하세요. 왼쪽이 빠름(100ms), 오른쪽이 느림(1500ms)입니다.

## 🔑 핵심 용어

| 용어 | 의미 |
|------|------|
| **의사결정트리** | 조건에 따라 가지를 나눠서 분류하는 알고리즘 |
| **특성 (Feature)** | 데이터의 속성 (예: 부리 길이, 꽃잎 너비) |
| **클래스 (Class)** | 분류 결과 (예: Adelie, Setosa) |
| **지니 불순도** | 데이터가 얼마나 섞여있는지 나타내는 수치 (0=순수, 0.5=혼합) |
| **분기 조건** | 트리의 각 노드에서 데이터를 나누는 기준 |
| **리프 노드** | 트리의 끝 노드, 최종 분류 결과를 담고 있음 |
| **최대 깊이** | 트리가 얼마나 깊이 가지를 칠 수 있는지의 한계 |
| **혼동행렬** | 실제 클래스 vs 예측 클래스를 비교하는 표 |
| **결정 경계** | 트리가 각 클래스를 나누는 2D 경계선 |
| **특성 중요도** | 각 특성이 분류에 얼마나 기여하는지의 비율 |
| **과적합** | 학습 데이터에 너무 맞춰져서 새 데이터에 약해지는 현상 |
