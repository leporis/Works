# 의사결정트리 머신러닝 시뮬레이터 이론해설서

**문서 버전: v1.0.1.0**

## 1. 의사결정트리란?

### 1.1 일상 속 의사결정트리

우리는 매일 의사결정트리를 사용합니다. 예를 들어, "오늘 우산을 가져갈까?"를 판단할 때:

```
        비가 올까?
       /         \
     예           아니오
    /               \
 우산 가져감      구름 많은가?
                /         \
              예           아니오
             /               \
        접이식 우산         안 가져감
```

의사결정트리는 **질문의 연속**으로 결론에 도달하는 방법입니다. 머신러닝에서는 컴퓨터가 데이터를 보고 스스로 "어떤 질문을, 어떤 순서로 할지"를 학습합니다.

### 1.2 머신러닝에서의 의사결정트리

| 항목 | 일상의 판단 | 머신러닝 의사결정트리 |
|------|------------|---------------------|
| 질문 | 사람이 경험으로 결정 | 데이터에서 자동으로 학습 |
| 기준 | 직관적 판단 | 수학적 최적화 (지니 불순도) |
| 목표 | 좋은 판단 | 정확한 분류/예측 |
| 검증 | 결과 확인 | 테스트 데이터로 평가 |

## 2. 시뮬레이터의 데이터: Palmer Penguins

### 2.1 펭귄 3종의 특성

시뮬레이터는 남극 Palmer 군도의 펭귄 134마리 데이터를 사용합니다.

| 종 | 부리 길이 | 부리 깊이 | 날개 길이 | 체중 |
|:--:|:---------:|:---------:|:---------:|:----:|
| Adelie | 33~46mm | 16~22mm | 172~198mm | 2700~4775g |
| Chinstrap | 41~53mm | 16~20mm | 189~212mm | 2700~4550g |
| Gentoo | 41~60mm | 13~18mm | 203~230mm | 3950~6300g |

### 2.2 왜 펭귄 데이터인가?

1. **3종 분류**: 2종보다 복잡하여 트리 구조가 풍부합니다
2. **특성 간 차이**: 종마다 뚜렷한 특성 차이가 있어 분류가 의미있습니다
3. **일부 겹침**: Adelie와 Chinstrap의 특성이 일부 겹쳐 완벽한 분류가 어렵습니다 — 이것이 교육적으로 가치 있습니다

### 2.3 분류 문제란?

```
입력 (특성값)         →  알고리즘  →  출력 (종 예측)

부리 길이: 39.1mm
부리 깊이: 18.7mm     →  의사결정트리  →  "Adelie"
날개 길이: 181mm
체중: 3750g
```

여러 특성값을 보고 정해진 카테고리(종) 중 하나를 맞추는 것이 **분류(Classification)** 문제입니다.

## 3. CART 알고리즘

### 3.1 CART란?

CART(Classification and Regression Trees)는 가장 널리 사용되는 의사결정트리 알고리즘입니다. 시뮬레이터는 이 알고리즘을 사용합니다.

핵심 아이디어는 **이진 분할(Binary Split)**: 매 단계에서 데이터를 딱 두 그룹으로 나누는 것입니다.

```
전체 데이터 (107마리)
    ↓
"부리 깊이 ≤ 15.6mm?"
    ↓              ↓
  예 (45마리)    아니오 (62마리)
  → 대부분 Gentoo   → Adelie 또는 Chinstrap
```

### 3.2 어떤 질문이 "좋은" 질문인가?

좋은 질문이란 **데이터를 가장 깨끗하게 나누는 질문**입니다.

나쁜 분할과 좋은 분할을 비교해봅시다:

```
나쁜 분할:                         좋은 분할:
┌──────────┐ ┌──────────┐        ┌──────────┐ ┌──────────┐
│ 🔴🔴🔴  │ │ 🟢🔵🔴  │        │ 🔴🔴🔴  │ │ 🟢🟢🔵  │
│ 🟢🔵🟢  │ │ 🔴🟢🔵  │        │ 🔴🔴🔴  │ │ 🔵🔵🔵  │
└──────────┘ └──────────┘        └──────────┘ └──────────┘
  여전히 섞여 있음 (불순)           한쪽은 거의 깨끗 (순수)
```

이 "깨끗함"을 수학적으로 측정하는 것이 **지니 불순도**입니다.

## 4. 지니 불순도 (Gini Impurity)

### 4.1 개념

지니 불순도는 "이 그룹에서 임의로 하나를 뽑았을 때, 잘못 분류할 확률"을 나타냅니다.

### 4.2 수식

```
Gini(S) = 1 - Σᵢ pᵢ²

여기서 pᵢ = (클래스 i에 속하는 데이터 수) / (전체 데이터 수)
```

### 4.3 계산 예시

**예시 1**: Gentoo만 있는 그룹 (44마리 중 44마리가 Gentoo)

```
p(Gentoo) = 44/44 = 1.0
p(Adelie) = 0/44 = 0
p(Chinstrap) = 0/44 = 0

Gini = 1 - (1.0² + 0² + 0²) = 1 - 1 = 0
```

→ **Gini = 0**: 완전히 순수한 그룹 (한 종만 존재)

**예시 2**: 3종이 균등한 그룹 (각 10마리씩)

```
p(Adelie) = 10/30 = 0.333
p(Chinstrap) = 10/30 = 0.333
p(Gentoo) = 10/30 = 0.333

Gini = 1 - (0.333² + 0.333² + 0.333²)
     = 1 - (0.111 + 0.111 + 0.111)
     = 1 - 0.333
     = 0.667
```

→ **Gini ≈ 0.667**: 최대 불순도 (3종 균등 혼합)

**예시 3**: Adelie 40마리, Chinstrap 5마리인 그룹

```
p(Adelie) = 40/45 = 0.889
p(Chinstrap) = 5/45 = 0.111

Gini = 1 - (0.889² + 0.111²)
     = 1 - (0.790 + 0.012)
     = 0.198
```

→ **Gini ≈ 0.198**: 비교적 순수한 그룹

### 4.4 지니 불순도의 범위

| 3종 분류 기준 | Gini 값 | 의미 |
|:---:|:-------:|------|
| 0 | 완전 순수 | 한 종만 존재 |
| 0.5 | 2종 균등 | 두 종이 반반 |
| 0.667 | 3종 균등 | 세 종이 균등 — 최대 불순도 |

## 5. 가중 지니 불순도 (최적 분할 선택)

### 5.1 개념

분할 후 두 그룹의 지니 불순도를 **데이터 수에 비례**하여 합산한 것입니다.

### 5.2 수식

```
가중 Gini = (|왼쪽| / |전체|) × Gini(왼쪽) + (|오른쪽| / |전체|) × Gini(오른쪽)
```

### 5.3 계산 예시

전체 107마리를 "부리 깊이 ≤ 15.6mm"로 나눈다면:

| 그룹 | 데이터 수 | Adelie | Chinstrap | Gentoo | Gini |
|------|:---------:|:------:|:---------:|:------:|:----:|
| 왼쪽 (≤15.6) | 45 | 0 | 1 | 44 | 0.044 |
| 오른쪽 (>15.6) | 62 | 40 | 22 | 0 | 0.460 |

```
가중 Gini = (45/107) × 0.044 + (62/107) × 0.460
          = 0.421 × 0.044 + 0.579 × 0.460
          = 0.019 + 0.266
          = 0.285
```

### 5.4 최적 분할 탐색

CART 알고리즘은 모든 특성의 모든 가능한 임계값에 대해 가중 지니를 계산하고, **가장 낮은 가중 지니**를 주는 (특성, 임계값) 쌍을 선택합니다.

```
시도 1: 부리길이 ≤ 42.0  → 가중 Gini = 0.512
시도 2: 부리깊이 ≤ 15.6  → 가중 Gini = 0.285  ← 최소!
시도 3: 날개길이 ≤ 206.5 → 가중 Gini = 0.298
시도 4: 체중 ≤ 4300     → 가중 Gini = 0.445
...

→ "부리깊이 ≤ 15.6" 선택!
```

## 6. 트리 구축 과정

### 6.1 재귀적 분할

CART는 **재귀적으로** 트리를 만듭니다. 각 노드에서:

1. 최적 분할을 찾는다
2. 데이터를 두 그룹으로 나눈다
3. 각 그룹에서 1~2를 반복한다
4. 종료 조건을 만족하면 리프 노드로 확정한다

```
전체 107마리
    │
    ▼ (부리깊이 ≤ 15.6?)
  ┌─┴──────────────────┐
  ▼                    ▼
45마리 (대부분 G)     62마리 (A + C)
  │                    │
  ▼ (최적 분할 탐색)   ▼ (최적 분할 탐색)
  ...                  ...
```

### 6.2 종료 조건

시뮬레이터에서 트리 분할이 멈추는 조건은 3가지입니다:

| 조건 | 시뮬레이터 설정 | 의미 |
|------|:-------------:|------|
| 최대 깊이 도달 | 사용자 설정 (1~8) | 트리가 설정된 깊이에 도달 |
| 최소 샘플 수 미만 | 사용자 설정 (1~10) | 노드의 데이터가 너무 적음 |
| 지니 불순도 ≤ 0.01 | 고정 | 이미 거의 순수한 그룹 |

### 6.3 리프 노드 결정

리프 노드에서의 분류 결과는 **다수결(Majority Vote)**로 결정됩니다. 해당 노드에 가장 많은 종이 예측값이 됩니다.

```
리프 노드에 Adelie 38, Chinstrap 2 → 예측: Adelie
```

## 7. 데이터 분할: 학습과 테스트

### 7.1 왜 데이터를 나누는가?

시험공부에 비유하면:

| 비유 | 머신러닝 |
|------|---------|
| 교과서로 공부 | 학습 데이터로 트리 생성 |
| 모의고사로 실력 확인 | 테스트 데이터로 성능 평가 |
| 교과서 문제만 풀 수 있음 | 학습 데이터만 잘 맞춤 (과적합) |
| 처음 보는 문제도 풀 수 있음 | 테스트 데이터도 잘 맞춤 (일반화) |

### 7.2 시뮬레이터의 분할

| 항목 | 값 | 역할 |
|------|-----|------|
| 학습 데이터 | 107마리 (80%) | 트리를 만드는 데 사용 |
| 테스트 데이터 | 27마리 (20%) | 트리를 평가하는 데만 사용 |
| 시드 | 42 | 항상 같은 분할 보장 |

## 8. 과적합과 과소적합

### 8.1 과적합 (Overfitting)

학습 데이터의 세세한 패턴(노이즈)까지 모두 외워버려서, 새로운 데이터에서는 오히려 성능이 떨어지는 현상입니다.

```
과적합 신호:
• 학습 정확도: 100% (완벽!)
• 테스트 정확도: 85% (갑자기 하락)
  → 차이가 클수록 과적합 심각
```

트리가 너무 깊으면 (깊이 6~8) 학습 데이터의 개별 샘플까지 분류하는 규칙을 만들게 됩니다.

### 8.2 과소적합 (Underfitting)

모델이 너무 단순하여 데이터의 기본 패턴도 제대로 학습하지 못하는 현상입니다.

```
과소적합 신호:
• 학습 정확도: 75%
• 테스트 정확도: 70%
  → 둘 다 낮음
```

트리 깊이가 1~2이면 3종을 구분할 규칙이 부족합니다.

### 8.3 적절한 복잡도

```
성능
 ↑
 │     ╭──────╮
 │    ╱        ╲  ← 테스트 정확도 (진짜 실력)
 │   ╱          ╲
 │  ╱    ╭───────────── 학습 정확도
 │ ╱   ╱
 │╱  ╱
 ├─┼────────────→ 트리 깊이
   ↑
   적절한 깊이
 (학습-테스트 차이 최소)
```

시뮬레이터의 깊이 실험에서 이 현상을 직접 관찰할 수 있습니다.

## 9. 혼동 행렬과 성능 평가

### 9.1 혼동 행렬이란?

테스트 데이터 27마리의 "실제 종"과 "예측 종"을 교차 비교한 표입니다.

```
                 예측
           Adelie  Chin  Gentoo
실 Adelie [  10     1      0  ]
제 Chin   [   0     5      0  ]
   Gentoo [   0     0     11  ]
```

### 9.2 읽는 법

- **대각선**: 정확히 맞춘 수 (10 + 5 + 11 = 26마리 정답)
- **비대각선**: 잘못 분류한 수 (Adelie 1마리를 Chinstrap으로 오분류)
- **정확도**: 26/27 = 96.3%

### 9.3 정확도 계산

```
정확도 = 대각선 합계 / 전체 데이터 수

학습 정확도 = 학습 데이터에서 맞춘 수 / 107
테스트 정확도 = 테스트 데이터에서 맞춘 수 / 27
```

## 10. 특성 중요도

### 10.1 어떤 특성이 중요한가?

의사결정트리에서 **루트 노드에 사용된 특성**이 가장 중요합니다. 전체 데이터를 가장 효과적으로 나누는 특성이기 때문입니다.

일반적으로 Palmer Penguins에서는:

| 순위 | 특성 | 이유 |
|:----:|------|------|
| 1 | 날개 길이 또는 부리 깊이 | Gentoo가 다른 종과 크게 구분됨 |
| 2 | 부리 길이 | Adelie와 Chinstrap 구분에 기여 |
| 3 | 체중 | 보조적 역할 |

### 10.2 시뮬레이터에서 확인하는 법

1. 특성을 하나씩만 선택하여 정확도 비교
2. 루트 노드의 분할 조건이 어떤 특성을 사용하는지 관찰
3. 학습된 규칙에서 자주 등장하는 특성 확인
