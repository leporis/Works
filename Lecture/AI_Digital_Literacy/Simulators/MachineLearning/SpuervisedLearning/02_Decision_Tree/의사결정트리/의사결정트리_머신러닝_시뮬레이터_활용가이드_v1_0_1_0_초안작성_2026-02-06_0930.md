# 의사결정트리 머신러닝 시뮬레이터 활용가이드

**문서 버전: v1.0.1.0**

---

## 1. 활용가이드 소개

### 1.1 목적

이 가이드는 의사결정트리 머신러닝 시뮬레이터를 수업 현장에서 효과적으로 활용하기 위한 **실습 활동 중심** 안내서입니다. 교수자가 바로 수업에 적용할 수 있도록 활동별 목표, 절차, 기록지, 토론 질문을 제공합니다.

### 1.2 대상

| 대상 | 설명 |
|------|------|
| 교수자 | 초등 예비교사 대상 AI/ML 교육 담당 교수 |
| 학습자 | 청주교대 '컴퓨팅사고와 AI교육' 수강 예비교사 |

### 1.3 선수 학습

| 항목 | 필수 여부 |
|------|:---------:|
| 머신러닝 기본 개념 (학습/예측) | 권장 |
| 분류 문제의 개념 | 권장 |
| 시뮬레이터 사용설명서 숙지 | 필수 |

---

## 2. 활동 1: 의사결정트리 첫 만남

### 2.1 활동 개요

| 항목 | 내용 |
|------|------|
| 목표 | 의사결정트리의 기본 구조(분기 노드, 리프 노드)를 이해한다 |
| 소요 시간 | 10분 |
| 준비물 | 시뮬레이터 (기본 설정 상태) |

### 2.2 활동 절차

**단계 1**: 시뮬레이터를 열고 기본 설정(깊이 4, 최소 샘플 2, 4개 특성 전체)을 확인합니다.

**단계 2**: [🌳 트리 생성] 버튼을 클릭합니다.

**단계 3**: 생성된 트리를 관찰하며 다음을 확인합니다.

| 확인 항목 | 기록 |
|-----------|------|
| 루트 노드의 분할 조건은 무엇인가요? | |
| 리프 노드는 총 몇 개인가요? | |
| 가장 깊은 경로는 몇 단계인가요? | |
| 학습 정확도는 얼마인가요? | |
| 테스트 정확도는 얼마인가요? | |

**단계 4**: 하단의 [📜 학습된 규칙]을 읽고, 가장 단순한 규칙과 가장 복잡한 규칙을 찾아봅니다.

### 2.3 토론 질문

1. 트리의 맨 위(루트)에서 어떤 특성으로 먼저 나누나요? 왜 그 특성이 선택되었을까요?
2. "부리 깊이 ≤ 15.6"이라는 규칙은 무슨 의미인가요? 일상 언어로 바꿔봅시다.
3. 리프 노드의 데이터가 한 종만 있으면 지니 불순도는 얼마인가요?

## 3. 활동 2: 깊이 실험 — 과적합 탐구

### 3.1 활동 개요

| 항목 | 내용 |
|------|------|
| 목표 | 트리 깊이에 따른 성능 변화를 관찰하고, 과적합 개념을 체험한다 |
| 소요 시간 | 15분 |
| 핵심 개념 | 과적합(Overfitting), 과소적합(Underfitting) |

### 3.2 활동 절차

**단계 1**: 최대 깊이를 1로 설정하고 트리를 생성합니다.

**단계 2**: 깊이를 2, 3, 4, 5, 6, 8로 차례로 변경하며 정확도를 기록합니다.

### 3.3 기록지

| 최대 깊이 | 학습 정확도 | 테스트 정확도 | 리프 노드 수 | 트리 깊이(실제) |
|:---------:|:-----------:|:------------:|:------------:|:---------------:|
| 1 | | | | |
| 2 | | | | |
| 3 | | | | |
| 4 | | | | |
| 5 | | | | |
| 6 | | | | |
| 8 | | | | |

### 3.4 분석 질문

1. 깊이를 늘릴수록 **학습 정확도**는 어떻게 변하나요?
2. 깊이를 늘릴수록 **테스트 정확도**는 계속 올라가나요?
3. 학습 정확도는 높은데 테스트 정확도는 낮은 경우가 있나요? 이것을 무엇이라 부르나요?
4. 가장 적절한 깊이는 몇이라고 생각하나요? 그 이유는?

### 3.5 교수자 참고

| 깊이 | 예상 현상 | 교육 포인트 |
|:----:|-----------|-------------|
| 1 | 학습/테스트 모두 낮음 | 과소적합: 모델이 너무 단순 |
| 3~4 | 학습/테스트 모두 높음 | 적절한 복잡도의 균형점 |
| 6~8 | 학습 매우 높고 테스트 정체/하락 | 과적합: 학습 데이터 암기 |

## 4. 활동 3: 특성 중요도 탐구

### 4.1 활동 개요

| 항목 | 내용 |
|------|------|
| 목표 | 어떤 특성이 펭귄 분류에 가장 중요한지 실험적으로 파악한다 |
| 소요 시간 | 15분 |
| 핵심 개념 | 특성 선택, 특성 중요도 |

### 4.2 활동 절차

**단계 1**: 깊이 4, 최소 샘플 2로 고정합니다.

**단계 2**: 특성을 하나씩만 선택하여 트리를 생성하고 정확도를 기록합니다.

**단계 3**: 특성을 2개씩 조합하여 트리를 생성하고 정확도를 기록합니다.

### 4.3 기록지 A: 단일 특성

| 선택 특성 | 테스트 정확도 | 루트 조건 | 관찰 |
|-----------|:------------:|-----------|------|
| 부리 길이만 | | | |
| 부리 깊이만 | | | |
| 날개 길이만 | | | |
| 체중만 | | | |

### 4.4 기록지 B: 2개 특성 조합

| 특성 조합 | 테스트 정확도 | 관찰 |
|-----------|:------------:|------|
| 부리 길이 + 부리 깊이 | | |
| 부리 깊이 + 날개 길이 | | |
| 날개 길이 + 체중 | | |
| 전체 4개 | | |

### 4.5 분석 질문

1. 단일 특성 중 가장 높은 정확도를 보인 특성은 무엇인가요?
2. 특성 1개일 때와 4개일 때 정확도 차이는 얼마나 되나요?
3. 특성이 많을수록 항상 정확도가 높아지나요?
4. 실생활에서 "어떤 정보가 중요한가"를 판단하는 상황과 비교해봅시다.

## 5. 활동 4: 추론 과정 추적

### 5.1 활동 개요

| 항목 | 내용 |
|------|------|
| 목표 | 의사결정트리가 새로운 데이터를 분류하는 과정을 단계별로 추적한다 |
| 소요 시간 | 10분 |
| 핵심 개념 | 추론(Inference), 경로 추적 |

### 5.2 활동 절차

**단계 1**: 학습 & 분석 모드에서 기본 설정으로 트리를 생성합니다.

**단계 2**: [🔍 추론 모드]로 전환합니다.

**단계 3**: 테스트 데이터 #1을 선택하고, 트리 위의 노란색 경로를 따라갑니다.

**단계 4**: 각 분기점에서 어떤 조건으로, 어떤 방향으로 이동했는지 기록합니다.

### 5.3 기록지

**테스트 샘플 #____ 추론 추적**

| 단계 | 분기 조건 | 샘플 값 | 판단 (≤/>) | 이동 방향 |
|:----:|-----------|---------|:----------:|:---------:|
| 1 | | | | |
| 2 | | | | |
| 3 | | | | |
| 4 | | | | |

| 항목 | 값 |
|------|-----|
| 최종 예측 | |
| 실제 종 | |
| 정답 여부 | ✅ / ❌ |

### 5.4 추가 활동

1. [🎲 랜덤] 버튼으로 3개 이상의 샘플을 추론하고, 틀린 것이 있는지 확인합니다.
2. 틀린 예측이 있다면, 왜 트리가 잘못 분류했는지 토론합니다.
3. [📊 전체 테스트 실행]으로 전체 정확도를 확인합니다.

## 8. 종합 활동: 최적 모델 찾기

### 8.1 활동 개요

| 항목 | 내용 |
|------|------|
| 목표 | 여러 파라미터를 조합하여 가장 좋은 성능의 모델을 찾는다 |
| 소요 시간 | 15분 |
| 핵심 개념 | 하이퍼파라미터 튜닝, 모델 선택 |

### 8.2 활동 절차

**단계 1**: 팀별(2~3명)로 최대 깊이, 최소 샘플 수, 특성 조합을 자유롭게 실험합니다.

**단계 2**: 가장 높은 **테스트 정확도**를 달성한 설정을 기록합니다.

**단계 3**: 팀별 결과를 발표하고 비교합니다.

### 8.3 기록지

| 시도 | 깊이 | 최소 샘플 | 선택 특성 | 학습 정확도 | 테스트 정확도 |
|:----:|:----:|:---------:|-----------|:-----------:|:------------:|
| 1 | | | | | |
| 2 | | | | | |
| 3 | | | | | |
| 4 | | | | | |
| 5 | | | | | |

**최적 설정**: 깊이 ____, 최소 샘플 ____, 특성 ____

**테스트 정확도**: _____%

### 8.4 발표 및 토론

1. 어떤 팀이 가장 높은 테스트 정확도를 달성했나요?
2. 높은 학습 정확도가 항상 높은 테스트 정확도를 의미하나요?
3. "좋은 모델"이란 무엇인지 정의해봅시다.
4. 실제 AI 개발에서도 이런 과정을 거칠까요?

## 9. 교수자 가이드

### 9.1 난이도별 활동 선택

| 난이도 | 권장 활동 | 소요 시간 |
|:------:|-----------|:---------:|
| 초급 | 활동 1 + 활동 4 | 20분 |
| 중급 | 활동 1 + 활동 2 + 활동 3 | 40분 |
| 심화 | 활동 2 + 활동 3 + 활동 5 + 종합 | 50분 |
| 전체 | 전 활동 | 85분 (2차시) |

### 9.2 자주 발생하는 질문과 대응

| 질문 | 교수자 대응 |
|------|------------|
| "왜 매번 같은 트리가 나와요?" | 시드 기반 셔플이므로 동일 설정이면 동일 결과. 교실에서 같은 결과를 보는 것이 교육적 장점 |
| "특성 0개 선택하면 어떻게 되나요?" | 알림 메시지가 나옴. 최소 1개 필요 |
| "학습 정확도가 100%인데 좋은 건가요?" | 과적합 토론으로 연결. 테스트 정확도가 진짜 실력 |
| "Adelie와 Chinstrap을 왜 혼동하나요?" | 두 종의 특성값 범위가 겹침. 데이터 보기로 확인 유도 |

### 9.3 확장 토론 주제

1. AI가 동물을 분류한다면 어떤 특성을 사용할까요?
2. 의사결정트리는 어떤 실생활 문제에 사용될 수 있을까요?
3. AI의 분류가 틀릴 수 있다면, 어떤 상황에서 위험할까요?
4. 의사결정트리와 우리가 일상에서 하는 판단의 공통점은?
