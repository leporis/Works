# 의사결정트리 시뮬레이터 개발일지

**문서 버전: v1.3.1.0**

---

## 1. 프로젝트 개요

### 1.1 프로젝트명
의사결정트리 학습 시뮬레이터

### 1.2 목적
초등 예비교사를 대상으로 **의사결정트리(Decision Tree)** 분류 알고리즘의 원리를 직관적으로 이해할 수 있도록 교육용 시뮬레이터를 개발한다.

### 1.3 교육 목표

| 목표 | 설명 |
|------|------|
| 의사결정트리 이해 | 특성 기반 조건 분기를 통한 분류 원리 학습 |
| 지니 불순도 체험 | 분류 기준 선택 원리(정보 이득) 시각화 |
| 트리 성장 관찰 | 애니메이션을 통한 트리 구축 과정 체험 |
| 특성 중요도 파악 | 각 입력 특성이 분류에 미치는 영향력 비교 |
| 추론 과정 학습 | 트리를 따라가며 새 데이터 분류 체험 |
| **다중 데이터 탐색** | **여러 데이터셋으로 알고리즘의 범용성 이해 (v1.3.0.0)** |
| **트리 구조 탐색** | **축소된 노드 호버로 세부 정보 확인, 트리 전체 구조 파악 (v1.3.1.0)** |

### 1.4 개발 배경
- v1.0.0.0 초기 버전에서 펭귄 데이터 전용으로 시작
- v1.1.0.0에서 직접 입력 추론, 특성 중요도 추가 (Phase 1)
- v1.2.0.0에서 트리 성장 애니메이션, 산점도/결정 경계 추가 (Phase 2)
- **v1.3.0.0에서 다중 데이터셋 지원, CSV 업로드 기능 추가 (Phase 3)**
- **v1.3.1.0에서 노드 축소, 호버 툴팁, 트리 영역 확장, 제목 변경**

## 2. 시스템 설계

### 2.1 데이터셋 레지스트리 (v1.3.0.0 신규)

시뮬레이터는 `DATASETS` 레지스트리 기반으로 여러 데이터셋을 동적으로 관리한다.

| 데이터셋 | 샘플 수 | 특성 수 | 클래스 수 | 분류 대상 |
|----------|:-------:|:-------:|:---------:|----------|
| 🐧 펭귄 (Palmer Penguins) | 134 | 4 | 3 | Adelie, Chinstrap, Gentoo |
| 🌺 붓꽃 (Iris) | 150 | 4 | 3 | Setosa, Versicolor, Virginica |
| 📄 CSV 업로드 | 가변 | 자동 감지 | 자동 감지 | 사용자 지정 |

### 2.2 펭귄 데이터셋 특성

| 특성 | 이모지 | 단위 | 범위 | 설명 |
|------|:------:|------|------|------|
| 부리 길이 | 🦷 | mm | 32.1~59.6 | 부리의 위에서 본 길이 |
| 부리 깊이 | 📏 | mm | 13.1~21.5 | 부리의 옆에서 본 높이 |
| 날개 길이 | 🪶 | mm | 172~231 | 앞날개(flipper) 길이 |
| 체중 | ⚖️ | g | 2700~6300 | 몸무게 |

### 2.3 붓꽃 데이터셋 특성

| 특성 | 이모지 | 단위 | 범위 | 설명 |
|------|:------:|------|------|------|
| 꽃받침 길이 | 🌿 | cm | 4.3~7.9 | Sepal Length |
| 꽃받침 너비 | 🌱 | cm | 2.0~4.4 | Sepal Width |
| 꽃잎 길이 | 🌸 | cm | 1.0~6.9 | Petal Length |
| 꽃잎 너비 | 🌷 | cm | 0.1~2.5 | Petal Width |

### 2.4 동적 CONFIG 구조 (v1.3.0.0 변경)

```
CONFIG (정적)
├── TREE: { MAX_DEPTH, MIN_SAMPLES, GINI_THRESHOLD, ... }
├── ANIMATION: { DEFAULT_SPEED, MIN_SPEED, MAX_SPEED }
├── SPLIT: { TRAIN_RATIO, RANDOM_SEED }
├── VIS: { NODE_*, SCATTER, COLORS }
│
└── (동적 - 데이터셋 전환 시 갱신)
    ├── FEATURES: { key: { label, unit, short, emoji, step, min, max, default } }
    ├── CLASSES: { name: { emoji, label, ko, color } }
    └── CLASS_KEY: 'species' | 'class' | 사용자 지정 열명
```

기존 v1.2.0.0까지는 FEATURES, CLASSES가 정적 하드코딩이었으나, v1.3.0.0에서 데이터셋 전환 시 자동 갱신되는 동적 구조로 변경되었다.

### 2.5 시뮬레이션 기본 설정

| 항목 | 기본값 | 조절 범위 |
|------|--------|-----------|
| 최대 깊이 | 4 | 1~8 |
| 최소 샘플 수 | 2 | 2~20 |
| 학습/테스트 비율 | 80:20 | 고정 |
| 랜덤 시드 | 42 | 고정 |
| 애니메이션 속도 | 600ms | 100~1500ms |

|------|------|------|
| 마크업 | HTML5 | 문서 구조 |

### 5.2 핵심 알고리즘

#### 지니 불순도 (Gini Impurity)
```
Gini(S) = 1 - Σ(pᵢ)²

pᵢ = (클래스 i의 샘플 수) / (전체 샘플 수)
```

#### 최적 분기점 탐색 (Best Split)
```
for each feature:
    sort data by feature value
    for each threshold (midpoint between consecutive values):
        split into left/right groups
        weighted_gini = (n_left × gini_left + n_right × gini_right) / n_total
        if weighted_gini < best_gini:
            update best split
```

#### 특성 중요도 (가중 지니 감소)
```
importance(feature) = Σ (n_node / n_total) × Δgini_node
    where feature was used for splitting

정규화: importance / Σ(all importances)
```

| 제목 | "분류 의사결정트리 시뮬레이터" | "의사결정트리 시뮬레이터" |

### 5.5 CSV 업로드 처리 흐름

```
1. handleCSVFile(event)
   └─ FileReader.readAsText() → parseCSV(text)

2. parseCSV(text)
   └─ 줄 분리 → 헤더 추출 → 행 파싱 → {headers, rows}

3. showCSVModal(parsed)
   └─ 미리보기 테이블 (5행) → 클래스 열 선택 드롭다운

4. confirmCSVUpload()
   ├─ 검증: 최소 5행, 숫자 특성 1개 이상
   ├─ 숫자 열 자동 감지 (클래스 열 제외)
   ├─ 클래스 목록 추출 → 이모지/색상 자동 할당
   ├─ DATASETS['csv_custom'] 등록
   └─ switchDataset('csv_custom')

5. switchDataset(datasetId)
   └─ applyDataset() → CONFIG 갱신 → UI 재생성 → 데이터 로드 → 초기화
```

### 5.6 데이터 검증 규칙

| 항목 | 조건 | 처리 |
|------|------|------|
| 최소 행 수 | 5행 이상 | 미달 시 경고 및 중단 |
| 숫자 특성 | 1개 이상 | 없으면 경고 및 중단 |
| 결측치 | 숫자 열에 NaN | 해당 행 제거 |
| 빈 클래스 값 | 공백 또는 비어있음 | 해당 행 제거 |
