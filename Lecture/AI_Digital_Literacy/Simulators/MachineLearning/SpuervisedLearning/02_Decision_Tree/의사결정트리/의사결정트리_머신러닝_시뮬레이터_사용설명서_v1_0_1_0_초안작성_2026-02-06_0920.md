# 의사결정트리 머신러닝 시뮬레이터 사용설명서

**문서 버전: v1.0.1.0**

## 1. 개요

### 1.1 시뮬레이터 소개

의사결정트리 머신러닝 시뮬레이터는 초등 예비교사를 대상으로 **의사결정트리(Decision Tree)** 분류 알고리즘의 원리를 직관적으로 학습할 수 있도록 설계된 교육용 웹 애플리케이션입니다.

| 항목 | 내용 |
|------|------|
| 대상 | 초등 예비교사 (비전공자 포함) |
| 데이터셋 | Palmer Penguins (134마리, 3종) |
| 알고리즘 | CART (Classification and Regression Trees) |
| 분할 기준 | 지니 불순도 (Gini Impurity) |
| 실행 환경 | 웹 브라우저 (Chrome, Edge, Firefox 권장) |
| 외부 의존성 | Google Fonts (온라인 접속 필요) |

### 1.2 실행 방법

`의사결정트리_머신러닝_시뮬레이터_v1.0.1.0.html` 파일을 웹 브라우저로 열면 바로 실행됩니다. 별도의 설치나 서버가 필요하지 않습니다.

### 1.3 화면 개요

시뮬레이터는 **학습 & 분석 모드**와 **추론 모드**, 두 가지 모드로 구성됩니다. 상단의 모드 탭을 클릭하여 전환합니다.

## 2. 학습 & 분석 모드

학습 & 분석 모드는 의사결정트리를 생성하고, 구조를 관찰하며, 성능을 평가하는 모드입니다.

### 2.1 화면 구성

| 영역 | 위치 | 기능 |
|------|------|------|
| 트리 설정 패널 | 왼쪽 | 파라미터 조절, 특성 선택, 트리 생성/초기화 |
| 의사결정트리 시각화 | 중앙 | Canvas 기반 트리 구조 렌더링 |
| 데이터 정보 패널 | 오른쪽 상단 | 데이터 분포, 분할 정보, 학습 데이터 테이블 |
| 모델 성능 패널 | 오른쪽 하단 | 정확도, 트리 깊이, 혼동 행렬 |
| 학습된 규칙 | 하단 왼쪽 | 분류 규칙 경로 목록 |
| 테스트 결과 상세 | 하단 오른쪽 | 전체 테스트 데이터 예측 비교 |

### 2.2 트리 설정 패널

#### 2.2.1 최대 깊이 (Max Depth)

| 항목 | 내용 |
|------|------|
| 범위 | 1 ~ 8 |
| 기본값 | 4 |
| 설명 | 트리가 깊어질 수 있는 최대 단계 수를 제한합니다 |

슬라이더를 좌우로 움직여 조절합니다. 값이 작을수록 단순한 트리, 클수록 복잡한 트리가 생성됩니다.

- 깊이 1~2: 매우 단순한 트리 (과소적합 가능)
- 깊이 3~5: 적절한 복잡도 (권장)
- 깊이 6~8: 복잡한 트리 (과적합 위험)

#### 2.2.2 최소 샘플 수 (Min Samples)

| 항목 | 내용 |
|------|------|
| 범위 | 1 ~ 10 |
| 기본값 | 2 |
| 설명 | 노드를 더 나누기 위해 필요한 최소 데이터 수입니다 |

값이 클수록 일반적인 규칙, 값이 작을수록 세밀한 규칙이 만들어집니다.

#### 2.2.3 사용할 특성

4개의 체크박스로 트리 생성에 사용할 특성을 선택합니다.

| 체크박스 | 특성 | 단위 |
|:--------:|------|------|
| ☑ | 부리 길이 (bill_length_mm) | mm |
| ☑ | 부리 깊이 (bill_depth_mm) | mm |
| ☑ | 날개 길이 (flipper_length_mm) | mm |
| ☑ | 체중 (body_mass_g) | g |

최소 1개 이상 선택해야 하며, 기본값은 4개 전체 선택입니다. 특성 수를 줄여가며 실험하면 어떤 특성이 분류에 중요한지 관찰할 수 있습니다.

#### 2.2.4 버튼

| 버튼 | 기능 |
|------|------|
| 🌳 트리 생성 | 현재 설정으로 의사결정트리를 학습합니다 |
| 🔄 초기화 | 트리와 모든 결과를 초기화합니다 |

#### 2.2.5 알고리즘 설명

패널 하단에 의사결정트리 알고리즘의 4단계 동작 원리가 간략하게 표시됩니다.

### 2.3 의사결정트리 시각화 (중앙 영역)

[🌳 트리 생성] 버튼을 클릭하면 중앙 Canvas에 트리가 그려집니다.

#### 2.3.1 분기 노드 읽기

```
┌─────────────────────┐
│  부리깊이 ≤ 15.6?   │  ← 분할 조건
│  gini=0.642 | n=107 │  ← 불순도 / 데이터 수
└─────────────────────┘
```

- **조건**: 해당 특성값이 임계값 이하인지 확인합니다
- **gini**: 지니 불순도 (0에 가까울수록 순수)
- **n**: 이 노드에 도달한 데이터 수
- **왼쪽 자식(≤)**: 조건을 만족하는 데이터
- **오른쪽 자식(>)**: 조건을 만족하지 않는 데이터

#### 2.3.2 리프 노드 읽기

```
┌─────────────────────┐
│  🔵 Gentoo          │  ← 최종 분류 결과
│  n=45 (A:0 C:1 G:44)│  ← 종별 데이터 분포
└─────────────────────┘
```

- **종 이모지 + 이름**: 이 노드에 도달한 데이터는 해당 종으로 분류
- **n**: 전체 데이터 수
- **(A:0 C:1 G:44)**: Adelie 0마리, Chinstrap 1마리, Gentoo 44마리

#### 2.3.3 간선 레이블

노드를 연결하는 선 옆에 `≤` (왼쪽) 또는 `>` (오른쪽) 표시가 있어 분기 방향을 확인할 수 있습니다.

### 2.4 데이터 정보 패널

#### 2.4.1 데이터 통계

| 표시 항목 | 설명 |
|-----------|------|
| 전체 데이터 | 134개 |
| 학습/테스트 | 107 / 27 (80/20 분할) |
| Adelie | 50개 (빨강 뱃지) |
| Chinstrap | 34개 (청록 뱃지) |
| Gentoo | 50개 (파랑 뱃지) |

#### 2.4.2 데이터 보기 버튼

[📋 데이터 보기] 버튼을 클릭하면 학습 데이터 107개의 전체 목록이 테이블로 표시됩니다. 각 행에는 4가지 특성값과 종(Species)이 표시됩니다. 다시 클릭하면 숨깁니다.

### 2.5 모델 성능 패널

트리 생성 후 자동으로 표시됩니다.

#### 2.5.1 성능 지표

| 지표 | 설명 | 색상 기준 |
|------|------|-----------|
| 학습 정확도 | 학습 데이터에서의 정확도 | 90%↑ 녹색, 70%↑ 노랑, 이하 빨강 |
| 테스트 정확도 | 테스트 데이터에서의 정확도 | 동일 기준 |
| 트리 깊이 | 실제 생성된 트리의 깊이 | - |

#### 2.5.2 혼동 행렬

3×3 표 형태로, 행은 실제 종, 열은 예측 종을 나타냅니다.

- **대각선 (녹색 배경)**: 정확하게 분류된 수
- **비대각선 (빨강 배경)**: 잘못 분류된 수
- **0 (흰색 배경)**: 해당 오분류 없음

### 2.6 하단 패널

#### 2.6.1 학습된 규칙

루트 노드에서 각 리프 노드까지의 분류 경로를 규칙 형태로 표시합니다.

```
부리깊이 ≤ 15.6 AND 날개길이 ≤ 210.0
→ 🔵 Gentoo (44/45개, 98%)
```

각 규칙은 AND 조건으로 연결되며, 해당 리프 노드의 다수 종과 비율을 보여줍니다.

#### 2.6.2 테스트 결과 상세

27개 테스트 데이터 각각에 대한 예측 결과를 테이블로 보여줍니다.

| 열 | 내용 |
|-----|------|
| # | 순번 |
| 부리길이~체중 | 4가지 특성값 |
| 실제 | 실제 종 (뱃지) |
| 예측 | 예측 종 (뱃지) |
| 결과 | ✅ 정답 / ❌ 오답 |

- 정답 행: 연녹색 배경
- 오답 행: 연빨강 배경

## 3. 추론 모드

추론 모드는 개별 테스트 데이터가 트리를 따라 분류되는 과정을 단계별로 관찰하는 모드입니다.

### 3.1 화면 구성

| 영역 | 위치 | 기능 |
|------|------|------|
| 테스트 데이터 선택 | 왼쪽 | 샘플 선택, 네비게이션, 전체 테스트 |
| 추론 경로 시각화 | 중앙 상단 | 경로 하이라이트된 트리 |
| 추론 과정 (단계별) | 중앙 하단 | 각 분기점의 판단 과정 |
| 추론 결과 | 오른쪽 | 예측 결과 및 정오답 표시 |

### 3.2 사전 조건

추론 모드를 사용하려면 먼저 **학습 & 분석 모드에서 트리를 생성**해야 합니다.

### 3.3 테스트 데이터 선택 패널

#### 3.3.1 드롭다운 선택

드롭다운 목록에서 테스트 샘플을 선택합니다. 각 항목에는 순번, 종, 부리 길이, 날개 길이가 표시됩니다.

#### 3.3.2 네비게이션 버튼

| 버튼 | 기능 |
|------|------|
| ◀ 이전 | 이전 테스트 샘플로 이동 |
| 다음 ▶ | 다음 테스트 샘플로 이동 |
| 🎲 랜덤 | 임의의 테스트 샘플 선택 |
| 📊 전체 테스트 실행 | 27개 전체 추론 결과 한 번에 확인 |

#### 3.3.3 샘플 정보

선택한 테스트 샘플의 4가지 특성값과 실제 종이 표시됩니다.

### 3.4 추론 경로 시각화

테스트 샘플을 선택하면 트리 위에 **노란색으로 추론 경로가 강조**됩니다.

- **노란 테두리 노드**: 추론이 통과한 노드
- **노란 굵은 간선**: 추론이 이동한 경로
- **노란 배경 리프**: 최종 분류 도달 노드

### 3.5 추론 과정 (단계별)

트리 아래에 각 분기점에서의 판단 과정이 순서대로 표시됩니다.

```
단계 1: 부리 깊이 = 14.1mm
        14.1 ≤ 15.6 → 왼쪽으로

단계 2: 날개 길이 = 210mm
        210 ≤ 212.5 → 왼쪽으로

🎯 최종 분류: Gentoo ✅ 정답!
```

각 단계는 녹색 배경으로 표시되며, 최종 결과는 정답(녹색) 또는 오답(빨강)으로 구분됩니다.

### 3.6 추론 결과 패널

| 표시 항목 | 설명 |
|-----------|------|
| 예측 결과 | 종 이모지 + 종명 |
| 정오답 | ✅ 정확한 분류! / ❌ 실제: (종명) |

정답이면 녹색 그라데이션 배경, 오답이면 빨강 그라데이션 배경으로 표시됩니다.

### 3.7 전체 테스트 실행

[📊 전체 테스트 실행] 버튼을 클릭하면 27개 테스트 데이터 전체의 추론 결과가 한 번에 표시됩니다.

- 상단: 전체 정확도 (%)와 정답 수
- 하단: 각 샘플별 실제 종, 예측 종, 정오답 표시
- 개별 결과를 클릭하면 해당 샘플의 상세 추론 과정을 볼 수 있습니다

### 4.1 Palmer Penguins

남극 Palmer 군도에 서식하는 3종 펭귄의 실측 데이터입니다.

| 종 | 영문명 | 한국어 | 이모지 | 색상 | 데이터 수 |
|:--:|--------|--------|:------:|------|:---------:|

### 4.2 특성 변수

| 특성 | 설명 | 범위 |
|------|------|------|
| 부리 길이(mm) | 부리의 길이 | 33.5 ~ 59.6 |
| 부리 깊이(mm) | 부리의 높이(두께) | 13.1 ~ 21.5 |
| 날개 길이(mm) | 플리퍼(날개)의 길이 | 172 ~ 230 |

### 4.3 데이터 분할

전체 134개 데이터를 시드 42 기반 셔플로 무작위 분할합니다.

- **학습 데이터 (80%, 107개)**: 트리를 만드는 데 사용
- **테스트 데이터 (20%, 27개)**: 트리 성능을 평가하는 데 사용
- 동일한 시드를 사용하므로 매번 같은 분할 결과가 나옵니다

## 5. 용어 사전

| 용어 | 영문 | 설명 |
|------|------|------|
| 의사결정트리 | Decision Tree | 질문(조건)의 연속으로 데이터를 분류하는 알고리즘 |
| CART | Classification and Regression Trees | 이진 분할 기반 트리 알고리즘 |
| 지니 불순도 | Gini Impurity | 데이터가 얼마나 혼합되어 있는지 나타내는 지표 (0=순수, 0.67=3종 균등) |
| 분기 노드 | Internal Node | 조건에 따라 좌/우로 데이터를 나누는 중간 노드 |
| 리프 노드 | Leaf Node | 최종 분류 결과를 내놓는 말단 노드 |
| 루트 노드 | Root Node | 트리의 최상위 노드 (첫 번째 분기) |
| 최대 깊이 | Max Depth | 트리가 깊어질 수 있는 최대 단계 수 |
| 최소 샘플 수 | Min Samples | 노드를 분할하기 위해 필요한 최소 데이터 수 |
| 혼동 행렬 | Confusion Matrix | 실제 분류와 예측 분류를 비교하는 표 |
| 과적합 | Overfitting | 학습 데이터에만 과도하게 맞춰져 새 데이터에서 성능이 떨어지는 현상 |
| 과소적합 | Underfitting | 모델이 너무 단순하여 데이터의 패턴을 충분히 학습하지 못하는 현상 |
| 학습/테스트 분할 | Train/Test Split | 데이터를 학습용과 평가용으로 나누는 것 |
