
### 3️⃣ 추론 체험
```
[🔍 추론 모드] 클릭 → 테스트 데이터 선택 → 단계별 분류 과정 관찰
```

## 🌳 트리 읽는 법

### 분기 노드 (내부 노드)
```
┌─────────────────────┐
│  부리깊이 ≤ 15.6?   │  ← 조건: 이 특성값 기준으로 분기
│  gini=0.642 | n=107 │  ← 불순도와 데이터 수
└─────────────────────┘
    ≤         >            ← 왼쪽: 조건 충족 / 오른쪽: 초과
```

### 리프 노드 (최종 분류)
```
┌─────────────────────┐
│  🔵 Gentoo          │  ← 이 노드에 도달하면 Gentoo로 분류
│  n=45 (A:0 C:1 G:44)│  ← 학습 데이터 분포
└─────────────────────┘
```

## 📜 학습된 규칙 읽기

예시:
```
부리깊이 ≤ 15.6 AND 날개길이 ≤ 210.0
→ 🔵 Gentoo (44/45개, 98%)
```

의미: "부리 깊이가 15.6mm 이하이고 날개가 210mm 이하이면 → Gentoo"

## 🔍 추론 과정 시각화

### 테스트 샘플 선택 후:

```
단계 1: 부리 깊이 = 14.1mm
        14.1 ≤ 15.6 → 왼쪽으로

단계 2: 날개 길이 = 210mm
        210 ≤ 212.5 → 왼쪽으로

🎯 최종 분류: Gentoo ✅ 정답!
```

- **트리 위 노란색 경로**: 추론이 지나간 경로
- **단계별 설명**: 각 분기점에서의 판단 근거

## ❓ 자주 묻는 질문

**Q: 지니 불순도가 뭔가요?**
→ 데이터가 얼마나 "섞여있는지" 나타내는 수치예요. 0이면 한 종만 있는 깨끗한 상태, 0.67이면 3종이 골고루 섞인 상태입니다.

**Q: 테스트 정확도가 학습 정확도보다 낮아요**
→ 정상입니다! 학습 데이터는 이미 본 데이터이므로 잘 맞추고, 테스트 데이터는 처음 보는 데이터라 더 어렵습니다.

**Q: 깊이를 늘릴수록 항상 좋나요?**
→ 아닙니다! 깊이가 너무 크면 학습 데이터에만 맞춰져서(과적합) 새로운 데이터에서 오히려 성능이 떨어질 수 있어요.

**Q: 어떤 특성이 가장 중요한가요?**
→ 트리의 루트(맨 위) 노드에 사용된 특성이 가장 중요합니다. 보통 날개 길이나 부리 깊이가 첫 번째로 선택됩니다.

## 🔑 핵심 용어

| 용어 | 의미 |
|------|------|
| **의사결정트리** | 질문(조건)의 연속으로 데이터를 분류하는 알고리즘 |
| **분기 노드** | 조건에 따라 좌/우로 데이터를 나누는 중간 노드 |
| **리프 노드** | 최종 분류 결과를 내놓는 말단 노드 |
| **지니 불순도** | 데이터의 혼합 정도 (0 = 순수, 0.67 = 3종 균등) |
| **최대 깊이** | 트리가 깊어질 수 있는 최대 단계 수 |
| **과적합** | 학습 데이터에만 맞춰져 새 데이터에서 성능 저하 |
| **혼동 행렬** | 실제 분류 vs 예측 분류를 비교하는 표 |
| **학습/테스트 분할** | 데이터를 학습용(80%)과 평가용(20%)으로 나눔 |
