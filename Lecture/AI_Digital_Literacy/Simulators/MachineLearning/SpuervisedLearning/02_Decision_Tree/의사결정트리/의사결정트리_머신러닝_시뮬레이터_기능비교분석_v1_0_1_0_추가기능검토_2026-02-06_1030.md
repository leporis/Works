## 1. 문서 목적

의사결정트리 머신러닝 시뮬레이터(v1.0.1.0)의 기능 확장을 위해, 범용 선형회귀 학습 시뮬레이터(v1.2.0.0)와 기능을 체계적으로 비교 분석하고, 추가할 기능의 우선순위를 정리한다.

## 6. 비적용 기능 (이유 명시)

범용 선형회귀 시뮬레이터의 일부 기능은 의사결정트리에 적용하지 않는다.

| 기능 | 비적용 이유 |
|------|------------|
| 학습률 (Learning Rate) | 의사결정트리는 경사하강법을 사용하지 않음. 파라미터 최적화 개념 자체가 다름 |
| 에포크 (Epochs) | 반복 학습 개념 없음. 트리는 한 번의 구축 과정으로 완성 |
| R² 결정 계수 | 회귀 문제 전용 지표. 분류 문제에는 정확도, F1 점수가 적합 |
| 다중 변수 가중치 | 의사결정트리는 특성별 가중치를 사용하지 않음. 알고리즘이 자동으로 최적 특성을 선택 |
| 차트 클릭 데이터 추가 | 분류 문제에서는 클래스(종) 지정이 필요하여 UX가 복잡해짐. 교육적 효과 대비 구현 비용이 큼 |
| 노이즈 레벨 조절 | 실측 데이터 사용이 교육적으로 적합. 인위적 노이즈 추가는 분류 문맥에서 부자연스러움 |

## 8. 기능별 상세 설계안

### 8.1 F4. 직접 입력 추론

추론 모드에서 테스트 데이터 선택 외에, 사용자가 직접 특성값을 입력하여 추론할 수 있는 기능이다.

**UI 설계**:
```
┌─────────────────────────────┐
│  🎯 직접 입력 추론           │
│                             │
│  🦷 부리 길이 (mm)          │
│  [====●========] 42.0       │
│  범위: 33.5 ~ 59.6         │
│                             │
│  📏 부리 깊이 (mm)          │
│  [======●======] 16.5       │
│  범위: 13.1 ~ 21.5         │
│                             │
│  🪶 날개 길이 (mm)          │
│  [========●====] 200        │
│  범위: 172 ~ 230           │
│                             │
│  ⚖️ 체중 (g)               │
│  [=====●=======] 3800       │
│  범위: 2700 ~ 6300         │
│                             │
│  [🔍 추론 실행]             │
│                             │
│  → 🔴 Adelie               │
│    (트리 경로 하이라이트)    │
└─────────────────────────────┘
```

**동작**: 슬라이더 조작 시 실시간으로 트리 경로가 하이라이트되고, 추론 단계별 설명이 업데이트된다. 범용 선형회귀의 추론 슬라이더와 동일한 즉시 반응 패턴이다.

**구현 포인트**:
- 각 특성의 min/max를 CONFIG.FEATURES에서 가져옴
- 기존 `predict()` 함수 재활용
- 기존 `getPathNodes()` + `drawTree()` 하이라이트 재활용
- 테스트 데이터 선택과 직접 입력을 탭으로 전환

### 8.2 F1. 트리 성장 애니메이션

트리가 한 단계씩 구축되는 과정을 애니메이션으로 재생하는 기능이다.

**UI 설계**:
```

│                                             │
│  재생 속도: [====●======]                    │
│            느림    보통    빠름              │
│                                             │
│  [▶️ 재생]  [⏹️ 정지]  [⏭️ 단계별]         │
│                                             │
│  ████████░░░░░░░░  단계 3/7                 │
│                                             │
│  현재: "부리깊이 ≤ 15.6" 분할 적용 중       │
│  지니 불순도: 0.642 → 0.285                 │
└─────────────────────────────────────────────┘
```

**동작**:
1. 트리 구축 과정을 단계별로 기록 (buildDecisionTree에 히스토리 추가)
2. 각 프레임에서 해당 단계까지의 트리만 렌더링
3. 새로 추가되는 노드는 강조 색상으로 표시
4. 각 단계의 분할 조건과 지니 불순도 감소를 텍스트로 표시

**구현 포인트**:
- `buildDecisionTree()` 함수에 스냅샷 기록 로직 추가
- 범용 선형회귀의 `trainingHistory[]` + `playFrame()` 패턴 차용
- 속도 조절: `setTimeout` 딜레이 조정 (20ms ~ 200ms)
- [⏭️ 단계별] 버튼: 한 스냅샷씩 수동 진행

### 8.3 F7. 특성 중요도 막대 그래프

각 특성이 트리 분류에 기여한 정도를 정량적으로 시각화한다.

**측정 방식**: 각 특성이 분할에 사용될 때의 **가중 지니 불순도 감소량**의 합

```
특성 중요도 = Σ (분할 전 Gini × n/N - 가중 Gini × n/N)
             해당 특성이 사용된 모든 노드에서
```

