| 단계 | 행동 | 확인 |
|:---:|------|:----:|
| 1 | 왼쪽 패널에서 **🍎 과일 분류** 선택 → **[✨ 데이터 생성]** 클릭 | 차트에 색깔 점이 나타났나요? |
| 2 | 오른쪽 패널에서 **[⚡ 분류 실행]** 클릭 | ◆ 모양 테스트 점에 ✅/❌가 붙나요? |

> 💡 30초만에 완료! 이제 K값 슬라이더를 바꿔서 정확도가 어떻게 변하는지 실험해보세요.

## 🚀 빠른 시작 (3단계)

### 1️⃣ 데이터 준비

**왼쪽 사이드바**에서 데이터를 준비합니다.

| 방법 | 조작 | 추천 상황 |
|------|------|----------|
| **프리셋 선택** | 드롭다운에서 데이터셋 선택 → [✨ 데이터 생성] | 처음 시작할 때 |
| **실 데이터** | 🌺 붓꽃 등 선택 → 변수 2개 고르기 → [✨ 데이터 생성] | 실제 데이터로 실험 |
| **CSV 업로드** | [📁 CSV 업로드] 탭 → 파일 선택 → 변수 매핑 | 나만의 데이터 사용 |

> 🎯 **첫 실습 추천**: 🍎 과일 분류, 데이터 수 60, 클래스 3, 분산도 3

### 2️⃣ 분류 실행

**오른쪽 사이드바**에서 설정 후 실행합니다.

| 설정 | 조절 | 기본값 |
|------|------|:------:|
| K값 | 슬라이더 1~15 (홀수 권장) | 5 |
| 거리 측정 | 유클리드 / 맨해튼 선택 | 유클리드 |
| 훈련:테스트 | 60:40 / **70:30** / 80:20 | 70:30 |

**[⚡ 분류 실행]** 클릭 → 테스트 점이 하나씩 평가되며 ✅/❌ 표시됩니다.

> 💡 **관찰**: 오른쪽 평가 진행 박스에서 정확도가 실시간으로 올라가는 걸 보세요!

### 3️⃣ K값 비교

**[▶️ K값 재생]** 클릭 → K=1부터 15까지 자동으로 재생됩니다.

| 관찰 | 어디서 볼까 |
|------|-----------|
| 결정 경계 변화 | 중앙 오른쪽 차트 — K가 커질수록 경계가 부드러워짐 |
| K별 정확도 | 재생 바 아래 칩 스트립 — 녹색 테두리 = 🏆 최적 K |
| 정확도 테이블 | 결과 패널 왼쪽 — 전체 K의 정확도 한눈에 비교 |

## 🔍 이런 장면을 찾아보세요!

| 관찰 대상 | 찾아볼 장면 | 의미 |
|----------|------------|------|
| **K=1 결정 경계** | 경계가 울퉁불퉁하게 꺾임 | 과적합(Overfitting) — 노이즈에 민감 |
| **K=15 결정 경계** | 경계가 매우 부드러워짐 | 과소적합(Underfitting) — 세부 패턴 무시 |
| **🏆 최적 K** | 테스트 정확도가 가장 높은 K | 일반화 성능이 가장 좋은 지점 |
| **분할 비율 차이** | 60:40 vs 80:20 정확도 비교 | 훈련 데이터가 많을수록 모델은 좋지만 테스트 수가 줄어 신뢰도↓ |
| **Train ● vs Test ◆** | 산점도에서 모양 구분 | 원(●)은 모델이 학습한 점, 다이아몬드(◆)는 처음 보는 점 |
| **혼동행렬 대각선** | 대각선 숫자가 크면 좋음 | 실제 클래스 = 예측 클래스 (정분류) |
| **추론 모드 연결선** | 차트 클릭 시 노란 점 주변 선 | 투표에 참여한 K개 이웃과의 거리 |

## ⚙️ 파라미터 설명

| 파라미터 | 범위 | 효과 | 추천 시작값 |
|---------|:----:|------|:----------:|
| **K값** | 1~15 | 작을수록 경계 복잡(과적합), 클수록 부드러움(과소적합) | 5 |
| **거리 측정** | 유클리드/맨해튼 | 유클리드: 직선 거리, 맨해튼: 격자 거리 | 유클리드 |
| **훈련:테스트** | 60:40~80:20 | 훈련↑ → 모델 좋지만 테스트 적어짐 | 70:30 |
| **데이터 수** | 20~150 | 많을수록 안정적 결과 | 60 |
| **클래스 수** | 2~4 | 많을수록 분류 어려워짐 | 3 |
| **분산도** | 1~5 | 클수록 클래스가 겹침 → 분류 어려워짐 | 3 |

## 🆘 도와주세요!

| 증상 | 원인 | 해결책 |
|------|------|--------|
| "데이터가 너무 적습니다!" 경고 | 데이터 10개 미만 | 데이터 수를 20 이상으로 설정 후 재생성 |
| 결정 경계가 안 보임 | 아직 분류 미실행 | **[⚡ 분류 실행]** 클릭 후 확인 |
| 정확도가 너무 낮음 (<50%) | 분산도가 너무 높거나 K값 부적절 | 분산도를 2~3으로 낮추고 K값 재생으로 최적 K 탐색 |
| 차트에 점이 안 보임 | 데이터 미생성 | **[✨ 데이터 생성]** 먼저 클릭 |
| 추론 모드에서 클릭해도 반응 없음 | 학습 모드에서 분류 미실행 | 🧠 학습 모드에서 먼저 분류 실행 후 🔮 추론 모드 전환 |
| CSV 업로드 오류 | 숫자형 컬럼 부족 | CSV에 숫자형 컬럼 2개 + 범주형 컬럼 1개 이상 필요 |
| K값 재생이 안 됨 | 데이터 미생성 | 먼저 데이터 생성 필요 (최소 10개 이상) |

## 🔑 핵심 용어

| 용어 | 학술 용어 | 의미 | 쉬운 설명 |
|------|----------|------|----------|
| KNN | K-Nearest Neighbors | K개의 가장 가까운 이웃으로 분류 | "가장 가까운 K명 친구에게 물어봐서 다수결로 결정" |
| K값 | K (hyperparameter) | 참조할 이웃의 수 | "몇 명한테 물어볼까?" |
| 결정 경계 | Decision Boundary | 클래스가 나뉘는 영역의 경계 | "여기서부터는 A팀, 저기서부터는 B팀" |
| 과적합 | Overfitting | 훈련 데이터에만 잘 맞고 새 데이터에 약함 | "시험 문제만 외워서 응용 못 하는 것" |
| Train/Test 분할 | Train/Test Split | 학습용과 평가용 데이터를 나누기 | "문제집과 시험지를 분리하는 것" |
| 혼동행렬 | Confusion Matrix | 실제 vs 예측 클래스 교차표 | "어떤 종류를 어떤 종류로 잘못 분류했는지 보는 표" |
| 유클리드 거리 | Euclidean Distance | 두 점 사이 직선 거리 | "새가 날아가는 최단 거리" |
| 맨해튼 거리 | Manhattan Distance | 축을 따라 이동하는 거리 | "바둑판 도로에서 걷는 거리" |

## 💭 생각해보기

> **K=1이 항상 나쁜 건 아닙니다.** 어떤 데이터에서는 K=1이 가장 좋은 결과를 낼 수도 있어요. 왜 그럴까요? (힌트: 클래스 경계가 매우 뚜렷한 데이터를 떠올려보세요.)

> **"좋은 모델"이란?** 훈련 데이터 정확도 100%인 모델과, 테스트 데이터 정확도 85%인 모델 중 어느 것이 더 좋을까요? Train/Test 분할이 왜 필수인지 생각해보세요.
