# 🎯 KNN 분류 시뮬레이터 퀵가이드

**문서 버전: v1.1.0.0**

---

## 한눈에 보기

| 항목 | 내용 |
|------|------|
| **목적** | K-최근접 이웃(KNN) 분류 알고리즘 학습 |
| **입력 변수** | X₁, X₂ (두 개의 특성) |
| **출력** | 분류 클래스 (다수결 투표) |
| **핵심 개념** | 유클리드 거리, K값, 결정 경계 |
| **데이터 소스** | 프리셋, CSV 업로드, 내장 데이터셋 |

---

## 📊 데이터 입력 방법 (v1.1.0 신규)

### 탭 1: 프리셋 데이터
기존 방식 - 과일/꽃/동물/랜덤 자동 생성

### 탭 2: CSV 파일 업로드
1. CSV 탭 클릭
2. 파일 드래그앤드롭 또는 클릭하여 업로드
3. X₁, X₂, 클래스 변수 선택
4. [📥 데이터 적용] 클릭

### 탭 3: 내장 데이터셋
| 데이터셋 | 설명 | 클래스 |
|----------|------|--------|
| 🌺 아이리스 | 꽃 크기로 품종 분류 | Setosa, Versicolor, Virginica |
| 🐧 펭귄 | 부리/지느러미로 종 분류 | Adelie, Chinstrap, Gentoo |
| 🍷 와인 | 화학 성분으로 품종 분류 | 클래스 1, 2, 3 |
| 🌾 씨앗 | 형태로 밀 품종 분류 | Kama, Rosa, Canadian |

## 🚀 빠른 시작 (3단계)

### 1️⃣ 데이터 준비
```
[프리셋] 선택 → [과일/꽃/동물/랜덤] 클릭 → [✨ 데이터 생성]

[내장] 선택 → [🌺 아이리스] 클릭 → [📥 데이터 적용]

```


### 3️⃣ 새 점 분류
```
[🔮 추론 모드] 클릭 → X₁/X₂ 슬라이더 조절 → 자동 분류
```

## 📐 KNN 알고리즘 원리

```
1. 새 점(빨간 ★)에서 모든 데이터까지 거리 계산
2. 가장 가까운 K개 이웃 선택
3. K개 이웃의 클래스 중 다수결로 분류

예) K=5일 때:
   - 🍎 사과 3개
   - 🍊 오렌지 2개
   → 다수결: 🍎 사과로 분류!
```

## 🎮 주요 컨트롤

### 분류 설정
| 컨트롤 | 범위 | 권장값 |
|--------|------|--------|
| K값 | 1 ~ 15 | 5 (홀수) |

### 데이터 설정 (프리셋)
| 컨트롤 | 범위 | 권장값 |
|--------|------|--------|
| 데이터 개수 | 5 ~ 100 | 30 |
| 노이즈 | 5% / 10% / 20% | 10% |
| 클래스 수 | 2 ~ 4 | 3 |

## 📈 그래프 읽는 법

### 산점도 & 결정 경계
- **점**: 학습 데이터 (클래스별 색상)
- **배경 색상**: 해당 영역의 예측 클래스
- **예측점**: 노란 원 (추론 시)
- **점선**: K개 이웃까지의 연결선

### K값 분석 그래프
- **X축**: K값 (1~15)
- **Y축**: 정확도 (%)
- 가장 높은 정확도의 K값 찾기!

## ❓ 자주 묻는 질문

**Q: K값은 왜 홀수가 좋나요?**
→ 동점(tie)을 피하기 위해서입니다. K=4일 때 2:2가 되면 결정하기 어렵습니다.

**Q: K가 너무 크면 어떻게 되나요?**
→ 결정 경계가 단순해지고, 세부적인 패턴을 놓칠 수 있습니다.

**Q: K가 1이면 어떻게 되나요?**
→ 가장 가까운 1개만 보므로 노이즈에 민감해집니다.

**Q: CSV 파일 형식은 어떻게 해야 하나요?**
→ 첫 행은 컬럼명, 이후 행은 데이터. 수치형 컬럼 2개 이상 필요.
