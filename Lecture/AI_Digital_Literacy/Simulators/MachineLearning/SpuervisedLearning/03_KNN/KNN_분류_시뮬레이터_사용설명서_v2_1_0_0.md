## 1. 소개

### 1.1 이 시뮬레이터는 무엇인가?

KNN 분류 시뮬레이터는 K-최근접 이웃(K-Nearest Neighbors) 알고리즘을 시각적으로 체험하는 교육용 웹 애플리케이션입니다. 데이터를 생성하고, 훈련과 테스트 데이터로 나눈 뒤, K값에 따라 분류 결과가 어떻게 달라지는지 직접 관찰할 수 있습니다.

### 1.2 KNN 알고리즘 요약

KNN은 새로운 데이터가 들어왔을 때, 기존 데이터 중 **가장 가까운 K개의 이웃**을 찾아 그 이웃들의 **다수결 투표**로 분류를 결정하는 알고리즘입니다.

```
새 데이터 → 가까운 K개 이웃 탐색 → 이웃들의 다수 클래스 → 예측 결과
```

### 1.3 관련 문서

| 문서 | 용도 | 읽는 시점 |
|------|------|----------|
| **이 문서 (사용설명서)** | 모든 기능의 상세 설명 | 사용 전·후 |
| 퀵가이드 | 빠른 참조, 실습 안내 | 사용 중 |
| 활용가이드 | 수업 적용 방법 | 수업 준비 시 |
| 개발일지 | 기술 구현 세부사항 | 개발·유지보수 시 |

```
├──────────────────────────────────────────────────────────────┤
│              [🧠 학습 모드]    [🔮 추론 모드]                │

│ ① 왼쪽  │    ② 중앙 시각화       │   ③ 오른쪽 설정         │

```

| 영역 | 내용 | 너비 |
|------|------|:----:|
| ① 왼쪽 사이드바 | 데이터셋 선택, 생성 설정, CSV 업로드 | 240px |

## 3. 데이터 입력

### 3.1 데이터 입력 방법

왼쪽 사이드바 상단의 탭에서 3가지 방법 중 하나를 선택합니다.

| 탭 | 방법 | 적합한 상황 |
|----|------|-----------|
| **📊 프리셋** | 내장 데이터셋 선택 | 처음 사용, 빠른 실습 |
| **📁 CSV 업로드** | 사용자 파일 업로드 | 직접 수집한 데이터 사용 |

### 3.2 프리셋 데이터셋

드롭다운에서 데이터셋을 선택합니다. 합성 데이터와 실 데이터 두 그룹으로 나뉩니다.

**합성 데이터 (4종)**

| 데이터셋 | 특성 X₁ | 특성 X₂ | 설명 |
|---------|---------|---------|------|
| 🍎 과일 분류 | 당도 (Brix) | 무게 (g) | 과일 품종 분류. 초보자 추천 |
| 🌸 꽃 분류 | 꽃잎 길이 (cm) | 꽃잎 너비 (cm) | 꽃 종류 분류 |
| 🐧 동물 분류 | 몸길이 (cm) | 체중 (kg) | 동물 종류 분류 |
| 🎲 랜덤 데이터 | 특성 1 | 특성 2 | 무작위 중심점. 다양한 실험용 |

합성 데이터는 아래 파라미터를 조절할 수 있습니다.

| 파라미터 | 범위 | 기본값 | 설명 |
|---------|:----:|:------:|------|
| 데이터 수 | 20~150 | 60 | 생성할 데이터 포인트 수 |
| 클래스 수 | 2~4 | 3 | 분류할 그룹 수 |
| 분산도 | 1~5 | 3 | 1=매우 밀집, 5=매우 분산. 높을수록 클래스 경계가 겹침 |

**실 데이터 (5종)**

| 데이터셋 | 출처 | 건수 | 클래스 | 선택 가능 변수 |
|---------|------|:----:|:------:|-------------|
| 🌺 붓꽃 (Iris) | Fisher(1936) | 150 | 3 | 꽃받침 길이/너비, 꽃잎 길이/너비 |
| 🐧 팔머 펭귄 | Horst(2020) | 333 | 3 | 부리 길이/깊이, 날개 길이, 체중 |
| 🍷 와인 품종 | Aeberhard(1991) | 178 | 3 | 알코올, 말산, 색상 강도, 프롤린 |
| 🌾 밀 씨앗 | Charytanowicz(2012) | 210 | 3 | 면적, 둘레, 콤팩트, 커널 너비 |
| 🏥 유방암 진단 | Wolberg(1995) | 569 | 2 | 평균 반지름/텍스처/둘레/면적 |

실 데이터는 4개 변수 중 **X₁과 X₂**에 사용할 변수 2개를 직접 선택합니다. 데이터셋을 선택하면 추천 변수 조합이 자동 설정되며, 데이터셋 정보 박스에 권장 설정이 표시됩니다.

### 3.3 CSV 업로드

**[📁 CSV 업로드]** 탭에서 사용자 CSV 파일을 불러옵니다.

**CSV 요구사항:**

| 항목 | 조건 |
|------|------|
| 파일 형식 | `.csv` (쉼표 구분) |
| 헤더 | 첫 행에 컬럼명 필수 |
| 숫자형 컬럼 | 최소 2개 (X₁, X₂로 사용) |
| 클래스 컬럼 | 1개 (범주형 또는 정수형) |

**업로드 절차:**

1. **[파일 선택]** 클릭 → CSV 파일 선택
2. 파싱 성공 시 **변수 선택 드롭다운** 3개가 나타남
3. X₁ 변수, X₂ 변수, 클래스 변수를 각각 선택
4. **[✨ 데이터 생성]** 클릭

> ⚠️ X₁과 X₂는 서로 다른 변수를 선택해야 합니다. 동일 변수 선택 시 오류 메시지가 표시됩니다.

### 3.4 데이터 생성 버튼

모든 입력 방법에서 공통으로 **[✨ 데이터 생성]** 버튼을 클릭해야 데이터가 차트에 표시됩니다. 이미 데이터가 있는 상태에서 다시 클릭하면 이전 데이터는 초기화되고 새로 생성됩니다.

## 4. KNN 설정

오른쪽 사이드바에서 KNN 알고리즘의 핵심 파라미터를 조절합니다.

### 4.1 K값 (이웃 수)

| 항목 | 값 |
|------|-----|
| 범위 | 1 ~ 15 |
| 기본값 | 5 |
| 단위 | 1 단위 |

K값은 분류 시 참조할 이웃의 수입니다.

| K값 | 특성 | 결정 경계 |
|:---:|------|----------|
| **작음** (1~3) | 노이즈에 민감, 과적합 위험 | 울퉁불퉁하고 복잡 |
| **중간** (5~7) | 적절한 균형점 | 비교적 매끄러움 |
| **큼** (9~15) | 안정적이나 세부 패턴 무시 | 매우 부드럽고 단순 |

> 💡 **홀수를 권장합니다.** 2개 클래스일 때 짝수 K를 사용하면 동점이 발생할 수 있습니다.

### 4.2 거리 측정 방식

| 방식 | 수식 | 특성 |
|------|------|------|
| **유클리드** | √((x₁-x₁')² + (x₂-x₂')²) | 직선 거리. 대부분의 상황에 적합 |
| **맨해튼** | \|x₁-x₁'\| + \|x₂-x₂'\| | 축 방향 거리. 격자형 데이터에 적합 |

### 4.3 훈련:테스트 비율

슬라이더로 데이터 분할 비율을 선택합니다.

| 비율 | 훈련 | 테스트 | 특성 |
|:----:|:----:|:------:|------|
| 60:40 | 60% | 40% | 테스트 데이터 많아 평가 신뢰도↑, 훈련 데이터 적어 모델 성능↓ |
| **70:30** | 70% | 30% | 균형 잡힌 기본값 |
| 80:20 | 80% | 20% | 훈련 데이터 많아 모델 성능↑, 테스트 데이터 적어 평가 변동↑ |

## 5. 분류 실행과 평가

### 5.1 실행 조건

**[⚡ 분류 실행]** 버튼을 클릭하면 다음 조건을 확인합니다.

| 조건 | 미충족 시 |
|------|----------|
| 데이터 존재 | "먼저 데이터를 생성해주세요!" 토스트 표시 |
| 데이터 10개 이상 | "데이터가 너무 적습니다! 최소 10개 이상 필요합니다." 토스트 표시 |

### 5.2 실행 과정

분류 실행은 다음 순서로 진행됩니다.

```
[1] 데이터 분할              [2] 차트 갱신              [3] 테스트 평가 애니메이션
    Fisher-Yates 셔플            산점도: Train ●/Test ◆      테스트 점 하나씩 평가
    비율에 따라 분리              결정 경계: 훈련 데이터 기준   ✅/❌ 실시간 표시
    분할 정보 박스 표시                                        프로그레스바 진행

[4] 결과 생성
    K=1~15 전체 테스트 정확도 계산
    현재 K의 혼동행렬 생성
    결과 패널(테이블, 혼동행렬, 차트) 표시
```

### 5.3 테스트 평가 애니메이션

오른쪽 사이드바의 **🔍 테스트 평가 진행** 박스에서 각 테스트 데이터 점이 순차적으로 평가되는 과정을 볼 수 있습니다.

| 요소 | 설명 |
|------|------|
| 프로그레스바 | 전체 평가 진행률 (%) |
| 카운터 | `N / M` 형태 (현재 / 전체) |
| 평가 목록 | 각 점의 ✅(정분류) 또는 ❌(오분류) + 실제→예측 클래스 |

산점도에서는 현재 평가 중인 테스트 점에 **주황색 맥동 링**이 표시되어 어떤 점이 평가 중인지 시각적으로 확인할 수 있습니다.

### 5.4 분할 정보 박스

분류 실행 후 오른쪽 사이드바에 표시됩니다.

| 정보 | 예시 |
|------|------|
| 비율 바 | 초록(훈련) + 파랑(테스트) 색상 바 |
| 훈련 데이터 | `42개 (70%)` |
| 테스트 데이터 | `18개 (30%)` |
| 테스트 정확도 | `83.3%` |

## 6. K값 재생

### 6.1 기능 설명

**[▶️ K값 재생]** 버튼을 클릭하면 K=1 → 3 → 5 → … → 15 순서로 자동 재생됩니다. 각 K값에서 결정 경계가 어떻게 변하는지, 그에 따라 테스트 정확도가 어떻게 달라지는지를 연속으로 관찰할 수 있습니다.

### 6.2 재생 중 표시 정보

| 요소 | 위치 | 내용 |
|------|------|------|
| 프로그레스바 | K값 재생 영역 | 전체 진행률 |
| K값 레이블 | 프로그레스바 아래 | `K=3` |
| 현재 정확도 | 프로그레스바 아래 | `정확도 86.7%` |
| K별 정확도 스트립 | 프로그레스바 아래 | 전체 K의 정확도 칩 나열 |

### 6.3 정확도 스트립 칩

재생이 시작되면 K=1부터 K=15까지 모든 홀수 K의 정확도가 작은 칩으로 표시됩니다.

| 칩 상태 | 의미 | 시각적 표시 |
|---------|------|-----------|
| 일반 | 해당 K의 정확도 | 회색 배경 |
| **파란 테두리** | 현재 재생 중인 K | 파란 테두리 + 연한 파랑 배경 |
| **녹색 테두리** | 최적 K (정확도 최고) | 녹색 테두리 + 연한 초록 배경 |

### 6.4 재생 완료 후

재생이 끝나면 자동으로 결과 패널이 갱신됩니다. 혼동행렬은 **최적 K** 기준으로 표시되며, 토스트 메시지로 최적 K와 그 정확도를 알려줍니다.

> 💡 재생 중 **[⏹️ 정지]** 버튼으로 원하는 K에서 멈출 수 있습니다.

## 7. 결과 해석

### 7.1 결과 패널 구성

분류 실행 또는 K값 재생 완료 후, 중앙 하단에 3열 결과 패널이 표시됩니다.

| 열 | 내용 | 용도 |
|----|------|------|
| **왼쪽** | K값별 테스트 정확도 테이블 | 어떤 K가 가장 좋은지 비교 |
| **가운데** | 혼동행렬 | 어떤 클래스를 잘못 분류했는지 확인 |
| **오른쪽** | K값별 정확도 차트 (막대그래프) | 정확도 추이를 시각적으로 파악 |

### 7.2 K값별 정확도 테이블

K=1, 3, 5, …, 15의 테스트 정확도가 한 번에 표시됩니다.

| 표시 | 의미 |
|------|------|
| 🏆 | 최적 K (정확도 최고) |
| ◀ | 현재 선택된 K |
| 🎯 우수 | 정확도 90% 초과 |
| ✅ 양호 | 정확도 70%~90% |
| ⚠️ 개선필요 | 정확도 70% 이하 |

### 7.3 혼동행렬

혼동행렬(Confusion Matrix)은 행이 **실제 클래스**, 열이 **예측 클래스**인 표입니다.

```
              예측
         A    B    C
실제 A [ 6    0    1 ]  ← 실제 A 중 6개 맞춤, 1개를 C로 오분류
     B [ 0    5    0 ]  ← 실제 B 중 5개 모두 맞춤
     C [ 1    0    5 ]  ← 실제 C 중 5개 맞춤, 1개를 A로 오분류
```

대각선 숫자가 클수록 좋은 결과입니다. 대각선 외의 숫자는 오분류 건수를 나타냅니다.

### 7.4 K값별 정확도 차트

Chart.js 막대그래프로 K값별 테스트 정확도를 시각화합니다. 현재 K의 막대가 강조 표시되며, 전체적인 정확도 추이를 한눈에 파악할 수 있습니다.

## 9. 거리 측정 방식

### 9.1 유클리드 거리 (Euclidean Distance)

두 점 사이의 **직선 거리**입니다. 피타고라스 정리를 확장한 것으로, 대부분의 상황에서 기본값으로 사용합니다.

```
d = √((x₁ - x₁')² + (x₂ - x₂')²)
```

예시: 점 (2, 3)과 점 (5, 7) 사이 → √(9 + 16) = √25 = **5.0**

### 9.2 맨해튼 거리 (Manhattan Distance)

두 점 사이를 **축 방향으로만 이동**하는 거리입니다. 바둑판 도로에서 걷는 거리와 같습니다.

```
d = |x₁ - x₁'| + |x₂ - x₂'|
```

예시: 점 (2, 3)과 점 (5, 7) 사이 → 3 + 4 = **7.0**

### 9.3 어떤 방식을 선택할까?

| 상황 | 추천 |
|------|------|
| 일반적인 경우 | 유클리드 |
| 특성 간 단위가 매우 다를 때 | 맨해튼 |
| 두 방식을 비교해보고 싶을 때 | 둘 다 실행 후 정확도 비교 |

## 10. Train/Test 분할 원리

### 10.1 왜 데이터를 나누는가?

모델의 진짜 실력은 **처음 보는 데이터**에서 발휘됩니다. 훈련에 사용한 데이터로 평가하면 시험 문제를 미리 본 것과 같아서 모델의 진짜 성능을 알 수 없습니다. 이를 **과적합(Overfitting)**이라 합니다.

```
❌ 잘못된 방법: 전체 데이터로 학습 + 전체 데이터로 평가 → 과적합 위험
✅ 올바른 방법: 훈련 데이터로 학습 + 테스트 데이터로 평가 → 일반화 성능 확인
```

### 10.2 분할 과정

1. **Fisher-Yates 셔플**: 데이터를 무작위로 섞습니다
2. **비율 분할**: 설정된 비율(60/70/80%)에 따라 앞부분을 훈련, 나머지를 테스트로 분리합니다
3. **결정 경계**: 훈련 데이터만 사용하여 생성됩니다
4. **정확도 평가**: 테스트 데이터만 사용하여 측정됩니다

### 10.3 시각적 구분

분류 실행 후 산점도에서 훈련과 테스트 데이터가 다른 모양으로 표시됩니다.

| 데이터 유형 | 모양 | 테두리 색상 |
|-----------|:----:|:----------:|
| 훈련 (Train) | ● 원형 | 흰색 |
| 테스트 (Test) | ◆ 다이아몬드 | 주황색 |

### 10.4 최소 데이터 요구

데이터가 10개 미만이면 분할 시 클래스별 대표성이 부족해집니다. 이 시뮬레이터는 **최소 10개 이상**의 데이터를 요구하며, 미달 시 분류 실행이 차단됩니다.

### 11.2 컬럼 구성
