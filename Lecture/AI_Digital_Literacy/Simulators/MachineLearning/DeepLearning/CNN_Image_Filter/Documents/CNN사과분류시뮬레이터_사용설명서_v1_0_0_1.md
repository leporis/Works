## 5. CNN 레이어 설명 패널

왼쪽 하단 **📖 CNN 레이어 설명** 패널에는 CNN 아키텍처의 8개 레이어가 카드 형태로 나열됩니다.

### 5.1 레이어 카드 구성

각 카드에는 3가지 정보가 표시됩니다.

| 요소 | 내용 | 예시 |
|------|------|------|
| 레이어명 | 번호 + 한국어명 + 영문 약칭 | ② 합성곱층 1 (Conv2D) |
| 출력 크기 | 해당 레이어의 출력 텐서 형태 | 62 × 62 × 8 |
| 설명 | 레이어의 역할을 한 문장으로 | 3×3 필터 8개로 에지, 색상 경계 등 저수준 특징을 추출합니다 |

### 5.2 8개 레이어 상세

| # | 레이어 | 출력 크기 | 역할 |
|:-:|--------|----------|------|
| ① | 입력층 (Input) | 64 × 64 × 3 (RGB) | 사과 이미지를 64×64 RGB 픽셀로 변환 |
| ② | 합성곱층 1 (Conv2D) | 62 × 62 × 8 | 3×3 필터 8개로 에지·색상 경계 추출 |
| ③ | 풀링층 1 (MaxPool) | 31 × 31 × 8 | 2×2 최댓값 풀링, 크기 절반 축소 |
| ④ | 합성곱층 2 (Conv2D) | 29 × 29 × 16 | 3×3 필터 16개로 색상 패턴·질감 추출 |
| ⑤ | 풀링층 2 (MaxPool) | 14 × 14 × 16 | 2×2 풀링, 핵심 특징만 유지 |
| ⑥ | 평탄화 (Flatten) | 3,136 | 2D → 1D 벡터 변환 |
| ⑦ | 완전연결층 (Dense) | 64 → 2 | 특징 조합, 분류 확률 계산 |
| ⑧ | 출력층 (Softmax) | 2 | 클래스별 확률 정규화 (합 = 1) |

### 5.3 크기 변화 흐름

```
64×64×3 → [Conv1] → 62×62×8 → [Pool1] → 31×31×8
         → [Conv2] → 29×29×16 → [Pool2] → 14×14×16
         → [Flatten] → 3136 → [Dense] → 64 → 2
```

### 5.4 수식 박스

패널 하단에 두 가지 핵심 수식이 표시됩니다.

**합성곱 출력 크기**: `(W − F + 2P) / S + 1`

- W=64 (입력), F=3 (필터), P=0 (패딩), S=1 (스트라이드) → `(64−3)/1+1 = 62`

**Softmax**: `P(y=k) = e^(z_k) / Σe^(z_j)`

- 두 클래스의 확률 합이 1이 되도록 정규화

### 5.5 클릭 상호작용

레이어 카드를 **클릭**하면 해당 레이어가 **활성화**되고, 중앙의 CNN 아키텍처 시각화에서 대응하는 블록이 **글로우 효과**로 하이라이트됩니다. 학습 중에는 에포크마다 자동으로 순차 하이라이트됩니다.

## 6. 손실 함수 및 정확도 차트

중앙 상단에 가로 2열로 배치된 **📉 손실 함수 (Loss)** 차트와 **📈 정확도 (Accuracy)** 차트는 학습 진행 상황을 실시간으로 시각화합니다.

### 6.1 차트 요소

| 요소 | 설명 |
|------|------|

| 손실 값 | 색상 | 의미 |

## 7. CNN 아키텍처 시각화

중앙의 **🏗️ CNN 아키텍처** 캔버스는 8개 레이어를 블록으로 표현하며, 현재 처리 중인 레이어를 시각적으로 강조합니다.

### 7.1 블록 시각적 표현

각 레이어는 출력 크기에 비례하는 사각형으로 표시됩니다. 블록 크기는 입력층에서 출력층으로 갈수록 작아져, CNN의 특징 추출·압축 과정을 직관적으로 보여줍니다.

| 레이어 | 색상 | 블록 크기 |
|--------|------|----------|
| Input | 🔴 #E53935 | 48×48 |
| Conv1, Conv2 | 🟣 #7B1FA2 | 44×44, 28×28 |
| Pool1, Pool2 | 🟠 #F57C00 | 33×33, 20×20 |
| Flatten | 🔵 #1976D2 | 7×52 (세로 긴 형태) |
| Dense | 🩵 #00897B | 7×32 |
| Output | 🟢 #43A047 | 7×14 |

### 7.2 활성 레이어 효과

현재 활성화된 레이어에는 3가지 시각 효과가 적용됩니다.

| 효과 | 설명 |

| 연결선 활성화 | 인접 레이어와의 연결선이 실선+진한 색상으로 변경 |

## 8. 특성맵 시각화

분류 모드에서 [🔍 CNN 분류 실행]을 누르면 **🔬 특성맵 시각화** 패널에 4단계의 처리 결과가 표시됩니다.

### 8.1 4블록 구성

| 블록 | 필터 수 | 출력 크기 | 시각화 색상 | 설명 |
|------|:------:|----------|:----------:|------|
| Conv1 — 에지 검출 | 8 | 62×62 | 청록색 계열 | 에지, 색상 경계 등 저수준 특징 |
| Pool1 — 맥스풀링 | 8 | 31×31 | 주황색 계열 | Conv1 결과를 2×2로 축소 |
| Conv2 — 패턴 인식 | 8 | 29×29 | 청록색 계열 | 복합 패턴, 질감 등 고수준 특징 |
| Pool2 — 맥스풀링 | 8 | 14×14 | 주황색 계열 | Conv2 결과를 2×2로 축소 |

### 8.2 필터 레이블

**Conv1 / Pool1**: 수평에지, 수직에지, 대각↘, 대각↗, 밝기, 반전, 샤프닝, 블러

**Conv2 / Pool2**: 패턴A, 패턴B, 패턴C, 패턴D, 색상A, 색상B, 텍스처A, 텍스처B

### 8.3 읽는 방법

특성맵의 **밝은 부분**은 해당 필터가 강하게 반응한 영역입니다. 예를 들어, 수평에지 필터의 특성맵에서 사과의 위아래 윤곽이 밝게 나타나면 필터가 수평 경계를 잘 감지한 것입니다.

## 10. 학습 현황 패널

오른쪽 **📊 학습 현황** 패널은 학습 진행 상황을 4칸 그리드와 텍스트 로그로 표시합니다.

### 10.1 통계 그리드

| 항목 | ID | 설명 |
|------|-----|------|
| 에포크 | statEpoch | 현재 학습 반복 횟수 |
| 학습 샘플 | statSamples | 에포크 × 데이터 수 |
| 현재 손실 | statLoss | 최근 에포크의 손실값 (색상으로 수준 표시) |
| 정확도 | statAcc | 최근 에포크의 정확도 (색상으로 수준 표시) |

### 10.2 학습 로그

에포크마다 한 줄씩 로그가 추가됩니다. 형식: `[Epoch 001] loss: 0.8234 | acc: 52.3% | layer: input`

최대 높이 110px까지 표시되며, 자동 스크롤로 최신 로그를 보여줍니다.
