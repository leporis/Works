## 1. 개요

### 1.1 목적

이 활용가이드는 CNN 사과분류 시뮬레이터를 **수업에 적용하는 방법**을 안내합니다. 45분 수업 흐름, 학습 활동지, 평가 질문을 포함하여 교사가 바로 수업에 활용할 수 있도록 구성했습니다.

### 1.2 대상 및 전제

| 항목 | 내용 |
|------|------|
| 교사 대상 | 초등 예비교사 (컴퓨팅 사고와 AI 교육 수강생) |
| 사전 지식 | 머신러닝의 기본 개념 (학습/예측, 지도학습) |
| 장비 | 컴퓨터 1대 + 브라우저 (Chrome 권장) |
| 시뮬레이터 버전 | v1.1.1.0 |

### 1.3 관련 문서

| 문서 | 참조 시점 |
|------|----------|
| 퀵가이드 | 수업 중 학생에게 배포 |
| 사용설명서 | 수업 전 교사 준비 |
| 활용가이드 (본 문서) | 수업 설계 시 |

### 2.2 단계별 교사 스크립트

**🎬 도입 (5분)**

"여러분, 마트에서 사과를 고를 때 무엇을 보나요? 색깔이죠! AI도 색깔을 기준으로 사과를 분류할 수 있어요. 오늘은 CNN이라는 인공지능이 어떻게 사과 사진을 보고 익었는지 판단하는지 직접 체험해볼 거예요."

**📚 개념 (10분)**

시뮬레이터의 📖 CNN 레이어 설명 패널에서 ①~⑧ 카드를 순서대로 클릭하며 설명합니다. 각 카드를 클릭할 때 중앙 CNN 아키텍처 시각화의 블록이 글로우 효과로 하이라이트되는 것을 함께 보여주세요.

핵심 비유:

| 레이어 | 비유 |
|--------|------|
| 합성곱 (Conv) | 🔍 돋보기로 부분을 자세히 살피기 |
| 풀링 (Pool) | 📝 핵심만 메모하고 나머지 버리기 |
| 평탄화 (Flatten) | 📋 메모를 한 줄로 나열하기 |
| 완전연결 (Dense) | 🤔 종합적으로 판단하기 |
| 출력 (Softmax) | 🎯 "익었을 확률 72%, 안 익었을 확률 28%" |

## 3. 활동 1: 학습률 비교 실험

### 3.1 활동 개요

| 항목 | 내용 |
|------|------|
| 목표 | 학습률이 학습 속도와 안정성에 미치는 영향을 이해한다 |
| 시간 | 10분 |
| 모드 | 🧠 학습 모드 |

### 3.2 실험 절차

**공통 설정** (3회 실험 모두 동일):

| 설정 | 값 |
|------|:---:|
| 데이터 수 | 100 |
| 노이즈 강도 | 15% |
| 밝기 변동폭 | 20% |
| 에포크 수 | 30 |
| 시각화 간격 | 매 에포크 |

**3회 실험** (학습률만 변경):

| 실험 | 학습률 | 예상 결과 |
|:----:|:------:|----------|
| A | 0.001 (매우 느림) | 손실이 천천히 내려감, 30 에포크 내 수렴 못할 수 있음 |
| B | 0.01 (보통) | 적당한 속도로 안정적 수렴 |
| C | 0.1 (매우 빠름) | 빠르게 내려가지만 울렁거림 (진동) 발생 가능 |

**순서**: 각 실험마다 [🗑️ 전체 초기화] → [🎲 데이터 생성] → 학습률 변경 → [▶ 시작] → 완료 후 손실 그래프 관찰

### 3.3 기록지

| 실험 | 학습률 | 최종 손실 | 최종 정확도 | 그래프 모양 |
|:----:|:------:|:---------:|:----------:|-----------|
| A | 0.001 | | | (스케치) |
| B | 0.01 | | | (스케치) |
| C | 0.1 | | | (스케치) |

### 3.4 토론 질문

- 학습률이 너무 작으면 어떤 문제가 생기나요?
- 학습률이 너무 크면 왜 그래프가 울렁거리나요?
- "적절한" 학습률은 어떻게 찾을 수 있을까요?

## 5. 활동 3: 분류 도전

### 5.1 활동 개요

| 항목 | 내용 |
|------|------|
| 목표 | 학습된 모델의 분류 능력을 테스트하고 한계를 탐구한다 |
| 시간 | 7~8분 |
| 모드 | 🔮 분류 모드 |

### 5.2 실험 절차

먼저 학습 모드에서 기본 설정으로 학습을 완료한 후, 분류 모드로 전환합니다.

**기본 분류 테스트** (5회):

| 시도 | 사과 타입 | 밝기 | 노이즈 | 회전 | 예측 결과 | 정답? |
|:----:|:--------:|:----:|:------:|:----:|:---------:|:-----:|
| 1 | 🍎 익은 | 0% | 10% | 0° | | |
| 2 | 🍏 덜 익은 | 0% | 10% | 0° | | |
| 3 | 🍎 익은 | -30% | 30% | 20° | | |
| 4 | 🍏 덜 익은 | +30% | 40% | -15° | | |
| 5 | 🎲 랜덤 | — | — | — | | |

**도전 과제**: 모델을 "속일 수 있는" 조합을 찾아보세요!

| 도전 | 조건 | 결과 |
|:----:|------|------|
| 1 | 익은 사과 + 밝기 -50% (매우 어둡게) | |
| 2 | 덜 익은 사과 + 노이즈 50% | |
| 3 | 어떤 조건에서든 오답이 나오는 설정 찾기 | |

### 5.3 토론 질문

- AI가 틀린 경우, 왜 틀렸다고 생각하나요?
- 특성맵에서 어떤 필터가 가장 유용해 보이나요?
- 밝기를 극단적으로 바꾸면 왜 분류가 어려워지나요?
