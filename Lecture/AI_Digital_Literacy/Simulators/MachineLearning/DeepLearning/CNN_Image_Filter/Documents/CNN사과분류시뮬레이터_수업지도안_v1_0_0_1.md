**문서 버전: v1.0.0.1**

## 1. 개요

이 수업지도안은 CNN 사과분류 시뮬레이터를 활용한 **2차시(45분×2) 교수·학습지도안**입니다. 초등 예비교사를 대상으로 합성곱 신경망(CNN)의 개념 이해와 실습을 체계적으로 진행할 수 있도록 설계되었습니다.

| 항목 | 내용 |
|------|------|
| **대상 시뮬레이터** | CNN 사과분류 시뮬레이터 v1.1.1.0 |
| **대상 학습자** | 초등 예비교사 (교육대학교 학부생) |
| **관련 교과** | 컴퓨팅사고와 AI교육 — 딥러닝 단원 |
| **차시 구성** | 1차시: CNN 개념 이해 + 기본 실습 / 2차시: 실험 탐구 + 심화 토론 |
| **소요 시간** | 45분 × 2차시 (총 90분) |
| **수업 환경** | 컴퓨터 실습실 (1인 1PC, 웹 브라우저) |

## 4. 1차시: CNN 개념 이해 + 기본 실습 (45분)

### 4.1 교수·학습 활동

| 단계 | 시간 | 교수 활동 | 학습 활동 | 자료·유의점 |
|:----:|:----:|----------|----------|------------|
| **도입** | 5분 | "이 사과가 익었는지, 컴퓨터는 어떻게 알 수 있을까요?" 질문 제시. 학생 의견 수렴 후 "오늘 CNN이라는 특별한 AI로 사과를 분류해 봅시다" 안내 | 자신만의 사과 판별 기준 발표 (색깔, 크기, 냄새 등) | 🍎 실물 사과 또는 사과 사진 2장(익은/덜 익은) 준비하면 효과적 |
| **개념①** | 5분 | CNN 3단계 비유 설명: 돋보기(합성곱)→메모(풀링)→판단(완전연결). 시뮬레이터 📖 레이어 설명 패널의 8개 카드를 순서대로 클릭하며 시연 | 교사 시연 관찰. "돋보기로 뭘 보나요?" 질문에 응답 | 이론해설서 §3.2 비유 활용. 레이어 카드 클릭 시 아키텍처 하이라이트 연동 강조 |
| **개념②** | 5분 | 학습이란? "처음에는 아무것도 모르지만, 사과 사진을 많이 보면서 패턴을 찾아가는 과정"임을 설명. 순전파→손실→역전파→갱신 순환을 "문제 풀기→채점→오답 노트→재학습"에 비유 | 비유에 대한 이해 확인. "왜 여러 번 반복해야 할까요?" 생각 | 이론해설서 §7.2 참조. 수식 없이 비유 중심으로 설명 |
| **시연** | 5분 | 시뮬레이터 기본 학습 시연: 데이터 생성(100개)→학습률 0.01→에포크 30→학습 시작. 손실 그래프가 내려가는 모습 관찰 유도. 학습 완료 후 분류 모드로 전환하여 사과 분류 시연 | 대형 화면으로 교사 시연 관찰. 손실 그래프 추세 관찰 | ⚠️ 시연 속도 조절: 시각화 간격 "매 에포크" 설정 권장 |
| **실습①** | 10분 | "여러분도 직접 해봅시다!" 학생들이 각자 시뮬레이터를 열고 기본 학습 실행. 순회 지도하며 조작 어려운 학생 도움 | ① 시뮬레이터 열기 ② 퀵가이드 '⚡ 1분 초속성 실습' 따라 하기 ③ 학습 실행 → 완료 확인 ④ 분류 모드에서 사과 3개 분류 | 퀵가이드 배포 필수. 30초 내 성공 경험(데이터 생성→학습→분류) 확보 |
| **실습②** | 10분 | 분류 모드에서 다양한 사과 실험 안내. "빨간색 비율을 바꿔보세요. 밝기를 바꿔보세요. 확률이 어떻게 변하나요?" | 사과 슬라이더 조작하며 분류 결과 관찰. 특성맵 시각화 패널에서 필터별 차이 관찰 | 관찰 결과를 활동지에 간단히 메모하도록 안내 |
| **정리** | 5분 | 핵심 개념 3가지 정리: ① CNN = 돋보기+메모+판단 ② 학습 = 반복해서 패턴 찾기 ③ 손실 감소 = 점점 똑똑해지는 과정. 다음 차시 예고 | 핵심 개념 복창. "오늘 가장 신기했던 점" 한 줄 발표 | "다음 시간에는 학습률을 바꿔보며 CNN이 어떻게 달라지는지 실험해 봅시다" |

```
│                                                                  │

│  🔍 돋보기 (합성곱) → 📝 메모 (풀링) → ⚖️ 판단 (완전연결)      │
```

## 5. 2차시: 실험 탐구 + 심화 토론 (45분)

### 5.1 교수·학습 활동

| 단계 | 시간 | 교수 활동 | 학습 활동 | 자료·유의점 |
|:----:|:----:|----------|----------|------------|
| **복습** | 3분 | "지난 시간에 배운 CNN 3단계를 기억하나요?" 퀵 퀴즈. "손실 그래프가 내려간다는 것은 무슨 뜻인가요?" | CNN 3단계 비유 복창. 손실 감소 = 학습 진행 확인 | 1차시 핵심 3가지를 칠판에 다시 게시 |
| **안내** | 2분 | 활동지 배포. "오늘은 과학자처럼 실험합니다. 하나만 바꾸고 나머지는 그대로 두는 것이 중요해요!" 변인 통제 개념 간단 안내 | 활동지 구성 확인. 실험 절차 읽기 | 활용가이드 §3·§4 기록지 활용 |
| **활동1** | 12분 | **학습률 비교 실험** 안내. 학습률 0.001→0.01→0.1로 3회 학습. "손실 그래프 모양이 어떻게 다른가요?" 순회 지도 | ① 학습률 3종(0.001, 0.01, 0.1)으로 각각 학습 실행 ② 최종 손실·정확도·그래프 모양을 기록지에 기록 ③ 3가지 결과 비교 | ⚠️ 매 실험 전 [🗑️ 전체 초기화] 필수. 에포크 30, 데이터 100개 고정. 시각화 간격 5 권장 (속도) |
| **발표①** | 3분 | 모둠별 또는 개인별 실험 결과 발표 요청. "0.1에서 그래프가 출렁거린 학생 있나요?" 핵심 관찰 공유 유도 | 기록지 기반으로 결과 발표. "0.001은 너무 느리고, 0.1은 불안정하고, 0.01이 적당했어요" 등 | 학습률 비유: "걸음 크기" 비유로 정리. 이론해설서 §7.3 참조 |
| **활동2** | 10분 | **노이즈 영향 실험** 안내. 노이즈 0%→25%→50%로 3회 학습 + 분류. "노이즈가 높으면 분류가 잘 안 되는 이유는?" | ① 노이즈 3종으로 각각 학습 후 분류 모드에서 사과 3개 분류 ② 기록지에 정확도·분류 성공 횟수·특성맵 선명도 기록 | 학습률 0.01, 에포크 30 고정. "실생활의 노이즈 = 흐린 사진, 역광, 그림자"로 연결 |
| **발표②** | 3분 | 노이즈 실험 결과 발표 요청. "깨끗한 데이터와 노이즈 많은 데이터, 어느 쪽이 더 정확했나요?" | 실험 결과 공유. "노이즈가 높으면 특성맵이 흐려지고 분류도 틀린다" 등 | ⚠️ 50% 노이즈에서 결과가 제각각일 수 있음 → "데이터 품질이 중요하다"는 메시지 |
| **심화** | 5분 | 특성맵 해석 활동. "분류 모드에서 익은 사과와 덜 익은 사과의 Conv1 특성맵을 비교해 봅시다. 어떤 필터에서 차이가 가장 큰가요?" | 두 종류 사과의 특성맵을 비교 관찰. "수평에지 필터에서 차이가 크다" 등 발견 공유 | 시뮬레이터 FILTER_NAMES_1 참조: 수평에지, 수직에지, 대각↘, 대각↗, 밝기, 반전, 샤프닝, 블러 |
| **토론** | 5분 | 심화 질문 제시: "① 사과 말고 다른 것도 분류할 수 있을까? ② 자율주행차의 신호등 인식도 비슷한 원리일까? ③ AI가 모든 사진을 완벽하게 분류할 수 있을까?" | 모둠 토론 후 대표 발표. CNN의 가능성과 한계 토론 | 정답보다 사고 과정을 중시. "AI는 데이터에서 패턴을 찾는 도구이지, 만능이 아니다"로 정리 |
| **정리** | 2분 | 전체 핵심 정리: ① 학습률 = 걸음 크기 ② 데이터 품질이 중요 ③ CNN은 강력하지만 한계도 있다. 소감 한 줄 발표 | "오늘 실험에서 가장 인상 깊었던 점" 한 줄 발표 | 활동지 수거 (평가 자료 활용) |


### 6.2 수준별 성취 기준

| 수준 | 개념 이해 | 실습 수행 | 탐구 능력 |
|:----:|----------|----------|----------|
| **상** | CNN 구조와 학습 과정을 비유와 함께 자신의 언어로 설명. 하이퍼파라미터의 역할을 이해 | 모든 기능을 독립적으로 활용. 다양한 설정을 자발적으로 실험 | 3회 실험을 정확히 수행하고 결과를 분석적으로 비교·해석 |
| **중** | CNN 3단계를 비유로 설명 가능. 학습률의 영향을 대략적으로 이해 | 교사 안내에 따라 학습·분류를 수행 가능 | 3회 실험을 수행하고 결과를 기록. 기본적인 비교 가능 |
| **하** | CNN이 이미지를 분류하는 AI라는 것을 이해. 학습 = 반복 연습이라는 것을 인지 | 교사 도움으로 기본 학습·분류를 수행 가능 | 교사 안내에 따라 실험을 수행하고 결과를 기록 |

## 7. 지도상 유의점

### 7.1 수업 운영

| # | 유의점 |
|:-:|--------|
| 1 | 수식을 최소화하고 **비유 중심**으로 설명합니다. "합성곱 = 돋보기", "풀링 = 메모 요약", "학습 = 문제 풀기 반복" 등의 비유를 일관되게 사용합니다 |
| 2 | 1차시 실습에서 **30초 내 첫 성공 경험**을 확보합니다. 퀵가이드 초속성 실습(데이터 생성→학습→분류)을 먼저 따라 하게 하여 자신감을 얻게 합니다 |
| 3 | 2차시 실험에서 **변인 통제** 개념을 강조합니다. "학습률만 바꾸고, 데이터 수·에포크·노이즈는 그대로!" 규칙을 반복 안내합니다 |
| 4 | 매 실험 전 **[🗑️ 전체 초기화]**를 반드시 실행하도록 안내합니다. 이전 학습이 남아 있으면 결과가 왜곡됩니다 |
| 5 | 학생마다 학습 결과가 다를 수 있습니다(랜덤 초기화). 이를 "같은 사람도 공부할 때마다 결과가 조금 다른 것"에 비유하여 자연스럽게 수용하도록 합니다 |

### 7.2 기술적 유의점

| # | 상황 | 대처 |
|:-:|------|------|
| 1 | 학습이 시작되지 않음 | [🎲 학습 데이터 생성] 버튼을 먼저 눌렀는지 확인 |
| 2 | 손실이 줄어들지 않음 | 학습률 0.01 또는 0.005로 변경 후 재학습 |
| 3 | 분류 결과가 계속 50:50 | 학습이 완료되었는지 확인. 미완료면 에포크 30 이상으로 재학습 |
| 4 | 브라우저가 느려짐 | 시각화 간격을 "5 에포크마다" 또는 "10 에포크마다"로 변경 |
| 5 | 특성맵이 표시되지 않음 | 분류 모드에서 [🔍 CNN 분류 실행]을 클릭해야 특성맵이 생성됨 |

### 7.3 학습자 수준 대응

| 학습자 수준 | 대응 전략 |
|:----------:|----------|
| 빠른 학습자 | 활동 3(분류 도전) 제공. "데이터 수 20개로 줄이면?", "에포크 100으로 늘리면?" 추가 실험 안내 |
| 보통 학습자 | 활동 1·2를 기록지와 함께 순서대로 진행. 핵심 관찰 포인트를 구두로 안내 |
| 느린 학습자 | 퀵가이드 초속성 실습만 반복 성공시킨 후, 활동 1의 학습률 0.001 vs 0.1 두 가지만 비교 |
