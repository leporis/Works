# 🍎 CNN 사과분류 시뮬레이터 개발일지

**문서 버전: v1.0.0.0**

---

## 1. 프로젝트 개요

### 1.1 목적

합성곱 신경망(CNN)의 동작 원리를 **시각적·체험적**으로 이해할 수 있는 교육용 시뮬레이터입니다. 초등 예비교사를 대상으로 CNN의 핵심 개념(합성곱, 풀링, 평탄화, 완전연결, Softmax)을 **사과 분류**라는 친숙한 맥락에서 학습할 수 있도록 설계했습니다.

### 1.2 교육 목표

| 수준 | 목표 |
|------|------|
| 이해 | CNN이 이미지를 처리하는 8단계 파이프라인의 역할과 흐름을 설명할 수 있다 |
| 체험 | 학습률·에포크·노이즈 등 하이퍼파라미터를 직접 조절하며 학습 결과 변화를 관찰한다 |
| 확인 | 학습된 모델로 새 사과 이미지를 분류하고, 정확도에 영향을 미치는 요인을 분석한다 |

### 1.3 배경

- **대상 과목**: 컴퓨팅 사고와 AI 교육 (청주교육대학교)
- **학습자**: 초등 예비교사 (비전공자)
- **핵심 알고리즘**: CNN (Convolutional Neural Network) — 이미지 분류
- **도메인**: 익은 사과 🍎 vs 덜 익은 사과 🍏 이진 분류

## 2. 시스템 설계

### 2.1 모드 구성

시뮬레이터는 **2개 모드**로 구성됩니다.

| 모드 | 목적 | 주요 패널 |
|------|------|----------|
| 🧠 학습 모드 | CNN 모델 훈련 과정 관찰 | 학습 데이터, 학습 설정, 손실/정확도 차트, CNN 아키텍처, 학습 현황, 학습 결과 |
| 🔮 분류 모드 | 학습된 모델로 새 이미지 분류 | 입력 이미지, 특성맵 시각화, 분류 결과 |

### 2.2 입출력 설계

**학습 모드 입력:**

| 파라미터 | 범위 | 기본값 | 역할 |
|---------|------|:------:|------|
| 데이터 수 | 20~300 (step 10) | 100 | 학습 데이터셋 크기 |
| 노이즈 강도 | 0~50% | 15% | 학습 데이터 품질 |
| 밝기 변동폭 | 0~50% | 20% | 이미지 다양성 |
| 학습률 | 0.001 / 0.005 / 0.01 / 0.05 / 0.1 | 0.01 | 경사하강법 스텝 크기 |
| 에포크 수 | 10 / 30 / 50 / 100 | 30 | 학습 반복 횟수 |
| 시각화 간격 | 1 / 2 / 5 / 10 | 5 | 차트 갱신 주기 |

**학습 모드 출력:**

| 출력 | 설명 |
|------|------|
| 손실 함수 차트 | 에포크별 Cross-Entropy 손실 변화 곡선 |
| 정확도 차트 | 에포크별 분류 정확도 변화 곡선 |
| CNN 아키텍처 시각화 | 8개 레이어 블록의 실시간 하이라이트 |
| 학습 현황 | 에포크, 학습 샘플 수, 현재 손실, 정확도 |
| 학습 로그 | 에포크별 loss / accuracy / 처리 레이어 텍스트 |
| 학습 결과 (3열) | 모델 성능 · 학습된 가중치 · 학습 설정 요약 |

**분류 모드 입력:**

| 파라미터 | 범위 | 역할 |
|---------|------|------|
| 사과 타입 | 🍎 익은 / 🍏 덜 익은 | 정답 레이블 |
| 밝기 변화 | -50~+50% | 입력 이미지 밝기 조절 |
| 노이즈 강도 | 0~50% | 분류 난이도 조절 |
| 회전 각도 | -30°~+30° | 이미지 회전 변환 |

**분류 모드 출력:**

| 출력 | 설명 |
|------|------|
| 특성맵 시각화 | Conv1 → Pool1 → Conv2 → Pool2 각 8개 필터 출력 |
| 분류 확률 바 | 🍎 익은 사과 / 🍏 덜 익은 사과 확률 |
| 판정 결과 | 예측 클래스 + 정답 여부 (✅/❌) |

### 2.3 CNN 아키텍처 (8층 구조)

| # | 레이어 | 출력 크기 | 역할 |
|:-:|--------|----------|------|
| ① | 입력층 (Input) | 64 × 64 × 3 | 사과 이미지를 RGB 픽셀로 변환 |
| ② | 합성곱층 1 (Conv2D) | 62 × 62 × 8 | 3×3 필터 8개로 에지·색상 경계 추출 |
| ③ | 풀링층 1 (MaxPool) | 31 × 31 × 8 | 2×2 영역 최댓값 선택, 크기 절반 축소 |
| ④ | 합성곱층 2 (Conv2D) | 29 × 29 × 16 | 3×3 필터 16개로 색상 패턴·질감 추출 |
| ⑤ | 풀링층 2 (MaxPool) | 14 × 14 × 16 | 2×2 풀링으로 핵심 특징만 유지 |
| ⑥ | 평탄화 (Flatten) | 3,136 | 2D 특성맵 → 1D 벡터 변환 |
| ⑦ | 완전연결층 (Dense) | 64 → 2 | 특징 조합하여 분류 확률 계산 |
| ⑧ | 출력층 (Softmax) | 2 | 클래스별 확률 정규화 (합 = 1) |

**핵심 수식:**

- 합성곱 출력 크기: `(W − F + 2P) / S + 1` → `(64 − 3) / 1 + 1 = 62`
- Softmax: `P(y=k) = e^(z_k) / Σe^(z_j)`

### 2.4 합성곱 필터 (Conv1)

| # | 필터명 | 3×3 커널 | 용도 |
|:-:|--------|---------|------|
| 0 | 수평에지 | `[[-1,-1,-1],[0,0,0],[1,1,1]]` | 수평 경계 검출 |
| 1 | 수직에지 | `[[-1,0,1],[-1,0,1],[-1,0,1]]` | 수직 경계 검출 |
| 2 | 대각↘ | `[[-1,-1,0],[-1,0,1],[0,1,1]]` | 대각선 경계 |
| 3 | 대각↗ | `[[0,-1,-1],[1,0,-1],[1,1,0]]` | 반대 대각선 |
| 4 | 밝기 | `[[0,0,0],[0,2,0],[0,0,0]]` | 밝기 강조 |
| 5 | 반전 | `[[0,0,0],[0,-1,0],[0,0,0]]` | 색상 반전 |
| 6 | 샤프닝 | `[[0,-1,0],[-1,5,-1],[0,-1,0]]` | 선명화 |
| 7 | 블러 | `[[1,1,1],[1,1,1],[1,1,1]]` | 흐림 (평균) |

## 3. 주요 기능

### 3.1 학습 모드

| 기능 | 설명 |
|------|------|
| 학습 데이터 생성 | 데이터 수·노이즈·밝기 슬라이더로 가상 학습 데이터 생성 |
| 학습 제어 (▶⏸⏭) | 시작/일시정지/단계별 실행 3종 컨트롤 |
| 실시간 차트 | Canvas API로 손실·정확도 곡선을 에포크마다 실시간 갱신 |
| CNN 아키텍처 시각화 | 8개 레이어 블록을 Canvas로 그리며, 현재 처리 단계를 글로우 효과로 하이라이트 |
| 레이어 설명 패널 | 8개 레이어 카드 (클릭 시 아키텍처 동기화), 출력 크기·역할 표시 |
| 학습 현황 | 에포크, 학습 샘플 수, 손실, 정확도 4칸 그리드 + 텍스트 로그 |
| 학습 과정 재생 | 완료 후 에포크별 스냅샷을 0.5x~5x 속도로 재생 |
| 학습 결과 패널 | 모델 성능·학습 가중치·학습 설정 3열 요약 |

### 3.2 분류 모드

| 기능 | 설명 |
|------|------|
| 사과 이미지 생성 | Canvas API로 익은/덜 익은 사과를 베지어 곡선으로 렌더링 |
| 이미지 변환 | 밝기·노이즈·회전 3가지 변환 실시간 적용 |
| 랜덤 샘플 | 타입·밝기·노이즈·회전을 무작위로 설정 |
| CNN 분류 실행 | 8단계 처리 과정을 오버레이 애니메이션으로 시각화 후 분류 |
| 특성맵 시각화 | Conv1·Pool1·Conv2·Pool2 각 8개 필터의 출력을 실시간 렌더링 |
| 분류 결과 | 익은/덜 익은 확률 바 + 판정 결과 + 정답 여부 표시 |

### 3.3 공통 기능

| 기능 | 설명 |
|------|------|
| 모드 전환 | 학습 ↔ 분류 탭 전환, 패널 표시/숨김 자동 관리 |
| CNN 아키텍처 | 양쪽 모드 공통 표시, 레이어 하이라이트 동기화 |
| 전체 초기화 | 모든 상태(가중치, 이력, 차트, UI)를 초기 상태로 복원 |
| 반응형 레이아웃 | 1100px 이하에서 1열 레이아웃으로 자동 전환 |

### 5.4 핵심 함수

| 함수 | 역할 |
|------|------|
| `init()` | 가중치 초기화, 특성맵 초기화, 아키텍처 그리기, 차트 초기화 |
| `generateTrainData()` | 학습 데이터 가상 생성 (데이터 수 표시) |
| `startTraining()` | 학습 시작/재개, 에포크 루프 시작 |
| `runSingleEpoch()` | 1 에포크 실행: 가중치 업데이트, 손실/정확도 계산, UI 갱신 |
| `finishTraining()` | 학습 완료 처리: 결과 패널 표시, 재생 패널 활성화 |
| `replayTraining()` | 에포크 스냅샷을 기반으로 학습 과정 재생 |
| `drawChart()` | Canvas 기반 실시간 차트 (면적 채우기 + 선 + 현재값 점) |
| `drawArchitecture()` | CNN 8개 레이어 블록을 Canvas로 시각화 |
| `highlightLayer(id)` | 특정 레이어에 글로우 효과 + 설명 패널 동기화 |
| `drawApple()` | 베지어 곡선 + 그라데이션으로 사과 이미지 생성 |
| `classifyApple()` | 8단계 오버레이 애니메이션 후 분류 실행 |
| `performClassification()` | RGB 분석 → 로지스틱 → Softmax 확률 계산 |
| `generateFeatureMaps()` | Conv1→Pool1→Conv2→Pool2 단계별 특성맵 생성 |
| `applyFilter()` | 3×3 커널 합성곱 연산 (ReLU 활성화) |
| `maxPool()` | 2×2 맥스풀링 다운샘플링 |
| `renderMaps()` | 특성맵 배열을 Canvas에 컬러맵으로 렌더링 |
| `resetAll()` | 전체 상태·UI 초기화 |

### 5.5 학습 시뮬레이션 알고리즘

학습 과정은 경사하강법을 간소화한 시뮬레이션으로, 가중치가 목표값에 점진적으로 수렴하는 과정을 보여줍니다.

**가중치 업데이트:**

```javascript
w.redWeight += (TARGET_RED_W - w.redWeight) * lr * (0.8 + noise());
```

**손실 함수:**

```javascript
const baseLoss = 0.85 * Math.exp(-epoch * lr * 3) + dataNoise * 0.1;
const loss = Math.max(0.02, baseLoss + random_perturbation);
```

**분류 확률 (로지스틱):**

```javascript
let logit = w.redWeight*aR + w.greenWeight*aG + w.textureWeight*(aR-aG) + w.bias;
let ripeProb = 1 / (1 + Math.exp(-logit * (2 + epoch*0.3)));
```

### 5.6 사과 이미지 렌더링

사과 이미지는 외부 리소스 없이 Canvas API만으로 생성됩니다.
