## 1. 시뮬레이터 소개

### 1.1 목적

RL 틱택토는 **Q-Learning 강화학습** 알고리즘이 틱택토 게임을 스스로 학습하는 과정을 실시간으로 관찰하고, 직접 AI와 대국할 수 있는 교육용 시뮬레이터입니다.

### 1.2 주요 특징

| 특징 | 설명 |
|------|------|
| 실시간 학습 관찰 | 그래프와 지표로 AI의 성장 과정을 눈으로 확인 |
| 대칭성 정규화 | 8가지 보드 대칭 변환으로 학습 효율 8배 향상 |
| 6가지 모드 | 자기대국, X 단독, O 단독, AI 대전, 사람 대전, 유휴 |
| Q-값 히트맵 | AI의 판단 근거를 색상으로 시각화 |
| 난이도 조절 | 0~100% 슬라이더로 AI 강도 세밀 조절 |
| 다크/라이트 테마 | 환경에 따라 즉시 전환 |
| 저장/불러오기 | 학습 데이터를 보존하고 공유 |

### 1.3 문서 안내

| 문서 | 목적 | 참조 시점 |
|------|------|----------|
| **사용설명서 (본 문서)** | 전체 기능 상세 설명 | 사용 전/후 |
| 퀵가이드 | 즉시 실행, 빠른 참조 | 사용 중 |
| 이론해설서 | Q-Learning 원리 학습 | 별도 학습 |
| 활용가이드 | 수업 적용 방법 | 수업 준비 |
| 개발일지 | 기술 설계, 아키텍처 | 개발 참고 |

## 3. 화면 구성 — 대시보드

대시보드는 시뮬레이터의 메인 화면으로, 좌측 컨트롤 패널과 우측 모니터링 영역으로 나뉩니다.

### 3.1 헤더 영역

| 요소 | 기능 |
|------|------|
| 🤖 RL 틱택토 | 시뮬레이터 제목 (파랑→빨강 그라데이션) |
| ☀️/🌙 | 다크/라이트 테마 토글 |
| ❓ 도움말 | 사용 가이드 모달 표시 |
| 📋 보고서 | Markdown 학습 보고서 다운로드 (100판 이상 시 활성화) |

### 3.2 좌측 컨트롤 패널 (160px)

#### 미니 보드

현재 게임 보드 상태를 소형(14px 셀)으로 표시합니다. 학습 중에는 실시간으로 갱신되지 않으며, 마지막 게임의 결과를 보여줍니다.

#### 속도 슬라이더

| 설정값 | 배치 크기 | 체감 속도 |
|:------:|:---------:|----------|
| 1ms | 500판/틱 | 가장 빠름 — 초당 ~10,000판 이상 |
| 5ms | 200판/틱 | 빠름 |
| 20ms | 50판/틱 | 보통 |
| 100ms | 10판/틱 | 느림 — 그래프 변화 세밀 관찰 |

> 💡 빠르게 학습시키려면 **1ms**, 그래프 변화를 자세히 보려면 **50~100ms**가 적당합니다.

#### 학습 방식 (📋)

| 방식 | X 에이전트 | O 에이전트 | 용도 |
|------|:----------:|:----------:|------|
| 🔄 자기대국 | 학습 | 학습 | 양쪽 AI가 서로 배우며 함께 성장 |
| 🔵 X 단독 | 학습 | 랜덤 | X만 집중 학습 (선공 전략 특화) |
| 🔴 O 단독 | 랜덤 | 학습 | O만 집중 학습 (후공 전략 특화) |

자기대국이 기본이며 가장 균형 잡힌 학습 결과를 제공합니다.

#### 파라미터 (⚙️)

세 가지 핵심 파라미터를 슬라이더로 조절합니다. **🔗동기** 모드에서는 X와 O가 같은 값을 사용하고, **🔓별도** 모드에서는 각각 다른 값을 설정할 수 있습니다.

| 파라미터 | 범위 | 기본값 | 의미 |
|----------|:----:|:------:|------|
| 학습률 α | 0.01~1.0 | 0.30 | 한 번의 경험에서 얼마나 배울지 |
| 할인율 γ | 0.1~1.0 | 0.90 | 미래 보상을 얼마나 중요하게 볼지 |
| ε 감소율 | 0.99~1.0 | 0.9995 | 탐험을 줄이는 속도 |

#### 액션 버튼

| 버튼 | 색상 | 기능 |
|------|:----:|------|
| ▶️ 학습 시작 / ⏹ 정지 | 🟢/🔴 | 학습 루프 시작/정지 |
| ⚔️ AI 대전 | 🟠 | 학습된 두 AI의 대국 관전 |
| 👤 사람 대전 | 🟣 | 사용자가 AI와 직접 대국 |

#### 저장/불러오기 버튼

| 아이콘 | 기능 |
|:------:|------|
| 💾 | 브라우저 localStorage에 저장 |
| 📤 | JSON 파일로 내보내기 |
| 📂 | localStorage에서 불러오기 |
| 📥 | JSON 파일 가져오기 |

#### 초기화 (🗑️)

모든 학습 데이터를 삭제합니다. **확인 모달**이 표시되어 실수로 삭제하는 것을 방지합니다. 1판 이상 학습해야 활성화됩니다.

### 3.3 우측 대시보드

#### 학습 단계 진행바

| 단계 | 아이콘 | 에피소드 범위 | 색상 | 의미 |
|------|:------:|:------------:|:----:|------|
| 초기 | 🌱 | 0 ~ 999 | 🔴 빨강 | Q-테이블 초기화, 대부분 랜덤 행동 |
| 탐색 | 🔍 | 1,000 ~ 9,999 | 🟠 주황 | 탐험과 활용의 균형기 |
| 학습 | 📚 | 10,000 ~ 29,999 | 🟡 노랑 | 전략이 구체화되는 시기 |
| 응용 | 🎯 | 30,000 ~ 49,999 | 🟢 초록 | 안정적인 전략 형성 |
| 숙련 | 🏆 | 50,000+ | 🔵 파랑 | 최적에 가까운 플레이 |

진행바 위에는 현재 단계 이름, 학습 중 여부(● 학습 중), 에피소드 수, 초당 학습 속도가 표시됩니다.

#### 지표 카드 (6개)

| 카드 | 아이콘 | 표시 내용 |
|------|:------:|----------|
| X 승률 | 🔵 | 최근 배치의 X 평균 승률 (%) |
| O 승률 | 🔴 | 최근 배치의 O 평균 승률 (%) |
| 무승부 | 🤝 | 최근 배치의 무승부율 (%) |
| ε | 🎲 | 현재 탐험률 (%) |
| 강도 | ⭐ | X 에이전트의 대(對) 랜덤 승률 |
| 등급 | 🎖️ | 강도 기반 등급 (F~S+) |

#### 그래프 (4종)

| 위치 | 그래프 | 색상 | 좋은 방향 |
|------|--------|:----:|:---------:|
| 좌상 | X/O 승률 추이 | 🔵/🔴 | ↑ |
| 우상 | 무승부율/탐험률 추이 | 🟡/🟣 | ↑/↓ |
| 좌하 | 강도/Q-상태 수 추이 | 🟢/🔵 | ↑/↑ |
| 우하 | 인사이트 + 학습 통계 | — | — |

각 그래프는 최근 500개 데이터 포인트를 표시하며, 바 형태로 시각화됩니다.

#### 인사이트 패널

**첫수 선호도**: X 에이전트가 빈 보드에서 각 위치 유형(중앙/모서리/변)에 부여한 평균 Q-값을 막대 그래프로 표시합니다. 최적 학습 시 🎯 중앙 > 📐 모서리 > ➖ 변 순서가 됩니다.

**학습 통계 테이블**:

| 항목 | 내용 |
|------|------|
| X/O Q-상태 | 각 에이전트가 학습한 고유 상태 수 |
| X/O ε | 각 에이전트의 현재 탐험률 |
| 학습 속도 | 초당 처리 에피소드 수 |
| Step 패널티 | 매 수마다 부과되는 패널티 (-0.01) |

## 4. 화면 구성 — 사람 대전

**[👤 사람 대전]** 버튼을 클릭하면 전환됩니다.

### 4.1 좌측: 게임 보드

3×3 보드(셀 크기 75px)가 화면 중앙에 표시됩니다.

| 요소 | 의미 |
|------|------|
| 빈 칸 | 클릭하여 돌을 놓을 수 있는 위치 |
| 빈 칸의 숫자 | Q-값 — AI가 해당 위치에 대해 평가한 가치 |
| 파란 배경 칸 | X가 놓인 위치 |
| 붉은 배경 칸 | O가 놓인 위치 |
| 시안 테두리 칸 | AI가 마지막으로 둔 위치 |
| 노란 테두리/빛 | 승리 라인 (3목 완성) |

### 4.2 차례 및 결과 표시

| 상태 | 표시 |
|------|------|
| 내 차례 | 🟢 당신의 차례 |
| AI 차례 | 🤔 AI 생각 중... |
| 승리 | 🎉 승리! (초록색) |
| 무승부 | 🤝 무승부! (노란색) |
| 패배 | 😔 패배... (빨간색) |

### 4.3 컨트롤

| 버튼/슬라이더 | 기능 |
|--------------|------|
| 🔄 새 게임 | 보드 초기화, 새 게임 시작 |
| 🔀 O로/X로 변경 | 선공/후공 전환 (게임도 리셋) |
| 🎚️ AI 난이도 (0~100%) | AI가 최적수를 두는 확률 |

**난이도 체계**:

| 범위 | 이름 | 색상 | 설명 |
|:----:|------|:----:|------|
| 0~19% | 🐣 초보 | 🟢 | AI가 거의 랜덤 — 누구나 이길 수 있음 |
| 20~49% | 중급 | 🟡 | 가끔 좋은 수를 둠 |
| 50~79% | 고급 | 🟠 | 대부분 최적수 — 실수를 유도해야 이김 |
| 80~100% | 🤖 최강 | 🔴 | 거의 완벽 — 충분히 학습했다면 무승부만 가능 |

### 4.4 우측: 정보 패널

**학습 단계 + 지표**: 현재 학습 단계(아이콘, 이름, 에피소드), 강도, 등급, ε, Q-상태 수를 2×2 카드로 표시합니다.

**🧠 AI 판단 근거 (Q-값 히트맵)**: 3×3 그리드에서 AI가 각 빈 칸에 대해 평가한 Q-값을 색상과 숫자로 표시합니다.

| 표시 | 의미 |
|------|------|
| 🟢 높은 값 (초록) | AI가 유리하다고 판단하는 위치 |
| 🔴 낮은 값 (빨강) | AI가 불리하다고 판단하는 위치 |
| ⭐최선 + 노란 테두리 | AI가 가장 좋다고 판단하는 위치 |

### 4.5 승/무/패 기록

현재 세션의 대국 기록이 하단에 표시됩니다. 새 게임을 시작해도 누적되며, 대시보드로 돌아가면 초기화됩니다.

### 5.1 좌측: 게임 보드

학습된 Agent X와 Agent O가 자동으로 대국합니다. 셀 크기 70px, AI의 마지막 수가 시안 테두리로 강조됩니다.

현재 차례인 에이전트의 판단 근거를 실시간으로 표시합니다. AI가 둘 때마다 히트맵이 갱신됩니다.

한 판이 끝나면 결과(X 승리/O 승리/무승부)가 표시된 후 2초 뒤 자동으로 다음 판이 시작됩니다.

## 6. 학습 기능 상세

### 6.1 Q-Learning 알고리즘

| 항목 | 내용 |
|------|------|
| 갱신 식 | Q(s,a) ← Q(s,a) + α × (r - Q(s,a)) |
| 행동 선택 | ε-greedy: ε 확률로 랜덤, (1-ε) 확률로 최적 |
| ε 감소 | 매 배치마다: ε ← max(0.01, ε × 감소율) |
| 이력 갱신 | 에피소드 종료 후 역방향(마지막 수→첫 수) 순회 |

### 6.2 대칭성 정규화

보드의 8가지 대칭 변환(항등, 90°, 180°, 270° 회전 × 좌우 반사)을 적용하여 동일 모양의 보드를 하나의 정규 상태로 통합합니다. 행동을 선택할 때는 정규 좌표계에서 선택한 뒤 역변환하여 원본 좌표로 반환합니다.

### 6.3 보상 체계

| 결과 | 보상 | 비고 |
|------|:----:|------|
| 승리 | +1.0 | 게임에서 이긴 에이전트 |
| 패배 | -1.0 | 진 에이전트 |
| 무승부 | +0.3 | 양쪽 모두 |
| 매 수 | -0.01 | 단계 패널티 — 빠른 승리 유도 |

### 6.4 강도 측정

2,000판마다 X 에이전트가 완전 랜덤 상대와 100판을 치러 승률을 측정합니다. 이 값이 ⭐ 강도이며, 등급 산정의 기준입니다.

| 등급 | 강도 범위 | 의미 |
|:----:|:---------:|------|
| F | 0~19 | 랜덤 수준 |
| D | 20~39 | 기초 전략 형성 |
| C | 40~59 | 중급 수준 |
| B | 60~74 | 고급 수준 |
| A | 75~89 | 거의 최적 |
| S | 90~95 | 최적 근접 |
| S+ | 96~100 | 이론적 최적 |

## 7. 파라미터 상세

### 7.1 학습률 α (Alpha)

| 값 | 효과 |
|:--:|------|
| 0.01~0.1 | 안정적이지만 느린 학습. 그래프가 부드럽게 변함 |
| 0.1~0.5 | 적당한 속도와 안정성의 균형 |
| 0.5~1.0 | 매우 빠르지만 불안정. 그래프가 요동칠 수 있음 |

### 7.2 할인율 γ (Gamma)

| 값 | 효과 |
|:--:|------|
| 0.1~0.3 | 눈앞의 보상만 중시 — 단기적 플레이 |
| 0.5~0.8 | 균형 잡힌 전략 |
| 0.9~1.0 | 미래 보상을 크게 반영 — 장기적 전략 |

### 7.3 ε 감소율 (Epsilon Decay)

| 값 | 효과 |
|:--:|------|
| 0.99~0.995 | 빠르게 탐험 종료 — 일찍 활용 전환 |
| 0.9995 (기본) | 적당한 탐험 기간 |
| 0.9999~1.0 | 오래 탐험 — 더 많은 상태 탐색, 느린 수렴 |

### 7.4 동기/별도 모드

**🔗동기**: X와 O가 같은 파라미터를 사용합니다 (기본).

**🔓별도**: X와 O에 서로 다른 파라미터를 설정할 수 있습니다. 예를 들어 X는 α=0.5로 공격적으로, O는 α=0.1로 신중하게 학습시킬 수 있습니다.

## 12. 자주 묻는 질문

### Q. AI를 절대 이길 수 없어요.
충분히 학습한 AI(S등급 이상)는 이론적으로 무승부가 최선입니다. 난이도를 50% 이하로 낮추면 AI가 가끔 실수하여 이길 기회가 생깁니다.

### Q. 그래프가 심하게 흔들려요.
학습률(α)이 너무 높으면 그래프가 요동칩니다. α를 0.1 이하로 낮춰 보세요.

### Q. 학습 속도가 너무 느려요.
속도를 1ms로 설정하세요. 초당 10,000판 이상 처리됩니다.

### Q. AI를 여러 컴퓨터에서 공유하고 싶어요.
📤 파일 내보내기로 JSON 파일을 저장하고, 다른 컴퓨터에서 📥 파일 가져오기로 불러오세요.

### Q. 학습 데이터가 사라졌어요.
브라우저의 "사이트 데이터 삭제" 기능을 사용하면 localStorage가 함께 삭제됩니다. 중요한 학습 데이터는 반드시 📤 파일 내보내기로 별도 보존하세요.

### Q. 왜 X가 O보다 항상 유리한가요?
틱택토에서 선공(X)이 후공(O)보다 구조적으로 유리합니다. 자기대국 학습 시 X 승률이 O보다 높은 것은 정상입니다. 양쪽 모두 최적 학습 후에는 100% 무승부가 됩니다.
