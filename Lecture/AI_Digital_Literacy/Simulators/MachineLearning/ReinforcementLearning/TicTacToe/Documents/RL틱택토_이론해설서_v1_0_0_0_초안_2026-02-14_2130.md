## 1. 강화학습이란?

### 1.1 세 가지 학습 방법

머신러닝에는 크게 세 가지 학습 방법이 있습니다.

| 학습 방법 | 핵심 | 비유 | 예시 |
|----------|------|------|------|
| 지도학습 | 정답이 있는 데이터로 학습 | 선생님이 정답을 알려줌 | 사과 가격 예측, 과일 분류 |
| 비지도학습 | 정답 없이 패턴을 찾음 | 스스로 분류하기 | K-means 군집화 |
| **강화학습** | **시행착오로 보상을 최대화** | **게임하며 스스로 배움** | **틱택토, 바둑, 로봇 제어** |

강화학습(Reinforcement Learning, RL)은 **에이전트(agent)** 가 **환경(environment)** 과 상호작용하며 **보상(reward)** 을 최대화하는 방법을 스스로 찾아가는 학습 방식입니다.

### 1.2 강화학습의 핵심 요소

| 요소 | 틱택토에서의 대응 | 설명 |
|------|-------------------|------|
| 에이전트 (Agent) | AI 플레이어 (X 또는 O) | 행동을 결정하는 학습 주체 |
| 환경 (Environment) | 틱택토 보드 | 에이전트가 상호작용하는 세계 |
| 상태 (State) | 현재 보드 배치 | 환경의 현재 모습 |
| 행동 (Action) | 빈 칸에 돌 놓기 | 에이전트가 취할 수 있는 선택 |
| 보상 (Reward) | 승리(+1), 패배(-1), 무승부(+0.3) | 행동의 결과에 대한 피드백 |
| 정책 (Policy) | Q-테이블 기반 선택 | 상태에서 어떤 행동을 할지의 전략 |

### 1.3 강화학습의 학습 흐름

```
에이전트가 현재 보드(상태)를 봅니다
         ↓
빈 칸 중 하나를 골라 돌을 놓습니다(행동)
         ↓
게임이 진행되고 결과가 나옵니다(보상)
         ↓
"이 상황에서 이 칸에 두니까 이겼구나/졌구나" — 경험을 기록합니다
         ↓
다음에 비슷한 상황이 오면 더 나은 선택을 합니다
```

이 과정을 수천~수만 번 반복하면, 에이전트는 점점 더 좋은 전략을 학습하게 됩니다.

## 2. Q-Learning 알고리즘

### 2.1 Q-값이란?

Q-값(Q-value)은 **"이 상태에서 이 행동을 하면 앞으로 얼마나 좋을까?"** 를 숫자로 나타낸 것입니다.

> 💡 **쉬운 비유**: 음식점 평점과 비슷해요. "이 가게에서 파스타를 시키면 4.5점, 피자를 시키면 3.8점" — 그러면 파스타를 주문하겠죠? Q-값도 "이 보드에서 중앙에 두면 0.42점, 모서리에 두면 0.18점" 이런 식이에요.

| 기호 | 읽는 법 | 의미 |
|:----:|---------|------|
| Q(s, a) | "큐 에스 에이" | 상태 s에서 행동 a의 가치 |
| s | state | 현재 보드 배치 |
| a | action | 돌을 놓을 위치 (0~8번 칸) |

### 2.2 Q-테이블

Q-테이블은 **모든 (상태, 행동) 쌍에 대한 Q-값을 저장하는 표**입니다.

```
Q-테이블 (예시)
┌─────────────────────┬──────┬──────┬──────┬──────┐
│ 상태 (보드 배치)     │ 0번칸 │ 1번칸 │ ... │ 8번칸 │
├─────────────────────┼──────┼──────┼──────┼──────┤
│ "X--O-X---"          │ 0.42 │ 0.15 │ ... │ -0.08│
│ "--XO-X-O-"          │ 0.31 │  -   │ ... │ 0.22 │
│ ...                  │ ...  │ ...  │ ... │ ...  │
└─────────────────────┴──────┴──────┴──────┴──────┘
     "-" = 이미 돌이 놓여 있어 선택 불가
```

시뮬레이터에서 **📚 Q-상태** 지표는 이 테이블에 몇 개의 상태가 기록되었는지를 보여줍니다.

### 2.3 Q-값 갱신 규칙

게임이 끝나면 에이전트는 자신이 둔 수들을 **마지막 수부터 거꾸로** 돌아보며 Q-값을 업데이트합니다.

**갱신 공식:**

```
Q(s, a) ← Q(s, a) + α × (보상 - Q(s, a))
```

| 기호 | 이름 | 의미 | 시뮬레이터 기본값 |
|:----:|------|------|:----------------:|
| α | 학습률 (alpha) | 새 경험을 얼마나 반영할지 | 0.30 |
| 보상 | reward | 해당 수의 결과에 대한 평가 | 아래 참조 |

> 💡 **쉬운 비유**: "기존 점수에 새 경험을 30%만 반영한다"는 뜻이에요. α=0.3이면 기존 생각 70% + 새 경험 30%로 평가를 업데이트해요.

### 2.4 역방향 갱신과 할인

시뮬레이터는 에피소드가 끝나면 **마지막 수 → 첫 수** 순서로 거슬러 올라가며 갱신합니다. 이때 **할인율(γ)** 이 적용됩니다.

```
5수째 (마지막): 보상 = 최종 보상 (예: 승리 +1.0)
4수째:          보상 = 단계패널티 + γ × (5수째 보상)
3수째:          보상 = 단계패널티 + γ × (4수째 보상)
...
```

| 기호 | 이름 | 의미 | 시뮬레이터 기본값 |
|:----:|------|------|:----------------:|
| γ | 할인율 (gamma) | 미래 보상을 현재에 얼마나 반영할지 | 0.90 |

γ=0.9이면 한 수 뒤의 보상은 90%로 줄어들고, 두 수 뒤는 81%로 줄어듭니다. **가까운 미래의 보상이 더 중요**하다는 뜻입니다.

### 2.5 갱신 예시

X가 승리(+1.0)한 에피소드에서 X가 3수를 뒀다고 가정합니다.

```
3수째 (마지막): reward = +1.0
  Q(s₃, a₃) ← Q(s₃, a₃) + 0.3 × (1.0 - Q(s₃, a₃))

2수째:          reward = -0.01 + 0.9 × 1.0 = +0.89
  Q(s₂, a₂) ← Q(s₂, a₂) + 0.3 × (0.89 - Q(s₂, a₂))

1수째 (첫 수):  reward = -0.01 + 0.9 × 0.89 = +0.80
  Q(s₁, a₁) ← Q(s₁, a₁) + 0.3 × (0.80 - Q(s₁, a₁))
```

승리한 경우 Q-값이 양의 방향으로 올라가고, 패배하면 음의 방향으로 내려갑니다. 수천 판을 거치면서 Q-테이블은 점점 정확한 평가를 담게 됩니다.

## 3. 탐험과 활용의 균형

### 3.1 ε-greedy 전략

에이전트는 매 수마다 두 가지 중 하나를 선택합니다.

| 확률 | 선택 방식 | 이름 |
|:----:|----------|------|
| ε (입실론) | 빈 칸 중 **무작위** 선택 | 🎲 탐험 (exploration) |
| 1 - ε | Q-값이 **가장 높은** 칸 선택 | 🎯 활용 (exploitation) |

> 💡 **쉬운 비유**: 매일 점심을 먹을 때, 가끔은 새로운 가게를 시도해보고(탐험), 대부분은 가장 맛있다고 알려진 가게에 가는(활용) 것과 같아요.

### 3.2 왜 탐험이 필요한가?

처음부터 활용만 하면 **"처음 우연히 이긴 수만 계속 두는"** 문제가 생깁니다. 아직 시도하지 않은 더 좋은 수가 있을 수 있기 때문이죠. 탐험은 다양한 상태를 경험하게 해서 Q-테이블을 풍부하게 채우는 역할을 합니다.

### 3.3 ε의 감소

학습 초반에는 탐험을 많이 하고, 점차 줄여나갑니다.

```
초기: ε = 100% — 거의 모든 수를 랜덤으로 (탐험 위주)
  ↓ 매 배치마다 ε ← ε × 감소율(0.9995)
  ↓
중기: ε ≈ 30% — 3번 중 1번은 랜덤
  ↓
후기: ε ≈ 1% — 거의 항상 최적수 선택 (활용 위주)
```

시뮬레이터의 **🎲 탐험률 ε** 그래프에서 이 감소 과정을 관찰할 수 있습니다.

### 3.4 감소율의 의미

| ε 감소율 | 1%까지 도달 시점 | 특징 |
|:--------:|:----------------:|------|
| 0.990 | ~460판 | 매우 빠른 전환 — 적은 탐험 |
| 0.995 | ~920판 | 빠른 전환 |
| 0.9995 (기본) | ~9,200판 | 적당한 탐험 기간 |
| 0.9999 | ~46,000판 | 긴 탐험 — 더 많은 상태 발견 |

## 4. 보상 설계

### 4.1 보상 체계

| 결과 | 보상값 | 설계 이유 |
|------|:------:|----------|
| 승리 | **+1.0** | 가장 바람직한 결과 |
| 패배 | **-1.0** | 가장 피해야 할 결과 |
| 무승부 | **+0.3** | 패배보다 낫지만 승리만큼은 아님 |
| 매 수 | **-0.01** | 빠른 승리를 유도하는 단계 패널티 |

### 4.2 무승부 보상이 양수인 이유

무승부 보상을 0으로 하면, 에이전트는 "이길 수 없는 상황에서 어떻게 해야 할지" 학습하기 어렵습니다. +0.3을 부여하면 패배(-1.0)를 피하고 무승부(+0.3)를 선택하는 **방어적 플레이**를 학습합니다.

### 4.3 단계 패널티의 역할

매 수마다 -0.01의 패널티가 있으면, AI는 **같은 결과라도 더 빨리** 달성하려 합니다. 예를 들어 3수 만에 이기면 패널티가 -0.03이지만, 5수 만에 이기면 -0.05입니다. 이는 더 효율적인 전략을 유도합니다.

### 4.4 보상이 학습에 미치는 영향

```
                 패배          무승부          승리
보상:  ←— -1.0 ———————— +0.3 ———————— +1.0 —→

에이전트의 학습:
  · 패배한 수의 Q-값 ↓ (다음에 피함)
  · 승리한 수의 Q-값 ↑ (다음에 선호)
  · 무승부한 수의 Q-값 ↗ (패배보다 나음)
```

## 5. 대칭성 정규화

### 5.1 대칭 변환이란?

틱택토 보드는 **8가지 대칭 변환**으로 같은 모양을 만들 수 있습니다.

```
원본      90° 회전   180° 회전  270° 회전
X · ·     · · X      · · ·      O · ·
· O ·     · O ·      · O ·      · O ·
· · O     O · ·      · · X      · · X

좌우 반사  + 90°      + 180°     + 270°
· · X     O · ·      · · ·      · · O
· O ·     · O ·      · O ·      · O ·
· · O     · · X      X · ·      X · ·
```

위 8가지 보드는 **전략적으로 완전히 동일**합니다.

### 5.2 정규화의 효과

대칭 변환이 없으면 AI는 8가지 상태를 각각 따로 학습해야 합니다. 정규화를 적용하면 하나의 상태로 통합됩니다.

| 항목 | 정규화 없음 | 정규화 적용 | 효율 |
|------|:----------:|:----------:|:----:|
| 빈 보드의 첫 수 선택지 | 9개 | 3개 (중앙, 모서리, 변) | **3배** |
| 전체 Q-상태 수 (추정) | ~5,000개 | ~765개 | **~7배** |
| 같은 성능 도달 시간 | 기준 | 약 1/7~1/8 | **7~8배** |

### 5.3 정규화 작동 방식

```
[1] 현재 보드를 8가지로 변환합니다
[2] 사전식 순서가 가장 작은 것을 "정규 상태"로 선택합니다
[3] 정규 좌표계에서 Q-값을 조회하고 행동을 선택합니다
[4] 선택한 행동을 원본 좌표로 역변환하여 실행합니다
```

시뮬레이터의 **📚 Q-상태 수** 그래프에서 대칭 압축된 상태 수의 증가를 관찰할 수 있습니다. 충분히 학습하면 약 765개 부근에서 포화됩니다.

## 6. 학습 과정의 이해

### 6.1 학습 5단계

시뮬레이터는 에피소드 수에 따라 학습 단계를 표시합니다.

| 단계 | 에피소드 | 무엇이 일어나는가 | 관찰 포인트 |
|------|:--------:|-------------------|-------------|
| 🌱 초기 | 0~499 | Q-테이블이 비어 있어 거의 랜덤 행동 | X/O 승률이 비슷, ε≈100% |
| 🔍 탐색 | 500~4,999 | 기본 패턴 발견 시작, 탐험 감소 중 | X 승률 서서히 상승 |
| 📚 학습 | 5,000~19,999 | 전략이 구체화, 무승부율 상승 | ε<20%, 강도 50+ |
| 🎯 응용 | 20,000~49,999 | 안정적 전략 형성, 미세 조정 | 무승부율 80%+, 강도 80+ |
| 🏆 숙련 | 50,000+ | 이론적 최적에 근접 | 무승부율 ~100%, 등급 S+ |

### 6.2 승률 추이의 해석

학습이 진행되면 그래프에서 특징적인 패턴이 나타납니다.

```
학습 초기           학습 중기            학습 완료
X 승률: ~40%        X 승률: ~25%         X 승률: ~0%
O 승률: ~30%        O 승률: ~10%         O 승률: ~0%
무승부:  ~30%        무승부:  ~65%         무승부:  ~100%
```

**양쪽 AI 모두 최적 학습을 마치면 무승부율이 100%에 수렴합니다.** 이는 틱택토의 이론적 결과(아래 §7 참조)와 일치합니다.

### 6.3 첫수 선호도의 변화

| 학습 단계 | 선호 순서 | 이유 |
|----------|----------|------|
| 초기 | 랜덤 (모두 Q≈0) | 아직 경험 부족 |
| 중기 | 중앙 > 모서리 ≈ 변 | 중앙의 이점 발견 |
| 완료 | 🎯 중앙 >> 📐 모서리 >> ➖ 변 | 최적 전략 확립 |

이 변화를 시뮬레이터의 **🧠 X의 첫수 선호도** 패널에서 관찰할 수 있습니다.

## 7. 틱택토의 게임 이론

### 7.1 완전 정보 게임

틱택토는 **완전 정보(perfect information) 제로섬(zero-sum) 유한(finite) 게임**입니다.

| 성질 | 의미 |
|------|------|
| 완전 정보 | 모든 플레이어가 보드 전체를 볼 수 있음 |
| 제로섬 | 한쪽이 이기면 다른 쪽은 반드시 짐 |
| 유한 | 게임이 반드시 끝남 (최대 9수) |

### 7.2 게임 트리

| 항목 | 값 |
|------|:--:|
| 가능한 게임 진행 수 (대칭 무시) | 255,168가지 |
| 서로 다른 보드 상태 | 5,478개 |
| 대칭 정규화 후 상태 | ~765개 |
| 이론적 결론 | **양쪽 최적 시 무승부** |

### 7.3 최적 전략

게임 이론에서 증명된 틱택토의 최적 전략은 다음과 같습니다.

**선공(X)의 최적 첫수**: 🎯 **중앙** (4번 칸)이 유일한 최적 선택입니다. 중앙을 차지하면 대각선 2개, 가로 1개, 세로 1개 — 총 4개의 승리선에 관여할 수 있습니다.

| 위치 유형 | 관여하는 승리선 수 | 전략적 가치 |
|----------|:-----------------:|:----------:|
| 🎯 중앙 (4번) | 4개 | 최고 |
| 📐 모서리 (0, 2, 6, 8번) | 3개 | 높음 |
| ➖ 변 (1, 3, 5, 7번) | 2개 | 낮음 |

AI가 충분히 학습하면 이 순서를 스스로 발견합니다.

### 7.4 Q-Learning과 미니맥스의 관계

| 비교 | Q-Learning | 미니맥스 |
|------|:----------:|:--------:|
| 접근 | 경험으로 학습 | 전체 탐색 |
| 사전 지식 | 불필요 | 게임 규칙 필요 |
| 최적성 | 충분한 학습 시 근사 | 항상 최적 |
| 확장성 | 큰 게임에도 적용 가능 | 큰 게임은 계산 불가 |
| 교육적 가치 | 학습 과정 관찰 가능 | 결과만 확인 가능 |

Q-Learning의 장점은 **AI가 규칙만 알고 스스로 전략을 발견하는 과정**을 관찰할 수 있다는 점입니다.

## 8. 파라미터가 학습에 미치는 영향

### 8.1 학습률 α (Alpha)

α는 **새로운 경험을 얼마나 강하게 반영할지** 결정합니다.

```
α가 작으면 (0.01):  오래된 경험 99% + 새 경험 1%  → 안정적이지만 느린 학습
α가 크면 (0.9):     오래된 경험 10% + 새 경험 90% → 빠르지만 불안정한 학습
```

| α 값 | 그래프 특성 | 학습 속도 | 안정성 |
|:----:|------------|:---------:|:------:|
| 0.01~0.1 | 매우 부드러운 곡선 | 느림 | 높음 |
| 0.1~0.5 | 적당한 변동 | 보통 | 보통 |
| 0.5~1.0 | 심한 요동 | 빠름 | 낮음 |

> **실험 아이디어**: 시뮬레이터에서 α를 0.01, 0.3, 0.9로 바꿔보고 X 승률 그래프가 어떻게 달라지는지 비교해 보세요.

### 8.2 할인율 γ (Gamma)

γ는 **미래의 보상을 현재에 얼마나 반영할지** 결정합니다.

```
γ=0.1: 한 수 뒤 보상의 10%만 반영 → 근시안적 플레이
γ=0.9: 한 수 뒤 보상의 90%를 반영 → 장기적 전략
```

| γ 값 | n수 뒤 반영률 | 전략 성향 |
|:----:|:------------:|----------|
| 0.1 | 1수: 10%, 2수: 1% | 눈앞의 이익만 추구 |
| 0.5 | 1수: 50%, 2수: 25% | 1~2수 앞을 봄 |
| 0.9 | 1수: 90%, 2수: 81% | 여러 수 앞을 봄 |
| 1.0 | 1수: 100%, 2수: 100% | 모든 미래를 동등하게 평가 |

> **실험 아이디어**: γ를 0.1로 낮추면 AI가 "즉시 이길 수 있는 수"만 찾고 장기 전략을 세우지 못하는 것을 관찰해 보세요.

### 8.3 ε 감소율 (Epsilon Decay)

감소율은 **탐험에서 활용으로 전환하는 속도**를 결정합니다.

| 감소율 | 탐험 기간 | 장점 | 단점 |
|:------:|:---------:|------|------|
| 0.990 | 매우 짧음 (~460판) | 빠른 수렴 | 미발견 전략 가능 |
| 0.9995 | 적당함 (~9,200판) | 충분한 탐험 + 적당한 수렴 | — |
| 0.9999 | 길음 (~46,000판) | 가장 철저한 탐험 | 느린 수렴 |

### 8.4 파라미터 상호작용

파라미터들은 서로 영향을 줍니다.

| 조합 | 결과 |
|------|------|
| 높은 α + 낮은 ε 감소 | 빠르지만 불안정 — 그래프가 심하게 요동 |
| 낮은 α + 높은 ε 감소 | 느리지만 안정 — 그래프가 천천히 올라감 |
| 높은 γ + 긴 탐험 | 장기 전략 학습에 충분한 시간 |
| 낮은 γ + 짧은 탐험 | 단기 전략으로 빠르게 수렴 |

## 9. 시뮬레이터와 이론의 연결

이론의 각 개념이 시뮬레이터의 어떤 요소에 대응되는지 정리합니다.

### 9.1 개념-UI 매핑

| 이론 개념 | 시뮬레이터 UI | 관찰 방법 |
|----------|-------------|----------|
| Q-값 | 보드 위 숫자, 히트맵 색상 | 사람 대전에서 빈 칸의 숫자 확인 |
| Q-테이블 크기 | 📚 Q-상태 수 | 대시보드 학습 통계 |
| ε (탐험률) | 🎲 ε 카드, ε 그래프 | 100%에서 1%로 감소 관찰 |
| α (학습률) | ⚙️ 파라미터 슬라이더 | 값을 바꾸고 승률 그래프 변화 관찰 |
| γ (할인율) | ⚙️ 파라미터 슬라이더 | 값을 바꾸고 전략 변화 관찰 |
| 보상 | 학습 통계의 Step 패널티 | -0.01 표시 확인 |
| 정책 | Q-값 히트맵의 ⭐최선 | AI가 선택하는 최적수 |
| 대칭 정규화 | Q-상태 수 ~765 | 비정규화 시 ~5,000과 비교 |
| 학습 수렴 | 무승부율 그래프 | 100%에 수렴하는 과정 |

### 9.2 관찰 시나리오

**시나리오 1: "AI가 중앙의 가치를 발견하는 순간"**
1. 초기화 후 학습 시작
2. 인사이트 패널의 첫수 선호도 관찰
3. 처음에는 중앙/모서리/변이 비슷 → 점차 중앙이 높아짐
4. 이론(§7.3)에서 중앙이 최적인 이유와 연결

**시나리오 2: "탐험이 줄어드는 과정"**
1. 학습 시작 후 🎲 탐험률 ε 그래프 관찰
2. 100%에서 시작 → 서서히 감소
3. ε이 낮아질수록 승률이 안정화되는 것을 확인
4. 이론(§3.3)의 탐험-활용 균형과 연결

**시나리오 3: "최적 플레이에 도달"**
1. 20,000판 이상 학습
2. 사람 대전에서 난이도 100%로 대국
3. 어떤 수를 두어도 무승부 — AI를 이길 수 없음
4. 이론(§7.2)의 "양쪽 최적 시 무승부"와 연결

## 10. 더 알아보기

### 10.1 Q-Learning의 한계

| 한계 | 설명 | 대안 |
|------|------|------|
| 상태 공간 폭발 | 바둑 같은 큰 게임은 Q-테이블 저장 불가 | Deep Q-Network (DQN) |
| 연속 상태 | 로봇 제어 등 연속적 상태에서 테이블 불가 | 함수 근사 방법 |
| 단일 에이전트 | 여러 에이전트 협력/경쟁 상황은 다름 | 멀티 에이전트 RL |

### 10.2 강화학습의 실제 응용

| 분야 | 사례 |
|------|------|
| 게임 | AlphaGo(바둑), OpenAI Five(Dota2), AlphaStar(스타크래프트) |
| 로봇 | 보행 학습, 물건 잡기, 자율 주행 |
| 추천 | 뉴스/영상 추천 알고리즘 |
| 의료 | 치료 순서 최적화 |
| 에너지 | 데이터센터 냉각 최적화 (DeepMind) |

### 10.3 용어 사전

| 한국어 | 영어 | 의미 |
|--------|------|------|
| 에이전트 | Agent | 학습하는 AI 주체 |
| 환경 | Environment | 에이전트가 상호작용하는 세계 |
| 상태 | State | 환경의 현재 모습 |
| 행동 | Action | 에이전트의 선택 |
| 보상 | Reward | 행동 결과에 대한 피드백 |
| 에피소드 | Episode | 한 판의 게임 (시작~끝) |
| Q-값 | Q-value | 상태-행동 쌍의 기대 가치 |
| Q-테이블 | Q-table | 모든 Q-값을 저장한 표 |
| 정책 | Policy | 상태에서 행동을 결정하는 규칙 |
| 탐험 | Exploration | 새로운 행동 시도 |
| 활용 | Exploitation | 최선으로 알려진 행동 선택 |
| 할인율 | Discount factor | 미래 보상의 현재 가치 비율 |
| 수렴 | Convergence | Q-값이 최적에 가까워지는 것 |
