## 1. 도입: AI, ML, DL의 포함 관계

### 1차시 복습

지난 시간에 우리는 AI(인공지능)가 "인간의 지능을 모방하는 컴퓨터 시스템"이라고 배웠어요. 그런데 AI라는 큰 세계 안에도 여러 가지 기술이 있답니다. 오늘은 그 중에서 가장 핵심적인 **기계학습(Machine Learning)**에 대해 알아보겠습니다.

### AI ⊃ ML ⊃ DL

**[그림 01] AI, ML, DL의 포함 관계**

AI, ML, DL은 서로 포함 관계에 있어요. 가장 큰 개념이 AI이고, 그 안에 ML이, ML 안에 DL이 들어 있습니다.

**🤖 AI (Artificial Intelligence, 인공지능)**

가장 큰 범주예요. 인간의 지능을 모방하는 **모든 기술**을 포함합니다. 규칙을 직접 작성하는 전통적인 방법부터, 데이터로 학습하는 최신 방법까지 모두 AI에 해당해요.

**🧠 ML (Machine Learning, 기계학습)**

AI의 하위 분야로, 사람이 규칙을 직접 작성하는 대신 **컴퓨터가 데이터에서 패턴을 스스로 학습**하는 기술이에요. 오늘 수업의 주인공이랍니다!

**🔥 DL (Deep Learning, 딥러닝)**

ML의 한 종류로, 인간의 뇌 신경망을 본뜬 **인공 신경망을 여러 층으로 쌓아** 복잡한 패턴을 학습하는 기술이에요. ChatGPT, AlphaGo가 대표적인 딥러닝 기술입니다.

> 💡 **쉬운 비유**: AI = 모든 탈것(자전거~비행기), ML = 엔진 있는 탈것(자동차~비행기), DL = 첨단 전기차

| 구분 | AI (인공지능) | ML (기계학습) | DL (딥러닝) |
|:---:|:---:|:---:|:---:|
| 범위 | 가장 넓음 | AI의 하위 | ML의 하위 |
| 방법 | 규칙 기반 포함 | 데이터 학습 | 신경망 학습 |
| 예시 | 체스 프로그램 | 스팸 필터 | ChatGPT |

> 📌 **핵심**: "모든 딥러닝은 기계학습이고, 모든 기계학습은 AI이지만, 그 반대는 성립하지 않아요!"

## 2. 전개 1: 기계학습의 정의

### 기계학습이란?

> **기계학습(Machine Learning)**: 컴퓨터가 명시적으로 프로그래밍되지 않고도, 데이터로부터 패턴을 스스로 학습하여 새로운 상황을 예측하는 AI 기술

쉽게 말하면, **"사람이 규칙을 작성하는 대신, 컴퓨터가 데이터에서 규칙을 찾아내는 기술"**이에요.

### 전통적 프로그래밍 vs 인공지능 프로그래밍

**[그림 02] 전통적 프로그래밍 vs 인공지능 프로그래밍**

이 그림이 오늘 수업의 핵심이에요! 왼쪽(전통적 방식)과 오른쪽(AI 방식)을 비교해 봅시다.

**전통적 프로그래밍** — 사람이 규칙을 작성

```
규칙(사람이 직접 작성) + 데이터(입력값) → 컴퓨터(명령 실행) → 결과(Answers)
```

프로그래머가 "이런 경우에는 이렇게 해라"라는 규칙을 하나하나 작성합니다.

예를 들어 스팸 메일 필터를 만든다면:
- IF "무료" 포함 AND "당첨" 포함 → 스팸
- IF "대출" 포함 AND "지금 바로" 포함 → 스팸
- IF 발신자가 주소록에 있음 → 정상
- ...

문제점이 보이시나요? 스패머들이 "무 료", "당/첨"처럼 글자를 바꾸면 규칙을 피할 수 있어요. 새로운 스팸 수법이 등장할 때마다 규칙을 계속 추가해야 합니다.

**인공지능 프로그래밍** — 컴퓨터가 규칙을 발견

AI 프로그래밍은 **두 단계**로 나뉘어요.

**① 학습 단계 (Learning)**

```
데이터(입력값) + 정답(레이블/Labels) → 컴퓨터(학습/Learning) → 모델(학습된 규칙)
```

수천 개의 메일 데이터(스팸/정상)를 제공하면, 컴퓨터가 스스로 "어떤 패턴이 스팸인지" **규칙을 생성**합니다. 사람이 생각하지 못한 미묘한 패턴까지 찾아내요!

**② 추론 단계 (Inference)**

```
새 입력(새로운 데이터) + 규칙(패턴)(= 학습된 모델) → 컴퓨터(추론/Inference) → 추측 결과(Predictions)
```

학습이 완료된 모델에 새 데이터를 넣으면, "이 사진은 아마 고양이입니다 (95%)"처럼 **예측 결과**를 내놓아요!

> 📌 **핵심: 규칙을 사람이 만드느냐 vs 데이터에서 컴퓨터가 찾아내느냐**

### 일상 속 기계학습

우리는 이미 매일 기계학습의 혜택을 누리고 있어요:

| 서비스 | 기계학습 활용 |
|:---|:---|
| 📧 Gmail | 스팸 메일 자동 분류 |
| 📺 유튜브 / 넷플릭스 | 취향 분석 → 콘텐츠 추천 |
| 🗣️ Siri / 빅스비 | 음성 인식 → 명령 수행 |
| 📸 스마트폰 카메라 | 얼굴 인식, 사진 분류 |
| 🛒 쿠팡 / 네이버 쇼핑 | 구매 기록 → 상품 추천 |

### 왜 기계학습이 필요할까?

> 🖼️ **[그림 18] 왜 기계학습이 필요할까요? — 전통적 프로그래밍으로 해결하기 어려운 문제**

기계학습이 필요한 이유는 크게 세 가지입니다:

| 이유 | 설명 | 예시 |
|:---:|:---|:---|
| ① **규칙 작성이 어려운 문제** | 특징이 수천 가지여서 규칙을 사람이 다 작성할 수 없음 | "이 사진이 고양이인지 어떤 규칙으로?" |
| ② **복잡한 패턴이 있는 문제** | 수백 개 변수가 복잡하게 얽혀 있음 | "이 환자의 질병 위험은 얼마나 높을까?" |
| ③ **변하는 환경을 따라가야 할 때** | 환경이 변하면 규칙도 바꿔야 하지만, 기계학습은 새 데이터로 자동 업데이트 | "내일의 스팸 메일은 어떤 패턴?" |

> 💡 **핵심 메시지**: "사람이 규칙을 만드는 대신, 데이터에서 규칙을 찾게 하자!" — 이것이 기계학습의 핵심 아이디어예요.

## 3. 전개 2: 머신러닝의 3가지 방법 + AI 사과농장 이야기

### 🅰️ 머신러닝의 3가지 방법 — 이론

기계학습은 **데이터로부터 학습하는 방법**에 따라 크게 3가지로 나뉩니다. 먼저 전체 지도를 본 뒤, 각각을 하나씩 살펴보겠습니다.

### 3.A-0 세 가지 학습 유형 개관

지금까지 배운 세 가지 학습 유형을 종합적으로 비교해 봅시다.

| 구분 | 지도학습 | 비지도학습 | 강화학습 |
|:---:|:---:|:---:|:---:|
| **라벨(정답)** | ⭕ 있음 | ❌ 없음 | 🎯 보상 |
| **데이터 구조** | 입력 + 라벨 | 입력만 | 입력 + 보상 |
| **학습 목표** | 정답 맞추기 | 패턴 찾기 | 보상 최대화 |
| **피드백** | 정답(라벨) | 없음 | 보상(점수) |
| **비유** | 시험 공부 | 그룹 나누기 | 게임 연습 |
| **사과농장** | 1·2·4·5장 | 3장 | — |
| **시뮬레이터** | KNN, 선형 회귀, 의사결정 트리 | K-Means | 스네이크 RL |
| **대표 예시** | 스팸 필터, 질병 진단 | 고객 세분화 | AlphaGo, 자율주행 |

> 📌 **핵심 메시지**: 문제 유형에 따라 적절한 학습 방식을 선택해요. 초등학교 AI 교육에서는 주로 **지도학습**(엔트리 이미지 분류)을 다룬답니다!

### 3.A-1 지도학습 (Supervised Learning)

> **지도학습**: 정답(라벨)이 있는 데이터로 학습하는 방식

"선생님이 문제와 정답을 함께 주고, 학생이 문제 유형을 파악하며 공부하는 것"과 같아요.

**학습 과정**:
1. 📊 데이터 + 정답(Label) 제공 → "이 사진은 고양이야", "이건 개야"
2. 🧠 컴퓨터가 데이터와 정답의 관계를 학습 → 패턴 발견
3. 🔮 새로운 데이터 → 정답 예측 → "이건 고양이일 확률 95%!"

**[그림 03] 지도학습 vs 비지도학습 데이터 구조 비교**

지도학습의 핵심은 **라벨(Label)**이에요. 라벨이란 데이터에 붙인 "정답 스티커"입니다.

```
지도학습 데이터:
  [고양이 사진] + "고양이" ✅   ← 라벨이 있음
  [개 사진]     + "개"     ✅   ← 라벨이 있음
```

지도학습에는 두 가지 주요 과제가 있어요:

| 과제 | 설명 | 사과농장 연결 |
|:---:|:---|:---|
| **분류 (Classification)** | 데이터를 범주로 나누기 | 1장 수확(익음/덜 익음), 2장 품종(홍로/부사) |
| **회귀 (Regression)** | 연속적인 값 예측하기 | 4장 가격(품질 → 가격 얼마?) |

#### 지도학습 vs 비지도학습 종합 비교

| 구분 | 지도학습 | 비지도학습 |
|:---:|:---:|:---:|
| **라벨(정답)** | ⭕ 있음 | ❌ 없음 |
| **데이터 구조** | 입력 + 라벨 | 입력만 |
| **학습 목표** | 정답 맞추기 | 패턴 찾기 |
| **비유** | 시험 공부 (문제+정답) | 그룹 나누기 (정답 없이) |
| **사과농장** | 1·2·4·5장 | 3장 |
| **시뮬레이터** | KNN, 선형 회귀, 의사결정 트리 | K-Means |

> 📌 **강조 포인트**: 지도학습 = 라벨(정답)이 있어야 학습 가능, 비지도학습 = 라벨 없이도 패턴 발견 가능. 둘 다 중요하고, 상황에 따라 선택해요!

### 3.A-2 비지도학습 (Unsupervised Learning)

> **비지도학습**: 라벨(정답 표시) 없이 데이터의 숨겨진 패턴을 스스로 찾는 방식

### "정답이 없다"의 정확한 의미

많은 사람들이 비지도학습을 "정답이 없는 학습"이라고 표현하는데, 이게 정확히 무슨 뜻일까요?

> 📌 **정확한 의미**: "정답이 없다" = **"라벨(정답 스티커)이 없다"**

지도학습에서는 모든 데이터에 라벨이 붙어 있었어요.

```
지도학습:    [고양이 사진] + "고양이" ✅  ← 라벨 있음
비지도학습:  [고객 데이터]  + ???      ❌  ← 라벨 없음
```

**[그림 04] 라벨이 없는 3가지 이유**

**왜 라벨이 없을까요?** 세 가지 경우가 있어요:

| # | 이유 | 예시 |
|:---:|:---|:---|
| 1 | **라벨 자체가 없을 때** | 신규 고객 — 어떤 유형인지 아직 몰라요 |
| 2 | **라벨 만들기 어려울 때** | 수백만 뉴스 기사에 일일이 분류 라벨을 붙일 수 없어요 |
| 3 | **숨겨진 패턴을 찾고 싶을 때** | 사람도 몰랐던 새로운 고객 유형을 발견하고 싶어요 |

### 군집화 (Clustering)

비지도학습의 대표적인 과제가 **군집화**예요. 비슷한 데이터끼리 자동으로 그룹을 만드는 것이죠.

**[그림 05] 군집화 3단계 과정**

```
1단계 입력 → 데이터만 제공 (라벨 없이)
2단계 과정 → AI가 유사성을 분석하여 자동 그룹화
3단계 출력 → 의미 있는 그룹 발견 (사람이 나중에 이름 붙임)
```

**[그림 06] 고객 세분화 결과 4개 그룹**

쇼핑몰 고객 데이터를 군집화하면:
- **그룹 A** (30%): 알뜰형 — 저가·할인 선호, 20~30대
- **그룹 B** (20%): 명품 선호형 — 고가·브랜드, 30~50대
- **그룹 C** (25%): 트렌드 민감형 — 신상품·유행, 10~20대
- **그룹 D** (25%): 실속형 — 가성비·리뷰 중심, 전연령

사람이 미리 "알뜰형", "명품형" 같은 라벨을 정하지 않았는데, AI가 스스로 의미 있는 그룹을 발견한 거예요! 이후에 사람이 각 그룹에 이름을 붙여주죠.

### 3.A-3 강화학습 (Reinforcement Learning)

> **강화학습**: 시행착오를 통해 **보상을 최대화**하는 방향으로 학습하는 방식

### 핵심 개념

"게임을 하면서 점수로 배우는 것"과 같아요. 아무도 "이렇게 해라"라고 알려주지 않지만, 점수(보상)를 보고 스스로 좋은 전략을 터득합니다.

**[그림 07] 강화학습 루프**

강화학습의 작동 원리는 반복되는 루프로 이루어져 있어요:

```
AI(Agent) → 행동(Action) 선택 → 환경(Environment)에서 실행
    ↑                                          ↓
    └─── 보상(Reward) 받고 다시 판단 ←──────────┘
```

### 강화학습의 구성 요소

| 요소 | 설명 | 게임 예시 | 강아지 훈련 |
|:---|:---|:---|:---|
| **Agent** | 학습하는 주체 | 게임 플레이어 | 강아지 |
| **Environment** | 활동하는 세계 | 게임 화면 | 집·공원 |
| **Action** | 선택하는 행동 | 상하좌우 이동 | 앉기, 짖기 |
| **Reward** | 행동에 대한 피드백 | 점수 +10 / -5 | 간식 / 무시 |

### 비유: 강아지 훈련

**[그림 08] 강아지 훈련 — 강화학습 비유**

#### 🔍 사과농장 2장: 품종이 섞였다! — KNN (K-최근접 이웃)

수확한 사과에 여러 품종이 섞여 있어요. 홍로, 부사, 아오리... 어떻게 구분할까요?

**KNN (K-Nearest Neighbors, K-최근접 이웃)**: 새 데이터가 들어오면, 기존 데이터 중에서 **가장 가까운 K개의 이웃**을 찾아 **다수결**로 분류하는 방법

**어떻게 작동하나요?**

새 사과가 들어오면:
1. 기존에 라벨이 붙어 있는 사과 데이터들 중에서
2. 새 사과와 특징(크기, 색상, 모양)이 **가장 비슷한 K개** 찾기
3. K개 이웃 중 **가장 많은 품종**으로 분류

| K값 | 가장 가까운 이웃들 | 결과 |
|:---:|:---|:---:|
| K=3 | 홍로, 홍로, 부사 | **홍로** (2:1) |
| K=5 | 홍로, 홍로, 부사, 부사, 홍로 | **홍로** (3:2) |

> 💡 **핵심**: KNN은 "유유상종" — 비슷한 것끼리 모이는 원리! 하지만 기존 데이터에 **라벨(품종명)**이 반드시 있어야 해요. 그래서 지도학습이랍니다.

#### 💰 사과농장 4장: 가격을 매기자 — 선형 회귀

사과의 품질(크기, 당도, 색상)과 가격 사이에는 어떤 관계가 있을까요?

**선형 회귀 (Linear Regression)**: 데이터의 추세를 나타내는 **직선(추세선)**을 찾아, 새로운 입력에 대한 **연속적인 값을 예측**하는 방법

| 품질 점수 | 실제 판매가 |
|:---:|:---:|
| 60점 | 800원 |
| 75점 | 1,200원 |
| 90점 | 1,500원 |
| 85점 | ??? |

이 데이터를 그래프에 점으로 찍으면, 점들이 대략 직선 위에 놓여요. 선형 회귀는 이 점들 사이에 **"가장 잘 맞는 직선"**을 찾아서, 새로운 품질 점수(85점)의 가격을 예측합니다.

> 💡 **핵심**: 분류(KNN)는 "어떤 종류?"를 맞추고, 회귀(선형 회귀)는 "얼마나?"를 예측해요.

| 구분 | 분류 (Classification) | 회귀 (Regression) |
|:---:|:---:|:---:|
| 출력 | 범주 (홍로/부사) | 숫자 (1,350원) |
| 예시 | 품종 구분 | 가격 예측 |
| 사과농장 | 2장 | 4장 |

#### 🚚 사과농장 5장: 어디로 보낼까? — 의사결정 트리

분류된 사과를 백화점, 대형마트, 온라인몰, 주스공장 중 어디로 보낼지 결정해야 해요.

**의사결정 트리 (Decision Tree)**: **예/아니오 질문을 반복**하여 최종 결정에 도달하는 방법

```
상품인가?
├── 예 → 대형인가?
│   ├── 예 → 🏬 백화점
│   └── 아니오 → 🛒 대형마트
└── 아니오 → 양호인가?
    ├── 예 → 📦 온라인몰
    └── 아니오 → 🧃 주스공장
```

마치 스무고개 게임처럼, 예/아니오 질문을 통해 점점 좁혀가며 답을 찾아요!

**의사결정 트리의 장점**:
- 사람이 이해하기 쉬움 (결정 과정이 투명)
- "왜 이 결정을 했는가?" 설명 가능
- 데이터에서 가장 효과적인 질문 순서를 **자동으로** 학습

> 💡 **핵심**: 의사결정 트리도 정답 라벨("백화점", "마트" 등)이 있어야 학습할 수 있어요. 지도학습이죠!

#### 📦 사과농장 3장: 크기별 등급 분류 — K-Means

수확한 사과를 크기별로 등급을 나눠야 하는데, 미리 정해진 기준이 없어요!

**K-Means 클러스터링**: 데이터를 **K개의 그룹**으로 나누되, 각 그룹의 **중심점(평균)**과 가장 가까운 데이터끼리 묶는 방법

사과농장에서의 활용:
- 사과들의 크기(무게, 지름) 데이터만 있고, "대/중/소" 라벨은 없음
- K=3으로 설정하면, AI가 자동으로 3개 그룹으로 나눔
- 결과: 큰 사과(상품), 중간 사과(중품), 작은 사과(하품)

**K-Means의 작동 원리**:
1. K개의 중심점을 임의로 배치
2. 각 데이터를 가장 가까운 중심점의 그룹에 배정
3. 각 그룹의 평균 위치로 중심점 이동
4. 2~3번 반복 → 중심점이 더 이상 움직이지 않으면 완료!

> 💡 **KNN vs K-Means — 이름이 비슷하지만 전혀 달라요!**

| 구분 | KNN (K-최근접 이웃) | K-Means (K-평균) |
|:---:|:---:|:---:|
| 학습 유형 | **지도학습** | **비지도학습** |
| 라벨 필요? | ⭕ 필요 | ❌ 불필요 |
| K의 의미 | 이웃 수 | 그룹 수 |
| 목적 | 새 데이터 분류 | 전체 데이터 그룹화 |
| 사과농장 | 2장 (품종 분류) | 3장 (등급 분류) |

##### 🖥️ 시뮬레이터 시연: K-Means 클러스터링

> "사과농장 3장에서 크기별로 등급을 나눈 원리를 직접 확인해 봅시다!"

> 🖼️ **[그림 16] K-Means 클러스터링 시뮬레이터 — 학습 모드 화면**
> 왼쪽: 데이터 설정 (데이터 개수·K값·분산도), 가운데: 클러스터 시각화 (색상별 군집 + 중심점), 오른쪽: 학습 설정 (최대 반복·초기화 방법)

| 단계 | 시연 내용 | 관찰 포인트 |
|:---:|:---|:---|
| 설정 | 라벨 없는 데이터 점들 배치 | 라벨 없이 점들의 분포만 보임 |
| 실행 | K=3 설정, 클러스터링 실행 | 중심점이 반복 이동하며 그룹 형성 |
| 해설 | K값 변경 (K=2, K=4) 비교 | 같은 데이터도 K에 따라 그룹 달라짐 |

> 📌 **시뮬레이터에서 확인하세요**: 중심점(십자 표시)이 반복적으로 이동하다가 안정되는 과정을 관찰! 이것이 K-Means의 핵심이에요.

> 🖼️ **[그림 21] Inertia(관성) 개념도 — 클러스터 품질을 어떻게 측정할까?**
> 왼쪽: 낮은 Inertia(좋음) — 데이터가 중심점에 모임, 가운데: 높은 Inertia(나쁨) — 데이터가 흩어짐, 오른쪽: 수식과 해석

📌 **Inertia가 낮을수록 좋은 클러스터!** 시뮬레이터에서 K값을 바꿔보면 Inertia 값이 변하는 것을 확인할 수 있어요. 적절한 K값을 찾는 것이 중요합니다.

## 4. 전개 3: 머신러닝 학습 과정 4단계

사과농장에서 본 모든 AI 기술은 공통된 **4단계 학습 과정**을 따라요.

> 💡 [그림 02]의 **① 학습 단계**와 **② 추론 단계**를 더 자세히 풀어보면 바로 이 4단계예요!

**[그림 10] 머신러닝 4단계 프로세스**

```
1단계 데이터 수집 → 2단계 학습 → 3단계 평가 → 4단계 예측
```

### 1단계: 데이터 수집 (Data Collection)

> 💡 **"좋은 데이터 = 좋은 AI"**

학습의 재료인 데이터를 모으는 단계예요. 데이터의 양과 질이 AI의 성능을 결정합니다.

**좋은 데이터의 조건**:

| 조건 | 설명 | 예시 |
|:---|:---|:---|
| ✅ 충분한 양 | 수백~수천 개 이상 | 사과 사진 1,000장 |
| ✅ 높은 품질 | 정확한 라벨, 선명한 이미지 | 흐릿한 사진 제외 |
| ✅ 다양성 | 다양한 조건 포함 | 맑은 날·흐린 날·다양한 품종 |

**[그림 11] 편향된 데이터 vs 좋은 데이터**

> ⚠️ **데이터 편향 주의!** 빨간 사과 사진만으로 학습하면, 초록 사과를 "사과가 아니다"라고 판단할 수 있어요.

**엔트리 연결**: 3~4주차 실습에서 웹캠으로 사진을 직접 촬영하는 것이 바로 1단계!

### 2단계: 학습 (Training)

수집한 데이터에서 **패턴을 찾는 과정**이에요. [그림 02]의 **① 학습 단계** — 데이터 + 정답 → 컴퓨터(학습) → 모델(학습된 규칙) — 가 바로 이 단계입니다!

- **에포크(Epoch)**: 전체 데이터를 1회 학습하는 단위 (교과서 한 번 읽기 = 1에포크)
- **가중치(Weight)**: 각 특징의 중요도를 나타내는 숫자
- 에포크를 반복할수록 가중치가 조정되며 성능이 향상

**엔트리 연결**: "모델 학습하기" 버튼 = 2단계 실행!

### 3단계: 평가 (Testing)

학습이 끝난 모델을 **새로운 데이터로 테스트**하는 단계예요.

- 학습에 사용하지 않은 새 데이터로 정확도 확인
- 시험 공부 후 모의고사를 보는 것과 같아요!

**엔트리 연결**: 학습 후 표시되는 정확도(%) = 3단계 결과!

### 4단계: 예측 (Prediction)

학습이 완료된 모델을 **실전에 적용**하는 단계예요. [그림 02]의 **② 추론 단계** — 새 입력 + 학습된 모델 → 추측 결과(Predictions) — 가 바로 이 단계입니다!

- 완전히 새로운 입력 데이터에 대해 결과를 예측
- 사과농장: 새 사과 사진 → "익었다!" → 수확

**엔트리 연결**: 새 이미지를 분류하는 것 = 4단계!

> 📌 **사과농장도 이 4단계를 따랐어요**: 사과 사진 수집(1단계) → CNN 학습(2단계) → 정확도 확인(3단계) → 새 사과 판별(4단계)

## 5. 정리

### 오늘 배운 핵심 개념 4가지

**① AI ⊃ ML ⊃ DL — 포함 관계**

AI가 가장 큰 개념이고, 그 안에 기계학습이, 기계학습 안에 딥러닝이 포함되어 있어요.

**② 전통적 프로그래밍 vs 인공지능 프로그래밍**

전통적(규칙 → 결과) vs AI(① 학습: 데이터 → 모델, ② 추론: 새 입력 → 예측)

**③ 3가지 학습 유형 (사과농장 연결)**

| 유형 | 핵심 | 사과농장 | 시뮬레이터 |
|:---|:---|:---|:---|
| 지도학습 | 라벨 있음 → 정답 맞추기 | 수확·품종·가격·배송 | KNN, 선형 회귀, 의사결정 트리 |
| 비지도학습 | 라벨 없음 → 패턴 발견 | 등급 분류 | K-Means |
| 강화학습 | 보상 → 최적 전략 | — | 스네이크 RL |

**④ 머신러닝 4단계 과정**

데이터 수집 → 학습 → 평가 → 예측

### 엔트리 AI 모델과의 연결

오늘 배운 이론은 3~4주차 엔트리 실습으로 이어집니다!

**엔트리 이미지 분류 모델 = 지도학습**:
1. 웹캠으로 고양이/개 사진 촬영 (데이터 수집)
2. "모델 학습하기" 버튼 클릭 (학습)
3. 정확도 확인 — 예: 92% (평가)
4. 새 이미지 실시간 분류 (예측)

→ **오늘 배운 지도학습의 4단계 과정이 그대로 적용됩니다!**

### 주요 용어 정리

| 용어 | 영문 | 의미 |
|:---|:---|:---|
| 모델 | Model | 데이터에서 학습된 규칙(패턴)의 집합 |
| 추론 | Inference | 학습된 모델로 새 데이터를 예측하는 과정 |
| 에포크 | Epoch | 전체 데이터 1회 학습 단위 |
| 가중치 | Weight | 특징의 중요도 숫자 |
| 정확도 | Accuracy | 맞춘 비율 (%) |
| 라벨 | Label | 데이터의 정답 표시 |
| 특징 | Feature | 데이터의 속성 (크기, 색상 등) |
| 군집화 | Clustering | 유사 데이터 그룹화 |
| 보상 | Reward | 행동에 대한 피드백 |
