# 🎬 도입: AI, ML, DL의 포함 관계 (5분)

## 1차시 복습

> 📌 **핵심 질문**: "1차시에서 배운 AI, 그 안에는 무엇이 있을까요?"

지난 시간에 우리는 AI의 정의와 유형을 배웠어요. AI는 "인간의 지능을 모방하는 컴퓨터 시스템"이라고 했죠. 그런데 AI를 구현하는 방법은 여러 가지가 있어요. 오늘은 그중에서도 가장 중요한 **기계학습(Machine Learning)**에 대해 알아보겠습니다.

## AI, ML, DL의 포함 관계

**[그림 08] AI, ML, DL의 포함 관계**

AI, ML, DL은 자주 혼동되는 용어입니다. 이 세 가지의 관계를 정확히 이해하는 것이 중요해요.

```
AI (인공지능)
 ⊃ ML (기계학습)
    ⊃ DL (딥러닝)
```

### 🤖 AI (Artificial Intelligence, 인공지능)

- **가장 큰 개념**: 인간의 지능을 모방하는 모든 기술
- **범위**: 규칙 기반 시스템부터 최신 딥러닝까지 모두 포함
- **예시**: 
  - 규칙 기반: 체스 프로그램 (if-then 규칙)
  - 기계학습: 스팸 필터, 추천 시스템
  - 전문가 시스템: 1980년대 의료 진단 시스템

### 🧠 ML (Machine Learning, 기계학습)

- **AI의 하위 개념**: AI를 구현하는 대표적 방법
- **핵심 특징**: 데이터로부터 패턴을 **자동으로 학습**
- **차이점**: 사람이 규칙을 일일이 작성하지 않음
- **예시**: 
  - 이메일 스팸 필터
  - 유튜브 추천 알고리즘
  - 날씨 예측 모델

### 🔥 DL (Deep Learning, 딥러닝)

- **ML의 하위 개념**: 기계학습의 한 종류
- **핵심 특징**: 인공신경망을 여러 층으로 쌓아 복잡한 패턴 학습
- **"Deep"의 의미**: 신경망의 층(Layer)이 깊다(많다)
- **예시**:
  - ChatGPT (자연어 처리)
  - AlphaGo (바둑)
  - 자율주행 (이미지 인식)

> 💡 **핵심 통찰**: AI ⊃ ML ⊃ DL 관계를 이해하면, 각 기술의 범위와 특성을 명확히 구분할 수 있습니다.

### 비유로 이해하기

AI를 "차량"에 비유하면:
- **AI** = 모든 차량 (자전거, 오토바이, 자동차, 비행기)
- **ML** = 엔진이 있는 차량 (오토바이, 자동차, 비행기)
- **DL** = 첨단 전기차 (테슬라)

기계학습은 AI의 일부이고, 딥러닝은 기계학습의 일부예요.

# 📘 전개 1: 기계학습의 정의 (15분)

## 기계학습이란?

> **기계학습(Machine Learning)**: 컴퓨터가 데이터로부터 패턴을 스스로 학습하는 AI 기술

기계학습은 1959년 아서 새뮤얼(Arthur Samuel)이 정의한 개념입니다: "명시적으로 프로그래밍하지 않고도 컴퓨터가 학습할 수 있는 능력"

## 전통적 프로그래밍 vs 기계학습

기계학습을 이해하려면, 먼저 전통적 프로그래밍과 어떻게 다른지 알아야 해요.

### 전통적 프로그래밍

```
규칙 + 데이터 → 결과
```

**예시: 스팸 메일 필터 (전통적 방식)**

1. 프로그래머가 규칙 작성:
   - IF "무료" 포함 AND "당첨" 포함 → 스팸
   - IF "대출" 포함 AND "즉시" 포함 → 스팸
   - IF 발신자가 블랙리스트 → 스팸

2. 문제점:
   - 규칙을 모두 나열하기 불가능
   - 새로운 스팸 기법에 대응 어려움
   - 유지보수가 복잡

### 기계학습

```
데이터 + 결과 → 규칙 (패턴)
```

**예시: 스팸 메일 필터 (기계학습 방식)**

1. 컴퓨터에게 수천 개의 메일 제공:
   - 스팸 메일 5,000개 (정답: 스팸)
   - 정상 메일 5,000개 (정답: 정상)

2. 컴퓨터가 스스로 패턴 발견:
   - "무료", "당첨"이 자주 나오면 스팸일 확률 높음
   - 친구 이메일에는 이름이 자주 나옴
   - 스팸은 링크가 많음

3. 장점:
   - 사람이 규칙을 작성하지 않아도 됨
   - 새로운 스팸 기법도 자동으로 학습
   - 데이터가 많을수록 성능 향상

## 왜 기계학습이 필요한가?

기계학습이 필요한 이유는 크게 세 가지예요.

### 1️⃣ 규칙을 작성하기 어려운 문제

**예시: 고양이 vs 개 구분**

전통적 프로그래밍으로 한다면:
- IF 귀가 뾰족하면 → 고양이?
- 하지만 치와와도 귀가 뾰족함
- IF 수염이 있으면 → 고양이?
- 하지만 일부 개도 수염이 있음

→ 규칙을 명확히 작성하기 불가능!

기계학습으로 한다면:
- 고양이 사진 1,000장 + 개 사진 1,000장 제공
- 컴퓨터가 스스로 패턴 발견
- 우리가 모르는 미세한 차이까지 학습

### 2️⃣ 복잡한 패턴이 있는 문제

**예시: 음성 인식**

- 같은 단어도 사람마다 발음이 다름
- 억양, 속도, 음높이가 천차만별
- 규칙으로 모든 경우를 다루기 불가능
- 기계학습으로 수백만 개 음성 데이터 학습

### 3️⃣ 환경이 계속 변하는 문제

**예시: 주식 가격 예측**

- 경제 상황이 계속 변함
- 과거 규칙이 미래에 안 맞을 수 있음
- 기계학습은 새 데이터로 계속 업데이트

## 일상 속 기계학습 예시

우리는 이미 기계학습을 매일 사용하고 있어요.

| 서비스 | 기계학습 활용 |
|:---|:---|
| 📧 **Gmail 스팸 필터** | 수십억 개의 메일 데이터로 스팸 패턴 학습 |
| 📺 **유튜브 추천** | 시청 기록으로 취향 분석, 관련 영상 추천 |
| 🗣️ **Siri, Alexa** | 수백만 명의 음성으로 음성 인식 학습 |
| 📸 **얼굴 인식** | 수천 장의 얼굴 사진으로 특징 학습 |
| 🚗 **자율주행** | 수백만 킬로미터 주행 데이터로 운전 학습 |
| 🛒 **쿠팡 추천** | 구매 기록으로 선호 상품 예측 |

> 💡 **핵심 메시지**: 기계학습은 "사람이 규칙을 작성하는 대신, 컴퓨터가 데이터에서 규칙을 찾는" 기술입니다.

# 📗 전개 2: 기계학습의 3가지 학습 유형 (15분)

기계학습은 크게 세 가지 방식으로 학습합니다. 각각의 특징을 이해해 봅시다.

## 1️⃣ 지도학습 (Supervised Learning)

> **지도학습**: 정답이 있는 데이터로 학습하는 방식

### 핵심 개념

"선생님이 문제와 정답을 주고 공부하는 것"과 같아요.

**학습 과정**:
1. 데이터 + 정답(Label) 제공
2. 컴퓨터가 데이터와 정답의 관계 학습
3. 새로운 데이터가 들어오면 정답 예측

### 지도학습의 두 가지 종류

#### 📊 분류 (Classification)

**입력 → 범주(카테고리)**

| 예시 | 입력 | 출력 (범주) |
|:---|:---|:---|
| 이메일 스팸 판별 | 메일 내용 | 스팸 / 정상 |
| 질병 진단 | 증상, 검사 결과 | 독감 / 감기 / 정상 |
| 꽃 종류 분류 | 꽃잎 길이, 너비 등 | 장미 / 튤립 / 백합 |
| 고양이 vs 개 | 사진 | 고양이 / 개 |

#### 📈 예측 (Regression)

**입력 → 숫자(연속적 값)**

| 예시 | 입력 | 출력 (숫자) |
|:---|:---|:---|
| 집값 예측 | 평수, 위치, 연식 | 5억 2천만 원 |
| 성적 예측 | 공부 시간, 출석률 | 85점 |
| 기온 예측 | 시간, 계절, 습도 | 23.5도 |
| 주식 가격 예측 | 과거 가격, 거래량 | 52,300원 |

### 지도학습 예시: 고양이 vs 개

**데이터 준비**:
- 고양이 사진 1,000장 (정답: 고양이)
- 개 사진 1,000장 (정답: 개)

**학습 과정**:
1. 컴퓨터에게 사진 + 정답 제공
2. 컴퓨터가 고양이와 개의 특징 학습
3. 귀 모양, 눈 모양, 털 패턴 등 스스로 발견

**예측**:
- 새로운 동물 사진 입력
- 컴퓨터가 "고양이" 또는 "개" 판별

### 엔트리 AI 모델 연결

💡 **중요**: 3주차에 배울 **엔트리 AI 이미지 분류 모델**이 바로 지도학습입니다!

엔트리에서 여러분이 직접:
1. 고양이/개 사진 찍기 (데이터 수집)
2. "이건 고양이", "이건 개" 라벨 붙이기 (정답 지정)
3. "모델 학습하기" 버튼 클릭 (학습)
4. 새 사진으로 테스트 (예측)

→ 이 전체 과정이 지도학습이에요!

## 2️⃣ 비지도학습 (Unsupervised Learning)

> **비지도학습**: 라벨(정답 표시) 없이 데이터의 패턴을 스스로 찾는 방식

### "정답이 없다"의 정확한 의미

💡 **매우 중요한 개념!**

**교수 전략**:
```
"비지도학습을 설명할 때 '정답이 없다'고 하는데,
이게 정확히 무슨 뜻일까요?

여러분, 잘 들어보세요.
'정답이 없다' = '라벨(정답 스티커)이 없다'는 뜻이에요!"
```

### 지도학습 vs 비지도학습 데이터 구조 비교

**판서 또는 슬라이드**:

```
지도학습 (Supervised Learning):
  데이터 + 라벨(정답)

  [고양이 사진] + "고양이" ✅
  [개 사진] + "개" ✅
  
  → 모든 데이터에 정답 스티커가 붙어 있음!

비지도학습 (Unsupervised Learning):
  데이터만 (라벨 없음)

  고객A: 25세, 구매액 10만원, 패션 선호
  고객B: 45세, 구매액 200만원, 전자제품 선호
  고객C: 28세, 구매액 15만원, 패션 선호
  
  → 어떤 그룹인지 정답 스티커가 없음! ❌
  → AI가 스스로 비슷한 사람끼리 묶어서 그룹 발견!
```

**강조할 점**:
```
지도학습: "이건 고양이"라는 라벨이 있음
비지도학습: "이 고객은 알뜰형"이라는 라벨이 없음

하지만 라벨이 없어도 AI는 스스로 패턴을 찾아
의미 있는 그룹을 만들어요!
```

### 왜 라벨이 없을까? (3가지 이유)

**교수 전략**: 각 이유를 구체적 예시와 함께 설명

#### 1️⃣ 라벨 자체가 없을 때

**예시: 쇼핑몰 신규 고객**

```
상황:
  쇼핑몰에 신규 고객 10,000명이 가입했어요.

문제:
  이 고객들이 어떤 취향인지 아직 몰라요.
  
  "이 고객은 알뜰형"
  "이 고객은 명품 선호형"
  
  이런 라벨을 붙일 수가 없어요!
  왜? → 아직 정보가 없으니까!

해결:
  ❌ 지도학습 불가능: 라벨이 없으니까
  ✅ 비지도학습: 구매 패턴만 보고 자동 그룹화
```

**교수 멘트**:
```
"여러분, 신규 고객의 취향을 어떻게 알 수 있을까요?
구매 이력이 없는데 '이 사람은 이런 스타일'이라고
라벨을 붙일 수가 없잖아요!

하지만 AI는 나이, 첫 구매 품목, 검색 키워드 등을 보고
비슷한 사람끼리 묶어서 그룹을 만들어요."
```

#### 2️⃣ 라벨 만들기 어려울 때

**예시: 뉴스 기사 분류**

```
상황:
  인터넷에 매일 수백만 개의 뉴스 기사가 올라와요.

문제:
  각 기사에 "정치", "경제", "스포츠" 라벨을 붙이려면?
  
  → 사람이 일일이 읽고 분류해야 함
  → 하루에 100만 개? 불가능!
  → 시간/비용이 너무 많이 듦

해결:
  ❌ 지도학습: 라벨 만들기 불가능 (시간/비용)
  ✅ 비지도학습: 단어 패턴만 보고 자동 분류
```

**교수 멘트**:
```
"여러분이 매일 올라오는 100만 개 뉴스에
일일이 '정치', '경제', '스포츠' 라벨을 붙인다고 상상해보세요.

하루 종일 해도 못 끝나겠죠?

하지만 AI는 기사에서 '국회', '의원' 같은 단어가 많으면
정치 그룹으로, '주식', '환율' 같은 단어가 많으면
경제 그룹으로 자동으로 묶어요!"
```

#### 3️⃣ 숨겨진 패턴을 발견하고 싶을 때

**예시: 새로운 고객 그룹 발견**

```
상황:
  기존에 알려진 고객 유형:
  - 10대 학생
  - 20대 직장인
  - 30대 부모

문제:
  하지만 이것만으로는 부족해요!
  사람도 몰랐던 새로운 패턴이 있을 수 있어요.

AI 발견:
  - "오전 출근 시간에 편의점에서 샐러드 사는 그룹"
    → 건강 관심형 직장인
  
  - "주말 저녁에 온라인 쇼핑하는 그룹"
    → 여가 시간 쇼핑형
  
  - "새벽 3시에 라면 구매하는 그룹"
    → 야간 근무자 or 수험생

해결:
  사람이 미리 정의하지 않은 새로운 패턴을
  AI가 데이터에서 스스로 발견!
```

**교수 멘트**:
```
"이게 비지도학습의 진짜 힘이에요!

사람이 생각하지 못한 패턴을 AI가 발견해요.

예를 들어 넷플릭스는 비지도학습으로
'금요일 밤 10시에 로맨스 영화 보는 사람들'
이런 새로운 시청 패턴을 발견했어요.

이건 사람이 미리 라벨로 만들 수 없는 거예요!"
```

### 군집화 (Clustering): 비지도학습의 대표 방법

**개념 설명**:

```
군집화 = 유사한 데이터끼리 묶어서 그룹(클러스터) 만들기
```

**3단계 과정**:

```
1단계: 입력
  고객 데이터만 제공 (라벨 없이)
  - 고객A: 나이 25, 구매액 10만원, 패션 선호
  - 고객B: 나이 45, 구매액 200만원, 전자제품 선호
  - 고객C: 나이 28, 구매액 15만원, 패션 선호
  ...

2단계: 과정
  AI가 유사한 고객끼리 자동 그룹화
  (라벨 없이 패턴만으로!)
  
  어떻게?
  - 나이가 비슷한 사람
  - 구매 금액이 비슷한 사람
  - 선호 카테고리가 비슷한 사람
  → 이런 기준으로 자동 분류

3단계: 출력
  그룹 A, B, C... (AI가 스스로 발견한 그룹)
  
  사람이 나중에 이름 붙임:
  - 그룹 A → "20대 패션 선호형"
  - 그룹 B → "40대 전자제품 선호형"
  - 그룹 C → "30대 생활용품 선호형"
```

### 구체적 예시 3가지

#### 예시 1: 쇼핑몰 고객 세분화 (상세)

```
┌─────────────────────────────────────┐
│ 예시 1: 쇼핑몰 고객 세분화          │
└─────────────────────────────────────┘

상황:
  신규 고객 10,000명
  취향 라벨 없음 (어떤 고객인지 모름)

❌ 지도학습이 불가능한 이유:
  "이 고객은 알뜰형"이라는 라벨이 없음!
  
✅ 비지도학습 적용:
  1. 데이터 입력: 나이, 구매액, 선호 카테고리
  2. AI 분석: 구매 패턴 자동 분석
  3. 자동 그룹화: 3~5개 그룹 발견

결과:
  그룹 1 (30%): 알뜰형
    - 특징: 저가 제품, 할인 쿠폰 활용
    - 나이: 20~30대
    - 구매액: 평균 5만원
  
  그룹 2 (20%): 명품 선호형
    - 특징: 고가 제품, 브랜드 중시
    - 나이: 30~50대
    - 구매액: 평균 50만원
  
  그룹 3 (25%): 트렌드 민감형
    - 특징: 신상품, 유행 제품
    - 나이: 10~20대
    - 구매액: 평균 15만원
  
  그룹 4 (25%): 실속형
    - 특징: 가성비 중시, 리뷰 확인
    - 나이: 전 연령
    - 구매액: 평균 10만원

활용:
  각 그룹에 맞춤 마케팅!
  - 알뜰형 → 할인 쿠폰 발송
  - 명품 선호형 → 신상 명품 소개
  - 트렌드 민감형 → 최신 트렌드 정보
  - 실속형 → 가성비 제품 추천
```

**교수 팁**:
```
"이 예시를 설명할 때:

1. 먼저 '신규 고객은 라벨이 없다'를 강조
2. 지도학습이 왜 불가능한지 명확히
3. 비지도학습으로 어떻게 해결하는지 설명
4. 결과가 실제 비즈니스에 어떻게 활용되는지 연결

학생들이 '아, 이래서 비지도학습이 필요하구나!'
하고 납득할 수 있도록!"
```

#### 예시 2: 넷플릭스 추천 시스템

```
┌─────────────────────────────────────┐
│ 예시 2: 넷플릭스 추천               │
└─────────────────────────────────────┘

상황:
  "좋아요" 데이터만으로는 취향 파악 어려움
  사용자가 무엇을 좋아할지 정확한 라벨 없음

비지도학습 적용:
  1. 시청 기록 수집
     - 어떤 영화를 봤나?
     - 언제 봤나?
     - 얼마나 봤나? (중간에 껐나?)
  
  2. 비슷한 시청 패턴을 가진 사용자끼리 그룹화
  
  3. 그룹 내 다른 사람들이 본 영화 추천

결과:
  "이 영화를 본 사람은 저 영화도 봤어요"
  
  → 이게 바로 비지도학습 결과!

왜 비지도학습?
  - "이 사람은 액션 선호형"이라는 명확한 라벨 없음
  - 사람의 취향은 복잡함 (액션+코미디+로맨스)
  - AI가 패턴을 발견해서 비슷한 사람 찾음
```

#### 예시 3: 뉴스 자동 분류

```
┌─────────────────────────────────────┐
│ 예시 3: 뉴스 자동 분류              │

상황:
  매일 수백만 개 뉴스 기사 생성

비지도학습 적용:
  1. 뉴스 기사 텍스트 입력
  
  2. 단어 패턴 분석
     - "국회", "의원", "법안" → ?

     - "축구", "야구", "승리" → ?
  
  3. 유사한 단어 패턴끼리 자동 그룹화

결과:
  그룹 A: 정치 기사 (국회, 선거, 정당...)
  그룹 B: 경제 기사 (주식, 환율, 기업...)
  그룹 C: 스포츠 기사 (축구, 야구, 올림픽...)

활용:

  - 유사 기사 묶기
  - 뉴스 추천 시스템
```

### 초등교육 비유

```
  이렇게 정답표를 미리 만들어 줌

비지도학습:
```

### 오개념 대응

| 학생 오개념 | 올바른 이해 | 대응 방법 |
|:---|:---|:---|
| "비지도학습은 아무렇게나 해도 된다" | 의미 있는 패턴을 찾아야 함 | "라벨이 없어도 품질 기준은 있어요" |

| "비지도학습은 쓸모없다" | 많은 실제 활용 사례 존재 | "추천 시스템, 이상 탐지 등" |

```
넷플릭스, 유튜브, 쇼핑몰 추천이 모두 비지도학습을 사용해요.
정답 라벨이 없어도 의미 있는 결과를 만들어내죠!"
```

## 3️⃣ 강화학습 (Reinforcement Learning)

> **강화학습**: 시행착오를 통해 보상을 최대화하는 방향으로 학습

### 핵심 개념

"게임하면서 점수로 배우기"

**학습 과정**:
1. AI가 행동(Action) 선택
2. 환경이 보상(Reward) 제공
   - 좋은 행동 → + 보상
   - 나쁜 행동 → - 보상
3. 보상을 최대화하는 방향으로 학습
4. 반복하며 최적의 전략 발견

### 강화학습의 구성 요소

| 요소 | 설명 | 예시 (게임) |
|:---|:---|:---|
| **Agent (에이전트)** | 학습하는 주체 | 게임 플레이어 |
| **Environment (환경)** | 에이전트가 상호작용하는 세계 | 게임 화면 |
| **State (상태)** | 현재 상황 | 점수, 위치, 체력 |
| **Action (행동)** | 에이전트가 선택할 수 있는 것 | 위/아래/좌/우 이동 |
| **Reward (보상)** | 행동에 대한 피드백 | 점수 +10, -5 |

### 예시 1: AlphaGo (바둑)

**목표**: 바둑에서 이기기

**학습 과정**:
1. AlphaGo가 바둑돌 놓기 (행동)
2. 게임 결과에 따라 보상:
   - 이기면 → +1 (큰 보상)
   - 지면 → -1 (큰 벌점)
   - 좋은 수 → +0.1 (작은 보상)
3. 수백만 번 자기 자신과 대국하며 학습
4. 이길 확률을 높이는 전략 발견

**결과**: 2016년 이세돌 9단을 4:1로 격파

### 예시 2: 자율주행

**목표**: 안전하고 효율적으로 운전

**학습 과정**:
1. AI가 핸들 조작 (행동)
2. 결과에 따라 보상:
   - 차선 유지 → +1
   - 차선 이탈 → -5
   - 사고 → -100
   - 목적지 도착 → +10
3. 시뮬레이션에서 수백만 번 주행하며 학습
4. 보상을 최대화하는 운전 기술 습득

### 예시 3: 게임 AI

**게임**: 슈퍼마리오

**학습 과정**:
1. AI가 버튼 누르기 (점프, 이동)
2. 보상:
   - 코인 획득 → +10
   - 적 제거 → +50
   - 게임 오버 → -100
   - 목표 도달 → +1000
3. 수천 번 게임하며 학습
4. 최고 점수를 얻는 플레이 방법 발견

### 비유로 이해하기

**지도학습**: 선생님이 정답 알려주며 공부
**비지도학습**: 혼자서 패턴 찾기
**강화학습**: 게임하며 점수로 배우기 (시행착오)

## 세 가지 학습 유형 비교

| 구분 | 지도학습 | 비지도학습 | 강화학습 |
|:---:|:---:|:---:|:---:|
| **정답** | ⭕ 있음 | ❌ 없음 | 🎯 보상 |
| **목표** | 정답 맞추기 | 패턴 찾기 | 보상 최대화 |
| **비유** | 시험 공부 | 그룹 나누기 | 게임 연습 |
| **데이터** | 입력 + 정답 | 입력만 | 상태 + 행동 + 보상 |
| **예시** | 스팸 필터<br>질병 진단<br>집값 예측 | 고객 세분화<br>뉴스 분류<br>추천 시스템 | AlphaGo<br>자율주행<br>게임 AI |

> 💡 **핵심 메시지**: 문제 유형에 따라 적절한 학습 방식을 선택하는 것이 중요합니다. 초등학교 AI 교육에서는 주로 **지도학습**(엔트리 이미지 분류)을 다룹니다.

# 📙 전개 3: 머신러닝 학습 과정 (10분)

기계학습이 실제로 어떻게 작동하는지 4단계 과정을 알아봅시다.

**[그림 09] 머신러닝 학습 과정**

```
데이터 수집 → 학습 → 평가 → 예측
```

## 1단계: 데이터 수집 (Data Collection)

> **"좋은 데이터 = 좋은 AI"**

### 데이터란?

기계학습에서 데이터는 "학습 재료"예요. 음식을 만들 때 재료가 좋아야 맛있는 요리가 나오듯, 좋은 데이터가 있어야 좋은 AI가 만들어집니다.

### 데이터 수집 예시

**고양이 vs 개 분류 모델 만들기**

1. 고양이 사진 1,000장 수집
2. 개 사진 1,000장 수집
3. 각 사진에 라벨 붙이기:
   - cat_001.jpg → "고양이"
   - dog_001.jpg → "개"

### 좋은 데이터의 조건

#### ✅ 1. 충분한 양

- 적은 데이터: 100장 → 성능 낮음
- 많은 데이터: 10,000장 → 성능 높음
- 일반적으로 수백~수천 개 필요

#### ✅ 2. 높은 품질

- 정확한 라벨 (고양이를 개라고 잘못 표시 X)
- 선명한 이미지 (흐릿한 사진 X)
- 오류가 없는 데이터

#### ✅ 3. 다양성

- 다양한 품종 포함
- 다양한 각도의 사진
- 다양한 배경, 조명

#### ❌ 편향된 데이터 주의

**나쁜 예시**:
- 고양이는 실내 사진만
- 개는 야외 사진만
- → AI가 "실내 = 고양이"로 잘못 학습

**좋은 예시**:
- 고양이: 실내 50% + 야외 50%
- 개: 실내 50% + 야외 50%

### 엔트리 연결

엔트리 AI 모델에서:
1. 웹캠으로 직접 사진 촬영 (데이터 수집)
2. "클래스 1", "클래스 2"로 분류 (라벨링)
3. 각 클래스당 20~30장 권장

## 2단계: 학습 (Training)

> **컴퓨터가 데이터에서 패턴을 찾는 과정**

### 학습이란?

컴퓨터가 수많은 예시를 보며 "고양이는 이런 특징이 있구나", "개는 저런 특징이 있구나"를 스스로 발견하는 과정이에요.

### 학습 과정

1. **초기 상태**: 아무것도 모름
   - 고양이인지 개인지 랜덤으로 맞춤
   - 정확도 50% (동전 던지기 수준)

2. **학습 시작**: 데이터를 반복해서 봄
   - 1회 (Epoch 1): "아, 귀 모양이 중요하구나"
   - 2회 (Epoch 2): "수염도 중요하네"
   - 3회 (Epoch 3): "눈 모양도 봐야겠다"

3. **가중치 조정**: 중요한 특징에 더 큰 점수 부여
   - 귀 모양: 가중치 높임
   - 털 색깔: 가중치 낮춤 (고양이/개 모두 다양)

4. **반복 학습**:
   - 10회, 20회, 50회... 반복
   - 점점 정확도 향상

### 학습 시간

| 데이터 크기 | 모델 복잡도 | 학습 시간 |
|:---:|:---:|:---:|
| 작음 | 간단 | 수 초 ~ 수 분 |
| 보통 | 보통 | 수 시간 ~ 수 일 |
| 큼 | 복잡 (딥러닝) | 수 주 ~ 수 개월 |

ChatGPT 같은 대형 모델은 수천 개의 GPU로 수개월간 학습했어요!

### 엔트리 연결

엔트리 AI 모델:
1. "모델 학습하기" 버튼 클릭
2. 진행 바가 100%까지 진행
3. 수십 초 ~ 수 분 소요 (간단한 모델이라 빠름)

## 3단계: 평가 (Testing)

> **학습한 모델이 얼마나 정확한지 확인**

### 평가란?

시험을 본다고 생각하면 돼요. 학습할 때 보지 못한 **새로운 데이터**로 테스트합니다.

### 왜 새로운 데이터로 평가할까?

**나쁜 평가 방법**: 학습 데이터로 평가
- 학생이 시험에 나온 문제를 미리 외워버림
- 100점을 받아도 진짜 실력인지 알 수 없음

**좋은 평가 방법**: 새로운 데이터로 평가
- 처음 보는 문제로 시험
- 진짜 실력을 측정 가능

### 데이터 분할

일반적으로 데이터를 다음과 같이 나눕니다:

```
전체 데이터 1,000장
 ├─ 학습 데이터 (Training): 700장 (70%)
 ├─ 검증 데이터 (Validation): 150장 (15%)
 └─ 테스트 데이터 (Testing): 150장 (15%)
```

- **학습 데이터**: 모델 학습에 사용
- **검증 데이터**: 학습 중간에 성능 확인
- **테스트 데이터**: 최종 성능 평가

### 평가 지표: 정확도 (Accuracy)

**정확도 = (맞춘 개수 / 전체 개수) × 100%**

예시:
- 테스트 데이터 100장
- 90장 맞춤, 10장 틀림
- 정확도 = 90%

### 성능 개선

정확도가 낮으면?
1. 데이터 더 수집
2. 모델 구조 변경
3. 학습 시간 늘리기
4. 다시 학습 (2단계로 돌아가기)

### 엔트리 연결

엔트리 AI 모델:
1. 학습 완료 후 정확도 표시 (예: 92%)
2. 웹캠으로 실시간 테스트
3. 잘못 분류하면 데이터 추가 후 재학습

## 4단계: 예측 (Prediction)

> **학습한 모델을 실전에 적용**

### 예측이란?

드디어 완성된 AI 모델을 실제로 사용하는 단계예요!

### 예측 과정

1. **새로운 입력**: 처음 보는 고양이 사진
2. **모델 적용**: 학습한 패턴으로 분석
3. **결과 출력**: "고양이입니다" (확률 95%)

### 실전 배포

완성된 모델을 다양한 곳에 활용:

| 분야 | 활용 예시 |
|:---|:---|
| 📱 **앱** | 구글 포토 (사진 자동 분류) |
| 🌐 **웹사이트** | Gmail (스팸 필터) |
| 🤖 **로봇** | 로봇 청소기 (장애물 인식) |
| 🚗 **자동차** | 테슬라 (자율주행) |
| 🏥 **병원** | AI 영상 진단 시스템 |

### 지속적인 모니터링

배포 후에도 계속 관리가 필요해요:

1. **성능 모니터링**: 정확도가 떨어지지 않는지 확인
2. **새 데이터 수집**: 환경이 변하면 새 데이터 추가
3. **재학습**: 주기적으로 모델 업데이트

예시: Gmail 스팸 필터
- 매일 새로운 스팸 기법 등장
- 지속적으로 데이터 수집 및 재학습
- 모델 계속 개선

### 엔트리 연결

엔트리 AI 모델:
1. 웹캠으로 새 이미지 입력
2. 실시간으로 분류 결과 표시
3. 블록 코딩과 연동하여 작품 제작
   - "만약 고양이면 → 야옹 소리"
   - "만약 개면 → 멍멍 소리"

## 4단계 과정 요약

| 단계 | 이름 | 설명 | 엔트리 |
|:---:|:---|:---|:---|
| 1 | 데이터 수집 | 학습할 재료 모으기 | 웹캠으로 사진 촬영 |
| 2 | 학습 | 패턴 발견 | "모델 학습하기" 버튼 |
| 3 | 평가 | 정확도 확인 | 정확도 % 표시 |
| 4 | 예측 | 실전 적용 | 새 이미지 분류 |

> 💡 **핵심 메시지**: 기계학습은 "데이터 수집 → 학습 → 평가 → 예측"의 4단계로 이루어집니다. 3주차 엔트리 실습에서 이 전 과정을 직접 체험하게 됩니다!

# 📝 정리 (5분)

## 핵심 개념 요약

오늘 배운 내용을 4가지로 정리해 봅시다.

### 1️⃣ AI ⊃ ML ⊃ DL 포함 관계

```
AI (인공지능) - 가장 큰 개념
 ⊃ ML (기계학습) - 데이터로 학습하는 AI
    ⊃ DL (딥러닝) - 신경망이 깊은 ML
```

- AI는 인간의 지능을 모방하는 모든 기술
- 기계학습은 AI를 구현하는 대표적 방법
- 딥러닝은 기계학습의 한 종류

### 2️⃣ 기계학습의 정의

> **기계학습**: 데이터로부터 패턴을 스스로 학습하는 AI 기술

**전통적 프로그래밍 vs 기계학습**
- 전통적: 규칙 작성 → 결과
- 기계학습: 데이터 + 결과 → 규칙 발견

### 3️⃣ 3가지 학습 유형

| 유형 | 특징 | 비유 | 대표 예시 |
|:---|:---|:---|:---|
| **지도학습** | 정답 있는 데이터로 학습 | 시험 공부 | 스팸 필터, 질병 진단 |
| **비지도학습** | 정답 없이 패턴 발견 | 그룹 나누기 | 고객 세분화 |
| **강화학습** | 시행착오로 보상 최대화 | 게임 연습 | AlphaGo, 자율주행 |

**💡 초등 AI 교육에서는 주로 지도학습을 다룹니다!**

### 4️⃣ 머신러닝 4단계 과정

```
1. 데이터 수집 → 2. 학습 → 3. 평가 → 4. 예측
```

- **1단계**: 좋은 데이터 모으기 (양, 질, 다양성)
- **2단계**: 패턴 발견 (반복 학습, 가중치 조정)
- **3단계**: 성능 확인 (새 데이터로 테스트)
- **4단계**: 실전 적용 (새로운 입력 예측)
