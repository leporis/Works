# 🎬 도입: AI, ML, DL의 포함 관계 (5분)

## 1차시 복습

> 📌 **핵심 질문**: "1차시에서 배운 AI, 그 안에는 무엇이 있을까요?"

지난 시간에 우리는 AI의 정의와 유형을 배웠어요. AI는 "인간의 지능을 모방하는 컴퓨터 시스템"이라고 했죠. 오늘은 AI를 구현하는 가장 중요한 방법인 **기계학습(Machine Learning)**에 대해 알아보겠습니다.

## AI, ML, DL의 포함 관계

**[그림 01] AI, ML, DL의 포함 관계**

```
AI (인공지능)
 ⊃ ML (기계학습)
    ⊃ DL (딥러닝)
```

### 🤖 AI (Artificial Intelligence, 인공지능)

- **가장 큰 개념**: 인간의 지능을 모방하는 모든 기술
- **예시**: 체스 프로그램(규칙 기반), 스팸 필터(기계학습), ChatGPT(딥러닝)

### 🧠 ML (Machine Learning, 기계학습)

- **AI의 하위 개념**: 데이터로부터 패턴을 **자동으로 학습**
- **예시**: 이메일 스팸 필터, 유튜브 추천, 날씨 예측

### 🔥 DL (Deep Learning, 딥러닝)

- **ML의 하위 개념**: 인공신경망을 여러 층으로 쌓아 복잡한 패턴 학습
- **예시**: ChatGPT, AlphaGo, 자율주행

> 💡 **비유**: AI = 모든 차량, ML = 엔진 있는 차량, DL = 첨단 전기차

# 📘 전개 1: 기계학습의 정의 (5분)

## 기계학습이란?

> **기계학습(Machine Learning)**: 컴퓨터가 데이터로부터 패턴을 스스로 학습하는 AI 기술

## 전통적 프로그래밍 vs 기계학습

**[그림 02] 전통적 프로그래밍 vs 기계학습**

### 전통적 프로그래밍

```
규칙 + 데이터 → 결과
```

프로그래머가 규칙을 직접 작성합니다. 예를 들어 스팸 필터라면:
- IF "무료" 포함 AND "당첨" 포함 → 스팸
- 문제점: 규칙을 모두 나열할 수 없고, 새로운 스팸 기법에 대응이 어려움

### 기계학습

```
데이터 + 결과 → 규칙 (패턴)
```

수천 개의 메일 데이터를 제공하면 컴퓨터가 스스로 패턴을 발견합니다.

## 일상 속 기계학습 예시

| 서비스 | 기계학습 활용 |
|:---|:---|
| 📧 Gmail 스팸 필터 | 메일 데이터로 스팸 패턴 학습 |
| 📺 유튜브 추천 | 시청 기록으로 취향 분석 |
| 🗣️ Siri, Alexa | 음성 데이터로 인식 학습 |
| 🛒 쿠팡 추천 | 구매 기록으로 선호 상품 예측 |

### 왜 기계학습이 필요한가?

> 🖼️ **그림 18** 왜 기계학습이 필요할까요? — 전통적 프로그래밍으로 해결하기 어려운 3가지 문제
>
> ① **규칙 작성이 어려운 문제**: "이 사진이 고양이인지 어떻게 규칙으로?" — 수천 가지 특징을 규칙으로 못 씀
> ② **복잡한 패턴이 있는 문제**: "이 환자의 질병 위험은?" — 수백 개 변수가 복잡하게 얽혀 있음
> ③ **변하는 환경을 따라가야 할 때**: "내일의 스팸 메일은 어떤 패턴?" — 기계학습은 새 데이터로 자동 업데이트

**교수 멘트**:
```
"스팸 필터를 떠올려 봅시다. 스팸 패턴이 매일 바뀌니까, 규칙을 사람이 매일 고칠 수 없잖아요?
기계학습은 새 데이터가 오면 스스로 규칙을 업데이트합니다. 이것이 기계학습의 강점이에요!"
```

> 💡 **핵심 메시지**: "사람이 규칙을 작성하는 대신, 컴퓨터가 데이터에서 규칙을 찾는 기술"

**전환**: "기계학습에는 크게 3가지 방법이 있어요. 먼저 그 방법들을 알아보고, 사과 농장 이야기에서 하나씩 만나봅시다!"

# 📗 전개 2: 머신러닝의 3가지 방법 + AI 사과농장 이야기 (30분)

## 🅰️ 머신러닝의 3가지 방법 — 이론 (12분)

### 3가지 학습 유형 개관 (2분)

기계학습은 **데이터로부터 학습하는 방법**에 따라 크게 3가지로 나뉩니다.

| 구분 | 지도학습 | 비지도학습 | 강화학습 |
|:---:|:---:|:---:|:---:|
| **라벨(정답)** | ⭕ 있음 | ❌ 없음 | 🎯 보상 |
| **학습 목표** | 정답 맞추기 | 패턴 찾기 | 보상 최대화 |
| **비유** | 시험 공부 | 그룹 나누기 | 게임 연습 |
| **대표 기술** | KNN, 선형회귀, 의사결정트리 | K-Means | Q-러닝 |

> 💡 **핵심**: 문제 유형에 따라 적절한 학습 방식을 선택합니다. 초등학교 AI 교육에서는 주로 **지도학습**(엔트리 이미지 분류)을 다룹니다.

**교수 멘트**:
```
"이 3가지를 기억해 두세요. 곧 사과농장 이야기에서 하나씩 만나게 됩니다!"
```

### 1️⃣ 지도학습 (Supervised Learning) — 이론 (4분)

> **지도학습**: 정답이 있는 데이터로 학습하는 방식

#### 핵심 개념

"선생님이 문제와 정답을 주고 공부하는 것"과 같아요.

**[그림 03] 지도학습 vs 비지도학습 데이터 구조 비교**

**학습 과정**:
1. 데이터 + 정답(Label) 제공
2. 컴퓨터가 데이터와 정답의 관계 학습
3. 새로운 데이터가 들어오면 정답 예측

#### 분류 vs 예측

| 유형 | 질문 | 출력 | 예시 |
|:---:|:---|:---|:---|
| **분류** (Classification) | "어떤 종류?" | 범주(카테고리) | 품종 구분, 배송지 결정 |
| **예측** (Regression) | "얼마?" | 숫자(연속값) | 가격 예측 |

#### 엔트리 AI 모델 연결

💡 3~4주차에 배울 **엔트리 AI 이미지 분류 모델**이 바로 지도학습!
- 고양이/개 사진 촬영 → 라벨 붙이기 → 학습 → 예측

---

### 2️⃣ 비지도학습 (Unsupervised Learning) — 이론 (3분)

> **비지도학습**: 라벨(정답 표시) 없이 데이터의 패턴을 스스로 찾는 방식

#### "정답이 없다"의 정확한 의미

💡 **"정답이 없다" = "라벨(정답 스티커)이 없다"**

**[그림 04] 라벨 없는 3가지 이유**

지도학습에서는 모든 데이터에 라벨이 붙어 있었어요.
- 지도학습: [고양이 사진] + "고양이" ✅
- 비지도학습: [고객 데이터] + ??? ❌ (어떤 그룹인지 모름)

**왜 라벨이 없을까?** 3가지 이유:
1. **라벨 자체가 없을 때** — 신규 고객의 취향을 아직 모름
2. **라벨 만들기 어려울 때** — 수백만 뉴스 기사에 일일이 분류 불가
3. **숨겨진 패턴을 찾고 싶을 때** — 사람도 몰랐던 새로운 그룹 발견

#### 군집화(Clustering) 개념

**[그림 05] 군집화 3단계 과정**

```
1단계: 입력 → 데이터만 제공 (라벨 없이)
2단계: 과정 → AI가 유사한 데이터끼리 자동 그룹화
3단계: 출력 → K개의 그룹 (사람이 나중에 이름 붙임)
```

#### 활용 사례: 고객 세분화

**[그림 06] 고객 세분화 결과 4개 그룹** — 쇼핑몰 고객 세분화 예시

### 3️⃣ 강화학습 (Reinforcement Learning) — 이론 (3분)

> **강화학습**: 시행착오를 통해 보상을 최대화하는 방향으로 학습

#### 핵심 개념

"게임하면서 점수로 배우기"

**[그림 07] 강화학습 루프**

**학습 과정**:
1. AI가 행동(Action) 선택
2. 환경이 보상(Reward) 제공: 좋은 행동 → +, 나쁜 행동 → -
3. 보상을 최대화하는 방향으로 학습

#### 강화학습의 구성 요소

| 요소 | 설명 | 예시 (게임) |
|:---|:---|:---|
| **Agent** | 학습하는 주체 | 게임 플레이어 |
| **Environment** | 상호작용하는 세계 | 게임 화면 |
| **Action** | 선택할 수 있는 행동 | 상하좌우 이동 |
| **Reward** | 행동에 대한 피드백 | 점수 +10, -5 |

#### 비유: 강아지 훈련

**[그림 08] 강아지 훈련 — 강화학습 비유**

- 앉으면 → 간식(+보상) 🦴
- 짖으면 → 무시(-보상) ❌
- 반복하며 좋은 행동을 학습!

#### 대표 예시

**[그림 09] 게임 점수 획득 — 강화학습 과정**

| 예시 | 보상(+) | 벌점(-) |
|:---|:---|:---|
| AlphaGo | 이기면 +1 | 지면 -1 |
| 자율주행 | 안전 주행 +1 | 사고 -100 |
| 게임 AI | 코인 획득 +10 | 게임 오버 -100 |

> 💡 사과농장에는 강화학습 장면이 없지만, 기계학습의 세 번째 기둥!
> "사과를 가장 효율적으로 수확하는 경로를 시행착오로 찾는 로봇"을 상상해 보세요.

### 🖥️ 지도학습 시뮬레이터 시연 (7분)

#### 사과농장 1장: 🌳 수확 — 딥러닝 CNN

농장에서 카메라로 사과를 촬영하면, CNN(합성곱 신경망)이 익은 정도를 분석합니다.
- 입력: 사과 사진 + 정답("익음 92%")
- 학습: 색상, 형태 패턴 학습
- 예측: 새 사과 → "익었다!" → 로봇이 수확

> 💡 CNN은 딥러닝의 일종이고, 딥러닝은 기계학습의 한 종류예요. 결국 지도학습의 확장판!

> 🖼️ **그림 22** CNN 사과분류 시뮬레이터 학습 모드 — 학습 데이터·손실/정확도 차트·CNN 아키텍처·학습 설정 패널 구성
>
> 📢 **예고**: "CNN이 사과를 분류하는 과정을 다음 주(2주차)에 이 시뮬레이터로 직접 체험해봅시다!"

#### 사과농장 2장: 🔍 품종 구분 — KNN + 시뮬레이터 시연 (2분)

수확한 사과에 여러 품종이 섞여 있어요. KNN(K-최근접 이웃)으로 분류합니다.
- 새 사과가 들어오면, 기존 데이터에서 **가장 가까운 K개 이웃** 확인
- 이웃 중 다수의 품종으로 분류
- 예: K=3일 때, 가까운 3개 중 "홍로"가 2개 → "홍로"로 분류

> **시연 목표**: "가장 가까운 이웃이 많은 쪽으로 분류" — 사과농장 2장 원리
>
> 🖼️ **그림 13** KNN 분류 시뮬레이터 학습 모드 — 데이터 설정·분류 시각화·학습 설정 3패널 구성

| 단계 | 시연 내용 | 설명 |
|:---:|:---|:---|
| 설정 (15초) | 두 범주 데이터 배치 | 🔵파란점(홍로), 🟢초록점(부사) |
| 실행 (1분) | K=3 설정, 새 데이터 분류 | 가까운 3개 이웃 중 다수결 |
| 해설 (45초) | 사과농장 연결 | "품종 라벨이 붙어 있어야 분류 가능 → 지도학습!" |

**교수 멘트**:
```
"사과농장 2장에서 섞인 품종을 구분했죠?
KNN은 새 사과가 기존의 어떤 품종 데이터와 가장 가까운지 보고 판단합니다.
정답 라벨이 있어야 학습할 수 있어요 — 이것이 지도학습입니다!"
```

#### 사과농장 3장: 📦 등급 분류 — K-Means + 시뮬레이터 시연

수확한 사과를 크기별로 나눠야 하는데, 미리 정해진 기준이 없어요.
- 크기 라벨("대/중/소")이 붙어 있지 않음
- K-Means가 비슷한 크기끼리 자동으로 3그룹으로 나눔
- AI가 스스로 "이 정도면 대, 이 정도면 중, 이 정도면 소"를 발견

> **시연 목표**: "라벨 없이 AI가 스스로 그룹을 만든다" — 사과농장 3장 원리
>
> 🖼️ **그림 16** K-Means 클러스터링 시뮬레이터 학습 모드 — 데이터 설정·클러스터 시각화·학습 설정 3패널 구성

| 단계 | 시연 내용 | 설명 |
|:---:|:---|:---|
| 설정 (30초) | 라벨 없는 데이터 배치 | 점들의 분포만 보임 |
| 실행 (1분30초) | K=3 설정, 클러스터링 실행 | 중심점이 이동하며 그룹 형성 |
| 해설 (1분) | 지도학습과 비교 | "KNN은 라벨 필요, K-Means는 라벨 불필요!" |

**교수 멘트**:
```
"아까 지도학습(KNN)에서는 라벨이 있어야 분류했죠?
지금 K-Means에는 라벨이 전혀 없어요!

그런데도 AI가 스스로 비슷한 점끼리 묶어서 3개 그룹을 만들어냈어요.
사과농장 3장에서 크기 기준 없이 대/중/소로 나눈 것이 바로 이 원리입니다!"
```

> 🖼️ **그림 21** Inertia(관성) 개념도 — 클러스터 품질 지표
>
> **Inertia가 낮을수록** 데이터가 중심점에 가까이 모여 있어 **좋은 클러스터**, 높으면 흩어진 클러스터.
> K-Means 알고리즘은 Inertia를 최소화하는 방향으로 중심점을 이동시킵니다.
