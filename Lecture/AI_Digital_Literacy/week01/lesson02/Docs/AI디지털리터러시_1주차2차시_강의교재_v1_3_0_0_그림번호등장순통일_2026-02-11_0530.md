# 🎬 도입: AI, ML, DL의 포함 관계

## 1차시 복습

> 📌 **핵심 질문**: "1차시에서 배운 AI, 그 안에는 무엇이 있을까요?"

지난 시간에 우리는 AI의 정의와 유형을 배웠어요. AI는 "인간의 지능을 모방하는 컴퓨터 시스템"이라고 했죠. 

**1차시 핵심 내용**:
- AI의 정의: 인간의 지능을 모방하는 컴퓨터 시스템
- 약한 AI vs 강한 AI: 현재 모든 AI는 특정 작업 전문가인 약한 AI
- 기능별 분류: 인식, 예측, 분류, 생성 AI
- AI 발전 역사: 봄과 겨울을 반복하며 발전, 현재 제3의 봄

그런데 AI를 구현하는 방법은 여러 가지가 있어요. 오늘은 그중에서도 가장 중요한 **기계학습(Machine Learning)**에 대해 알아보겠습니다.

## AI, ML, DL의 포함 관계

**[그림 01] AI, ML, DL의 포함 관계**

![AI, ML, DL의 포함 관계](AI디지털리터러시_1주차1차시_06_AI_ML_DL포함관계_v1_0_1_0_점선삭제_20260208_0010.png)

AI, ML, DL은 자주 혼동되는 용어입니다. 이 세 가지의 관계를 정확히 이해하는 것이 오늘 수업의 첫 번째 목표예요.

```
AI (인공지능)
 ⊃ ML (기계학습)
    ⊃ DL (딥러닝)
```

### 🤖 AI (Artificial Intelligence, 인공지능)

**정의**: 인간의 지능을 모방하는 모든 컴퓨터 기술

AI는 가장 큰 개념이에요. 인간처럼 생각하고, 학습하고, 문제를 해결하는 모든 컴퓨터 시스템을 포함합니다.

**AI의 범위**:
- 규칙 기반 시스템 (Rule-based Systems)
- 전문가 시스템 (Expert Systems)
- 기계학습 (Machine Learning)
- 딥러닝 (Deep Learning)

**역사적 예시**:
- **1950~1960년대**: 규칙 기반 체스 프로그램
  - 사람이 "이런 상황에서는 이렇게 둬라" 규칙을 모두 작성
  - IF-THEN 규칙의 집합
  
- **1980년대**: 전문가 시스템
  - 의사의 진단 지식을 규칙으로 만든 MYCIN
  - 수백 개의 IF-THEN 규칙으로 구성
  
- **2000년대 이후**: 기계학습과 딥러닝
  - 규칙을 작성하는 대신 데이터로 학습
  - 알파고, ChatGPT 등

### 🧠 ML (Machine Learning, 기계학습)

**정의**: AI를 구현하는 방법 중 하나로, 데이터로부터 패턴을 자동으로 학습하는 기술

기계학습은 AI의 하위 개념이자, 현대 AI의 핵심 기술이에요.

**핵심 특징**: 
- 사람이 규칙을 일일이 작성하지 않음
- 데이터만 제공하면 컴퓨터가 스스로 패턴 발견
- 데이터가 많을수록 성능 향상

**등장 배경**:
1950년대 후반, 아서 새뮤얼(Arthur Samuel)이 체커 게임 프로그램을 만들면서 기계학습이라는 용어를 처음 사용했어요. 

> "컴퓨터에게 명시적으로 프로그래밍하지 않고도 학습할 수 있는 능력을 부여하는 연구 분야"

**일상 속 기계학습**:
- 이메일 스팸 필터
- 유튜브 동영상 추천
- 음성 인식 (Siri, Alexa)
- 얼굴 인식 (스마트폰 잠금 해제)
- 상품 추천 (쿠팡, 아마존)

### 🔥 DL (Deep Learning, 딥러닝)

**정의**: 기계학습의 한 종류로, 인공신경망을 여러 층으로 쌓아 복잡한 패턴을 학습하는 기술

딥러닝은 기계학습의 하위 개념이에요. 모든 딥러닝은 기계학습이지만, 모든 기계학습이 딥러닝은 아니에요.

**"Deep"의 의미**:
- 신경망의 층(Layer)이 깊다(많다)는 뜻
- 기존 기계학습: 층이 1~2개
- 딥러닝: 층이 수십~수백 개

**특징**:
- 복잡한 패턴 학습 가능
- 대량의 데이터와 컴퓨팅 파워 필요
- 이미지, 음성, 텍스트 처리에 탁월

**대표 사례**:
- **ChatGPT** (자연어 처리)
  - 수천억 개의 텍스트 데이터로 학습
  - 수백 개 층의 신경망 사용
  
- **AlphaGo** (바둑)
  - 수백만 개의 바둑 기보 학습
  - 스스로 대국하며 실력 향상
  
- **테슬라 자율주행** (이미지 인식)
  - 수백만 킬로미터 주행 데이터
  - 도로, 신호등, 차량 등을 실시간 인식

## 세 개념의 관계 정리

**포함 관계**:
```
AI (가장 넓은 개념)
├─ 규칙 기반 시스템
├─ 전문가 시스템
└─ ML (AI의 일부)
   ├─ 의사결정 트리
   ├─ 서포트 벡터 머신
   └─ DL (ML의 일부)
      ├─ 합성곱 신경망 (CNN)
      ├─ 순환 신경망 (RNN)
      └─ 트랜스포머 (Transformer)
```

**비유로 이해하기**:

AI를 "차량"에 비유하면:
- **AI** = 모든 종류의 차량 (자전거, 오토바이, 자동차, 비행기)
- **ML** = 엔진이 있는 차량 (오토바이, 자동차, 비행기)
- **DL** = 첨단 전기차 (테슬라, 리비안)

자전거(규칙 기반)도 차량이지만, 엔진(학습 능력)은 없어요. 오토바이와 자동차(기계학습)는 엔진으로 움직이고, 테슬라(딥러닝)는 가장 첨단 기술이 적용된 거죠.

**시대별 주력 기술**:
- **1950~1980년대**: 규칙 기반 AI
- **1990~2010년대**: 기계학습 (ML)
- **2010년대~현재**: 딥러닝 (DL)

> 💡 **핵심 메시지**: AI ⊃ ML ⊃ DL 관계를 이해하면, 각 기술의 범위와 특성을 명확히 구분할 수 있습니다. 오늘 우리가 집중해서 배울 내용은 **기계학습(ML)**입니다!

# 📘 전개 1: 기계학습의 정의

## 기계학습이란?

> **기계학습(Machine Learning)**: 컴퓨터가 데이터로부터 패턴을 스스로 학습하는 AI 기술

기계학습은 1959년 아서 새뮤얼(Arthur Samuel)이 정의한 개념입니다:

> "명시적으로 프로그래밍하지 않고도 컴퓨터가 학습할 수 있는 능력을 연구하는 분야"

이 정의를 풀어서 설명하면:
- **명시적으로 프로그래밍하지 않고** = 사람이 규칙을 일일이 작성하지 않고
- **컴퓨터가 학습할 수 있는 능력** = 데이터에서 스스로 패턴을 발견하는 능력

## 전통적 프로그래밍 vs 기계학습

기계학습을 이해하려면, 먼저 전통적 프로그래밍과 어떻게 다른지 알아야 해요.

### 전통적 프로그래밍

```
규칙 (사람이 작성) + 데이터 → 결과
```

**작동 방식**:
1. 프로그래머가 모든 규칙을 작성
2. 컴퓨터는 규칙에 따라 데이터 처리
3. 결과 출력

**예시 1: 스팸 메일 필터 (전통적 방식)**

프로그래머가 규칙 작성:
```
IF 메일에 "무료" 포함 AND "당첨" 포함 → 스팸
IF 메일에 "대출" 포함 AND "즉시" 포함 → 스팸
IF 발신자가 블랙리스트에 있음 → 스팸
IF 링크가 10개 이상 → 스팸
...
```

**문제점**:
1. **규칙을 모두 나열하기 불가능**
   - 스팸 메일의 종류는 무한히 많음
   - 새로운 스팸 기법이 계속 등장
   
2. **유지보수가 복잡**
   - 새 규칙 추가 시 기존 규칙과 충돌 가능
   - 규칙이 수천 개가 되면 관리 불가능
   
3. **적응력 부족**
   - 환경이 변하면 규칙을 다시 작성해야 함
   - "당첨"이라는 단어를 쓰는 정상 메일도 걸림

### 기계학습

```
데이터 + 결과(정답) → 규칙(패턴) 발견
```

**작동 방식**:
1. 컴퓨터에게 데이터와 정답을 제공
2. 컴퓨터가 스스로 패턴 학습
3. 새로운 데이터에 학습한 패턴 적용

**예시 2: 스팸 메일 필터 (기계학습 방식)**

컴퓨터에게 수천 개의 메일 제공:
- 스팸 메일 5,000개 (정답: "스팸")
- 정상 메일 5,000개 (정답: "정상")

컴퓨터가 스스로 패턴 발견:
- "무료", "당첨"이 자주 나오면 스팸일 확률 ↑
- 친구 이메일에는 이름이 자주 나옴
- 스팸은 링크가 많고, 이미지 비율이 높음
- 문장 구조가 이상한 경우가 많음

**장점**:
1. **사람이 규칙을 작성하지 않아도 됨**
   - 데이터만 제공하면 자동으로 학습
   
2. **새로운 패턴 자동 학습**
   - 새로운 스팸 기법도 데이터로 학습
   - 규칙 수정 불필요
   
3. **성능 향상**
   - 데이터가 많을수록 정확도 ↑
   - 인간이 발견 못한 패턴도 찾음

## 비교표

| 구분 | 전통적 프로그래밍 | 기계학습 |
|:---|:---|:---|
| **입력** | 규칙 + 데이터 | 데이터 + 정답 |
| **출력** | 결과 | 규칙(패턴) |
| **규칙 작성** | 사람이 일일이 작성 | 컴퓨터가 자동 발견 |
| **적응력** | 낮음 (환경 변화 대응 어려움) | 높음 (새 데이터로 재학습) |
| **예시** | IF-THEN 규칙 체스 프로그램 | 스팸 필터, 추천 시스템 |
| **장점** | 명확하고 예측 가능 | 복잡한 패턴 학습 가능 |
| **단점** | 복잡한 문제 해결 어려움 | 대량의 데이터 필요 |

## 왜 기계학습이 필요한가?

기계학습이 필요한 이유는 크게 세 가지예요.

### 1️⃣ 규칙을 작성하기 어려운 문제

**예시: 고양이 vs 개 구분**

전통적 프로그래밍으로 한다면:
```
IF 귀가 뾰족하면 → 고양이?
→ 문제: 치와와도 귀가 뾰족함 ❌

IF 수염이 있으면 → 고양이?
→ 문제: 일부 개도 수염이 있음 ❌

IF 크기가 작으면 → 고양이?
→ 문제: 페르시안 고양이는 큼, 치와와는 작음 ❌
```

규칙을 명확히 작성하기 불가능해요! 귀 모양, 눈 모양, 털 패턴, 체형, 꼬리 등 수백 가지 요소를 어떻게 조합해야 할지 정확한 규칙을 만들 수 없어요.

**기계학습으로 한다면**:
1. 고양이 사진 1,000장 + "고양이" 라벨
2. 개 사진 1,000장 + "개" 라벨
3. 컴퓨터가 스스로 특징 학습
4. 우리가 모르는 미세한 차이까지 발견

### 2️⃣ 복잡한 패턴이 있는 문제

**예시: 음성 인식**

음성 인식의 어려움:
- 같은 단어도 사람마다 발음이 다름
  - 남성/여성 목소리
  - 어린이/성인 목소리
  - 지역 억양 (서울, 경상도, 전라도...)
  
- 말하는 속도가 제각각
  - 빠르게 말하는 사람
  - 천천히 또박또박 말하는 사람
  
- 주변 소음
  - 조용한 실내
  - 시끄러운 거리
  - 바람 소리, 음악 소리

이런 모든 경우를 규칙으로 만들기는 불가능해요!

**기계학습의 해결**:
- 수백만 개의 음성 데이터로 학습
- 다양한 환경, 다양한 사람의 목소리 학습
- 패턴을 자동으로 발견하고 적응

### 3️⃣ 환경이 계속 변하는 문제

**예시: 주식 가격 예측**

주식 시장의 특징:
- 경제 상황이 계속 변함
- 국제 정세, 환율, 금리 등 영향
- 과거 규칙이 미래에 안 맞을 수 있음

전통적 프로그래밍:
- "금리가 오르면 주가 하락" 규칙 작성
- 하지만 예외 상황이 너무 많음
- 코로나19 같은 예상 못한 사건 대응 불가

기계학습:
- 최신 데이터로 계속 재학습
- 새로운 패턴을 지속적으로 발견
- 환경 변화에 자동 적응

| 📺 **유튜브 추천** | 시청 기록 분석, 관련 영상 추천 | 수조 개의 시청 기록 |
| 🗣️ **Siri, Alexa** | 음성을 텍스트로 변환 | 수백만 명의 음성 |
| 📸 **얼굴 인식** | 얼굴 특징 추출, 본인 확인 | 수천 장의 얼굴 사진 |
| 🚗 **자율주행** | 도로, 차량, 신호등 인식 | 수백만 km 주행 데이터 |
| 🛒 **쿠팡 추천** | 구매 기록으로 선호 상품 예측 | 수억 건의 구매 기록 |
| 📰 **네이버 뉴스** | 관심사 분석, 맞춤 뉴스 추천 | 수억 건의 클릭 기록 |
| 🎬 **넷플릭스** | 시청 패턴으로 영화/드라마 추천 | 수억 명의 시청 기록 |

**공통점**:
- 모두 대량의 데이터가 필요
- 사람이 규칙을 작성하지 않음
- 데이터가 많을수록 정확도 향상
- 사용자 행동에 따라 계속 학습

> 💡 **핵심 메시지**: 기계학습은 "사람이 규칙을 작성하는 대신, 컴퓨터가 데이터에서 규칙을 찾는" 기술입니다. 규칙을 명확히 정의하기 어렵거나, 패턴이 복잡하거나, 환경이 계속 변하는 문제에 특히 유용해요!

# 📗 전개 2: 기계학습의 3가지 학습 유형

기계학습은 크게 세 가지 방식으로 학습합니다. 각각의 특징을 자세히 알아봅시다.

## 1️⃣ 지도학습 (Supervised Learning)

> **지도학습**: 정답(라벨)이 있는 데이터로 학습하는 방식

### 핵심 개념

"선생님이 문제와 정답을 주고 공부하는 것"과 같아요.

**비유**:
- 학생이 시험 공부할 때
- 문제집에 문제 + 정답이 있음
- 문제를 풀고, 정답과 비교하며 학습
- AI도 마찬가지로 데이터 + 정답으로 학습

**학습 과정**:
1. **데이터 + 정답(Label) 제공**
   - 예: 고양이 사진 + "고양이"
   - 예: 개 사진 + "개"
   
2. **컴퓨터가 데이터와 정답의 관계 학습**
   - 고양이의 특징: 뾰족한 귀, 동그란 눈, 작은 코...
   - 개의 특징: 큰 코, 축 처진 귀, 긴 주둥이...
   
3. **새로운 데이터가 들어오면 정답 예측**
   - 새 동물 사진 → "고양이일 확률 95%"

### 데이터 구조

지도학습 데이터는 반드시 **입력 + 라벨(정답)**이 쌍으로 있어야 해요.

```
[입력 데이터]          [라벨]
고양이 사진   →   "고양이"
개 사진      →   "개"
토끼 사진    →   "토끼"
```

라벨이 없으면 지도학습을 할 수 없어요!

### 지도학습의 두 가지 종류

#### 📊 분류 (Classification)

**정의**: 입력 데이터를 범주(카테고리)로 구분

**특징**:
- 출력이 이산적 (불연속적)
- "A 또는 B 또는 C" 형태
- 범주의 개수가 정해져 있음

**예시**:

| 문제 | 입력 | 출력 (범주) | 범주 개수 |
|:---|:---|:---|:---:|
| 이메일 분류 | 메일 내용 | 스팸 / 정상 | 2개 |
| 질병 진단 | 증상, 검사 결과 | 독감 / 감기 / 정상 | 3개 |
| 꽃 종류 분류 | 꽃잎 길이, 너비 등 | 장미 / 튤립 / 백합 / 국화 | 4개 |
| 손글씨 인식 | 숫자 이미지 | 0~9 | 10개 |
| 동물 분류 | 사진 | 고양이 / 개 / 토끼 / 새... | 여러 개 |

**실생활 예시**:
- **스팸 필터**: 스팸 or 정상
- **얼굴 인식**: 본인 or 타인
- **질병 진단**: 암 or 정상
- **이미지 인식**: 개 or 고양이 or 사람

#### 📈 예측/회귀 (Regression)

**정의**: 입력 데이터로 연속적인 숫자 값을 예측

💡 **쉬운 이해**: 
- **회귀**는 "**얼마나?**"라는 질문에 답하는 것
- 연속적인 숫자를 맞히는 학습

**교수 멘트**:
```
"회귀(Regression)라는 말이 어렵게 느껴지죠?
간단하게 말하면 '숫자를 맞히는 것'이에요.

예를 들어볼까요?
- 이 집은 **얼마**일까? → 3억 5천만원
- 시험 **몇 점** 받을까? → 85점
- 내일 기온이 **몇 도**일까? → 18도

이렇게 숫자로 된 답을 맞히는 게 회귀예요!"
```

**회귀 vs 분류 비교표**:

| 구분 | 회귀 | 분류 |
|:---:|:---|:---|
| **질문** | "얼마나?" | "어느 것?" |
| **답 형태** | 숫자 (연속적) | 범주 (구분) |
| **예시 1** | 시험 점수: 85점 | 합격/불합격: 합격 |
| **예시 2** | 집값: 3억 5천 | 아파트/빌라: 아파트 |
| **예시 3** | 기온: 18도 | 더위/추위: 더위 |
| **예시 4** | 과일 무게: 180g | 과일 종류: 사과 |

**강조**:
- 회귀는 0~100점, 0원~10억원처럼 **연속적인 범위**의 숫자
- 분류는 합격/불합격, 고양이/개처럼 **정해진 범주** 중 하나

**예시**:

| 문제 | 입력 | 출력 (숫자) |
|:---|:---|:---|
| 집값 예측 | 평수, 위치, 연식 | 5억 2천만 원 |
| 성적 예측 | 공부 시간, 출석률 | 85점 |
| 기온 예측 | 시간, 계절, 습도 | 23.5도 |
| 주식 가격 예측 | 과거 가격, 거래량 | 52,300원 |

### 지도학습 상세 예시: 고양이 vs 개 분류

**1단계: 데이터 준비**
```
학습 데이터:
- 고양이 사진 1,000장 + 라벨 "고양이"
- 개 사진 1,000장 + 라벨 "개"
```

**2단계: 학습 과정**

컴퓨터가 스스로 특징 발견:
- 귀 모양: 고양이는 뾰족, 개는 축 처짐
- 눈 모양: 고양이는 동그랗고 큼, 개는 작음
- 털 패턴: 고양이는 부드러움, 개는 거침
- 체형: 고양이는 날씬, 개는 근육질
- 얼굴 비율: 코와 눈의 위치 등

**3단계: 평가**

새로운 사진으로 테스트:
- 고양이 사진 200장 → 정확도 95%
- 개 사진 200장 → 정확도 93%
- 전체 정확도: 94%

**4단계: 예측**

실전 적용:
```
입력: 새로운 동물 사진
→ AI 분석
→ 출력: "고양이일 확률 98%"
```

### 엔트리 AI 모델 연결

💡 **매우 중요**: 3주차에 배울 **엔트리 AI 이미지 분류 모델**이 바로 지도학습입니다!

**엔트리에서 직접 체험**:
1. **데이터 수집**: 웹캠으로 고양이/개 사진 찍기
2. **라벨 부여**: "이건 고양이", "이건 개" 표시
3. **모델 학습**: "모델 학습하기" 버튼 클릭
4. **평가**: 정확도 확인 (예: 92%)
5. **예측**: 새 사진으로 실시간 테스트

→ 이 전체 과정이 바로 지도학습이에요!

## 2️⃣ 비지도학습 (Unsupervised Learning)

> **비지도학습**: 라벨(정답 표시) 없이 데이터의 패턴을 스스로 찾는 방식

### "정답이 없다"의 정확한 의미

💡 **매우 중요한 개념!**

많은 사람들이 비지도학습을 "정답이 없다"고 표현하는데, 이게 정확히 무슨 뜻일까요?

**정확한 의미**: "정답이 없다" = "라벨(정답 스티커)이 없다"

### 지도학습 vs 비지도학습 데이터 구조 비교

**[그림 02] 지도학습 vs 비지도학습 데이터 구조 비교**

![지도학습vs비지도학습 데이터구조 비교](AI디지털리터러시_1주차2차시_15_지도학습vs비지도학습_데이터구조비교.png)

**지도학습 (Supervised Learning)**:
```
데이터 + 라벨(정답)

[고양이 사진] + "고양이" ✅
[개 사진] + "개" ✅
[토끼 사진] + "토끼" ✅

→ 모든 데이터에 정답 스티커가 붙어 있음!
```

**비지도학습 (Unsupervised Learning)**:
```
데이터만 (라벨 없음)

[고객 A 데이터] (어떤 그룹인지 모름)
[고객 B 데이터] (어떤 그룹인지 모름)
[고객 C 데이터] (어떤 그룹인지 모름)

→ 정답 스티커가 없어도 AI가 스스로 그룹 발견!
```

### 왜 비지도학습을 사용하는가?

**[그림 03] 라벨이 없는 3가지 이유**

![라벨없는 3가지 이유](AI디지털리터러시_1주차2차시_17_라벨없는_3가지이유.png)

라벨이 있으면 더 좋은데, 왜 굳이 라벨 없이 학습할까요? 세 가지 이유가 있어요.

#### 1️⃣ 라벨 자체가 없을 때

**예시: 신규 고객 세분화**
```
상황: 온라인 쇼핑몰에 신규 고객 10,000명 가입
데이터: 나이, 성별, 거주지, 관심사 등
문제: 이들이 어떤 그룹인지 아직 모름 (라벨 없음)
```

해결 방법:
- ❌ 지도학습 불가능: "이 고객은 알뜰형"이라는 라벨이 없음
- ✅ 비지도학습: 구매 패턴 분석 → 자동으로 3~5개 그룹 발견
- 결과: "알뜰형", "명품 선호형", "트렌드 민감형" 등 자동 분류

#### 2️⃣ 라벨 만들기 어려울 때

**예시: 뉴스 기사 자동 분류**
```
상황: 매일 수백만 개 뉴스 기사 생성
데이터: 뉴스 텍스트
문제: 일일이 라벨 붙이기 불가능 (시간/비용)
```

라벨을 만들려면:
- 사람이 수백만 개 기사를 읽고
- 하나하나 "정치", "경제", "스포츠" 라벨 붙여야 함
- 하루에 수천 개씩 쏟아지는데 불가능!

해결 방법:
- 비지도학습: 단어 패턴으로 유사 기사끼리 자동 분류
- AI가 스스로 주제별로 그룹화
- 결과: 정치, 경제, 스포츠 등 자동 분류

#### 3️⃣ 숨겨진 패턴을 발견하고 싶을 때

**예시: 새로운 고객 그룹 찾기**
```
상황: 기존에 "남성/여성" 2개 그룹으로만 분류
데이터: 고객 구매 패턴
목표: 사람도 몰랐던 새로운 그룹 발견
```

해결 방법:
- 비지도학습으로 데이터 분석
- AI가 5개의 새로운 그룹 발견:
  1. "10~20대 남성 게이머"
  2. "30~40대 여성 육아맘"
  3. "50대 남성 골프 애호가"
  4. "20대 여성 K팝 팬"
  5. "60대 이상 건강 관심층"

→ 사람이 미리 정한 그룹이 아닌, 데이터에서 드러난 진짜 그룹!

### 군집화 (Clustering): 비지도학습의 대표 방법

**정의**: 유사한 데이터끼리 자동으로 그룹화하는 방법

**작동 원리**:
1. **입력**: 라벨 없는 데이터
   - 예: 고객 1,000명의 구매 기록
   
2. **과정**: 유사도 계산
   - 구매 패턴이 비슷한 고객끼리 묶음
   - 거리 계산, 패턴 분석 등
   
3. **출력**: 그룹 A, B, C...
   - AI가 스스로 3~5개 그룹 발견
   - 각 그룹의 특징 분석

### 비지도학습 구체적 예시

#### 예시 1: 쇼핑몰 고객 세분화

**[그림 05] 고객 세분화 결과 (4개 그룹)**

![고객세분화 결과 4개 그룹](AI디지털리터러시_1주차2차시_18_고객세분화결과_4개그룹.png)

**상황**:
- 신규 가입 고객 10,000명
- 데이터: 나이, 구매 금액, 방문 횟수, 관심 카테고리
- 라벨 없음 (어떤 그룹인지 모름)

**비지도학습 적용**:
1. 구매 패턴 분석
2. 유사한 고객끼리 자동 그룹화
3. 4개 그룹 발견

**결과**:
- **그룹 A (알뜰형)**: 저가 상품, 할인 행사 때만 구매
- **그룹 B (명품 선호형)**: 고가 브랜드, 정가 구매
- **그룹 C (트렌드 민감형)**: 신상품 즉시 구매, 20~30대
- **그룹 D (실속형)**: 가성비 중시, 리뷰 꼼꼼히 확인

**활용**:
- 각 그룹별 맞춤 마케팅
- 그룹 A에게 할인 쿠폰 발송
- 그룹 B에게 신상품 VIP 미리보기

#### 예시 2: 넷플릭스 추천 시스템

**상황**:
- 수억 명의 사용자
- 데이터: 시청 기록, "좋아요" 기록
- 문제: "이 영화를 본 사람은?" 추천

**비지도학습 적용**:
1. 시청 패턴이 비슷한 사용자끼리 그룹화
2. 그룹 내에서 자주 시청한 콘텐츠 추출
3. 비슷한 사람들이 본 영화 추천

**예시**:
```
사용자 A: 〈기생충〉, 〈설국열차〉, 〈옥자〉 시청
        → 봉준호 감독 좋아하는 그룹

비슷한 그룹 사람들이 본 영화:
→ 〈마더〉, 〈살인의 추억〉 추천
```

#### 예시 3: 뉴스 자동 주제 분류

**상황**:
- 매일 수백만 개 뉴스 기사
- 데이터: 기사 텍스트
- 라벨 없음 (주제를 일일이 붙일 수 없음)

**비지도학습 적용**:
1. 기사의 단어 빈도 분석
2. 비슷한 단어가 많은 기사끼리 그룹화
3. 주제별 자동 분류

**결과**:
- 그룹 1: "대통령", "국회", "정당" → 정치
- 그룹 2: "주가", "환율", "금리" → 경제
- 그룹 3: "골프", "야구", "축구" → 스포츠
- 그룹 4: "배우", "영화", "드라마" → 연예

### 초등교육 비유

**비유 1: 친구 그룹 나누기**
```
상황: 반 친구들 30명을 그룹으로 나누려고 해요
방법 1 (지도학습): 선생님이 미리 "조용한 아이", "활발한 아이" 라벨 붙임
방법 2 (비지도학습): 친구들이 스스로 비슷한 애들끼리 모임
```

**비유 2: 레고 블록 정리**
```
지도학습: 색깔별로 빨강/파랑/노랑 통에 정리 (정답 있음)
비지도학습: 비슷하게 생긴 블록끼리 자유롭게 묶기 (정답 없이 스스로 그룹화)
```

### 지도학습 vs 비지도학습 종합 비교

| 구분 | 지도학습 | 비지도학습 |
|:---:|:---:|:---:|
| **라벨(정답)** | ⭕ 있음 | ❌ 없음 |
| **데이터 구조** | 입력 + 라벨 | 입력만 |
| **학습 목표** | 정답 맞추기 | 패턴 찾기 |
| **비유** | 시험 공부<br>(문제+정답) | 친구 그룹 나누기<br>(정답표 없이) |
| **예시** | 고양이/개 구분<br>스팸 필터 | 고객 세분화<br>뉴스 분류 |
| **장점** | 정확한 예측 가능 | 라벨 작업 불필요 |
| **단점** | 라벨 작업 필요 | 결과 해석 필요 |
| **사용 시기** | 정답을 아는 경우 | 정답을 모르는 경우 |

**강조 포인트**:
- 지도학습: 모든 데이터에 "정답 스티커"가 붙어 있음 ✅
- 비지도학습: "정답 스티커"가 없어도 AI가 스스로 그룹 발견 🔍
- 둘 다 중요하고, 상황에 따라 선택!

## 3️⃣ 강화학습 (Reinforcement Learning)

> **강화학습**: 시행착오를 통해 보상을 최대화하는 방향으로 학습하는 방식

### 핵심 개념

"게임을 하면서 점수로 배우는 것"과 같아요.

**비유**:
- 게임을 처음 시작할 때
- 이것저것 눌러보며 시행착오
- 점수가 오르면 "잘했구나!" → 그 행동 반복
- 게임 오버되면 "실수했구나!" → 그 행동 피함

### 작동 원리

**[그림 06] 강화학습 루프**

![강화학습 루프](AI디지털리터러시_1주차2차시_14_강화학습루프.png)

**구성 요소**:

1. **Agent (에이전트)**
   - 학습하는 주체 (AI)
   - 행동을 선택하고 실행
   
2. **Environment (환경)**
   - AI가 활동하는 공간
   - 게임 화면, 도로, 바둑판 등
   
3. **State (상태)**
   - 현재 상황
   - 게임 화면, 차의 위치, 바둑판 상태 등
   
4. **Action (행동)**
   - AI가 취할 수 있는 선택
   - 위/아래/좌/우, 가속/감속, 바둑돌 놓기 등
   
5. **Reward (보상)**
   - 행동의 결과
   - +점수 (좋은 행동) or -점수 (나쁜 행동)

### 학습 과정

**단계별 설명**:

1. **초기 상태**
   - AI가 환경을 관찰
   - 예: 게임 시작 화면
   
2. **행동 선택**
   - AI가 행동을 선택
   - 처음에는 랜덤 (무작위)
   
3. **행동 실행**
   - 선택한 행동을 환경에 적용
   - 예: 오른쪽으로 이동
   
4. **보상 받기**
   - 행동의 결과를 보상으로 받음
   - 코인 먹음 → +10점
   - 구덩이 빠짐 → -100점
   
5. **학습**
   - 보상이 높은 행동 → 더 자주 선택
   - 보상이 낮은 행동 → 피함
   
6. **반복**
   - 수천~수만 번 반복하며 최적 전략 학습

### 시각화로 이해하기

#### **[그림 07] 강아지 훈련으로 이해하기** 🐕

![강아지 훈련 - 강화학습 비유](AI디지털리터러시_1주차2차시_19_강아지훈련_강화학습비유.png)

강화학습의 원리를 일상 생활의 강아지 훈련으로 쉽게 설명합니다.

**교수 전략**:
```
"강화학습을 강아지 훈련으로 생각해볼까요?

1. 상황 (State): 강아지가 마당에 있어요
2. 행동 (Action): '앉아!' 명령 → 강아지가 앉음
3. 보상 (Reward): 잘했어! 간식 제공! 🦴
4. 학습 (Learning): '앉아 = 간식'으로 기억

다음번엔? 앉아 명령만 들어도 바로 앉아요!
왜? 간식을 받았던 기억 때문이죠.

AI도 똑같이 배워요!
- 좋은 결과 → 그 방법 기억
- 나쁜 결과 → 다른 방법 시도"
```

**초등학생 설명**:
```
"여러분이 강아지를 훈련시킨다고 생각해보세요.
강아지가 앉으면 간식을 주고,
앉지 않으면 아무것도 안 주죠?

그럼 강아지는 어떻게 할까요?
간식을 받기 위해 계속 앉으려고 하겠죠!

AI도 똑같아요!
좋은 점수를 받으면 그 방법을 기억하고,
나쁜 점수를 받으면 다른 방법을 시도해요."
```

#### **[그림 08] 게임 점수 획득 과정** 🎮

![게임 점수 획득 - 강화학습 과정](AI디지털리터러시_1주차2차시_20_게임점수획득_강화학습과정.png)

게임 AI의 시행착오 학습 과정을 시각화합니다.

**교수 전략**:
```
"게임으로 더 재미있게 이해해볼까요?

게임 목표: 장애물 피하면서 최대한 멀리 가기

시도 1 (실패):
- 장애물 앞 → 점프 → 부딪힘! 💥
- 점수: -10점 ❌
- 학습: "점프는 위험해..."

시도 2 (성공):
- 장애물 앞 → 옆으로 피함 → 성공! ✨
- 점수: +50점 ✅
- 학습: "옆으로 피하는 게 좋구나!"

100만 번 반복하면?
- 1,000번: 50점 😢
- 10,000번: 300점 🙂
- 100,000번: 850점 😊
- 1,000,000번: 2,500점 🏆

→ 최고 점수 달성!

AI는 지치지 않고 24시간 연습할 수 있어요!"
```

**오개념 방지**:

| 오개념 | 올바른 이해 |
|:---|:---|
| "AI가 게임을 재미있다고 느낀다" | AI는 감정이 없고, 점수 최대화만 목표로 함 |
| "한 번에 완벽하게 배운다" | 수만~수백만 번 시행착오 필요 |
| "사람처럼 생각하며 배운다" | 보상 수치만 보고 통계적으로 학습 |

### 강화학습 예시

#### 예시 1: AlphaGo (바둑 AI)

**환경**: 바둑판 19×19

**상태**: 현재 바둑판의 돌 배치

**행동**: 361개 위치 중 하나에 돌 놓기

**보상**:
- 승리 → +1점
- 패배 → -1점
- 무승부 → 0점

**학습 과정**:
1. 처음에는 랜덤으로 돌을 놓음
2. 수백만 판 자가 대국
3. 이긴 경우의 패턴 학습
4. 점점 실력 향상
5. 최종적으로 세계 챔피언 이김 (2016년)

#### 예시 2: 테슬라 자율주행

**환경**: 도로, 차량, 신호등, 보행자 등

**상태**: 카메라로 본 도로 상황

**행동**: 가속, 감속, 좌회전, 우회전, 정지 등

**보상**:
- 안전 주행 → +점수
- 차선 유지 → +점수
- 사고 → 큰 -점수
- 급정거 → -점수

**학습 과정**:
1. 시뮬레이터에서 수백만 km 학습
2. 안전한 주행 패턴 학습
3. 실제 도로 데이터로 계속 개선

#### 예시 3: 게임 AI (슈퍼마리오)

**환경**: 게임 화면

**상태**: 마리오 위치, 적 위치, 장애물 등

**행동**: 좌/우 이동, 점프, 달리기

**보상**:
- 코인 먹음 → +10점
- 적 밟음 → +100점
- 죽음 → -1000점
- 골 도달 → +5000점

**학습 과정**:
1. 처음에는 무작위로 움직임
2. 수천 번 게임 오버
3. 점차 코인 먹는 법, 적 피하는 법 학습
4. 최종적으로 완벽한 플레이

### 지도학습 vs 강화학습 비교

| 구분 | 지도학습 | 강화학습 |
|:---:|:---:|:---:|
| **피드백** | 정답 (라벨) | 보상 (점수) |
| **피드백 시기** | 즉시 (데이터마다) | 지연될 수 있음 (게임 끝날 때) |
| **목표** | 정답 맞추기 | 보상 최대화 |
| **예시** | 고양이/개 구분 | AlphaGo, 게임 AI |
| **데이터** | 미리 준비 | 직접 경험하며 생성 |

### 강화학습의 특징

**장점**:
- 정답 데이터 없이도 학습 가능
- 복잡한 전략 게임에 강함
- 환경과 상호작용하며 학습

**단점**:
- 학습 시간이 매우 오래 걸림
- 많은 시행착오 필요
- 보상 설계가 어려움

**활용 분야**:
- 게임 AI (바둑, 체스, 스타크래프트)
- 로봇 제어 (걷기, 물건 잡기)
- 자율주행 (운전 학습)
- 추천 시스템 (클릭률 최대화)

**[그림 09] 3가지 학습유형 비교**

![3가지 학습유형 비교](AI디지털리터러시_1주차2차시_10_3가지학습유형비교.png)

| 구분 | 지도학습 | 비지도학습 | 강화학습 |
|:---:|:---:|:---:|:---:|
| **데이터 구조** | 입력 + 라벨 | 입력만 | 입력 + 보상 |
| **학습 목표** | 정답 맞추기 | 패턴 찾기 | 보상 최대화 |
| **피드백** | 정답(라벨) | 없음 | 보상(점수) |
| **비유** | 시험 공부<br>(문제+정답) | 친구 그룹 나누기<br>(정답표 없이) | 게임 연습<br>(점수로 배우기) |
| **예시** | 고양이/개 구분<br>스팸 필터<br>질병 진단 | 고객 세분화<br>뉴스 분류<br>추천 시스템 | AlphaGo<br>자율주행<br>게임 AI |
| **장점** | 정확한 예측 | 라벨 불필요 | 복잡한 전략 학습 |
| **단점** | 라벨 작업 필요 | 결과 해석 어려움 | 학습 시간 오래 걸림 |
| **활용 분야** | 분류, 예측 문제 | 데이터 탐색, 군집화 | 게임, 로봇, 자율주행 |

> 💡 **핵심 메시지**: 
> - **지도학습** = 정답으로 배우기 (가장 일반적)
> - **비지도학습** = 스스로 패턴 찾기 (탐색적)
> - **강화학습** = 보상으로 배우기 (상호작용적)
> 
> 3주차 엔트리 AI 실습은 **지도학습**입니다!

# 📙 전개 3: 머신러닝 학습 과정 4단계

기계학습이 실제로 어떻게 이루어지는지, 4단계 과정을 자세히 알아봅시다.

**[그림 10] 머신러닝 프로세스**

![머신러닝 프로세스](머신러닝프로세스.jpg)

## 1단계: 데이터 수집 (Data Collection)

> **좋은 데이터 = 좋은 AI**

### 데이터의 중요성

기계학습에서 데이터는 연료와 같아요. 자동차에 좋은 휘발유를 넣으면 성능이 좋아지듯이, AI에게 좋은 데이터를 주면 성능이 향상됩니다.

**데이터 = AI의 연료**
```
많은 데이터 + 좋은 품질 = 똑똑한 AI
적은 데이터 + 나쁜 품질 = 멍청한 AI
```

### 좋은 데이터의 3가지 조건

#### 1️⃣ 데이터의 양 (Quantity)

**많을수록 좋아요!**

| 데이터 양 | 성능 | 예시 |
|:---|:---|:---|
| 100개 | 낮음 (60%) | 간단한 패턴만 학습 |
| 1,000개 | 보통 (75%) | 기본적인 패턴 학습 |
| 10,000개 | 좋음 (90%) | 복잡한 패턴 학습 |
| 100,000개 | 매우 좋음 (95%) | 미세한 차이까지 학습 |

**예시**:
- 고양이/개 분류: 각각 1,000장 이상
- 음성 인식: 수만 시간의 음성
- 자율주행: 수백만 km 주행 데이터

#### 2️⃣ 데이터의 질 (Quality)

**정확하고 깨끗해야 해요!**

나쁜 데이터 예시:
- ❌ 흐릿한 사진 (화질 나쁨)
- ❌ 잘못된 라벨 (고양이 사진에 "개" 라벨)
- ❌ 중복된 데이터 (똑같은 사진 여러 장)
- ❌ 노이즈 많은 음성 (잡음이 심함)

좋은 데이터 예시:
- ✅ 선명한 사진 (화질 좋음)
- ✅ 정확한 라벨 (고양이 사진에 "고양이" 라벨)
- ✅ 다양한 데이터 (여러 품종의 고양이)
- ✅ 깨끗한 음성 (잡음 없음)

#### 3️⃣ 데이터의 다양성 (Diversity)

**다양한 상황을 포함해야 해요!**

**나쁜 예 - 편향된 데이터**:
```
고양이 사진 1,000장
→ 전부 실내에서 촬영
→ 전부 페르시안 고양이
→ 전부 흰색

문제: 야외 고양이, 검은 고양이는 인식 못함!
```

**좋은 예 - 다양한 데이터**:
```
고양이 사진 1,000장
→ 실내/야외 반반
→ 페르시안, 샴, 코리안숏헤어 등 다양
→ 흰색, 검은색, 갈색, 얼룩 등 다양

결과: 어떤 고양이도 잘 인식!
```

**[그림 11] 편향된 데이터 vs 좋은 데이터**

![편향된 데이터 vs 좋은 데이터](AI디지털리터러시_1주차2차시_13_편향된데이터vs좋은데이터.png)

### 데이터 수집 방법

| 방법 | 설명 | 예시 |
|:---|:---|:---|
| **직접 수집** | 사람이 직접 데이터 만들기 | 사진 촬영, 설문조사 |
| **크롤링** | 웹에서 자동 수집 | 뉴스 기사, 이미지 검색 |
| **센서** | 기계로 자동 수집 | 온도, 습도, GPS 데이터 |
| **공개 데이터셋** | 이미 만들어진 데이터 사용 | ImageNet, MNIST |

### 엔트리 연결

엔트리 AI 모델에서의 데이터 수집:
1. 웹캠으로 직접 사진 촬영
2. 고양이/개 각각 20~30장 권장
3. 다양한 각도, 밝기로 촬영
4. "고양이"/"개" 라벨 자동 부여

## 2단계: 학습 (Training)

> **컴퓨터가 데이터에서 패턴을 발견하는 과정**

### 학습이란?

데이터를 반복해서 보면서 패턴을 찾는 과정이에요.

**비유**: 
- 학생이 문제집을 여러 번 풀며 유형 파악
- 운동선수가 같은 동작을 반복 연습

### 학습 과정

**1단계: 초기 설정**
```
AI: "귀가 뾰족하면 고양이일 확률 50%" (랜덤 추측)
     (아직 배우지 않아서 모름)
```

**2단계: 데이터 보기**
```
AI가 고양이 사진을 보고 예측:
"귀가 뾰족하니까... 고양이!"
정답 확인: "맞아, 고양이야!"
→ "귀 뾰족 → 고양이" 확률을 60%로 조정
```

**3단계: 반복 학습**
```
같은 과정을 수천 번 반복:
- 고양이 사진 1,000장
- 개 사진 1,000장

점점 정확도 향상:
처음: 50% → 60% → 70% → 80% → 90% → 95%
```

**4단계: 패턴 발견**
```
AI가 스스로 발견한 패턴:
- 귀가 뾰족하면 → 고양이 확률 높음
- 눈이 크고 동그라면 → 고양이 확률 높음
- 코가 크면 → 개 확률 높음
- 주둥이가 길면 → 개 확률 높음
```

### 핵심 개념

#### 에포크 (Epoch)

**정의**: 전체 데이터를 1회 학습하는 단위

**비유**: 교과서를 한 번 읽는 것 = 1 에포크

**예시**:
```
데이터: 고양이 1,000장 + 개 1,000장 = 총 2,000장

1 에포크: 2,000장을 모두 1회씩 학습
10 에포크: 2,000장을 10회 반복 학습
50 에포크: 2,000장을 50회 반복 학습

→ 에포크가 많을수록 학습 시간 ↑, 성능 ↑
```

**[그림 12] 학습 곡선**

![학습 곡선](AI디지털리터러시_1주차2차시_12_학습곡선.png)

#### 가중치 (Weight)

**정의**: 각 특징의 중요도를 나타내는 수치

**비유**: 
- 시험에서 각 과목의 배점 비율
- 요리할 때 재료의 양

**예시**:
```
고양이 판별 시 중요도:
- 귀 모양: 가중치 0.8 (매우 중요)
- 눈 모양: 가중치 0.6 (중요)
- 털 색깔: 가중치 0.2 (덜 중요)

→ 귀 모양이 가장 중요한 특징!
```

학습 과정에서 AI가 가중치를 자동으로 조정해요.

### 엔트리 연결

엔트리 AI 모델 학습:
1. "모델 학습하기" 버튼 클릭
2. 진행 상황 표시 (에포크 진행)
3. 학습 중 정확도 상승 그래프
4. 학습 완료 후 최종 정확도 표시

## 3단계: 평가 (Testing)

> **학습한 모델의 성능을 측정하는 과정**

### 평가란?

AI가 잘 배웠는지 시험 보는 단계예요.

**비유**:
- 학생이 모의고사 보기
- 운동선수가 연습 경기 하기

**중요**: 학습할 때 본 데이터가 아닌, **새로운 데이터**로 테스트해야 해요!

### 왜 새 데이터로 테스트해야 할까?

**나쁜 예 - 학습 데이터로 테스트**:
```
학습: 고양이 사진 A, B, C로 학습
평가: 똑같은 사진 A, B, C로 테스트
→ 100% 정확!

문제: 외워버린 것! (진짜 실력 아님)
```

**좋은 예 - 새 데이터로 테스트**:
```
학습: 고양이 사진 A, B, C로 학습
평가: 처음 보는 사진 D, E, F로 테스트
→ 90% 정확

결과: 진짜 실력 측정!
```

### 데이터 분할

**[그림 13] 데이터 분할 (Train/Validation/Test)**

![데이터 분할](AI디지털리터러시_1주차2차시_11_데이터분할.png)

전체 데이터를 세 부분으로 나눠요:

```
전체 데이터 1,000장
├─ 학습용 (Train): 700장 (70%)
│  → 모델 학습에 사용
├─ 검증용 (Validation): 150장 (15%)
│  → 학습 중간에 성능 확인
└─ 평가용 (Test): 150장 (15%)
   → 최종 성능 평가
```

**각 데이터의 역할**:

| 구분 | 비율 | 역할 | 비유 |
|:---|:---:|:---|:---|
| **학습용** | 70% | 모델이 배우는 데이터 | 문제집 |
| **검증용** | 15% | 학습 중 성능 확인 | 모의고사 |
| **평가용** | 15% | 최종 성능 측정 | 수능 시험 |

**왜 나눌까요?**

1. **과적합 (Overfitting) 방지**
   - 문제: 학습 데이터만 너무 잘 맞춤
   - 증상: 학습 데이터 100%, 새 데이터 60%
   - 비유: 시험 문제만 달달 외움 (응용력 없음)
   
2. **일반화 성능 평가**
   - 목적: 처음 보는 데이터도 잘 예측하는지 확인
   - 방법: 검증/평가 데이터로 테스트

### 정확도 측정

**정의**: 전체 중에서 맞춘 비율

```
정확도 = (맞춘 개수 ÷ 전체 개수) × 100%
```

**예시**:
```
평가용 데이터 200장으로 테스트:
- 고양이 100장 → 95장 정답 (95%)
- 개 100장 → 90장 정답 (90%)

전체 정확도 = (95+90) ÷ 200 = 92.5%
```

### 성능이 낮을 때 대응

정확도가 낮으면 다시 개선해야 해요.

**개선 방법**:
1. 데이터 더 수집
2. 모델 구조 변경
3. 학습 시간 늘리기 (에포크 증가)
4. 다시 학습 (2단계로 돌아가기)

### 엔트리 연결

엔트리 AI 모델:
1. 학습 완료 후 정확도 표시 (예: 92%)
2. 웹캠으로 실시간 테스트
3. 잘못 분류하면 데이터 추가 후 재학습

## 4단계: 예측 (Prediction)

> **학습한 모델을 실전에 적용**

### 예측이란?

드디어 완성된 AI 모델을 실제로 사용하는 단계예요!

### 예측 과정

**1단계: 새로운 입력**
```
처음 보는 고양이 사진 입력
```

**2단계: 모델 적용**
```
AI가 학습한 패턴으로 분석:
- 귀가 뾰족함 → 고양이 가능성 ↑
- 눈이 크고 둥글음 → 고양이 가능성 ↑
- 코가 작음 → 고양이 가능성 ↑
```

**3단계: 결과 출력**
```
"고양이입니다" (확률 95%)
```

### 실전 배포

완성된 모델을 다양한 곳에 활용할 수 있어요.

| 분야 | 활용 예시 | 설명 |
|:---|:---|:---|
| 📱 **앱** | 구글 포토 | 사진을 자동으로 "사람", "고양이", "풍경" 분류 |
| 🌐 **웹사이트** | Gmail 스팸 필터 | 새 메일이 오면 자동으로 스팸 판별 |
| 🤖 **로봇** | 로봇 청소기 | 장애물을 인식하고 피해서 청소 |
| 🚗 **자동차** | 테슬라 자율주행 | 도로의 차선, 신호등, 차량 실시간 인식 |
| 🏥 **병원** | AI 영상 진단 | X-ray 사진에서 폐렴 자동 진단 |
| 🏪 **편의점** | 무인 계산대 | 상품을 자동으로 인식하고 계산 |

### 지속적인 모니터링

배포 후에도 계속 관리가 필요해요.

**1. 성능 모니터링**
```
정확도가 떨어지지 않는지 확인
예: 처음 95% → 시간 지나니 85%
원인: 환경 변화, 새로운 패턴 등장
```

**2. 새 데이터 수집**
```
환경이 변하면 새 데이터 추가
예: 새로운 품종의 고양이 등장
→ 해당 데이터 수집하여 재학습
```

**3. 재학습**
```
주기적으로 모델 업데이트
예: Gmail 스팸 필터
- 매일 새로운 스팸 기법 등장
- 지속적으로 데이터 수집 및 재학습
- 모델 계속 개선
```

### 엔트리 연결

엔트리 AI 모델 예측:
1. 웹캠으로 새 이미지 입력
2. 실시간으로 분류 결과 표시
3. 블록 코딩과 연동하여 작품 제작:
   ```
   만약 (AI 모델 결과) = "고양이"라면
      → "야옹" 소리 재생
   만약 (AI 모델 결과) = "개"라면
      → "멍멍" 소리 재생
   ```

# 📝 정리

## 핵심 개념 요약

오늘 배운 내용을 4가지로 정리해 봅시다.

### 1️⃣ AI ⊃ ML ⊃ DL 포함 관계

```
AI (인공지능) - 가장 큰 개념
 ⊃ ML (기계학습) - 데이터로 학습하는 AI
    ⊃ DL (딥러닝) - 신경망이 깊은 ML
```

**핵심**:
- AI는 인간의 지능을 모방하는 모든 기술
- 기계학습은 AI를 구현하는 대표적 방법
- 딥러닝은 기계학습의 한 종류

### 2️⃣ 기계학습의 정의

> **기계학습**: 데이터로부터 패턴을 스스로 학습하는 AI 기술

**전통적 프로그래밍 vs 기계학습**:
- 전통적: 규칙 작성 → 결과
- 기계학습: 데이터 + 결과 → 규칙 발견

**장점**:
- 사람이 규칙을 작성하지 않아도 됨
- 복잡한 패턴 학습 가능
- 새로운 환경에 적응

### 3️⃣ 3가지 학습 유형

| 유형 | 특징 | 비유 | 대표 예시 |
|:---|:---|:---|:---|
| **지도학습** | 정답 있는 데이터로 학습 | 시험 공부 | 스팸 필터, 질병 진단 |
| **비지도학습** | 정답 없이 패턴 발견 | 그룹 나누기 | 고객 세분화, 뉴스 분류 |
| **강화학습** | 시행착오로 보상 최대화 | 게임 연습 | AlphaGo, 자율주행 |

**강조 포인트**:
- **지도학습**: 라벨(정답) 있음 ✅
- **비지도학습**: 라벨(정답) 없음 ❌ → AI가 스스로 그룹 발견
- **강화학습**: 보상(점수)으로 학습 🎯

**💡 초등 AI 교육에서는 주로 지도학습을 다룹니다!**

### 4️⃣ 머신러닝 4단계 과정

```
1. 데이터 수집 → 2. 학습 → 3. 평가 → 4. 예측
```

**각 단계**:
- **1단계**: 좋은 데이터 모으기 (양, 질, 다양성)
- **2단계**: 패턴 발견 (반복 학습, 가중치 조정)
- **3단계**: 성능 확인 (새 데이터로 테스트)
- **4단계**: 실전 적용 (새로운 입력 예측)
