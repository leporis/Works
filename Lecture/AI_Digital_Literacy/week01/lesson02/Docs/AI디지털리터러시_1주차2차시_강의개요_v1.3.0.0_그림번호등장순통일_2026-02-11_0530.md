### 4.3 전개 2: 기계학습의 3가지 학습 유형 (15분)

| 순서 | 학습 유형 | 핵심 특징 | 일상 예시 |
|:---:|:---|:---|:---|
| 1 | **지도학습 (Supervised Learning)** | 정답이 있는 데이터로 학습 | 시험 공부 (문제+정답) |
| 2 | 지도학습 세부: 분류 vs 예측 | 분류(범주), 예측(수치) | 고양이/개 구분, 집값 예측 |
| 3 | **비지도학습 (Unsupervised Learning)** | 정답 없이 패턴 발견 | 친구들을 비슷한 그룹으로 나누기 |
| 4 | 비지도학습 세부: 군집화 | 유사한 데이터끼리 그룹화 | 고객 세분화, 뉴스 주제 분류 |
| 5 | **강화학습 (Reinforcement Learning)** | 시행착오로 보상 최대화 | 게임 연습 (잘하면 점수, 못하면 게임오버) |
| 6 | 강화학습 예시 | AlphaGo, 자율주행, 게임 AI | 바둑, 운전, 게임 플레이 |

**사용 그림**: 그림 02 지도학습vs비지도학습_데이터구조비교, 그림 03 라벨없는_3가지이유, 그림 04 군집화_3단계과정, 그림 05 고객세분화결과_4개그룹, 그림 06 강화학습루프, 그림 07 강아지훈련_강화학습비유, 그림 08 게임점수획득_강화학습과정, 그림 09 3가지학습유형비교

**교수 전략**:

**1) 지도학습 설명 (7분)**
- **비유**: 선생님이 문제와 정답을 주고 공부하는 것
- **분류 (Classification)**:
  - 입력: 이미지
  - 출력: "고양이" 또는 "개" (범주)
  - 예시: 이메일 스팸 판별, 질병 진단
- **예측 (Regression)**:
  - 입력: 집의 평수, 위치, 연식
  - 출력: 집값 (숫자)
  - 예시: 주식 가격 예측, 기온 예측
- **엔트리 AI 모델 연결**: 3주차에 배울 엔트리의 이미지 분류 모델이 바로 지도학습!

**2) 비지도학습 설명 (4분)**

**"정답이 없다"의 정확한 의미**:
- **핵심**: "정답이 없다" = "라벨(정답 표시)이 없다"
- **지도학습과 비교**:
  - 지도학습: 데이터 + 라벨(정답) → 예: 고양이 사진 + "고양이" 스티커 ✅
  - 비지도학습: 데이터만 (라벨 없음) → 예: 고객 데이터 (어떤 그룹인지 스티커 없음) ❌

**왜 비지도학습을 사용하는가?** (3가지 이유)
1. **라벨 자체가 없을 때**
   - 예: 신규 고객 10,000명의 취향을 아직 모름
   - 라벨을 붙일 근거가 없음
   
2. **라벨 만들기 어려울 때**
   - 예: 매일 수백만 개 뉴스 기사에 일일이 라벨 붙이기 불가능
   - 시간/비용이 너무 많이 듦
   
3. **숨겨진 패턴을 발견하고 싶을 때**
   - 예: 사람도 몰랐던 새로운 고객 그룹 찾기
   - AI가 데이터에서 새로운 구조 발견

**군집화 (Clustering)**: 비지도학습의 대표 방법
- **입력**: 고객 데이터 (나이, 구매 내역, 거주지 등)
- **과정**: 유사한 데이터끼리 자동 그룹화 (라벨 없이 패턴만으로)
- **출력**: 그룹 A, B, C... (AI가 스스로 발견한 그룹)

**구체적 예시**:
- **예시 1 - 쇼핑몰 고객 세분화**:
  - 상황: 신규 고객 10,000명, 취향 라벨 없음
  - ❌ 지도학습 불가능: "이 고객은 알뜰형"이라는 라벨이 없음
  - ✅ 비지도학습: 구매 패턴 분석 → 자동으로 3~5개 그룹 발견
  - 결과: "알뜰형", "명품 선호형", "트렌드 민감형" 등 자동 분류

- **예시 2 - 넷플릭스 추천**:
  - 상황: "좋아요" 데이터만으로는 취향 파악 어려움
  - 비지도학습: 시청 패턴이 비슷한 사용자끼리 그룹화
  - 결과: "이 영화를 본 사람은 저 영화도 봤어요" (추천 시스템)

- **예시 3 - 뉴스 자동 분류**:
  - 상황: 매일 수백만 개 뉴스, 라벨 붙이기 불가능
  - 비지도학습: 단어 패턴으로 유사 기사끼리 자동 분류
  - 결과: 정치, 경제, 스포츠 등 주제별 자동 그룹화

**초등교육 비유**: 
"친구들을 성격별로 나누는데, 선생님이 만든 정답표가 없을 때.
AI가 스스로 비슷한 친구들끼리 묶어서 그룹을 만들어요!"

**3) 강화학습 설명 (4분)**
- **비유**: 게임하며 점수로 배우기
- **작동 원리**:
  1. 행동(Action): AI가 선택
  2. 보상(Reward): 좋으면 +, 나쁘면 -
  3. 반복 학습: 보상 최대화 방향으로 학습
- **예시**:
  - AlphaGo: 바둑에서 이기면 +, 지면 -
  - 자율주행: 안전 운전 +, 사고 -
  - 게임 AI: 점수 획득 +, 게임 오버 -

**비교표 활용** (판서 또는 슬라이드):

| 구분 | 지도학습 | 비지도학습 | 강화학습 |
|:---:|:---:|:---:|:---:|
| **라벨(정답)** | ⭕ 있음 | ❌ 없음 | 🎯 보상 |
| **데이터 구조** | 입력 + 라벨 | 입력만 | 입력 + 보상 |
| **학습 목표** | 정답 맞추기 | 패턴 찾기 | 보상 최대화 |
| **비유** | 시험 공부<br>(문제+정답) | 친구 그룹 나누기<br>(정답표 없이) | 게임 연습<br>(점수로 배우기) |
| **예시** | 고양이/개 구분<br>스팸 필터 | 고객 세분화<br>뉴스 분류 | AlphaGo<br>자율주행 |

**강조 포인트**:
- 지도학습: 모든 데이터에 "정답 스티커"가 붙어 있음
- 비지도학습: "정답 스티커"가 없어도 AI가 스스로 그룹 발견
- 강화학습: 정답 대신 "점수(보상)"로 좋고 나쁨 판단

### 4.4 전개 3: 머신러닝 학습 과정 (10분)

| 순서 | 단계 | 설명 | 엔트리 연결 |
|:---:|:---|:---|:---|
| 1 | **데이터 수집** | 학습할 데이터 모으기 | 엔트리: 고양이/개 사진 촬영 |
| 2 | **학습 (Training)** | 패턴 찾기 | 엔트리: "모델 학습하기" 버튼 |
| 3 | **평가 (Testing)** | 정확도 확인 | 엔트리: 정확도 % 확인 |
| 4 | **예측 (Prediction)** | 새 데이터에 적용 | 엔트리: 새 사진 분류 |

**사용 그림**: 그림 10 머신러닝프로세스, 그림 11 편향된데이터vs좋은데이터, 그림 12 학습곡선, 그림 13 데이터분할

**상세 설명**:

**1단계: 데이터 수집**
- 좋은 데이터 = 좋은 AI
- 데이터 양: 많을수록 좋음 (수백~수천 개)
- 데이터 질: 정확하고 다양해야 함
- **주의**: 편향된 데이터 → 편향된 AI

**2단계: 학습**
- 컴퓨터가 데이터에서 패턴 발견
- 반복 학습 (Epoch)
- 가중치(Weight) 조정

**3단계: 평가**
- 새로운 데이터로 테스트
- 정확도 측정 (예: 90% 정확)
- 성능 개선 필요 시 재학습

**4단계: 예측**
- 실전 적용
- 새로운 입력 → 결과 출력
- 지속적 모니터링

**엔트리 AI 모델 시연** (가능하면):
- 엔트리에서 이미지 분류 모델 학습 시연
- 4단계 과정이 실제로 어떻게 진행되는지 보여주기

### 4.5 정리 (5분)

**핵심 개념 4가지 요약**:

1. **AI ⊃ ML ⊃ DL**: AI > 기계학습 > 딥러닝 포함 관계
2. **기계학습**: 데이터로부터 패턴을 학습하는 AI 기술
3. **3가지 학습 유형**:
   - 지도학습: 정답 있는 데이터로 학습 (분류/예측)
   - 비지도학습: 정답 없이 패턴 발견 (군집화)
   - 강화학습: 시행착오로 보상 최대화 (게임 AI)
4. **머신러닝 과정**: 데이터 수집 → 학습 → 평가 → 예측

**엔트리 AI 연결**:
- 3주차에 배울 **엔트리 AI 모델 학습**은 바로 **지도학습** 원리!
- 이미지 분류: 고양이/개 사진으로 학습 → 새 사진 판별

**다음 주 퀴즈 안내**:
- 2주차 1차시 시작 시 **1주차 전체 내용** 퀴즈 (10분)
- 5지선다형 10문항 + 단답형 5문항 = 10점
- 출제 범위: 1주차 1차시 (AI 이해) + 1주차 2차시 (기계학습 원리)
- 복습 권장!

**다음 차시 예고**: 2주차 1차시 - 데이터와 딥러닝

## 6. 교수 전략 및 유의사항

### 6.1 학생 수준 고려

**예비교사 대상 특성**:
- 기술적 세부사항보다 **교육적 의미** 강조
- 초등학생에게 어떻게 가르칠지 관점에서 접근
- 엔트리 AI 모델 실습과 연결

### 6.2 개념 설명 전략

**추상적 개념을 구체화**:
- 지도학습 = 시험 공부 (문제+정답)
- 비지도학습 = 친구 그룹 나누기
- 강화학습 = 게임 연습 (점수로 배우기)

**일상 예시 활용**:
- 유튜브 추천 (지도학습)
- 넷플릭스 사용자 그룹화 (비지도학습)
- AlphaGo (강화학습)

### 6.3 엔트리 연결 강조

**3~4주차 실습 예고**:
- "지금 배우는 이론이 바로 엔트리 AI 모델의 원리!"
- 이미지 분류 모델 = 지도학습
- 데이터 수집 → 학습 → 평가 → 예측 과정 체험

### 6.4 학생 오개념 대응

| 오개념 | 올바른 이해 |
|:---|:---|
| "AI는 스스로 생각한다" | AI는 데이터 패턴을 학습할 뿐, 사고력 없음 |
| "학습=사람처럼 이해" | 통계적 패턴 발견, 의미 이해 아님 |
| "데이터 많으면 무조건 좋음" | 편향된 데이터 → 편향된 AI |

## 7. 주요 용어 해설

기계학습 수업에서 사용되는 핵심 전문용어를 학생들이 이해할 수 있도록 쉽게 설명합니다.

| 용어 (한글) | 영문 | 간단 정의 | 비유/예시 |
|:---|:---|:---|:---|
| **에포크** | Epoch | 전체 데이터를 1회 학습하는 단위 | 교과서를 한 번 읽는 것 = 1에포크, 10번 반복 = 10에포크 |
| **가중치** | Weight | 각 특징의 중요도를 나타내는 수치 | 귀 모양(중요) = 높은 가중치, 털 색깔(덜 중요) = 낮은 가중치 |
| **정확도** | Accuracy | 전체 중 맞춘 비율 | 맞춘 개수 ÷ 전체 개수 × 100% (예: 90/100 = 90%) |
| **학습** | Training | 모델이 데이터에서 패턴을 배우는 과정 | 학생이 문제집 풀며 공부하는 과정 |
| **평가** | Testing | 학습한 모델의 성능을 측정하는 과정 | 시험 보기 - 처음 보는 문제로 실력 확인 |
| **검증** | Validation | 학습 중간에 성능을 확인하는 과정 | 모의고사 - 최종 시험 전 중간 점검 |
| **라벨** | Label | 데이터의 정답 표시 | "고양이" 사진에 "고양이" 스티커 붙이기 |
| **특징** | Feature | 데이터의 속성이나 특성 | 고양이 특징: 귀 모양, 수염, 눈 모양, 털 길이 등 |

**교수 전략:**
- 용어 첫 등장 시 한글 + 영문 병기 후 간단히 설명
- 예시: "에포크(Epoch)는 전체 데이터를 1회 학습하는 단위입니다. 교과서를 한 번 읽는 것과 같아요."
- 학생들이 영문 용어보다 한글 설명에 집중하도록 유도
