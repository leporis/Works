## 1. 교육 활용 개요

### 1.1 교육적 가치

KNN 분류 시뮬레이터는 다음 세 가지 측면에서 교육적 가치를 제공합니다.

**① 가장 직관적인 ML 알고리즘 체험:**
KNN은 "가장 가까운 이웃을 보고 따라 하기"라는 일상적 판단 방식과 동일하여, 머신러닝을 처음 접하는 학습자도 원리를 즉시 이해할 수 있습니다. 수학 공식 없이 "거리 계산 → 이웃 찾기 → 다수결 투표"의 3단계로 설명할 수 있습니다.

**② K값에 따른 과적합/일반화 개념:**
K=1일 때의 복잡한 결정 경계(과적합)와 K=15일 때의 부드러운 경계(일반화)를 시각적으로 비교하면서, 모델 복잡도와 일반화 성능의 관계를 체험할 수 있습니다.

**③ 분류 성능 평가 체험:**
혼동행렬, 정확도, K값별 성능 비교 등 모델 평가의 기본 개념을 실습할 수 있습니다.

### 1.2 대상 학습자

| 대상 | 적합도 | 활용 방법 |
|------|:------:|----------|
| 초등 5~6학년 | ⭐⭐ | 과일/동물 프리셋으로 "이웃 찾기" 개념 체험 |
| 예비교사 | ⭐⭐⭐ | 알고리즘 원리 이해 + 수업 시나리오 설계 |
| 현직교사 연수 | ⭐⭐⭐ | KNN 개념 + 교실 활용 방법 습득 |
| 통계·데이터 교육 | ⭐⭐⭐ | 분류 알고리즘, 교차검증, 혼동행렬 심화 |

## 3. 수업 시나리오

### 시나리오 A: KNN 분류 입문 (15분)

**목표:** KNN 알고리즘의 기본 원리를 이해하고, 데이터를 분류할 수 있다.

| 시간 | 활동 | 교사 발문 |
|:----:|------|----------|
| 0~3분 | 🍎 과일 프리셋 선택 → 데이터 60개 생성 | "차트에 있는 점들은 각각 무엇을 나타낼까요?" |
| 3~5분 | K=5로 분류 실행 → 결정 경계 관찰 | "색깔 영역은 무엇을 의미할까요?" |
| 5~8분 | K=1로 변경 → 재실행 → 경계 비교 | "K=1과 K=5일 때 경계가 어떻게 다른가요?" |
| 8~11분 | 추론 모드 → 슬라이더로 과일 분류 체험 | "당도 70, 무게 40인 과일은 어떤 과일일까요?" |
| 11~15분 | K개 이웃 목록과 투표 결과 확인 | "왜 이 과일이 '귤'로 분류되었는지 설명해 볼까요?" |

**학생 도달 기대:**
- "가장 가까운 이웃을 보고 다수결로 정한다"는 KNN 원리를 자신의 말로 설명할 수 있음
- K값을 바꾸면 분류 결과가 달라진다는 것을 이해함

---

### 시나리오 B: K값 실험 수업 (25분)

**목표:** K값 변화가 분류 정확도와 결정 경계에 미치는 영향을 실험으로 확인한다.

| 시간 | 활동 | 교사 지시 |
|:----:|------|----------|
| 0~5분 | 🍎 과일, 데이터 80개, 클래스 3개 생성 | "모두 같은 설정으로 데이터를 만들어 주세요" |
| 5~10분 | K=1로 분류 → 정확도·경계 기록 | "활동지에 정확도와 경계 모양을 기록하세요" |
| 10~13분 | K=3으로 변경 → 재실행 → 기록 | "K=1과 비교하여 무엇이 달라졌나요?" |
| 13~16분 | K=7로 변경 → 재실행 → 기록 | "경계가 점점 어떻게 변하나요?" |
| 16~19분 | K=15로 변경 → 재실행 → 기록 | "K가 너무 크면 어떤 문제가 생길까요?" |
| 19~22분 | K값별 정확도 차트 분석 | "어떤 K값이 가장 좋은 정확도를 보이나요?" |
| 22~25분 | 전체 토의: 최적 K값 논의 | "너무 작지도 너무 크지도 않은 K가 왜 좋을까요?" |

**핵심 발견:**
- K=1: 경계 복잡, 노이즈 민감 (과적합)
- K=중간: 적절한 경계 (최적)
- K=15: 경계 과도하게 단순 (과소적합)

---

### 시나리오 C: 거리 방식 비교 탐구 (20분)

**목표:** 유클리드 거리와 맨해튼 거리의 차이를 이해하고, 거리 정의에 따라 분류 결과가 달라짐을 체험한다.

| 시간 | 활동 | 교사 발문 |
|:----:|------|----------|
| 0~3분 | 🎲 랜덤 프리셋, 데이터 60개, 클래스 2개 생성 | "2개 클래스면 결과가 더 명확합니다" |
| 3~7분 | 유클리드 거리 + K=5 → 분류 실행 → 정확도·경계 기록 | "이 결정 경계의 모양을 관찰하세요" |
| 7~11분 | 맨해튼 거리로 변경 → 재실행 → 비교 | "경계 모양이 어떻게 달라졌나요?" |
| 11~15분 | 추론 모드에서 같은 좌표 → 두 거리 방식 비교 | "같은 데이터인데 이웃 순서가 다른 이유는?" |
| 15~20분 | 전체 토의: "거리"의 의미 | "일상에서 거리를 재는 다른 방법이 있을까요?" |

**핵심 발견:**
- 유클리드: 직선(대각선) 거리 → 원형 이웃 영역
- 맨해튼: 격자 거리 → 마름모형 이웃 영역
- 거리 정의에 따라 "가장 가까운 이웃"이 달라짐
