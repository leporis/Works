## 🚀 빠른 시작

### 학습 모드 흐름

```
① 프리셋 선택 → ② 데이터 설정 → ③ 데이터 생성 → ④ K값·거리 설정 → ⑤ 분류 실행
```

| 단계 | 어디서 | 무엇을 |
|:---:|--------|--------|
| ① | 좌측 패널 프리셋 탭 | 🍎 과일 / 🌸 꽃 / 🐧 동물 / 🎲 랜덤 중 선택 |
| ② | 좌측 패널 슬라이더 | 데이터 개수(20~150), 클래스 수(2~4), 분산도(1~5) 조절 |
| ③ | 좌측 **✨ 데이터 생성** 버튼 | 클릭 → 산점도에 데이터 표시 |
| ④ | 우측 패널 "⚙️ KNN 설정" | K값(1~15, 기본 5), 거리 방식(유클리드/맨해튼) 선택 |
| ⑤ | 우측 **⚡ 분류 실행** 버튼 | 결정 경계 + 정확도 + 혼동행렬 + K값별 정확도 차트 표시 |

### 추론 모드 흐름

```
① 추론 모드 전환 → ② 좌표 입력 → ③ 결과 확인
```

| 단계 | 어디서 | 무엇을 |
|:---:|--------|--------|
| ① | 상단 **🔮 추론 모드** 탭 | 클릭 → 추론 화면 전환 |
| ② | 좌측 슬라이더 X₁, X₂ | 분류할 데이터의 좌표 입력 (0~100) |
| ③ | 우측 결과 패널 + 좌측 이웃 목록 | 예측 클래스, 신뢰도, K개 이웃 거리, 투표 결과 확인 |

## 📂 데이터 입력

### 방법 1: 프리셋 데이터 (가장 간편)

좌측 패널의 프리셋 버튼으로 빠르게 시작할 수 있습니다.

| 프리셋 | 축 레이블 | 클래스 예시 |
|--------|----------|------------|
| 🍎 과일 | 당도(Brix) × 무게(g) | 사과, 배, 귤, 포도 |
| 🌸 꽃 | 꽃잎 길이(cm) × 꽃잎 너비(cm) | 장미, 튤립, 백합, 국화 |
| 🐧 동물 | 몸길이(cm) × 체중(kg) | 고양이, 강아지, 토끼, 햄스터 |
| 🎲 랜덤 | 특성 1(X₁) × 특성 2(X₂) | 클래스 A, B, C, D |

**데이터 생성 설정:**
- **데이터 개수**: 20~150개 (기본 60)
- **클래스 수**: 2~4개 (기본 3)
- **분산도**: 매우 밀집 ~ 매우 분산 (5단계)

### 방법 2: CSV 업로드

1. 좌측 패널에서 **CSV** 탭 클릭
2. CSV 파일 드래그&드롭 또는 클릭하여 업로드
3. X₁, X₂ 변수 및 클래스 열 선택
4. **📥 데이터 적용** 클릭

### 방법 3: 내장 데이터셋

좌측 패널에서 **내장** 탭을 클릭하면 실제 ML 연구에 사용되는 데이터셋을 불러올 수 있습니다.

| 데이터셋 | 설명 | 클래스 수 |
|---------|------|:--------:|
| 🌺 아이리스 | 꽃잎/꽃받침 크기로 품종 분류 | 3 |
| 🐧 펭귄 | 부리/지느러미 크기로 종 분류 | 3 |
| 🍷 와인 | 화학 성분으로 품종 분류 | 3 |
| 🌾 씨앗 | 형태 측정으로 밀 품종 분류 | 3 |

내장 데이터셋은 X₁, X₂로 사용할 변수를 직접 선택할 수 있습니다.

## ⚙️ KNN 설정 가이드

### K값 (이웃 수)

| K값 | 특성 | 추천 상황 |
|:---:|------|----------|
| 1~3 | 민감한 분류, 노이즈 영향 큼 | 데이터가 깨끗하고 밀집된 경우 |
| 5~7 | **균형 잡힌 분류** (기본값 5) | 대부분의 경우 추천 |
| 9~15 | 안정적 분류, 경계 부드러움 | 노이즈가 많은 데이터 |

> 💡 **팁**: 짝수 K값은 동점(tie)이 발생할 수 있으므로 **홀수**를 권장합니다.

### 거리 측정 방식

| 방식 | 수식 | 특성 |
|------|------|------|
| 유클리드 거리 | √((x₁-x₂)² + (y₁-y₂)²) | 직선 거리, 가장 일반적 |
| 맨해튼 거리 | \|x₁-x₂\| + \|y₁-y₂\| | 격자형 거리, 이상값에 덜 민감 |

### 결과 패널 읽기

분류 실행 후 차트 아래에 3가지 결과가 표시됩니다.

| 결과 | 의미 |
|------|------|
| 📊 분류 결과 | 현재 K값의 정확도와 상태 (🎯우수 >90%, ✅양호 >70%, ⚠️개선필요) |
| 📋 혼동행렬 | 실제 클래스 vs 예측 클래스 교차표 (대각선 = 정분류) |
| 📈 K값별 정확도 | K=1~15까지 홀수 K값의 정확도 비교 차트 |

## 🆘 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| 차트가 빈 화면 | 데이터 미생성 | **✨ 데이터 생성** 클릭 |
| 결정 경계가 안 보임 | 분류 미실행 | **⚡ 분류 실행** 클릭 |
| 정확도가 매우 낮음 (<50%) | K값 부적절 또는 분산도 과다 | K값별 정확도 차트에서 최적 K값 확인 |
| 추론 모드에서 "--" 표시 | 학습 모드에서 데이터 미생성 | 학습 모드로 돌아가 데이터 생성 후 재전환 |
| CSV 업로드 실패 | 숫자형 컬럼 부족 | 최소 2개 이상의 숫자형 컬럼 필요 |
| 내장 데이터 로드 후 분포가 이상 | X₁, X₂ 변수 선택 부적절 | 변수 조합을 바꿔서 시도 |
| 분류 실행이 느림 | 데이터 150개 + K값 분석 연산 | 데이터 수를 줄여서 시도 |
| 이웃 연결선이 안 보임 | 학습 모드 차트임 | 추론 모드에서만 이웃 연결선 표시 |

## 🔑 핵심 용어

| 용어 | 의미 |
|------|------|
| **KNN** | K-Nearest Neighbors. 가장 가까운 K개의 이웃으로 분류하는 알고리즘 |
| **K값** | 참조할 이웃의 수. 값이 클수록 안정적이나 경계가 부드러워짐 |
| **유클리드 거리** | 두 점 사이의 직선 거리 (√((Δx)² + (Δy)²)) |
| **맨해튼 거리** | 두 점 사이의 격자형 거리 (\|Δx\| + \|Δy\|) |
| **다수결 투표** | K개 이웃의 클래스 중 가장 많은 클래스로 분류 |
| **결정 경계** | 분류 결과가 바뀌는 경계선 (색깔 영역 구분선) |
| **혼동행렬** | 실제 클래스와 예측 클래스의 교차 빈도표 |
| **정확도** | 전체 데이터 중 올바르게 분류한 비율 (%) |
| **LOOCV** | Leave-One-Out 교차검증. 하나씩 빼서 나머지로 분류 정확도 측정 |
| **신뢰도** | K개 이웃 중 예측 클래스에 투표한 비율 (%) |
