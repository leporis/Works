# ML 시뮬레이터 문서 삽입 그림 작성 계획

**문서 버전: v1.0.0.0**

---

## 1. 현황 분석

### 1.1 대상 문서

7개 시뮬레이터 × 3종 문서(퀵가이드·사용설명서·활용가이드) = 21개 문서 완성 상태.

현재 문서 내 도해(다이어그램)는 모두 **ASCII 유니코드 박스문자**로 표현되어 있으며, SVG/PNG 형태의 정식 그림은 **0건**이다 (썸네일 7장 제외).

### 1.2 기존 ASCII 다이어그램 현황

| # | 시뮬레이터 | 퀵가이드 | 사용설명서 | 활용가이드 | 소계 |
|:-:|-----------|:-------:|:---------:|:---------:|:----:|
| 1 | AI사과농장이야기 애니메이션 | 1 | 5 | 1 | **7** |
| 2 | 범용 선형회귀 시뮬레이터 | 1 | 6 | 3 | **10** |
| 3 | KNN 분류 시뮬레이터 | 4 | 0 | 0 | **4** |
| 4 | 분류 의사결정트리 시뮬레이터 | 4 | 0 | 0 | **4** |
| 5 | K-means 클러스터링 시뮬레이터 | 5 | 6 | 0 | **11** |
| 6 | 스네이크 강화학습 시뮬레이터 | 2 | 3 | 0 | **5** |
| 7 | RL 틱택토 시뮬레이터 | 3 | 4 | 0 | **7** |
| | **합계** | **20** | **24** | **4** | **48** |

### 1.3 기존 시각 자산

| 자산 유형 | 수량 | 비고 |
|----------|:----:|------|
| 썸네일 (1400×600 PNG) | 7 | 시뮬레이터별 1장, 완성 |
| SVG 다이어그램 | 0 | 미작성 |
| 스크린샷 (PNG) | 0 | 미촬영 |

## 2. 그림 유형 분류

### 2.1 유형 정의

| 유형 | 코드 | 설명 | 제작 방법 | viewBox |
|------|:----:|------|----------|---------|
| **UI 레이아웃** | `UI` | 시뮬레이터 화면 구조도 | SVG 직접 제작 | 800×400~500 |
| **사용 흐름도** | `WF` | 단계별 사용 워크플로우 | SVG 직접 제작 | 800×300~400 |
| **알고리즘 개념도** | `AL` | ML 알고리즘 원리 시각화 | SVG 직접 제작 | 800×350~450 |
| **활동지 도해** | `AC` | 실습 활동 구조·기록표 | SVG 직접 제작 | 800×300 |

### 2.2 우선순위 등급

| 등급 | 기준 | 비고 |
|:----:|------|------|
| **P1 필수** | 사용자가 소프트웨어를 처음 볼 때 반드시 필요한 그림 | UI 레이아웃 |
| **P2 권장** | 사용 흐름 이해를 크게 돕는 그림 | 핵심 워크플로우 |
| **P3 선택** | ASCII로도 충분하나, SVG화하면 품질 향상 | 알고리즘·활동지 |

### 2.3 공유 가능성 분석

동일 시뮬레이터의 퀵가이드·사용설명서는 같은 UI 레이아웃 그림을 **공유**할 수 있다. 또한 스네이크·틱택토는 Q-Learning 개념도를 공유할 수 있다.

| 공유 그룹 | 공유 대상 | 절감 수 |
|----------|----------|:-------:|
| UI 레이아웃 (퀵가이드↔사용설명서) | 7개 시뮬레이터 × 모드별 레이아웃 | 약 10장 |
| Q-Learning 개념도 | 스네이크 ↔ 틱택토 | 1장 |
| **절감 소계** | | **약 11장** |

## 7. 제작 순서 및 작업 흐름

### 7.1 권장 제작 순서

```
1단계: P1 — UI 레이아웃 9장
  ├─ ① 범용 선형회귀 (학습+추론) → 가장 대표적, 3열 레이아웃 패턴 확립
  ├─ ② KNN 분류 (학습+추론) → 같은 3열 패턴 재사용
  ├─ ③ 의사결정트리 (학습분석+추론) → 3열+하단 패턴
  ├─ ④ K-means 클러스터링 (학습+추론) → 같은 3열 패턴 재사용
  ├─ ⑤ 스네이크 강화학습 (전체) → 상단바+3열 패턴
  ├─ ⑥ RL 틱택토 (대시보드+사람대전) → 2열+특수모드 패턴
  └─ ⑦ AI사과농장 (전체) → 세로형 특수 레이아웃

2단계: P2 — 핵심 워크플로우·개념도 17장
  ├─ 공유 Q-Learning 개념도 1장
  ├─ 시뮬레이터별 사용 흐름도 (7장)
  ├─ 시뮬레이터별 상세 워크플로우 (7장)
  └─ AI사과농장 장면 전환 흐름 (2장)

3단계: P3 — 알고리즘·활동지 11장
  ├─ 범용 선형회귀: 경사하강법, R² 등 (4장)
  ├─ K-means: Inertia, 거리계산, 비교실험 (3장)
  ├─ 범용 선형회귀 활용가이드: 활동지 (2장)
  └─ AI사과농장 사용설명서: 타임라인 (1장)
  └─ 스네이크/틱택토 추가 워크플로우 (1장)
```

### 7.2 각 그림의 작업 흐름

문서관리지침 §3.2 이미지 제작 8단계 프로세스를 준수한다.

```
[1] 대상 문서의 ASCII 다이어그램 확인 (내용·구조 파악)
 ↓
[2] 시뮬레이터 HTML 소스 대조 (실제 UI와 정합성 검증)
 ↓
[3] 유니코드 다이어그램 또는 생성 프롬프트 작성 → 사용자 승인
 ↓
[4] SVG 제작 (코딩관리지침 §3~4 준수)
 ↓
[5] 한글 렌더링 확인
 ↓
[6] 문서에 삽입 위치 확정 및 문서 갱신
```

## 9. 의사 결정 필요 사항

### 9.1 스크린샷 병행 여부

| 선택지 | 장점 | 단점 |
|--------|------|------|
| A. SVG만 제작 | 버전 변경에 강함, 일관된 스타일 | 실제 화면과 차이 발생 가능 |
| B. SVG + 스크린샷 병행 | 실제 화면 대조 가능 | 시뮬레이터 업데이트 시 재촬영 필요 |
| C. 스크린샷 우선, SVG 보조 | 가장 정확한 UI 표현 | 스타일 불일치, 유지보수 부담 |

> **권장**: A안 (SVG만). 시뮬레이터가 지속 업데이트되므로 SVG가 유지보수에 유리.

### 9.2 P3 단계 범위

P3 그림 11장은 현재 ASCII로도 기능하고 있어, 전체 제작 여부는 P1·P2 완료 후 결정 가능.

### 9.3 제작 시작 시뮬레이터

| 선택지 | 근거 |
|--------|------|
| A. 범용 선형회귀 (권장) | 3열 레이아웃의 대표격, 패턴 확립 후 타 시뮬레이터에 재사용 |
| B. AI사과농장이야기 | 가장 특수한 레이아웃, 별도 처리 필요 |
| C. 시뮬레이터 무관, P1 일괄 | 모든 UI 레이아웃을 한꺼번에 제작 |
