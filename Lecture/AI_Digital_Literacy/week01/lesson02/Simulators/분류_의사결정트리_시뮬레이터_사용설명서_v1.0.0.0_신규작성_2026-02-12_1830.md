## 1. 소프트웨어 개요

### 1.1 목적

분류 의사결정트리 시뮬레이터는 의사결정트리(Decision Tree) 알고리즘이 데이터를 분류하는 원리를 시각적으로 이해할 수 있도록 설계된 교육용 웹 애플리케이션입니다. 트리가 데이터를 조건별로 나누어 분류하는 과정을 노드 단위 성장 애니메이션, 결정 경계 시각화, 추론 경로 하이라이트 등을 통해 직관적으로 체험할 수 있습니다.

### 1.2 교육 목표

1. **의사결정트리의 구조 이해**: 루트 노드, 내부 노드, 리프 노드의 역할 학습
2. **분할 기준 이해**: 지니 불순도를 기반으로 최적 분할이 선택되는 과정 파악
3. **과적합 체험**: 최대 깊이와 최소 샘플 수 변경을 통한 과적합/과소적합 관찰
4. **모델 평가**: 학습/테스트 정확도, 혼동행렬, 특성 중요도 해석
5. **규칙 추출**: 트리에서 IF-THEN 규칙으로 변환하는 과정 이해
6. **추론 과정 추적**: 새 데이터가 트리를 따라 분류되는 경로 단계별 확인

### 1.3 주요 특징

| 특징 | 설명 |
|------|------|
| 2가지 모드 | 학습 & 분석 모드 + 추론 모드 |
| 2종 내장 데이터셋 | 팔머 펭귄(130+개), 아이리스(150개) |
| CSV 업로드 | 외부 데이터 불러오기 (클래스 열 지정) |
| 트리 성장 애니메이션 | 노드 단위 순차 표시 + 속도 조절 |
| 2D 결정 경계 | 100×100 격자 기반 산점도 + 경계 시각화 |
| 특성 중요도 | 지니 불순도 감소 기반 기여도 막대 차트 |
| IF-THEN 규칙 | 트리에서 자동 추출한 분류 규칙 목록 |
| 추론 경로 하이라이트 | 선택한 데이터의 트리 경로를 노란색으로 강조 |
| 2종 추론 입력 | 테스트 데이터 선택 또는 슬라이더 직접 입력 |

## 3. 화면 구성

### 3.1 전체 레이아웃

시뮬레이터는 상단 헤더(데이터셋 선택바), 모드 탭, 3열 본문(좌측 사이드바 + 중앙 영역 + 우측 사이드바), 하단 패널로 구성됩니다.

**상단 헤더:**
- 시뮬레이터 제목 및 버전 배지
- 📦 데이터셋 드롭다운 선택기
- 📂 CSV 업로드 버튼

**모드 탭:**
- **🌳 학습 & 분석**: 트리 생성, 시각화, 성능 분석
- **🔍 추론 모드**: 개별 데이터 분류 과정 추적

### 3.2 학습 & 분석 모드

**좌측 사이드바 — ⚙️ 트리 설정:**
- 최대 깊이(Max Depth) 슬라이더: 1~8, 기본값 4
- 최소 샘플 수(Min Samples) 슬라이더: 1~10, 기본값 2
- 사용할 특성 체크박스: 데이터셋의 전체 특성 표시
- **🌳 트리 생성** 버튼, **🔄 초기화** 버튼
- 🎬 트리 성장 애니메이션 컨트롤 (트리 생성 후 표시):
  - ▶ 재생, ⏭ 단계, ⏮ 처음 버튼
  - 🐢-🐇 속도 조절 슬라이더 (100~1500ms)
  - 진행률 표시 (현재/전체 노드)
- 📖 의사결정트리란? 알고리즘 설명 박스 (4단계)

**중앙 영역:**
- 🌳 의사결정트리 시각화: Canvas 기반 트리 렌더링 (노드, 엣지, 조건, 클래스)
- 🎯 2D 결정 경계 (특성 2개 이상 선택 시): 산점도 + 결정 경계 색상 영역
  - X축/Y축 특성 선택 드롭다운
  - 학습 데이터(●) + 테스트 데이터(▲) 구분 표시

**우측 사이드바:**
- 📊 데이터 정보: 전체 데이터 수, 학습/테스트 분할 수, 클래스별 분포, 데이터 테이블
- 📏 모델 성능 (트리 생성 후): 학습 정확도, 테스트 정확도, 트리 깊이, 혼동행렬
- 📊 특성 중요도 (트리 생성 후): 지니 불순도 감소 기반 기여도 막대 차트

**하단 패널 (트리 생성 후):**
- 📜 학습된 규칙: 트리에서 추출한 IF-THEN 분류 규칙
- 📊 테스트 결과 상세: 각 테스트 데이터의 예측/실제 클래스, 정오 여부

### 3.3 추론 모드

**좌측 사이드바 — 🔍 추론 입력:**
- **📋 테스트 데이터** 탭: 테스트 데이터 드롭다운 선택, 이전/다음/랜덤/전체 추론 버튼, 샘플 정보 표시
- **✏️ 직접 입력** 탭: 각 특성별 슬라이더 (범위: 데이터 최솟값~최댓값), 실시간 추론

**중앙 영역:**
- 🔍 추론 경로 시각화: 트리에서 해당 데이터의 경로를 노란색으로 하이라이트
- 📋 추론 과정 (단계별): 각 노드에서의 조건 판단 + 방향(왼쪽/오른쪽) + 최종 분류

**우측 사이드바 — 💡 추론 결과:**
- 예측 클래스 (이모지 + 클래스명)
- 정오 판정 (테스트 데이터 사용 시)

## 4. 데이터 입력

### 4.1 내장 데이터셋

**🐧 펭귄 (Palmer Penguins):**
- 출처: Palmer Station, Antarctica
- 약 130개 데이터, 3클래스 (Adelie 🔴, Chinstrap 🟢, Gentoo 🔵)
- 4개 특성:
  - 🦷 부리 길이 (mm): 32.1~59.6
  - 📏 부리 깊이 (mm): 13.1~21.5
  - 🪶 날개 길이 (mm): 172~231
  - ⚖️ 체중 (g): 2700~6300

**🌺 붓꽃 (Iris):**
- 출처: Fisher(1936)
- 150개 데이터, 3클래스 (Setosa 🔴, Versicolor 🟢, Virginica 🔵)
- 4개 특성:
  - 🌿 꽃받침 길이 (cm): 4.3~7.9
  - 🌱 꽃받침 너비 (cm): 2.0~4.4
  - 🌸 꽃잎 길이 (cm): 1.0~6.9
  - 🌷 꽃잎 너비 (cm): 0.1~2.5

### 4.2 CSV 업로드

**절차:**
1. 상단 **📂 CSV 업로드** 버튼 클릭
2. CSV 파일 선택 → 미리보기 모달 표시 (데이터 테이블)
3. 🏷️ 클래스(분류 대상) 열 드롭다운에서 선택
4. **✅ 적용** 클릭

**요구사항:**
- CSV 형식 (쉼표 구분, UTF-8 인코딩)
- 나머지 숫자형 열이 자동으로 특성으로 등록
- 문자열 열 중 하나를 클래스 열로 지정

**CSV 데이터 제거:**
상단 파일명 옆 **✕** 버튼으로 CSV 데이터를 제거하고 내장 데이터셋으로 복귀할 수 있습니다.

### 4.3 데이터 분할

데이터 로드 시 자동으로 80:20 비율로 학습/테스트 데이터를 분할합니다.
- 분할 방식: 시드 42 기반 의사난수 셔플 (Fisher-Yates)
- 동일 데이터셋은 항상 같은 분할 결과 보장

## 5. 학습 & 분석 모드

### 5.1 트리 생성 설정

| 설정 | 범위 | 기본값 | 설명 |
|------|------|--------|------|
| 최대 깊이 | 1~8 | 4 | 트리의 최대 레벨 수. 클수록 복잡, 작을수록 단순 |
| 최소 샘플 수 | 1~10 | 2 | 노드 분할에 필요한 최소 데이터 수. 클수록 조기 중단 |
| 사용할 특성 | 체크박스 | 전체 선택 | 선택한 특성만 분할 기준 후보로 사용 |

### 5.2 트리 생성 알고리즘

**🌳 트리 생성** 클릭 시 다음 과정이 수행됩니다:

1. **지니 불순도 계산**: 현재 노드 데이터의 혼합 정도 측정
2. **최적 분할 탐색**: 선택된 모든 특성 × 모든 임곗값에서 지니 불순도 감소가 최대인 분할 선택
3. **분할 실행**: 조건(특성 ≤ 임곗값)에 따라 데이터를 좌측(≤)/우측(>)으로 분배
4. **재귀 반복**: 최대 깊이, 최소 샘플 수, 지니 임곗값(0.01) 도달 시 리프 노드 생성
5. **성능 평가**: 학습/테스트 정확도, 혼동행렬, 특성 중요도 계산
6. **트리 시각화**: Canvas에 노드/엣지 렌더링 + 성장 애니메이션 재생

### 5.3 트리 시각화

Canvas 기반 트리 렌더링:

**내부 노드 표시:**
- 특성 이모지 + 분할 조건 (예: "🦷 부리 길이 ≤ 43.3")
- 노드의 데이터 수
- 지니 불순도 값

**리프 노드 표시:**
- 예측 클래스 이모지 + 이름 (예: "🔴 Adelie")
- 클래스별 데이터 수

**엣지 표시:**
- 좌측: "≤ 임곗값"
- 우측: "> 임곗값"

### 5.4 트리 성장 애니메이션

트리 생성 후 자동으로 성장 애니메이션이 재생됩니다:

- **▶ 재생**: 노드가 하나씩 순차적으로 나타남 (전위 순회)
- **⏭ 단계**: 다음 노드 1개만 추가
- **⏮ 처음**: 모든 노드 숨기고 처음부터 다시
- **🐢-🐇 속도**: 100ms(빠름) ~ 1500ms(느림), 기본 600ms
- 진행률: "현재 노드 / 전체 노드" + 완료 시 "✅ 트리 완성!"

### 5.5 2D 결정 경계

특성을 2개 이상 선택한 경우 산점도 + 결정 경계가 표시됩니다:

- 100×100 해상도 격자에서 각 점을 트리로 분류
- 분류 결과에 해당하는 색상을 25% 투명도로 채색
- 학습 데이터: ● (원) 표시
- 테스트 데이터: ▲ (삼각형) 표시
- X축/Y축 특성은 드롭다운으로 변경 가능

### 5.6 성능 지표

**학습 정확도 / 테스트 정확도:**
- 학습 데이터(80%)와 테스트 데이터(20%)에 대한 분류 정확도
- 색상 코드: >90% 초록, >70% 노랑, 이하 빨강

**트리 깊이:**
- 실제 생성된 트리의 깊이 (최대 깊이 이하)

**혼동행렬:**
- 테스트 데이터 기반
- 행: 실제 클래스, 열: 예측 클래스
- 대각선: 정분류 (하이라이트)

### 5.7 특성 중요도

지니 불순도 감소(Gini Importance) 기반:
- 각 특성이 트리 분할에 사용될 때의 불순도 감소량 합계
- 전체 합 = 100%로 정규화하여 막대 차트 표시
- 퍼센트 + 진행 막대로 시각화

### 5.8 학습된 규칙

트리의 루트에서 각 리프까지의 경로를 IF-THEN 규칙으로 변환:
- 예: "IF 부리 길이 ≤ 43.3 AND 날개 길이 ≤ 206 → Adelie"
- 각 규칙에 해당하는 클래스 색상 배지 표시

### 5.9 테스트 결과 상세

테스트 데이터 각각에 대해:
- 각 특성값 표시
- 실제 클래스 vs 예측 클래스
- 정분류(✅)/오분류(❌) 판정

## 6. 추론 모드

### 6.1 테스트 데이터 추론

**📋 테스트 데이터** 탭에서 드롭다운으로 테스트 데이터를 선택합니다:

- 각 테스트 데이터의 특성값, 실제 클래스 표시
- 추론 과정을 단계별로 표시:
  - 각 단계: 특성 = 값, 조건(≤/>), 방향(왼쪽/오른쪽)
  - 최종 분류: 예측 클래스 + 정오 판정

**네비게이션 버튼:**
- ◀ 이전 / ▶ 다음: 순차 이동
- 🎲 랜덤: 임의 테스트 데이터
- 📊 전체 추론: 모든 테스트 데이터 일괄 추론 → 정확도 표시

### 6.2 직접 입력 추론

**✏️ 직접 입력** 탭에서 슬라이더로 각 특성값을 설정합니다:

- 각 특성별 슬라이더 (범위: 데이터의 최솟값~최댓값)
- 슬라이더 이동 시 **실시간으로 추론 결과 갱신**
- 추론 과정(단계별 조건)과 트리 경로 하이라이트 동시 표시

### 6.3 추론 경로 시각화

추론 모드에서 트리 시각화는 해당 데이터의 경로를 노란색(#F9A825)으로 하이라이트합니다:
- 경로 노드: 노란색 배경 + 두꺼운 테두리
- 경로 엣지: 노란색 + 두꺼운 선
- 비경로 노드/엣지: 기본 스타일 유지

### 7.1 기술 스택

| 항목 | 기술 |
|------|------|
| 마크업 | HTML5 |

### 7.2 알고리즘 상세

**지니 불순도 (Gini Impurity):**
- 수식: Gini(D) = 1 − Σ(pᵢ²), 여기서 pᵢ는 클래스 i의 비율
- Gini = 0: 완전 순수 (단일 클래스)
- Gini = 0.5: 2클래스 균등 혼합 (최대 불순도)

**최적 분할 선택:**
1. 선택된 특성별로 모든 고유값을 임곗값 후보로 설정
2. 각 임곗값에서 좌측(≤)/우측(>)으로 데이터 분할
3. 가중 평균 지니 불순도 계산: (|left|/|total|)×Gini(left) + (|right|/|total|)×Gini(right)
4. 가중 평균 지니가 최소인 (특성, 임곗값) 조합 선택

**분할 중단 조건:**
- 최대 깊이 도달
- 데이터 수 < 최소 샘플 수
- 지니 불순도 < 0.01 (거의 순수)
- 유효한 분할이 없음 (한쪽이 비는 경우)

**특성 중요도 계산:**
- 각 내부 노드에서: (해당 노드의 데이터 비율) × (지니 불순도 감소량)
- 같은 특성이 여러 노드에서 사용된 경우 합산
- 전체 합으로 정규화 (0~100%)

## 9. FAQ

**Q1. 왜 지니 불순도를 사용하나요?**
A1. 지니 불순도는 계산이 간단하면서도 데이터의 혼합 정도를 효과적으로 측정합니다. 다른 기준(정보 이득/엔트로피)도 사용 가능하지만, 본 시뮬레이터는 교육용으로 가장 직관적인 지니 불순도를 채택했습니다.

**Q2. 최대 깊이를 높이면 항상 정확도가 올라가나요?**
A2. 학습 정확도는 올라가지만 테스트 정확도는 오히려 떨어질 수 있습니다. 이것이 과적합입니다. 트리가 학습 데이터의 노이즈까지 외워버려 새 데이터에 대한 일반화 능력이 떨어집니다.

**Q3. 특성 중요도가 0%인 특성이 있는 이유는?**
A3. 해당 특성이 트리 분할에 한 번도 사용되지 않았다는 의미입니다. 다른 특성이 분류에 더 효과적이었기 때문입니다.

**Q4. 학습된 규칙은 어떻게 읽나요?**
A4. "IF 조건1 AND 조건2 → 클래스" 형식입니다. 트리의 루트에서 리프까지의 경로를 나타냅니다. 예를 들어 "IF 꽃잎 길이 ≤ 2.5 → Setosa"는 꽃잎 길이가 2.5cm 이하면 Setosa로 분류한다는 의미입니다.

**Q5. 시드 42란 무엇인가요?**
A5. 데이터 분할 시 사용하는 난수 시드입니다. 동일한 시드를 사용하면 같은 데이터셋에서 항상 같은 학습/테스트 분할이 보장되어, 실험 결과를 재현할 수 있습니다.
