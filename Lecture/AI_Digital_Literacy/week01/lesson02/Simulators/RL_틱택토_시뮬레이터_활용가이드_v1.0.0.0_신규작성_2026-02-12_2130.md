## 1. 교육 활용 개요

### 1.1 교육적 가치

RL 틱택토 시뮬레이터는 강화학습의 핵심 개념을 3×3 보드 게임이라는 친숙한 맥락에서 학습할 수 있는 교육 도구입니다. 학습자는 AI가 "아무것도 모르는 상태"에서 출발하여 시행착오를 통해 최적 전략을 발견하는 전체 과정을 직접 관찰할 수 있습니다.

| 교육적 가치 | 설명 |
|------------|------|
| **즉각적 관찰** | 학습 시작 → 성장 → 숙련까지 수분 내 관찰 가능 |
| **수학적 이해** | Q-값, 보상, 할인율 등 강화학습의 수학적 요소를 시각적으로 체험 |
| **게임이론 연결** | 최적 플레이 시 항상 무승부라는 틱택토의 게임이론적 성질 발견 |
| **실험 설계** | 파라미터 변경 → 결과 관찰 → 분석의 과학적 탐구 과정 체험 |
| **인간 vs AI** | 직접 대결로 AI의 성장을 체감하고 AI의 판단 근거 분석 |

### 1.2 대상 학습자

| 대상 | 활용 방법 |
|------|----------|
| 초등 5~6학년 | 게임을 통한 AI 개념 소개, 사람 대전으로 AI 체험 |
| 중학생 | 학습 단계 관찰, 파라미터 실험, 탐험/활용 개념 학습 |
| 예비교사 | 강화학습 원리 이해, 수업 시나리오 설계, 3가지 학습유형 비교 교수법 |
| AI 입문자 | Q-Learning 알고리즘 체험, 하이퍼파라미터 튜닝 실습 |

| 디지털 리터러시 | AI의 학습 방법 이해 | 보상·벌점을 통한 행동 개선 관찰 |

### 2.2 학습 목표 매핑

| 학습 목표 | 시뮬레이터 기능 | 관찰 포인트 |
|----------|----------------|------------|
| 강화학습이 무엇인지 설명할 수 있다 | 학습 시작 → 성장 관찰 | ε 감소, 승률 변화, 등급 향상 |

| 보상이 학습에 미치는 영향을 이해할 수 있다 | 보상 구조 분석 | +1.0(승) / -1.0(패) / +0.3(무) |
| AI의 판단 근거를 해석할 수 있다 | Q-값 히트맵 분석 | 초록(유리) / 빨간(불리) 수 판별 |
| AI와 사람의 차이를 비교할 수 있다 | 사람 대전 모드 | AI의 착수와 나의 판단 비교 |

## 3. 수업 시나리오

### 시나리오 A: 강화학습 입문 — "AI에게 틱택토를 가르쳐볼까요?" (20분)

**학습 목표:** 강화학습의 기본 원리(시행착오, 보상, 학습)를 설명할 수 있다.

| 단계 | 시간 | 교사 활동 | 학생 활동 | 시뮬레이터 |
|:----:|:----:|----------|----------|-----------|
| 도입 | 3분 | "틱택토를 한 번도 안 해본 아이에게 가르친다면?" | 자신의 경험 공유 | - |
| 관찰 | 5분 | 시뮬레이터 시연 (자기대국 시작) | 화면 관찰 | 🔄자기대국 + 속도 1ms |
| 발견 | 7분 | "AI가 어떻게 변하고 있나요?" 유도 질문 | 승률·등급 변화 관찰 기록 | 대시보드 차트 |
| 체험 | 3분 | 사람 대전 모드 전환 | AI와 직접 대결 | 👤 사람 대전 |
| 정리 | 2분 | "AI는 어떻게 배웠을까?" 정리 | 핵심 개념 정리 | - |

**교사 발문 예시:**
- "처음에 AI의 승률이 왜 이렇게 불규칙할까요?" → 아직 아무것도 모르기 때문
- "시간이 지나면서 무승부가 왜 늘어날까요?" → 양쪽 모두 잘 배워서
- "최적의 틱택토는 어떤 결과가 나올까요?" → 항상 무승부

---

### 시나리오 B: 탐험 vs 활용 — "모험할까, 안전하게 갈까?" (20분)

**학습 목표:** 강화학습에서 탐험(exploration)과 활용(exploitation)의 균형이 왜 중요한지 설명할 수 있다.

| 단계 | 시간 | 활동 | 시뮬레이터 설정 |
|:----:|:----:|------|---------------|
| 도입 | 3분 | 비유: "맨날 가던 식당 vs 새 식당" | - |
| 실험1 | 5분 | ε 감소율 0.99 (빠른 전환) 학습 | ε감소=0.99, 10,000판 |
| 실험2 | 5분 | ε 감소율 0.9999 (느린 전환) 학습 | ε감소=0.9999, 10,000판 |
| 비교 | 5분 | 두 결과 비교 토론 | 📋 보고서 다운로드 |
| 정리 | 2분 | 너무 빨리 안주하면 vs 너무 오래 탐험하면 | - |

**핵심 발견:** ε 감소가 너무 빠르면(0.99) 초기 경험에 갇히고, 너무 느리면(0.9999) 랜덤 수가 오래 지속됩니다. 적절한 균형(0.9995)이 최적 학습을 이끕니다.

---

### 시나리오 C: 파라미터 실험 — "AI 튜닝 실험실" (25분)

**학습 목표:** 학습률(α)과 할인율(γ)이 학습 결과에 미치는 영향을 실험으로 확인할 수 있다.

| 단계 | 시간 | 활동 |
|:----:|:----:|------|
| 도입 | 3분 | "자전거를 빨리 배우려면?" → 파라미터 개념 소개 |
| 실험 | 15분 | 조별로 다른 파라미터 설정 후 20,000판 학습 |
| 발표 | 5분 | 조별 결과(등급, 강도) 발표 및 비교 |
| 정리 | 2분 | 최적 파라미터 조합 토론 |

**실험 조건표:**

| 조 | α (학습률) | γ (할인율) | 예상 효과 |
|:--:|:---------:|:---------:|----------|
| 1조 | 0.10 (낮음) | 0.90 | 안정적이나 느린 학습 |
| 2조 | 0.50 (높음) | 0.90 | 빠르지만 불안정할 수 있음 |
| 3조 | 0.30 (기본) | 0.50 (낮음) | 즉각적 보상 중시, 장기 전략 약함 |
| 4조 | 0.30 (기본) | 0.99 (높음) | 장기 전략 중시, 안정적 |

### 시나리오 D: 사람 vs AI — "내가 AI를 이길 수 있을까?" (15분)

**학습 목표:** 학습된 AI의 의사결정을 Q-값 히트맵으로 분석하고, 사람의 직관과 비교할 수 있다.

| 단계 | 시간 | 활동 | 시뮬레이터 |
|:----:|:----:|------|-----------|
| 준비 | 3분 | 50,000판 이상 학습된 AI 준비 | 사전 학습 or 파일 가져오기 |
| 대전 | 7분 | AI와 5판 대결 + Q-값 관찰 | 👤 사람 대전 + Q-히트맵 |
| 분석 | 3분 | "AI는 왜 그 수를 뒀을까?" 토론 | Q-값 해석 |
| 정리 | 2분 | AI의 판단과 사람의 직관 비교 | - |

**Q-값 히트맵 읽기 안내:**
- 초록 배경 + 양수: AI가 유리하다고 판단하는 수
- 빨간 배경 + 음수: AI가 불리하다고 판단하는 수
- 가장 큰 양수: AI가 최적이라고 판단하는 수

---

### 시나리오 E: 학습 방식 비교 — "혼자 연습 vs 함께 연습" (20분)

**학습 목표:** 자기대국과 단독 학습의 차이를 비교하고, 학습 환경이 결과에 미치는 영향을 설명할 수 있다.

| 단계 | 시간 | 활동 |
|:----:|:----:|------|
| 도입 | 3분 | "축구 혼자 연습 vs 친구와 연습" 비유 |
| 실험 | 12분 | 3가지 방식으로 각 10,000판 학습 후 비교 |
| 분석 | 3분 | 보고서 비교: 강도·등급·승률 차이 |
| 정리 | 2분 | "함께 배우면 왜 더 잘 배울까?" |

**비교 항목:** 🔄자기대국(X=85, O=80) vs 🔵X단독(X=75, O=약함) vs 🔴O단독(X=약함, O=70) — 실제 수치는 실험마다 달라질 수 있습니다.

**질문:**
1. AI가 첫수로 어디를 선호하나요? 왜 그곳이 좋을까요?

## 5. 교수·학습 팁

### 5.1 비유 활용

| 개념 | 비유 | 설명 |
|------|------|------|
| Q-테이블 | 시험 오답노트 | 각 상황에서 좋았던/나빴던 수를 기록해두는 노트 |
| ε (탐험률) | 새 식당 도전 확률 | 처음엔 100% 모험 → 점점 단골집(최적 수) 선호 |
| 학습률 α | 새 경험의 반영 정도 | 높으면 최근 결과에 크게 영향받고, 낮으면 안정적 |
| 할인율 γ | 미래 계획의 비중 | 높으면 장기 전략 중시, 낮으면 당장 이득 중시 |
| 대칭성 정규화 | "어차피 같은 상황이잖아" | 보드를 돌려도 같은 판이면 하나로 통합 |
| 무승부 = 성공 | "뺏기지 않는 것도 잘하는 것" | 최적 틱택토는 항상 무승부 |

### 5.2 흔한 오개념과 교정

| 오개념 | 교정 |
|--------|------|
| "AI가 규칙을 미리 알고 있다" | AI는 규칙만 알고 전략은 모릅니다. 시행착오로 배웁니다 |
| "학습이 많으면 무조건 이긴다" | 틱택토는 최적 시 무승부입니다. 이기는 AI는 상대가 실수한 것 |
| "무승부가 많으면 학습 실패" | 오히려 무승부율 증가가 올바른 학습의 신호입니다 |
| "ε이 0이 되어야 최적" | 약간의 탐험(ε=1%)이 새로운 상황 대응에 도움이 됩니다 |
| "X가 항상 유리하다" | 선공 유리가 있지만, 최적 플레이 시 후공도 무승부 가능 |
| "AI는 창의적으로 둔다" | Q-테이블에 기록된 패턴을 따를 뿐, 진정한 '창의성'은 없습니다 |

### 5.3 3종 학습유형 비교 교수법

강화학습 시뮬레이터는 3가지 기계학습 유형을 비교 설명하는 데 활용할 수 있습니다.

| 학습유형 | 핵심 | 틱택토 비유 |
|----------|------|------------|
| 지도학습 | 정답이 있는 데이터로 학습 | "이 상황에선 여기 두는 게 정답이야" (가르쳐줌) |
| 비지도학습 | 패턴을 스스로 발견 | "이 보드들을 비슷한 것끼리 묶어볼래?" (분류) |
| **강화학습** | 시행착오 + 보상으로 학습 | "일단 둬봐 → 이기면 +1, 지면 -1" (**이 시뮬레이터**) |

## 6. 평가 연계

### 6.1 과정 평가 관찰 포인트

| 평가 영역 | 관찰 항목 | 수준 |
|----------|----------|:----:|
| **AI 개념 이해** | 강화학습의 핵심 요소(에이전트, 보상, 탐험/활용)를 설명할 수 있는가 | 상/중/하 |
| **실험 설계** | 파라미터를 변경하고 결과를 비교하는 실험을 설계할 수 있는가 | 상/중/하 |
| **데이터 해석** | 차트(승률, ε, 강도)를 읽고 학습 진행 상황을 해석할 수 있는가 | 상/중/하 |
| **비판적 사고** | Q-값 히트맵을 분석하여 AI의 판단 근거를 설명할 수 있는가 | 상/중/하 |
| **실생활 연결** | 강화학습 원리를 실생활 사례에 적용하여 설명할 수 있는가 | 상/중/하 |

### 6.2 포트폴리오 과제 예시
