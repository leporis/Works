## 1. 교육 활용 개요

### 1.1 교육적 가치

분류 의사결정트리 시뮬레이터는 다음과 같은 교육적 가치를 제공합니다.

| 가치 | 설명 |
|------|------|
| **해석 가능성 체험** | 트리 구조를 시각적으로 확인하여 "AI가 왜 그런 결정을 내렸는지" 이해 |
| **규칙 기반 사고** | 예/아니오 질문의 연쇄가 복잡한 분류를 수행하는 과정을 직관적으로 관찰 |
| **과적합 실험** | 최대 깊이와 최소 샘플 수를 조절하여 모델의 일반화 능력을 직접 실험 |
| **특성 중요도 분석** | 어떤 특성이 분류에 가장 큰 영향을 미치는지 시각적으로 확인 |
| **추론 과정 추적** | 추론 모드에서 데이터가 트리를 따라가는 경로를 단계별로 관찰 |

### 1.2 대상 학습자

| 대상 | 활용 초점 | 권장 활동 |
|------|----------|----------|
| 예비교사 (대학생) | ML 알고리즘 원리 이해 + 수업 설계 역량 | 시나리오 B, C + 활동지 A, B |
| 현직 교사 (연수) | 수업 적용 방법 파악 | 시나리오 A + 활동지 A |
| 초등 고학년 (5~6학년) | AI의 분류 원리 체험 | 시나리오 A + 관찰 위주 활동 |

| [6실05-03] | 인공지능 활용 사례를 살펴보고 사회에 미치는 영향을 토의한다 | 펭귄/붓꽃 분류 사례, 의료·금융 확장 토의 |

| 특성 중요도를 해석하고 핵심 특성을 판별할 수 있다 | 특성 중요도 차트 | 활동지 C 분석 |
| 새 데이터의 분류 결과를 트리 경로로 추적할 수 있다 | 추론 모드 + 경로 하이라이트 | 추론 실습 기록 |

## 3. 수업 시나리오

### 시나리오 A: 의사결정트리 입문 (15분)

> **목표**: 의사결정트리가 어떻게 데이터를 분류하는지 처음으로 체험한다.

| 단계 | 시간 | 교수 활동 | 학생 활동 | 시뮬레이터 기능 |
|:----:|:----:|----------|----------|---------------|
| 도입 | 3분 | "스무고개 놀이 해본 적 있나요?" 예/아니오 질문의 힘 설명 | 스무고개 비유에 공감 | - |
| 체험 | 5분 | 🐧 펭귄 데이터셋으로 트리 생성 시연 | 트리가 자라는 애니메이션 관찰, 각 노드의 조건 읽기 | 트리 생성 + 성장 애니메이션 |
| 추론 | 4분 | 추론 모드에서 테스트 데이터 선택 시연 | 노란색 추론 경로를 따라가며 "왜 이 결과인지" 추적 | 추론 모드 + 경로 하이라이트 |
| 정리 | 3분 | "트리의 첫 번째 질문이 가장 중요한 이유는?" 발문 | 특성 중요도 차트와 연결하여 답변 | 특성 중요도 |

### 시나리오 B: 과적합 실험 수업 (25분)

> **목표**: 최대 깊이를 변경하며 과적합(overfitting)의 개념을 체험적으로 이해한다.

| 단계 | 시간 | 교수 활동 | 학생 활동 | 시뮬레이터 기능 |
|:----:|:----:|----------|----------|---------------|
| 도입 | 3분 | "외우기 vs 이해하기" 비유로 과적합 개념 도입 | 비유 이해 | - |
| 실험1 | 5분 | 깊이 1로 트리 생성 → "너무 단순한 트리" 관찰 | 정확도 기록, 결정 경계 관찰 (넓은 영역) | 최대 깊이=1, 결정 경계 |
| 실험2 | 5분 | 깊이 4로 트리 생성 → "적절한 트리" 관찰 | 정확도 기록, 결정 경계 관찰 (균형 잡힌 영역) | 최대 깊이=4, 결정 경계 |
| 실험3 | 5분 | 깊이 8로 트리 생성 → "너무 복잡한 트리" 관찰 | 정확도 기록, 결정 경계 관찰 (세밀한 영역) | 최대 깊이=8, 결정 경계 |
| 비교 | 4분 | 3개 결과 비교 토론 유도: "훈련 정확도 vs 테스트 정확도" | 활동지 B 기록표 작성, 최적 깊이 판단 | 혼동행렬 비교 |
| 정리 | 3분 | 과적합/과소적합 정리, 실생활 비유 강화 | 핵심 개념 정리 | - |

### 시나리오 C: 특성 선택과 해석 탐구 (20분)

> **목표**: 어떤 특성을 사용하느냐에 따라 트리 구조와 성능이 달라지는 것을 실험하고, 특성 중요도를 해석한다.

| 단계 | 시간 | 교수 활동 | 학생 활동 | 시뮬레이터 기능 |
|:----:|:----:|----------|----------|---------------|
| 도입 | 3분 | "펭귄을 구분할 때 어떤 특징을 보면 좋을까?" 발문 | 자유 예측 (부리 길이? 체중?) | - |
| 실험1 | 5분 | 모든 특성 사용하여 트리 생성 | 특성 중요도 차트 관찰, 가장 중요한 특성 기록 | 특성 선택 (전체) + 특성 중요도 |
| 실험2 | 5분 | 중요도 높은 특성 2개만 선택하여 트리 생성 | 정확도 변화 관찰, 결정 경계 시각화 비교 | 특성 선택 (2개) + 결정 경계 |
| 실험3 | 4분 | 중요도 낮은 특성 2개만 선택하여 트리 생성 | 정확도 하락 관찰, "왜 이 특성은 효과가 없는지" 토론 | 특성 선택 (2개) + 학습된 규칙 |
| 정리 | 3분 | 특성 선택의 중요성 정리, 실생활 적용 토론 | 활동지 C 작성 | - |

## 4. 학생 활동지

### 활동지 A: 트리 규칙 읽기

**목표**: 의사결정트리의 분류 규칙을 읽고 해석할 수 있다.

**준비**: 🐧 펭귄 데이터셋, 최대 깊이 3, 기본 설정

| 단계 | 지시 사항 |
|:----:|----------|
| 1 | 🐧 펭귄 데이터셋을 선택하고 **🌳 트리 생성**을 클릭하세요 |
| 2 | 트리의 **루트 노드**(맨 위 노드)에 적힌 조건을 아래에 적으세요 |
| 3 | "학습된 규칙" 패널에서 규칙 3개를 골라 아래 표에 적으세요 |
| 4 | **🔍 추론 모드**에서 테스트 데이터 3개를 선택하고 추론 경로를 적으세요 |

**기록표 — 학습된 규칙**:

| # | 규칙 (IF ... THEN ...) | 분류 결과 | 해당 데이터 수 |
|:-:|----------------------|----------|:------------:|
| 1 | | | |
| 2 | | | |
| 3 | | | |

**기록표 — 추론 경로**:

| # | 테스트 데이터 | 1번째 분기 조건 | 2번째 분기 조건 | 최종 분류 | 정답 여부 |
|:-:|-------------|----------------|----------------|---------|:---------:|
| 1 | | | | | |
| 2 | | | | | |
| 3 | | | | | |

**생각해보기**: 트리의 첫 번째 질문(루트 노드)이 가장 중요한 이유는 무엇일까요?

---

### 활동지 B: 과적합 실험 기록

**목표**: 최대 깊이에 따른 트리 복잡도와 정확도의 관계를 실험적으로 발견한다.

**준비**: 🐧 펭귄 데이터셋, 최소 샘플 수 2 (기본값)

| 단계 | 지시 사항 |
|:----:|----------|
| 1 | 최대 깊이를 **1, 2, 4, 6, 8** 순서로 바꾸며 각각 트리를 생성하세요 |
| 2 | 매번 정확도와 노드 수, 결정 경계 모양을 기록하세요 |
| 3 | 모든 실험 후 "최적의 깊이"가 무엇인지 판단하세요 |

**기록표**:

| 최대 깊이 | 전체 정확도 (%) | 노드 수 | 결정 경계 모양 (간단히 스케치/설명) |
|:---------:|:--------------:|:------:|----------------------------------|
| 1 | | | |
| 2 | | | |
| 4 | | | |
| 6 | | | |
| 8 | | | |

**분석 질문**:
1. 깊이가 깊어질수록 훈련 정확도는 어떻게 변하나요?
2. 깊이가 너무 깊으면 어떤 문제가 생길 수 있을까요? (결정 경계를 보고 판단)
3. "딱 좋은" 깊이는 몇일까요? 그 이유는?

---

### 활동지 C: 특성 중요도 분석

**목표**: 특성 선택이 분류 성능에 미치는 영향을 분석한다.

**준비**: 🐧 펭귄 데이터셋, 최대 깊이 4

| 단계 | 지시 사항 |
|:----:|----------|
| 1 | 모든 특성(4개)을 선택하고 트리를 생성하세요 → 특성 중요도 순위를 기록하세요 |
| 2 | 중요도 1, 2위 특성만 선택하여 트리를 다시 생성하세요 → 정확도를 기록하세요 |
| 3 | 중요도 3, 4위 특성만 선택하여 트리를 생성하세요 → 정확도를 기록하세요 |
| 4 | 특성 1개만 선택하여 트리를 생성하세요 → 정확도를 기록하세요 |

**기록표**:

| 사용 특성 | 정확도 (%) | 트리 깊이 | 특성 중요도 순위 |
|----------|:---------:|:--------:|:----------------|
| 전체 (4개) | | | 1위:__ 2위:__ 3위:__ 4위:__ |
| 상위 2개 (__, __) | | | |
| 하위 2개 (__, __) | | | |
| 1개만 (__) | | | |

**분석 질문**:
1. 상위 2개 특성만으로도 전체와 비슷한 정확도가 나오나요?
2. 어떤 특성이 펭귄 종 분류에 가장 중요한가요? 왜 그렇다고 생각하나요?
3. 불필요한 특성을 제거하면 어떤 장점이 있을까요?

## 5. 학습 질문 설계

### 5.1 기능별 발문

| 시뮬레이터 기능 | 발문 | 기대 관찰/답변 |
|---------------|------|---------------|
| 트리 성장 애니메이션 | "트리가 자라는 순서를 보세요. 왜 이 질문을 먼저 했을까요?" | 지니 불순도가 가장 많이 감소하는 질문이 먼저 선택됨 |
| 결정 경계 (깊이 1) | "영역이 2개뿐이네요. 이 트리로 3종을 정확히 구분할 수 있을까요?" | 깊이가 부족하면 세밀한 분류 불가 (과소적합) |
| 결정 경계 (깊이 8) | "경계가 아주 복잡하네요. 새 데이터도 잘 맞출 수 있을까요?" | 훈련 데이터에 과도하게 맞추면 일반화 실패 (과적합) |
| 혼동행렬 | "Adelie를 Chinstrap으로 잘못 분류한 데이터가 몇 개인가요?" | 혼동행렬의 비대각 요소 읽기 |
| 특성 중요도 | "막대 길이가 0에 가까운 특성을 빼면 정확도가 떨어질까요?" | 실험으로 확인 → 중요도 낮은 특성 제거해도 큰 차이 없음 |
| 학습된 규칙 | "이 규칙을 사람이 읽을 수 있나요? 다른 AI 모델도 이렇게 설명 가능한가요?" | 의사결정트리의 해석 가능성(interpretability) 장점 |
| 추론 경로 하이라이트 | "노란색 경로를 따라가 보세요. 몇 번의 질문으로 답을 찾았나요?" | 깊이 = 질문 횟수, 적은 질문으로 분류 가능 |

### 5.2 블룸 수준별 질문

| 수준 | 질문 예시 |
|------|----------|
| **기억** | 의사결정트리의 루트 노드에 적힌 조건은 무엇인가요? |
| **이해** | 의사결정트리가 데이터를 분류하는 방법을 스무고개에 비유하여 설명하세요. |
| **적용** | 최대 깊이를 2에서 6으로 바꾸면 정확도와 결정 경계가 어떻게 달라지나요? |
| **분석** | 펭귄 데이터에서 "부리 길이"와 "체중" 중 어느 것이 분류에 더 중요한가요? 근거를 제시하세요. |
| **평가** | 깊이 2 트리와 깊이 8 트리 중 새 펭귄 데이터를 분류할 때 어느 것이 더 신뢰할 만한가요? 이유를 설명하세요. |
| **창의** | 의사결정트리를 "학교 급식 메뉴 선택"에 적용한다면, 어떤 질문들로 트리를 만들 수 있을까요? |

## 6. 추천 데이터 설정

### 6.1 수업 목적별 추천

### 6.2 내장 데이터셋 활용 팁

| 항목 | 내용 |
|------|------|
| 클래스 | Adelie, Chinstrap, Gentoo (3종) |
| 특성 | 부리 길이(mm), 부리 깊이(mm), 날개 길이(mm), 체중(g) |

## 7. 시뮬레이터 연계 학습

### 7.1 다른 분류 시뮬레이터와의 비교

이 시뮬레이터는 같은 데이터셋(펭귄, 붓꽃)을 다른 알고리즘으로도 분류할 수 있는 KNN 분류 시뮬레이터와 함께 활용하면 효과적입니다.

| 비교 항목 | 의사결정트리 | KNN |
|----------|------------|-----|
| 분류 방식 | 규칙 기반 (질문의 연쇄) | 거리 기반 (가장 가까운 이웃) |
| 결정 경계 | 축에 평행한 직선 (계단형) | 곡선 (보로노이 다이어그램) |
| 해석 가능성 | 높음 (규칙을 읽을 수 있음) | 낮음 (이웃 데이터에 의존) |
| 핵심 파라미터 | 최대 깊이, 최소 샘플 수 | K값, 거리 방식 |
| 장점 | 사람이 이해하기 쉬움 | 단순하고 직관적 |

**연계 활동 제안**: 같은 펭귄 데이터셋으로 두 시뮬레이터를 실행하고, 결정 경계의 모양과 정확도를 비교하는 활동을 진행합니다.

### 7.2 AI 사과 농장 이야기와의 연계

AI 사과 농장 이야기 애니메이션의 5장(출하 결정)에서 결정 트리가 등장합니다. 애니메이션에서 개념을 먼저 접한 뒤, 이 시뮬레이터로 직접 트리를 만들어보는 순서가 효과적입니다.
