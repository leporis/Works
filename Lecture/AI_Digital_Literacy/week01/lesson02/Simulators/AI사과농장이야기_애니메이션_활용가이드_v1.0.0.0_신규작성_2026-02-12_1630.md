## 1. 교육 활용 개요

### 1.1 이 애니메이션의 교육적 가치

AI 사과 농장 이야기는 다음 세 가지 교육적 강점을 가집니다.

**① 스토리 기반 통합 학습**: 5가지 ML 알고리즘을 사과 농장이라는 하나의 서사로 연결합니다. 개별 알고리즘을 따로 배우면 "왜 여러 알고리즘이 필요한지" 이해하기 어렵지만, 농장의 문제 해결 과정을 따라가면 각 알고리즘의 적합한 활용 맥락이 자연스럽게 드러납니다.

**② 비유와 대화 기반 설명**: 농부와 로봇의 대화 구조는 "문제 → 해결 → 이해 확인"이라는 교수-학습 패턴을 그대로 반영합니다. 농부가 학습자의 대리인으로서 "아하!" 반응을 보여주어 학습자의 이해를 자연스럽게 안내합니다.

**③ 지도학습 vs 비지도학습 대비**: 5개 알고리즘 중 4개가 지도학습, 1개(K-means)가 비지도학습으로 구성되어, "정답이 있는 학습"과 "정답 없는 학습"의 차이가 스토리 속에서 명확하게 드러납니다.

### 1.2 대상 학습자

| 대상 | 적합도 | 활용 방법 |
|------|:-----:|----------|
| 초등학생 (5~6학년) | ⭐⭐⭐ | 전체 시청 + 간단한 관찰 활동 |
| 예비교사 (대학생) | ⭐⭐⭐ | CK(내용지식) 강화 + 교수 설계 분석 |
| 현직교사 연수 | ⭐⭐ | AI 교육 도입 사례 + 교구 활용법 체험 |
| 일반인 AI 입문 | ⭐⭐ | 비전공자를 위한 ML 개념 소개 |

### 1.3 활용 가능한 수업 형태

| 형태 | 시간 | 설명 |
|------|:---:|------|
| **도입 영상** | 5~7분 | ML 개념 수업 시작 전 동기 유발 |
| **개념 정리** | 10~15분 | ML 알고리즘 학습 후 종합 복습 |
| **시뮬레이터 안내** | 5분 + α | 다른 ML 시뮬레이터 체험 전 맥락 제공 |
| **독립 수업** | 45분 | 애니메이션 + 활동지 + 토론 (아래 시나리오 참조) |
| **과제** | 자율 | 사전/사후 학습용 개인 시청 |

## 2. 교육과정 연계

### 2.1 2022 개정 교육과정 성취기준

이 애니메이션이 직접적으로 연결되는 성취기준은 다음과 같습니다.

| 성취기준 코드 | 내용 | 관련 장면 |
|-------------|------|----------|
| [6실05-04] | 인공지능이 만들어지는 과정을 체험한다 | 1~5장 전체 |
| [6실05-05] | 인공지능 기술을 활용하여 실생활 문제 해결 방법을 탐색한다 | 에필로그 종합 |

### 2.2 교육 목표 매핑

| 장면 | 다루는 개념 | 학습 목표 (학생이 할 수 있는 것) |
|------|-----------|---------------------------|
| 프롤로그 | AI 도입 배경 | AI가 인간의 노동을 "돕는" 도구임을 말할 수 있다 |
| 1장 | CNN, 지도학습 | "정답을 알려주고 배운다"는 지도학습 원리를 설명할 수 있다 |
| 2장 | KNN, 유사도 | "비슷한 것끼리 비교하여 분류한다"는 원리를 설명할 수 있다 |
| 3장 | K-means, 비지도학습 | "정답 없이 스스로 묶는다"는 비지도학습을 지도학습과 구분할 수 있다 |
| 4장 | 선형회귀, 예측 | "데이터에서 규칙을 찾아 예측한다"는 원리를 설명할 수 있다 |
| 5장 | 결정트리, 의사결정 | "예/아니오 질문으로 결정한다"는 과정을 따라갈 수 있다 |
| 에필로그 | 종합 | 문제에 맞는 AI 도구를 선택하는 것이 중요함을 말할 수 있다 |

### 2.3 블룸 분류체계 수준

이 애니메이션은 블룸 분류체계의 **이해(Understand)** 수준을 주로 다루며, 활동 설계에 따라 **적용(Apply)** 및 **분석(Analyze)** 수준까지 확장할 수 있습니다.

| 블룸 수준 | 이 애니메이션에서 | 활동으로 확장 |
|----------|----------------|------------|
| 기억 | 5가지 알고리즘 이름 나열 | 카드 매칭 활동 |
| **이해** | 각 알고리즘의 원리를 비유로 설명 | 자기 말로 다시 설명하기 |
| 적용 | (해당 없음) | 시뮬레이터로 직접 실행 |
| 분석 | (해당 없음) | 지도학습/비지도학습 비교 토론 |
| 평가 | (해당 없음) | "어떤 문제에 어떤 알고리즘?" 판단 |
| 창조 | (해당 없음) | 새로운 비유 이야기 만들기 |

## 3. 수업 시나리오

### 3.1 시나리오 A: 도입 영상 (7분)

**활용 상황**: ML 개념 수업의 첫 번째 활동으로, 학습 동기를 유발합니다.

| 단계 | 시간 | 교수 활동 | 학생 활동 |
|:---:|:---:|----------|----------|
| 안내 | 1분 | "오늘은 AI가 사과 농장을 돕는 이야기를 먼저 보겠습니다" | 화면 주목 |
| 시청 | 5분 | 애니메이션 전체 재생 (1.15x 빠르게) | 시청 |
| 발문 | 1분 | "농장에서 AI가 어떤 일을 했나요? 몇 가지였나요?" | 자유 답변 |

> **💡 팁**: 도입용으로 사용할 때는 1.15x 빠르게 모드로 재생하면 약 4분 내로 끝낼 수 있습니다. 세부 이해보다 전체 흐름 파악이 목적입니다.

### 3.2 시나리오 B: 독립 수업 (45분)

**활용 상황**: ML 알고리즘 개념을 이 애니메이션 중심으로 가르치는 한 차시 수업입니다.

| 단계 | 시간 | 교수 활동 | 학생 활동 | 자료 |
|:---:|:---:|----------|----------|:---:|
| **도입** | 3분 | "여러분이 사과 농장 주인이라면?" 질문 | 브레인스토밍 | — |
| **시청 1회차** | 6분 | 전체 재생 (보통 속도), 중간 멈춤 없이 | 전체 시청 + 관찰 활동지 기록 | 활동지 A |
| **중간 정리** | 5분 | 장면별 핵심 질문 (학생 응답 유도) | 구두 응답 | — |
| **시청 2회차** | 10분 | 1장→3장 선택 재생 (장면 인디케이터 활용), 장면마다 일시정지 후 설명 보충 | 활동지 B 기록 | 활동지 B |
| **토론** | 10분 | "지도학습 4개 vs 비지도학습 1개, 왜 이런 비율일까?" | 모둠 토론 | 질문 카드 |
| **확장** | 7분 | "사과 농장 외에 이 알고리즘이 쓰일 곳은?" | 실생활 예시 찾기 | 활동지 C |
| **정리** | 4분 | 5가지 알고리즘 한 줄 요약 함께 읽기 | 핵심 노트 정리 | — |

> **💡 팁**: 시청 2회차에서 모든 장면을 다시 보면 시간이 부족합니다. 1장(지도학습 대표)과 3장(비지도학습)만 선택 재생하여 두 학습 유형의 대비를 집중적으로 다루는 것을 권장합니다.

### 3.3 시나리오 C: 시뮬레이터 체험 전 맥락 제공 (5분 + α)

**활용 상황**: KNN, K-means, 선형회귀 등 개별 시뮬레이터 체험 수업 전에, 해당 알고리즘이 실생활에서 어떤 문제를 해결하는지 맥락을 제공합니다.

| 순서 | 시간 | 활동 |
|:---:|:---:|------|
| 1 | 3분 | 해당 장면만 선택 재생 (예: KNN 시뮬레이터 체험 전 → 2장만 재생) |
| 2 | 2분 | "방금 본 것처럼, 이 알고리즘을 이제 직접 실행해봅시다" → 시뮬레이터 이동 |
| 3 | α분 | 개별 시뮬레이터 실습 |

**장면-시뮬레이터 매핑**:

| 장면 | 알고리즘 | 연계 시뮬레이터 |
|------|---------|--------------|
| 1장: 수확 | CNN/딥러닝 | (별도 딥러닝 시뮬레이터 또는 Teachable Machine) |
| 2장: 품종 | KNN | KNN 분류 시뮬레이터 v1.2.0.1 |
| 3장: 등급 | K-means | K-means 클러스터링 시뮬레이터 v1.0.0.1 |
| 4장: 가격 | 선형회귀 | 범용 선형회귀 시뮬레이터 v1.4.0.0 |
| 5장: 출하 | 결정트리 | 분류 의사결정트리 시뮬레이터 v1.3.0.0 |

## 5. 학습 질문 설계

### 5.1 장면별 교수 발문

수업 중 특정 장면에서 일시정지한 후 활용할 수 있는 발문입니다.

**프롤로그 후**
- "농부 김사과의 고민은 무엇이었나요?"
- "AI 로봇이 도와준다고 했는데, AI가 농부를 대체한 건가요?"

**1장(CNN) 후**
- "로봇은 어떻게 익은 사과를 알아봤나요?"
- "수천 장의 사과 사진으로 학습했다고 했는데, 사진이 많아야 하는 이유는 뭘까요?"
- "92%라는 숫자는 무엇을 의미하나요? 100%가 아닌 이유는?"

**2장(KNN) 후**
- "'가장 가까운 3개 친구'라는 표현에서 '가깝다'는 것은 무엇을 의미하나요?"
- "만약 3개가 아니라 1개만 비교하면 어떤 문제가 생길까요?"

**3장(K-means) 후** ⭐ 가장 중요한 토론 포인트
- "1~2장과 3장의 가장 큰 차이는 무엇인가요?"
- "정답이 없는데 어떻게 분류할 수 있었을까요?"
- "왜 작년 기준으로 나누면 안 되었나요?"

**4장(선형회귀) 후**
- "'가격 = 30 × 품질 + 1000'에서 30은 무슨 의미인가요?"
- "이 공식은 누가 만든 건가요? 사람이요? AI가요?"

**5장(결정트리) 후**
- "예/아니오 질문의 순서가 바뀌면 결과도 달라질까요?"
- "질문이 2개면 몇 가지 결과가 나올 수 있나요? 3개면?"

**에필로그 후**
- "5가지 AI 중에서 하나만 선택해야 한다면 어떤 걸 고르겠어요? 왜?"

## 6. 시뮬레이터 연계

### 6.1 연계 학습 경로

이 애니메이션은 다른 ML 시뮬레이터와 함께 사용할 때 교육 효과가 극대화됩니다.

```
[1단계: 이야기로 맥락 이해]
    AI 사과 농장 이야기 전체 시청 (5~7분)
              ↓
[2단계: 개별 알고리즘 체험]
    해당 시뮬레이터로 직접 실험 (10~15분씩)
              ↓
[3단계: 종합 정리]
    에필로그 재시청 + 비교 활동 (5분)
```

### 6.2 장면별 시뮬레이터 연계 상세

| 장면 | 시뮬레이터 | 연계 활동 |
|------|-----------|----------|
| 2장: 품종 (KNN) | KNN 분류 시뮬레이터 | "이야기에서 K=3이었죠? K를 바꾸면 결과가 달라지는지 확인해보세요" |
| 3장: 등급 (K-means) | K-means 클러스터링 시뮬레이터 | "이야기에서 3그룹이었죠? K를 2나 4로 바꾸면 어떻게 될까요?" |
| 4장: 가격 (선형회귀) | 범용 선형회귀 시뮬레이터 | "이야기의 '가격=30×품질+1000'처럼, 직선이 데이터를 따라가는 걸 직접 보세요" |
| 5장: 출하 (결정트리) | 분류 의사결정트리 시뮬레이터 | "이야기의 예/아니오 질문 트리를 데이터로 직접 만들어보세요" |

### 6.3 추가 연계 가능 외부 도구

| 도구 | 연계 장면 | 활용 |
|------|----------|------|

| 엔트리 AI 블록 | 전체 | 이미지 분류, 테이블 기반 학습 |
