## 1. 소프트웨어 개요

### 1.1 목적

스네이크 강화학습 시뮬레이터는 Q-Learning 알고리즘을 활용하여 AI 에이전트가 스네이크 게임을 스스로 학습하는 과정을 시각적으로 관찰할 수 있는 교육용 웹 애플리케이션입니다. 보상과 벌점을 통해 행동을 개선해나가는 강화학습의 핵심 원리를 게임이라는 친숙한 맥락에서 직관적으로 이해할 수 있습니다.

### 1.2 교육 목표

1. **강화학습의 핵심 개념 이해**: 에이전트, 환경, 상태, 행동, 보상의 관계 파악
2. **Q-Learning 알고리즘 체험**: Q-테이블 업데이트 과정과 가치 함수의 역할 학습
3. **탐험 vs 활용 딜레마 관찰**: ε-greedy 전략에서 탐험률 변화의 효과 관찰
4. **하이퍼파라미터의 영향 탐구**: 학습률, 할인율, 보상 구조 변경에 따른 학습 성능 변화 실험
5. **학습 과정의 시각적 분석**: 보상 추이, 점수 추이, Q-값 히트맵을 통한 학습 진행 해석

### 1.3 주요 특징

| 특징 | 설명 |
|------|------|
| 3가지 모드 | AI 학습, 직접 플레이(방향키), AI 시연 |
| 실시간 학습 시각화 | 뱀의 이동, 보상 획득, 사망을 실시간 관찰 |
| 학습 성적표 | 5단계 레벨 시스템(🥚→🐣→🐥→🐔→🦅) + 구간별 추이 |
| 속도 조절 | 1~100 FPS (60+ 터보 모드) |
| 보상 커스터마이징 | 먹이/충돌/접근/멀어짐 보상 개별 조절 |
| 차트 분석 | 보상 추이, 점수 추이, Q-값 히트맵 |
| 이벤트 시스템 | 먹이 획득(🍎), 사망(💀) 시 일시정지 + 시각 피드백 |
| 학습 로그 | 에피소드별 점수, 보상, 마일스톤 기록 |

## 4. 기능 상세

### 4.1 Q-Learning 알고리즘

**Q-값 업데이트 공식:**
```
Q(s, a) ← Q(s, a) + α × [r + γ × max Q(s', a') - Q(s, a)]
```

- `s`: 현재 상태, `a`: 선택한 행동
- `r`: 즉시 보상, `s'`: 다음 상태
- `α`: 학습률, `γ`: 할인율

**상태 표현 (8차원):**
뱀의 상태는 8개의 특성으로 표현됩니다:
- 4방향 위험 여부: 상/하/좌/우로 한 칸 이동 시 벽 또는 몸통 충돌 여부 (0 또는 1)
- 먹이 상대 방향: X축(좌:-1, 동일:0, 우:+1), Y축(상:-1, 동일:0, 하:+1)

**행동 공간:** 4가지 (상/하/좌/우)

**행동 선택 (ε-greedy):**
- 확률 ε: 무작위 행동 (탐험)
- 확률 1-ε: Q-값이 가장 높은 행동 (활용)
- ε는 매 에피소드마다 감소율을 곱하여 점차 줄어듦

### 4.2 보상 체계

| 이벤트 | 기본 보상 | 설명 |
|--------|:--------:|------|
| 🍎 먹이 획득 | +10 | 뱀이 먹이를 먹으면 길이 증가 + 점수 |
| 💀 벽/몸통 충돌 | -10 | 게임 오버 (에피소드 종료) |
| 먹이에 접근 | +1 | 맨해튼 거리가 줄어들면 |
| 먹이에서 멀어짐 | -1 | 맨해튼 거리가 늘어나면 |
| 최대 스텝 초과 | 사망 처리 | 200 + 점수×20 스텝 초과 시 |

### 4.3 게임 메커니즘

| 항목 | 설명 |
|------|------|
| 초기 길이 | 3칸 (격자 중앙에서 우측 방향) |
| 먹이 위치 | 빈 칸에 무작위 배치 |
| 사망 조건 | 벽 충돌, 자기 몸통 충돌, 최대 스텝 초과 |
| 에피소드 | 게임 시작~사망까지 1회 |
| 점수 | 먹이 획득 횟수 |
| 최대 스텝 | 200 + (현재 점수 × 20) |

### 4.4 속도 시스템

| FPS 범위 | 모드 | 동작 |
|:---:|------|------|
| 1~59 | 일반 | 초당 FPS 프레임 렌더링 |
| 60~100 | 터보 | 프레임당 복수 스텝 실행 (화면은 60fps 고정) |

터보 모드에서의 프레임당 스텝 수: `ceil((FPS - 60) / 5) + 1`

### 4.5 학습 레벨 시스템

최근 50 에피소드의 평균 점수 기준으로 레벨이 결정됩니다:

| 레벨 | 이모지 | 이름 | 기준 점수 | 색상 | 설명 |
|:---:|:---:|------|:---:|:---:|------|
| 0 | 🥚 | 미학습 | 0 | #666 | 학습 시작 전 |
| 1 | 🐣 | 방황 중 | 0.1+ | #f44 | 벽을 피하는 법을 배우는 중 |
| 2 | 🐥 | 눈치 있음 | 0.5+ | #f80 | 먹이 방향을 인식하기 시작 |
| 3 | 🐔 | 사냥꾼 | 2.0+ | #fc0 | 안정적으로 먹이를 찾아감 |
| 4 | 🦅 | 마스터 | 5.0+ | #0f8 | 효율적 경로로 빠르게 사냥 |

### 4.6 이벤트 시스템

먹이 획득/사망 시 게임을 일시정지하고 시각적 피드백을 표시합니다:

**성공 이벤트 (🍎):**
- 먹이 위치에 초록색 원 + 점수 표시
- 기본 700ms 정지 후 자동 재개

**실패 이벤트 (💀):**
- 뱀 머리 위치에 빨간색 원 + 사망 표시
- 기본 900ms 정지 후 다음 에피소드 시작

> 터보 모드(60+ FPS)에서는 이벤트 표시가 자동 생략되어 학습 속도를 유지합니다.

### 4.7 학습 성적표

**비교 카드 (4개):**
- 최근 50회 평균점수: 이전 50회 대비 변화량(△) 표시
- 최근 50회 평균보상: 이전 50회 대비 변화량(△) 표시
- 최근 50회 평균생존(스텝): 이전 50회 대비 변화량(△) 표시
- Q-테이블 크기: 학습된 상태-행동 쌍의 수

**구간별 추이 막대:**
50 에피소드 단위로 평균 점수를 막대 그래프로 표시하여 학습 진행을 한눈에 파악.

## 6. 사용 워크플로우

### 6.1 기본 학습 워크플로우

```
① 파라미터 확인 → ② 학습 시작 → ③ 속도 조절 → ④ 성장 관찰 → ⑤ AI 시연
```

### 6.2 비교 실험 워크플로우

```
실험 A: 학습률 비교
① α=0.01로 200 에피소드 학습 → 결과 기록 → 🔄 초기화
② α=0.10으로 200 에피소드 학습 → 결과 비교

실험 B: 보상 구조 비교
① 기본 보상으로 학습 → 결과 기록 → 🔄 초기화
② 접근 보상=0, 멀어짐 벌점=0으로 학습 → 결과 비교

실험 C: 맵 크기 비교
① 10×10 격자에서 학습 → 결과 기록 → 🔄 초기화
② 20×20 격자에서 학습 → 결과 비교
```

| 6 | 터보 모드에서 이벤트 안 보임 | 정상 동작 (터보 시 자동 생략) | 속도를 30 이하로 낮춰 관찰 |
| 7 | 차트에 데이터 없음 | 에피소드 0회 | 학습을 시작해야 데이터 생성 |
| 8 | 레벨이 🦅까지 안 올라감 | 큰 그리드 난이도 | 그리드 줄이기 또는 500+ 에피소드 학습 |
| 9 | ε가 0에 가까운데 탐험 행동 | ε 최솟값 설정 | 정상: ε_min > 0이면 항상 소량의 탐험 유지 |
| 10 | Q-값 히트맵이 모두 0 | 학습 미시작 | 학습 시작 후 관찰 |

## 9. FAQ

**Q1. 왜 처음에 뱀이 무작위로 움직이나요?**
A1. 초기 ε=1.0이므로 100% 무작위 행동(탐험)을 합니다. 다양한 상황을 경험해야 Q-테이블에 정보가 쌓이므로, 초기의 무작위 탐험은 학습에 필수적입니다.

**Q2. ε 감소율은 어떤 효과가 있나요?**
A2. 감소율이 0.995이면 매 에피소드마다 ε에 0.995를 곱합니다. 약 460 에피소드 후 ε ≈ 0.1이 됩니다. 감소율이 높을수록(0.999에 가까울수록) 천천히 탐험을 줄이고, 낮을수록(0.9에 가까울수록) 빨리 활용 위주로 전환합니다.

**Q3. 맵 크기를 바꾸면 왜 학습이 리셋되나요?**
A3. 그리드 크기가 달라지면 상태 공간이 변하므로 기존 Q-테이블이 유효하지 않습니다. 새로운 환경에서 처음부터 학습해야 합니다.

**Q4. "접근 보상"과 "멀어짐 벌점"은 왜 필요한가요?**
A4. 먹이를 먹기까지 많은 스텝이 필요한데, 먹이 보상(+10)만 있으면 그 사이 행동에 대한 피드백이 없습니다. 접근/멀어짐 보상은 먹이를 향한 "방향 힌트"를 제공하여 학습을 가속합니다.

**Q5. Q-테이블 크기가 계속 커지나요?**
A5. 새로운 (상태, 행동) 조합을 만날 때마다 Q-테이블에 항목이 추가됩니다. 상태 공간은 유한하므로(위험×먹이방향 조합) 일정 크기 이상으로는 거의 증가하지 않습니다.
