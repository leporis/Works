## 1. 소프트웨어 개요

### 1.1 목적

RL 틱택토 시뮬레이터는 Q-Learning 알고리즘을 활용하여 두 AI 에이전트(X, O)가 틱택토를 스스로 학습하는 과정을 시각적으로 관찰할 수 있는 교육용 웹 애플리케이션입니다. 강화학습의 핵심인 "시행착오를 통한 전략 발견" 과정을 3×3 보드 게임이라는 친숙한 맥락에서 직관적으로 이해할 수 있습니다.

### 1.2 교육 목표

1. **Q-Learning 알고리즘 이해**: Q-테이블에 상태-행동 가치를 기록하고 업데이트하는 원리 학습
2. **탐험 vs 활용 딜레마 관찰**: ε-greedy 전략에서 탐험률(ε) 감소에 따른 학습 전략 변화 관찰
3. **자기대국을 통한 발전**: 두 에이전트가 상호 대국하며 함께 성장하는 과정 체험
4. **대칭성 정규화 개념**: 회전·반사 변환으로 학습 효율을 높이는 기법 이해
5. **하이퍼파라미터 실험**: 학습률·할인율·ε 감소율 변경에 따른 학습 성능 비교
6. **사람 vs AI 대결**: 학습된 AI와 직접 대국하여 AI의 성장을 체감

### 1.3 주요 특징

| 특징 | 설명 |
|------|------|
| 4가지 모드 | 대시보드(학습), AI 대전(관전), 사람 대전(대국), 학습 방식 선택 |
| Q-Learning 구현 | 에피소드 단위 역방향 업데이트 + 대칭성 정규화(8배 효율) |
| 3가지 학습 방식 | 자기대국, X 단독, O 단독 |

| 5단계 성장 시스템 | 🌱초기 → 🔍탐색 → 📚학습 → 🎯응용 → 🏆숙련 |
| 등급 체계 | F → D → C → B → A → S (강도 점수 기준) |

| 파라미터 동기화 | X·O 파라미터 동기/별도 전환 |
| 데이터 관리 | 브라우저 저장, JSON 내보내기/가져오기, 학습 보고서 |

## 4. 기능 상세

### 4.1 Q-Learning 알고리즘

**학습 원리:**

Q-Learning은 에이전트가 각 상태(보드 배치)에서 각 행동(착수 위치)의 가치(Q-값)를 시행착오를 통해 학습하는 알고리즘입니다.

**업데이트 방식:**

이 시뮬레이터는 에피소드 종료 후 역방향(마지막 수→첫 수)으로 Q-값을 일괄 업데이트합니다. 마지막 수에는 최종 보상(승리/패배/무승부)이, 이전 수에는 할인된 미래 보상 + Step 패널티가 적용됩니다.

```
Q[상태][행동] += α × (목표보상 - Q[상태][행동])
목표보상 = 최종보상 (마지막 수) 또는 step패널티 + γ × 다음수목표보상 (이전 수)
```

**ε-greedy 행동 선택:**

| 상황 | 확률 | 행동 |
|------|:----:|------|
| 탐험 | ε | 가능한 수 중 무작위 선택 |
| 활용 | 1-ε | Q-값이 가장 높은 수 선택 |

ε은 매 배치(학습 묶음)마다 `ε × epsilonDecay`로 감소하여, 초기에는 다양한 수를 탐험하고 점차 학습된 최적 수를 활용하도록 전환됩니다.

### 4.2 대칭성 정규화

틱택토 보드는 회전(0°, 90°, 180°, 270°)과 반사(좌우, 상하, 대각선 2개)를 합쳐 8가지 대칭 변환이 가능합니다. 이 시뮬레이터는 대칭 변환을 적용하여 동일한 전략적 의미를 가진 보드를 하나의 정규 상태(canonical state)로 통합합니다.

| 항목 | 설명 |
|------|------|
| 변환 수 | 8가지 (4회전 × 2반사) |
| 정규화 방법 | 8가지 변환 중 사전순으로 가장 작은 문자열을 정규 상태로 선택 |
| 효과 | Q-테이블 크기 최대 8배 축소, 학습 속도 향상 |
| 역변환 | Q-값 조회 시 정규 좌표 → 원래 보드 좌표로 역변환 |

### 4.3 보상 체계

| 상황 | 보상값 | 설계 의도 |
|------|:------:|----------|
| 승리 | +1.0 | 이기는 것이 최고 목표 |
| 패배 | -1.0 | 지는 것은 최대 패널티 |
| 무승부 | +0.3 | 무승부도 긍정 평가 (틱택토는 최적 시 무승부) |
| 매 수(Step) | -0.01 | 불필요한 수 억제, 빠른 결판 유도 |

### 4.4 학습 방식

| 방식 | 모드 키 | X 에이전트 | O 에이전트 | 용도 |
|------|:-------:|:----------:|:----------:|------|
| 🔄 자기대국 | selfplay | Q-Learning | Q-Learning | 양쪽 상호 발전 (기본) |
| 🔵 X 단독 | solo-x | Q-Learning | 랜덤 | 선공 전략 집중 훈련 |
| 🔴 O 단독 | solo-o | 랜덤 | Q-Learning | 후공 전략 집중 훈련 |

### 4.5 학습 단계 (STAGES)

| 단계 | 아이콘 | 에피소드 범위 | 색상 | 특징 |
|------|:------:|:------------:|------|------|
| 초기 | 🌱 | 0 ~ 500 | 빨강 | ε=100%, 완전 랜덤, 탐색 시작 전 |
| 탐색 | 🔍 | 500 ~ 5,000 | 주황 | ε 감소 시작, 경험 축적 |
| 학습 | 📚 | 5,000 ~ 20,000 | 노랑 | 전략 형성, 승률 안정화 |
| 응용 | 🎯 | 20,000 ~ 50,000 | 초록 | ε→1%, 최적 근접 |
| 숙련 | 🏆 | 50,000 ~ 100,000 | 시안 | 완벽 플레이, 무승부 지배 |

### 4.6 등급 시스템

등급은 강도(Strength) 점수에 기반합니다. 강도는 랜덤 상대와 200판 대전하여 `(승수 + 무승부수 × 0.5) / 200 × 100`으로 산출됩니다.

| 등급 | 강도 범위 | 색상 | 설명 |
|:----:|:---------:|------|------|
| S | 95+ | 🟡 금색 | 완벽! |
| A | 85~94 | 🟢 초록 | 훌륭함 |
| B | 75~84 | 🔵 파랑 | 좋음 |
| C | 60~74 | 🟡 노랑 | 보통 |
| D | 40~59 | 🟠 주황 | 부족 |
| F | 0~39 | 🔴 빨강 | 학습 필요 |

### 4.7 차트 시스템

6개 차트 모두 동일한 구조를 사용합니다. 최근 60개 데이터 포인트를 막대 그래프로 표시하며, 추세선(선형 회귀 기울기)에 따른 상태 표시를 포함합니다.

| 차트 | 데이터 | 좋은 방향 |
|------|--------|:---------:|
| 🔵 X 승률 | 최근 50판 X 승리 비율 | ↗ 상승 |
| 🔴 O 승률 | 최근 50판 O 승리 비율 | ↗ 상승 |
| 🤝 무승부율 | 최근 50판 무승부 비율 | ↗ 상승 |
| 🎲 탐험률 ε | 현재 ε × 100 | ↘ 하강 |
| ⭐ X 강도 | 랜덤 상대 대비 강도 점수 | ↗ 상승 |
| 📚 Q-상태 수 | Q-테이블 항목 수 (대칭 압축) | ↗ 상승 |

**추세 상태 표시:**

| 상태 | 아이콘 | 조건 |
|------|:------:|------|
| 좋음! | ✅ | 기울기가 좋은 방향으로 0.1 이상 |
| 주의 | ⚠️ | 기울기가 나쁜 방향으로 0.1 이상 |
| 안정 | ➡️ | 기울기 절대값 0.1 미만 |
| 수집중 | ⏳ | 데이터 30개 미만 |

### 4.8 Q-값 히트맵

사람 대전 모드와 AI 대전 모드에서 AI의 판단 근거를 보드 위에 시각화합니다.

| 요소 | 표시 방법 |
|------|----------|
| 빈 칸의 Q-값 | 수치(소수점 2자리) + 배경 색상 |
| 최적 수 | 🔸 마크 + 강조 테두리 |
| 높은 Q-값 | 🟢 초록 배경 (유리한 수) |
| 낮은 Q-값 | 🔴 빨간 배경 (불리한 수) |

### 4.9 학습 인사이트 패널

대시보드 모드에서 X 에이전트의 첫수(오프닝) 선호도를 실시간 분석합니다.

| 범주 | 위치 | 아이콘 |
|------|------|:------:|
| 중앙 | [4] | 🎯 |
| 모서리 | [0, 2, 6, 8] | 📐 |
| 변 | [1, 3, 5, 7] | ➖ |

각 범주의 평균 Q-값을 수평 막대 그래프로 표시합니다. 학습이 진행되면 일반적으로 중앙 또는 모서리 선호가 높아지는 것을 관찰할 수 있습니다.

| ε 초기값 | 1.0 | (고정) | - |
| ε 최솟값 | 0.01 | (고정) | - |

### 5.3 보상 상수

| 상수명 | 값 |
|--------|:---:|
| REWARDS.win | +1.0 |
| REWARDS.lose | -1.0 |
| REWARDS.draw | +0.3 |
| REWARDS.step | -0.01 |

### 7.2 주요 클래스 및 함수

| 이름 | 유형 | 역할 |
|------|------|------|
| `QAgent` | 클래스 | Q-테이블 관리, 행동 선택, 학습 업데이트 |

| `trainEpisode()` | 함수 | 1판 학습 실행 (보드 초기화 → 교대 착수 → 결과 업데이트) |

### 7.3 학습 루프 구조

학습 시작 시 `setInterval`로 반복 실행되며, 각 루프에서 배치 크기만큼 에피소드를 실행합니다. 속도 슬라이더 값이 인터벌 주기(ms)를 결정합니다. 5,000판마다 자동 저장되며, 주기적으로 강도 측정이 수행됩니다.

## 8. 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| 학습이 시작되지 않는다 | 모드가 idle 상태 | **▶️ 학습 시작** 클릭 |
| 학습 속도가 느리다 | 속도 값이 높음 | 속도 슬라이더를 **1ms**로 설정 |
| 승률이 불규칙하다 | 학습 초기 단계 | 5,000판 이상 학습 (ε 감소 필요) |
| 사람 대전에서 AI가 약하다 | 학습 부족 | 20,000판 이상 학습 후 재도전 |
| 무승부만 나온다 | 최적 학습 완료! | 정상 — 최적 틱택토는 항상 무승부 |
| AI가 나쁜 수를 둔다 | ε이 아직 높음 | ε이 5% 이하까지 학습 지속 |
| ε이 안 줄어든다 | 감소율이 1에 가까움 | ε 감소율을 0.999로 낮추기 |
| X만 계속 이긴다 | 단독 학습 모드 확인 | 🔄 자기대국으로 변경 |
| 저장이 안 된다 | 브라우저 저장소 제한 | 📤 파일 내보내기 사용 |
| 보고서 버튼이 비활성 | 학습 100판 미만 | 100판 이상 학습 후 시도 |
| 화면 레이아웃이 깨진다 | 화면이 너무 작음 | 1280px 이상 해상도에서 사용 |
| 테마 색상이 이상하다 | 테마 전환 필요 | ☀️/🌙 버튼으로 전환 |

## 9. FAQ

**Q: 틱택토에서 AI가 완벽해지면 어떻게 되나요?**
A: 양쪽 모두 최적 플레이하면 항상 무승부입니다. 무승부율이 90% 이상이면 거의 최적에 도달한 것입니다.

**Q: 자기대국과 단독 학습의 차이는?**
A: 자기대국은 X와 O 모두 Q-Learning으로 학습하여 상호 발전합니다. 단독 학습은 한쪽만 학습하고 상대는 랜덤으로 두어 특정 역할의 전략을 집중 훈련합니다.

**Q: 대칭성 정규화를 쓰면 무엇이 좋은가요?**
A: 회전·반사로 동일한 전략적 의미의 보드를 하나로 통합하여 Q-테이블 크기를 최대 8배 줄이고 학습 속도를 높입니다.

**Q: Q-값 히트맵에서 숫자가 의미하는 것은?**
A: 각 빈 칸의 Q-값으로, AI가 해당 위치에 착수했을 때 기대하는 미래 보상입니다. 양수일수록 유리하고 음수일수록 불리합니다.

**Q: 파라미터 동기화 모드의 차이는?**
A: 🔗동기는 X·O의 α, γ, ε 감소율을 동일하게 유지합니다. 🔓별도는 각각 다른 값을 설정할 수 있어 비대칭 실험이 가능합니다.

**Q: 학습 데이터를 다른 컴퓨터로 옮길 수 있나요?**
A: 📤 파일 내보내기로 JSON을 다운로드하고, 다른 컴퓨터에서 📥 파일 가져오기로 불러올 수 있습니다.
