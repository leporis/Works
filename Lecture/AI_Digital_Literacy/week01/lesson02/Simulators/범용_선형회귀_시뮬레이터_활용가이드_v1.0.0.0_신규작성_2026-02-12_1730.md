## 1. 교육 활용 개요

### 1.1 교육적 가치

이 시뮬레이터는 다음 세 가지 교육적 강점을 가집니다.

**① 완전한 ML 파이프라인 체험**: 데이터 로딩 → 변수 선택 → 학습 → 평가 → 예측의 전 과정을 한 화면에서 체험합니다. "머신러닝이란 무엇인가"의 전체 그림을 이해하는 데 적합합니다.

**② 학습률의 직관적 이해**: 학습률을 0.001에서 0.1까지 조절하며 "너무 작으면 느리고, 너무 크면 발산한다"는 핵심 원리를 눈으로 확인합니다.

**③ 실제 데이터 활용**: 23개 공개 데이터셋이 내장되어 있어, 교과서의 인위적 예시가 아닌 실제 데이터로 학습합니다.

### 1.2 대상 학습자

| 대상 | 적합도 | 활용 방법 |
|------|:-----:|----------|
| 초등학생 (5~6학년) | ⭐⭐ | 공부시간·성적 데이터로 "예측" 개념만 체험 |
| 예비교사 (대학생) | ⭐⭐⭐ | 전체 파이프라인 + 학습률 실험 + 다중변수 |
| 현직교사 연수 | ⭐⭐⭐ | AI 교구 활용법 체험 |
| 통계 교육 | ⭐⭐⭐ | 앤스컴 데이터로 시각화의 중요성 학습 |

## 2. 교육과정 연계

### 2.1 성취기준

| 성취기준 코드 | 내용 | 이 시뮬레이터의 역할 |
|-------------|------|-------------------|
| [6실05-04] | 인공지능이 만들어지는 과정을 체험한다 | 데이터→학습→예측 전 과정 직접 조작 |
| [6실05-05] | AI 기술을 활용하여 실생활 문제 해결 방법을 탐색한다 | 23개 실제 데이터로 문제 해결 체험 |

### 2.2 학습 목표 매핑

| 기능 | 학습 목표 |
|------|----------|
| 데이터 로드 + 산점도 | 데이터의 경향을 시각적으로 파악할 수 있다 |
| 학습 실행 + 회귀선 | "데이터에서 규칙을 찾는 것"이 머신러닝임을 이해한다 |
| 학습률 조절 | 학습의 속도와 안정성의 관계를 이해한다 |

| 추론 모드 | 학습된 모델로 새로운 값을 예측하는 과정을 체험한다 |
| 앤스컴 데이터 | 통계량만으로는 데이터를 판단할 수 없음을 이해한다 |

## 3. 수업 시나리오

### 3.1 시나리오 A: 선형회귀 입문 (15분 시연)

**상황**: ML 수업에서 지도학습(예측) 개념 시연

| 단계 | 시간 | 교수 활동 | 사용 기능 |
|:---:|:---:|----------|----------|
| 동기유발 | 2분 | "경력이 많을수록 연봉이 높을까?" | — |
| 데이터 | 3분 | 💼 경력과 연봉 로드 → 산점도 관찰 | 데이터셋 로드 |
| 학습 | 3분 | ⚡ 학습 실행 → 직선이 나타남 | 학습 실행 |
| 해석 | 3분 | "w=200이면 경력 1년=연봉 200만원" | 결과 패널 |
| 예측 | 2분 | 추론 모드 → "경력 15년이면?" | 추론 슬라이더 |
| 정리 | 2분 | "데이터에서 규칙을 찾아 예측하는 것이 ML" | — |

### 3.2 시나리오 B: 학습률 실험 수업 (25분)

**상황**: 경사하강법의 학습률 역할을 실험으로 이해

| 단계 | 시간 | 교수 활동 | 학생 활동 |
|:---:|:---:|----------|----------|
| 도입 | 3분 | "산을 내려가는 보폭" 비유 | 경청 |
| 실험 1 | 5분 | 학습률=0.001로 학습 | 손실 그래프 관찰 → 활동지 기록 |
| 실험 2 | 5분 | 학습률=0.01로 학습 | 비교 기록 |
| 실험 3 | 5분 | 학습률=0.1로 학습 | "발산" 현상 관찰 |
| 토론 | 5분 | "왜 중간 값이 최적일까?" | 모둠 토론 |
| 정리 | 2분 | 학습률의 균형 원칙 정리 | 결론 기록 |

### 3.3 시나리오 C: 앤스컴 데이터 탐구 (20분)

**상황**: "통계량만으로는 부족하다" — 시각화의 중요성

| 단계 | 시간 | 교수 활동 |
|:---:|:---:|----------|
| 1 | 3분 | 앤스컴 1 로드 → 학습 → R² 기록 |
| 2 | 3분 | 앤스컴 2 로드 → 학습 → R² 기록 (거의 같음!) |
| 3 | 3분 | 앤스컴 3, 4 반복 |
| 4 | 5분 | "R²가 같은데 산점도는 완전히 다르다!" 토론 |
| 5 | 3분 | 오차선 표시 → 각 데이터셋의 오차 패턴 비교 |
| 6 | 3분 | "숫자만 보지 말고 그래프를 봐야 한다" 정리 |

## 5. 학습 질문 설계

### 5.1 기능별 발문

| 기능 | 발문 |
|------|------|
| 산점도 관찰 | "점들이 어떤 방향으로 모여 있나요? 올라가나요 내려가나요?" |
| 학습 실행 | "빨간 직선은 어떻게 나타났나요? 누가 그렸나요?" |
| 회귀식 해석 | "w=200이면 X가 1 늘 때 Y는 얼마나 변하나요?" |
| 손실 그래프 | "그래프가 내려가다 평평해지는 건 무슨 의미인가요?" |
| 학습 과정 재생 | "직선이 처음엔 엉뚱한 곳에 있다가 점점 맞아가는 이유는?" |
| 오차선 | "빨간 수직선이 긴 점은 어떤 점인가요?" |
| 추론 | "범위 밖의 값(외삽)을 예측하면 믿을 수 있을까요?" |
| 재분할 | "왜 버튼을 누를 때마다 결과가 달라지나요?" |

### 5.2 블룸 수준별 질문

| 수준 | 질문 |
|------|------|
| 기억 | Y = wX + b에서 w는 무엇이고 b는 무엇인가요? |
| 이해 | 학습률이 커지면 왜 발산하나요? 비유로 설명해보세요 |
| 적용 | 우리 반 키와 몸무게 데이터로 예측 모델을 만들 수 있을까요? |
| 분석 | 앤스컴 4개 데이터의 R²가 같은데 패턴이 다른 이유는? |
| 평가 | R²=0.5인 모델을 실생활에서 써도 될까요? |
| 창조 | 선형회귀가 적합하지 않은 데이터 예시를 만들어보세요 |

## 6. 추천 데이터셋

### 6.1 수업 목적별 추천

| 수업 목적 | 추천 데이터셋 | 이유 |
|----------|-------------|------|
| **입문 시연** | 💼 경력과 연봉 | 직관적 관계, 적은 데이터(N=30) |
| **학생 공감** | 📚 공부시간·성적 | 학생 경험과 연결, r 높음 |
| **높은 상관** | 🚕 뉴욕 택시 | r=0.95, 깔끔한 선형 관계 |
| **시각화 교육** | 📊 앤스컴 1~4 | 같은 통계량, 다른 패턴 |
| **실제 데이터** | 🌸 붓꽃 | ML 분야 대표 데이터 |
| **다중 변수** | 🚗 자동차 연비 | 마력·무게 등 다중 요인 |
| **음의 상관** | 🚗 자동차 연비 | 무게↑→연비↓ (r=-0.83) |
| **약한 상관** | 🏥 당뇨병 | r=0.59, 현실적 한계 체험 |
