## 1. 프로젝트 개요

### 1.1 목적

「AI 사과 농장 이야기」는 **5가지 머신러닝 알고리즘**을 사과 농장이라는 친숙한 이야기 속에서 자연스럽게 이해할 수 있도록 만든 **교육용 TTS 애니메이션**입니다. 농부 김사과와 AI 로봇 알파의 대화를 통해 각 알고리즘이 실제 농장 문제를 어떻게 해결하는지 보여줍니다.

### 1.2 교육 목표

| 목표 | 설명 |
|------|------|
| **ML 알고리즘 개념 이해** | CNN, KNN, K-평균, 선형회귀, 결정트리의 핵심 원리 파악 |
| **지도학습 vs 비지도학습 구분** | 정답이 필요한 학습과 필요 없는 학습의 차이 인식 |
| **문제-알고리즘 매칭** | 문제 유형에 따라 적절한 AI 기술을 선택하는 사고력 향상 |
| **AI에 대한 친근감 형성** | 이야기를 통해 AI를 어렵지 않은 도구로 인식 |

### 1.3 대상

예비 초등교사(교육대학교 학생), AI 교육에 관심 있는 교사 및 일반인. 프로그래밍이나 수학적 배경 지식이 없어도 이해할 수 있도록 설계되었습니다.

### 1.4 실행 환경

| 항목 | 요구 사항 |
|------|----------|
| 브라우저 | Chrome(권장), Edge, Safari, Firefox |
| 인터넷 | 웹 폰트 로딩을 위해 초기 1회 필요 (이후 캐시 활용) |
| 스피커/이어폰 | TTS 음성 재생을 위해 권장 |
| 화면 크기 | 최소 768px 너비 권장 (모바일은 가로 모드) |
| 파일 형식 | 단일 HTML 파일 (`AI사과농장이야기_애니메이션_v1.5.0.0.html`) |

> ⚠️ **Chrome 브라우저**를 가장 권장합니다. Chrome의 TTS 엔진이 한국어 음성을 가장 자연스럽게 재생합니다.

## 2. 등장인물

이 애니메이션에는 3명의 캐릭터가 등장합니다. 각 캐릭터는 고유한 음성(TTS 설정)과 역할을 가지고 있습니다.

### 2.1 농부 김사과 👨‍🌾

| 항목 | 설명 |
|------|------|
| 역할 | 사과 농장 주인. 실제 농장 문제를 제시하고, 알파의 설명을 듣고 이해하는 **학습자 대리인** |
| 성격 | 호기심 많고 따뜻한 말투. "아이고", "오호", "아하!" 등 감탄사를 자주 사용 |
| 교육적 기능 | 학습자가 가질 수 있는 질문을 대신 던지고, 핵심 내용을 자기 말로 요약 |
| 음성 설정 | 속도 0.9배, 피치 0.85 (낮고 느린 어른 목소리) |
| 말풍선 색상 | 갈색 계열 (`#8D6E63` → `#6D4C41`) |

### 2.2 AI 로봇 알파 🤖

| 항목 | 설명 |
|------|------|
| 역할 | AI 기술 전문가. 각 장면에서 적절한 알고리즘을 소개하고 원리를 설명하는 **교사 대리인** |
| 성격 | 정확하고 친절한 말투. "~입니다", "~해요" 등 정중한 어미 사용 |
| 교육적 기능 | 알고리즘의 작동 원리를 쉬운 비유로 설명하고, 결과를 수치로 제시 |
| 음성 설정 | 속도 1.05배, 피치 1.4 (빠르고 높은 로봇 목소리) |
| 말풍선 색상 | 파란 계열 (`#2196F3` → `#1565C0`) |

### 2.3 내레이터 📖

| 항목 | 설명 |
|------|------|
| 역할 | 이야기의 흐름을 안내하는 해설자. 각 장의 시작과 마무리를 담당 |
| 성격 | 차분하고 중립적인 해설 톤 |
| 교육적 기능 | 장면 전환 안내, 전체 흐름 요약 |
| 음성 설정 | 속도 1.0배, 피치 1.0 (기본 음성) |
| 말풍선 색상 | 회색 계열 (기본) |

| 3 | 👨‍🌾 김사과 | 홍로, 부사, 아오리, 감홍... 생긴 게 다 비슷해서 구분이 안 돼. | - |

| 8 | 🤖 알파 | 이 사과와 가장 가까운 3개 사과가 모두 부사네요. 이 사과는 부사입니다! | 부사 사과 |

**교육 포인트:**

- **KNN의 핵심 원리**: "가장 비슷한 K개의 이웃을 찾아 다수결로 결정"
- **특성(Feature) 개념**: 당도, 산도, 무게 = 사과를 설명하는 측정값
- **K값의 의미**: K=3은 가장 가까운 3개를 참고한다는 뜻
- **지도학습 연속**: 1장의 CNN과 마찬가지로 "정답을 아는 데이터"로부터 학습

| 3 | 👨‍🌾 김사과 | 문제는 매년 사과 크기가 달라진다는 거야. 올해는 비가 많이 와서 사과가 전체적으로 크거든. | - |
| 4 | 👨‍🌾 김사과 | 작년 기준으로 나누면 올해는 전부 상품이 되어버려. 어떻게 하면 좋지? | - |
| 5 | 🤖 알파 | 이럴 때는 K평균 클러스터링을 사용합니다! | K-means 클러스터 |
| 6 | 🤖 알파 | 미리 정해진 기준이 아니라, 올해 사과들 중에서 자동으로 3개 그룹을 만들어요. | - |
| 7 | 🤖 알파 | 이건 비지도 학습이라서 정답 없이도 스스로 분류할 수 있어요. | - |
| 8 | 👨‍🌾 김사과 | 오! 정답을 안 알려줘도 스스로 나눈다고? 정말 신기하구나! | 중품 (중) |
| 9 | 🤖 알파 | 네! 그래서 매년 달라지는 데이터에도 유연하게 적응할 수 있습니다. | 하품 (소) |

**교육 포인트:**

- **비지도학습의 등장**: 1~2장은 지도학습, 3장에서 비지도학습으로 전환되는 극적 대비
- **K-평균 클러스터링의 원리**: "비슷한 것끼리 자동으로 모으기"
- **고정 기준 vs 상대 기준**: "작년 기준이 올해는 맞지 않는다" — 데이터 기반 유연한 분류의 필요성
- **김사과의 놀라움**: "정답 없이도 스스로 나눈다"라는 반응이 학습자의 감정을 대변

| AI 해결 방법 | 과거 판매 데이터로 품질-가격 관계를 학습하여 공식 도출 |
| 핵심 수치 | 가격 = 30 × 품질 + 1000원 (품질 1점 = 30원) |

**내레이션 스크립트:**

| # | 화자 | 대사 | 이미지 |
|:-:|:----:|------|:------:|
| 1 | 📖 내레이터 | 네 번째 이야기. 가격을 매깁니다. | - |
| 2 | 👨‍🌾 김사과 | 등급도 나눴으니 이제 가격을 정해야겠네. | - |
| 3 | 👨‍🌾 김사과 | 당도가 높고 무게가 나가면 비싸게 팔아야 하는데... 정확히 얼마로 해야 할까? | 품질 점수 |
| 4 | 🤖 알파 | 제가 선형 회귀로 도와드릴게요! | - |
| 5 | 🤖 알파 | 당도와 무게를 합쳐서 품질 점수를 만들고, 과거 판매 데이터로 가격과의 관계를 학습했어요. | - |
| 6 | 🤖 알파 | 분석 결과, 가격은 30 곱하기 품질 더하기 1000원 입니다! | 가격 공식 |
| 7 | 👨‍🌾 김사과 | 오호, 그러니까 품질이 1점 오르면 가격이 30원씩 오른다는 거지? | - |
| 8 | 🤖 알파 | 정확합니다! 이것도 지도 학습이에요. 과거 가격 정답을 보고 규칙을 찾아낸 거죠. | - |

**교육 포인트:**

- **선형회귀의 핵심**: "데이터에서 직선 관계(y = wx + b)를 찾아내는 것"
- **기울기(w)의 의미**: "품질 1점 = 가격 30원" — 기울기가 현실적 의미를 가진다는 점
- **지도학습으로 회귀**: 분류(1·2장)와 달리 연속적인 수치를 예측하는 "회귀" 개념 소개
- **품질 점수**: 당도 + 무게 → 복합 지표라는 특성 공학(Feature Engineering) 개념의 자연스러운 도입

### 5.3 속도 옵션

기본 제공되는 3가지 속도 옵션입니다. (내부적으로 7단계까지 지원하나 UI에는 3개만 표시)

| 옵션 | 배속 | 권장 용도 |
|------|:----:|----------|
| 느리게 | 0.85x | 내용을 꼼꼼히 이해하고 싶을 때, 필기하면서 볼 때 |
| **보통** | **1.0x** | **기본 감상** (기본 선택) |
| 빠르게 | 1.15x | 복습하거나 전체 흐름만 빠르게 파악할 때 |

### 5.4 음소거 모드

음소거 시에도 애니메이션은 계속 진행됩니다. TTS 음성 대신 텍스트 길이에 기반한 대기 시간이 자동 계산됩니다.

| 설정 | 계산 방식 |
|------|----------|
| 최소 대기 시간 | 2,000ms (2초) |
| 글자당 대기 시간 | 80ms/글자 |
| 실제 대기 시간 | max(2000, 글자수 × 80) ms |

> 💡 음소거 모드는 **스피커 없는 환경**(도서관 등)에서 자막만 보며 감상하기에 적합합니다.

## 6. TTS (음성 합성) 시스템

### 6.1 개요

이 애니메이션은 브라우저 내장 **Web Speech API**를 사용하여 한국어 음성을 합성합니다. 별도의 음성 파일이나 서버가 필요 없으며, 브라우저의 TTS 엔진에 의존합니다.

### 6.2 캐릭터별 음성 설정

| 캐릭터 | 속도 (rate) | 높낮이 (pitch) | 음성 특징 |
|--------|:----------:|:-------------:|----------|
| 📖 내레이터 | 1.0 | 1.0 | 기본 음성, 차분한 해설 톤 |
| 👨‍🌾 김사과 | 0.9 | 0.85 | 느리고 낮은 음, 따뜻한 어른 |
| 🤖 알파 | 1.05 | 1.4 | 빠르고 높은 음, 로봇 느낌 |

### 6.3 TTS 상태 표시

| 상태 | 점 색상 | 라벨 | 의미 |
|------|:------:|------|------|
| 미지원 | 🔴 | TTS 미지원 | 브라우저가 Web Speech API를 지원하지 않음 |
| 로딩중 | 🟡 | 로딩중... | 한국어 음성 엔진을 불러오는 중 |
| 준비됨 | 🟢 | 준비됨 | 재생 가능 상태 |
| 재생중 | 🔵 | 재생중 | 현재 음성 출력 중 |

### 6.4 브라우저별 호환성

| 브라우저 | 한국어 TTS | 음성 품질 | 비고 |
|---------|:----------:|:--------:|------|
| **Chrome** | ✅ | ⭐⭐⭐ | **최적**, Google TTS 엔진 사용 |
| Edge | ✅ | ⭐⭐⭐ | Microsoft TTS 엔진, 품질 양호 |
| Safari | ✅ | ⭐⭐ | macOS/iOS 기본 한국어 음성 |
| Firefox | ⚠️ | ⭐ | OS 의존적, 한국어 음성이 없을 수 있음 |

> ⚠️ **Chrome 5초 제한 대응**: Chrome에서는 TTS 발화가 15초 이상 지속되면 자동 중단되는 버그가 있습니다. 이 애니메이션은 5초 간격으로 TTS를 갱신(resume)하여 이 문제를 자동 우회합니다.

## 7. 장면 전환 효과

### 7.1 시각 전환

장면이 바뀔 때 0.8초 동안 현재 장면이 페이드 아웃되고 새 장면이 페이드 인됩니다. 새 장면의 제목은 아래에서 위로 슬라이드하며 나타나고(fadeInUp), 부제와 일러스트도 순차적으로 등장합니다.

| 요소 | 등장 시점 | 애니메이션 |
|------|:--------:|----------|
| 장면 배경 | 0ms | 페이드 인 (0.8초) |
| 제목 | 0ms | 아래→위 슬라이드 (0.8초) |
| 부제 | 300ms | 아래→위 슬라이드 (0.8초) |
| 일러스트 | 500ms | 페이드 인 (0.8초) |
| 캐릭터 | 700ms | 아래→위 슬라이드 (0.8초) |

### 7.2 전환 효과음

장면 전환 시 C5→E5→G5 세 음의 **차임벨 효과음**이 재생됩니다 (Web Audio API 사용). 음소거 상태에서도 전환 효과음은 별도로 제어됩니다.

| 음계 | 주파수 | 시작 시점 | 지속 시간 |
|:----:|:------:|:--------:|:--------:|
| C5 | 523.25 Hz | 0초 | 0.4초 |
| E5 | 659.25 Hz | 0.3초 | 0.4초 |
| G5 | 783.99 Hz | 0.6초 | 0.6초 |

전환 효과음의 전체 지속 시간은 2.5초이며, 이 시간 동안 내레이션은 시작되지 않습니다.
