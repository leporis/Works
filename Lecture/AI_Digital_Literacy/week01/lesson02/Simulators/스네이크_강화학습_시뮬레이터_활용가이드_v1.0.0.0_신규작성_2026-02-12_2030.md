## 1. 교육 활용 개요

### 1.1 교육적 가치

**① 강화학습의 본질적 체험:**
"보상을 통해 행동을 개선한다"는 강화학습의 핵심을 게임이라는 친숙한 맥락에서 직관적으로 경험합니다. 뱀이 무작위로 움직이다가 점점 먹이를 잘 찾게 되는 과정은 "시행착오를 통한 학습"의 본질을 보여줍니다.

**② 탐험-활용 딜레마의 시각화:**
ε(탐험률)이 1.0에서 0.01로 줄어가는 과정에서 무작위 탐험이 점차 학습된 정책으로 대체되는 모습을 실시간으로 관찰할 수 있습니다. 이는 새로운 것을 시도하는 것과 이미 아는 좋은 방법을 활용하는 것 사이의 균형이라는 보편적 교훈과 연결됩니다.

**③ 보상 설계의 중요성 탐구:**
보상 구조를 직접 변경하고 학습 결과를 비교함으로써, "어떤 행동에 칭찬/벌을 줄 것인가"가 학습 결과에 결정적 영향을 미친다는 점을 깨달을 수 있습니다.

### 1.2 대상 학습자

| 대상 | 적합도 | 활용 방법 |
|------|:------:|----------|
| 초등 5~6학년 | ⭐⭐⭐ | "강아지 훈련" 비유 + 게임으로 강화학습 개념 체험 |
| 예비교사 | ⭐⭐⭐ | 강화학습 원리 이해 + 파라미터 실험 + 수업 시나리오 설계 |
| 현직교사 연수 | ⭐⭐⭐ | 강화학습의 교육적 비유 개발 + 수업 적용 |
| 통계·데이터 교육 | ⭐⭐ | Q-Learning 알고리즘 이해 + 하이퍼파라미터 실험 |

### 2.2 학습 목표 매핑

| 시뮬레이터 기능 | 학습 목표 |

| 학습 성적표 관찰 | 학습 진행의 정량적 분석 방법 학습 |
| Q-값 히트맵 | AI의 "판단 기준" 시각화 체험 |

## 3. 수업 시나리오

### 시나리오 A: 강화학습 입문 — "AI에게 게임을 가르쳐볼까요?" (20분)

**목표:** 강화학습의 기본 개념(보상, 벌점, 학습)을 게임 맥락에서 이해한다.

| 시간 | 활동 | 교사 발문 |
|:----:|------|----------|
| 0~3분 | 동기유발: "뱀 게임을 AI에게 가르칠 수 있을까?" | "컴퓨터가 스스로 게임을 잘하게 되려면 뭐가 필요할까요?" |
| 3~5분 | 🎮 직접 플레이 체험 → 점수 기록 | "여러분의 점수는 몇 점인가요?" |
| 5~8분 | 🧠 AI 학습 시작 (속도 15fps) → 뱀 관찰 | "AI 뱀이 처음에 어떻게 움직이나요? 왜 그럴까요?" |
| 8~12분 | 속도를 터보(60+)로 올려 200 에피소드 학습 | "학습 성적표의 레벨이 어떻게 변했나요?" |
| 12~15분 | 속도를 15fps로 낮추어 AI 행동 관찰 | "처음과 비교해서 뱀의 움직임이 어떻게 달라졌나요?" |
| 15~18분 | 🤖 AI 시연 모드로 전환 → 관찰 | "학습된 AI는 여러분보다 잘하나요? 어떤 점이 다른가요?" |
| 18~20분 | 정리: "강아지 훈련"과 비교 토론 | "간식 주기(보상)로 강아지를 훈련하는 것과 어떻게 비슷한가요?" |

**학생 도달 기대:**
- "AI는 보상과 벌점을 통해 좋은 행동을 스스로 배운다"고 설명할 수 있음
- 학습 초기(무작위)와 후기(목표 지향)의 행동 차이를 인식함

---

### 시나리오 B: 탐험 vs 활용 실험 — "모험과 안전, 무엇이 중요할까?" (20분)

**목표:** ε(탐험률)의 변화가 학습에 미치는 영향을 관찰하고, 탐험과 활용의 균형을 이해한다.

| 시간 | 활동 | 교사 발문 |
|:----:|------|----------|
| 0~3분 | 기본 설정으로 학습 시작 → 헤더의 ε 값 관찰 | "ε가 1.00에서 시작하는데, 이것은 무슨 뜻일까요?" |
| 3~7분 | 터보 학습 → ε가 0.5로 줄어드는 과정 관찰 | "ε가 줄어들면서 뱀의 행동이 어떻게 변하나요?" |
| 7~10분 | 🔄 초기화 → ε 감소율을 0.9로 변경 → 학습 | "감소율을 바꾸니 ε가 얼마나 빨리 줄어드나요?" |
| 10~14분 | 🔄 초기화 → ε 최솟값을 0.1로 변경 → 학습 | "항상 10%는 모험하게 하면 어떻게 될까요?" |
| 14~17분 | 🔄 초기화 → ε=0.01(거의 탐험 없이) 시작 → 학습 | "처음부터 모험을 안 하면 잘 배울 수 있을까요?" |
| 17~20분 | 결과 비교 토론 | "왜 처음에는 많이 탐험하고, 나중에는 줄이는 게 좋을까요?" |

**📝 학생 기록표:**

| 실험 | 설정 변경 | 100 에피소드 후 ε | 최고점수 | 관찰 |
|:----:|----------|:---:|:---:|------|
| 기본 | 감소율 0.995 | | | |
| 빠른 감소 | 감소율 0.9 | | | |
| 높은 최솟값 | ε 최솟값 0.1 | | | |
| 탐험 없음 | 초기 ε=0.01 | | | |

---

### 시나리오 C: 보상 설계 실험 — "당근과 채찍" (25분)

**목표:** 보상 구조가 AI 학습에 미치는 영향을 실험으로 확인하고, "좋은 보상 설계"의 중요성을 이해한다.

| 시간 | 활동 | 교사 지시 |
|:----:|------|----------|
| 0~3분 | 기본 보상으로 200 에피소드 학습 (터보) → 결과 기록 | "기본 설정의 결과를 기록하세요" |
| 3~7분 | 🔄 초기화 → 접근보상=0, 멀어짐벌점=0 → 200 에피소드 | "방향 힌트 없이도 잘 배울까요?" |
| 7~11분 | 🔄 초기화 → 먹이보상=50, 충돌벌점=-1 → 200 에피소드 | "당근을 아주 크게 주면 어떨까요?" |
| 11~15분 | 🔄 초기화 → 먹이보상=1, 충돌벌점=-50 → 200 에피소드 | "채찍을 아주 크게 하면 어떨까요?" |
| 15~20분 | 4가지 결과 비교 분석 | "어떤 보상 구조에서 AI가 가장 잘 배웠나요?" |
| 20~25분 | 토론: 실생활 보상 설계 연결 | "학교에서 '잘한 칭찬'과 '잘못한 벌'의 균형은?" |

**📝 보상 실험 기록표:**

| 실험 | 먹이보상 | 충돌벌점 | 접근 | 멀어짐 | 200회 후 최고점수 | 200회 후 평균보상 |
|:----:|:---:|:---:|:---:|:---:|:---:|:---:|
| 기본 | +10 | -10 | +1 | -1 | | |
| 방향힌트 없음 | +10 | -10 | 0 | 0 | | |
| 큰 당근 | +50 | -1 | +1 | -1 | | |
| 큰 채찍 | +1 | -50 | +1 | -1 | | |

### 시나리오 D: 사람 vs AI 비교 — "누가 더 잘할까?" (15분)

**목표:** 사람의 의사결정과 AI의 학습된 행동을 비교하며 강화학습의 특성을 이해한다.

| 시간 | 활동 | 교사 발문 |
|:----:|------|----------|
| 0~3분 | 🎮 직접 플레이: 학생 3번 시도 → 최고점수 기록 | "여러분의 최고 점수를 기록하세요" |
| 3~8분 | 🧠 AI 학습 300 에피소드 (터보) | "AI가 학습하는 동안 차트 변화를 관찰하세요" |
| 8~11분 | 🤖 AI 시연: AI 3번 시도 → 최고점수 기록 | "AI의 점수와 여러분의 점수를 비교해보세요" |
| 11~15분 | 비교 토론 | "AI가 사람보다 잘하는 점과 못하는 점은?" |

## 5. 교수·학습 팁

### 5.1 비유 활용

| 강화학습 개념 | 쉬운 비유 |
|-------------|----------|
| 에이전트 (뱀) | 훈련 중인 강아지 |
| 환경 (게임) | 강아지가 놀아다니는 마당 |
| 보상 (+) | 간식 주기, 칭찬하기 |
| 벌점 (-) | "안 돼!" 라고 말하기 |
| Q-테이블 | 강아지의 경험 기억 |
| 탐험 (ε) | 새로운 장소 탐험하기 |
| 활용 (1-ε) | 간식 있는 곳으로 바로 가기 |
| 에피소드 | 하루의 훈련 세션 |

### 5.2 흔한 오개념과 교정

| 오개념 | 교정 |
|--------|------|
| "AI가 미리 규칙을 알고 있다" | AI는 규칙을 모르고 시작하며, 시행착오로만 학습. 처음의 무작위 움직임이 증거 |
| "탐험은 나쁘다" | 탐험 없이는 새로운 좋은 전략을 발견할 수 없음. 초기 탐험이 학습의 핵심 |
| "벌점만 주면 빨리 배운다" | 큰 벌점은 AI를 "소극적"으로 만들어 먹이를 피하게 됨. 당근과 채찍의 균형 필요 |
| "AI가 게임을 '이해'한다" | AI는 게임 규칙이나 목표를 이해하지 않고, 오직 보상을 최대화하는 패턴만 학습 |

### 5.3 3종 학습유형 비교 교수법

스네이크 RL을 다른 시뮬레이터와 함께 사용하면 3가지 학습 유형을 효과적으로 비교할 수 있습니다:

| 비교 항목 | 지도학습 (KNN) | 비지도학습 (K-means) | 강화학습 (스네이크) |
|----------|:---:|:---:|:---:|
| 데이터 | 레이블 있음 | 레이블 없음 | 보상/벌점 |
| 학습 방식 | 정답과 비교 | 패턴 발견 | 시행착오 |
| 교사 역할 | 정답 제공 | 없음 | 평가(보상/벌점) |
| 비유 | 교과서로 공부 | 스스로 분류 | 게임하며 학습 |
