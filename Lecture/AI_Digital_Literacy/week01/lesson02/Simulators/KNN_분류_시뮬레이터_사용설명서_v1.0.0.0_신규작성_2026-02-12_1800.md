## 1. 소프트웨어 개요

### 1.1 목적

KNN 분류 시뮬레이터는 K-최근접 이웃(K-Nearest Neighbors) 알고리즘의 작동 원리를 시각적으로 이해할 수 있도록 설계된 교육용 웹 애플리케이션입니다. 새로운 데이터가 들어왔을 때 가장 가까운 K개의 이웃을 찾아 다수결 투표로 분류하는 과정을 단계별로 체험할 수 있습니다.

### 1.2 교육 목표

1. **KNN 알고리즘의 원리 이해**: 거리 기반 분류의 핵심 개념 학습
2. **K값의 효과 체험**: K값 변화에 따른 결정 경계와 정확도 변화 관찰
3. **거리 측정 방식 비교**: 유클리드 거리와 맨해튼 거리의 차이 이해
4. **결정 경계 시각화**: 분류 영역이 어떻게 형성되는지 직관적 파악
5. **혼동행렬 해석**: 분류 성능 평가 방법 학습
6. **실제 데이터 활용**: 아이리스, 펭귄 등 실제 ML 데이터셋 체험

### 1.3 주요 특징

| 특징 | 설명 |
|------|------|
| 2가지 모드 | 학습 모드(데이터 생성 + 분류 실행) + 추론 모드(개별 데이터 분류) |
| 3가지 데이터 입력 | 프리셋 생성, CSV 업로드, 내장 데이터셋 |
| 결정 경계 시각화 | 50×50 격자 기반 실시간 결정 경계 렌더링 |
| K값 분석 | K=1~15 홀수값의 LOOCV 정확도 자동 비교 |
| 혼동행렬 | 클래스별 정분류/오분류 교차표 자동 생성 |
| 2종 거리 측정 | 유클리드 거리, 맨해튼 거리 선택 |
| 추론 시각화 | K개 이웃 연결선, 투표 결과, 신뢰도 표시 |

## 3. 화면 구성

### 3.1 전체 레이아웃

시뮬레이터는 상단 헤더, 좌측 사이드바(240px), 중앙 차트 영역, 우측 사이드바(260px)의 3열 구조로 구성됩니다.

**상단 헤더:**
- 시뮬레이터 제목 및 버전 배지
- **🧠 학습 모드** / **🔮 추론 모드** 탭 전환 버튼

### 3.2 학습 모드 화면

**좌측 사이드바 — 📊 데이터 입력:**
- 데이터 탭 (프리셋 / CSV / 내장): 3가지 데이터 소스 전환
- 프리셋 탭: 과일·꽃·동물·랜덤 프리셋 + 데이터 개수·클래스 수·분산도 슬라이더 + 생성/초기화 버튼
- CSV 탭: 파일 드래그&드롭 영역 + 변수 선택 드롭다운(X₁, X₂, 클래스)
- 내장 탭: 아이리스·펭귄·와인·씨앗 선택 버튼 + 변수 선택 드롭다운
- KNN 알고리즘 설명 박스
- 데이터 테이블 (#, X₁, X₂, 클래스)

**중앙 차트 영역:**
- 📈 데이터 분포 산점도: 클래스별 색상 점 + 결정 경계(분류 실행 후) + 범례
- 결과 패널(분류 실행 후 표시):
  - 📊 분류 결과: K값, 정확도(%), 상태(🎯우수/✅양호/⚠️개선필요)
  - 📋 혼동행렬: 클래스별 정분류/오분류 교차표
  - 📈 K값별 정확도: K=1~15 홀수값 정확도 막대 차트

**우측 사이드바 — ⚙️ KNN 설정:**
- K값(이웃 수) 슬라이더: 1~15, 기본값 5
- 거리 측정 방식 드롭다운: 유클리드 거리 / 맨해튼 거리
- **⚡ 분류 실행** 버튼
- K값 선택 팁 안내 박스

### 3.3 추론 모드 화면

**좌측 사이드바 — 🎯 분류할 데이터:**
- X₁ 슬라이더: 0~100, 0.5 단위
- X₂ 슬라이더: 0~100, 0.5 단위
- 🔗 K개의 최근접 이웃 목록: 순위, 거리, 클래스
- 🗳️ 다수결 투표 결과: 클래스별 투표 수 막대

**중앙 차트 영역 — 🔮 분류 시각화:**
- 전체 데이터 점 표시 (클래스별 색상)
- 예측 지점 강조 표시 (큰 원 + 물결 효과)
- K개 이웃까지 연결선 (점선)
- 이웃 점 강조 표시 (흰색 테두리)
- 결정 경계 배경 (선택적)

**우측 사이드바 — 🏷️ 분류 결과:**
- 예측 클래스: 이모지 + 클래스명 (색상 배경)
- 사용된 K값
- 신뢰도 (%)
- 거리 방식

## 4. 데이터 입력

### 4.1 프리셋 데이터 생성

프리셋은 미리 정의된 주제의 가상 데이터를 자동 생성합니다.

**4개 프리셋:**

| 프리셋 | X₁ 축 | X₂ 축 | 클래스(최대 4개) |
|--------|-------|-------|-----------------|
| 🍎 과일 | 당도(Brix) | 무게(g) | 사과, 배, 귤, 포도 |
| 🌸 꽃 | 꽃잎 길이(cm) | 꽃잎 너비(cm) | 장미, 튤립, 백합, 국화 |
| 🐧 동물 | 몸길이(cm) | 체중(kg) | 고양이, 강아지, 토끼, 햄스터 |
| 🎲 랜덤 | 특성 1(X₁) | 특성 2(X₂) | 클래스 A, B, C, D |

**생성 설정:**

| 설정 | 범위 | 기본값 | 설명 |
|------|------|--------|------|
| 데이터 개수 | 20~150 | 60 | 전체 데이터 포인트 수 |
| 클래스 수 | 2~4 | 3 | 분류 카테고리 수 |
| 분산도 | 1~5단계 | 보통(3) | 1=매우 밀집 ~ 5=매우 분산 |

데이터 생성 시 각 프리셋에 정의된 중심점(centers) 주변에 가우시안 분포로 데이터가 생성됩니다. 분산도 값은 내부적으로 ×8 배수로 변환되어 표준편차로 사용됩니다.

### 4.2 CSV 업로드

외부 CSV 파일의 데이터를 불러올 수 있습니다.

**절차:**
1. **CSV** 탭 클릭
2. 파일 드래그&드롭 또는 클릭하여 업로드
3. 파일명, 컬럼 수, 데이터 수 자동 표시
4. X₁ 변수, X₂ 변수, 클래스 변수 드롭다운에서 선택
5. **📥 데이터 적용** 클릭

**요구사항:**
- CSV 형식(쉼표 구분, UTF-8 인코딩)
- 최소 2개의 숫자형 컬럼 + 1개의 클래스 컬럼
- 첫 행은 헤더(컬럼명)
- X₁과 X₂는 서로 다른 변수여야 함

CSV 데이터는 0~100 범위로 자동 정규화되어 시각화에 최적화됩니다.

## 5. 학습 모드

### 5.1 분류 실행 과정

**⚡ 분류 실행** 클릭 시 내부적으로 다음 과정이 수행됩니다:

1. **K값별 정확도 계산**: K=1부터 15까지 홀수값에 대해 LOOCV(Leave-One-Out Cross Validation) 수행
2. **현재 K값 정확도 표시**: 사용자가 선택한 K값의 정확도
3. **혼동행렬 생성**: 현재 K값으로 LOOCV 수행 결과의 교차표
4. **결정 경계 렌더링**: 50×50 격자의 각 셀을 KNN으로 분류하여 색상 영역 표시
5. **결과 패널 표시**: 정확도, 혼동행렬, K값별 정확도 차트

### 5.2 LOOCV (Leave-One-Out 교차검증)

본 시뮬레이터는 별도의 훈련/테스트 분할 없이 LOOCV 방식으로 정확도를 측정합니다:

1. 데이터에서 1개를 테스트용으로 빼냄
2. 나머지 데이터로 KNN 분류 수행
3. 분류 결과와 실제 클래스 비교
4. 모든 데이터에 대해 반복
5. 정분류 비율 = 정확도

이 방식은 소규모 데이터셋에서 편향 없는 성능 추정이 가능합니다.

### 5.3 결정 경계

분류 실행 후 차트 배경에 반투명 색상 영역으로 결정 경계가 표시됩니다:

- 50×50 해상도의 격자 생성
- 각 격자점에서 현재 K값과 거리 방식으로 KNN 분류 수행
- 분류 결과에 해당하는 클래스 색상을 30% 투명도로 채색
- K값이 작을수록 경계가 복잡(과적합), 클수록 부드러움(일반화)

### 5.4 혼동행렬

LOOCV 결과를 기반으로 생성되는 교차표입니다:

- 행(세로): 실제 클래스
- 열(가로): 예측 클래스
- 대각선 셀: 정분류(하이라이트)
- 비대각선 셀: 오분류

### 5.5 K값별 정확도 차트

K=1, 3, 5, 7, 9, 11, 13, 15의 8개 홀수 K값에 대한 LOOCV 정확도를 막대 차트로 표시합니다. 최적 K값을 시각적으로 비교할 수 있습니다.

### 5.6 정확도 상태 표시

| 정확도 | 상태 | 아이콘 |
|--------|------|--------|
| 90% 초과 | 우수 | 🎯 |
| 70%~90% | 양호 | ✅ |
| 70% 미만 | 개선필요 | ⚠️ |

## 6. 추론 모드

### 6.1 모드 전환

상단 **🔮 추론 모드** 탭을 클릭하면 추론 화면으로 전환됩니다. 학습 모드에서 생성한 데이터가 유지된 채 개별 데이터 포인트의 분류 과정을 상세히 관찰할 수 있습니다.

### 6.2 예측 입력

좌측 패널의 X₁, X₂ 슬라이더로 분류할 데이터의 좌표를 입력합니다:
- X₁: 0.0 ~ 100.0 (0.5 단위)
- X₂: 0.0 ~ 100.0 (0.5 단위)

슬라이더를 움직이면 실시간으로 분류 결과가 갱신됩니다.

### 6.3 추론 결과 표시

**K개의 최근접 이웃 목록:**
- 거리순으로 정렬된 K개 이웃
- 각 이웃의 순위(#1~#K), 거리, 클래스 표시

**다수결 투표 결과:**
- 클래스별 투표 수를 막대 그래프로 표시
- 가장 많은 표를 얻은 클래스가 예측 결과

**분류 결과 패널(우측):**
- 예측 클래스: 이모지 + 클래스명 (해당 색상 배경)
- 사용된 K값
- 신뢰도: 예측 클래스 투표 비율 (%)
- 거리 방식: 유클리드 / 맨해튼

### 6.4 추론 차트 시각화

추론 모드의 차트에는 학습 모드보다 풍부한 시각 정보가 표시됩니다:

1. **전체 데이터 점**: 클래스별 색상 원
2. **예측 지점**: 큰 원 + 흰색 테두리 + 물결 애니메이션
3. **이웃 연결선**: 예측 지점에서 K개 이웃까지 점선
4. **이웃 강조**: K개 이웃 점에 큰 원 + 두꺼운 테두리
5. **결정 경계**: 반투명 색상 영역 (선택적)
6. **축 눈금**: 0~100 범위, 20 단위 격자

### 7.2 알고리즘 상세

**KNN 분류 절차:**
1. 예측 지점과 모든 데이터 간 거리 계산
2. 거리 기준 오름차순 정렬
3. 상위 K개 선택 (최근접 이웃)
4. K개 이웃의 클래스별 투표 집계
5. 최다 득표 클래스를 예측 결과로 반환
6. 신뢰도 = 최다 득표 수 / K × 100

**거리 계산:**
- 유클리드: √((x₁₁ − x₁₂)² + (x₂₁ − x₂₂)²)
- 맨해튼: |x₁₁ − x₁₂| + |x₂₁ − x₂₂|

**데이터 정규화 (CSV/내장 데이터셋):**
- 각 변수의 최솟값/최댓값 기준 0~100 범위로 Min-Max 정규화
- 정규화 공식: x_norm = (x − min) / (max − min) × 100

**결정 경계 렌더링:**
- 해상도: 50×50 = 2,500개 격자점
- 각 격자점에서 KNN 분류 수행
- 분류 결과 색상을 30% 투명도(alpha=0x30)로 채색

### 7.3 색상 체계

## 9. FAQ

**Q1. 왜 K값은 홀수를 권장하나요?**
A1. 짝수 K값은 두 클래스의 투표 수가 동점이 될 수 있습니다. 이 경우 예측이 불안정해질 수 있으므로, 동점을 피하기 위해 홀수를 권장합니다. 본 시뮬레이터에서 K값별 정확도 분석도 홀수(1, 3, 5, ..., 15)만 수행합니다.

**Q2. 유클리드 거리와 맨해튼 거리의 차이는 무엇인가요?**
A2. 유클리드 거리는 두 점 사이의 직선(대각선) 거리이고, 맨해튼 거리는 격자(바둑판)를 따라 이동하는 거리입니다. 유클리드는 일반적으로 많이 사용되며, 맨해튼은 이상값(outlier)에 상대적으로 덜 민감합니다.

**Q3. LOOCV는 왜 사용하나요?**
A3. 소규모 데이터셋(20~150개)에서는 훈련/테스트 분할 시 테스트 데이터가 너무 적어 평가가 불안정합니다. LOOCV는 모든 데이터를 한 번씩 테스트 데이터로 사용하므로 소규모 데이터에서도 편향 없는 정확도 추정이 가능합니다.

**Q4. K값이 커지면 왜 정확도가 떨어질 수 있나요?**
A4. K값이 과도하게 커지면 멀리 있는(다른 클래스의) 데이터까지 이웃으로 포함되어 분류가 부정확해집니다. 극단적으로 K=전체 데이터 수이면 항상 최다 클래스로만 예측합니다. 반대로 K=1이면 노이즈에 과민 반응합니다.

**Q5. 혼동행렬에서 대각선 값이 높으면 좋은 건가요?**
A5. 네. 대각선은 실제 클래스와 예측 클래스가 일치하는 정분류 건수입니다. 대각선 값이 높고 비대각선 값이 낮을수록 분류 성능이 우수합니다.
