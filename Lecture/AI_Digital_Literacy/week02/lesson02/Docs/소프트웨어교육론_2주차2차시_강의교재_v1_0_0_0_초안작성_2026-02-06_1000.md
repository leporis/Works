# 소프트웨어교육론 2주차 2차시 강의교재

**주제: 자료 제작 실습**

**문서 버전: v1.0.0.0**

---

## 서문

이번 차시는 지난 시간에 배운 프롬프트 작성 원칙을 실제로 적용해보는 실습 시간입니다. 생성형 AI를 활용하여 초등학교 수업에서 사용할 수 있는 다양한 교육 자료를 직접 제작해 봅니다.

---

## 1. 오늘의 실습 과제

### 1.1 과제 소개

오늘 여러분이 수행할 과제는 다음과 같습니다.

> 💡 **실습 과제**
> 
> "초등학교 5학년 수학 학습지 1장을 생성형 AI로 제작하세요."

이 과제를 통해 여러분은 1차시에서 배운 프롬프트 작성 4대 원칙(구체성, 맥락 제공, 형식 지정, 단계별 요청)을 실제로 적용해보게 됩니다.

### 1.2 과제 요구사항

학습지를 제작할 때 다음 요구사항을 충족해야 합니다.

| 항목 | 내용 |
|:---:|:---|
| **대상 학년** | 초등학교 5학년 |
| **교과** | 수학 |
| **자료 형태** | 학습지 1장 |
| **분량** | A4 1장 분량 |
| **문제 수** | 5~10개 권장 |
| **난이도** | 쉬움-중간-어려움 골고루 |

### 1.3 사용 가능한 AI 도구

실습에 사용할 수 있는 생성형 AI 도구입니다.

| 도구 | 접속 주소 | 특징 |
|:---:|:---|:---|
| **ChatGPT** | chat.openai.com | 가장 널리 사용, 한글 지원 우수 |
| **Claude** | claude.ai | 긴 텍스트 처리에 강함 |
| **Gemini** | gemini.google.com | Google 서비스 연동 |

어떤 도구를 사용해도 괜찮습니다. 평소 익숙한 도구를 사용하세요.

## 2. 평가 기준

### 2.1 평가 개요

이번 실습 과제는 총 15점으로 평가됩니다.

| 평가 항목 | 배점 | 평가 내용 |
|:---:|:---:|:---|
| **프롬프트의 질** | 5점 | 프롬프트 작성 원칙 적용 정도 |
| **결과물의 질** | 10점 | 완성도, 정확성, 교육적 적절성 |
| **합계** | **15점** | |

### 2.2 프롬프트 평가 기준 (5점)

프롬프트의 질은 1차시에서 배운 4대 원칙 적용 여부로 평가합니다.

| 점수 | 기준 |
|:---:|:---|
| **5점** | 4대 원칙 모두 적용, 체계적으로 구조화된 프롬프트 |
| **4점** | 3가지 원칙 적용, 구조화 시도 |
| **3점** | 2가지 원칙 적용 |
| **2점** | 1가지 원칙만 적용 |
| **1점** | 원칙 적용 없이 단순 요청 |

**4대 원칙 복습**

1. **구체적으로 작성하기**: 추상적 표현 피하고 명확하게
2. **맥락과 배경 정보 제공하기**: 대상, 목적, 상황 설명
3. **원하는 출력 형식 지정하기**: 표, 목록, 문단 등 형태 지정
4. **단계별로 나누어 요청하기**: 복잡한 요청은 분리

### 2.3 결과물 평가 기준 (10점)

| 항목 | 배점 | 세부 기준 |
|:---:|:---:|:---|
| **완성도** | 4점 | 요청한 형식과 분량 충족, 체계적 구성 |
| **정확성** | 3점 | 내용 오류 없음, 수학 문제 정답 정확 |
| **적절성** | 3점 | 학년 수준에 맞음, 교육적 가치 있음 |

### 2.4 제출 방법

| 항목 | 내용 |
|:---:|:---|
| **제출 기한** | 수업 종료 후 10분 이내 (절대 기한) |
| **제출 방법** | LMS 과제 제출 |
| **제출물 1** | 최종 결과물 (학습지) |
| **제출물 2** | 사용한 프롬프트 전문 |

> ⚠️ **중요**
> 
> 프롬프트 전문을 반드시 함께 제출해야 합니다. 
> 결과물만 제출하면 프롬프트 평가(5점)를 받을 수 없습니다.

## 3. 프롬프트 작성 가이드

### 3.1 나쁜 프롬프트 vs 좋은 프롬프트

**나쁜 프롬프트 예시** ❌

```
수학 학습지 만들어줘
```

이 프롬프트의 문제점을 살펴봅시다.

| 문제점 | 설명 |
|:---|:---|
| 학년 정보 없음 | 어느 학년을 위한 것인지 알 수 없음 |
| 단원 정보 없음 | 어떤 수학 내용인지 불명확 |
| 분량 정보 없음 | 문제 수, 페이지 수 등 미지정 |
| 형식 정보 없음 | 어떤 레이아웃을 원하는지 불명확 |

**좋은 프롬프트 예시** ✅

```
[기본 정보]
- 대상: 초등학교 5학년
- 과목: 수학
- 단원: 분수의 덧셈과 뺄셈
- 목적: 수업 후 형성평가용 학습지

[문제 구성]
- 총 8문제
- 난이도 배분: 쉬움 3문제, 중간 3문제, 어려움 2문제
- 유형: 계산 문제 5개, 문장제 문제 3개

[형식]
- A4 1장 분량
- 상단에 제목과 이름 쓰는 칸 포함
- 문제 번호와 배점 표시
- 풀이 공간 여유있게 배치

이 조건에 맞는 학습지를 만들어주세요.
```

이 프롬프트가 좋은 이유:

| 적용 원칙 | 적용 내용 |
|:---|:---|
| ① 구체적으로 | 학년, 단원, 문제 수, 난이도까지 명시 |
| ② 맥락 제공 | "형성평가용"이라는 목적 제시 |
| ③ 형식 지정 | A4 1장, 제목란, 풀이 공간 등 레이아웃 지정 |
| ④ 단계별 구조 | [기본 정보], [문제 구성], [형식]으로 분리 |

### 3.2 프롬프트 구조화 템플릿

학습지 제작 시 활용할 수 있는 템플릿입니다.

```
[기본 정보]
- 대상: (학년)
- 과목: (과목명)
- 단원: (단원/주제)
- 목적: (활용 목적)

[문제 구성]
- 총 문제 수: (숫자)개
- 난이도: (배분 방법)
- 문제 유형: (유형 나열)

[형식]
- 분량: (A4 몇 장)
- 레이아웃: (상세 설명)
- 기타: (추가 요청)
```

### 3.3 자료 유형별 프롬프트 팁

**① 학습지/평가 문항**

| 포함 요소 | 예시 |
|:---|:---|
| 기본 정보 | "5학년 1학기 분수 단원" |
| 문제 구성 | "계산 5문제 + 문장제 3문제" |
| 난이도 | "쉬움:중간:어려움 = 3:3:2" |
| 형식 | "문제 사이 풀이 공간 3줄" |

**② 안내문/가정통신문**

| 포함 요소 | 예시 |
|:---|:---|
| 발신자 | "OO초등학교 5학년 1반 담임" |
| 수신자 | "학부모님" |
| 안내 내용 | "현장학습 일시, 장소, 준비물" |
| 형식 | "공문 형식, 하단에 회신란" |

**③ 표/데이터 정리**

| 포함 요소 | 예시 |
|:---|:---|
| 표 구조 | "5행 4열" |
| 헤더 | "이름, 국어, 수학, 평균" |
| 형식 | "마크다운 표 형식" |

## 4. 실습 수행 가이드

### 4.1 실습 절차

실습은 다음 순서로 진행합니다.

```
┌─────────────────────────────────────┐
│         실습 진행 순서              │
├─────────────────────────────────────┤
│                                     │
│  [1] 프롬프트 작성 (5분)            │
│       │                             │
│       ▼                             │
│  [2] AI에게 요청 (1분)              │
│       │                             │
│       ▼                             │
│  [3] 결과 확인 (3분)                │
│       │                             │
│       ▼                             │
│  [4] 만족? ──No──▶ 프롬프트 수정    │
│       │              │              │
│      Yes             │              │
│       │              │              │
│       ▼◀─────────────┘              │
│  [5] 최종 결과물 확정               │
│       │                             │
│       ▼                             │
│  [6] 프롬프트 전문 저장             │
│                                     │
└─────────────────────────────────────┘
```

### 4.2 단계별 상세 안내

**[1단계] 프롬프트 작성**

- 3.2의 템플릿을 참고하여 작성
- 4대 원칙을 의식하며 작성
- 처음부터 완벽할 필요 없음 (반복 개선 예정)

**[2단계] AI에게 요청**

- 선택한 AI 도구에 프롬프트 입력
- Enter 또는 전송 버튼 클릭

**[3단계] 결과 확인**

- 생성된 결과물 꼼꼼히 확인
- 특히 **수학 문제의 정답이 맞는지** 반드시 확인!

**[4단계] 만족 여부 판단**

만족스럽지 않다면 프롬프트를 수정합니다.

| 불만족 사항 | 수정 방법 |
|:---|:---|
| 난이도가 안 맞음 | "쉬움-중간-어려움 비율 3:3:2로" |
| 분량이 안 맞음 | "A4 1장 분량으로 조정해줘" |
| 형식이 이상함 | 원하는 형식을 더 구체적으로 설명 |
| 정답이 틀림 | 직접 수정하거나 "N번 문제 정답 확인해줘" |

**[5단계] 최종 결과물 확정**

- 수정이 완료되면 최종 결과물로 확정
- 파일로 저장 (PDF, DOCX, 이미지 등)

**[6단계] 프롬프트 전문 저장**

- 사용한 프롬프트 전체를 복사
- 텍스트 파일로 저장
- **제출 시 반드시 포함!**

### 4.3 실습 중 체크리스트

실습하면서 다음을 확인하세요.

**프롬프트 작성 시** ✓

- [ ] 학년과 단원을 명시했는가?
- [ ] 문제 수와 난이도를 지정했는가?
- [ ] 원하는 형식(레이아웃)을 설명했는가?
- [ ] 프롬프트를 구조화했는가?

**결과 확인 시** ✓

- [ ] 문제 난이도가 적절한가?
- [ ] **수학 정답이 맞는가?** (필수!)
- [ ] 학년 수준에 맞는 용어를 사용했는가?
- [ ] 요청한 형식과 분량이 맞는가?

## 5. AI 결과물 검토

### 5.1 왜 검토가 필요한가?

생성형 AI는 매우 유용하지만, 완벽하지 않습니다. AI가 생성한 결과물에는 오류가 포함될 수 있습니다.

**AI 결과물의 특징**

| 장점 | 단점 |
|:---|:---|
| 빠른 초안 생성 | 정확성 보장 안 됨 |
| 다양한 형식 지원 | 가끔 사실과 다른 내용 생성 |
| 반복 수정 가능 | 맥락을 잘못 이해할 수 있음 |

### 5.2 환각(Hallucination) 현상

> 🚨 **환각(Hallucination)이란?**
> 
> AI가 그럴듯하지만 실제로는 틀린 정보를 생성하는 현상입니다.

**교육 자료에서 발생할 수 있는 환각 현상**

| 자료 유형 | 환각 예시 |
|:---|:---|
| 수학 문제 | 제시된 정답이 실제로는 틀림 |
| 역사 내용 | 역사적 사실이 부정확함 |
| 과학 내용 | 과학적 사실 오류 |
| 인용/출처 | 존재하지 않는 책이나 논문 인용 |

**실제 사례**

```
AI가 생성한 수학 문제:
"3/4 + 1/2를 계산하세요. (정답: 5/6)"

실제 정답: 5/4 (또는 1과 1/4)

AI가 틀린 정답을 제시한 경우입니다!
```

### 5.3 검토 체크리스트

AI가 생성한 결과물을 검토할 때 확인해야 할 항목입니다.

| 검토 항목 | 확인 내용 |
|:---|:---|
| **정확성** | 수학 정답이 맞는가? 사실 정보가 정확한가? |
| **적절성** | 학년 수준에 맞는가? 교육과정에 부합하는가? |
| **완성도** | 요청한 형식과 분량에 맞는가? |
| **윤리성** | 편견이나 부적절한 내용은 없는가? |

### 5.4 교사의 역할

> 💡 **기억하세요!**
> 
> AI는 **도구**입니다. 최종 책임은 **교사**에게 있습니다.
> 
> - AI가 생성한 결과물은 반드시 교사가 검토해야 합니다.
> - 오류가 발견되면 직접 수정합니다.
> - AI는 초안 생성을 도와주는 조수 역할입니다.

## 6. 프롬프트 개선 전략

### 6.1 반복 개선의 중요성

처음부터 완벽한 결과를 얻기는 어렵습니다. 프롬프트를 수정하며 점진적으로 개선하는 것이 중요합니다.

```
반복 개선 과정:

1차 시도 → 결과 확인 → 부족한 점 파악
    ↓
프롬프트 수정 → 2차 시도 → 결과 확인
    ↓
필요시 추가 수정 → 3차 시도 → 최종 확정
```

### 6.2 상황별 개선 방법

**결과가 불만족스러울 때**

| 문제 상황 | 개선 방법 |
|:---|:---|
| 내용이 부족함 | "더 자세히 설명해줘", "예시를 추가해줘" |
| 난이도가 맞지 않음 | 구체적인 비율 제시 "쉬움:중간:어려움 = 3:4:3" |
| 형식이 이상함 | 원하는 형식을 더 상세히 설명 |
| 분량이 맞지 않음 | "A4 1장 분량으로 조정해줘" |
| 정답이 틀림 | "N번 문제 정답을 다시 계산해줘" |

### 6.3 반복 개선 예시

실제 프롬프트 개선 과정을 살펴봅시다.

**1차 시도**

```
5학년 수학 학습지 만들어줘
```

결과: 문제가 너무 어렵고, 형식이 불명확함

**2차 시도**

```
5학년 1학기 분수 덧셈 학습지를 만들어줘.
- 문제 8개
- 쉬움-중간-어려움 골고루
```

결과: 내용은 좋은데 레이아웃이 아쉬움

**3차 시도**

```
위 내용을 유지하되:
- 상단에 제목과 이름란 추가
- 문제 사이 여백 넓게
- 각 문제에 배점 표시
```

결과: 만족스러운 학습지 완성!

### 6.4 후속 요청 활용

AI와 대화하듯이 후속 요청을 활용할 수 있습니다.

**유용한 후속 요청 예시**

| 상황 | 후속 요청 |
|:---|:---|
| 문제 수 조정 | "문제를 5개로 줄여줘" |
| 난이도 조정 | "1~3번 문제를 더 쉽게 만들어줘" |
| 형식 수정 | "표 형식으로 다시 정리해줘" |
| 정답지 요청 | "정답지를 별도로 만들어줘" |
| 해설 추가 | "각 문제에 풀이 과정도 추가해줘" |

## 8. 증강 관점 복습

### 8.1 증강(Augmentation)이란?

오늘 실습은 **증강** 관점의 AI 활용입니다.

> 💡 **증강(Augmentation)**
> 
> 인간이 주도하고, AI가 보조하는 방식의 AI 활용

**증강 vs 자동화**

| 관점 | 설명 | 인간의 역할 |
|:---:|:---|:---|
| **증강** | 인간이 주도, AI가 보조 | 방향 설정, 최종 검토 |
| 자동화 | AI가 반복 작업 수행 | 지시, 감독 |

### 8.2 오늘 실습에서의 증강

오늘 여러분이 수행한 실습을 증강 관점에서 분석해봅시다.

| 단계 | 주체 | 역할 |
|:---|:---:|:---|
| 과제 파악 | 인간 | 어떤 학습지가 필요한지 판단 |
| 프롬프트 작성 | 인간 | 요구사항을 AI가 이해할 수 있게 전달 |
| 초안 생성 | AI | 프롬프트에 맞는 학습지 초안 생성 |
| 검토 및 수정 | 인간 | 오류 확인, 수정 지시 |
| 최종 확정 | 인간 | 최종 결과물 승인 |

**핵심 포인트**: 교사가 전 과정의 **주도권**을 가지고 있습니다!

### 8.3 다음 주 예고: 자동화

다음 주(3주차)에는 **자동화** 관점을 배웁니다.

- 반복되는 작업을 AI가 대신 수행
- 템플릿 활용, 대량 처리 기법
- 실습: 자동화 과제 수행

- 템플릿 활용: [기본 정보], [문제 구성], [형식]

**4. 결과물 검토**
- 환각 현상 주의
- 수학 정답 반드시 확인
- 교사의 최종 검토 필수

**5. 증강 관점**
- 인간이 주도, AI가 보조
- 교사가 전 과정의 주도권 보유
