# 소프트웨어교육론 2주차 2차시 강의노트

**주제: 자료 제작 실습**

**문서 버전: v1.0.0.0**

---

## 1. 수업 개요

| 항목 | 내용 |
|:---:|:---|
| **과목명** | 소프트웨어교육론 |
| **주차/차시** | 2주차 2차시 |
| **주제** | 자료 제작 실습 |
| **수업 시간** | 50분 |
| **수업 형태** | 실습 중심 |
| **선수 학습** | 2주차 1차시 (프롬프트 작성 원칙) |

---

## 2. 수업 목표

이번 차시가 끝나면 학습자는 다음을 할 수 있습니다.

1. **생성형 AI를 활용하여** 다양한 형태의 교육 자료(학습지, 평가 문항, 표 등)를 제작할 수 있다.
2. **1차시에서 배운 프롬프트 작성 원칙**을 실제 자료 제작에 적용할 수 있다.
3. **결과물의 품질을 평가**하고 프롬프트를 개선할 수 있다.
4. **AI 생성 결과물의 정확성**을 검토하는 습관을 갖출 수 있다.

---

## 3. 시간 배분

| 단계 | 시간 | 내용 |
|:---:|:---:|:---|
| **도입** | 5분 | 실습 과제 안내, 평가 기준 설명, 제출 방법 안내 |
| **전개** | 35분 | 개별 실습 수행, 순회 지도 |
| **정리** | 10분 | 실습 결과 정리, 프롬프트 전문 준비, 과제 제출 |

## 4. 도입 (5분)

### 4.1 오늘의 실습 과제 소개

> 💡 **오늘의 과제**
> 
> "초등학교 5학년 수학 학습지 1장을 생성형 AI로 제작하세요."

오늘 수업은 지난 시간에 배운 프롬프트 작성 원칙을 실제로 적용해보는 실습 시간입니다. 여러분이 직접 생성형 AI에게 프롬프트를 작성하여 교육 자료를 만들어볼 거예요.

### 4.2 과제 요구사항

| 항목 | 내용 |
|:---:|:---|
| **대상 학년** | 초등학교 5학년 |
| **교과** | 수학 |
| **자료 형태** | 학습지 1장 |
| **분량** | A4 1장 분량 (문제 5~10개 권장) |
| **난이도** | 쉬움-중간-어려움 문제 골고루 포함 |

### 4.3 평가 기준 안내

| 평가 항목 | 배점 | 평가 내용 |
|:---:|:---:|:---|
| **프롬프트의 질** | 5점 | 구체성, 맥락 제공, 형식 지정, 단계적 요청 |
| **결과물의 질** | 10점 | 완성도, 정확성, 교육적 적절성 |
| **합계** | **15점** | |

### 4.4 제출 안내

- **제출 기한**: 수업 종료 후 10분 이내 (절대 기한)
- **제출 방법**: LMS 과제 제출
- **제출물**: 
  1. 최종 결과물 (학습지)
  2. 사용한 프롬프트 전문

> ⚠️ **주의사항**
> 
> 프롬프트 전문을 반드시 함께 제출해야 합니다. 결과물만 제출하면 프롬프트 평가(5점)를 받을 수 없습니다.

## 5. 전개 (35분)

### 5.1 실습 준비

**사용 도구 선택**

| 도구 | 특징 | 접속 방법 |
|:---:|:---|:---|
| **ChatGPT** | 가장 널리 사용, 한글 지원 | chat.openai.com |
| **Claude** | 긴 문장 처리에 강함, 한글 지원 | claude.ai |
| **Gemini** | Google 서비스 연동 | gemini.google.com |

어떤 도구를 사용해도 괜찮습니다. 평소 익숙한 도구를 사용하세요.

### 5.2 프롬프트 작성 가이드

**1차시 복습: 프롬프트 4대 원칙** 🔄

| 원칙 | 설명 | 적용 예시 |
|:---:|:---|:---|
| **① 구체적으로** | 추상적 표현 피하기 | "수학 문제" → "5학년 분수의 덧셈 문제" |
| **② 맥락 제공** | 배경 정보 주기 | "초등학교 5학년 학생을 위한", "1학기 과정" |
| **③ 형식 지정** | 원하는 출력 형태 명시 | "표 형식으로", "번호를 붙여서" |
| **④ 단계별 요청** | 복잡한 요청은 나누기 | 먼저 개요 → 그 다음 세부 내용 |

### 5.3 프롬프트 예시

**나쁜 프롬프트 예시** ❌

```
수학 학습지 만들어줘
```

이 프롬프트의 문제점:
- 학년이 없음
- 어떤 단원인지 없음
- 문제 수, 난이도 정보 없음
- 형식 지정 없음

**좋은 프롬프트 예시** ✅

```
[기본 정보]
- 대상: 초등학교 5학년
- 과목: 수학
- 단원: 분수의 덧셈과 뺄셈
- 목적: 수업 후 형성평가용 학습지

[문제 구성]
- 총 8문제
- 난이도 배분: 쉬움 3문제, 중간 3문제, 어려움 2문제
- 유형: 계산 문제 5개, 문장제 문제 3개

[형식]
- A4 1장 분량
- 상단에 제목과 이름 쓰는 칸 포함
- 문제 번호와 배점 표시
- 풀이 공간 여유있게 배치

이 조건에 맞는 학습지를 만들어주세요.
```

### 5.4 자료 유형별 프롬프트 팁

**① 학습지/평가 문항 제작 시**

| 요소 | 포함할 내용 |
|:---|:---|
| 기본 정보 | 학년, 과목, 단원 |
| 문제 구성 | 문제 수, 유형, 난이도 배분 |
| 형식 | 레이아웃, 답안 작성 공간 |
| 추가 요청 | 정답지 별도 제작 여부 |

**② 안내문/가정통신문 제작 시**

| 요소 | 포함할 내용 |
|:---|:---|
| 기본 정보 | 발신자(학교명, 담임), 수신자(학부모) |
| 내용 | 안내 목적, 핵심 일정/정보 |
| 형식 | 공문 형식, 인사말 포함 여부 |
| 톤 | 격식체, 친근한 어조 등 |

**③ 표/데이터 정리 시**

| 요소 | 포함할 내용 |
|:---|:---|
| 표 구조 | 행/열 개수, 헤더 내용 |
| 데이터 | 포함할 항목 나열 |
| 형식 | 마크다운 표, 엑셀 형식 등 |

### 5.5 실습 수행 (개별 활동)

**실습 절차**

```
[1] 프롬프트 작성 (5분)
    ↓
[2] AI에게 요청 및 결과 확인 (5분)
    ↓
[3] 결과 검토 및 프롬프트 수정 (반복)
    ↓
[4] 최종 결과물 확정
    ↓
[5] 프롬프트 전문 저장
```

**실습 중 체크포인트** ✓

□ 프롬프트에 학년과 단원을 명시했는가?
□ 문제 수와 난이도를 지정했는가?
□ 원하는 형식(레이아웃)을 설명했는가?
□ 결과물의 정확성을 확인했는가? (특히 수학 문제의 정답!)

### 5.6 순회 지도 포인트 (교수자용)

**확인 사항**

| 항목 | 지도 내용 |
|:---|:---|
| **프롬프트 확인** | 너무 막연하지 않은지, 4대 원칙 적용 여부 |
| **결과물 검토** | 학년 수준에 맞는지, 오류는 없는지 |
| **환각 현상 주의** | 수학 문제의 정답이 틀리지 않았는지 반드시 확인 안내 |
| **시간 관리** | 한 가지에 너무 많은 시간 쓰지 않도록 안내 |

**자주 발생하는 문제와 해결책**

| 문제 상황 | 해결 방법 |
|:---|:---|
| "결과가 마음에 안 들어요" | 프롬프트에서 구체적으로 어떤 부분이 부족한지 추가 요청 |
| "문제가 너무 어려워요" | "초등학교 5학년 1학기 수준에 맞게"라고 난이도 명시 |
| "형식이 이상해요" | "마크다운 형식으로", "표로 정리해서" 등 형식 지정 |
| "수학 정답이 틀렸어요" | AI 생성 결과는 반드시 검토! 틀린 부분 직접 수정 |

### 5.7 AI 생성 결과물 검토 방법

**필수 검토 항목**

| 검토 항목 | 확인 내용 |
|:---|:---|
| **정확성** | 수학 문제의 정답이 맞는지, 사실 정보가 정확한지 |
| **적절성** | 학년 수준에 맞는지, 교육과정에 부합하는지 |
| **완성도** | 요청한 형식과 분량에 맞는지 |
| **윤리성** | 편견이나 부적절한 내용이 없는지 |

> 🚨 **환각(Hallucination) 현상 주의!**
> 
> AI는 가끔 그럴듯하지만 틀린 정보를 생성합니다.
> 
> - 수학 문제의 **정답이 틀릴 수 있습니다**
> - 역사적 사실이 **부정확할 수 있습니다**
> - 인용이나 출처가 **허구일 수 있습니다**
> 
> **교사가 반드시 최종 검토해야 합니다!**

### 5.8 프롬프트 개선 전략

**결과가 불만족스러울 때**

| 문제 | 개선 방법 |
|:---|:---|
| 내용이 부족함 | "더 자세히", "예시를 추가해서" |
| 난이도가 안 맞음 | "쉬움-중간-어려움 비율 3:4:3으로" |
| 형식이 이상함 | 원하는 형식을 구체적으로 다시 설명 |
| 분량이 안 맞음 | "A4 1장 분량으로", "문제 8개로 조정해서" |

**반복 개선 예시**

```
1차 시도: "5학년 수학 학습지 만들어줘"
→ 결과: 너무 어렵고 형식이 불명확

2차 시도: "5학년 1학기 분수 덧셈 학습지, 문제 8개, 쉬움-중간-어려움 골고루"
→ 결과: 내용은 좋은데 레이아웃이 아쉬움

3차 시도: "위 내용 유지하되, 상단에 제목과 이름란 추가, 문제 사이 여백 넓게"
→ 결과: 만족!
```

## 6. 정리 (10분)

### 6.1 실습 결과 정리

**최종 점검 체크리스트**

□ 학습지가 완성되었는가?
□ 문제의 정답을 확인했는가?
□ 프롬프트 전문을 복사해두었는가?
□ 제출할 파일이 준비되었는가?

### 6.2 과제 제출 준비

**제출 파일 구성**

| 파일 | 내용 | 형식 |
|:---:|:---|:---:|
| **파일 1** | 최종 결과물 (학습지) | PDF, DOCX, 이미지 |
| **파일 2** | 사용한 프롬프트 전문 | TXT, DOCX |

**프롬프트 제출 양식 예시**

```
[사용한 AI 도구]: ChatGPT / Claude / Gemini
[프롬프트 전문]:
(여기에 프롬프트 전체 내용 복사)

[수정 이력] (선택)
- 1차 시도: ...
- 2차 시도: ...
```

### 6.3 평가 기준 상세

**프롬프트의 질 (5점)**

| 점수 | 기준 |
|:---:|:---|
| 5점 | 4대 원칙 모두 적용, 체계적 구조 |
| 4점 | 3가지 원칙 적용, 구조화 시도 |
| 3점 | 2가지 원칙 적용 |
| 2점 | 1가지 원칙만 적용 |
| 1점 | 원칙 적용 없이 단순 요청 |

**결과물의 질 (10점)**

| 항목 | 배점 | 기준 |
|:---:|:---:|:---|
| 완성도 | 4점 | 요청한 형식과 분량 충족 |
| 정확성 | 3점 | 내용 오류 없음, 정답 정확 |
| 적절성 | 3점 | 학년 수준에 맞음, 교육적 가치 있음 |

## 7. 보충 자료

### 7.1 추가 프롬프트 예시

**가정통신문 예시**

```
[기본 정보]
- 학교명: OO초등학교
- 담당: 5학년 1반 담임교사
- 수신: 학부모님

[안내 내용]
- 다음 주 현장학습 안내
- 일시: 5월 15일(금)
- 장소: 과학관
- 준비물: 도시락, 필기도구
- 복장: 편한 복장

[형식]
- 공문 형식
- 존댓말 사용
- 하단에 회신 부분 포함
```

**평가 기준표 예시**

```
[과제]
초등학교 5학년 발표 수업 평가 기준표 제작

[구성]
- 평가 영역: 내용, 발표력, 태도, 협력
- 각 영역별 수준: 매우 잘함 / 잘함 / 보통 / 노력 필요
- 각 수준별 구체적 행동 지표 포함

[형식]
- 표 형식
- A4 가로 1장
```
