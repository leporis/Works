# 소프트웨어교육론 2주차 2차시 강의교재

**주제: 자료 제작 실습**

**문서 버전: v1.1.0.0 (완성)**

---

## 1. 학습 목표

이번 차시를 마치면 다음을 할 수 있습니다.

- 생성형 AI를 활용하여 다양한 형태의 교육 자료를 제작할 수 있다.
- 프롬프트 작성 원칙을 실제 자료 제작에 적용할 수 있다.
- 결과물의 품질을 평가하고 프롬프트를 개선할 수 있다.

---

## 2. 도입: 오늘의 실습 안내

### 2.1 과제 개요

오늘은 1차시에서 배운 프롬프트 작성 원칙을 실제로 적용해봅니다. 초등학교 현장에서 사용할 수 있는 교육 자료를 생성형 AI로 직접 제작하는 것이 과제입니다.

### 2.2 선택 가능한 자료 유형

아래 4가지 유형 중 하나를 선택하여 제작합니다.

| 자료 유형 | 설명 | 예시 |
|:---|:---|:---|
| **학습지** | 특정 단원의 연습 문제, 활동지 | 3학년 수학 분수 학습지 |
| **안내문** | 현장학습 안내, 가정통신문 | 가을 현장학습 안내문 |
| **평가 문항** | 단원평가, 수행평가 문항 | 과학 단원평가 10문항 |
| **표/데이터** | 학습 자료용 데이터 표 | 세계 주요 국가 정보표 |

### 2.3 평가 기준

| 평가 영역 | 배점 | 세부 기준 |
|:---:|:---:|:---|
| 프롬프트 질 | 5점 | 명확성, 구체성, 형식 지정, 맥락 제공 |
| 결과물 품질 | 10점 | 목표 달성도, 완성도, 활용 가능성 |
| **합계** | **15점** | |

### 2.4 제출 요건

- 제출 시한: 수업 종료 후 10분 이내
- 제출물: 프롬프트 전문 + 최종 결과물
- 형식: 문서 파일 (한글, Word, PDF)

## 3. 1차시 복습: 프롬프트 작성 5원칙

실습에 앞서 1차시에서 배운 프롬프트 작성 원칙을 복습합니다. 모호한 프롬프트를 어떻게 명확하게 개선할 수 있는지 확인하세요.

![그림 2. 프롬프트 개선 과정](강의교재_그림2_프롬프트개선과정.svg)

### 3.1 5원칙 요약

| 원칙 | 핵심 내용 | 적용 예시 |
|:---:|:---|:---|
| **명확성** | 모호하지 않게 구체적으로 요청 | "수학 문제" → "3학년 분수 덧셈 문제 5개" |
| **맥락** | 배경 정보와 사용 목적 제공 | "초등학교 3학년 수업용으로..." |
| **형식** | 원하는 결과물의 형태 지정 | "표 형식으로", "10문항으로" |
| **단계** | 복잡한 요청은 단계별로 나눔 | "조건1: ..., 조건2: ..., 조건3: ..." |
| **예시** | 원하는 결과의 샘플 제시 | "예시: 1/4 + 2/4 = __" |

### 3.2 모호한 프롬프트 vs 명확한 프롬프트

**모호한 프롬프트:**

```
학습지 만들어줘
```

- 학년: 불명확
- 과목: 불명확
- 문항 수: 불명확
- 형식: 불명확

**명확한 프롬프트:**

```
초등학교 3학년 수학 '분수의 덧셈' 학습지를 만들어줘.

조건:
- 문항 수: 8문항
- 난이도: 기본 (분모가 같은 분수)
- 형식: 빈칸 채우기 4문항 + 계산 문제 4문항
- 상단에 이름 쓰는 란 포함
```

- 학년: 초등학교 3학년 ✓
- 과목: 수학 - 분수의 덧셈 ✓
- 문항 수: 8문항 ✓
- 형식: 빈칸 채우기 + 계산 문제 ✓

## 4. 실습 가이드

오늘 실습은 4단계로 진행됩니다. 각 단계에서 무엇을 해야 하는지 먼저 확인하세요.

![그림 1. 자료 제작 실습 4단계](강의교재_그림1_실습단계흐름도.svg)

### 4.1 실습 단계

| 단계 | 시간 | 활동 내용 |
|:---:|:---:|:---|
| 1단계 | 5분 | 자료 유형 선택 및 프롬프트 초안 작성 |
| 2단계 | 10분 | AI에게 요청, 결과물 확인 및 문제점 파악 |
| 3단계 | 15분 | 프롬프트 수정, 반복 시도, 최적 결과물 도출 |
| 4단계 | 5분 | 결과물 정리, 필요 시 직접 수정 |

### 4.2 실습 진행 순서

**1단계: 프롬프트 설계**

1. 만들고 싶은 자료 유형을 선택합니다.
2. 해당 자료에 필요한 조건을 정리합니다.
3. 5원칙을 적용하여 프롬프트 초안을 작성합니다.

**2단계: 1차 시도**

1. 작성한 프롬프트를 AI에게 입력합니다.
2. 생성된 결과물을 꼼꼼히 확인합니다.
3. 문제점이나 개선할 점을 파악합니다.

**3단계: 개선 및 재시도**

1. 파악한 문제점을 해결하도록 프롬프트를 수정합니다.
2. 수정된 프롬프트로 다시 요청합니다.
3. 만족스러운 결과물이 나올 때까지 반복합니다.

**4단계: 마무리**

1. 최종 결과물을 선택합니다.
2. 필요하다면 직접 수정하여 완성도를 높입니다.
3. 프롬프트와 결과물을 제출 문서로 정리합니다.

## 5. 자료 유형별 제작 가이드

### 5.1 학습지 제작

학습지는 특정 단원의 학습 내용을 연습하기 위한 자료입니다.

**프롬프트 작성 시 포함할 요소:**

| 요소 | 설명 | 예시 |
|:---|:---|:---|
| 학년 | 대상 학년 | 초등학교 3학년 |
| 과목/단원 | 구체적인 단원명 | 수학 분수의 덧셈 |
| 문항 수 | 필요한 문제 개수 | 10문항 |
| 난이도 | 상/중/하 또는 구체적 기준 | 기본 난이도 |
| 형식 | 문항 유형, 배치 방식 | 객관식 5 + 서술형 5 |
| 부가 요소 | 이름란, 풀이 공간 등 | 상단에 이름란 포함 |

**좋은 프롬프트 예시:**

```
초등학교 4학년 국어 '높임 표현' 단원 학습지를 만들어줘.

조건:
- 문항 수: 총 6문항
- 난이도: 기본
- 형식: 
  · 1~3번: 높임 표현 찾기 (문장 속에서 높임 표현 밑줄 긋기)
  · 4~5번: 높임 표현으로 바꾸기 (일반체를 존칭으로)
  · 6번: 짧은 글쓰기 (선생님께 드리는 편지)
- 상단에 이름 쓰는 란 포함
- 각 문항 사이에 충분한 쓰기 공간 확보
```

**결과물 점검 포인트:**

- 지정한 학년 수준에 맞는가?
- 문항 수와 형식이 요청과 일치하는가?
- 교육과정 성취기준과 연결되는가?

---

### 5.2 안내문 제작

안내문은 학부모나 학생에게 정보를 전달하기 위한 문서입니다.

**프롬프트 작성 시 포함할 요소:**

| 요소 | 설명 | 예시 |
|:---|:---|:---|
| 문서 종류 | 안내문의 유형 | 현장학습 안내문 |
| 대상 | 수신자 명시 | 학부모님께 |
| 목적 | 안내 목적 | 가을 현장학습 안내 |
| 포함 정보 | 필수 기재 항목 | 날짜, 장소, 준비물, 비용 |
| 톤 | 문체와 분위기 | 격식체, 정중한 어조 |
| 분량 | 적정 길이 | A4 1장 |

**좋은 프롬프트 예시:**

```
초등학교 5학년 봄 운동회 안내문을 작성해줘.

조건:
- 대상: 학부모님
- 형식: 가정통신문
- 포함 내용:
  · 일시: 5월 10일(금) 오전 9시~오후 2시
  · 장소: 학교 운동장
  · 준비물: 체육복, 모자, 물, 간식
  · 우천 시: 5월 17일로 연기
- 분량: A4 1장
- 톤: 정중하고 명확한 안내체
- 하단에 참관 신청 회신란 포함
```

**결과물 점검 포인트:**

- 필수 정보가 빠짐없이 포함되었는가?
- 학부모가 읽기에 적절한 어조인가?
- 회신이나 서명이 필요하면 해당 란이 있는가?

---

### 5.3 평가 문항 제작

평가 문항은 학생들의 학습 성취도를 확인하기 위한 시험 문제입니다.

**프롬프트 작성 시 포함할 요소:**

| 요소 | 설명 | 예시 |
|:---|:---|:---|
| 과목 | 평가 과목 | 과학 |
| 단원/성취기준 | 평가 범위 | 식물의 한살이 |
| 문항 유형 | 객관식, 서술형 등 | 4지선다형 |
| 난이도 분포 | 난이도 비율 | 하 30%, 중 50%, 상 20% |
| 수량 | 총 문항 수 | 10문항 |
| 정답 요청 | 정답 포함 여부 | 정답과 해설 별도 포함 |

**좋은 프롬프트 예시:**

```
초등학교 3학년 사회 '우리 고장의 모습' 단원평가 문제를 만들어줘.

조건:
- 문항 수: 8문항
- 유형: 4지선다형 객관식
- 난이도: 쉬움 2문항, 보통 4문항, 어려움 2문항
- 다루는 개념: 고장의 자연환경, 고장의 특산물, 고장의 축제
- 정답과 간단한 해설을 문항 아래에 별도 제시
```

**결과물 점검 포인트:**

- 성취기준에 부합하는 문항인가?
- 난이도가 적절하게 분포되었는가?
- 오답 선지가 그럴듯하게 구성되었는가?
- **정답이 정확한가?** (반드시 검증 필요!)

⚠️ **중요: 정답 검증**

AI가 생성한 정답은 틀릴 수 있습니다. 특히 수학 계산 문제나 과학 개념 문제는 반드시 직접 확인하세요.

---

### 5.4 표/데이터 제작

표/데이터는 정보를 정리하여 시각적으로 제시하는 자료입니다.

**프롬프트 작성 시 포함할 요소:**

| 요소 | 설명 | 예시 |
|:---|:---|:---|
| 데이터 종류 | 어떤 내용의 데이터인지 | 세계 주요 국가별 인구 |
| 열/행 구성 | 표의 구조 | 국가명, 인구, 수도, 대륙 |
| 정렬 기준 | 데이터 정렬 방식 | 인구 많은 순 |
| 출처/정확성 | 데이터의 근거 | 2024년 기준 추정치 |
| 용도 | 활용 맥락 | 사회과 수업 자료용 |

**좋은 프롬프트 예시:**

```
초등학교 4학년 과학 '계절의 변화' 수업에 사용할 데이터 표를 만들어줘.

조건:
- 내용: 우리나라 4계절의 평균 기온과 특징
- 포함 항목: 계절, 대표 월, 평균 기온, 날씨 특징, 대표 행사
- 형식: 표 형태로 정리
- 기온 데이터는 서울 기준
```

**결과물 점검 포인트:**

- 데이터가 학습 목적에 적합한가?
- 표의 구조가 읽기 쉬운가?
- **데이터의 정확성을 확인할 수 있는가?**

⚠️ **중요: 데이터 정확성**

AI가 생성한 수치 데이터(인구, 면적, 기온 등)는 부정확할 수 있습니다. 중요한 수치는 공신력 있는 출처로 확인하세요.

## 6. 프롬프트 개선 전략

결과물이 만족스럽지 않을 때 프롬프트를 어떻게 개선할 수 있는지 알아봅니다.

### 6.1 일반적인 문제와 해결 방법

| 문제 상황 | 원인 | 해결 방법 |
|:---|:---|:---|
| 결과물이 너무 어려움 | 난이도 미지정 | "초등학교 O학년 수준으로" 추가 |
| 문항 수가 다름 | 구체적 수량 미지정 | "정확히 10문항으로" 명시 |
| 형식이 다름 | 형식 미지정 | "표 형식으로", "번호 목록으로" 지정 |
| 내용이 빠짐 | 포함 항목 미명시 | 포함할 요소를 구체적으로 나열 |
| 어조가 맞지 않음 | 톤 미지정 | "정중한 어조로", "친근하게" 추가 |

### 6.2 단계적 개선 예시

**1차 시도:**

```
분수 학습지 만들어줘
```

**결과:** 중학교 수준의 어려운 문제 10개 생성

**2차 시도 (학년 추가):**

```
초등학교 3학년 분수 학습지 만들어줘
```

**결과:** 문항 수가 20개로 너무 많음

**3차 시도 (문항 수 추가):**

```
초등학교 3학년 분수 학습지 만들어줘.
- 문항 수: 8문항
```

**결과:** 모든 문항이 같은 형식

**4차 시도 (형식 추가):**

```
초등학교 3학년 분수 학습지 만들어줘.
- 문항 수: 8문항
- 형식: 빈칸 채우기 4문항 + 계산 문제 4문항
- 난이도: 기본 (분모가 같은 분수)
```

**결과:** 원하는 학습지 완성!

## 8. 제출 문서 작성법

### 8.1 제출 문서 구성

제출 문서에는 다음 내용을 포함합니다.

**필수 포함 항목:**

1. 제작한 자료 유형
2. 최종 프롬프트 전문
3. AI 생성 결과물

**선택 포함 항목 (가산점):**

4. 중간 과정 프롬프트
5. 개선 과정 설명

### 8.2 제출 문서 예시

```
==============================================
소프트웨어교육론 2주차 2차시 실습 과제
학번: 20XXXXXX  이름: OOO
==============================================

1. 제작 자료 유형: 학습지

2. 최종 프롬프트:

초등학교 3학년 수학 '분수의 덧셈' 학습지를 만들어줘.

조건:
- 문항 수: 8문항
- 난이도: 기본 (분모가 같은 분수)
- 형식: 빈칸 채우기 4문항 + 계산 문제 4문항
- 상단에 이름 쓰는 란 포함
- 각 문항 사이에 충분한 풀이 공간 확보

3. AI 생성 결과물:

[AI가 생성한 학습지 내용 붙여넣기]

---

4. (선택) 개선 과정:

1차 프롬프트: "분수 학습지 만들어줘"
→ 문제점: 학년과 난이도가 맞지 않음

2차 프롬프트: "초등학교 3학년 분수 학습지 만들어줘"
→ 문제점: 문항 수가 너무 많음

최종 프롬프트: 위의 조건을 모두 추가하여 완성
```

## 9. 수업 정리

### 9.1 오늘 배운 내용

- 프롬프트 작성 5원칙을 실제 교육 자료 제작에 적용했습니다.
- 프롬프트를 반복적으로 개선하여 결과물의 품질을 높이는 방법을 경험했습니다.
- AI가 생성한 결과물을 비판적으로 검토하는 태도를 기를 수 있었습니다.

### 9.2 핵심 교훈

| 교훈 | 설명 |
|:---|:---|
| **구체성이 핵심** | 프롬프트가 구체적일수록 원하는 결과를 얻을 확률이 높아집니다. |
| **반복 개선** | 한 번에 완벽한 결과를 기대하기보다 점진적으로 개선합니다. |
| **비판적 검토** | AI 결과물을 무비판적으로 수용하지 않고 직접 확인합니다. |
| **인간이 주도** | AI는 도구이며, 최종 판단과 수정은 사람의 몫입니다. |

### 9.3 다음 차시 예고
