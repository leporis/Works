## 2. 수업 목표

### 2.1 학습 목표

1. 생성형 AI를 활용하여 다양한 형태의 교육 자료를 제작할 수 있다.
2. 1차시에서 배운 프롬프트 작성 원칙을 실제 자료 제작에 적용할 수 있다.
3. 결과물의 품질을 평가하고 프롬프트를 개선할 수 있다.

### 2.2 1차시 복습 연결점

| 1차시 학습 내용 | 2차시 적용 |
|:---|:---|
| 프롬프트 작성 5원칙 | 실제 자료 제작 프롬프트에 적용 |
| 모호한/명확한 프롬프트 비교 | 직접 프롬프트를 개선하며 체험 |
| 생성형 AI의 특성 이해 | 도구로서 AI를 활용하는 실습 |
| 증강 관점 | 인간이 주도하고 AI가 보조하는 협업 |

## 4. 도입 (5분)

### 4.1 실습 과제 제시 (2분)

**교수자 발화 예시:**

> "오늘은 지난 시간에 배운 프롬프트 작성 원칙을 실제로 적용해보는 시간입니다. 여러분이 초등학교 현장에서 사용할 수 있는 자료를 생성형 AI로 직접 만들어볼 거예요."

**과제 내용 안내:**

초등학교 수업에 활용할 수 있는 교육 자료를 생성형 AI로 제작하는 것이 과제입니다. 학생들은 아래 4가지 유형 중 하나를 선택합니다.

| 자료 유형 | 설명 | 난이도 |
|:---|:---|:---:|
| **학습지** | 특정 단원의 연습 문제, 활동지 | ★★☆ |
| **안내문** | 현장학습, 행사 안내문, 가정통신문 | ★☆☆ |
| **평가 문항** | 단원평가, 수행평가 문항 | ★★★ |
| **표/데이터** | 학습 자료용 데이터 표, 비교표 | ★★☆ |

### 4.2 평가 기준 안내 (2분)

**교수자 발화 예시:**

> "제출물은 두 부분으로 평가됩니다. 첫째는 프롬프트의 질, 둘째는 결과물의 품질입니다. 프롬프트 없이 결과물만 제출하면 프롬프트 점수를 받을 수 없으니 주의하세요."

**평가 배점 표:**

| 평가 영역 | 배점 | 세부 기준 |
|:---:|:---:|:---|
| **프롬프트 질** | 5점 | 명확성, 구체성, 형식 지정, 맥락 제공 |
| **결과물 품질** | 10점 | 목표 달성도(5점), 완성도(3점), 활용 가능성(2점) |
| **합계** | **15점** | |

### 4.3 제출 방법 안내 (1분)

**제출 요건:**

- 제출 시한: 수업 종료 후 10분 이내
- 제출 방법: LMS 과제란에 업로드
- 제출 형식: 문서 파일 (한글, Word, PDF 등)

**제출물 구성:**

1. 프롬프트 전문 (최종 버전)
2. AI 생성 결과물
3. (선택) 중간 과정 프롬프트 및 결과물

### 5.2 자료 유형별 지도 가이드

#### 5.2.1 학습지 제작

**프롬프트에 포함할 요소:**

| 요소 | 설명 | 예시 |
|:---|:---|:---|
| 학년 | 대상 학년 명시 | "초등학교 3학년" |
| 과목/단원 | 구체적인 단원명 | "수학 분수의 덧셈" |
| 문항 수 | 필요한 문제 개수 | "10문항" |
| 난이도 | 상/중/하 또는 구체적 기준 | "기본 난이도" |
| 형식 | 문항 유형, 배치 방식 | "객관식 5문항 + 서술형 5문항" |
| 부가 요소 | 그림, 표, 이름란 등 | "상단에 이름 쓰는 란 포함" |

**좋은 프롬프트 예시:**

```
초등학교 3학년 수학 '분수' 단원 학습지를 만들어줘.

조건:
- 문항 수: 총 8문항
- 난이도: 기본 (분모가 같은 분수의 덧셈)
- 형식: 빈칸 채우기 4문항 + 계산 문제 4문항
- 상단에 "이름: ___" 란 포함
- 각 문항 사이에 충분한 풀이 공간 확보
```

**모호한 프롬프트 예시:**

```
분수 학습지 만들어줘.
```

→ 학년, 난이도, 문항 수, 형식 모두 불명확

**순회 지도 시 확인 사항:**

- 학년 수준에 맞는 문항인가?
- 교육과정 성취기준과 연결되는가?
- 문항 형식이 다양하게 구성되었는가?

---

#### 5.2.2 안내문 제작

**프롬프트에 포함할 요소:**

| 요소 | 설명 | 예시 |
|:---|:---|:---|
| 문서 종류 | 안내문의 유형 | "현장학습 안내문" |
| 대상 | 수신자 명시 | "학부모님께" |
| 목적 | 안내 목적 | "가을 현장학습 일정 안내" |
| 포함 정보 | 필수 기재 항목 | "날짜, 장소, 준비물, 비용" |
| 톤 | 문체와 분위기 | "격식체, 정중한 어조" |
| 분량 | 적정 길이 | "A4 1장" |

**좋은 프롬프트 예시:**

```
초등학교 3학년 가을 현장학습 안내문을 작성해줘.

조건:
- 대상: 학부모님
- 형식: 가정통신문 형태
- 포함 내용: 일시(10월 15일), 장소(OO수목원), 준비물(도시락, 물, 편한 복장), 비용(입장료 무료)
- 분량: A4 1장
- 톤: 정중하고 명확한 안내체
- 하단에 회신란(참가 동의 서명) 포함
```

**모호한 프롬프트 예시:**

```
현장학습 안내문 써줘.
```

→ 일시, 장소, 대상 등 핵심 정보 없음

**순회 지도 시 확인 사항:**

- 필수 정보가 모두 포함되었는가?
- 학부모가 읽기에 적절한 어조인가?
- 회신이 필요한 경우 회신란이 있는가?

---

#### 5.2.3 평가 문항 제작

**프롬프트에 포함할 요소:**

| 요소 | 설명 | 예시 |
|:---|:---|:---|
| 과목 | 평가 과목 | "과학" |
| 단원/성취기준 | 평가 범위 | "식물의 한살이" |
| 문항 유형 | 객관식, 서술형 등 | "객관식 10문항" |
| 난이도 분포 | 난이도 비율 | "하 30%, 중 50%, 상 20%" |
| 수량 | 총 문항 수 | "10문항" |
| 정답 여부 | 정답 포함 요청 | "정답과 해설 별도 포함" |

**좋은 프롬프트 예시:**

```
초등학교 4학년 과학 '식물의 한살이' 단원평가 문제를 만들어줘.

조건:
- 문항 수: 10문항
- 유형: 4지선다형 객관식
- 난이도: 쉬움 3문항, 보통 5문항, 어려움 2문항
- 다루는 개념: 씨앗의 구조, 발아 조건, 성장 과정, 꽃의 역할
- 정답과 간단한 해설을 문항 아래에 별도로 제시
```

**모호한 프롬프트 예시:**

```
과학 시험문제 만들어줘.
```

→ 학년, 단원, 문항 수, 난이도 전부 불명확

**순회 지도 시 확인 사항:**

- 성취기준에 부합하는 문항인가?
- 난이도가 적절하게 분포되었는가?
- 오답 선지가 그럴듯한가? (너무 명백한 오답은 좋지 않음)
- 정답이 정확한가? (AI가 잘못된 정답을 제시할 수 있음)

**⚠️ 주의: 정답 검증 필수**

평가 문항의 정답은 반드시 교수자 또는 학생 본인이 검증해야 합니다. AI가 생성한 정답이 틀린 경우가 있으므로, 특히 수학 계산 문제나 과학 개념 문제는 반드시 확인합니다.

---

#### 5.2.4 표/데이터 제작

**프롬프트에 포함할 요소:**

| 요소 | 설명 | 예시 |
|:---|:---|:---|
| 데이터 종류 | 어떤 내용의 데이터인지 | "세계 주요 국가별 인구" |
| 열/행 구성 | 표의 구조 | "국가명, 인구, 수도, 대륙" |
| 정렬 기준 | 데이터 정렬 방식 | "인구 많은 순" |
| 출처/정확성 | 데이터의 근거 | "2024년 기준 추정치" |
| 용도 | 활용 맥락 | "사회과 수업 자료용" |

**좋은 프롬프트 예시:**

```
초등학교 6학년 사회과 세계 지리 수업에 사용할 '세계 주요 국가 정보' 표를 만들어줘.

조건:
- 국가 수: 10개국 (아시아 3, 유럽 3, 아메리카 2, 오세아니아 1, 아프리카 1)
- 포함 항목: 국가명, 수도, 대륙, 인구(단위: 만 명)
- 정렬: 인구 많은 순
- 형식: 표 형태로 정리
- 인구 데이터는 2024년 기준 추정치
```

**모호한 프롬프트 예시:**

```
나라 정보 표 만들어줘.
```

→ 어떤 나라, 어떤 정보, 몇 개국인지 불명확

**순회 지도 시 확인 사항:**

- 데이터가 학습 목적에 적합한가?
- 표의 구조가 읽기 쉬운가?
- 데이터의 정확성을 검증할 수 있는가?

**⚠️ 주의: 데이터 정확성 검증**

AI가 생성한 데이터(인구, 면적, 연도 등)는 부정확할 수 있습니다. 중요한 수치는 공신력 있는 출처로 확인하도록 안내합니다.

### 5.3 순회 지도 체크리스트

교수자는 실습 시간 동안 학생들의 활동을 순회하며 다음 사항을 점검합니다.

**1단계 점검 (프롬프트 설계 단계):**

| 점검 항목 | 확인 내용 |
|:---|:---|
| 자료 유형 선택 | 학생이 자신에게 맞는 유형을 선택했는가? |
| 1차시 원칙 적용 | 명확성, 맥락, 형식, 단계, 예시 원칙을 떠올리고 있는가? |
| 초안 작성 | 프롬프트 초안을 종이나 메모장에 먼저 작성하는가? |

**2단계 점검 (1차 시도 후):**

| 점검 항목 | 확인 내용 |
|:---|:---|
| 결과물 분석 | 결과물의 문제점을 스스로 파악하는가? |
| 구체적 개선점 | "뭔가 이상해요"가 아닌 구체적 문제를 인식하는가? |

**3단계 점검 (개선 단계):**

| 점검 항목 | 확인 내용 |
|:---|:---|
| 프롬프트 수정 | 구체적인 조건을 추가하거나 수정하는가? |
| 반복 시도 | 한 번에 만족하지 않고 여러 번 시도하는가? |
| 비교 분석 | 여러 결과물을 비교하며 최적의 것을 선택하는가? |

---

### 5.4 순회 지도 시 피드백 예시

학생들의 상황에 따라 다음과 같은 피드백을 제공합니다.

**상황 1: 프롬프트가 너무 짧은 경우**

| 학생 프롬프트 | 교수자 피드백 |
|:---|:---|
| "수학 학습지 만들어줘" | "어떤 학년, 어떤 단원인지 AI가 알 수 있을까요? 1차시에서 배운 '맥락 제공' 원칙을 떠올려보세요." |

**상황 2: 결과물이 만족스럽지 않다고 할 때**

| 학생 발언 | 교수자 피드백 |
|:---|:---|
| "뭔가 이상한데 뭘 고쳐야 할지 모르겠어요" | "구체적으로 어떤 점이 이상한가요? 난이도? 문항 수? 형식? 문제점을 구체화하면 프롬프트도 구체화할 수 있어요." |

**상황 3: 한 번의 시도로 만족하는 경우**

| 학생 발언 | 교수자 피드백 |
|:---|:---|
| "한 번에 잘 나왔어요. 제출하면 되나요?" | "좋아요! 그런데 이 자료를 실제 수업에서 바로 쓸 수 있을까요? 초등학생 눈높이에 맞는지, 오류는 없는지 한 번 더 검토해보세요." |

**상황 4: AI 도구 접속 문제**

| 학생 발언 | 교수자 피드백 |
|:---|:---|
| "AI가 접속이 안 돼요" | "다른 AI 도구를 시도해보세요. ChatGPT가 안 되면 Claude를 써보고, 그래도 안 되면 Gemini나 Microsoft Copilot을 사용해보세요." |

**상황 5: 자료 유형 선택을 못하는 경우**

| 학생 발언 | 교수자 피드백 |
|:---|:---|
| "뭘 만들어야 할지 모르겠어요" | "가장 익숙한 '학습지'로 시작해보세요. 담당 과목이나 좋아하는 과목의 연습문제 학습지가 만들기 가장 수월해요." |

### 5.5 1차시 프롬프트 5원칙 상기

실습 중 학생들에게 1차시에서 배운 5원칙을 상기시킬 수 있도록 다음 표를 참조합니다.

| 원칙 | 핵심 질문 | 실습 적용 |
|:---:|:---|:---|
| **명확성** | 요청이 명확한가? | "~를 만들어줘"보다 "~를 만들어줘. 조건은 A, B, C..." |
| **맥락** | 배경 정보를 주었는가? | 학년, 과목, 단원, 사용 목적 명시 |
| **형식** | 원하는 형태를 지정했는가? | 문항 수, 표 구조, 분량 등 명시 |
| **단계** | 복잡한 요청을 나누었는가? | 여러 조건이 있으면 단계별로 제시 |
| **예시** | 원하는 결과의 예시를 주었는가? | "예시: 1. 3+5=__" 형태로 보여주기 |

## 6. 정리 (10분)

### 6.1 우수 사례 공유 (3분)

**진행 방법:**

1. 실습 중 발견한 우수 사례 1~2개 선정
2. 해당 학생의 프롬프트와 결과물을 전체 공유 (동의하에)
3. 어떤 점이 좋았는지 간략히 코멘트

**우수 사례 선정 기준:**

| 기준 | 설명 |
|:---|:---|
| 프롬프트의 구체성 | 조건이 명확하고 상세하게 기술됨 |
| 반복 개선 과정 | 여러 번 수정하여 결과물을 발전시킴 |
| 결과물의 완성도 | 실제 교육 현장에서 바로 활용 가능한 수준 |
| 창의적 접근 | 독특하거나 참신한 자료 아이디어 |

**교수자 발화 예시:**

> "OO 학생의 프롬프트를 보면, 학년과 단원뿐만 아니라 문항 형식, 난이도 배분까지 구체적으로 지정했어요. 이렇게 상세하게 요청하면 AI가 더 정확한 결과물을 만들어줍니다."

---

### 6.2 과제 정리 (5분)

**안내 사항:**

- 최종 프롬프트와 결과물을 하나의 문서로 정리
- 프롬프트가 누락되면 프롬프트 점수(5점) 획득 불가
- 중간 과정을 포함하면 가산점 부여 (개선 과정을 보여주면 좋음)

**제출 문서 구성 예시:**

```
1. 제작 자료 유형: (예: 학습지)

2. 최종 프롬프트:
   [프롬프트 내용]

3. AI 생성 결과물:
   [결과물 내용]

4. (선택) 개선 과정:
   - 1차 프롬프트: ...
   - 1차 결과물: ...
   - 수정 사항: ...
   - 2차 프롬프트: ...
```

---

### 6.3 제출 안내 (2분)

**제출 요건 재확인:**

| 항목 | 내용 |
|:---|:---|
| 제출 시한 | 수업 종료 후 10분 이내 |
| 제출 장소 | LMS 과제란 |
| 파일 형식 | 한글, Word, PDF 중 택일 |
| 파일명 | 학번_이름_2주차2차시.확장자 |

**교수자 발화 예시:**

> "지금부터 5분간 제출 문서를 정리하시고, 수업 종료 후 10분 안에 LMS에 업로드해주세요. 프롬프트 전문을 반드시 포함해야 합니다."

## 7. 평가 세부 기준

### 7.1 프롬프트 질 평가 (5점)

| 점수 | 기준 |
|:---:|:---|
| 5점 | 명확성, 구체성, 형식 지정, 맥락 제공 모두 우수 |
| 4점 | 대부분의 요소가 포함되었으나 일부 미흡 |
| 3점 | 기본적인 요소는 포함되었으나 구체성 부족 |
| 2점 | 핵심 요소 일부만 포함 |
| 1점 | 매우 간략하여 프롬프트라고 보기 어려움 |
| 0점 | 프롬프트 미제출 |

### 7.2 결과물 품질 평가 (10점)

**목표 달성도 (5점):**

| 점수 | 기준 |
|:---:|:---|
| 5점 | 의도한 자료를 완벽하게 생성 |
| 4점 | 의도한 자료를 대부분 생성 |
| 3점 | 의도한 자료를 일부 생성 |
| 2점 | 의도와 다소 다른 결과물 |
| 1점 | 의도와 상당히 다른 결과물 |

**완성도 (3점):**

| 점수 | 기준 |
|:---:|:---|
| 3점 | 추가 수정 없이 바로 사용 가능 |
| 2점 | 약간의 수정 후 사용 가능 |
| 1점 | 상당한 수정이 필요 |

**활용 가능성 (2점):**

| 점수 | 기준 |
|:---:|:---|
| 2점 | 실제 초등학교 수업에서 바로 활용 가능 |
| 1점 | 수정하면 활용 가능 |
| 0점 | 활용이 어려움 |
