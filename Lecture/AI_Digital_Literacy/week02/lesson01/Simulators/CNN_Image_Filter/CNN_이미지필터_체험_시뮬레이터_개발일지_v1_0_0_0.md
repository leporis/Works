# 🔍 CNN 이미지 필터 체험 시뮬레이터 개발일지

**문서 버전: v1.0.0.0**
**대상 시뮬레이터: CNN_이미지필터_체험_시뮬레이터_v1.0.0.0**

## 1. 프로젝트 개요

### 1.1 목적

합성곱 신경망(CNN)의 핵심 원리인 **필터(커널)를 이용한 특징 추출 과정**을 시각적·체험적으로 이해할 수 있는 교육용 시뮬레이터입니다. 10×10 이진 이미지에 3×3 필터를 적용하여 합성곱 연산이 어떻게 수행되는지 한 칸씩 따라가며 관찰할 수 있습니다.

### 1.2 교육 목표

| 목표 | 설명 |
|------|------|
| 디지털 이미지의 수치 표현 이해 | 이미지가 숫자(0/255) 배열이라는 것을 체감 |
| 합성곱 연산 과정 이해 | 필터 슬라이딩 → 원소별 곱셈 → 합산의 3단계 관찰 |
| 필터 특성 비교 | 서로 다른 커널이 서로 다른 특징(윤곽선, 블러 등)을 추출함을 확인 |
| CNN 자동학습 개념 인식 | 사람이 설계하는 수동 필터와 CNN이 자동 학습하는 필터의 차이 |

### 1.3 배경

청주교육대학교 「컴퓨팅 사고와 AI 교육」 교과목의 CNN 단원 교보재로 개발되었습니다. 기존 프로젝트의 ML 시뮬레이터 포트폴리오(선형 회귀, KNN, 의사결정트리, K-means, 경사하강법, 신경망)에 이어 **CNN 합성곱 연산 체험** 시뮬레이터를 추가합니다. 별도로 제작 중인 CNN 이론해설서의 실습 도구 역할도 겸합니다.

## 2. 시스템 설계

### 2.1 입출력 설계

```
┌────────────────────┐      ┌──────────────┐      ┌────────────────────┐
│    입력 이미지      │      │  필터(커널)   │      │    출력 결과       │
│    (10×10 이진)    │  ✱   │  (3×3 가중치) │  →   │    (8×8 특징맵)   │
│                    │      │              │      │                    │
│  값: 0 또는 255    │      │  값: 정수     │      │  값: 실수          │
│  프리셋 8종        │      │  필터 7종     │      │  색상 매핑 표시    │
│  직접 그리기 가능   │      │              │      │                    │
└────────────────────┘      └──────────────┘      └────────────────────┘
```

### 2.2 합성곱 알고리즘

```
입력: grid[N][N], kernel[K][K], divisor (선택)
출력: output[N-K+1][N-K+1]

for r = 0 to N-K:
    for c = 0 to N-K:
        sum = 0
        for kr = 0 to K-1:
            for kc = 0 to K-1:
                sum += grid[r+kr][c+kc] × kernel[kr][kc]
        if divisor: sum /= divisor
        output[r][c] = sum
```

### 2.3 설정 상수 (CONFIG)

| 카테고리 | 키 | 값 | 설명 |
|---------|-----|:---:|------|
| GRID | SIZE | 10 | N×N 입력 그리드 크기 |
| GRID | CELL_ON_VALUE | 255 | 채워진 셀 내부값 |
| GRID | CELL_OFF_VALUE | 0 | 빈 셀 내부값 |
| GRID | CELL_ON_COLOR | #1A237E | 채워진 셀 표시 색상 |
| GRID | CELL_OFF_COLOR | #FAFAFA | 빈 셀 표시 색상 |
| KERNEL | SIZE | 3 | K×K 커널 크기 |
| OUTPUT | POSITIVE_COLOR | [21,101,192] | 양수 출력 RGB (파란) |
| OUTPUT | NEGATIVE_COLOR | [198,40,40] | 음수 출력 RGB (빨간) |
| OUTPUT | ZERO_COLOR | [236,239,241] | 0 출력 RGB (회색) |
| OUTPUT | CLAMP_VALUE | 255 | 출력 클램핑 최대값 |
| ANIMATION | DEFAULT_SPEED | 300 | 기본 재생 속도 (ms) |
| ANIMATION | MIN_SPEED | 50 | 최소 속도 (가장 빠름) |
| ANIMATION | MAX_SPEED | 800 | 최대 속도 (가장 느림) |

### 5.4 출력 색상 매핑 알고리즘

```
1. 전체 출력값에서 최대 절대값(maxAbs) 계산
2. 각 셀의 정규화값 = val / maxAbs (범위: -1.0 ~ +1.0)
3. 색상 결정:
   - norm > 0.01  → 파란색(POSITIVE_COLOR), 투명도 = norm×0.8 + 0.1
   - norm < -0.01 → 빨간색(NEGATIVE_COLOR), 투명도 = |norm|×0.8 + 0.1
   - |norm| ≤ 0.01 → 회색(ZERO_COLOR)
4. 텍스트 색상: 투명도 > 0.5이면 흰색, 아니면 검정
```
