## ⚡ 1분 초속성 실습

| 단계 | 행동 | 확인 |
|:---:|------|:----:|
| 1 | 왼쪽 **[━ 가로선]** 프리셋 클릭 | 10×10 그리드에 가로 줄무늬가 나타났나요? |
| 2 | **[⚡ 즉시 적용]** 버튼 클릭 | 오른쪽 출력 그리드에 색상이 나타났나요? |
| 3 | 왼쪽 필터 목록에서 **[┃ 세로 윤곽선]** 클릭 | 출력이 자동으로 바뀌었나요? → 거의 안 보이면 🎉 |

> 💡 **왜 안 보일까요?** 세로 윤곽선 필터는 세로 방향 변화만 찾는데, 입력에는 가로선만 있으니까요! 이것이 **필터의 핵심 원리**입니다.

## 🚀 빠른 시작 (3단계)

### 1️⃣ 입력 이미지 만들기

왼쪽 사이드바 **📋 입력 이미지** 패널에서 선택합니다.

| 방법 | 설명 |
|------|------|
| **프리셋 선택** | 8가지 기본 도형 중 클릭 (가로선, 세로선, 십자, 대각선, 사각형, 원형, T 글자, 화살표) |
| **직접 그리기** | 입력 그리드에서 **좌클릭 = 그리기**, **우클릭 = 지우기** |
| **도구 활용** | 🗑️ 지우기(전체 초기화), 🔄 반전(흑백 뒤바꿈) |

### 2️⃣ 필터 선택하기

**🎛️ 필터(커널) 선택** 패널에서 7가지 필터 중 하나를 클릭합니다.

| 필터 | 한마디 | 태그 |
|------|--------|:----:|
| ⬜ 원본 (항등) | 아무것도 안 함 — 비교 기준! | 기본 |
| ━ 가로 윤곽선 | 가로선만 찾는 탐정 | 윤곽 |
| ┃ 세로 윤곽선 | 세로선만 찾는 탐정 | 윤곽 |
| ✦ 전체 윤곽선 | 모든 선을 찾는 만능 탐정 | 윤곽 |
| 🔺 선명하게 | 경계를 또렷하게 | 선명 |
| 🌫️ 흐리게 (블러) | 부드럽게 뭉개기 | 흐림 |
| 🏔️ 엠보싱 | 입체감 살리기 | 입체 |

> 💡 출력 결과가 이미 있으면, 필터를 바꿀 때 **자동으로 재적용**됩니다.

### 3️⃣ 합성곱 과정 관찰하기

| 버튼 | 기능 | 추천 상황 |
|------|------|----------|
| **⚡ 즉시 적용** | 전체 결과를 한 번에 표시 | 결과만 빨리 보고 싶을 때 |
| **▶️ 단계별 재생** | 필터가 한 칸씩 이동하며 계산 | 합성곱 원리를 이해하고 싶을 때 |
| **⏭️ 한 칸 이동** | 다음 위치 한 칸만 계산 | 특정 위치를 천천히 보고 싶을 때 |
| **⏹️ 정지** | 재생 중지 | 재생 중 멈추고 싶을 때 |

> 💡 **속도 슬라이더**로 애니메이션 속도를 50ms(빠름)~800ms(느림) 조절할 수 있어요.

## 🔍 이런 장면을 찾아보세요!

| 관찰 대상 | 찾아볼 장면 | 의미 |
|----------|------------|------|
| 입력 하이라이트 | 주황 테두리가 3×3 영역을 감싸고 이동 | 필터가 슬라이딩하는 현재 위치 |
| 출력 색상 | 파란색이 진한 셀 | 필터가 강하게 반응한 곳 (양수 큰 값) |
| 출력 색상 | 빨간색이 진한 셀 | 필터가 반대로 반응한 곳 (음수 큰 값) |
| 출력 색상 | 회색 셀 | 필터가 거의 반응하지 않은 곳 (0에 가까움) |
| 계산 과정 | `255×1 + 0×0 + 255×(-1) = 0` | 입력값 × 필터값을 더한 합성곱 결과 |
| 커널 표시 | 파란 셀 = 양수, 빨간 셀 = 음수 | 필터 커널의 가중치 부호 |

## 🆘 도와주세요!

| 증상 | 원인 | 해결책 |
|------|------|--------|
| 출력 그리드가 비어 있어요 | 필터가 아직 적용되지 않음 | **[⚡ 즉시 적용]** 또는 **[▶️ 단계별 재생]** 클릭 |
| 출력이 전부 회색이에요 | 입력 이미지가 비어 있음 | 프리셋을 선택하거나 직접 그리기 |
| 출력이 전부 회색이에요 (그림 있는데!) | 원본(항등) 필터가 아니라 윤곽선 필터인데 특징이 없는 방향 | 다른 프리셋이나 다른 필터를 시도해 보세요 |
| 계산 과정이 안 보여요 | 체크박스가 꺼짐 | ⚙️ 옵션 → **[계산 과정 표시]** 체크 |
| 셀 숫자가 안 보여요 | 체크박스가 꺼짐 | ⚙️ 옵션 → **[셀 값 표시]** 체크 |
| 그리기가 안 돼요 | 출력 그리드를 클릭함 | **입력 이미지** (왼쪽 그리드)에서 그리세요 |
| 단계별 재생이 너무 빨라요 | 속도 설정이 낮음 | 속도 슬라이더를 오른쪽(800ms)으로 이동 |
| 화면이 깨져 보여요 | 브라우저 창이 좁음 | 브라우저 창을 넓히거나, 1024px 이상 권장 |

## 🔑 핵심 용어

| 용어 | 학술 용어 | 의미 | 쉬운 설명 |
|------|----------|------|----------|
| 합성곱 | Convolution | 필터와 입력의 원소별 곱셈 후 합산 | 돋보기로 한 칸씩 살펴보며 점수 매기기 |
| 필터 / 커널 | Filter / Kernel | 특징 추출용 작은 숫자 배열 (3×3) | "이런 패턴을 찾아라!"라고 적힌 레시피 |
| 윤곽선 검출 | Edge Detection | 밝기가 급변하는 경계 찾기 | 그림의 테두리선만 골라내기 |
| 슬라이딩 | Sliding | 필터가 입력 위를 한 칸씩 이동 | 돋보기를 왼쪽→오른쪽, 위→아래로 옮기기 |
| 출력 맵 | Feature Map | 합성곱 결과로 나온 숫자 배열 | 필터가 매긴 "반응 점수표" |
| 블러 | Blur | 주변 값의 평균을 구해 부드럽게 | 안경 없이 보는 것처럼 뿌옇게 |
| 엠보싱 | Embossing | 입체감을 주는 필터 효과 | 종이에 눌러 찍은 듯한 볼록/오목 느낌 |
| CNN | Convolutional Neural Network | 합성곱 신경망 | 필터를 자동으로 학습하는 AI |

## 💭 생각해보기

> **"가로 윤곽선 필터"로 가로선 프리셋을 적용하면 강하게 반응하는데, 세로선 프리셋에는 거의 반응하지 않습니다. 그렇다면 CNN이 고양이 사진에서 "귀의 윤곽"을 찾으려면, 어떤 종류의 필터가 필요할까요? 그리고 그 필터는 누가 만들까요?**

> 💡 힌트: CNN은 필터 값을 사람이 정하지 않고, **학습 데이터에서 자동으로** 찾아냅니다!
