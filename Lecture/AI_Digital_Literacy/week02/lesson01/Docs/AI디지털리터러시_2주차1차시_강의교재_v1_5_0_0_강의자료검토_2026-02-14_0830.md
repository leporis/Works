# 📘 제1장: 데이터의 역할 — AI 학습의 연료

## 1.1 머신러닝 4단계 복습

> 📌 **핵심 질문**: "AI가 학습하려면 무엇이 필요할까요?"

지난 시간에 배운 머신러닝의 학습 과정 4단계를 떠올려 봅시다.

**[그림 01] 머신러닝 4단계 프로세스** (1주차 2차시 재사용)

![머신러닝 4단계 프로세스](images/AI디지털리터러시_2주차1차시_01_머신러닝4단계프로세스.png)

| 단계 | 내용 | 비유 |
|:---:|:---|:---|
| **① 데이터 수집** | 학습할 데이터 모으기 | 📚 문제집 사기 |
| **② 학습** | 데이터에서 패턴 찾기 | ✍️ 열심히 공부하기 |
| **③ 평가** | 정확도 확인 | 📋 시험 보기 |
| **④ 예측** | 새 데이터에 적용 | 🎯 실전 응용 |

4단계 중 가장 먼저 나오는 것이 **데이터 수집**입니다. 좋은 학습은 좋은 데이터에서 시작됩니다. 오늘은 이 데이터에 대해 깊이 알아봅니다.

\newpage

## 1.2 데이터 = AI 학습의 연료

AI에게 데이터는 자동차의 **연료**와 같습니다.

| 자동차 🚗 | AI 🤖 |
|:---:|:---:|
| 좋은 연료 → 잘 달린다 | 좋은 데이터 → 정확한 AI |
| 나쁜 연료 → 고장난다 | 나쁜 데이터 → 부정확한 AI |
| 연료 없으면 → 멈춘다 | 데이터 없으면 → 학습 불가 |

AI의 성능은 결국 **어떤 데이터로 학습했느냐**에 달려 있습니다. 아무리 뛰어난 알고리즘도 나쁜 데이터로는 좋은 결과를 낼 수 없어요. 이것이 AI 분야에서 자주 인용되는 말의 배경이에요.

> 🗣️ **"Garbage In, Garbage Out"** (쓰레기가 들어가면, 쓰레기가 나온다)

\newpage

## 1.3 AI가 학습하는 데이터의 유형

**[그림 02] 데이터 유형 — 이미지, 텍스트, 소리**

![데이터 유형 — 이미지, 텍스트, 소리](images/AI디지털리터러시_2주차1차시_02_데이터유형_이미지텍스트소리_v1_0_0_0_신규작성_20260213_2230.png)

AI가 학습하는 데이터는 크게 세 가지 유형으로 나눌 수 있습니다.

### 🖼️ 이미지 데이터

컴퓨터에게 이미지는 **숫자의 격자**입니다. 각 픽셀(점)이 색상 값을 가지고 있어요. 예를 들어, 흑백 이미지에서 0은 흰색, 255는 검은색을 의미합니다.

| 항목 | 설명 |
|:---|:---|
| **형태** | 픽셀(점)로 구성된 2차원 격자 |
| **값의 범위** | 흑백: 0(흰)~255(검), 컬러: RGB 3채널 |
| **AI 활용** | 이미지 분류, 객체 인식, 의료 진단 |
| **일상 예시** | 고양이/개 사진 구분, 얼굴 인식, X-ray 판독 |
| **교육 연결** | 3~4주차 엔트리 이미지 분류 모델에서 직접 체험 |

### 📝 텍스트 데이터

우리가 매일 쓰고 읽는 글자도 AI의 학습 재료가 됩니다. AI는 단어와 문장의 패턴을 학습하여 의미를 파악합니다.

| 항목 | 설명 |
|:---|:---|
| **형태** | 문자, 단어, 문장의 연속 |
| **처리 방법** | 단어를 숫자(벡터)로 변환하여 계산 |
| **AI 활용** | 감정 분석, 기계 번역, 텍스트 요약, 대화 |
| **일상 예시** | 스팸 메일 필터, 네이버 파파고, ChatGPT |
| **교육 연결** | 2차시 생성형 AI(LLM)에서 상세히 학습 |

### 🎵 소리 데이터

음성, 음악, 환경 소리도 모두 디지털 데이터로 변환되어 AI의 학습 재료가 됩니다.

| 항목 | 설명 |
|:---|:---|
| **형태** | 시간에 따른 음파 신호 (진폭과 주파수) |
| **처리 방법** | 음파를 스펙트로그램(시각적 표현)으로 변환 |
| **AI 활용** | 음성 인식, 음악 추천, 소리 분류 |
| **일상 예시** | "시리야", "알렉사", 음악 앱의 노래 인식 |
| **교육 연결** | 3주차 엔트리 음성인식 블록에서 직접 체험 |

> 💡 **초등교육 연결**: "우리가 교과서로 공부하듯이, AI는 이미지·글·소리를 '디지털 교과서'로 공부합니다. 어떤 교과서를 주느냐에 따라 AI의 실력이 달라져요."

## 1.4 데이터의 양과 질

AI 학습에서는 데이터의 **양**과 **질** 모두 중요합니다.

### 양(Quantity): 많을수록 좋지만...

데이터가 많을수록 AI는 더 다양한 패턴을 학습할 수 있습니다. 예를 들어, 고양이 사진 100장보다 10,000장으로 학습하면 다양한 종류·각도·환경의 고양이를 인식할 수 있어요.

### 질(Quality): 양보다 더 중요한 것

그러나 **양이 아무리 많아도 질이 나쁘면 문제**입니다.

| 상황 | 결과 |
|:---|:---|
| 양 ↑ + 질 ↑ | ✅ 좋은 AI — 이상적인 상황 |
| 양 ↑ + 질 ↓ | ⚠️ 여전히 문제 — 편향·오류가 확대됨 |
| 양 ↓ + 질 ↑ | 🔶 제한적이지만 정확한 AI |
| 양 ↓ + 질 ↓ | ❌ 나쁜 AI — 학습 자체가 어려움 |

"실내에서 찍은 고양이 사진만 10만 장" vs "다양한 환경(실내, 실외, 밤, 낮)의 고양이 사진 1만 장" — 어느 쪽이 더 좋은 학습 데이터일까요?

다양한 조건의 1만 장이 더 좋습니다. 한쪽에 치우친 데이터는 양이 많아도 **편향** 문제를 일으키기 때문이에요. 이 문제를 다음 장에서 자세히 알아봅시다.

# 📗 제2장: 데이터 편향 — AI도 편견을 배운다

## 2.1 편향이란 무엇인가?

> 📌 **핵심 질문**: "데이터가 한쪽에 치우치면 어떤 일이 벌어질까요?"

**편향(Bias)** = 데이터가 특정 방향에 치우친 상태

1주차 2차시에서 머신러닝 학습 과정을 배우면서, 데이터가 편향될 수 있다는 점을 간단히 다뤘어요. 오늘은 실제 사례를 통해 편향이 얼마나 심각한 문제를 일으킬 수 있는지 알아봅니다.

**[그림 03] 편향된 데이터 vs 좋은 데이터** (1주차 2차시 재사용)

![편향된 데이터 vs 좋은 데이터](images/AI디지털리터러시_2주차1차시_03_편향된데이터vs좋은데이터.png)

편향을 비유하면 **한쪽에 치우친 안경**을 쓰고 세상을 보는 것과 같습니다.

| 좋은 안경 🤓 | 삐뚤어진 안경 🥴 |
|:---:|:---:|
| 다양한 데이터 | 치우친 데이터 |
| 세상을 정확히 봄 | 세상을 삐뚤게 봄 |
| 공정한 판단 | 불공정한 판단 |

## 2.2 실제 편향 사례

### 사례 1: 👤 얼굴인식 AI의 편향

**문제**: 특정 피부색의 얼굴 인식 정확도가 현저히 낮은 AI

| 항목 | 내용 |
|:---|:---|
| **연구** | MIT 미디어랩 조이 부올라므위니(Joy Buolamwini) 연구 |
| **발견** | 밝은 피부 남성: 오류율 0.8% vs 어두운 피부 여성: 오류율 34.7% |
| **원인** | 학습 데이터의 약 77%가 밝은 피부, 남성 위주 |
| **영향** | 공항 보안, 경찰 신원 확인 등에서 차별적 결과 |

이 연구는 전 세계적으로 큰 반향을 일으켰습니다. AI가 아무리 뛰어나도, 학습 데이터가 편향되면 **차별적인 결과**를 만들어냅니다.

### 사례 2: 💼 채용 AI의 편향

**문제**: 여성 지원자에게 체계적으로 불이익을 주는 AI

| 항목 | 내용 |
|:---|:---|
| **기업** | 글로벌 IT 기업 (2018년 보도) |
| **발견** | 이력서에 "여성" 관련 단어(여대, 여성 동아리) → 감점 |
| **원인** | 과거 10년간 채용 기록 학습 (당시 남성 위주 합격) |
| **결과** | AI는 "남성 = 좋은 지원자"라고 잘못 학습 |
| **조치** | 해당 기업은 이 AI 시스템을 **폐기** |

과거 데이터에 이미 편향이 존재했고, AI는 이 편향을 그대로 학습한 것입니다. **AI는 편견도 학습합니다**.

### 사례 3: 🌐 번역 AI의 편향

**문제**: 직업에 따라 성별을 자동으로 고정하여 번역

| 원문 (성별 중립) | AI 번역 결과 |
|:---|:---|
| "그 사람은 의사입니다" | "**He** is a doctor" (항상 남성) |
| "그 사람은 간호사입니다" | "**She** is a nurse" (항상 여성) |
| "그 사람은 CEO입니다" | "**He** is a CEO" (항상 남성) |

인터넷에서 수집한 텍스트 데이터에 이미 성별 고정관념이 포함되어 있기 때문입니다. AI가 이 패턴을 그대로 반영한 결과입니다.

\newpage

## 2.3 편향의 원인과 해결 방향

편향은 왜 발생하고, 어떻게 해결할 수 있을까요?

| 원인 | 구체적 예 | 해결 방향 |
|:---|:---|:---|
| **수집 범위의 편향** | 밝은 피부 사진 위주 수집 | 다양한 집단의 데이터 **균형 수집** |
| **역사적 편향** | 과거 남성 위주 채용 기록 | 편향 감지 도구로 **사전 점검** |
| **고정관념 포함** | 성별 고정관념이 포함된 텍스트 | 데이터 전처리 과정에서 **편향 제거** |
| **수집자의 무의식** | 특정 지역/환경만 촬영 | **다양성 체크리스트** 활용 |

### 교육에서의 의미

초등학생에게 AI를 가르칠 때 중요한 것은 "데이터가 중요하다"는 사실만이 아닙니다. **"다양한 데이터가 공정한 AI를 만든다"**는 점까지 이해시키는 것이 핵심입니다.

3~4주차 엔트리 실습에서 학생들이 직접 이미지 데이터를 수집할 때, 다음과 같이 안내할 수 있습니다.

| 잘못된 수집 예 | 올바른 수집 예 |
|:---|:---|
| 밝은 곳에서만 촬영 | 밝은 곳 + 어두운 곳 |
| 한 각도에서만 촬영 | 여러 각도에서 촬영 |
| 비슷한 대상만 촬영 | 다양한 크기·색상·배경 |

# 📙 제3장: 딥러닝과 신경망 — AI의 뇌 구조

## 3.1 신경망이란 무엇인가?

> 📌 **핵심 질문**: "AI의 뇌는 어떻게 생겼을까요?"

1주차 1차시에서 AI ⊃ ML ⊃ DL의 포함 관계를 배웠습니다. 오늘은 이 중에서 가장 안쪽에 있는 **DL(딥러닝, Deep Learning)**에 대해 자세히 알아봅니다.

**신경망(Neural Network)**은 인간의 뇌에서 **영감**을 얻은 AI 모델입니다.

> ⚠️ **주의**: "영감"이라는 단어가 중요합니다. 신경망은 실제 뇌와 같지 않습니다. 뇌의 연결 구조에서 아이디어를 얻었을 뿐, 작동 방식은 매우 다릅니다.

### ⭐ 심화: 생물학적 뉴런 vs 인공 뉴런

> 📖 **심화 학습**: 뉴런·퍼셉트론 비교는 신경망 이해를 돕기 위한 심화 내용입니다. 퀴즈 출제 범위에 포함되지 않습니다.

**[그림 04] 뉴런의 관점 — 생물학적 뇌 vs 인공지능**

![뉴런과 퍼셉트론](images/AI디지털리터러시_2주차1차시_04_뉴런과퍼셉트론_v1_0_0_0_신규작성_20260213_2230.png)

우리 뇌의 신경세포(뉴런)와 인공 뉴런(퍼셉트론)을 비교해 봅시다.

| 생물학적 뉴런 | 역할 | 인공 뉴런 (퍼셉트론) |
|:---:|:---:|:---:|
| **수상돌기** | 신호 수용 & 중요도 반영 | **입력값 × 가중치** |
| **세포체** | 정보 통합 & 발화 여부 결정 | **합산 및 편향 (Σ+)** |
| **축삭 → 시냅스** | 결과 전달 & 다음 단계 연결 | **활성화 함수 → 출력값** |

> 💡 **핵심 메시지**: 인공 뉴런은 생물학적 뉴런의 정보 처리 방식을 수학적으로 **'모방'**하여 데이터를 학습하고 판단합니다. 다만, 실제 뇌의 뉴런은 유연하고 복잡한 전기/화학 반응으로 작동하는 반면, 인공 뉴런은 정교한 수학적 연산과 모델링으로 동작한다는 차이가 있습니다.

\newpage

## 3.2 신경망의 기본 구조

**[그림 05] 신경망 기본 구조 — 입력층 → 은닉층 → 출력층**

![신경망 기본 구조](images/AI디지털리터러시_2주차1차시_05_신경망기본구조_v1_0_0_0_신규작성_20260213_2230.png)

신경망은 크게 **3가지 층**으로 구성됩니다.

### 입력층 (Input Layer) — 👁️ 눈

데이터가 숫자로 변환되어 들어오는 곳입니다.

| 데이터 종류 | 입력 예시 |
|:---|:---|
| 과일 데이터 | "크기=7.5", "빨간 정도=8.2" → 숫자 2개 |
| 이미지 데이터 | 각 픽셀의 밝기 값 (0~255) → 픽셀 수만큼의 숫자 |
| 텍스트 데이터 | 단어를 숫자(벡터)로 변환 |

### 은닉층 (Hidden Layer) — 🧠 뇌

입력 데이터에서 **특징(Feature)**을 추출하는 곳입니다. "은닉"이라 불리는 이유는 외부에서 직접 관찰하기 어렵기 때문이에요.

| 역할 | 설명 |
|:---|:---|
| 특징 추출 | 데이터의 중요한 패턴을 찾아냄 |
| 가중치 조정 | 어떤 특징이 중요한지 판단 → **이것이 학습!** |
| 비선형 변환 | 단순한 입력을 복잡한 표현으로 변환 |

### 출력층 (Output Layer) — ✋ 손

최종 결과가 나오는 곳입니다.

| 문제 유형 | 출력 예시 |
|:---|:---|
| 분류 | "사과 90%, 오렌지 5%, 레몬 3%, 포도 2%" |
| 예측 | "내일 기온: 22.5°C" |
| 생성 | "The cat sat on the ___" → "mat" |

## 3.3 신경망의 학습 원리 — 가중치와 연결

신경망에서 학습이란 무엇일까요? 바로 **가중치(Weight)**를 조정하는 것입니다.

각 노드(뉴런) 사이의 연결에는 **가중치**라는 숫자가 있습니다. 이 가중치가 클수록 해당 연결이 중요하다는 의미예요.

| 학습 전 | 학습 후 |
|:---|:---|
| 모든 가중치가 무작위 값 | 중요한 연결은 큰 가중치 |
| "아무것도 모르는 상태" | "패턴을 파악한 상태" |
| 분류 정확도 낮음 (거의 랜덤) | 분류 정확도 높음 |

학습 과정에서 AI는 다음을 반복합니다.

1. **순전파**: 입력 데이터를 넣고 결과를 계산
2. **오차 계산**: 정답과 비교하여 얼마나 틀렸는지 확인
3. **역전파**: 오차를 줄이는 방향으로 가중치 조정
4. **반복**: 전체 데이터를 여러 번 반복 (에포크)

> 💡 **비유**: "시험을 보고(순전파), 채점한 뒤(오차계산), 틀린 부분을 복습하고(역전파), 다시 문제를 푸는(반복) 과정과 같아요!"

## 3.4 🍎 신경망과일분류시뮬레이터 체험

이론을 시뮬레이터로 직접 확인해 봅시다!

### 시뮬레이터 사양

| 항목 | 내용 |
|:---|:---|
| **이름** | 신경망과일분류시뮬레이터 v1.0.0.0 |
| **입력 (2개)** | 크기(Size), 빨간 정도(Redness) |
| **은닉층 (6개)** | 특징 추출 뉴런 6개 |
| **출력 (4개)** | 🍎 사과, 🍊 오렌지, 🍋 레몬, 🍇 포도 |
| **학습 데이터** | 12개 (과일 4종 × 3개씩) |

### 체험 활동 ① 구조 관찰

시뮬레이터를 열면, 신경망의 구조가 시각적으로 표시됩니다.

| 관찰 포인트 | 확인 내용 |
|:---|:---|
| 왼쪽 **입력 노드 2개** | "크기"와 "빨간 정도" — 과일을 숫자 2개로 표현 |
| 가운데 **은닉 노드 6개** | 특징을 분석하는 뉴런들 |
| 오른쪽 **출력 노드 4개** | 사과, 오렌지, 레몬, 포도 — 분류 결과 |
| **연결선** | 모든 노드가 연결 → 각 연결에 가중치 존재 |

> 📌 "입력 2개에서 출력 4개까지, 이 모든 연결선에 **가중치**라는 숫자가 있어요. 학습이란 바로 이 가중치를 조정하는 것입니다!"

### 체험 활동 ② 학습 과정 관찰

[학습 시작] 버튼을 클릭하여 학습 과정을 관찰합니다.

| 관찰 포인트 | 변화 |
|:---|:---|
| **연결선 굵기** | 굵어지거나 얇아짐 = 가중치 조정 중 |
| **에포크(Epoch) 수** | 증가 = 전체 데이터 반복 학습 |
| **오차(Loss)** | 감소 = 학습이 잘 되는 중 |
| **정확도** | 상승 = 분류를 잘하게 됨 |

연결선 굵기가 변하는 것이 보이시죠? 이것이 바로 **학습**입니다. AI가 "이 연결은 중요하니까 더 강하게, 이 연결은 덜 중요하니까 약하게" 조정하고 있어요.

### 체험 활동 ③ 학습 결과 테스트

학습이 완료된 후, 새로운 데이터로 테스트합니다.

| 테스트 방법 | 설명 | 기대 결과 |
|:---|:---|:---|
| **테스트 카드** | 미리 준비된 과일 데이터 | 대부분 정확히 분류 ✅ |
| **랜덤 입력** | 임의의 크기/빨간 정도 | 어떤 과일로 분류되는지 관찰 |
| **경계 영역** | 애매한 값 (예: 중간 크기, 중간 색) | 확률이 비슷하게 나뉨 |

> 💡 "학습하지 않은 **새 과일**도 분류할 수 있어요! 이것이 머신러닝의 핵심 — 패턴을 학습해서 새로운 상황에 적용하는 것입니다."

## 3.5 딥러닝 = 깊은(Deep) 신경망

지금 체험한 시뮬레이터는 은닉층이 **1개**인 **얕은 신경망**입니다. 만약 은닉층을 여러 개로 늘리면 어떻게 될까요?

| 구분 | 얕은 신경망 (Shallow) | 깊은 신경망 (Deep) |
|:---:|:---|:---|
| **은닉층 수** | 1~2개 | 수십~수백 개 |
| **학습 능력** | 단순한 패턴 (크기/색으로 구분) | 매우 복잡한 패턴 (사진으로 구분) |
| **예시** | 과일 분류 (숫자 2개) | 이미지 분류 (픽셀 수만 개) |
| **이름** | 기계학습 (ML) | **딥러닝 (DL)** |

**딥러닝(Deep Learning) = "깊은 학습"**

왜 "깊다(Deep)"고 할까요? **은닉층이 여러 개** 쌓여 있기 때문입니다!

### 은닉층이 많으면 무엇이 달라지나?

**[그림 06] 딥러닝 특징 추출 — 단계별 과정**

![딥러닝 특징 추출](images/AI디지털리터러시_2주차1차시_06_딥러닝특징추출_v1_0_0_0_신규작성_20260213_2230.png)

각 은닉층은 점점 더 **복잡한 특징**을 추출합니다.

| 은닉층 단계 | 추출하는 특징 | 예시 (고양이 사진 인식) |
|:---:|:---|:---|
| **1층** (초기) | 선, 윤곽선, 모서리 | 가로선, 세로선, 대각선 |
| **2층** | 모양, 질감 | 동그란 형태, 줄무늬 패턴 |
| **3층** | 부분 (Part) | 눈, 귀, 코 등 부분 인식 |
| **4층+** (깊은 층) | 전체 객체 | "이것은 고양이 얼굴이다!" |

1층에서는 단순한 선만 보이지만, 층이 깊어질수록 점점 복잡한 특징을 인식하게 됩니다. 마치 우리가 그림을 볼 때 "선 → 모양 → 부분 → 전체"로 인식하는 과정과 비슷해요.

이것이 딥러닝의 핵심 원리입니다. **층이 많을수록 더 복잡한 패턴을 학습할 수 있습니다.**

## ⭐ 3.6 심화: CNN — 이미지 전문가 신경망

> 📖 **심화 학습**: CNN(합성곱 신경망) 개념과 시뮬레이터 체험(§3.6~3.7)은 딥러닝 이해를 돕기 위한 심화 내용입니다. 퀴즈 출제 범위에 포함되지 않습니다.

딥러닝 중에서 **이미지 처리에 특화된 구조**가 있습니다. 바로 **CNN(Convolutional Neural Network, 합성곱 신경망)**입니다.

### CNN의 핵심 아이디어: 필터(커널)

CNN의 핵심은 **필터(커널, Kernel)**라는 작은 격자를 이미지 위에서 이동하며 특징을 추출하는 것입니다.

| 개념 | 설명 | 비유 |
|:---|:---|:---|
| **필터(커널)** | 특정 특징을 감지하는 작은 격자 (예: 3×3) | 🔍 특수 안경 — 각 안경마다 다른 것이 보임 |
| **합성곱** | 필터를 이미지 위에서 한 칸씩 이동하며 계산 | 돋보기로 이미지를 훑어봄 |
| **특징 맵** | 합성곱의 결과 이미지 | 특수 안경으로 본 결과물 |

### 필터의 종류와 역할

같은 이미지에 서로 다른 필터를 적용하면, 다른 특징이 추출됩니다.

| 필터 종류 | 추출하는 특징 | 역할 |
|:---|:---|:---|
| 가로 윤곽선 필터 | 가로 방향의 선 | 수평 경계 감지 |
| 세로 윤곽선 필터 | 세로 방향의 선 | 수직 경계 감지 |
| 전체 윤곽선 필터 | 모든 방향의 경계 | 물체의 전체 형태 감지 |

**CNN의 핵심**: 이런 필터를 사람이 만드는 것이 아니라, **AI가 데이터로부터 자동으로 학습**합니다!

## ⭐ 3.7 심화: 🔍 CNN 이미지필터 시뮬레이터 체험

필터가 실제로 어떻게 작동하는지 직접 체험해 봅시다!

### 시뮬레이터 사양

| 항목 | 내용 |
|:---|:---|
| **이름** | CNN 이미지필터 체험 시뮬레이터 v1.0.0.0 |
| **입력** | 10×10 픽셀 그리드 (클릭/드래그로 그리기, 흑백) |
| **프리셋 이미지** | 8종 (가로선, 세로선, 십자, 대각선, 사각형, 원형, T글자, 화살표) |
| **필터(커널)** | 7종 3×3 커널 |
| **출력** | 8×8 결과 그리드 (양수=파랑, 음수=빨강, 0=회색) |
| **시각화** | 합성곱 연산 단계별 애니메이션 |

### 사용 가능한 7가지 필터

| # | 필터 이름 | 효과 |
|:---:|:---|:---|
| 1 | 원본 (항등) | 입력 이미지 그대로 유지 |
| 2 | 가로 윤곽선 | 가로 방향의 선만 강조 |
| 3 | 세로 윤곽선 | 세로 방향의 선만 강조 |
| 4 | 전체 윤곽선 (라플라시안) | 모든 방향의 윤곽선 강조 |
| 5 | 선명하게 | 이미지의 경계를 또렷하게 |
| 6 | 흐리게 (블러) | 이미지를 부드럽게 |
| 7 | 엠보싱 | 양각 효과 (입체감) |

### 체험 활동 ① 입력 이미지 만들기

1. 프리셋에서 **"십자(+)"** 모양을 선택합니다
2. 10×10 격자에 + 모양이 표시됩니다
3. 검은 칸 = 255, 흰 칸 = 0으로 AI에게 이미지가 전달됩니다

> 📌 "AI에게 이미지는 이런 숫자 격자로 보입니다. 우리 눈에는 그림이지만, AI에게는 숫자예요!"

### 체험 활동 ② 필터 적용 및 결과 비교

같은 십자(+) 이미지에 서로 다른 필터를 적용해 봅니다.

| 순서 | 적용할 필터 | 관찰 포인트 |
|:---:|:---|:---|
| 1 | **가로 윤곽선** | 가로 방향 선만 강조 → 세로선은 사라짐! |
| 2 | **세로 윤곽선** | 세로 방향 선만 강조 → 가로선은 사라짐! |
| 3 | **전체 윤곽선** | 십자의 모든 윤곽선이 한꺼번에! |

> 💡 "같은 그림인데 필터에 따라 결과가 완전히 달라지죠? 각 필터는 이미지에서 **특정 특징만 골라봅니다**."

### 체험 활동 ③ 합성곱 애니메이션 관찰

[단계별 재생] 버튼을 클릭하면, 필터가 이미지 위를 한 칸씩 이동하는 과정을 볼 수 있습니다.

| 관찰 포인트 | 설명 |
|:---|:---|
| 입력 이미지에서 **3×3 영역**이 하이라이트됨 | 필터가 현재 보고 있는 위치 |
| 필터 값 × 이미지 값 = 곱셈·덧셈 과정 표시 | 합성곱 연산의 실체 |
| 결과 격자가 한 칸씩 채워짐 | 왼쪽 위 → 오른쪽 아래 순서 |

> 📌 "필터가 이미지 위를 돋보기처럼 훑으면서 특징을 찾아내는 것, 이것이 **합성곱(Convolution)**입니다!"

\newpage

### 전통적 방법 vs CNN 비교

| 항목 | 전통적 방법 | CNN (딥러닝) |
|:---:|:---|:---|
| **필터 설계** | 사람이 수작업으로 만듦 | AI가 데이터로부터 **자동 학습** |
| **특징 추출** | 전문가의 경험과 지식 필요 | 데이터만 있으면 가능 |
| **적응성** | 새 문제마다 새 필터 설계 필요 | 데이터만 바꾸면 재학습 가능 |
| **성능** | 제한적 | 대규모 데이터에서 압도적 |

> 💡 **핵심 메시지**: "CNN은 이런 필터를 **스스로** 학습합니다. 어떤 필터가 좋은지, 어떻게 조합하면 좋은지를 AI가 데이터로부터 배워요. 이것이 딥러닝이 강력한 이유입니다!"
