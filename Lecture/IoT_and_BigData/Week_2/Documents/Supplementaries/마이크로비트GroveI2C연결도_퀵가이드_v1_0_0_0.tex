\section*{\emoji{thought-balloon} 생각해보기}

\begin{tcolorbox}[summarybox]
\textbf{"왜 센서마다 별도의 선을 연결하지 않을까?"}

\vspace{0.3em}
센서 6개를 각각 연결하면 선이 12개 이상 필요합니다.

I2C 버스를 사용하면 \textbf{단 2개 선}으로 모든 센서와 통신할 수 있습니다.

\vspace{0.3em}
이것은 마치 \textbf{교실 방송 시스템}과 같습니다:

- 스피커 1개(SDA)와 마이크 1개(SCL)로 교실 전체에 방송

- "3번 학생(0x44번 센서), 온도 알려주세요!" \textrightarrow{} 해당 센서만 응답

\vspace{0.3em}
단, I2C 주소가 겹치면 두 센서가 동시에 대답해서 혼란이 생깁니다!

실제 프로젝트에서 주소 충돌을 확인하는 것이 중요합니다.

\end{tcolorbox}
