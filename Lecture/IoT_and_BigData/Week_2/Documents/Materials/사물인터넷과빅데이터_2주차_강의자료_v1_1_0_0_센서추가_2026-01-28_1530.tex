\subsection{1주차 과제 확인}

지난 주에는 마이크로비트를 처음 접하고 LED에 자기 이름을 표시하는 과제를 수행했습니다.

\begin{keypoint}
\textbf{점검 사항}
\begin{itemize}
  \item MakeCode 에디터 접속 성공 여부
  \item 마이크로비트 연결 및 프로그래밍 성공 여부
  \item LED 문자 표시 원리 이해 여부
\end{itemize}
\end{keypoint}

\subsection{마이크로비트란?}

마이크로비트는 영국 BBC에서 교육용으로 개발한 초소형 컴퓨터입니다. 현재 V2 버전이 출시되어 더 많은 기능을 제공합니다.

\begin{figure}[H]
\centering
\includegraphics[width=0.6\textwidth]{microbitdrawing2bluemicrophoneon.png}
\caption{마이크로비트 V2 앞면 - LED, 버튼, 마이크 등}
\end{figure}

\subsubsection{주요 구성 요소}

\textbf{앞면 (Front)}
\begin{itemize}
  \item \textbf{5×5 LED 디스플레이}: 25개 LED로 문자, 숫자, 그림 표시
  \item \textbf{버튼 A, B}: 입력 장치
  \item \textbf{터치 로고}: V2에서 추가된 터치 센서
  \item \textbf{마이크}: V2에서 추가된 소리 감지 센서
  \item \textbf{엣지 커넥터}: 외부 장치 연결용 핀 (0, 1, 2, 3V, GND 등)
\end{itemize}

\begin{figure}[H]
\centering
\includegraphics[width=0.6\textwidth]{microbitdrawingback13.png}
\caption{마이크로비트 V2 뒷면 - 프로세서, 센서, 연결부}
\end{figure}

\textbf{뒷면 (Back)}
\begin{itemize}
  \item \textbf{프로세서}: 프로그램을 실행하는 CPU
  \item \textbf{가속도 센서}: 움직임, 기울기 감지
  \item \textbf{나침반 (자기 센서)}: 방위 측정
  \item \textbf{USB 포트}: 프로그래밍 및 전원 공급
  \item \textbf{배터리 커넥터}: 휴대용 전원 연결
  \item \textbf{리셋 버튼}: 프로그램 재시작
\end{itemize}

\end{figure}

\subsection{MakeCode 에디터 복습}

MakeCode는 마이크로비트를 프로그래밍할 수 있는 블록 코딩 환경입니다.

\textbf{주요 영역}
\begin{enumerate}
  \item \textbf{시뮬레이터}: 코드 실행 결과를 바로 확인
  \item \textbf{블록 팔레트}: 사용 가능한 블록들 (기본, 입력, LED, 루프 등)
  \item \textbf{코딩 영역}: 블록을 끌어다 놓아 프로그램 작성
\end{enumerate}

\begin{keypoint}
\textbf{기본 블록 구조}

모든 마이크로비트 프로그램은 다음 블록으로 시작합니다:
\begin{itemize}
  \item \textbf{시작하면}: 전원이 켜지거나 리셋할 때 한 번 실행
  \item \textbf{무한반복}: 계속 반복해서 실행
\end{itemize}
\end{keypoint}

\subsection{바이브코딩의 개념}

\begin{tcolorbox}[enhanced, colback=blue!10, colframe=mainblue, 
                  arc=5mm, fontupper=\Large\bfseries, center title]
"코드를 짜지 않고 코드를 만든다"
\end{tcolorbox}

\vspace{1em}

바이브코딩(Vibe Coding)은 \textbf{AI와 대화하며 프로그램을 만드는} 새로운 프로그래밍 방식입니다. 전통적인 코딩에서는 문법을 배우고 직접 코드를 작성해야 했지만, 바이브코딩에서는 \textbf{자연어로 원하는 것을 설명}하면 AI가 코드를 생성해줍니다.

\subsection{전통적 프로그래밍 vs 바이브코딩}

\begin{table}[H]
\centering

\hline
\rowcolor{lightgray}
\textbf{전통적 프로그래밍} & \textbf{바이브코딩} \\
\hline
문법 학습 필수 & 문법 학습 불필요 \\
\hline
직접 코드 작성 & AI와 대화로 작성 \\
\hline
오류 수정에 시간 소요 & AI가 즉시 수정 \\
\hline
시간 소요 +++ & 시간 소요 + \\
\hline
난이도 높음 & 난이도 낮음 \\
\hline
\textbf{코드 작성 능력 중요} & \textbf{문제 정의 능력 중요} \\
\hline
\end{tabularx}
\caption{전통적 프로그래밍과 바이브코딩 비교}

\subsection{왜 바이브코딩인가?}

\subsubsection{초등교사에게 필요한 역량}

초등교사는 학생들에게 \textbf{프로그래밍 문법}을 가르치는 것보다 \textbf{컴퓨팅 사고력}과 \textbf{문제 해결 능력}을 길러주는 것이 더 중요합니다.

\begin{keypoint}
\textbf{바이브코딩으로 기를 수 있는 역량}
\begin{enumerate}
  \item \textbf{문제 정의 능력}: "무엇을 만들고 싶은가?"를 명확히 하기
  \item \textbf{비판적 사고}: "이 결과가 맞나? 확인해보자"
  \item \textbf{AI 협업 능력}: AI에게 효과적으로 요청하는 방법
  \item \textbf{컴퓨팅 사고력}: 문제를 단계별로 나누어 해결하기
\end{enumerate}
\end{keypoint}

\subsubsection{2022 개정 교육과정과의 연계}

2022 개정 교육과정에서는 초등학생의 \textbf{SW/AI 교육}이 강조됩니다. 바이브코딩은 다음과 같이 교육과정과 연계됩니다:

\begin{table}[H]
\centering
\begin{tabularx}{\textwidth}{|l|X|}
\hline
\rowcolor{lightgray}
\textbf{교육과정 요소} & \textbf{바이브코딩 연계} \\
\hline
실과 - 피지컬 컴퓨팅 & 마이크로비트 + 센서 활용 \\
\hline
실과 - 데이터 수집·분석 & 센서 데이터 수집 및 시각화 \\
\hline
SW/AI 교육 & AI 협업, 문제 해결 \\
\hline
컴퓨팅 사고력 & 문제 정의, 추상화, 분해 \\
\hline
\end{tabularx}
\end{table}

\begin{warning}
\textbf{바이브코딩의 오해}

바이브코딩이 "모든 것을 AI가 해준다"는 의미는 \textbf{아닙니다}. 가장 중요한 것은:
\begin{itemize}
  \item 문제를 \textbf{명확히 정의}하는 것
  \item 결과를 \textbf{비판적으로 검증}하는 것
  \item 만족할 때까지 \textbf{반복 수정}하는 것
\end{itemize}
\end{warning}

\subsection{Claude란?}

\textbf{Claude}는 Anthropic에서 개발한 AI 어시스턴트로, 대화를 통해 코드를 생성하고 문제를 해결할 수 있습니다.

\subsubsection{Claude의 특징}

\begin{itemize}
  \item \textbf{긴 대화 컨텍스트 지원}: 이전 대화 내용을 기억
  \item \textbf{코드 생성 및 분석 특화}: HTML, JavaScript, Python 등
  \item \textbf{단계적 수정 가능}: 결과를 보고 수정 요청
  \item \textbf{교육용으로 적합}: 친절한 설명과 예시 제공
\end{itemize}

\textbf{접속 방법}
\begin{itemize}
  \item 웹사이트: \url{https://claude.ai}
  \item 계정 생성 (무료)
  \item 바로 대화 시작
\end{itemize}

\begin{keypoint}
\textbf{다른 AI도 사용 가능합니다}

ChatGPT, Copilot, Gemini 등 다른 AI도 바이브코딩에 사용할 수 있습니다. 이 강의에서는 Claude를 예시로 사용합니다.
\end{keypoint}

\subsection{효과적인 프롬프트 작성법}

\textbf{프롬프트(Prompt)}는 AI에게 보내는 요청이나 질문입니다. 프롬프트를 잘 작성하면 원하는 결과를 더 빠르게 얻을 수 있습니다.

\subsubsection{프롬프트 작성 원칙}

\begin{enumerate}
  \item \textbf{구체적으로 요청하기}
  \begin{itemize}
    \item ❌ 나쁜 예: "웹페이지 만들어줘"
    \item ✅ 좋은 예: "자기소개 웹페이지를 만들어줘. 이름, 전공, 취미를 포함하고 파란색 테마로"
  \end{itemize}
  
  \item \textbf{형식 지정하기}
  \begin{itemize}
    \item "HTML 파일 하나로 만들어줘"
    \item "CSS는 파일 안에 포함해줘"
    \item "설명은 주석으로 넣어줘"
  \end{itemize}
  
  \item \textbf{단계적으로 수정하기}
  \begin{itemize}
    \item 일단 기본 버전을 만들고
    \item 결과를 확인한 후
    \item 수정할 부분을 구체적으로 요청
  \end{itemize}
  
  \item \textbf{예시 제공하기}
  \begin{itemize}
    \item 원하는 스타일의 예시 제시
    \item 데이터 형식 예시 제공
    \item 비슷한 결과물 참고 자료
  \end{itemize}
\end{enumerate}

\subsection{프롬프트 예시}

\begin{promptbox}
\textbf{예시 1: 자기소개 웹페이지}

\begin{verbatim}
자기소개 웹페이지를 만들어줘.

이름: 김철수
전공: 수학교육과
취미: 등산, 독서

\subsection{Grove I2C 연결 방식}

Grove 센서는 \textbf{Grove Shield}를 통해 마이크로비트에 연결됩니다.

\begin{figure}[H]
\centering

\caption{Grove Shield와 센서 연결 예시}

\subsubsection{I2C 통신이란?}

\textbf{I2C}(Inter-Integrated Circuit)는 \textbf{2개의 선(SDA, SCL)}으로 여러 장치를 연결하는 통신 방식입니다.

\begin{keypoint}
\textbf{I2C 통신의 특징}
\begin{itemize}
  \item \textbf{2개 선 공유}: SDA(데이터), SCL(클럭)
  \item \textbf{고유 주소}: 각 센서는 고유 주소로 구분 (예: 0x44, 0x76)
  \item \textbf{다중 연결}: 이론상 127개, 실용적으로 5~8개 장치 연결 가능
  \item \textbf{간단한 배선}: 납땜 없이 Grove 커넥터로 쉽게 연결
\end{itemize}
\end{keypoint}

\subsubsection{연결 구조}

\begin{warning}
\textbf{센서의 한계 이해하기}

센서는 \textbf{현재 상태}만 측정할 뿐입니다:
\begin{itemize}
  \item ❌ 센서는 \textbf{미래를 예측하지 않습니다}
  \item ❌ 센서는 \textbf{원인을 알려주지 않습니다}
  \item ❌ 센서는 \textbf{판단하지 않습니다}
  \item ✅ 센서는 \textbf{지금 이 순간의 값}만 알려줍니다
\end{itemize}

예: 온도 센서가 "28도"를 측정했다면
\begin{itemize}
  \item ✅ "지금 온도는 28도다" (사실)
  \item ❌ "더워서 학생들이 집중을 못한다" (추정)
\end{itemize}
\end{warning}
