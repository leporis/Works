## 3. 학습 내용 개요

### 3.1 바이브코딩이란? (활동 1)

- 바이브코딩의 정의: "코드를 짜지 않고, AI에게 말로 설명하여 코드를 만드는 방법"
- 전통 코딩 vs 바이브코딩 비교
- 바이브코딩의 장점과 한계
- 이 수업에서 바이브코딩을 사용하는 이유: 코드 문법이 아닌 **문제 해결 과정**에 집중

### 3.2 Claude 사용법 (활동 2)

- Claude 접속 및 기본 인터페이스 안내
- 좋은 프롬프트 작성법 (구체적으로, 단계적으로, 예시 포함)
- 프롬프트 → 결과물 → 수정 요청의 반복 과정
- Artifact(코드 결과물) 확인 및 다운로드 방법
- 대화목록 PDF 생성 방법 (과제 제출용)

### 3.3 바이브코딩 체험 (활동 3)

- 실습 과제: Claude에게 자기소개 홈페이지 만들어달라고 요청
- 첫 프롬프트 작성 → 결과 확인 → 수정 요청 → 완성
- HTML 파일 다운로드 및 브라우저에서 확인
- 체험 후 소감: "코딩을 몰라도 웹페이지를 만들 수 있다!"

### 3.4 센서 탐색 (활동 4)

- **마이크로비트 V2 내장 센서** (회로 연결 없이 사용)
  - 온도 센서, 마이크(소음), 조도 센서, 가속도계, 나침반, 터치 로고
  - 각 센서의 측정 범위, 특징, 한계
  - 1주차 복습 연결: LED 이름 표시에서 → 센서 데이터 활용으로 확장

- **Grove 확장 센서** (I2C 플러그앤플레이)
  - Grove Shield 연결 방식 소개 (케이블 꽂기만 하면 됨)
  - I2C 통신 개념 (간단히: "주소로 센서를 구분하는 버스")
  - 주요 Grove I2C 센서: 온습도(SHT40), 조도(TSL2561), 색상(TCS34725), 거리(VL53L0X), 기압(BMP280), 환경(BME680), 가속도(LIS3DHTR)
  - 센서 조합의 다양성: 내장 + 외장 조합으로 848가지 가능

- **센서 시뮬레이터 시연** (참고 도구)
  - 센서 시뮬레이터 v9: 센서 슬롯 배치 및 가상 데이터 확인
  - 센서 검색 도구: "교실 온도와 습도를 측정하고 싶어요" → 센서 추천
  - I2C 연결도: 센서별 주소와 연결 구조 시각화

### 3.5 프로젝트 브레인스토밍 안내 (활동 5)

- 다음 주(3주차) 프로젝트 기획 예고
- "학교 현장에서 무엇을 측정할 수 있을까?" 생각해오기
- 프로젝트 예시 간단 소개: 교실 환경, 복도 소음, 급식실 대기, 운동장 활동량
