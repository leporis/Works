## 1. 지난 주 돌아보기

### 1.1 1주차 과제 점검

지난 시간에 마이크로비트 LED에 자기 이름을 표시하는 과제를 했어요. 간단히 점검해 볼게요.

- MakeCode 에디터에 접속해서 블록 코딩을 해봤나요?
- "시작하면" 블록 안에 "문자열 표시" 블록을 넣어서 이름을 표시했나요?
- 시뮬레이터에서 LED가 스크롤되면서 이름이 나타나는 걸 확인했나요?

잘 됐든, 잘 안 됐든 괜찮아요. 오늘부터는 블록 코딩 대신 **완전히 새로운 방식**으로 프로그래밍을 해볼 거예요.

### 1.2 오늘 배울 내용 미리보기

오늘 수업은 크게 두 가지예요.

첫째, **바이브코딩**이라는 새로운 코딩 방식을 배워요. AI에게 "이런 걸 만들어줘"라고 말하면 코드가 뚝딱 나오는 방법이에요.

둘째, **센서 탐색**을 해요. 마이크로비트에 내장된 센서와 추가로 연결할 수 있는 Grove 센서를 살펴보면서, "우리 프로젝트에서 뭘 측정할 수 있을까?"를 생각해 볼 거예요.

## 2. 바이브코딩이란?

### 2.1 전통 코딩 vs 바이브코딩

여러분이 알고 있는 코딩은 아마 이런 모습일 거예요.

**전통 코딩 방식:**

```
프로그래밍 언어 공부 → 문법 암기 → 코드 작성 → 오류 수정 → 완성
```

변수, 함수, 반복문, 조건문... 이런 문법을 하나하나 배우고, 한 글자라도 틀리면 에러가 나죠. 세미콜론 하나 빠뜨려도 프로그램이 안 돌아가는 경험, 혹시 해보셨나요?

**바이브코딩 방식:**

```
만들고 싶은 것을 말로 설명 → AI가 코드 생성 → 결과 확인 → 수정 요청 → 완성
```

바이브코딩(Vibe Coding)은 **코드를 직접 짜지 않고, AI에게 자연어로 설명하여 코드를 만드는 방법**이에요. "바이브(vibe)"는 분위기, 느낌이라는 뜻인데, 내가 원하는 느낌을 AI에게 전달하면 AI가 코드로 바꿔준다는 거예요.

### 2.2 바이브코딩의 핵심 아이디어

바이브코딩에서 가장 중요한 건 **"무엇을 만들고 싶은지 명확하게 설명하는 능력"**이에요.

기존 코딩에서는 "어떻게(How) 만드는가"가 중요했어요. `for` 반복문을 써야 하는지, `while`을 써야 하는지, 배열 인덱스는 0부터 시작하는지... 이런 기술적 지식이 필요했죠.

바이브코딩에서는 "무엇을(What) 만들고 싶은가"가 중요해요. "교실 온도를 그래프로 보여주는 웹페이지를 만들어줘"처럼, 원하는 결과를 잘 설명하면 돼요.

### 2.3 바이브코딩의 장점과 한계

**장점:**

- 프로그래밍 언어를 몰라도 프로그램을 만들 수 있어요
- 아이디어를 빠르게 시제품(프로토타입)으로 만들 수 있어요
- 코드 문법 대신 문제 해결 과정에 집중할 수 있어요
- 결과물을 보면서 "이건 좀 바꿔줘"라고 점진적으로 개선할 수 있어요

**한계:**

- AI가 만든 코드가 항상 완벽하지는 않아요
- 복잡한 프로그램일수록 원하는 대로 만들기 어려워져요
- AI가 코드를 "이해"하는 게 아니라 패턴을 기반으로 생성하는 거예요
- 결과물을 검토하고 판단하는 건 결국 사람의 몫이에요

> 💡 **핵심 메시지**: "AI는 코드를 이해하지 않아요. 우리가 만든 설명을 바탕으로 코드를 생성할 뿐이에요. 그래서 결과를 꼭 확인해야 해요!"

### 2.4 이 수업에서 바이브코딩을 사용하는 이유

이 수업의 목표는 코딩 실력을 키우는 게 아니에요. **데이터를 수집하고, 분석하고, 해석하는 과정**을 경험하는 거예요.

바이브코딩을 사용하면 코드 문법에 시간을 쏟지 않고, 진짜 중요한 것들에 집중할 수 있어요.

- "어떤 센서로 뭘 측정할까?" (문제 정의)
- "수집한 데이터에서 어떤 패턴이 보일까?" (데이터 분석)
- "이 패턴이 정말 원인-결과 관계일까?" (비판적 사고)

코딩은 AI에게 맡기고, 우리는 **생각하는 일**에 집중하는 거예요.

## 3. Claude 사용법

### 3.1 Claude 접속하기

Claude는 Anthropic이라는 회사에서 만든 AI 도구예요. 웹 브라우저에서 바로 사용할 수 있어요.

**접속 방법:**

1. 웹 브라우저에서 `claude.ai`에 접속해요
2. 학교에서 제공하는 계정으로 로그인해요
3. "새 대화" 버튼을 눌러 대화를 시작해요

### 3.2 Claude 화면 구성

Claude 화면은 크게 세 영역으로 나뉘어요.

- **왼쪽 사이드바**: 이전 대화 목록이 있어요. 나중에 다시 찾아볼 수 있어요.
- **가운데 대화 영역**: 여러분이 입력한 메시지와 Claude의 답변이 표시돼요.
- **입력창**: 화면 하단에 있어요. 여기에 프롬프트를 입력해요.

### 3.3 프롬프트 작성법

프롬프트(Prompt)란 AI에게 보내는 요청 메시지예요. 프롬프트를 잘 쓰면 원하는 결과를 더 정확하게 얻을 수 있어요.

**좋은 프롬프트의 3가지 원칙:**

**원칙 1: 구체적으로 써요**

| ❌ 모호한 프롬프트 | ✅ 구체적인 프롬프트 |
|-------------------|---------------------|
| "웹페이지 만들어줘" | "내 이름, 취미, 좋아하는 과목을 소개하는 웹페이지를 만들어줘" |
| "그래프 그려줘" | "온도 데이터를 시간순으로 꺾은선 그래프로 그려줘" |

**원칙 2: 단계적으로 요청해요**

한 번에 모든 걸 요청하기보다, 단계별로 나눠서 요청하면 더 좋은 결과를 얻을 수 있어요.

```
[1단계] "자기소개 홈페이지를 만들어줘. 이름은 홍길동이야."
[2단계] "배경색을 하늘색으로 바꿔줘."
[3단계] "좋아하는 과목 목록을 추가해줘."
```

**원칙 3: 예시를 포함해요**

원하는 결과의 예시를 보여주면 AI가 더 잘 이해해요.

```
"표를 만들어줘. 예를 들면 이런 식이야:
| 시간 | 온도 | 소음 |
| 9:00 | 24℃ | 45dB |"
```

### 3.4 Artifact 확인과 다운로드

Claude가 코드를 만들면 **Artifact**라는 별도 패널에 결과가 표시돼요.

- **미리보기**: Artifact 패널에서 결과물을 바로 확인할 수 있어요
- **코드 보기**: "Code" 탭을 누르면 실제 코드를 볼 수 있어요
- **다운로드**: 우측 상단 다운로드 버튼으로 HTML 파일을 저장할 수 있어요

### 3.5 대화목록 PDF 생성 방법

과제 제출 시 대화 내역을 PDF로 제출해야 해요. Claude에게 다음 프롬프트를 입력하면 자동으로 생성돼요.

```
이 대화에서 사용한 프롬프트와 답변을 포함한 대화 전문과 
프롬프트와 답변을 정리한 요약본을 PDF 문서로 만들어줘. 
오늘 날짜, 수업주차 2주차, 작성자 이름 OOO, 학번 XXXXXX을 추가해줘.
```

이렇게 하면 대화 전문 PDF와 요약본 PDF가 자동으로 만들어져요.

## 4. 바이브코딩 체험 실습

### 4.1 실습 과제: 자기소개 홈페이지 만들기

이제 직접 바이브코딩을 해볼 거예요. Claude에게 자기소개 홈페이지를 만들어달라고 요청해 보세요.

**첫 번째 프롬프트 예시:**

```
나를 소개하는 웹페이지를 만들어줘.
- 이름: 홍길동
- 학교: 청주교육대학교 4학년
- 좋아하는 과목: 체육, 미술
- 취미: 독서, 영화 감상
- 배경색은 밝고 예쁘게 해줘
```

### 4.2 결과 확인 → 수정 요청

Claude가 만든 결과물을 보고, 마음에 안 드는 부분이 있으면 수정을 요청해요.

**수정 요청 예시:**

```
"글씨가 좀 작아. 제목을 더 크게 해줘."
"사진 넣을 자리를 추가해줘."
"색깔을 좀 더 파스텔 톤으로 바꿔줘."
"이모지를 추가해서 더 귀엽게 만들어줘."
```

이 과정이 바로 바이브코딩의 핵심이에요. **프롬프트 → 결과 확인 → 수정 요청**을 반복하면서 원하는 결과물에 점점 가까워지는 거예요.

### 4.3 실습 후 생각해 볼 점

바이브코딩 체험을 하면서 다음을 생각해 보세요.

- Claude가 내 의도를 잘 이해했나요?
- 프롬프트를 어떻게 바꾸니까 결과가 달라졌나요?
- Claude가 만든 코드에 혹시 이상한 부분은 없었나요?
- 이 방법으로 더 복잡한 프로그램도 만들 수 있을까요?

> 💡 **기억하세요**: "Claude가 만든 결과를 그대로 믿지 말고, 꼭 확인하는 습관을 들이세요!"

## 5. 센서 탐색: 마이크로비트 내장 센서

### 5.1 마이크로비트 V2에 들어있는 센서들

지난 시간에 마이크로비트로 LED에 이름을 표시했죠? 사실 마이크로비트 안에는 **여러 종류의 센서**가 들어있어요. 추가 장치를 연결하지 않아도 바로 사용할 수 있는 센서들이에요.

**마이크로비트 V2 내장 센서 목록:**

| 센서 | 측정 대상 | 값 범위 | 특징 |
|------|----------|---------|------|
| 온도 센서 | 온도 (℃) | −5 ~ 50℃ | CPU 근처에 있어서 실제보다 2~5℃ 높을 수 있어요 |
| 마이크 🎤 | 소리 크기 | 0 ~ 255 | V2에서 새로 추가! 상대적 크기예요 |
| 조도 센서 | 밝기 | 0 ~ 255 | LED를 거꾸로 이용해서 빛을 감지해요 |
| 가속도계 | 움직임, 기울기 | −2g ~ +2g (3축) | 흔들림, 떨어뜨림도 감지해요 |
| 나침반 | 방향 | 0° ~ 360° | 지구 자기장을 감지해요 |
| 터치 로고 | 터치 여부 | 터치/비터치 | V2에서 새로 추가! 금색 로고를 터치하면 감지 |

### 5.2 내장 센서의 특징과 한계

내장 센서는 편리하지만, 몇 가지 알아둘 점이 있어요.

**온도 센서의 특이한 점:**

마이크로비트의 온도 센서는 별도 센서가 아니라 **프로세서(CPU) 안에 들어있는 온도 센서**예요. 그래서 CPU가 작동하면서 발생하는 열 때문에 실제 공기 온도보다 2~5℃ 높게 측정될 수 있어요. 정확한 온도를 알고 싶으면 나중에 배울 Grove 외부 온도 센서를 사용하는 게 좋아요.

**마이크의 특이한 점:**

마이크가 측정하는 값은 0~255 사이의 **상대적인 숫자**예요. "소리 크기 = 150"이라고 해서 150데시벨이라는 뜻이 아니에요. 같은 교실에서 여러 마이크로비트로 측정하면 값이 조금씩 다를 수도 있어요.

**조도 센서의 특이한 점:**

마이크로비트에는 별도 조도 센서가 없어요. 대신 **5×5 LED를 거꾸로 사용**해서 빛을 감지해요. 꽤 기발하지만, 정확한 조도(lux)를 알기는 어려워요.

### 5.3 내장 센서로 할 수 있는 것 vs 없는 것

| 할 수 있는 것 ✅ | 할 수 없는 것 ❌ |
|-----------------|-----------------|
| 실내 온도 변화 추적 | 정확한 기온 측정 |
| 소음 수준 비교 (시끄러움/조용함) | 정확한 데시벨(dB) 측정 |
| 밝음/어두움 구분 | 정확한 조도(lux) 측정 |
| 움직임/기울기 감지 | 습도 측정 |
| 방향 감지 | 거리 측정 |
| | 공기질/CO₂ 측정 |

내장 센서만으로도 충분히 재미있는 프로젝트를 할 수 있지만, 더 다양하고 정밀한 측정을 원하면 **외부 센서**를 추가해야 해요. 그게 바로 다음에 소개할 Grove 확장 센서예요!

## 6. 센서 탐색: Grove 확장 센서

### 6.1 Grove 시스템이란?

Grove는 Seeed Studio라는 회사에서 만든 **플러그앤플레이 센서 시스템**이에요.

전통적인 방법으로 센서를 연결하려면 브레드보드에 점퍼선을 꽂고, 저항을 달고, 회로를 구성해야 해요. 잘못 연결하면 센서가 고장날 수도 있고요.

Grove 시스템에서는 **전용 케이블을 꽂기만 하면 끝**이에요. 마치 USB를 꽂는 것처럼 간단해요.

**연결 방식 비교:**

| 전통 방식 | Grove 방식 |
|----------|-----------|
| 브레드보드 + 점퍼선 + 저항 | 전용 케이블 한 개 |
| 회로 지식 필요 | 회로 지식 불필요 |
| 잘못 연결 시 고장 위험 | 잘못 끼울 수가 없는 구조 |
| 연결에 10~30분 | 연결에 10초 |

### 6.2 Grove Shield와 마이크로비트 연결

마이크로비트에 Grove 센서를 연결하려면 **Grove Shield**라는 확장 보드가 필요해요.

**연결 방법:**

1. 마이크로비트를 Grove Shield의 슬롯에 끼워요
2. Grove 센서를 Shield의 I2C 포트에 케이블로 연결해요
3. USB로 컴퓨터에 연결하면 끝!

### 6.3 I2C 통신이란?

Grove 센서 대부분은 **I2C**(아이-투-씨)라는 통신 방식을 사용해요. 어려운 개념 같지만, 비유로 쉽게 이해할 수 있어요.

**I2C를 교실에 비유하면:**

하나의 I2C 버스는 하나의 **교실 인터콤 시스템**과 같아요.

- **마이크로비트** = 선생님 (질문을 하는 사람)
- **센서들** = 학생들 (대답하는 사람들)
- **I2C 버스** = 인터콤 선 (모두 같은 선에 연결)
- **I2C 주소** = 학생 번호 (각 센서의 고유 번호)

선생님이 "3번 학생, 지금 온도가 몇 도야?"라고 물으면, 3번 학생만 대답해요. "7번 학생, 습도는?"이라고 물으면 7번 학생이 대답하고요. 이렇게 **하나의 선에 여러 센서를 연결**할 수 있어요.

**센서별 I2C 주소 예시:**

| 센서 | I2C 주소 | 역할 |
|------|:--------:|------|
| SHT40 | 0x44 | 온도·습도 |
| TSL2561 | 0x29 | 조도 (정밀) |
| TCS34725 | 0x29 | 색상 |
| VL53L0X | 0x29 | 거리 (레이저) |
| BMP280 | 0x76 | 기압·고도 |
| BME680 | 0x76 | 환경 (온도·습도·기압·공기질) |
| LIS3DHTR | 0x18 | 가속도 (정밀) |

> 💡 **참고**: 같은 주소를 가진 센서들은 동시에 연결할 수 없어요. 예를 들어 TSL2561과 TCS34725는 둘 다 0x29이라 하나만 사용할 수 있어요.

### 6.4 주요 Grove I2C 센서 소개

**🌡️ SHT40 — 온도·습도 센서**

| 항목 | 내용 |
|------|------|
| 측정 | 온도 (−40~125℃), 습도 (0~100%) |
| 정확도 | 온도 ±0.2℃, 습도 ±1.8% |
| 특징 | 마이크로비트 내장 온도보다 훨씬 정확해요 |
| 프로젝트 활용 | 교실 환경 모니터링, 습도와 집중도 관계 |

**💡 TSL2561 — 조도 센서**

| 항목 | 내용 |
|------|------|
| 측정 | 조도 (0.1~40,000 lux) |
| 특징 | 사람 눈과 비슷한 감도, 정확한 lux 값 제공 |
| 프로젝트 활용 | 교실 밝기 측정, 조명과 집중도 관계 |

**🎨 TCS34725 — 색상 센서**

| 항목 | 내용 |
|------|------|
| 측정 | RGB 색상값 + 밝기 |
| 특징 | 물체의 색을 감지할 수 있어요 |
| 프로젝트 활용 | 과학 실험(색 변화), 미술 활동 |

**📏 VL53L0X — 거리 센서 (ToF)**

| 항목 | 내용 |
|------|------|
| 측정 | 거리 (30~2000mm) |
| 원리 | 레이저를 쏘고 돌아오는 시간을 측정 (Time of Flight) |
| 특징 | 초음파보다 정확하고 빠름 |
| 프로젝트 활용 | 사람 감지, 줄 길이 측정, 자세 변화 |

**🌤️ BMP280 — 기압 센서**

| 항목 | 내용 |
|------|------|
| 측정 | 기압 (300~1100 hPa), 온도 |
| 특징 | 고도 추정 가능 (기압으로 높이 계산) |
| 프로젝트 활용 | 날씨 변화 추적, 층수 감지 |

**🌍 BME680 — 환경 종합 센서**

| 항목 | 내용 |
|------|------|
| 측정 | 온도, 습도, 기압, 공기질(VOC 가스) |
| 특징 | 4가지를 한 번에 측정하는 만능 센서 |
| 프로젝트 활용 | 교실 환경 종합 분석, 환기 상태 |

**🏃 LIS3DHTR — 가속도 센서**

| 항목 | 내용 |
|------|------|
| 측정 | 3축 가속도 (±2g/4g/8g/16g) |
| 특징 | 마이크로비트 내장 가속도계보다 정밀하고 설정 범위가 넓어요 |
| 프로젝트 활용 | 운동량 측정, 진동 분석 |

### 6.5 센서 조합의 다양성

마이크로비트 내장 센서 4종(온도, 소음, 빛, 가속도)과 Grove 외부 센서를 조합하면, **848가지 이상의 센서 조합**이 가능해요.

예를 들어 "교실 환경과 집중도" 프로젝트를 한다면 다음과 같은 조합을 사용할 수 있어요.

| 내장 센서 | Grove 센서 | 측정 항목 |
|----------|-----------|----------|
| 온도, 소음, 빛 | SHT40 (습도) | 온도·습도·소음·밝기 4가지 |
| 소음 | BME680 (환경 종합) | 소음·온도·습도·기압·공기질 5가지 |
| 가속도 | VL53L0X (거리) | 움직임·거리(사람 감지) 2가지 |

어떤 센서 조합이 내 프로젝트에 적합한지는 다음에 소개할 **센서 검색 도구**로 쉽게 찾아볼 수 있어요.

## 7. 센서 시뮬레이터 소개

### 7.1 센서 시뮬레이터 v9

센서 시뮬레이터는 **실제 하드웨어 없이 센서 데이터를 체험**할 수 있는 웹 도구예요.

**주요 기능:**

- 마이크로비트와 Grove Shield를 화면에서 보여줘요
- 최대 6개의 Grove 센서 슬롯에 센서를 배치할 수 있어요
- 각 센서의 가상 데이터를 실시간으로 확인할 수 있어요
- I2C 주소 충돌을 자동으로 감지해 줘요

이 시뮬레이터는 수업 중 참고 도구로 활용하고, 나중에 실제 하드웨어를 받기 전에 프로젝트 센서 구성을 미리 계획할 때 유용해요.

### 7.2 센서 검색 도구

센서 검색 도구는 **자연어로 센서를 추천**받을 수 있는 도구예요.

예를 들어 "교실 온도와 습도를 측정하고 싶어요"라고 입력하면, 사용 가능한 내장 센서와 외부 센서를 추천해 줘요. 센서 수, 데이터 유형 수, 내장/외장 비율까지 알려주니까, 프로젝트 계획을 세울 때 참고하면 좋아요.

### 7.3 I2C 연결도

I2C 연결도는 **센서별 I2C 주소와 연결 구조**를 시각적으로 보여주는 도구예요.

마이크로비트에서 I2C 버스(SDA, SCL 두 선)가 나오고, 여기에 여러 센서가 각자의 주소로 연결되는 구조를 한눈에 확인할 수 있어요. 같은 주소를 가진 센서끼리는 연결할 수 없다는 것도 이 도구로 확인할 수 있어요.

## 8. 프로젝트 브레인스토밍 안내

### 8.1 다음 주 예고: 프로젝트 기획

다음 주(3주차)에는 여러분만의 **프로젝트 주제**를 정할 거예요. 실습학교에서 실제로 데이터를 수집할 프로젝트를 기획하는 시간이에요.

### 8.2 "학교 현장에서 뭘 측정할 수 있을까?"

프로젝트 주제를 정하려면 먼저 **"학교에서 센서로 무엇을 측정할 수 있을까?"**를 생각해 봐야 해요.

**프로젝트 예시:**

| 주제 | 센서 | 핵심 질문 |
|------|------|----------|
| 교실 환경과 집중도 | 온도, 소음, 빛, 습도 | 교실 환경이 집중도에 영향을 줄까? |
| 복도 소음 분석 | 소음 | 쉬는 시간 vs 수업 시간, 소음 차이는? |
| 급식실 대기 시간 | 거리 (사람 감지) | 줄이 가장 긴 시간대는? |
| 운동장 활동량 | 가속도 | 체육 시간에 아이들이 얼마나 움직일까? |
| 도서관 이용 패턴 | 빛, 소음 | 가장 조용한 시간대는? |

### 8.3 다음 주까지 생각해 올 것

- 내가 실습학교에서 측정하고 싶은 것은?
- 어떤 센서가 필요할까?
- 데이터를 모으면 어떤 질문에 답할 수 있을까?

아이디어가 떠오르면 메모해 두세요. 다음 시간에 함께 나눠봐요!

## 9. 오늘 수업 정리

### 9.1 핵심 정리

오늘 배운 내용을 정리하면 다음과 같아요.

**바이브코딩**: 코드를 직접 짜지 않고, AI에게 자연어로 설명하여 코드를 만드는 방법이에요. 프롬프트를 잘 쓰는 게 핵심이에요.

**Grove 확장 센서**: I2C 방식으로 플러그앤플레이 연결이 가능한 외부 센서예요. 온습도(SHT40), 조도(TSL2561), 색상(TCS34725), 거리(VL53L0X), 기압(BMP280), 환경(BME680), 가속도(LIS3DHTR) 등 다양한 센서를 추가할 수 있어요.
