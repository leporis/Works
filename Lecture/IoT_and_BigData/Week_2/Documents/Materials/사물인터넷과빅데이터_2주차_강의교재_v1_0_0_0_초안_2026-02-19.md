## 2. 바이브코딩이란?

### 2.1 코딩의 두 가지 방식

여러분이 알고 있는 코딩은 아마 이런 모습일 거예요.

**전통 코딩 방식:**

프로그래밍 언어를 공부하고, 문법을 암기하고, 코드를 한 줄씩 직접 작성하는 거예요. 변수, 함수, 반복문, 조건문... 이런 개념을 하나하나 배우고, 한 글자라도 틀리면 에러가 나죠.

```
전통 코딩: 프로그래밍 언어 학습 → 문법 암기 → 코드 직접 작성 → 오류 수정 → 완성
```

세미콜론 하나 빠뜨려도 프로그램이 안 돌아가는 경험, 혹시 해보셨나요? 전통 코딩에서는 **"어떻게(How) 만드는가"**에 많은 시간을 쏟게 돼요.

그런데 이제 완전히 다른 방식이 가능해졌어요.

**바이브코딩 방식:**

AI에게 내가 원하는 것을 **말로 설명**하면, AI가 코드를 생성해 줘요. 결과를 보고 마음에 안 들면 "이 부분 좀 바꿔줘"라고 요청하면 돼요.

```
바이브코딩: 원하는 것을 말로 설명 → AI가 코드 생성 → 결과 확인 → 수정 요청 → 완성
```

바이브코딩에서는 **"무엇을(What) 만들고 싶은가"**가 핵심이에요.

> **[그림 1] 전통 코딩 vs 바이브코딩 비교 다이어그램**

### 2.2 바이브코딩의 정의

바이브코딩(Vibe Coding)은 **코드를 직접 짜지 않고, AI에게 자연어로 설명하여 코드를 만드는 방법**이에요.

"바이브(vibe)"는 영어로 분위기, 느낌이라는 뜻이에요. 내가 원하는 **느낌(vibe)**을 AI에게 전달하면, AI가 그 느낌에 맞는 코드를 만들어준다는 거예요.

| 항목 | 전통 코딩 | 바이브코딩 |
|------|----------|-----------|
| 핵심 역량 | 프로그래밍 문법 지식 | 원하는 것을 명확히 설명하는 능력 |
| 소통 대상 | 컴퓨터 (코드로) | AI (자연어로) |
| 오류 수정 | 코드를 직접 디버깅 | AI에게 수정 요청 |
| 결과물 | 코드 파일 | 코드 파일 (AI가 생성) |
| 필요 지식 | 프로그래밍 언어, 알고리즘 | 문제 정의, 프롬프트 작성 |

### 2.3 바이브코딩의 장점

바이브코딩은 특히 프로그래밍을 처음 접하는 분들에게 큰 장점이 있어요.

**🚀 진입 장벽이 낮아요**

프로그래밍 언어를 몰라도 프로그램을 만들 수 있어요. 한국어로 "이런 걸 만들어줘"라고 하면 되니까요.

**⚡ 아이디어를 빠르게 구현해요**

전통 코딩으로 웹페이지 하나 만들려면 HTML, CSS, JavaScript를 배워야 하는데, 바이브코딩으로는 5분이면 기본 웹페이지가 완성돼요.

**🎯 진짜 중요한 것에 집중해요**

코드 문법에 시간을 쏟는 대신, "무엇을 만들지", "데이터를 어떻게 해석할지" 같은 **문제 해결 과정**에 집중할 수 있어요.

**🔄 점진적 개선이 쉬워요**

결과물을 보면서 "이건 좀 바꿔줘", "여기에 이것도 추가해줘"라고 계속 수정할 수 있어요.

### 2.4 바이브코딩의 한계

장점이 많지만, 반드시 알아둬야 할 한계도 있어요.

**⚠️ AI가 만든 코드가 항상 완벽하지는 않아요**

AI가 가끔 잘못된 코드를 만들거나, 의도와 다른 결과를 내놓을 수 있어요. 그래서 결과를 **반드시 확인**해야 해요.

**⚠️ AI는 코드를 '이해'하는 게 아니에요**

AI가 코드를 쓸 줄 아는 것처럼 보이지만, 실제로는 대량의 데이터에서 학습한 **패턴을 기반으로 생성**하는 거예요. "이해"와 "생성"은 다른 거예요.

**⚠️ 복잡한 프로그램일수록 어려워져요**

간단한 웹페이지는 쉽지만, 복잡한 시스템을 만들 때는 프롬프트를 매우 정교하게 써야 해요.

**⚠️ 결과를 판단하는 건 사람의 몫이에요**

AI가 "이 코드가 맞아요"라고 해도, 정말 맞는지 확인하는 건 결국 사람이 해야 해요.

> 💡 **핵심 메시지**: "AI는 코드를 이해하지 않아요. 우리가 만든 설명을 바탕으로 코드를 생성할 뿐이에요. 그래서 결과를 꼭 확인해야 해요!"

### 2.5 이 수업에서 바이브코딩을 사용하는 이유

이 수업의 목표는 **코딩 실력을 키우는 게 아니에요**. 데이터를 수집하고, 분석하고, 해석하는 과정을 경험하는 거예요.

바이브코딩을 사용하면 코드 문법에 시간을 쏟지 않고, 진짜 중요한 것들에 집중할 수 있어요.

| 바이브코딩으로 AI가 하는 일 | 여러분이 집중하는 일 |
|---------------------------|---------------------|
| HTML/CSS/JavaScript 코드 작성 | 어떤 센서로 뭘 측정할까? (문제 정의) |
| 그래프 그리는 코드 작성 | 수집한 데이터에서 어떤 패턴이 보일까? (데이터 분석) |
| 데이터 처리 코드 작성 | 이 패턴이 정말 원인-결과 관계일까? (비판적 사고) |

코딩은 AI에게 맡기고, 우리는 **생각하는 일**에 집중해요!

## 3. Claude 사용법

### 3.1 Claude란?

Claude는 Anthropic이라는 회사에서 만든 **대화형 AI**예요. 텍스트로 대화하면서 질문에 답하거나, 글을 쓰거나, 코드를 생성할 수 있어요.

이 수업에서는 Claude를 **바이브코딩 도구**로 사용해요. Claude에게 "이런 프로그램을 만들어줘"라고 요청하면 코드를 생성해 주고, "이 부분을 수정해줘"라고 하면 코드를 고쳐줘요.

### 3.2 접속 방법

1. 웹 브라우저에서 **claude.ai**에 접속해요
2. 학교에서 제공하는 계정으로 로그인해요
3. 왼쪽 상단의 **"새 대화"** 버튼을 눌러 대화를 시작해요

> **[그림 2] Claude 인터페이스 화면 구성**

### 3.3 화면 구성

Claude 화면은 크게 세 영역으로 나뉘어요.

| 영역 | 위치 | 기능 |
|------|------|------|
| 사이드바 | 왼쪽 | 이전 대화 목록, 새 대화 시작 |
| 대화 영역 | 가운데 | 입력한 메시지와 Claude 답변 표시 |
| 입력창 | 하단 | 프롬프트 입력, 파일 첨부 |

대화 영역에서 Claude가 코드를 생성하면, 별도의 **Artifact 패널**이 오른쪽에 나타나요. 여기서 결과물을 미리보기하거나 다운로드할 수 있어요.

### 3.4 좋은 프롬프트 작성법

프롬프트(Prompt)란 **AI에게 보내는 요청 메시지**예요. 프롬프트를 어떻게 쓰느냐에 따라 결과 품질이 크게 달라져요.

**좋은 프롬프트의 3가지 원칙:**

#### 원칙 1: 구체적으로 써요

모호하게 쓰면 AI도 모호한 결과를 내놓아요. 원하는 것을 최대한 자세히 써주세요.

| ❌ 모호한 프롬프트 | ✅ 구체적인 프롬프트 |
|-------------------|---------------------|
| "웹페이지 만들어줘" | "내 이름, 취미, 좋아하는 과목을 소개하는 웹페이지를 만들어줘. 배경은 밝은 파란색으로 해줘." |
| "그래프 그려줘" | "온도 데이터를 시간순으로 꺾은선 그래프로 그려줘. X축은 시간, Y축은 온도(℃)야." |
| "데이터 분석해줘" | "CSV 파일의 온도와 소음 데이터에서 상관관계를 계산하고 산점도를 그려줘." |

#### 원칙 2: 단계적으로 요청해요

한 번에 모든 걸 요청하기보다, 단계별로 나눠서 요청하면 더 좋은 결과를 얻을 수 있어요.

```
[1단계] "자기소개 홈페이지를 만들어줘. 이름은 홍길동이야."
   → 기본 페이지 완성

[2단계] "배경색을 하늘색으로 바꿔줘."
   → 디자인 개선

[3단계] "좋아하는 과목 목록을 표로 추가해줘."
   → 내용 추가

[4단계] "이모지를 넣어서 더 귀엽게 만들어줘."
   → 마무리
```

#### 원칙 3: 예시를 포함해요

원하는 결과의 예시를 보여주면 AI가 더 정확하게 이해해요.

```
"표를 만들어줘. 예를 들면 이런 형식이야:

| 시간 | 온도(℃) | 소음(dB) |
| 9:00 | 24 | 45 |
| 10:00 | 26 | 52 |"
```

> **[그림 3] 프롬프트 → 결과 → 수정의 반복 흐름도**

### 3.5 Artifact: 결과물 확인과 다운로드

Claude가 코드를 생성하면 **Artifact**라는 별도 패널에 결과가 표시돼요.

**Artifact 패널의 기능:**

| 기능 | 설명 |
|------|------|
| 미리보기(Preview) | 웹페이지 결과를 바로 확인할 수 있어요 |
| 코드 보기(Code) | 실제 생성된 HTML/CSS/JS 코드를 볼 수 있어요 |
| 다운로드 | 오른쪽 상단 버튼으로 HTML 파일을 저장할 수 있어요 |

다운로드한 HTML 파일은 웹 브라우저에서 열면 바로 확인할 수 있어요. 이 파일이 바로 과제 제출물이 돼요.

### 3.6 대화목록 PDF 생성 방법

이 수업에서는 바이브코딩 활용 주차(2주차, 9~13주차)에 **대화목록 전문**과 **대화목록 요약본**을 PDF로 제출해야 해요.

Claude에게 다음 프롬프트를 입력하면 자동으로 생성돼요.

**대화목록 PDF 생성 프롬프트:**

```
이 대화에서 사용한 프롬프트와 답변을 포함한 대화 전문과 
프롬프트와 답변을 정리한 요약본을 PDF 문서로 만들어줘. 
오늘 날짜, 수업주차 2주차, 작성자 이름 OOO, 학번 XXXXXX을 추가해줘.
```

**생성되는 파일:**

| 파일 | 내용 |
|------|------|
| 대화 전문 PDF | 프롬프트 원문 + Claude 응답 전체 |
| 대화 요약본 PDF | 프롬프트 원문 + Claude 응답 3줄 요약 |

## 4. 바이브코딩 체험 실습

### 4.1 실습 과제: 자기소개 홈페이지 만들기

이제 직접 바이브코딩을 해볼 거예요! Claude에게 자기소개 홈페이지를 만들어달라고 요청해 보세요.

**첫 번째 프롬프트 예시:**

```
나를 소개하는 웹페이지를 만들어줘.
- 이름: 홍길동
- 학교: 청주교육대학교 컴퓨터교육과 4학년
- 좋아하는 과목: 체육, 미술, 음악
- 취미: 독서, 영화 감상, 요리
- 장래 희망: 초등학교 선생님
- 배경은 밝고 예쁘게, 이모지도 넣어줘
```

이 프롬프트를 Claude에게 입력하면, 바로 HTML 코드가 생성되고 Artifact 패널에서 미리보기를 할 수 있어요.

### 4.2 결과 확인하기

Claude가 만든 결과물을 Artifact 패널에서 확인해 보세요.

**확인 포인트:**

| 항목 | 확인 질문 |
|------|----------|
| 내용 | 내가 요청한 정보가 다 들어갔나요? |
| 디자인 | 배경색, 글씨 크기, 레이아웃이 마음에 드나요? |
| 이모지 | 이모지가 적절한 위치에 있나요? |
| 오류 | 이상한 내용이나 깨진 부분은 없나요? |

### 4.3 수정 요청하기

마음에 안 드는 부분이 있으면 Claude에게 수정을 요청해요. 이게 바이브코딩의 핵심 과정이에요!

**수정 요청 예시:**

| 수정 요청 | 기대 효과 |
|----------|----------|
| "글씨가 좀 작아. 제목을 더 크게 해줘." | 타이포그래피 개선 |
| "사진 넣을 자리를 추가해줘." | 레이아웃 변경 |

## 5. 마이크로비트 V2 내장 센서

### 5.1 마이크로비트 복습

지난 시간에 마이크로비트로 LED에 이름을 표시했죠? 사실 마이크로비트 안에는 LED 말고도 **여러 종류의 센서**가 들어있어요. 추가 장치를 연결하지 않아도 바로 사용할 수 있는 센서들이에요.

> **[그림 4] 마이크로비트 V2 내장 센서 배치도**

마이크로비트 V2의 주요 구성을 다시 확인해 볼게요.

**앞면:**
- 5×5 LED 디스플레이 (문자, 숫자, 그림 표시 + 조도 센서 역할)
- 버튼 A, B (입력 장치)
- 터치 로고 (V2에서 추가)
- 마이크 (V2에서 추가)

**뒷면:**
- 프로세서 (프로그램 실행 + 온도 센서 내장)
- 가속도 센서 (움직임, 기울기 감지)
- 나침반 (방위 측정)
- USB 포트, 배터리 커넥터, 리셋 버튼

### 5.2 내장 센서 상세

마이크로비트 V2에는 6가지 센서가 내장되어 있어요.

#### 🌡️ 온도 센서

| 항목 | 내용 |
|------|------|
| 측정 대상 | 온도 (℃) |
| 값 범위 | −5℃ ~ 50℃ |
| 위치 | 프로세서(CPU) 내부 |

**특징과 한계:**

마이크로비트의 온도 센서는 별도 센서가 아니라 **프로세서(CPU) 안에 있는 온도 센서**예요. CPU가 작동하면서 발생하는 열 때문에 실제 공기 온도보다 **2~5℃ 높게** 측정될 수 있어요.

교실 온도가 24℃인데 마이크로비트가 27℃를 표시한다면, 마이크로비트가 고장난 게 아니라 CPU 열 때문이에요. 정밀한 온도 측정이 필요하면 나중에 배울 Grove 외부 온도 센서(SHT40)를 사용하면 돼요.

> 💡 **토론 질문**: "마이크로비트가 표시하는 온도를 그대로 '교실 온도'라고 써도 될까요? 왜 그렇게 생각하나요?"

#### 🎤 마이크 (소음 센서)

| 항목 | 내용 |
|------|------|
| 측정 대상 | 소리 크기 |
| 값 범위 | 0 ~ 255 |
| 위치 | 앞면 하단 |
| 특이사항 | V2에서 새로 추가된 센서 |

**특징과 한계:**

마이크가 측정하는 값은 **상대적인 숫자(0~255)**예요. "소리 크기 = 150"이라고 해서 150데시벨이라는 뜻이 아니에요!

이 값은 "조용한 상태(0에 가까움) vs 시끄러운 상태(255에 가까움)"를 비교하는 데 유용해요. 하지만 정확한 데시벨(dB) 값을 알기는 어려워요.

또한, 같은 교실에서 여러 마이크로비트로 동시에 측정해도 값이 조금씩 다를 수 있어요. 이건 개별 하드웨어의 차이 때문이에요.

#### 💡 조도 센서 (빛 센서)

| 항목 | 내용 |
|------|------|
| 측정 대상 | 밝기 |
| 값 범위 | 0 ~ 255 |
| 위치 | 앞면 (5×5 LED를 활용) |

**특징과 한계:**

마이크로비트에는 별도 조도 센서가 없어요. 대신 **5×5 LED를 거꾸로 사용**해서 빛을 감지해요. LED는 원래 빛을 내보내는 장치인데, 빛을 받으면 약간의 전류가 흐르는 성질을 이용한 거예요. 꽤 기발하지만, 정확한 조도(lux)를 알기는 어려워요.

"밝음(교실 조명 켜짐)" vs "어두움(조명 꺼짐)" 정도의 구분은 잘 해요.

#### 🏃 가속도계

| 항목 | 내용 |
|------|------|
| 측정 대상 | 움직임, 기울기, 흔들림 |
| 값 범위 | −2g ~ +2g (3축: X, Y, Z) |
| 위치 | 뒷면 |

**특징과 한계:**

3축(X, Y, Z) 방향으로 움직임을 감지해요. 기울기, 흔들림, 떨어뜨림, 자유낙하까지 감지할 수 있어요.

MakeCode에서는 "흔들림 감지", "로고 위로", "로고 아래로", "화면 위로", "화면 아래로", "왼쪽으로 기울이기", "오른쪽으로 기울이기", "자유낙하" 등의 이벤트로 활용할 수 있어요.

#### 🧭 나침반

| 항목 | 내용 |
|------|------|
| 측정 대상 | 방향 (자기장) |
| 값 범위 | 0° ~ 360° |
| 위치 | 뒷면 |

지구 자기장을 감지해서 북쪽 방향을 알려줘요. 처음 사용 시 보정(캘리브레이션)이 필요해요. 일반적인 IoT 프로젝트에서는 잘 사용하지 않지만, 방향 관련 프로젝트에서 활용할 수 있어요.

#### ✋ 터치 로고

| 항목 | 내용 |
|------|------|
| 측정 대상 | 터치 여부 |
| 값 | 터치됨 / 터치 안 됨 |
| 위치 | 앞면 상단 (금색 로고) |
| 특이사항 | V2에서 새로 추가 |

금색 로고 부분을 손가락으로 터치하면 감지돼요. 정전식 방식이라 맨손으로 터치해야 작동해요. 추가 입력 버튼처럼 활용할 수 있어요.

### 5.3 내장 센서로 할 수 있는 것 vs 없는 것

| 할 수 있는 것 ✅ | 할 수 없는 것 ❌ |
|-----------------|-----------------|
| 실내 온도 변화 추적 (대략적) | 정확한 기온 측정 (외부 센서 필요) |
| 소음 수준 비교 (상대값) | 정확한 데시벨(dB) 측정 |
| 밝음/어두움 구분 | 정확한 조도(lux) 측정 |
| 움직임, 기울기 감지 | 습도 측정 |
| 방향 감지 | 거리 측정 |
| 터치 입력 | 공기질/CO₂ 측정 |
| | 색상 인식 |

내장 센서만으로도 재미있는 프로젝트를 할 수 있어요. 하지만 더 다양하고 정밀한 측정을 원한다면? 바로 다음에 소개할 **Grove 확장 센서**가 필요해요!

## 6. Grove 확장 센서

### 6.1 Grove 시스템이란?

Grove는 Seeed Studio라는 회사에서 만든 **플러그앤플레이 센서 시스템**이에요.

전통적인 방법으로 센서를 연결하려면 브레드보드에 점퍼선을 꽂고, 저항을 달고, 납땜을 해야 할 수도 있어요. 잘못 연결하면 센서가 고장나기도 하고요.

Grove 시스템에서는 **전용 케이블을 꽂기만 하면 끝**이에요. USB 케이블을 꽂는 것처럼 간단해요. 커넥터 모양이 정해져 있어서 잘못 끼울 수가 없는 구조예요.

> **[그림 5] Grove Shield 연결 다이어그램**

**연결 방식 비교:**

| 항목 | 전통 방식 | Grove 방식 |
|------|----------|-----------|
| 필요 부품 | 브레드보드 + 점퍼선 + 저항 | 전용 케이블 1개 |
| 회로 지식 | 필요 | 불필요 |
| 연결 시간 | 10~30분 | 10초 |
| 잘못 연결 위험 | 있음 (고장 가능) | 없음 (방향 고정) |
| 견고함 | 점퍼선 빠질 수 있음 | 케이블이 단단히 고정 |

### 6.2 Grove Shield와 마이크로비트 연결

마이크로비트에 Grove 센서를 연결하려면 **Grove Shield for micro:bit**라는 확장 보드가 필요해요.

**연결 순서:**

1. **마이크로비트를 Grove Shield에 끼워요** — 엣지 커넥터 슬롯에 쏙 들어가요
2. **Grove 센서를 I2C 포트에 연결해요** — 4핀 Grove 케이블로 꽂기만 하면 돼요
3. **USB로 컴퓨터에 연결해요** — 프로그래밍과 전원 공급이 동시에 돼요

Grove Shield에는 여러 개의 I2C 포트가 있어서 **센서를 동시에 여러 개 연결**할 수 있어요. 이게 I2C 통신의 큰 장점이에요.

### 6.3 I2C 통신 이해하기

Grove 센서 대부분은 **I2C**(아이-투-씨, Inter-Integrated Circuit)라는 통신 방식을 사용해요.

> **[그림 6] I2C 버스 개념도**

#### I2C를 교실 인터콤에 비유하면

I2C 버스를 교실의 **인터콤 시스템**에 비유해 볼게요.

| I2C 구성 요소 | 교실 비유 |
|--------------|----------|
| 마이크로비트 (마스터) | 선생님 (질문하는 사람) |
| 센서들 (슬레이브) | 학생들 (대답하는 사람들) |
| I2C 버스 (SDA + SCL) | 인터콤 선 (모두 같은 선에 연결) |
| I2C 주소 | 학생 번호 (각 센서의 고유 번호) |

선생님이 "3번 학생, 지금 온도가 몇 도야?"라고 물으면, 3번 학생만 대답해요. "7번 학생, 습도는?"이라고 물으면 7번 학생이 대답하고요. 이렇게 **하나의 버스에 여러 센서를 연결**하고, 주소로 구분해서 각각의 센서에 개별적으로 질문할 수 있어요.

#### I2C의 핵심 포인트

| 특징 | 설명 |
|------|------|
| 2개의 선 | SDA(데이터 선) + SCL(클럭 선)만 있으면 돼요 |
| 고유 주소 | 각 센서에 고유한 주소(예: 0x44)가 있어요 |
| 동시 연결 | 하나의 버스에 여러 센서를 연결할 수 있어요 |
| 주소 충돌 주의 | 같은 주소를 가진 센서 2개를 동시에 연결하면 안 돼요 |

### 6.4 주요 Grove I2C 센서 소개

이제 우리 수업에서 사용할 수 있는 주요 Grove I2C 센서를 소개할게요. 각 센서의 특징과 프로젝트 활용 가능성을 함께 살펴봐요.

> **[그림 7] Grove I2C 센서 종류 (사진 모음)**

#### 🌡️ SHT40 — 온도·습도 센서

| 항목 | 내용 |
|------|------|
| I2C 주소 | 0x44 |
| 측정 항목 | 온도 (−40~125℃), 습도 (0~100%) |
| 정확도 | 온도 ±0.2℃, 습도 ±1.8% |

마이크로비트 내장 온도 센서보다 **훨씬 정확**해요. 그리고 마이크로비트로는 측정할 수 없는 **습도**까지 측정할 수 있어요. 교실 환경 모니터링 프로젝트의 핵심 센서예요.

**활용 예시:** 교실 온도·습도 변화 추적, "습도가 높을수록 졸린가?" 탐구

#### 💡 TSL2561 — 조도 센서

| 항목 | 내용 |
|------|------|
| I2C 주소 | 0x29 |
| 측정 항목 | 조도 (0.1~40,000 lux) |
| 특징 | 사람 눈과 비슷한 감도, 실제 lux 값 제공 |

마이크로비트 내장 조도 센서(LED 역이용)와 달리, **실제 lux 값**을 정확하게 측정할 수 있어요. 교실 조명이 학습 환경 기준(300 lux 이상)에 맞는지 확인할 수 있어요.

**활용 예시:** 교실 밝기 측정, 시간대별 자연광 변화 추적

#### 🎨 TCS34725 — 색상 센서

| 항목 | 내용 |
|------|------|
| I2C 주소 | 0x29 |
| 측정 항목 | RGB 색상값 (빨강, 초록, 파랑) + 밝기 |
| 특징 | 물체의 색을 RGB 숫자로 감지 |

물체의 색상을 감지할 수 있는 센서예요. 과학 실험(용액 색 변화), 미술 활동(색 혼합) 등 창의적인 프로젝트에 활용할 수 있어요.

> ⚠️ **주의**: TSL2561과 I2C 주소가 동일(0x29)해서 **두 센서를 동시에 연결할 수 없어요**.

**활용 예시:** 과학 실험 색 변화 관찰, 물체 분류

#### 📏 VL53L0X — 거리 센서 (ToF)

| 항목 | 내용 |
|------|------|
| I2C 주소 | 0x29 |
| 측정 항목 | 거리 (30~2000mm) |
| 원리 | 레이저를 쏘고 돌아오는 시간 측정 (Time of Flight) |

레이저를 이용해서 거리를 측정하는 센서예요. **초음파보다 정확하고 빠르며**, 측정 범위는 약 3cm~2m예요.

**활용 예시:** 사람 유무 감지, 줄 길이 측정, 자세 변화 감지

#### 🌤️ BMP280 — 기압 센서

| 항목 | 내용 |
|------|------|
| I2C 주소 | 0x76 |
| 측정 항목 | 기압 (300~1100 hPa), 온도 |
| 특징 | 기압으로 고도 추정 가능 |

기압과 온도를 측정하는 센서예요. 기압 변화를 통해 **날씨 변화를 추적**하거나, 건물의 **층수를 감지**할 수도 있어요.

**활용 예시:** 날씨 변화와 학생 행동 관계, 층간 이동 추적

#### 🌍 BME680 — 환경 종합 센서

| 항목 | 내용 |
|------|------|
| I2C 주소 | 0x76 |
| 측정 항목 | 온도, 습도, 기압, 공기질(VOC 가스) |
| 특징 | 4가지를 한 번에 측정하는 만능 센서 |

센서 하나로 온도·습도·기압·공기질 **4가지를 동시에 측정**할 수 있는 만능 센서예요. 교실 환경을 종합적으로 분석하고 싶을 때 가장 효율적이에요.

> ⚠️ **주의**: BMP280과 I2C 주소가 동일(0x76)해서 **두 센서를 동시에 연결할 수 없어요**. BME680이 BMP280의 상위 호환이므로, 둘 중 하나만 선택한다면 BME680이 유리해요.

**활용 예시:** 교실 환경 종합 분석, 환기 전후 공기질 비교

#### 🏃 LIS3DHTR — 가속도 센서

| 항목 | 내용 |
|------|------|
| I2C 주소 | 0x18 |
| 측정 항목 | 3축 가속도 (±2g/4g/8g/16g) |
| 특징 | 마이크로비트 내장보다 정밀, 설정 범위 넓음 |

마이크로비트 내장 가속도계보다 **정밀하고 설정 범위가 넓은** 외부 가속도 센서예요. 운동량 측정, 진동 분석 등 활동량 관련 프로젝트에 유용해요.

**활용 예시:** 체육 시간 활동량 측정, 진동 패턴 분석

### 6.5 센서 주소 충돌 정리

I2C 센서를 조합할 때, 같은 주소를 가진 센서끼리는 동시에 사용할 수 없어요. 조합을 계획할 때 이 점을 주의해야 해요.

| I2C 주소 | 해당 센서들 | 동시 사용 |
|:--------:|-----------|:---------:|
| 0x18 | LIS3DHTR | 단독 사용 가능 |
| 0x29 | TSL2561, TCS34725, VL53L0X | 3개 중 **1개만** 선택 |
| 0x44 | SHT40 | 단독 사용 가능 |
| 0x76 | BMP280, BME680 | 2개 중 **1개만** 선택 |

### 6.6 센서 조합의 다양성

마이크로비트 내장 센서 4종(온도, 소음, 빛, 가속도)과 Grove 외부 센서를 조합하면, **848가지 이상의 센서 조합**이 가능해요.

**프로젝트별 추천 센서 조합 예시:**

| 프로젝트 주제 | 내장 센서 | Grove 센서 | 측정 항목 수 |
|-------------|----------|-----------|:----------:|
| 교실 환경 종합 | 소음, 빛 | BME680 (환경 종합) | 6가지 |
| 교실 온습도·조도 | 소음 | SHT40, TSL2561 | 5가지 |
| 운동장 활동량 | 가속도 | LIS3DHTR, SHT40 | 5가지 |
| 사람 감지·환경 | 소음 | VL53L0X, SHT40 | 4가지 |
| 미니멀 환경 | 온도, 소음, 빛 | (없음) | 3가지 |

## 7. 센서 시뮬레이터 소개

### 7.1 왜 시뮬레이터가 필요할까?

실제 마이크로비트와 Grove 센서는 다음 시간(3~4주차)부터 본격적으로 사용해요. 하지만 오늘 **시뮬레이터**를 통해 미리 센서 조합을 체험해 볼 수 있어요.

시뮬레이터를 사용하면 다음과 같은 이점이 있어요.

- 실제 하드웨어 없이 센서 배치와 데이터를 확인할 수 있어요
- 센서 조합을 자유롭게 실험해 볼 수 있어요
- I2C 주소 충돌을 자동으로 감지해 줘요
- 프로젝트 계획을 세우기 전에 미리 검토할 수 있어요

### 7.2 센서 시뮬레이터 v9

센서 시뮬레이터는 마이크로비트와 Grove Shield를 화면에서 보여주고, 최대 6개의 센서를 배치하면서 가상 데이터를 확인할 수 있는 웹 도구예요.

> **[그림 8] 센서 시뮬레이터 화면 캡처**

**주요 기능:**

| 기능 | 설명 |
|------|------|
| 센서 슬롯 배치 | 6개 슬롯에 원하는 센서를 드래그하여 배치 |
| 가상 데이터 | 각 센서의 실시간 가상 측정값 표시 |
| I2C 주소 충돌 감지 | 같은 주소 센서를 넣으면 경고 표시 |
| 조합 통계 | 총 센서 수, 데이터 유형 수 자동 계산 |

### 7.3 센서 검색 도구

"어떤 센서가 내 프로젝트에 맞을까?" 고민될 때 사용하는 도구예요. 자연어로 원하는 것을 입력하면 적합한 센서를 추천해 줘요.

> **[그림 9] 센서 검색 도구 화면 캡처**

**사용 예시:**

| 입력 | 추천 결과 |
|------|----------|
| "교실 온도와 습도를 측정하고 싶어요" | 내장 온도 + SHT40 (습도) |
| "사람이 지나가는지 알고 싶어요" | VL53L0X (거리 센서) |
| "교실 환경을 종합적으로 측정하고 싶어요" | BME680 (환경 종합) + 내장 소음 |

### 7.4 I2C 연결도

I2C 연결도는 **센서별 I2C 주소와 연결 구조**를 시각적으로 보여주는 도구예요.

> **[그림 10] I2C 연결도 화면 캡처**

마이크로비트에서 나오는 I2C 버스(SDA, SCL 두 선)에 각 센서가 고유 주소로 연결되는 구조를 한눈에 확인할 수 있어요. 주소가 같은 센서끼리 충돌하는 것도 시각적으로 확인할 수 있어요.

## 8. 프로젝트 브레인스토밍 안내

### 8.1 다음 주 예고: 프로젝트 기획

다음 주(3주차)에는 여러분만의 **프로젝트 주제**를 정할 거예요. 실습학교(5~8주차)에서 실제로 데이터를 수집할 프로젝트를 기획하는 시간이에요.

프로젝트의 전체 흐름을 미리 확인해 볼게요.

```
[3주차] 프로젝트 기획 → 주제 선정, 센서 선택, 수집 계획서 작성
   ↓
[4주차] 데이터수집기 제작 → 마이크로비트+Grove 프로그래밍, 테스트
   ↓
[5~8주차] 실습학교에서 데이터 수집 → 실제 데이터 확보!
   ↓
[9~13주차] 바이브코딩으로 분석 → 시각화, 패턴 발견, AI 모델
   ↓
[14~15주차] 보고서 + 발표 → 결과 공유
```

### 8.2 학교 현장에서 뭘 측정할 수 있을까?

프로젝트 주제를 정하려면 "학교에서 센서로 무엇을 측정할 수 있을까?"를 생각해 봐야 해요.

> **[그림 11] 프로젝트 예시 개요도**

**프로젝트 예시:**

| 주제 | 사용 센서 | 핵심 질문 |
|------|----------|----------|
| 교실 환경과 집중도 | 온도, 소음, 빛, 습도 | 교실 환경이 집중도에 영향을 줄까? |
| 복도 소음 분석 | 소음 | 쉬는 시간 vs 수업 시간, 소음 차이는? |
| 급식실 대기 시간 | 거리 (사람 감지) | 줄이 가장 긴 시간대는? |
| 운동장 활동량 | 가속도 | 체육 시간에 얼마나 움직일까? |
| 도서관 이용 패턴 | 빛, 소음 | 가장 조용한 시간대는? |

### 8.3 좋은 프로젝트 주제의 조건

| 조건 | 설명 | 예시 |
|------|------|------|
| 측정 가능 | 센서로 측정할 수 있는 데이터여야 해요 | ✅ "교실 온도" / ❌ "학생 기분" |
| 비교 가능 | 두 가지 이상을 비교하면 더 흥미로워요 | "수업 시간 vs 쉬는 시간 소음 비교" |
| 학교 현장 | 실습학교에서 수집 가능해야 해요 | "교실", "복도", "운동장" 등 |
| 충분한 양 | 데이터가 너무 적으면 분석이 어려워요 | 최소 3일 이상 수집 권장 |

### 8.4 다음 주까지 생각해 올 것

- 내가 실습학교에서 측정하고 싶은 것은?
- 어떤 센서가 필요할까?
- 데이터를 모으면 어떤 질문에 답할 수 있을까?

아이디어가 떠오르면 메모해 두세요. 다음 시간에 함께 나눠봐요!

## 9. 오늘 수업 정리

### 9.1 핵심 정리

오늘 배운 내용을 정리해 볼게요.

**바이브코딩:**

바이브코딩은 코드를 직접 짜지 않고, AI에게 자연어로 설명하여 코드를 만드는 방법이에요. 프롬프트를 구체적으로, 단계적으로, 예시를 포함해서 작성하면 좋은 결과를 얻을 수 있어요.

**Claude 사용법:**

claude.ai에 접속해서 프롬프트를 입력하면 코드를 생성해 줘요. Artifact 패널에서 미리보기와 다운로드가 가능하고, 대화목록 PDF도 자동으로 생성할 수 있어요.

**마이크로비트 내장 센서:**

온도, 소음(마이크), 빛(조도), 가속도, 나침반, 터치 6가지 센서가 내장되어 있어요. 추가 장치 없이 바로 사용 가능하지만, 정밀도에 한계가 있어요.

**Grove 확장 센서:**

I2C 방식의 플러그앤플레이 센서예요. 케이블을 꽂기만 하면 돼요. 온습도(SHT40), 조도(TSL2561), 색상(TCS34725), 거리(VL53L0X), 기압(BMP280), 환경 종합(BME680), 가속도(LIS3DHTR) 등 다양한 센서를 추가할 수 있어요.

### 9.2 핵심 메시지 되새기기

| 핵심 메시지 | 오늘 수업에서의 의미 |
|------------|---------------------|
| "AI는 판단하지 않는다, 계산할 뿐이다" | Claude가 코드를 '이해'하는 게 아니라 패턴으로 생성해요 |
| "코딩을 의심하는 방법을 가르친다" | 바이브코딩 결과물을 무조건 믿지 말고 확인하세요 |
| "데이터는 원인을 말하지 않는다" | 센서가 측정하는 것과 우리가 알고 싶은 것은 달라요 |
