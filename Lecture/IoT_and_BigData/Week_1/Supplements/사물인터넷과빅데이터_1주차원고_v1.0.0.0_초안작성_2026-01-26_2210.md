## 1. 강좌 개요

### 1.1 기본 정보

- **강좌명**: 사물인터넷과 빅데이터
- **시간**: 수요일 6-7교시
- **학점**: 2학점 / 주 2시간
- **대상**: 컴퓨터교육과 4학년

### 1.2 목표

마이크로비트와 Grove 센서로 **데이터를 수집**하고, AI와 협업하여 **분석**하는 전 과정을 경험합니다.

**흐름**: 센서 → 데이터 수집 → 시각화 → 분석 → AI 모델

### 1.3 왜 이 강좌를 배우는가?

**2022 개정 교육과정**에서 초등학교 실과에 **피지컬 컴퓨팅**과 **데이터 수집·분석**이 포함되었습니다.

**연계 내용**:
- 실과 - 피지컬 컴퓨팅
- 실과 - 데이터 수집·분석
- SW/AI 교육
- 컴퓨팅 사고력

**이 강좌에서 배우는 것**:
- 센서 활용 능력
- 데이터 분석 기초
- AI 도구 활용
- 교육적 적용

## 2. 핵심 메시지

### 2.1 메시지 1: 데이터와 원인

> "데이터는 원인을 말하지 않는다. 우리가 원인을 '추정'할 뿐이다."

센서 데이터는 **현상**만 보여줍니다. **원인**은 우리가 해석해야 합니다.

### 2.2 메시지 2: AI와 기준

> "AI는 집중도를 판단하지 않는다. 우리가 만든 기준을 계산할 뿐이다."

AI는 **도구**입니다. **기준**과 **해석**은 사람의 몫입니다.

### 2.3 메시지 3: 코딩을 의심하기

> "이 수업은 코딩을 가르치지 않는다. 코딩을 의심하는 방법을 가르친다."

**바이브코딩**으로 코드를 생성하고, **비판적 사고**로 검증합니다.

## 4. 예시 프로젝트: 수업 집중도

### 4.1 핵심 질문

> 온도가 높으면 집중도가 낮아질까?

**상관관계** vs **인과관계**

### 4.2 측정 데이터

- 온도 센서 → 실내 온도
- 소음 센서 → 소음 수준
- 설문 조사 → 집중도 (1~5점)

### 4.3 프로젝트 흐름

1. 센서로 데이터 수집
2. 바이브코딩으로 시각화
3. 패턴 발견 및 해석
4. AI 모델 제작 및 비교
5. 결과 발표

## 5. 수업 집중도 시뮬레이터

### 5.1 탭 1: 데이터 입력

센서 데이터와 집중도 점수를 입력하는 화면

**입력 항목**:
- 온도 (°C)
- 소음 (dB)
- 집중도 (1-5)

### 5.2 탭 2: 데이터 분석

산점도와 상관관계를 분석하는 화면

**분석 결과 예시**:
- 상관계수: -0.72
- 해석: 강한 음의 상관
- 온도↑ → 집중도↓

**주의**: 상관관계 ≠ 인과관계

### 5.3 탭 3: 학생 모델 (규칙 기반 AI)

학생이 직접 규칙을 정하는 AI 모델

**규칙 예시**:
```
IF 온도 < 25°C AND 소음 < 50dB
THEN 집중도 = 높음

IF 온도 ≥ 28°C OR 소음 ≥ 60dB
THEN 집중도 = 낮음
```

**정확도**: 68%

### 5.4 탭 4: 교수 모델 (머신러닝 AI)

데이터가 스스로 규칙을 찾는 AI 모델

- 데이터를 입력하면 AI가 자동으로 최적의 규칙을 찾음
- 학습된 규칙: 온도×0.3 + 소음×0.5 + ...

**정확도**: 85%

### 5.5 탭 5: 모델 비교

학생 모델과 교수 모델의 정확도를 비교

| 모델 | 정확도 |
|------|--------|
| 학생 모델 (규칙 기반) | 68% |
| 교수 모델 (머신러닝) | 85% |

**핵심 메시지**: 머신러닝이 더 정확하지만, 규칙을 설명할 수 없다!

## 6. 바이브코딩 소개

### 6.1 정의

**바이브코딩(Vibe Coding)** = 코드를 직접 작성하지 않고, AI와의 대화를 통해 프로그램을 만드는 방식

### 6.2 전통적 프로그래밍 vs 바이브코딩

| 전통적 프로그래밍 | 바이브코딩 |
|------------------|-----------|
| 문법 학습 | 문제 정의 |
| 코드 작성 | AI에게 요청 |
| 디버깅 | 결과 검증 |
| 반복... | 개선 요청 |

### 6.3 바이브코딩 시연

**프롬프트 예시**:
> "마이크로비트 LED에 하트 모양을 표시하는 MakeCode 블록 코드를 알려줘"

**결과**: Claude가 블록 코드를 설명해줌

### 6.4 핵심

- 문법 몰라도 OK
- 아이디어만 있으면 가능
- 단, **검증은 필수**!

## 8. Grove 확장 시스템

### 8.1 Grove 확장이란?

I2C 포트로 다양한 외부 센서를 마이크로비트에 연결하여 데이터 수집 범위를 확장

### 8.2 연결 가능 센서

- 온습도 센서 (SHT40)
- 기압 센서 (BMP280)
- CO2/VOC 센서 (SGP30)
- 거리 센서 (VL53L0X)
- 색상/조도 센서

### 8.3 최대 연결

I2C 포트로 최대 **6개** 센서 동시 연결 가능

## 9. MakeCode 소개

### 9.1 MakeCode란?

마이크로비트 프로그래밍을 위한 **블록 코딩** 환경

### 9.2 장점

- 드래그 앤 드롭으로 코딩
- 실시간 시뮬레이터
- 한글 지원
- 웹 브라우저에서 실행

### 9.3 접속 방법

**makecode.microbit.org** → 지금 바로 접속해봅시다!

3. **대화목록 요약본** (PDF): 프롬프트 + Claude 응답 3줄 요약

**제출 안내**:
