
## 1. 개요

- **대상 시뮬레이터**: MakeCode시뮬레이터LED이름표시 (1주차 강의자료)
- **이미지 현황 요약**: PNG 스크린샷 2장 + SVG 다이어그램 9장

| 항목 | 내용 |
|------------|------------------------------------------------------------|
| 총 이미지 수 | 11장 (PNG 2 + SVG 9) |
| 형식 | PNG (스크린샷), SVG (벡터 다이어그램) |
| 용도 | 개발일지, 퀵가이드, 사용설명서, 활용가이드, 이론해설서에 삽입 |

## 2. 디자인 규격

### 2.1 PNG 스크린샷

| 항목 | 값 |
|------|-----|
| 캡처 대상 | 시뮬레이터 HTML (아티팩트 뷰어) |
| 배경색 | #1a1a2e (시뮬레이터 기본 배경) |
| 주요 색상 | #7c5fd6 (기본 블록), #6d4bc5 (LED 블록), #00d4aa (LED 점등) |
| 해상도 | 원본 캡처 해상도 유지 |

### 2.2 SVG 다이어그램

| 항목 | 값 |
|------|-----|
| 폰트 | Noto Sans CJK KR |
| 배경색 | 흰색 (레이아웃 다이어그램은 #1a1a2e) |
| 변환 | SVG → PDF (rsvg-convert), LaTeX \includegraphics 삽입 |
| 생성 방법 | 방법 A (유니코드 다이어그램 → SVG) — 문서관리지침 §3 |

## 5. 재생성 가이드

### 5.1 PNG 스크린샷

1. 시뮬레이터 HTML을 아티팩트 뷰어 또는 브라우저에서 열기
2. 원하는 상태 재현 (초기 화면 / 블록 배치 후 실행)
3. 전체 화면 캡처 (블록 팔레트 + 코딩 영역 + 시뮬레이터 패널 포함)
4. PNG 형식 저장, 파일명 규칙 준수

### 5.2 SVG 다이어그램

1. SVG 원본 편집 (텍스트 에디터 또는 Claude 재생성)
2. font-family: `Noto Sans CJK KR` 사용
3. PDF 변환: `rsvg-convert input.svg -f pdf -o output.pdf`
4. LaTeX 삽입: `\includegraphics[width=...]{파일명.pdf}`

### 5.3 파일명 규칙

```
[문서종류]_[번호]_[내용].확장자  ← 원본
[문서종류]_[번호]_[내용]_v버전_수정키워드_날짜_시간.확장자  ← 수정 시
```

| 접두어 | 설명 |
|--------|------|
| 공통 | 여러 문서에서 공유하는 이미지 |
| 활용가이드, 이론해설서 | 해당 문서 전용 이미지 |
| 번호 | 01부터 2자리 순번 |
