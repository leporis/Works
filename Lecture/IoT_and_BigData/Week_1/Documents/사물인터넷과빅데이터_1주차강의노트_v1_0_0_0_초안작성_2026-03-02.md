## 1. 강좌에 오신 것을 환영합니다! 🎉

### 1.1 이 강좌에서 배우는 것

여러분, 반갑습니다! 사물인터넷과 빅데이터 강좌에 오신 것을 환영해요.

혹시 이런 생각 해본 적 있나요?

> "교실이 왜 이렇게 덥지? 애들이 집중을 못 하는 것 같은데…"
> "운동장 소음이 너무 심한데, 수업에 영향이 있을까?"

이 강좌에서는 이런 **현장 교사의 궁금증**을 데이터로 탐구하는 방법을 배워요.

핵심 흐름: **센서 → 데이터 수집 → 시각화 → 분석 → AI 모델**

그리고 가장 신기한 건, 이 모든 과정에서 **코드를 한 줄도 직접 작성하지 않아요!** 바로 바이브코딩 덕분이에요.

## 2. 사물인터넷(IoT)과 빅데이터란?

### 2.1 사물인터넷(IoT)이란?

**IoT = Internet of Things = 사물인터넷**

쉽게 말하면, "인터넷에 연결된 모든 것"이에요.

- 온도계가 인터넷에 연결되면 → **스마트 온도계**
- 전구가 인터넷에 연결되면 → **스마트 전구**
- 교실 환경 센서가 인터넷에 연결되면 → **스마트 교실**

| 분류 | 예시 |
|:----:|:-----|
| 스마트홈 | 스마트 전구, AI 스피커, 스마트 도어록 |
| 스마트시티 | 환경 모니터링 센서, 교통 신호 제어 |
| 스마트팜 | 온습도 자동 제어, 자동 관개 시스템 |
| 스마트스쿨 | 출결 관리, 교실 환경 모니터링 |

이 강좌에서는 **마이크로비트 + Grove 센서**로 간단한 IoT 데이터 수집기를 만들어요.

### 2.2 빅데이터란?

**빅데이터**는 단순히 "데이터가 많다"는 뜻이 아니에요. 3V로 설명해요.

| V | 이름 | 의미 | 예시 |
|:-:|:----:|:-----|:-----|
| Volume | 규모 | 데이터 양이 방대하다 | 하루 SNS 게시글 수 억 건 |
| Velocity | 속도 | 데이터 생성 속도가 빠르다 | 실시간 센서 데이터 |
| Variety | 다양성 | 다양한 형태의 데이터 | 숫자, 사진, 음성, 텍스트 |

> 💡 **이 강좌에서의 빅데이터**: 우리는 규모는 작지만 **실제 현장 데이터**를 수집해요. 센서 데이터(숫자) + 설문 데이터(응답)를 결합하는 방식이 실제 빅데이터 분석과 동일한 접근법이에요.

### 2.3 IoT + 빅데이터 = 이 강좌의 흐름

```
[센서] → [데이터 수집] → [빅데이터 저장] → [분석·시각화] → [AI 모델] → [의사결정]
```

우리 강좌에 적용하면:

```
[마이크로비트+Grove] → [실습학교 현장 수집] → [엑셀 정리] → [바이브코딩 시각화] → [규칙 기반 AI] → [교육 환경 개선]
```

## 3. 이 강좌의 핵심 메시지 3가지 💬

이 강좌 전체를 관통하는 메시지 3가지를 꼭 기억해 두세요!

### 3.1 메시지 1: 데이터는 원인을 말하지 않아요

> "데이터는 원인을 말하지 않는다. 우리가 원인을 '추정'할 뿐이다."

온도가 28°C이고 집중도가 2.3점이라고 해서, "온도 때문에 집중도가 떨어졌다"고 말할 수 없어요.

데이터는 **현상만 보여줄 뿐**, 원인은 우리가 추정하는 거예요.

**왜 이게 중요할까요?**
- 교사가 "온도가 높아서 집중도가 낮다"고 결론 내리면 → 에어컨만 켜면 된다고 생각할 수 있어요.
- 하지만 실제 원인은 "점심 시간 직후 수업"이거나 "단조로운 수업 방식"일 수도 있어요.
- **데이터 분석가의 역할**: 데이터에서 패턴을 찾되, 원인 추정은 신중하게 해야 해요.

### 3.2 메시지 2: AI는 판단하지 않아요

> "AI는 집중도를 판단하지 않는다. 우리가 만든 기준을 계산할 뿐이다."

AI가 "집중도 낮음"이라고 판정한다면, 그건 **우리가 정해준 규칙**을 실행한 결과예요.

- `IF 온도 >= 28 → 집중도 낮음` → 이 규칙을 만든 건 사람이에요.
- AI는 이 규칙에 따라 계산할 뿐, 스스로 판단하지 않아요.

**교육 현장에서의 의미**: AI 보조 교사 시스템이 "이 학생은 집중력이 낮아요"라고 말한다면, 그 기준은 누가 정했는지 반드시 확인해야 해요.

### 3.3 메시지 3: 코딩을 의심하세요

> "이 수업은 코딩을 가르치지 않는다. 코딩을 의심하는 방법을 가르친다."

AI가 만든 코드가 항상 맞지는 않아요. 결과를 검증하고 수정 요청하는 방법을 배워요.

- AI는 틀린 코드를 자신 있게 제시할 수 있어요.
- 코드가 "작동한다"는 것이 "올바르다"는 뜻은 아니에요.
- **비판적 사고**가 바이브코딩의 핵심이에요.

## 5. 예시 프로젝트: 수업 집중도 📊

### 5.1 핵심 질문

**"온도가 높으면 집중도가 낮아질까?"**

이 질문 하나로 15주 전체 과정을 보여드릴게요.

### 5.2 데이터 수집 계획

- **측정 변수 (센서)**: 온도(°C), 소음(dB)
- **주관 변수 (설문)**: 집중도 점수(1~5점)
- **수집 방법**: 마이크로비트로 온도·소음 측정 + 설문지로 집중도 수집
- **수집 시기**: 교육실습 기간 (5–8주)

### 5.3 상관관계 vs 인과관계

데이터를 분석하다 보면 "온도가 높을 때 집중도가 낮네?"라는 패턴을 발견할 수 있어요. 이것이 **상관관계**예요.

| 구분 | 설명 | 확인 방법 |
|:----:|:-----|:---------|
| **상관관계** (Correlation) | 두 변수가 함께 변하는 경향 | 데이터 분석으로 확인 가능 |
| **인과관계** (Causation) | 한 변수가 다른 변수의 원인 | 통제 실험이 필요 |

> ⚠️ **주의**: 상관관계 ≠ 인과관계
>
> "아이스크림 판매량이 늘면 익사 사고가 늘어난다"는 상관관계가 있어요. 하지만 아이스크림이 익사의 원인은 아니에요. **공통 원인**(더운 여름)이 있는 거예요.

### 5.4 반례를 찾아라! 🔍

"온도가 높으면 집중도가 낮다"는 가설이 맞을까요?

- 여름에 에어컨 빵빵한 교실에서도 집중 못 하는 날이 있어요 → 반례!
- 온도가 25°C인데도 집중도 5점인 수업이 있어요 → 반례!

**반례가 있다면**: 가설을 수정하거나, 다른 요인을 고려해야 해요.

이것이 **비판적 데이터 분석**이에요.

## 6. 수업집중도 분석 프로그램 시연 🖥️

### 6.1 바이브코딩으로 만든 프로그램

이 프로그램은 **Claude AI와 대화만으로** 만들어진 웹 애플리케이션이에요. 코드 한 줄 직접 작성하지 않았어요!

5개 탭으로 구성되어 있어요.

### 6.2 탭 1: 데이터 입력

온도(°C), 소음(dB), 집중도(1~5점)를 입력하는 탭이에요.

- **센서 데이터**: 마이크로비트로 측정한 객관적 수치
- **설문 데이터**: 학생들이 응답한 주관적 수치

> 💡 **포인트**: 센서 데이터는 **객관적**, 집중도 설문은 **주관적**이에요. 두 종류의 데이터를 결합해서 분석하는 게 이 프로젝트의 핵심이에요.

### 6.3 탭 2: 데이터 분석

산점도(Scatter Plot)와 상관계수를 자동으로 계산해요.

**상관계수(r) 해석**:

| 범위 | 해석 |
|:----:|:-----|
| 0.8 ~ 1.0 | 강한 양의 상관 |
| 0.5 ~ 0.8 | 중간 양의 상관 |
| -0.5 ~ 0.5 | 약한 상관 또는 없음 |
| -0.8 ~ -0.5 | 중간 음의 상관 |
| -1.0 ~ -0.8 | 강한 음의 상관 |

예시 데이터에서 상관계수 **r = -0.72** → 강한 음의 상관 (온도↑ → 집중도↓ 경향)

> ⚠️ **주의**: r = -0.72라고 해서 "온도가 원인"이라고 단정할 수 없어요!

### 6.4 탭 3: 학생 모델 (규칙 기반 AI)

여러분이 직접 규칙을 만들어요.

```
IF 온도 < 25 AND 소음 < 50 → 집중도 = 높음
IF 온도 >= 28 OR 소음 >= 60 → 집중도 = 낮음
그 외                        → 집중도 = 보통
```

숫자를 바꿔가며 정확도를 높여보세요. 정확도 68%를 달성했어요.

**학생 모델의 장점**: 규칙을 직접 만들었기 때문에 "왜 이런 결과인지" 설명할 수 있어요!

### 6.5 탭 4: 교수 모델 (머신러닝 AI)

데이터를 보고 스스로 수식을 찾아내요.

```
집중도 = -0.3 × 온도 - 0.2 × 소음 + 12.5
```

정확도 85%를 달성했어요. 학생 모델(68%)보다 훨씬 높아요!

하지만 **블랙박스 문제**가 있어요.
- 계수 `-0.3`, `-0.2`가 무엇을 의미하는지 직관적으로 이해하기 어려워요.
- 교육 현장에서는 "왜?"를 설명할 수 있어야 해요.

### 6.6 탭 5: 모델 비교

| 모델 | 정확도 | 설명 가능성 |
|:----:|:------:|:-----------:|
| 학생 모델 (규칙 기반) | 68% | ✅ 높음 |
| 교수 모델 (머신러닝) | 85% | ❌ 낮음 |

> 🎯 **핵심 메시지**: 머신러닝이 더 정확하지만, "왜 그런 결과가 나왔는지" 설명하기 어려워요. 이걸 **블랙박스 문제**라고 해요.
>
> 초등학교 교육 현장에서는 규칙 기반 모델(학생 모델)도 중요한 가치가 있어요!

## 7. 바이브코딩이란? 💻

### 7.1 정의

**바이브코딩(Vibe Coding)** = 코드를 직접 작성하지 않고, AI와의 대화를 통해 프로그램을 만드는 방식

2025년 Andrej Karpathy(전 Tesla AI 디렉터)가 처음 제안한 개념이에요.

### 7.2 전통적 프로그래밍 vs 바이브코딩

| 구분 | 전통적 프로그래밍 | 바이브코딩 |
|:----:|:----------------:|:---------:|
| 시작 방법 | 문법 공부 | 문제 정의 |
| 코드 작성 | 직접 작성 | AI에게 요청 |
| 오류 해결 | 오류 메시지 해석 | 결과 확인 후 수정 요청 |
| 소요 시간 | 길다 | 짧다 |
| 필요 역량 | 코딩 문법 | 문제 정의 능력 |

### 7.3 바이브코딩의 3단계

```
[1단계] 문제 정의
  "내가 원하는 게 뭔지" 명확히 정의하기
         ↓
[2단계] AI에게 요청 (프롬프트 작성)
  Claude에게 자연어로 설명하기
         ↓
[3단계] 결과 확인 및 수정 요청
  결과가 맞는지 확인 → 수정 요청 반복
```

### 7.4 좋은 프롬프트 작성법

바이브코딩의 핵심은 **좋은 프롬프트**를 작성하는 거예요.

**좋은 프롬프트의 3가지 요소**:

1. **구체적**: 모호한 표현 없이 정확하게
2. **조건 명시**: 동작 조건과 예외 상황 포함
3. **출력 형식 지정**: 어떤 형태로 결과를 원하는지

**예시 비교**:

| ❌ 나쁜 프롬프트 | ✅ 좋은 프롬프트 |
|:--------------|:--------------|
| "마이크로비트 코드 만들어줘" | "마이크로비트 LED에 '홍길동'을 한 글자씩 표시해줘. 각 글자는 1초 표시 후 다음 글자로 넘어가고, 무한 반복해줘. MakeCode 블록 코드로." |
| "집중도 분석해줘" | "온도, 소음, 집중도 데이터 CSV를 입력받아 상관계수를 계산하고 산점도를 그리는 HTML 파일을 만들어줘" |

### 7.5 바이브코딩에서의 비판적 사고

AI가 준 코드를 그냥 쓰면 안 돼요. 꼭 이것들을 확인하세요.

- **기능이 제대로 작동하나요?** → 직접 테스트
- **의도한 대로 작동하나요?** → 엣지 케이스(극단적 상황) 확인
- **코드가 이해되나요?** → 이해 안 되는 부분은 AI에게 설명 요청

## 8. 마이크로비트 소개 🔬

### 8.1 마이크로비트란?

**마이크로비트(micro:bit)**는 영국 BBC가 개발한 교육용 마이크로컨트롤러예요.

- 손바닥만 한 크기
- 25개 LED 매트릭스
- 다양한 내장 센서
- MakeCode로 쉽게 프로그래밍 가능

> 📚 **역사**: 2016년 영국 중학교 1학년 학생 100만 명에게 무료 배포하며 시작됐어요. 지금은 전 세계 수천만 명이 사용해요.

### 8.2 내장 센서 6종

마이크로비트 V2에는 **6가지 센서**가 내장되어 있어요. 별도 하드웨어 없이 바로 사용할 수 있어요!

| 센서 | 측정 대상 | 활용 예시 | 단위 |
|:----:|:--------:|:---------|:----:|
| 온도 센서 | 실내 온도 | 교실 쾌적도 측정 | °C |
| 빛 센서 | 조도 | 학습 환경 밝기 | 0–255 |
| 가속도 센서 | 움직임, 기울기 | 운동량, 낙하 감지 | mg |
| 나침반 센서 | 방향 | 방위 탐색 | 0–360° |
| 마이크 (V2) | 소리 크기 | 소음 수준 측정 | dB |
| 터치 로고 (V2) | 터치 입력 | 버튼 대용 | 켜짐/꺼짐 |

> ⚠️ **주의**: 마이크로비트의 온도 센서는 CPU 온도를 측정해요. 따라서 실제 실내 온도보다 약 2–5°C 높게 측정될 수 있어요. 정밀한 측정이 필요할 때는 Grove 외부 센서를 사용해요.

### 8.3 LED 매트릭스 활용

5×5 = 25개 LED로 다양한 표현이 가능해요.

- 문자 표시: 한 글자씩 스크롤
- 숫자 표시: 측정값 표시
- 아이콘 표시: 하트, 체크 등
- 애니메이션: 연속 이미지 전환

## 9. Grove 확장 시스템 🌿

### 9.1 Grove란?

**Grove**는 Seeed Studio에서 만든 **모듈형 센서 시스템**이에요.

- 모든 센서가 **같은 커넥터 규격**을 사용해요
- 연결 방향이 정해져 있어서 **거꾸로 꽂을 수 없어요**
- 다양한 센서를 쉽게 교체할 수 있어요

### 9.2 Grove Shield (확장 보드)

마이크로비트에 Grove 센서를 연결하려면 **Grove Shield**가 필요해요.

마이크로비트를 Shield에 꽂으면 I2C 포트 4개가 생겨요. 여기에 Grove 센서를 연결해요.

### 9.3 I2C 통신

Grove 센서는 **I2C (I-Squared-C)** 통신을 사용해요.

| 특징 | 내용 |
|:----:|:-----|
| 연결 선 수 | 2개 (SDA: 데이터, SCL: 클럭) |
| 연결 가능 센서 | 최대 6개 동시 연결 |
| 주소 구분 | 각 센서가 고유 주소(0x44, 0x39 등) 사용 |
| 장점 | 적은 선으로 여러 센서 연결 가능 |

**I2C 연결 구조**:
```
마이크로비트 V2
      ↓
Grove Shield (확장 보드)
      ↓
  ┌───┴───┬───────┐
SHT35   TSL2561   VL53L0X  ... (최대 6개)
(온습도) (조도)   (거리)
0x44    0x39     0x29
```

### 9.4 주요 Grove 센서 10종

이 강좌에서 활용할 수 있는 Grove 센서들이에요.

| # | 센서명 | 측정 항목 | I2C 주소 | 교육 활용 |
|:-:|:------:|:--------:|:--------:|:---------|
| 1 | SHT35 | 온도, 습도 | 0x44 | 교실 쾌적도 |
| 2 | TSL2561 | 조도(lux) | 0x39 | 학습 환경 밝기 |
| 3 | BMP280 | 기압, 고도 | 0x76 | 날씨 변화 탐구 |
| 4 | SGP30 | CO₂, TVOC | 0x58 | 환기 필요성 판단 |
| 5 | VL53L0X | 거리(mm) | 0x29 | 출석 감지, 접근 감지 |
| 6 | TCS34725 | 색상(RGB) | 0x29 | 색 분류, 미술 활동 |
| 7 | MCP9808 | 정밀 온도 | 0x18 | 정밀 온도 측정 |
| 8 | AHT20 | 온도, 습도 | 0x38 | 저렴한 온습도 측정 |
| 9 | ADXL345 | 가속도 | 0x53 | 운동 분석 |
| 10 | OLED | 디스플레이 | 0x3C | 결과 현장 표시 |

> 💡 **프로젝트 힌트**: 어떤 센서를 선택할지가 곧 "무엇을 측정할지"를 결정해요. 3–4주차에 프로젝트 주제와 함께 센서를 선택해요.

## 10. MakeCode 소개 🧩

### 10.1 MakeCode란?

**MakeCode**는 마이크로비트를 프로그래밍하는 **블록 코딩 환경**이에요.

- 웹 브라우저에서 실행 (https://makecode.microbit.org)
- 설치 없이 사용 가능
- 한글 지원
- 블록 코딩 + JavaScript 동시 지원

### 10.2 화면 구성

MakeCode 화면은 크게 3부분으로 나뉘어요.

| 영역 | 역할 |
|:----:|:-----|
| 왼쪽 블록 팔레트 | 사용 가능한 블록 목록 |
| 가운데 코드 영역 | 블록을 드래그해서 코딩 |
| 오른쪽 시뮬레이터 | 코드 실행 결과 미리보기 |

### 10.3 기본 블록 4가지

| 블록 | 역할 | 언제 사용 |
|:----:|:-----|:---------|
| `시작하면 실행` | 전원 켜면 1번만 실행 | 초기 설정 |
| `무한반복 실행` | 계속 반복 | 지속적 측정 |
| `문자열 보여주기` | LED에 문자 스크롤 | 값 표시 |
| `일시정지 (ms)` | 지정 시간 대기 | 속도 조절 |

### 10.4 마이크로비트에 업로드하기

1. USB 케이블로 마이크로비트 연결
2. MakeCode에서 "다운로드" 클릭
3. `.hex` 파일이 다운로드됨
4. 파일을 마이크로비트 드라이브로 복사 (드래그 앤 드롭)
5. 마이크로비트가 자동으로 프로그램 실행!

> 💡 **시뮬레이터 활용**: 마이크로비트 없이도 MakeCode의 내장 시뮬레이터로 동작을 먼저 확인할 수 있어요.

## 11. 오늘의 실습: LED 이름 표시 🔴

### 11.1 실습 목표

마이크로비트 LED에 **자신의 이름을 한 글자씩 표시**하는 프로그램을 바이브코딩으로 만들어요.

이 실습을 통해 배우는 것:
- Claude(AI)에게 코딩 요청하는 방법
- MakeCode에서 프로그램 만들기
- 마이크로비트에 업로드하기
- 바이브코딩의 작동 방식 직접 체험

### 11.2 바이브코딩 실습 단계

**[1단계]** Claude에게 프롬프트 작성

```
"마이크로비트 LED에 '홍길동'을 한 글자씩 순서대로 표시하는 MakeCode
블록 코드를 알려줘. 각 글자는 1초간 표시되고 다음 글자로 넘어가도록 해줘.
프로그램이 시작되면 무한 반복해줘."
```

**[2단계]** Claude의 답변에서 블록 코드 확인

**[3단계]** MakeCode에서 구현 (블록 직접 배치 또는 JavaScript 붙여넣기)

**[4단계]** 시뮬레이터에서 결과 확인

**[5단계]** 마이크로비트에 업로드

**[6단계]** 실행 사진 촬영 📸

### 11.3 실습 심화: 개선 요청해보기

기본 실습이 완료되면 AI에게 개선을 요청해봐요.

- "글자 표시 속도를 2초로 바꿔줘"
- "이름 표시가 끝나면 하트 아이콘을 보여줘"
- "버튼 A를 누를 때만 이름이 표시되도록 해줘"

개선 요청 → 결과 확인 → 다시 개선 요청의 반복이 바이브코딩이에요.

### 11.4 트러블슈팅 가이드

실습 중 문제가 생기면 이렇게 해결해요.

| 문제 | 해결 방법 |
|:----:|:---------|
| 마이크로비트가 인식 안 됨 | USB 케이블 확인 (데이터 전송용인지 충전용인지) |
| LED에 아무것도 안 나옴 | 전원 확인, 프로그램 다시 업로드 |
| 한글이 깨짐 | 마이크로비트는 한글 지원 안 함 → 이니셜(영문)로 변경 |
| AI 답변이 이상함 | 프롬프트를 더 구체적으로 수정해서 다시 요청 |

## 12. 데이터 리터러시란? 📈

### 12.1 정의

**데이터 리터러시(Data Literacy)** = 데이터를 읽고, 이해하고, 분석하고, 활용하는 능력

### 12.2 데이터 리터러시의 3단계

| 단계 | 이름 | 내용 |
|:----:|:----:|:-----|
| 1단계 | 읽기 | 그래프, 표를 이해하기 |
| 2단계 | 분석 | 패턴 찾기, 상관관계 파악 |
| 3단계 | 비판 | 데이터의 한계 인식, 원인 추정의 근거 제시 |

이 강좌는 3단계 모두를 다뤄요. 특히 **3단계(비판)**에 집중해요.

### 12.3 초등 교사에게 필요한 이유

2022 개정 교육과정 초등 실과에 데이터 수집·분석이 포함됐어요.

- 학생들이 "데이터는 뭐예요?"라고 물으면?
- 학생들이 "AI가 이렇다고 했어요!"라고 하면?

이 강좌에서 직접 경험한 내용이 그 대답의 근거가 돼요.
