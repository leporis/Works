## 2. 주간 과제 (35%)

### 2.1 운영 방식

| 항목 | 내용 |
|:---|:---|
| **배점** | 주당 5% × 7주 = 35% |
| **제출 주차** | 2-4주, 8-13주 (1주차는 점검용) |
| **제출 기한** | 수업 종료 후 10분까지 (LMS) |
| **지각 감점** | 1일당 10% 감점 |
| **출석 인정** | 과제 제출로 출석 대체 |

### 2.2 주차별 과제 및 제출물

| 주차 | 과제 | VC | 제출물 | 배점 |
|:---:|:---|:---:|:---|:---:|
| **1** | LED 이름 표시 | | 사진 + 개발일지(PDF) | **점검용** |
| **2** | 바이브코딩 체험 | O | 결과물(HTML) + 대화목록 전문(PDF) + 요약본(PDF) | 5% |
| **3** | 프로젝트 기획 | | 데이터 수집 계획서 + 개발일지(PDF) | 5% |
| **4** | 데이터수집기 제작 | | MakeCode 파일 + 테스트 결과 + 개발일지(PDF) | 5% |
| **5-7** | 데이터 수집 | | 없음 (교육실습 기간, 8주차에 통합 제출) | - |
| **8** | 데이터 정리 | | 엑셀 파일 + 개발일지(PDF) | 5% |
| **9** | 바이브코딩 시각화 | O | 결과물(HTML) + 대화목록 전문(PDF) + 요약본(PDF) | 5% |
| **10** | 바이브코딩 분석 | O | 결과물(HTML) + 해석 메모 + 대화목록(PDF) | 5% |
| **11** | 바이브코딩 분석 심화 | O | 결과물(HTML) + 반례 분석 메모 + 대화목록(PDF) | 5% |
| **12** | 바이브코딩 AI 모델 | O | 결과물(HTML) + 대화목록(PDF) | 5% |
| **13** | 바이브코딩 분석 완료 | O | 결과물(HTML) + 보고서 초안(PDF) + 대화목록(PDF) | 5% |

> **VC = 바이브코딩**. O 표시 주차는 대화목록 전문 + 요약본 필수 제출

|:---:|:---:|:---|

| **B** | 85% | 과제 대부분 수행 + 개발일지 충실 |

| **D** | 55% | 과제 미흡 또는 개발일지 부실 |

## 3. 대화목록 평가 기준

### 3.1 평가 영역

바이브코딩 활용 주차(2, 9-13주)의 대화목록은 다음 기준으로 평가합니다.

| 영역 | 비중 | 설명 |
|:---|:---:|:---|
| **프롬프트 적절성** | 50% | 목표·맥락·조건 명확성 |
| **대화 과정 적절성** | 50% | 개선·수정·검증 과정 |

### 3.2 등급별 기준

| 등급 | 점수 | 프롬프트 적절성 | 대화 과정 적절성 |
|:---:|:---:|:---|:---|
| **A** | 100% | 목표·맥락·조건 명확, 논리적 구조 | 결과 검증 + 문제 발견 시 수정 요청 + 개선 반복 |
| **B** | 85% | 목표·맥락 명확, 기본 구조 갖춤 | 결과 확인 + 필요시 수정 요청 |
| **C** | 70% | 기본 요청 전달, 일부 모호함 | 결과 수용, 최소한의 수정 |
| **D** | 55% | 모호한 요청, 맥락 부족 | 무비판적 수용, 개선 노력 없음 |

### 3.3 등급별 대화 예시

**A등급: 목표·맥락·조건 명확 + 결과 검증 + 개선 반복**

> **[대화 1]** "마이크로비트로 교실 온도를 측정하는 프로그램을 만들고 싶어.
> - 목표: 5초마다 온도를 측정해서 LED에 표시
> - 조건: 25도 이상이면 '더움' 아이콘, 미만이면 '추움' 아이콘
> - 출력: MakeCode 블록 코드로 설명해줘"
>
> **[대화 2]** ← 테스트 후 문제 발견, 수정 요청
>
> "테스트해보니까 온도가 너무 자주 바뀌어서 아이콘이 깜빡거려. 최근 3번 측정의 평균으로 바꿔줄 수 있어?"

**B등급: 목표·맥락 명확 + 필요시 수정 요청**

> **[대화 1]** "마이크로비트로 교실 온도 측정하는 코드 만들어줘. 5초마다 측정해서 LED에 표시하고 싶어."
>
> **[대화 2]** ← 수정 요청
>
> "숫자 대신 아이콘으로 바꿔줘. 더우면 해 모양, 추우면 눈 모양으로."

**C등급: 기본 요청 전달 + 최소한의 수정**

> **[대화 1]** "마이크로비트 온도 측정 코드 만들어줘"
>
> **[대화 2]** ← 단순 수정 요청
>
> "아이콘으로 바꿔줘"

**D등급: 모호한 요청 + 무비판적 수용**

> **[대화 1]** "온도 코드"
>
> **[대화 2]** "마이크로비트"

## 4. 피어 평가 (5%)

### 4.1 운영 방식

| 항목 | 내용 |
|:---|:---|
| **실시 시기** | 15주차 최종 발표 시 |
| **평가 대상** | 동료의 발표 |
| **평가 기준** | 발표 내용, 통찰력, 완성도 |

### 4.2 등급별 기준

| 점수 | 기준 |
|:---:|:---|
| **3점** | 우수 - 통찰력 있는 분석, 창의적 시도 |
| **2점** | 보통 - 기본 요구사항 충족 |
| **1점** | 미흡 - 내용 일부만 충족 |
| **0점** | 부족 - 발표 불참 또는 미완성 |
