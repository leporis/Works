## 슬라이드 9: 측정 가능 vs 측정 불가능

```
🔄 "측정 불가능"을 "측정 가능"으로 바꾸기

[그림 3-4] 측정 가능 vs 측정 불가능 비교 다이어그램

> 📎 파일: `그림3-4_측정가능vs불가능.svg`

| 측정 불가능 ❌         | →  | 측정 가능 ✅                        |
|----------------------|:--:|-------------------------------------|
| "덥다"               | →  | "온도가 28℃ 이상"                   |
| "시끄럽다"            | →  | "소음이 70 이상"                    |
| "쾌적하다"            | →  | "22~26℃이고 40~60%인 시간 비율"     |
| "집중한다"            | →  | "소음 50 이하인 시간 비율"           |
| "맛있다"              | ✕  | (센서 변환 불가)                    |

변환 공식: 추상적 개념 → 센서 숫자 → 기준값 설정 → 비율/빈도
```

```

[그림 3-6] Grove I2C 연결 다이어그램

> 📎 파일: `그림3-6_GroveI2C연결도_3주차용.svg`

| 센서  | 모듈명   | 측정 항목        | I2C 주소 |
|------|---------|-----------------|:--------:|
| 온습도 | SHT40   | 온도 ±0.2℃, 습도 | 0x44     |
| 공기질 | SGP30   | CO₂, TVOC       | 0x58     |
| 거리  | VL53L0X | 30~2,000 mm     | 0x29     |
| 조도  | TSL2561 | 0.1~40,000 lux  | 0x39*    |
| 기압  | BMP280  | 300~1,100 hPa   | 0x76     |

💡 I2C = 마이크로비트와 센서가 대화하는 규칙
   각 센서에 고유 "주소"가 있음
   *TSL2561 기본 주소는 0x29 (VL53L0X와 동일). 동시 사용 시 0x39로 변경 필요
```

## 슬라이드 21: 데이터 양 계산

```
🧮 예상 데이터 양 계산

공식:
  예상 데이터 개수 = (하루 수집 시간 ÷ 수집 간격) × 수집 일수

[그림 3-15] 수집 간격별 저장 용량 시뮬레이션 그래프

> 📎 파일: `그림3-15_저장용량시뮬레이션.svg`

계산 예시:
  6시간 ÷ 5분 × 5일 = 72 × 5 = 360개    ✅ 여유
  6시간 ÷ 10초 × 5일 = 2,160 × 5 = 10,800개  ⚠️ 한계

💡 권장 수집 기간: 5~10일
   요일별·시간대별 패턴을 충분히 볼 수 있는 양
```

## 슬라이드 28: 핵심 요약

```
📌 오늘의 핵심 요약

  1. 좋은 주제 = 측정 가능 + 학교 수집 + 비교 가능 + 교육적 의미

  2. "측정 불가능"을 "측정 가능"으로 바꾸는 것이 핵심

  3. 도구 3종: 검색 → 카탈로그 → 시뮬레이터

  4. 계획서 10개 항목을 꼼꼼히 작성

  5. ⭐ 센서는 현재 상태만 측정한다.
        판단과 예측은 사람의 몫이다.
```
