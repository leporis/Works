## 1. 오늘의 수업 — 프로젝트의 첫걸음

### 1.1 전체 프로젝트, 지금 어디쯤?

이번 학기 프로젝트는 크게 4단계로 이루어져 있어요. 오늘은 그중 **첫 번째 단계**에 해당합니다.

```
[1단계] 기획 (3주차) ◀── 🔵 지금 여기!
   "무엇을 측정할까? 어떤 센서를 쓸까?"
        │
        ▼
[2단계] 제작 (4주차)
   "MakeCode로 데이터수집기를 만들자"
        │
        ▼
[3단계] 수집 (5~8주차)
   "실습학교에서 실제 데이터를 모으자"
        │
        ▼
[4단계] 분석 (9~13주차)
   "바이브코딩으로 데이터를 시각화하고 해석하자"
```

**[그림 3-1] 프로젝트 진행 4단계 흐름도**

> 📎 파일: `그림3-1_프로젝트4단계흐름도.svg`

오늘 만드는 **데이터 수집 계획서**는 단순한 과제가 아니에요. 이 계획서가 4주차에 만들 데이터수집기의 설계도가 되고, 5~8주차에 실습학교에서 데이터를 모을 때의 가이드가 됩니다. 오늘 기획이 탄탄해야 나머지 10주가 순탄하게 흘러갑니다.

### 1.2 지난 주에 했던 것

2주차에는 **바이브코딩**을 처음 체험했어요. Claude에게 말로 설명하면 코드가 나온다는 경험을 했고, 센서의 종류도 간단히 살펴봤습니다.

| 2주차 활동 | 핵심 경험 |
|-----------|----------|
| Claude로 자기소개 홈페이지 제작 | "AI에게 말로 설명 → 코드 생성" 체험 |
| 센서 탐색 | 마이크로비트 내장 센서 + Grove 확장 센서 종류 확인 |
| 대화목록 PDF 작성 | AI와의 대화 기록을 문서로 정리하는 방법 학습 |

오늘은 코딩이 아니라 **생각하는 시간**이에요. "무엇을 측정할지", "왜 그것을 측정하고 싶은지", "어떤 센서를 써야 할지"를 고민합니다.

## 2. 좋은 프로젝트 주제란?

### 2.1 좋은 주제의 4가지 조건

학기 프로젝트 주제를 고르기 전에, "좋은 주제"의 기준을 먼저 알아봐요. 아무 주제나 고르면 5~8주차 현장 수집에서 막히거나, 9주차 이후에 분석할 게 없는 상황이 될 수 있어요.

**[그림 3-2] 좋은 프로젝트 주제 4가지 조건 다이어그램**

> 📎 파일: `그림3-2_좋은주제4조건.svg`

---

#### 조건 1: 📊 센서로 측정 가능한가?

센서는 **숫자를 만들어내는 장치**예요. 온도, 소음, 밝기, 거리처럼 숫자로 바꿀 수 있는 것만 측정할 수 있어요.

| 센서로 측정할 수 있는 것 | 센서로 측정할 수 없는 것 |
|------------------------|----------------------|
| 온도 (25.3℃) | 학생의 기분 |
| 소음 크기 (72) | 수업의 재미 |
| 밝기 (350 lux) | 급식의 맛 |
| 거리 (1,500 mm) | 선생님의 카리스마 |
| CO₂ 농도 (800 ppm) | 교실의 분위기 |

핵심은 **"숫자로 표현할 수 있는가?"**예요. "덥다"는 측정할 수 없지만, "온도가 28℃ 이상이다"는 센서로 바로 확인할 수 있죠.

---

#### 조건 2: 🏫 학교에서 수집 가능한가?

5~8주차에 실습학교에서 데이터를 수집해요. 따라서 **학교 안에서 마이크로비트를 설치하고 운영할 수 있는 장소**여야 해요.

| ✅ 수집 가능한 장소 | ❌ 현실적으로 어려운 장소 |
|-------------------|----------------------|
| 교실 (책상 위, 창가) | 야외 하천, 산속 |
| 복도, 계단 | 학교 밖 주택가 |
| 급식실 입구 | 지하실, 옥상 (접근 제한) |
| 도서관, 체육관 | 다른 학교 |
| 운동장 (벤치 부근) | 교통량 많은 도로변 |

> 💡 센서 설치 위치도 중요해요. 마이크로비트가 비에 젖거나 학생들이 실수로 건드리지 않는 곳을 골라야 합니다.

---

#### 조건 3: 📈 비교·분석이 가능한가?

데이터를 모았는데 항상 같은 값만 나오면 분석할 거리가 없어요. **변화가 있어야 이야기가 됩니다.**

비교할 수 있는 변인의 예시를 살펴볼까요?

| 비교 유형 | 예시 |
|----------|------|
| **시간** 비교 | 1교시 vs 5교시, 오전 vs 오후 |
| **장소** 비교 | 1층 복도 vs 3층 복도, 창가 vs 중앙 |
| **조건** 비교 | 창문 열림 vs 닫힘, 에어컨 켜짐 vs 꺼짐 |
| **요일** 비교 | 월요일 vs 금요일 |
| **날씨** 비교 | 맑은 날 vs 흐린 날 |

> 📌 비교할 수 있는 변인이 **최소 2가지** 이상 있는 주제가 좋아요.

---

#### 조건 4: 🎓 교육적 의미가 있는가?

여러분은 컴퓨터교육과 학생이에요. 졸업하면 초등학교에서 SW·AI 교육을 담당하게 됩니다. **초등학교 현장에 적용할 수 있는 주제**를 고르면, 프로젝트 경험이 훗날 수업 설계에도 도움이 돼요.

| 교육 연계 가능 교과 | 프로젝트 주제 예시 |
|-------------------|------------------|
| 과학 — 날씨와 생활 | 교실 온습도 변화, 자연광 변화 |
| 과학 — 공기 질과 건강 | 교실 CO₂ 농도 측정, 환기 효과 |
| 수학 — 통계와 측정 | 복도 통행량, 급식 대기 시간 |
| 체육 — 안전한 운동 환경 | 운동장 온습도와 자외선 |
| 생활 — 소음과 건강 | 복도·급식실 소음 패턴 |

> 💡 모든 주제가 4가지 조건을 100% 만족할 필요는 없어요. 하지만 **조건 1(측정 가능)과 조건 2(학교에서 수집 가능)**는 반드시 충족해야 합니다.

### 2.2 프로젝트 주제 예시 카드

"어떤 주제를 골라야 할지 모르겠다!" 걱정하지 마세요. 8가지 예시를 준비했어요. 이 중에서 골라도 되고, 변형해도 되고, 아예 새로운 주제를 만들어도 됩니다.

**[그림 3-3] 프로젝트 주제 예시 카드 8종 (2×4 그리드)**

> 📎 파일: `그림3-3_주제카드8종.svg`

---

#### 🌡️ 카드 1: 교실 쾌적도 측정

| 항목 | 내용 |
|------|------|
| 연구 질문 | "언제 교실이 가장 쾌적할까?" |
| 측정 항목 | 온도 (℃), 습도 (%), CO₂ 농도 (ppm) |
| 센서 | SHT40 (온습도) + SGP30 (공기질) |
| 수집 장소 | 교실 창가, 교실 중앙, 복도 쪽 |
| 비교 변인 | 시간대 (1교시 vs 5교시), 위치 (창가 vs 중앙), 환기 전/후 |
| 난이도 | ⭐⭐ |
| 교육 연계 | 과학 — 날씨와 생활 |

이 주제는 **가장 인기 있는 프로젝트 주제** 중 하나예요. 교실 온도·습도·CO₂ 데이터를 모아서 "쾌적한 학습 환경"의 조건을 찾아볼 수 있어요.

**이런 질문에 답할 수 있어요:**
- 수업 시간이 지나면서 CO₂ 농도가 얼마나 올라갈까?
- 창문을 열면 얼마나 빨리 CO₂가 줄어들까?
- 같은 시간에 창가와 중앙의 온도 차이는 얼마나 될까?

---

#### 🔊 카드 2: 복도 소음 패턴

| 항목 | 내용 |
|------|------|
| 연구 질문 | "쉬는 시간에 복도 소음이 얼마나 클까?" |
| 측정 항목 | 소리 크기 (0~255 상대값) |
| 센서 | 내장 마이크 |
| 수집 장소 | 1층 복도, 3층 복도, 계단 입구 |
| 비교 변인 | 시간대 (수업 중 vs 쉬는 시간), 층수, 요일 |
| 난이도 | ⭐ |
| 교육 연계 | 생활 — 소음과 건강 |

내장 마이크만 사용하므로 **가장 쉬운 프로젝트**예요. 확장 센서 없이 바로 시작할 수 있어요. 쉬는 시간과 수업 시간의 소음 차이, 층별 소음 차이를 비교할 수 있습니다.

> ⚠️ 내장 마이크는 절대값(dB)이 아닌 **상대값(0~255)**을 출력해요. "80dB이 넘으면 건강에 해롭다" 같은 절대적 기준과 직접 비교하기는 어렵지만, 시간대별 상대적 변화를 분석하기에는 충분합니다.

---

#### 🍽️ 카드 3: 급식실 대기 환경

| 항목 | 내용 |
|------|------|
| 연구 질문 | "급식 대기 줄이 길 때 온도는 어떻게 변할까?" |
| 측정 항목 | 거리 (mm), 온도 (℃), 습도 (%) |
| 센서 | VL53L0X (거리) + SHT40 (온습도) |
| 수집 장소 | 급식실 입구, 복도 대기 줄 |
| 비교 변인 | 학년별 급식 시간, 메뉴별 대기 길이 |
| 난이도 | ⭐⭐ |
| 교육 연계 | 수학 — 시간과 길이 측정 |

거리 센서(VL53L0X)를 급식실 입구에 설치하면, 사람이 지나갈 때 거리 값이 변해요. 이 변화를 세면 **통행량**을 추정할 수 있습니다.

---

#### ☀️ 카드 4: 운동장 활동 환경

| 항목 | 내용 |
|------|------|
| 연구 질문 | "운동하기 좋은 날씨 조건은 무엇일까?" |
| 측정 항목 | 온도 (℃), 습도 (%), 밝기 (lux) |
| 센서 | SHT40 (온습도) + TSL2561 (조도) |
| 수집 장소 | 운동장 그늘, 운동장 양지, 체육관 |
| 비교 변인 | 맑은 날 vs 흐린 날, 오전 vs 오후, 그늘 vs 양지 |
| 난이도 | ⭐⭐ |
| 교육 연계 | 체육/과학 — 안전한 운동 환경 |

> ⚠️ 운동장에 마이크로비트를 설치할 때는 **비·먼지·충격으로부터 보호**하는 방법을 미리 계획해야 해요. 밀폐 용기(지퍼백 등)에 넣되, 센서 부분은 공기 접촉이 가능하도록 약간 열어둡니다.

---

#### 📚 카드 5: 도서관 조용함

| 항목 | 내용 |
|------|------|
| 연구 질문 | "어느 자리가 가장 집중하기 좋을까?" |
| 측정 항목 | 소리 크기 (상대값), 밝기 (상대값) |
| 센서 | 내장 마이크 + 내장 빛 센서 |
| 수집 장소 | 도서관 창가, 중앙, 입구 근처 |
| 비교 변인 | 위치별, 시간대별 (점심시간 vs 방과 후) |
| 난이도 | ⭐ |
| 교육 연계 | 국어 — 독서 환경 |

내장 센서만 사용하는 **난이도 ⭐ 프로젝트**예요. 소음과 밝기를 동시에 측정하여 "조용하면서 밝기도 적당한 자리"를 찾을 수 있어요.

#### 🚻 카드 6: 화장실 사용 패턴

| 항목 | 내용 |
|------|------|
| 연구 질문 | "쉬는 시간에 화장실 사용 빈도는 어떻게 변할까?" |
| 측정 항목 | 거리 변화 (사람 감지) |
| 센서 | VL53L0X (거리) |
| 수집 장소 | 화장실 입구 (프라이버시 보호 위치) |
| 비교 변인 | 쉬는 시간 vs 수업 시간, 층별, 시간대 |
| 난이도 | ⭐⭐ |
| 교육 연계 | 수학 — 빈도와 통계 |

> 📌 **프라이버시 주의**: 거리 센서는 화장실 **입구 천장**에 설치하여 통과하는 사람 수만 세는 방식이에요. 화장실 안이 아닌, 입구의 통행량만 측정합니다.

---

#### 🌬️ 카드 7: 교실 환기 알림

| 항목 | 내용 |
|------|------|
| 연구 질문 | "언제 창문을 열어야 할까?" |
| 측정 항목 | CO₂ 농도 (ppm), 온도 (℃) |
| 센서 | SGP30 (CO₂) + SHT40 (온습도) |
| 수집 장소 | 교실 내부 (학생 책상 높이) |
| 비교 변인 | 환기 전/후, 학생 수, 수업 시간 경과 |
| 난이도 | ⭐⭐⭐ |
| 교육 연계 | 과학/보건 — 공기 질과 건강 |

난이도가 가장 높은 프로젝트예요. CO₂ 센서(SGP30)는 **워밍업 시간**(약 15초)이 필요하고, 초기값이 불안정할 수 있어요. 하지만 환기 전후의 CO₂ 변화를 보는 것은 매우 흥미로운 데이터를 만들어 줍니다.

**CO₂ 농도 기준 참고:**

| CO₂ 농도 | 상태 | 조치 |
|:--------:|------|------|
| 400~600 ppm | 🟢 쾌적 | 유지 |
| 600~1,000 ppm | 🟡 보통 | 환기 권장 |
| 1,000~2,000 ppm | 🟠 나쁨 | 즉시 환기 |
| 2,000 ppm 이상 | 🔴 매우 나쁨 | 긴급 환기 |

#### 💡 카드 8: 자연광 변화 관찰

| 항목 | 내용 |
|------|------|
| 연구 질문 | "교실 밝기가 시간대별로 어떻게 변할까?" |
| 측정 항목 | 밝기 (lux) |
| 센서 | TSL2561 (조도) |
| 수집 장소 | 교실 창가, 중앙, 복도 쪽 |
| 비교 변인 | 시간대, 날씨, 커튼/블라인드 여부 |
| 난이도 | ⭐ |
| 교육 연계 | 과학 — 빛과 그림자 |

조도 센서 1개만으로 할 수 있는 간단한 프로젝트예요. 하루 동안 교실 밝기가 어떻게 변하는지 관찰하면, 조명을 언제 켜고 끄는 것이 에너지 절약에 좋은지 분석할 수 있어요.

**밝기(lux) 기준 참고:**

| 밝기 | 환경 예시 |
|:----:|----------|
| 1~10 lux | 어두운 복도, 야간 |
| 100~300 lux | 일반 교실 (조명) |
| 300~500 lux | 밝은 교실 (조명 + 자연광) |
| 1,000~5,000 lux | 창가 직사광선 |
| 10,000 lux 이상 | 맑은 날 야외 |

### 3.2 추상적 표현을 구체적 지표로 바꾸기

많은 학생이 처음에 "교실이 쾌적한가?"같은 추상적 질문을 만들어요. 이런 질문은 센서로 직접 답할 수 없어요. 하지만 **구체적 지표로 바꾸면** 센서가 답할 수 있는 질문이 됩니다.

**변환 예시:**

| 추상적 질문 ❌ | 변환 과정 | 측정 가능한 질문 ✅ |
|---------------|----------|-------------------|
| "교실이 덥다" | 온도 센서 수치로 표현 | "교실 온도가 28℃ 이상인 시간은 하루 중 몇 %인가?" |
| "복도가 시끄럽다" | 소음 센서 수치로 표현 | "쉬는 시간 복도 소음이 수업 시간의 몇 배인가?" |
| "교실이 쾌적하다" | 복합 지표로 표현 | "온도 22~26℃이고 습도 40~60%인 시간 비율은?" |
| "급식실이 혼잡하다" | 통행량으로 표현 | "급식 시간에 1분당 거리 센서 감지 횟수는?" |
| "도서관이 집중하기 좋다" | 소음+밝기 조합 | "소음이 낮고(하위 25%) 밝기가 적절한(200~500 lux) 자리는?" |

**변환 공식:**

```
추상적 개념 → 센서로 측정 가능한 숫자 → 기준(임계값) 설정 → 비율/빈도로 질문
```

### 3.3 핵심 메시지: 센서와 AI의 한계

여기서 이번 학기에서 가장 중요한 메시지 하나를 짚고 넘어갈게요.

> 📌 **"센서는 현재 상태만 측정한다. 미래를 예측하지 않는다."**

센서가 "온도 30℃"라고 알려주면, 그건 **지금 이 순간 30℃**라는 뜻이에요. "내일도 30℃일 것이다"라는 예측은 센서가 하는 것이 아니라, **사람이 데이터를 보고 추정하는 것**입니다.

> 📌 **"AI는 집중도를 판단하지 않는다. 우리가 만든 기준을 계산할 뿐이다."**

9~13주차에 데이터를 분석하고 AI 모델을 만들 때, AI가 "이 교실은 집중하기 좋다"라고 말한다면, 그건 AI가 판단한 것이 아니에요. **우리가 "집중하기 좋다 = 소음 50 이하 + 온도 22~26℃"라는 기준을 만들고**, AI가 그 기준에 따라 계산한 것이에요.

| # | 질문 | 측정 가능? | 변환 (측정 불가능한 경우) |

| 3 | "교실 소음이 50 이하이다" | ✅ | — |

| 5 | "복도에 사람이 지나간다" | ✅ | — |

마이크로비트 V2에는 5종류의 센서가 기본으로 탑재되어 있어요. 별도의 부품 없이 바로 사용할 수 있습니다.

**[그림 3-5] 마이크로비트 V2 내장 센서 배치도**

| 센서 | 측정 항목 | 출력 범위 | 정밀도 | 특징 |
|------|----------|:--------:|--------|------|
| 온도 센서 | 온도 (℃) | -5~50℃ | ±3℃ | CPU 내부 온도 기반, 실제보다 2~3℃ 높을 수 있음 |

| 장점 | 단점 |
|------|------|

### 4.2 Grove 확장 센서 (I2C 연결)

내장 센서로 부족할 때 **Grove 확장 센서**를 추가할 수 있어요. Grove 시스템은 커넥터를 꽂기만 하면 되므로 **납땜이 필요 없고**, I2C 통신이라는 방식으로 마이크로비트와 데이터를 주고받습니다.

**I2C 통신이란?**

쉽게 말하면 마이크로비트와 센서가 대화하는 규칙이에요. 각 센서에는 고유한 "주소"가 있어서, 마이크로비트가 "0x44번 센서야, 온도 알려줘"라고 요청하면 센서가 값을 보내줍니다.

**[그림 3-6] Grove 확장 센서 I2C 연결 다이어그램**

> 📎 파일: `그림3-6_GroveI2C연결도_3주차용.svg`

**이 강좌에서 사용 가능한 Grove 센서:**

| 센서 | 모듈명 | 측정 항목 | 정밀도 | I2C 주소 | 적합한 주제 |
|------|--------|----------|--------|:--------:|-----------|
| 온습도 | SHT40 | 온도 ±0.2℃, 습도 ±1.8% | 매우 높음 | 0x44 | 교실 쾌적도, 운동장 환경 |
| 공기질 | SGP30 | CO₂ 400~60,000 ppm, TVOC | 높음 | 0x58 | 교실 환기, 공기 질 |
| 거리 | VL53L0X | 30~2,000 mm | 높음 | 0x29 | 통행량, 대기 줄 |
| 조도 | TSL2561 | 0.1~40,000 lux | 높음 | 0x29 또는 0x39 | 자연광 변화, 밝기 |
| 기압 | BMP280 | 300~1,100 hPa, 온도 | 높음 | 0x76 또는 0x77 | 날씨 변화 |

**Grove 연결 구조:**

```
마이크로비트 V2
    │
    └── Grove 쉴드 (어댑터 보드)
            │
            ├── I2C 포트 1 → 센서 A
            ├── I2C 포트 2 → 센서 B
            ├── I2C 포트 3 → 센서 C
            └── I2C 포트 4 → 센서 D
```

> 📌 **I2C 포트에는 최대 4개 센서를 동시에 연결**할 수 있어요. 단, 같은 I2C 주소를 가진 센서 2개를 동시에 연결할 수는 없어요. 예를 들어 VL53L0X(0x29)와 TSL2561(기본 0x29)은 주소가 같아서 동시 사용 시 TSL2561의 주소를 0x39로 변경해야 합니다.

```
              └─ 충돌 있음 → 주소 변경 또는 다른 센서로 대체
```

## 5. 데이터 수집 계획서 작성

### 5.1 계획서란?

데이터 수집 계획서는 **프로젝트의 설계도**예요. "무엇을, 어디서, 얼마나 자주, 얼마나 오래 수집할 것인가"를 미리 정해둬요. 계획서가 탄탄하면 4주차 수집기 제작과 5~8주차 현장 수집이 순탄합니다.

**[그림 3-10] 데이터 수집 계획서 양식 (작성 예시 포함)**

> 📎 파일: `그림3-10_수집계획서양식.svg`

### 5.2 계획서 10개 항목 상세 가이드

각 항목의 의미와 작성 방법을 알아볼게요.

#### 항목 1: 프로젝트 제목

프로젝트 전체를 한 문장으로 요약하는 제목이에요.

| ❌ 나쁜 예 | ✅ 좋은 예 |
|-----------|-----------|
| "교실 측정" | "교실 쾌적도 시간대별 변화 측정" |
| "데이터 수집" | "1~3층 복도 소음 패턴 비교 분석" |
| "센서 프로젝트" | "도서관 자리별 집중 환경 비교" |

> 💡 좋은 제목에는 **장소 + 측정 대상 + 비교 방법**이 포함되어 있어요.

---

#### 항목 2: 연구 질문

"~할까?", "~일까?" 형태의 질문이에요. **센서로 답할 수 있는 구체적인 질문**이어야 해요.

| ❌ 나쁜 연구 질문 | ✅ 좋은 연구 질문 |
|----------------|-----------------|
| "교실이 쾌적한가?" | "1교시와 5교시 중 온도·습도가 쾌적 범위(22~26℃, 40~60%)에 있는 시간은 언제가 더 긴가?" |
| "복도가 시끄러운가?" | "쉬는 시간 복도 소음은 수업 시간 대비 평균 몇 배인가?" |
| "밝기가 변하는가?" | "하루 중 교실 창가 밝기가 가장 높은 시간대는 언제인가?" |

---

#### 항목 3~4: 측정 항목과 사용 센서

3번 항목에는 **수집할 데이터의 종류**를, 4번 항목에는 **사용할 센서**를 적어요.

**작성 예시:**

| 측정 항목 | 단위 | 센서 | 내장/확장 |
|----------|------|------|:--------:|
| 온도 | ℃ | SHT40 | 확장 |
| 습도 | % | SHT40 | 확장 |
| CO₂ 농도 | ppm | SGP30 | 확장 |

---

#### 항목 5: 수집 장소

학교 내 구체적인 위치를 적어요. **5주차에 실습학교에 가서 확정**하면 되지만, 지금은 대략적인 위치를 적어둡니다.

| 수준 | 예시 |
|:----:|------|
| 너무 모호 | "교실" |
| 적절 | "3-1 교실 창가 책상 위" |
| 더 좋음 | "3-1 교실 창가 2번째 줄 책상 위 (높이 약 70cm)" |

> 💡 **높이**도 중요해요. 같은 교실에서도 바닥과 천장의 온도 차이가 2~3℃ 날 수 있어요.

#### 항목 6: 수집 간격

"몇 초/분마다 데이터를 기록할 것인가"를 결정해요. 측정 대상의 변화 속도에 따라 적절한 간격이 달라집니다.

**[그림 3-11] 수집 간격별 데이터 양 비교 인포그래픽**

> 📎 파일: `그림3-11_수집간격별데이터양.svg`

**수집 간격 추천표:**

| 측정 대상 | 변화 속도 | 권장 간격 | 하루 6시간 수집 시 데이터 개수 |
|----------|:--------:|:--------:|:------------------------:|
| 온도·습도 | 느림 | 5분 | 72개 |
| CO₂ 농도 | 느림~보통 | 5분 | 72개 |
| 밝기 (자연광) | 보통 | 1~5분 | 72~360개 |
| 소음 | 빠름 | 10초~1분 | 360~2,160개 |
| 거리 (통행량) | 매우 빠름 | 1초 | 21,600개 |

**데이터 양 계산 공식:**

```
예상 데이터 개수 = (하루 수집 시간 ÷ 수집 간격) × 수집 일수

예시: 6시간 ÷ 5분 × 5일 = 72 × 5 = 360개
예시: 6시간 ÷ 10초 × 5일 = 2,160 × 5 = 10,800개
```

> ⚠️ **저장 용량 주의**: 마이크로비트 Data Logger의 저장 공간은 약 64KB예요. 데이터 1개가 약 6~8바이트라면 약 **8,000~10,000개**까지 저장할 수 있어요. 1초 간격으로 6시간 수집하면 하루에 21,600개로 용량을 초과해요! 이 경우 수집 간격을 늘리거나, 매일 데이터를 다운로드해야 합니다.

**[그림 3-15] 수집 간격에 따른 저장 용량 시뮬레이션**

> 📎 파일: `그림3-15_저장용량시뮬레이션.svg`

#### 항목 7: 수집 기간

5~8주차(4주간) 중 실제로 몇 일 동안 수집할 것인가를 정해요.

| 수집 기간 | 장점 | 단점 |
|:--------:|------|------|
| 1~2일 | 빠르게 완료 | 데이터가 적어 패턴 파악 어려움 |
| 3~5일 | 적절한 양 | 요일별 비교 가능 |
| 1~2주 | 풍부한 데이터 | 배터리·관리 부담 |
| 3~4주 | 매우 풍부 | 기기 고장·분실 위험 |

> 💡 **권장: 5~10일**. 이 정도면 요일별·시간대별 패턴을 충분히 볼 수 있고, 관리 부담도 적당해요.

#### 항목 8: 예상 데이터 양

6번(수집 간격)과 7번(수집 기간)을 곱하여 예상 데이터 양을 계산해요.

**계산 예시:**

| 수집 간격 | 하루 시간 | 수집 일수 | 예상 데이터 양 |
|:--------:|:--------:|:--------:|:------------:|
| 5분 | 6시간 | 5일 | 360개 |
| 1분 | 6시간 | 5일 | 1,800개 |
| 10초 | 6시간 | 3일 | 6,480개 |
| 1초 | 1시간 | 5일 | 18,000개 |

#### 항목 9: 예상 결과

데이터를 모으기 전에 **어떤 패턴을 예상하는지** 적어요. 예상이 맞든 틀리든 상관없어요. 오히려 예상과 다른 결과가 나오면 더 흥미로운 분석이 됩니다.

**작성 예시:**

| 주제 | 예상 결과 |
|------|----------|
| 교실 쾌적도 | "오후(5교시)가 오전(1교시)보다 온도가 높고 CO₂도 높을 것이다" |
| 복도 소음 | "쉬는 시간(10분)에 소음이 수업 시간의 3배 이상일 것이다" |
| 자연광 변화 | "남향 교실 창가 밝기는 오전 10시~오후 2시에 가장 높을 것이다" |

#### 항목 10: 주의 사항

현장 수집에서 발생할 수 있는 문제를 미리 생각해 둬요.

**자주 발생하는 문제와 대비책:**
