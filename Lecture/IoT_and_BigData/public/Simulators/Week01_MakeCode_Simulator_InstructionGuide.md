# 🎓 MakeCode 시뮬레이터 — LED 이름 표시 활용가이드

**문서 버전: v1.0.0.0**
**시뮬레이터 버전: v1.0.2.4**

---

## 1. 개요

### 1.1 이 문서의 목적

본 활용가이드는 MakeCode 시뮬레이터를 대학 수업에서 효과적으로 활용하기 위한 교수·학습 방법을 안내한다. 「사물인터넷과 빅데이터」 과목 1주차 강의자료로 설계되었으며, 프로그래밍 경험이 없는 예비 초등교사(4학년)를 대상으로 한다.

### 1.2 학습 목표

| 영역 | 학습 목표 |
|------|----------|
| **지식** | 블록 코딩의 기본 구조(이벤트, 순차 실행)를 설명할 수 있다 |
| **기능** | 시뮬레이터에서 블록을 배치하여 LED에 자기 이름을 표시할 수 있다 |
| **태도** | 코딩에 대한 두려움을 줄이고 탐구적 자세를 갖는다 |

### 1.3 관련 교육과정

| 항목 | 내용 |
|------|------|
| 대상 과목 | 사물인터넷과 빅데이터 (4학년) |
| 주차 | 1주차 |
| 주제 | LED에 이름 표시하기 |
| 수강 인원 | 21명 |
| 선수 학습 | 없음 (프로그래밍 비전공 대상) |

## 2. 수업 시나리오

### 2.1 기본 수업 흐름 (75분)

| 단계 | 시간 | 활동 | 교수자 역할 | 시뮬레이터 기능 |
|:----:|:----:|------|-----------|----------------|
| **도입** | 10분 | micro:bit 소개 및 동기 유발 | 실물 micro:bit 시연 | — |
| **시연** | 10분 | 교수자가 시뮬레이터로 "Hello!" 표시 | 화면 공유, 단계별 설명 | 전체 기능 시연 |
| **탐색** | 15분 | 학생 자율 탐색 (블록 드래그 체험) | 순회 지도, 개별 질문 대응 | 블록 팔레트, 드래그 앤 드롭 |
| **실습** | 25분 | 자기 이름 LED에 표시하기 | 활동지 배부, 관찰 포인트 안내 | 문자열 보여주기, 실행 |
| **심화** | 10분 | 인사 메시지 만들기 (복합 블록) | 도전 과제 제시 | 아이콘, 일시정지 블록 |
| **정리** | 5분 | 학습 내용 요약, 질의응답 | 핵심 개념 정리 | — |

### 2.2 도입 단계 상세 (10분)

**목표**: micro:bit가 무엇인지 알고, LED로 무엇을 할 수 있는지 관심을 갖는다.

| 시간 | 교수자 활동 | 학생 활동 |
|:----:|-----------|----------|
| 3분 | 실물 micro:bit 보여주기 + "이 작은 컴퓨터로 무엇을 할 수 있을까?" | 관찰, 추측 발표 |
| 4분 | micro:bit LED에 이름 표시 시연 (실물 또는 영상) | 관찰, "오!" 반응 |
| 3분 | "오늘 우리도 LED에 이름을 표시해 볼 거예요. 진짜 micro:bit 대신 시뮬레이터를 사용합니다." | 시뮬레이터 접속 |

> 💡 **팁**: 실물 micro:bit가 없으면 MakeCode 공식 시뮬레이터 영상으로 대체 가능하다.

### 2.3 시연 단계 상세 (10분)

**목표**: 시뮬레이터의 기본 조작법을 관찰한다.

교수자가 화면을 공유하며 다음을 순서대로 시연한다.

| 순서 | 시연 내용 | 강조 포인트 |
|:----:|----------|------------|
| 1 | 팔레트에서 "문자열 보여주기" 블록 드래그 | "왼쪽에서 블록을 꺼내요" |
| 2 | "시작하면 실행" 영역에 드롭 | "보라색 영역 안에 놓아야 해요" |
| 3 | `Hello!`를 `Hi!`로 수정 | "글씨를 클릭하면 바꿀 수 있어요" |
| 4 | ▶️ 실행 버튼 클릭 | "실행 버튼을 누르면 LED가 켜져요" |
| 5 | LED에 "Hi!" 표시되는 것 확인 | "한 글자씩 나타나는 거 보이죠?" |
| 6 | ⏹️ 정지 → 🔄 리셋 | "멈추고 싶으면 정지, 처음부터 하려면 리셋" |

### 2.4 탐색 단계 상세 (15분)

**목표**: 블록 종류와 드래그 앤 드롭을 자유롭게 체험한다.

| 활동 | 안내 멘트 |
|------|----------|
| 자유 탐색 | "5가지 블록이 있어요. 다 한 번씩 써보세요!" |
| 블록 조합 | "블록을 여러 개 넣으면 어떻게 될까요?" |
| 삭제 체험 | "× 버튼으로 블록을 지울 수 있어요" |
| 두 영역 비교 | "'시작하면 실행'과 '무한반복'에 각각 넣어보세요. 뭐가 다를까요?" |

> ⚠️ **순회 지도 시 확인 사항**: 블록을 이벤트 영역 밖에 놓으려는 학생 → 점선 안에 놓도록 안내

### 2.5 실습 단계 상세 (25분)

**목표**: 자기 이름을 LED에 표시하는 프로그램을 완성한다.

#### 활동 1: 내 이름 표시하기 (15분)

| 단계 | 안내 | 예상 시간 |
|:----:|------|:--------:|
| 1 | "문자열 보여주기" 블록을 "시작하면 실행"에 놓기 | 2분 |
| 2 | `Hello!`를 자기 이름(영어 또는 한글 자모)으로 변경 | 3분 |
| 3 | ▶️ 실행하여 LED 확인 | 1분 |
| 4 | 일시정지 블록 추가 (이름 표시 후 잠깐 대기) | 3분 |
| 5 | 아이콘 보여주기(❤️) 추가 | 3분 |
| 6 | 전체 실행하여 결과 확인 | 3분 |

#### 활동 2: 무한 이름 스크롤 (10분)

| 단계 | 안내 | 예상 시간 |
|:----:|------|:--------:|
| 1 | 리셋 후 "무한반복 실행"에 이름 블록 배치 | 3분 |
| 2 | 실행하여 반복 확인 | 2분 |
| 3 | "시작하면 실행" vs "무한반복 실행" 차이 토론 | 5분 |

### 2.6 심화 단계 상세 (10분)

**목표**: 블록 조합의 창의적 활용을 시도한다.

| 도전 과제 | 힌트 |
|----------|------|
| 🌟 이름 + 하트 반복 | 무한반복에 문자열 → 일시정지 → 아이콘 → 일시정지 |
| 🌟🌟 카운트다운 (3, 2, 1, GO!) | 시작하면 실행에 숫자 3 → 일시정지 → 숫자 2 → ... → 문자열 GO! |
| 🌟🌟🌟 LED 깜빡이기 | 무한반복에 아이콘 → 일시정지 → 화면 지우기 → 일시정지 |

## 5. 교수자 팁

### 5.1 흔한 학생 실수와 대처

| 실수 | 증상 | 대처 |
|------|------|------|
| 블록을 영역 밖에 드롭 | 블록이 사라짐 | "점선 안에 놓아야 해요" 안내 |
| 실행 버튼을 안 누름 | LED 변화 없음 | "오른쪽 ▶️ 버튼 눌러보세요" |
| 한글 완성형 입력 | LED에 아무것도 안 나옴 | "영어나 한글 자모(ㄱ, ㅏ)를 써보세요" |
| 일시정지 값 0ms | 글자가 너무 빠름 | "1000 이상으로 설정해 보세요" |
| 무한반복 정지 못함 | 패닉 | "빨간 정지 버튼을 눌러보세요" 침착히 안내 |

### 5.2 수업 운영 권장 사항

| 항목 | 권장 |
|------|------|
| 학습 환경 | 개인 노트북/PC + 크롬 브라우저 권장 |
| 파일 배포 | LMS에 HTML 파일 업로드 → 학생 다운로드 후 더블클릭 실행 |
| 화면 공유 | 교수자 시연 시 프로젝터/줌 화면 공유 |
| 모둠 활동 | 2인 1조 (1인 조작, 1인 관찰 후 교대) 권장 |
| 시간 여유 | 탐색 15분은 최소 10분 확보 (첫 드래그 성공 경험이 중요) |

### 5.3 평가 연계

| 평가 요소 | 확인 방법 |
|----------|----------|
| 블록 배치 능력 | 활동지 1 완성 여부 |
| 이벤트 이해 | 활동지 2의 결론 문장 |
| 순차 실행 이해 | "블록 순서를 바꾸면?" 질문 응답 |
| 창의적 활용 | 심화 도전 과제 도전 여부 |

## 6. 초등학교 수업 적용 아이디어
