**시뮬레이터 버전: v1.0.2.4**

## 📖 이 문서 사용법

> **이 퀵가이드는 MakeCode 시뮬레이터와 함께 보는 내비게이션입니다.**

| 상황 | 참조 섹션 |
|------|----------|
| 처음 시작 | ⚡ 1분 초속성 실습 |
| 블록 종류가 궁금 | 🚀 빠른 시작 |
| 버튼 위치 모름 | 🖥️ 화면 구성 |
| LED가 안 켜짐 | 🆘 도와주세요! |
| 용어가 낯설음 | 🔑 핵심 용어 |

📚 **관련 문서 읽기 순서**: 퀵가이드(지금) → 사용설명서(전체 기능) → 활용가이드(수업 적용) → 이론해설서(원리 심화)

## ⚡ 1분 초속성 실습

| 단계 | 행동 | 확인 |
|:---:|------|:----:|
| 1 | 왼쪽 팔레트에서 **문자열 보여주기** 블록을 드래그 | 손가락 모양 커서로 바뀌었나요? |
| 2 | **🚀 시작하면 실행** 영역에 드롭 | 블록이 보라색 영역 안에 들어갔나요? |
| 3 | 블록의 `Hello!`를 **자기 이름**으로 수정 | 텍스트가 바뀌었나요? |
| 4 | 오른쪽 **▶️ 실행** 버튼 클릭 | LED에 이름이 한 글자씩 나타나면 🎉 |

> 💡 **팁**: 실행 중인 블록은 **노란색 테두리**로 강조돼요!

---

## 🚀 빠른 시작 (3단계)

### 1️⃣ 블록 선택하기

왼쪽 **블록 팔레트**에서 원하는 블록을 찾으세요.

| 카테고리 | 블록 | 어떤 블록인가요? |
|:--------:|------|-----------------|
| ■ **기본** | 문자열 보여주기 | LED에 글자를 한 자씩 스크롤해요 |
| ■ **기본** | 숫자 보여주기 | LED에 숫자를 표시해요 |
| ■ **기본** | 아이콘 보여주기 ❤️ | LED에 하트 모양을 띄워요 |
| ■ **기본** | 일시정지 | 잠시 멈추고 기다려요 (ms 단위) |
| 💡 **LED** | 화면 지우기 | 모든 LED를 꺼요 |

### 2️⃣ 블록 배치하기

블록을 **드래그**해서 가운데 코딩 영역에 놓으세요.

| 영역 | 언제 실행? | 어디에 쓰나요? |
|------|-----------|--------------|
| 🚀 **시작하면 실행** | 프로그램 시작 시 **1번만** | 이름 표시, 인사 메시지 |
| 🔄 **무한반복 실행** | 끝없이 **계속 반복** | 이름 무한 스크롤, 깜빡이기 |

### 3️⃣ 실행하고 관찰하기

오른쪽 시뮬레이터 패널의 버튼을 사용하세요.

| 버튼 | 기능 |
|:----:|------|
| **▶️ 실행** | 블록 프로그램 실행 시작 |
| **⏹️ 정지** | 실행 즉시 멈춤 |
| **🔄 리셋** | 블록 모두 제거 + LED 초기화 |

## 🎯 실습 활동

### 활동 1: 내 이름 LED에 표시하기

| 단계 | 행동 | 확인 |
|:---:|------|:----:|
| 1 | **문자열 보여주기** → 🚀 시작하면 실행 | 배치 완료? |
| 2 | `Hello!` → 자기 이름으로 변경 | 이름 입력? |
| 3 | **▶️ 실행** 클릭 | LED에 이름 표시? 🎉 |

### 활동 2: 인사 메시지 만들기

| 단계 | 행동 | 확인 |
|:---:|------|:----:|
| 1 | 문자열 보여주기 `Hi!` → 🚀 시작하면 실행 | 배치 완료? |
| 2 | **일시정지** `1000` ms 추가 | 두 번째 블록? |
| 3 | 아이콘 보여주기 ❤️ 추가 | 세 번째 블록? |
| 4 | **▶️ 실행** → "Hi!" → 1초 대기 → ❤️ 순서 확인 | 순서대로? 🎉 |

### 활동 3: 무한 이름 스크롤

| 단계 | 행동 | 확인 |
|:---:|------|:----:|
| 1 | 문자열 보여주기 (이름) → 🔄 **무한반복 실행** | 배치 완료? |
| 2 | **▶️ 실행** | 이름이 계속 반복? |
| 3 | **⏹️ 정지** → 멈추는지 확인 | 즉시 정지? 🎉 |

## 🆘 도와주세요!

| 증상 | 원인 | 해결책 |
|------|------|--------|
| LED가 전혀 안 켜져요 | 블록을 배치하지 않음 | 팔레트에서 블록을 드래그하여 코딩 영역에 놓으세요 |
| 블록을 놓을 수 없어요 | 이벤트 영역 밖에 드롭 | 보라색 🚀 또는 🔄 영역 **안**에 놓으세요 |
| 실행해도 변화가 없어요 | 블록이 이벤트 밖에 있음 | 블록이 점선 영역 안에 있는지 확인하세요 |
| 글자가 너무 빨리 지나가요 | 일시정지 미사용 | **일시정지** 블록(2000ms 이상)을 블록 사이에 추가하세요 |
| LED에 이상한 모양이 나와요 | 지원하지 않는 문자 | 영어(A~Z), 숫자(0~9), 한글 자모(ㄱ~ㅎ, ㅏ~ㅣ)를 사용하세요 |
| 무한반복이 안 멈춰요 | 정상 동작입니다! | **⏹️ 정지** 버튼을 클릭하세요 |
| 블록 팔레트가 안 보여요 | 좁은 화면에서 스크롤 필요 | 위로 스크롤하면 팔레트가 보여요 |

## 🔑 핵심 용어

| 용어 | 학술 용어 | 의미 | 쉬운 설명 |
|------|----------|------|----------|
| 블록 | Block | 하나의 명령어 단위 | 레고 조각처럼 끼우는 코딩 명령 |
| 이벤트 | Event | 특정 상황 발생 시 실행 | "~하면 ~해라"의 '~하면' 부분 |
| 순차 실행 | Sequential Execution | 위에서 아래로 순서대로 실행 | 블록을 놓은 순서대로 하나씩 실행 |
| 반복 | Loop | 같은 동작을 여러 번 수행 | "무한반복"은 멈출 때까지 계속 |
| LED 매트릭스 | LED Matrix | 5×5 격자 형태의 LED 배열 | 25개 작은 불로 이루어진 화면 |
| 일시정지 | Pause/Delay | 지정 시간 동안 대기 | ms(밀리초) 단위: 1000ms = 1초 |
| 드래그 앤 드롭 | Drag and Drop | 끌어서 놓기 조작 | 마우스로 잡아서 원하는 곳에 놓기 |

## 💭 생각해보기

> **블록의 순서를 바꾸면 결과가 달라질까요?**
>
> "문자열 보여주기" → "일시정지" → "아이콘 보여주기" 순서와
> "아이콘 보여주기" → "일시정지" → "문자열 보여주기" 순서의 차이를 비교해 보세요.
> 프로그래밍에서 **순서**가 왜 중요한지 생각해 봅시다.

> **"시작하면 실행"과 "무한반복 실행"은 언제 각각 사용해야 할까요?**
>
> 인사 메시지를 한 번만 보여주고 싶을 때와, 이름을 계속 스크롤하고 싶을 때를 떠올려 보세요.
