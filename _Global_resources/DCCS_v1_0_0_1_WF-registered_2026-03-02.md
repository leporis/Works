## 1. 점검 목적과 범위

### 1.1 목적

문서가 많아질수록 한 문서의 버전이 올라갈 때 이를 참조하는 다른 문서의 참고자료 표가 자동으로 갱신되지 않는다. 이 체계는 그 누락을 정기적으로 찾아 수정함으로써 **"어떤 문서를 펼쳐도 참조 버전이 현재와 일치한다"** 는 상태를 유지하는 것이 목표다.

### 1.2 범위

프로젝트 내 모든 관리 문서 — Policy, Procedure, Standard (L1~L3) — 의 **참고자료 표**에 기재된 타 문서 버전과, SKILL.md의 **참고자료 표 및 문서 체계 맵**.

## 3. 점검 항목

문서마다 아래 5개 항목을 순서대로 확인한다.

| # | 점검 항목 | 확인 방법 | 기준 |
|:-:|----------|----------|------|
| 1 | **마스터 버전표 vs 각 문서 표지 버전** | §2 표 ↔ 각 문서 "문서 버전:" 행 | 완전 일치 |
| 2 | **참고자료 표의 타 문서 버전** | 각 문서 참고자료 표 ↔ §2 마스터 버전표 | 최신 버전 일치 |
| 3 | **SKILL.md 참고자료 표 버전** | SKILL.md 참고자료 표 ↔ §2 마스터 버전표 | 완전 일치 |
| 4 | **SKILL.md 문서 체계 맵 버전** | SKILL.md §6 트리 ↔ §2 마스터 버전표 | 완전 일치 |
| 5 | **생성/수정 이력 표의 최신 행 버전** | 이력 표 마지막 행 버전 ↔ 문서 표지 버전 | 완전 일치 |

## 4. 점검 주기와 트리거

### 4.1 정기 점검 (분기 1회)

| 시점 | 담당 | 방법 |
|------|------|------|
| 매 분기 첫째 주 | Changmo Yang & Claude AI | §5 점검 절차 전체 수행 |

### 4.2 이벤트 트리거 (즉시 점검)

아래 상황이 발생하면 즉시 §5를 수행한다.

| 트리거 | 이유 |
|--------|------|
| 어떤 문서든 **마이너 이상** 버전 업 발생 | 참조 버전 불일치 가능성 높음 |
| 새 문서 추가 | 마스터 버전표·SKILL.md 즉시 갱신 필요 |
| 문서 폐기·통합 | 참고자료 표에서 해당 문서 참조 제거 필요 |
| 정합성 오류 발견 시 | 즉시 §7 처리 절차 수행 |

## 5. 점검 절차

### Step 1: 마스터 버전표 최신화

모든 문서의 표지에서 현재 버전을 읽어 §2 표를 갱신한다.

```
확인 명령 (grep 예시):
grep "문서 버전:" [문서명].md
```

### Step 2: 각 문서 참고자료 표 대조

각 문서의 참고자료 표를 열고, 타 문서 버전을 §2와 대조한다.
불일치 발견 시 →  §6 점검 기록에 기재 → §7 처리 절차 수행.

### Step 3: SKILL.md 이중 점검

SKILL.md의 두 곳을 각각 확인한다.
- `§참고자료` 표의 버전 열
- `§6 문서 체계 맵`의 버전 표기

### Step 4: 이력 표 일관성 확인

각 문서의 생성/수정 이력 표에서 마지막 행의 버전이 표지 버전과 일치하는지 확인한다.

### Step 5: 점검 기록 작성

§6 양식에 따라 점검 결과를 기록하고 이 문서의 이력 표에 추가한다.

## 7. 불일치 발견 시 처리 절차

### 7.1 경미한 불일치 (참고자료 버전 오기 — 패치)

```
1. §2 마스터 버전표에서 정확한 버전 확인
2. 해당 문서의 참고자료 표 수정
3. 이력 표에 패치 기록 추가 (버전 W+1)
4. §6 점검 기록에 조치 내용 기재
```

### 7.2 심각한 불일치 (내용 참조 오류 — 리비전 이상)

```
1. 불일치 영향 범위 파악 (어느 섹션이 구 버전 기준으로 쓰였는가)
2. 최신 버전 문서 내용 기준으로 해당 섹션 재검토 및 수정
3. 변경 수준 판단 (리비전/마이너) 후 버전 증가
4. SKILL.md 트리거 맵 및 참고자료 표 동기화
5. §6 점검 기록에 상세 조치 기재
```

### 7.3 문서 추가 시

```
1. §2 마스터 버전표에 새 행 추가
2. SKILL.md 참고자료 표 + §6 문서 체계 맵 추가
3. 관련 문서(상위 표준 등)의 참고자료 표에 신규 문서 추가
4. 다음 분기 점검 시 신규 문서 포함하여 점검
```
